# PRD: 修复 selectBestAccount 忽略 extra_used 账号

## 问题

`selectBestAccount()` 只过滤 `five_hour_pct >= 80%` 的账号，但不过滤 `extra_used=True` 的账号。
account1 的 `extra_used=True`（超额用量已耗尽），调用 sonnet 立即失败（exit code 1，2秒内）。
由于 account1 使用率显示 4%，被误选为最优账号，导致所有 mouth/thalamus/cortex LLM 调用失败。

## 修复

在 `.filter()` 里加一条：`&& !a.extraUsed`，跳过 extra_used=True 的账号。
