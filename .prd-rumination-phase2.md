# 反刍回路 Phase 2 — 完善与可视化

## 背景

Phase 1（PR #24）已合并：反刍核心流程可用（空闲时消化 learnings → LLM 洞察 → memory_stream）。
但存在 5 个未完成项需要补齐。

## 目标

补齐反刍系统的可操作性和可观测性：
1. 手动触发反刍（不再只能等 tick 空闲）
2. Actionable 洞察自动创建可执行 task
3. 知识归档机制（防 learnings 表无限增长）
4. Dashboard 知识消化页面（可视化状态）
5. 完整的 API 端点（状态查询 + 列表）

## 成功标准

- `POST /api/brain/ruminate` 能手动触发消化
- `GET /api/brain/rumination/status` 返回完整状态
- `GET /api/brain/learnings` 支持分页和筛选
- LLM 输出含 `[ACTION:]` 时自动创建 task
- 90 天已消化知识自动归档
- Dashboard `/knowledge/digestion` 页面可用
- 所有测试通过，facts-check + version-sync 通过
