# Task Card: .dev-mode.{branch} 状态层持久化

## 需求（What & Why）
**功能描述**: 修改 .gitignore 允许 .dev-mode.{branch} 被 git 追踪，并在关键写入点（spec/code/sprint-contract）写完 .dev-mode.{branch} 后立即 git add + commit，让 git worktree add 后能自动恢复 step 状态。
**背景**: devloop-check.sh 所有条件都读 .dev-mode，但 .dev-mode.* 被 gitignore 忽略，上下文压缩后文件消失，devloop-check 失明认为 step_1 pending，agent 从头开始。这是"状态层"的根本漏洞。
**不做什么**: 不改变 .dev-mode 的格式和内容；不修改 devloop-check.sh 的读取逻辑；不影响 .dev-mode（无后缀主 symlink）的行为。

## 成功标准
> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [ARTIFACT] .gitignore 中 `.dev-mode.*` 的忽略规则被移除（只保留忽略 `.dev-mode` 无后缀）
2. [BEHAVIOR] 执行完 Stage 1 Spec 后，`.dev-mode.{branch}` 已被 git commit 进功能分支，可被 git worktree add 自动恢复
3. [BEHAVIOR] sprint-contract-loop.sh 在收敛（exit 0）时同时 commit `.dev-mode.{branch}`
4. [GATE] CI 全部通过（engine-ci.yml）
