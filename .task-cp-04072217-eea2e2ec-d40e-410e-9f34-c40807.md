# Task Card: 修复 context.js KR 进度恒为 0% 的问题

## 需求（What & Why）
**功能描述**: `/api/brain/context` 接口返回的 Objective 进度始终为 0%，而 `/api/brain/okr/current` 显示真实进度。
**背景**: `context.js` 用 `current_value / target_value` 计算进度，但 kr-verifier 只更新 `key_results.progress` 列，`current_value` 从未被更新，导致 context 接口永远返回 0。
**不做什么**: 不修改 kr-verifier、okr/current、数据库 schema。

## 根因
- `kr-verifier.js` → 更新 `key_results.progress`（整数百分比）
- `context.js` → 读 `current_value / target_value`（始终 0.00）
- 两者脱钩

## 成功标准
> [BEHAVIOR] GET /api/brain/context 返回的 OKR progress_pct 与 /api/brain/okr/current 一致（非全零）

## 验收条件（DoD）

- [x] [BEHAVIOR] 新 SQL 用 AVG(COALESCE(kr.progress,0)) 计算目标进度为非零值
  Test: manual:psql -h localhost -U cecelia -d cecelia -t -c "SELECT COALESCE(ROUND(AVG(COALESCE(kr.progress,0))),0) as pct FROM objectives o LEFT JOIN key_results kr ON kr.objective_id=o.id AND kr.status<>'archived' WHERE o.status<>'archived' GROUP BY o.id" | grep -v '^ 0$' | grep -q '[0-9]' 

- [x] [ARTIFACT] context.js 使用 AVG(kr.progress) 而非 current_value/target_value
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/routes/context.js','utf8');if(!c.includes('AVG(COALESCE(kr.progress'))process.exit(1)"

## 实现方案
**要改的文件**: `packages/brain/src/routes/context.js`
**修改点**: 将 SQL `AVG(kr.current_value::numeric / NULLIF(kr.target_value::numeric, 0) * 100)` 改为 `AVG(COALESCE(kr.progress, 0))`
**不改什么**: 不改 kr-verifier、okr-hierarchy、数据库 schema
