## PRD - Brain dispatch 故障：auth 错误误触发熔断器

**需求来源**: Brain 自动派发 - P0++ CRITICAL，ZenithJoy 派发系统 96h 冻结诊断与修复

**根因诊断**:
- `account2` OAuth token 过期，`heartbeat zenithjoy` 任务使用 account2 运行时报错：
  `"Failed to authenticate. API Error: 401 OAuth token has expired"`
- 该错误消息未匹配 `AUTH_PATTERNS`（`authentication\s+failed` 不匹配 `Failed to authenticate`）
- 被分类为 `TASK_ERROR`（默认）而非 `auth`
- `routes.js` 只旁路 `billing_cap` 和 `rate_limit/network`，不旁路 `auth`
- 导致 `cbFailure('cecelia-run')` 被调用，3次后 circuit breaker OPEN，所有 dispatch 阻塞 30min
- 上一轮 95% 的 dispatch failure 都是 `circuit_breaker_open`

**功能描述**:
1. `quarantine.js`：AUTH_PATTERNS 添加 "Failed to authenticate" 和 "OAuth token has expired" 匹配规则
2. `routes.js`：execution-callback 将 `auth` 类失败加入熔断器旁路（auth failure = 凭据问题，不代表 cecelia-run 系统故障）

**涉及文件**:
- `packages/brain/src/quarantine.js`（AUTH_PATTERNS 新增两个规则）
- `packages/brain/src/routes.js`（cbFailure 旁路条件扩展）

## 成功标准
- auth 类失败（OAuth token 过期、认证失败）不再触发 cbFailure('cecelia-run')
- classifyFailure("Failed to authenticate...OAuth token has expired") 返回 class='auth'
- circuit breaker 不因单账户 OAuth 过期而 OPEN
- 单元测试覆盖新增 AUTH_PATTERNS 和旁路逻辑

**非目标**: 不自动刷新 OAuth token（需用户手动操作），不修改隔离区内已有任务状态
