# PRD - refactor(startup): 统一 startup-recovery.js 与 executor.js 孤儿处理逻辑

## 背景
两条并行孤儿处理路径存在优先级冲突：
- 路径A（startup-recovery.js）无条件将 in_progress 重置为 queued，先于路径B运行
- 路径B（executor.js::syncOrphanTasksOnStartup）智能逻辑实际等到的是空集

## 目标
删除 startup-recovery.js 中的 DB 孤儿恢复段，统一由 syncOrphanTasksOnStartup 负责。

## 变更范围
1. startup-recovery.js：删除 DB orphan task recovery 段，移除 pool 参数
2. server.js：显式调用 syncOrphanTasksOnStartup（在 runStartupRecovery 之后、initTickLoop 之前）
3. tick.js：移除 initTickLoop 内的 syncOrphanTasksOnStartup 调用（避免双调用）
4. startup-recovery.test.js：删除 DB requeue 测试，添加 pool.query 未调用断言
5. startup-recovery-enhanced.test.js：移除 requeued 字段期望

## 成功标准
- startup-recovery.js 中无 UPDATE tasks 语句
- server.js 显式调用 syncOrphanTasksOnStartup
- startup-recovery.test.js 中无重置为 queued 测试
- Brain L4 集成测试通过
