# Task Card: 修复 rumination 链路 — NotebookLM 响应清洗 + dedup 修复

## 需求（What & Why）

**功能描述**: rumination 链路的唯一故障点：NotebookLM 切换 notebook 时返回前缀噪音文本
`"Different notebook specified, starting new conversation...\nAnswer:\n[真实内容]"`，
该混合文本通过 `length > 50` 质量检查，被写入 memory_stream 作为垃圾洞察。
同时 `cecelia_events` 中无 `rumination_output` 记录导致 dedup 完全失效，垃圾洞察持续写入。

**背景**: SelfDrive 调度的 P1 任务，影响系统稳定性 KR 达成。
历史失败：两次 OOM（exit_code=137）均为 Claude Code 进程被 kill，与本次代码修复无关。

**不做什么**: 不改 NotebookLM notebook ID，不修改 tick 调度频率。

## 成功标准

- [ ] rumination 运行后 memory_stream 中的反刍洞察不含 "Different notebook specified" 前缀
- [ ] cecelia_events 中存在 rumination_output 类型记录（dedup 机制恢复工作）
- [ ] 手动触发 `/api/brain/rumination/run?force=true` 时，返回的 insights 为真实洞察文本

## 验收条件（DoD）

- [ ] [PRESERVE] 现有 rumination 流程（callLLM fallback、去重逻辑、memory_stream 写入、suggestions 写入）不变
  Test: tests/brain/rumination.test.js

- [ ] [ARTIFACT] `cleanNotebookResponse(text)` 函数存在于 rumination.js 中
  Test: manual:node -e "const {cleanNotebookResponse} = await import('./packages/brain/src/rumination.js'); const r = cleanNotebookResponse('Different notebook specified, starting new conversation...\nContinuing conversation abc..\nAnswer:\n[反刍洞察] 测试洞察内容'); if(!r.includes('测试洞察内容') || r.includes('Different notebook')) process.exit(1); console.log('PASS')"

- [ ] [BEHAVIOR] NotebookLM 返回含 "Different notebook specified" 前缀时，cleanNotebookResponse 提取 Answer 后内容作为洞察
  Test: tests/brain/__tests__/rumination.test.js

- [ ] [BEHAVIOR] 清洗后洞察长度 < 50 时，fallback 到 callLLM（不写入垃圾）
  Test: tests/brain/__tests__/rumination.test.js

- [ ] [GATE] 所有现有测试通过
  Test: manual:node -e "require('fs').accessSync('packages/brain/src/rumination.js')"

## 实现方案

**要改的文件**: `packages/brain/src/rumination.js`

**具体改动**:
1. 新增 `cleanNotebookResponse(text)` 导出函数：
   - 如果 text 包含 "Different notebook specified" → 提取 "Answer:" 后的内容
   - 支持 "Answer:\n" 和 "Answer: " 两种格式
   - 返回清洗后的文本（或原始文本如果无前缀）
2. 在 `digestLearnings()` 中，获得 NotebookLM insight 后立即调用 `cleanNotebookResponse()`
3. 清洗后重新执行 `length > 50` 检查，不足则 fallback 到 callLLM
4. cecelia_events INSERT 参数：传对象 `{ content_hash: contentHash }` 而非 `JSON.stringify()`（避免 jsonb 类型推断问题）

**Scope 锚定**: 仅改 `packages/brain/src/rumination.js`，不动其他文件
