# PRD: Feishu 消息统一经过丘脑路由

## 背景

丘脑（thalamus.js）设计目标是所有信息的统一 Gateway，但 Feishu 消息历史上直接调 handleChat，绕过了丘脑。PR #1062 已给丘脑注入 Skills 地图 + 任务队列感知，现在丘脑有能力做路由决策。

## 目标

让 Feishu 消息先经过丘脑路由，丘脑决定是直接回复、调 handleChat 还是派发任务。

## 成功标准

1. Feishu 私聊消息进来时，先调用 thalamusProcessEvent(USER_MESSAGE 事件)，再根据返回决定后续动作
2. 丘脑返回 handle_chat 时，走原有 handleChat 逻辑（兜底）
3. 丘脑返回 dispatch_task/create_task 时，创建 Brain 任务而不调 handleChat
4. 丘脑调用失败时，fallback 到原 handleChat（不影响正常对话）
5. 群聊 @mention 消息同样先过丘脑
6. Brain unit tests 通过，无现有测试破坏

## 不做什么

- 不改变 handleChat 本身的逻辑
- 不修改丘脑的 LLM prompt（只利用现有能力）
- 不改变群聊/私聊的消息获取逻辑
