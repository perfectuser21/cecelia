# Task Card: CI DoD BEHAVIOR 命令真正执行

## 需求（What & Why）

**功能描述**: 在 `.github/workflows/ci.yml` 的 engine-tests job 中新增一步，读取 task-card.md 中所有 `[BEHAVIOR]` 条目的 `Test: manual:node ...` 命令并真正执行，失败则阻断 CI。

**背景**: 目前 DoD `[BEHAVIOR]` 的 `Test: manual:xxx` 是荣誉制——CI 只检查格式（有没有 `[BEHAVIOR]` 标签、是否全部勾选），但从不实际运行这些验证命令。改成硬检查后，每次 push 时 node 类验证命令会真正跑，确保文件检查类断言有效。

**不做什么**:
- 不执行 `manual:curl`（需要运行中的 Brain 服务，CI 环境无法连接）
- 不执行 `manual:chrome:`（浏览器命令 CI 无法跑）
- 不执行 `manual:psql`、`manual:bash`（服务依赖类）
- 只执行 `manual:node`，其余前缀跳过

## 成功标准

- CI engine-tests job 新增 "DoD BEHAVIOR 命令执行" 步骤
- 步骤自动提取 task-card.md 中所有 `Test: manual:node ...` 行并执行
- `manual:curl`/`manual:chrome:`/`manual:psql`/`manual:bash` 跳过并打印 `⏭️ 跳过`
- `manual:node` 命令执行失败时 CI 报错退出
- 无 task-card.md 时整步跳过（不影响无 task-card 的 PR）

## 验收条件（DoD）

- [x] [BEHAVIOR] CI engine-tests 新增 "DoD BEHAVIOR 命令执行" 步骤，能提取并执行 manual:node 命令
  Test: manual:node -e "const c=require('fs').readFileSync('.github/workflows/ci.yml','utf8');if(!c.includes('DoD BEHAVIOR 命令执行'))process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] manual:curl / manual:chrome: / manual:psql / manual:bash 被跳过（不报错）
  Test: manual:node -e "const c=require('fs').readFileSync('.github/workflows/ci.yml','utf8');if(!c.includes('manual:curl') && !c.includes('chrome:'))process.exit(1);console.log('skip-logic present')"

- [x] [ARTIFACT] ci.yml 中新增步骤存在
  Test: manual:node -e "const c=require('fs').readFileSync('.github/workflows/ci.yml','utf8');if(!c.includes('DoD BEHAVIOR 命令执行'))process.exit(1);console.log('step exists')"

## 实现方案

**要改的文件**: `.github/workflows/ci.yml`

**修改位置**: `engine-tests` job 中，在 "DoD 格式检查" 步骤之后新增 "DoD BEHAVIOR 命令执行" 步骤

**核心逻辑**:
1. 找 task-card 文件（`.task-*.md` 或 `task-card.md`）
2. 提取所有 `Test: manual:node ` 行
3. 逐条执行，失败则 `exit 1`
4. `manual:curl`/`manual:chrome:`/`manual:psql`/`manual:bash` 打印跳过信息

**不改什么**: 不改现有 "DoD 格式检查" 步骤，不改其他 job
