# PRD：对话深度驱动记忆下钻 — L0→L1→全文 + 嘴巴主动关联叙述

## 背景

记忆系统有三层结构（L0 摘要 / L1 中间层 / 全文），但现在每轮对话都返回相同粒度（固定 L0 片段），不管用户是第一次提到某话题还是已经深聊了好几轮。

另外，嘴巴找到相关记忆后的行为是"问用户是不是这个"，不够主动——应该像真正的助理一样，点出关联、讲出上下文，让用户自己判断。

## 目标

1. **深度感知**：根据对话上下文判断话题深度（first_mention / follow_up / deep_dive）
2. **分层返回**：depth=0 → L0 摘要点一下；depth=1 → L1 更多细节；depth=2 → 全文
3. **主动关联叙述**：嘴巴 prompt 改为"我注意到这和 [Initiative: XXX] 有关，那个当时做了 A，还缺 B"，而不是"是这个吗？"

## 技术范围

### 深度检测（memory-retriever.js）

- `buildMemoryContext` 接收 `conversationHistory`（最近 5 条）
- 用 Jaccard 计算当前 query 与最近历史消息的相似度
- 超过 threshold（0.25）视为同话题 → depth++
- depth 传入 `searchSemanticMemory` 控制返回粒度

### 分层返回（memory-retriever.js）

- `formatItem` 根据 depth 决定返回字段：
  - depth=0：`summary` 或 description 前 80 字
  - depth=1：`l1_content` 或 description 前 200 字（learnings）；goals/projects 加 status + task count
  - depth=2：`content` 全文（learnings）；goals/projects 加详细 task 列表

### 嘴巴 prompt（orchestrator-chat.js）

- 删除"是这个吗"确认模式
- 改为主动关联叙述：找到相关结构化对象时，点出关联 + 状态 + 缺口，让用户自行判断

## 不在范围

- LLM 意图重写
