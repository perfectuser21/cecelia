# Task Card: CI Gate — 新增 hook/lib 文件必须有 RCI 条目

## 目标

当 PR 新增 `packages/engine/hooks/` 或 `packages/engine/scripts/devgate/` 下的文件时，
CI L2 检查该文件是否已在 `regression-contract.yaml` 中有对应 RCI 条目。
防止新能力无 RCI 覆盖悄悄滑入生产。

## 成功标准

1. 新增 `check-new-files-need-rci.cjs`，导出 `scanMissingRci(addedFiles, contractContent)` 函数
2. CI L2 新增 `new-files-need-rci-check` job（检测 hooks/ 和 scripts/devgate/ 下新增文件）
3. 三重降级保护（无新增目标文件 / 文件已有 RCI / 降级跳过 → 不失败）
4. 测试文件覆盖率 ≥ 60%

## 实现方案

### 新增脚本：check-new-files-need-rci.cjs

- 接受参数：新增文件路径列表（相对 repo 根目录）
- 目标目录：`packages/engine/hooks/` 和 `packages/engine/scripts/devgate/`
- 读取 `packages/engine/regression-contract.yaml`，提取所有 `file:` 字段
- 对每个目标目录下的新增文件，检查是否有 RCI 条目
- 导出 `scanMissingRci(addedFiles, contractContent)` 供测试 import
- Exit 0 = 通过，Exit 1 = 有文件缺失 RCI

### CI L2 新增 job

- `new-files-need-rci-check` job
- 获取新增文件：`git diff --name-only --diff-filter=A origin/main...HEAD`
- 过滤目标路径后调用脚本
- 三重降级：无新增文件 → 跳过；无目标路径文件 → 跳过；脚本不存在 → 跳过
