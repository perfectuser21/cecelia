# Task Card: 记忆系统 PR4 — Rumination 质量改进

## 需求（What & Why）

**功能描述**:
当前 Rumination 仅从 `learnings` 表 FIFO 取输入，不利用 PR2 已写入的高显著性对话记录。
导致反刍与真实对话脱节：Cecelia 在深度思考时没有考虑最近的重要对话内容。

**背景**: PR2 已把高显著性对话写入 `memory_stream`（salience_score、emotion_tag）。
PR4 让 Rumination 也消化这些条目，形成"对话 → 记忆 → 反刍 → 洞察"的完整闭环。

**不做什么**:
- 不重写 digestLearnings 核心逻辑
- 不修改 NotebookLM 调用方式
- 不改变 memory_stream 写入格式（PR2 已定义）

## 成功标准

1. [ARTIFACT] rumination.js 新增 `fetchMemoryStreamItems(db, limit)` 函数
2. [BEHAVIOR] runRumination 在 learnings 不足时补充 memory_stream 条目（salience_score ≥ 0.7，source_type='conversation_turn'，status='active'）
3. [BEHAVIOR] buildNotebookQuery 在 items 含 emotion_tag 时，查询字符串包含情绪上下文
4. [BEHAVIOR] 消化完成后，已处理的 memory_stream 条目 status 更新为 'ruminated'
5. [GATE] CI L1-L4 全部通过
