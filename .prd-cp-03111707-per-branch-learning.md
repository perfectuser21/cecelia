# PRD: Per-Branch Learning 文件——消除并行 /dev 的 LEARNINGS.md 冲突

## 背景

多个 /dev 并行运行时，每个 PR 都在 Step 10 向同一个 `docs/LEARNINGS.md` 追加内容。
先合并的 PR 修改了文件 → 后面的 PR 必然冲突 → CONFLICTING 状态 → CI 不触发 → 手动解决冲突浪费时间。

## 目标

将 Learning 写入 per-branch 独立文件（`docs/learnings/<branch>.md`），消除并行冲突。
CI 的 Learning Format Gate 改为检查 `docs/learnings/` 目录下有新文件。

## 范围

### 需要修改的文件
1. `packages/engine/skills/dev/steps/10-learning.md` — Step 10 写 learning 的目标路径
2. `.github/workflows/ci-l1-process.yml` — Learning Format Gate 检查逻辑
3. `packages/engine/skills/dev/steps/11-cleanup.md` — cleanup 时合并 learning（可选）

### 不需要修改的文件
- Brain 代码
- stop hook（已在 PR #852 修复）
- 其他 hooks

## 成功标准

1. Step 10 将 learning 写到 `docs/learnings/<branch>.md` 而非 `docs/LEARNINGS.md`
2. CI Learning Format Gate 检查 `docs/learnings/` 目录下有新文件
3. 并行多个 /dev 不再产生 LEARNINGS.md 合并冲突
4. 旧的 `docs/LEARNINGS.md` 保留（历史记录），新 learning 不再追加到此文件
