# Task Card: 全平台数据采集端点完整化 — 补齐缺失链路

## 需求（What & Why）
**功能描述**: 在 PR #1945（已有写回端点和 platform_scrape.sh 支持）基础上，补齐以下缺失链路：
1. 每日全平台采集调度器 — Brain 自动创建 platform_scraper 任务（目前无自动触发）
2. LLM 算力消耗监控 — 每日 account_usage_cache 快照到 llm_usage_snapshots 表
3. 算力消耗查询端点 — 供周报和选题引擎使用

**背景**: content_analytics 表当前为空（0 rows），因为：
- platform_scrape.sh 需要 BRAIN_API_URL 才能回写，但没有触发器
- 无每日定时采集机制 → 数据采集链路断裂 → 周报/选题引擎无法工作

**不做什么**:
- 不修改爬虫本身（在 CN 机器上）
- 不添加 N8N 工作流配置
