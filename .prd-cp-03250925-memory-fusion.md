# PRD: Memory Fusion — Claude Code auto-memory ↔ Cecelia 文档系统融合

## 背景

Claude Code auto-memory 系统（`~/.claude-account1/projects/.../memory/*.md`）已积累 36 个记忆文件，
记录了项目架构、流程规则、反馈教训。但这些文件只有 AI 能读，Dashboard 看不到。

Cecelia 文档系统（design_docs / decisions 表）已存在 API 和前端页面，但：
- design_docs：只有 1 条日记（空的）
- decisions：10466 条但都是无类别的 AI 动作日志，DecisionRegistry 页面实际为空

**目标**：打通两套系统，让每次对话的知识沉淀自动出现在 Dashboard。

## 成功标准

1. `decisions` 表有 >= 20 条有 category 的人类可读决策（来自 CLAUDE.md 和 memory feedback 文件）
2. `design_docs` 表有 >= 15 条来自 memory project/reference 文件的内容
3. Brain tick 每轮调用 memory-sync，新的 memory 文件会自动同步
4. DecisionRegistry 和 DesignVault 页面不再空白
