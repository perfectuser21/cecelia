# PRD: 修复 Brain 内部 LLM 调用超时太短

## 背景

Brain 每次 tick 触发多个并发 LLM 调用（emotion-layer、memory scoring、reflection），
实测用真实大小 prompt（~3000 token）并发 Sonnet 需要 20-30s。
当前超时设置过短，导致所有 L1/L2 调用 100% 超时失败，Brain 降级为纯 L0 脑干运行。

## 根因

- 超时是按"短 prompt 单次调用"设计的（假设 ~8s）
- 真实 prompt 含 memoryContextBlock + 系统提示 + 事件 JSON，实测 20-30s
- 并发 3 个调用时竞争 Claude Max 订阅带宽，进一步加长

## 修复范围

| 文件 | 当前值 | 目标值 |
|------|--------|--------|
| `emotion-layer.js` | 15s | 90s |
| `thalamus.js` callThalamLLM default | 30s | 90s |
| `memory.js` / batchScoreImportance | 30s（默认）| 90s |
| `reflection.js` Opus 调用 | 60s | 150s |
| `cecelia-bridge.js` 上限 | 120s | 180s |
