# PRD: Pipeline 审查前置 — Stage 2 完成后 Codex 审代码，CI 前完成所有审查

## 背景
当前流程：Stage 1 → spec_review → Stage 2 → Stage 3（push → CI → code_review）
问题：code_review 在 CI 之后做，导致 CI 通过后还要等 Codex 审查，浪费时间。

## 目标
把 code_review 从 CI 后移到 push 前，实现"每步审完再推"：
Stage 1 → spec_review → Stage 2 → code_review → Stage 3（push → CI 一次过）→ Stage 4

## 改动范围
1. `packages/engine/skills/dev/steps/02-code.md` — Stage 2 末尾加 code_review_gate 派发
2. `packages/engine/lib/devloop-check.sh` — 条件 2.5 插入 code_review 检查
3. `packages/engine/skills/dev/steps/03-integrate.md` — 删除 3.3 节 code_review 派发
4. `packages/engine/skills/dev/SKILL.md` — 更新流程描述

## 成功标准
- Stage 2 完成后派发 code_review_gate 给 Codex
- devloop-check.sh 在 step_2_code done 后检查 code_review
- Stage 3 不再派发 code_review_gate
- SKILL.md 流程描述与实际一致
