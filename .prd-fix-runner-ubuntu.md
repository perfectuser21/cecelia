# PRD: 修复 CI 并发卡队——将 self-hosted 改为 ubuntu-latest

## 问题
cecelia 仓库所有 6 个 workflow 均使用 [self-hosted, hk-vps]，
香港 VPS 每个仓库只有 1 个 runner 实例，多个 PR 同时触发时全部排队，
导致 CI 一早上卡住过不去。

## 目标
将所有 workflow 的 runs-on 从 [self-hosted, hk-vps] 改为 ubuntu-latest，
利用 GitHub Pro 计划的 3,000 分钟/月额度，实现天然并发。

## 变更范围
- .github/workflows/brain-ci.yml
- .github/workflows/devgate.yml
- .github/workflows/engine-ci.yml
- .github/workflows/quality-ci.yml
- .github/workflows/workflows-ci.yml
- .github/workflows/workspace-ci.yml

## 验收标准
1. 所有 workflow 中无 self-hosted 或 hk-vps 字样
2. CI 能在 GitHub hosted runner 上正常通过
