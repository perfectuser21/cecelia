# Task Card: 记忆系统 PR12 — RPE 闭环：RPE 信号影响任务权重

## 需求（What & Why）

**功能描述**：在 task-weight.js 中加入 RPE 历史查询，使奖赏预测误差信号真正影响任务调度权重。历史 RPE 偏高（超预期）的任务类型适当提升权重，RPE 偏低（不如预期）的适当降低。

**背景**：PR5 已实现 computeRPE 和 rpe_signal 事件记录，但 RPE 结果从未被用于影响后续任务权重。信号是孤岛——记录了但没有形成闭环。

**不做什么**：不修改现有同步 `calculateTaskWeight` 的签名；不在 tick 主路径引入强制 DB 查询（向后兼容）。

## 成功标准

1. [ARTIFACT] task-weight.js 导出 `calculateTaskWeightAsync` 和 `getTaskRPEAdjustment`，使用 rpe_signal 历史查询
2. [BEHAVIOR] RPE 历史平均值 > 0 → 权重提升；< 0 → 权重降低；无数据 → 权重不变
3. [GATE] CI 全部通过

- RPE_WEIGHT_SCALE = 4（avg_rpe * 4 = weight adjustment）
- RPE_WEIGHT_CAP = 10（最大调整量 ±10）
- RPE_SAMPLE_SIZE = 20（最近 20 条 rpe_signal 事件）
