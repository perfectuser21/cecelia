# Task Card: Capability Probe 系统 + Auto-Fix 链路修复

## 需求（What & Why）

**功能描述**: 让 Cecelia 能感知自己身体里每条关键链路通不通，断了自己修，不需要通知用户。

**背景**:
- auto-fix 链路代码完整但从未触发过（0次），原因是 `callCortexForRca()` 只给 Cortex 5个字段，导致 RCA 信心度永远 < 70%
- Cecelia 有 37 个 capability、63+ 个 skill，但不知道哪些能用哪些是孤岛
- 174 条 failure_pattern learning，0 条被应用
- 数据：5046 次隔离、752 次熔断、189 完成 vs 200 失败

**不做什么**:
- 不新增 LLM agent（用现有 Cortex Sonnet）
- 不改变 auto-fix 的判断逻辑（shouldAutoFix 阈值不变）
- 不改变任务派发架构

## 成功标准

1. [BEHAVIOR] callCortexForRca() 给 Cortex 提供完整上下文（失败日志、run_events 详情、相关代码路径），使 RCA 信心度有机会达到 70%+
2. [ARTIFACT] 新增 capability-probe.js 模块，定期探测关键链路健康状态
3. [BEHAVIOR] Probe 发现链路故障时自动创建修复任务（走 auto-fix 路径）
4. [GATE] 所有现有测试通过
