# Task Card: codex-bridge 账号无状态化 — US Brain 注入 token

## 需求（What & Why）

**功能描述**：
改造 Codex 执行架构，使 Xi'an 机器（codex-bridge）不再持有本地账号信息。
所有 Codex auth token 由美国 Brain 统一管理，在派发任务时随 request body 注入。

**背景**：
- 当前：Xi'an bridge 读取本地 `~/.codex-teamX/auth.json`，token 过期需要在 Xi'an 重新登录
- 目标：美国 M4 持有所有 5 个账号的 auth.json，派发任务时选最优账号并注入 token
- 效果：Xi'an 成为纯执行节点，换机器零迁移成本

**不做什么**：
- 不改动 `codex-account-usage.cjs`（仍用于 Xi'an 本地 fallback）
- 不改动 runner.sh
- 不删除 Xi'an 本地 auth.json（向后兼容）

## 接口定义

### accounts 字段 schema

```json
{
  "accounts": [
    {
      "id": "team1",
      "auth": {
        "auth_mode": "chatgpt",
        "tokens": {
          "access_token": "eyJ...",
          "id_token": "eyJ...",
          "refresh_token": "...",
          "account_id": "org-..."
        },
        "last_refresh": "2026-03-21T..."
      }
    }
  ]
}
```

`auth` 字段即 `~/.codex-teamX/auth.json` 的完整内容（JSON object），直接序列化传输。

### 临时目录命名

```
/tmp/codex-inj-{taskId}-{timestamp}/
  team1/auth.json
  team2/auth.json
```

- `taskId`：Brain Task UUID（`f6b254da-...`）
- `timestamp`：`Date.now()`，保证同一 task 并发安全

目录权限：`chmod 700`（进程级隔离）

### 涉及文件

- `packages/brain/scripts/codex-bridge/codex-bridge.cjs`：Xi'an 端（接收注入）
- `packages/brain/src/routes/brain-meta.js`：US Brain 端（账号选择 + 注入）

US Brain 的 codex 派发端点：`POST /api/brain/codex-exec`（`brain-meta.js` 中 `/api/brain/codex-exec` 路由）

3. [BEHAVIOR] 无 `accounts` 字段时 codex-bridge 降级到本地 `selectBestCodexAccount()`
4. [BEHAVIOR] 临时目录权限 700，进程退出/崩溃后自动清理
5. [GATE] CI 全部通过，单元测试覆盖账号排序和降级逻辑

## DoD 条目

### D1：codex-bridge.cjs — 接收注入 token

- `POST /run` 和 `POST /execute` body 支持 `accounts: [{id, auth}]`
- 有 `accounts`：
  - 写 `/tmp/codex-inj-{taskId}-{timestamp}/{id}/auth.json`（目录权限 700，文件权限 600）
  - 第一个账号为主账号（primaryCodexHome），其余为轮换账号（CODEX_HOMES）
  - `finally` 块 `fs.rmSync(tmpDir, { recursive: true, force: true })` 清理
- 无 `accounts`：降级到原有 `selectBestCodexAccount()` 逻辑（零改动）
- `POST /run` codex_dev 类型：CODEX_HOMES 由临时目录列表组成（冒号分隔）

### D2：brain-meta.js — 账号选择 + 注入

新增 `selectBestAccountFromLocal(maxAccounts = 3)` 函数：
1. 遍历 `~/.codex-team{1,2,3,4,5}/auth.json`，跳过不存在的
2. 并行调用 `https://chatgpt.com/backend-api/wham/usage`（`Authorization: Bearer {access_token}`, `ChatGPT-Account-Id: {account_id}`）
3. 按 `rate_limit.primary_window.used_percent` 升序排序
4. 返回前 `maxAccounts` 个 `{id, auth}` 对象

在 `POST /api/brain/codex-exec` 路由调用 Xi'an bridge 前：
- 调用 `selectBestAccountFromLocal()`
- 若成功：request body 加入 `accounts` 字段
- 若失败/超时（8s）：降级，不加 `accounts`，Xi'an 自选本地账号

### D3：自动化测试

在 `packages/brain/scripts/codex-bridge/` 新增 `codex-bridge.test.cjs`：

```
测试用例：
1. 有 accounts → 临时目录创建、CODEX_HOME 正确设置
2. 无 accounts → selectBestCodexAccount() 被调用（mock）
3. 执行完毕后临时目录不存在（finally 清理）
4. accounts 为空数组 → 降级到本地选账号
```

在 `packages/brain/src/` 新增 `selectBestAccountFromLocal.test.js`（或同文件 describe）：

```
测试用例：
1. 多账号按 used_percent 升序排序
2. wham/usage 返回 401 → 该账号跳过
3. 所有账号 wham/usage 失败 → 返回空数组（触发降级）
4. 最多返回 maxAccounts 个
```
