# PRD - 方案A Hook Gates：7个真锁实现（bash-guard/stop-dev/branch-protect）

**需求来源**: Brain 自动调度任务 — agent_ops 领域
**功能描述**: /dev 工作流中 99% 的"必须要求"是 .md 文字假锁，只有 bash-guard/stop-dev/branch-protect 是真正能阻止操作的真锁。本任务实现 7 个 Hook Gate 真锁，确保质量门禁从源头拦截而非依赖 CI 事后检查。
**涉及文件**:
- `packages/engine/hooks/bash-guard.sh` — 拦截 git push/commit
- `packages/engine/hooks/stop-dev.sh` — 放行前验证 Learning 格式、Brain 代码更新
- `packages/engine/hooks/branch-protect.sh` — 写 PRD/DoD/engine/skills 时触发对应检查
**非目标**: 不修改 CI 流程、不修改 check-prd.sh / check-dod-mapping.cjs 的逻辑本身

## 7 个真锁详情

| # | Hook | 触发条件 | 执行动作 |
|---|------|----------|----------|
| 1 | bash-guard.sh | git push | 先跑 local-precheck.sh，失败则阻止 push |
| 2 | stop-dev.sh | stop hook 触发（会话结束前） | 跑 check-learning.sh 验证内容格式 |
| 3 | branch-protect.sh | 写 .prd-*.md 文件 | 跑 check-prd.sh 验证 PRD 格式 |
| 4 | branch-protect.sh | 写 .dod-*.md 文件 | 跑 check-dod-mapping.cjs 验证 DoD 映射 |
| 5 | bash-guard.sh | git commit | 验证 commit message 符合 feat:/fix:/docs: 格式 |
| 6 | stop-dev.sh | Brain 代码文件变更 | 检查 DEFINITION.md 是否已更新 |
| 7 | branch-protect.sh | 改动 engine/skills/ 路径 | 检查 Engine 三要素（版本/registry/LEARNINGS） |

## 成功标准

- git push 被拦截时，local-precheck.sh 自动运行且失败时 push 被阻止
- git commit message 不符合 feat:/fix:/docs: 等格式时被阻止
- 写 .prd-*.md 时 check-prd.sh 自动运行（exit 非零时阻止写入或给出错误）
- 写 .dod-*.md 时 check-dod-mapping.cjs 自动运行
- stop-dev.sh 放行前验证 Learning 内容格式（exit 非零时阻止会话结束）
- Brain 代码改动时检查 DEFINITION.md 是否同步更新
- engine/skills/ 改动时检查 Engine 三要素

### 异常处理 & 报警规范

N/A — 无 IO 操作，纯本地 hook 逻辑
