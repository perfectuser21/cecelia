# Task Card: fix(brain): executeCopywriting 降级策略 + 清除重复函数

## 背景

content-pipeline 文案生成管道中，"亲爱的安先生"等关键词反复失败（4次）。

**根本原因（已确认）**：
1. 旧版代码在 `notebook_id = null` 时创建占位符 finding，`brand_relevance: 2`，`source: '系统'`
2. `executeCopywriting` 过滤 `brand_relevance >= 3`，占位符被全部剔除 → 硬失败
3. 没有降级策略 — 遇到低分 findings 应当降级继续，让 copy-review 做质量判断
4. `content-pipeline-executors.js` 中有重复定义的 `findOutputDir` 函数

注：PR #1754 已修复 executeResearch（加入 web search），新生成的 findings 均为 brand_relevance: 4。
本 PR 修复的是：旧占位数据导致当前 copywriting 硬失败的问题，以及代码冗余。

## 成功标准

1. `executeCopywriting` 收到全部 brand_relevance < 3 的 findings 时，降级使用全部 findings 继续执行，不硬失败
2. `executeCopywriting` 在低质量 findings 时降级继续，不硬失败

## 修改目标

文件：`packages/brain/src/content-pipeline-executors.js`

**`executeCopywriting` 降级路径**：
- 若 `top.length === 0` 但 `findings.length > 0` → 降级使用全部 findings，记录 warn 日志
- 若 `findings.length === 0` → 保持原有硬失败逻辑（research 从未运行）
- `top` 改为 `let` 声明
