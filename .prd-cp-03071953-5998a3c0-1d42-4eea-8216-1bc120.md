# PRD: strategy_session task_type 实现

## 背景

`strategy_session` task_type 已注册到 Brain，skill 也已创建，但 execution-callback 缺少解析 KR 并写入 goals 的逻辑。

## 当前状态

### 已完成 ✅
1. task_type 注册（task-router.js, executor.js）
2. model-registry agent 注册
3. /strategy-session skill 创建
4. strategy-session-parser.js 解析器
5. parser 测试用例

### 待实现 🚧
- execution-callback 中解析 KR JSON → 写入 goals 表

## 功能需求

### D4: execution-callback 解析 KR JSON
- 从 task output 中提取 `krs` 数组
- 支持 ` ```json ... ``` ` 块格式
- 支持纯 JSON 字符串格式

### D5: goals 表写入
- 每个 KR 写入 `goals` 表
- 字段映射：
  - `title` → KR title
  - `domain` → KR domain (product/tech/ops/finance/growth)
  - `owner_role` → KR owner_role (转为小写: cto/cfo/coo/ceo)
  - `priority` → KR priority (P0/P1/P2，无效值默认 P1)
  - `status` → 'pending'
  - `progress` → 0
  - `type` → 'area_okr'
- 关联 `project_id`（从 task payload 中获取）

### D6: task payload 写入
- `meeting_summary` 写入 task payload
- `key_tensions` 写入 task payload

### D7: 错误处理
- JSON 解析失败时不抛出，记录警告日志
- krs 为空数组时不执行写入
- 缺少 domain/owner_role 时使用 null

## 验收标准

1. 创建 `strategy_session` 任务并执行后，goals 表新增对应 KR 记录
2. KR 包含正确的 domain 和 owner_role
3. task payload 包含 meeting_summary
4. JSON 解析异常不导致 callback 失败
