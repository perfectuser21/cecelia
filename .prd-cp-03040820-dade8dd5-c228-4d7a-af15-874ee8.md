## PRD - Dashboard 48h系统简报展示和端到端测试

**需求来源**: Brain 自动调度任务 - Initiative「48h简报自动化调度集成」第三个 PR
**功能描述**: 在 Dashboard 新增简报展示页面（列表+详情），在 Brain 新增简报 API 端点，并完成端到端测试验证
**涉及文件**:
- `apps/dashboard/src/pages/Reports.tsx` - 简报列表页
- `apps/dashboard/src/pages/ReportDetail.tsx` - 简报详情页
- `apps/dashboard/src/router.tsx` - 路由配置
- `packages/brain/src/server.js` - Brain API 端点

**成功标准**:
1. Brain API GET /api/brain/reports 返回简报列表
2. Brain API GET /api/brain/reports/:id 返回简报详情
3. Dashboard /reports 页面正确显示简报列表
4. Dashboard /reports/:id 页面正确显示简报详情
5. 端到端测试全部通过
6. CI 通过，PR 合并

**非目标**:
- 不修改 tick.js 48h 定时检查逻辑（已在前两个 PR 完成）
- 不修改 cortex 简报生成接口（已在前两个 PR 完成）
- 不做简报编辑功能

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| API 查询 system_reports 表失败 | 返回 500 + 错误信息 | P2 | reports_query_error |
| 简报 ID 不存在 | 返回 404 | P3 | report_not_found |
| Dashboard 加载 API 失败 | 显示错误状态 + 重试按钮 | P3 | dashboard_reports_load_error |
