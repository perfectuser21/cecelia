# PRD: 统一 cecelia-run — Mac 适配代码回归 git 源文件

## 背景

`/Users/administrator/bin/cecelia-run`（620 行）是生产运行版本，包含大量 Mac 适配代码（root 处理、PATH 设置、失败分类、resume 支持等），这些代码从未合并回 git 源文件 `packages/brain/scripts/cecelia-run.sh`（545 行）。

LaunchDaemon plist 硬编码指向 `~/bin/cecelia-run`，导致：
1. git 源文件和运行版本长期不一致（偏差 75 行）
2. 任何对运行版本的修改都绕过 CI 和代码审查
3. 重新部署时改动会丢失

## 目标

1. 将 `~/bin/cecelia-run` 的所有 Mac 适配代码合并进 `packages/brain/scripts/cecelia-run.sh`
2. 同时加入 resume 修复（--session-id / --resume，解决超时重头跑浪费 token 的问题）
3. `~/bin/cecelia-run` 改为指向 git 源文件的 symlink（或直接更新 plist）
4. git 源文件成为唯一真实来源（SSOT）
5. 修复 Stop Hook：CI pending 时改为 exit 2（阻止退出），而不是 exit 0

## 成功标准

- `packages/brain/scripts/cecelia-run.sh` 包含所有 Mac 适配代码
- `~/bin/cecelia-run` 是 symlink 指向 git 源文件
- Brain plist 的 CECELIA_RUN_PATH 指向 git 版本路径
- Brain 重启后正常运行
- Stop Hook 在 CI pending 时返回 exit 2（不退出）
