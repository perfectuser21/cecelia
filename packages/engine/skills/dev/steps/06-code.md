- `功能.ts` → `功能.test.ts`
- 例：`login.ts` → `login.test.ts`

## Step 6.5：Simplify（强制，不可跳过）

> **代码写完后立刻执行。不做完不能进 Step 7。**

### 扫描范围

只扫描**本次改动的文件**，不是全仓库：

```bash
git diff --name-only HEAD
```

### 必须清理的 4 类问题

**类型 1：被替代的旧代码**

新函数/新逻辑已经替代了旧的 → 旧的必须删。包括：
- 被新函数替代的旧函数
- 被新变量替代的旧变量
- 注释掉的旧实现块（`// old: ...` / `/* deprecated */`）

git 有历史记录，不需要用注释保留旧代码。

**类型 2：重复逻辑**

改动文件里出现两段几乎一样的代码 → 提取成共用函数，两处都改成调用它。

**类型 3：矛盾注释**

- 注释说"做 A"，代码实际做 B → 删注释或改注释
- 注释说"TODO: 临时方案"但已经是正式代码 → 删
- 注释描述的行为和代码行为不一致 → 以代码为准，修注释

**类型 4：无用 import**

改动文件里有 import 但代码里没用到 → 删。

### 完成标准

改动文件里确认：
- ✅ 无被替代的旧函数/旧变量残留
- ✅ 无注释掉的旧代码块
- ✅ 无矛盾注释（注释和代码行为一致）
- ✅ 无未使用的 import

全部满足后才能继续。
