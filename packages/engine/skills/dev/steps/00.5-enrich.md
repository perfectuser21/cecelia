# Step 0.5: PRD Enrich — autonomous_mode 粗 PRD 自动丰满

> 仅 autonomous_mode=true 激活。粗糙 PRD 先派 Enrich Subagent 多轮自反思补全成完整 PRD，再进 Stage 1。

## 0. 激活条件

```bash
BRANCH_NAME=$(git rev-parse --abbrev-ref HEAD)
DEV_MODE_FILE=".dev-mode.${BRANCH_NAME}"
AUTONOMOUS_MODE=$(grep "^autonomous_mode:" "$DEV_MODE_FILE" 2>/dev/null | awk '{print $2}' || echo "false")

[[ "$AUTONOMOUS_MODE" != "true" ]] && exit 0  # 非 autonomous 跳过
```

## 1. 判断是否需要 enrich

把用户传入的 PRD 写入临时文件，用 enrich-decide.sh 判断：

```bash
RAW_PRD=".raw-prd-${BRANCH_NAME}.md"
# 用户 PRD 写入 $RAW_PRD（由 /dev 启动时主 agent 写）

bash packages/engine/skills/dev/scripts/enrich-decide.sh "$RAW_PRD"
if [[ $? -eq 0 ]]; then
    echo "PRD 已足够丰满，跳过 enrich"
    exit 0
fi
```

判断规则（任一不满足即 thin）:
- 长度 >= 500 字节
- 含 `## 成功标准` section
- 含 `## 不做` section

## 2. 派 Enrich Subagent

**使用 `superpowers:brainstorming` 的自反思框架，但全程 subagent 内部执行，不问用户**。

主 agent 调 Task/Agent 工具派遣 Enrich Subagent，prompt 如下：

---

### Enrich Subagent 指令模板

你是 PRD Enrich Subagent。读 raw PRD，通过多轮自反思产出一份详细、可行、零歧义的 enriched PRD，供后续 /dev Stage 1 使用。

**严禁问用户问题**。一切空白自己用代码探索 + 合理默认填。

### 步骤

1. **读 raw PRD**（用户原始输入，可能只有一句话）

2. **探索代码库**找背景：
   - 搜 PRD 提到的关键词 (grep/glob)
   - 读相关现有文件
   - 查 git log 找相关 commit

3. **5 自问**（brainstorming 骨架，自己回答）：
   - 真实目的是什么？用户为什么要这个？
   - 技术约束是什么？（已有架构、依赖、兼容性）
   - 成功怎么验证？（测试命令、预期输出、可衡量指标）
   - 有哪些实现方案？列 2-3 个，Good/Bad 对比
   - Scope 边界在哪？明确"不做"列表

4. **3 轮自 review**：
   - 轮 1：初稿，填所有 section
   - 轮 2：找漏洞 — 有没有遗漏的边界？命令能跑吗？
   - 轮 3：最终检查零占位符 — TBD/TODO/稍后/适当 全无

5. **产出 enriched PRD** 格式：

```markdown
# PRD: <主题>

## 背景
<用户原 PRD + 代码探索结论>

## 真实目的
<推导自代码探索 + 第 1 问>

## 成功标准
1. <精确可测>
2. <精确可测>
...

## 方案选择
| 方案 | Good | Bad | 选 |
|------|------|-----|-----|
| A | ... | ... | 是 |
| B | ... | ... | 否 |

## 涉及文件
- <精确路径>（新建/修改）

## 不做
- <明确列出 scope 外的事>

## 假设
- <所有填空的默认假设>
```

### 输出

- 写入 `.enriched-prd-<branch>.md`
- 返回 DONE 或 BLOCKED

---

## 3. 主 agent 最终 review

Enrich Subagent 返回后，主 agent 读 enriched PRD，做一轮最终 check：

```bash
bash packages/engine/skills/dev/scripts/enrich-decide.sh ".enriched-prd-${BRANCH_NAME}.md"
if [[ $? -eq 1 ]]; then
    echo "Enrich 产出仍 thin，派一次 revise"
    # 再派一次 Enrich Subagent 带 feedback
fi
```

达标则进 Stage 1。

## 4. Stage 1 输入切换

Stage 1 `01-spec.md` autonomous 分支 **优先读 `.enriched-prd-${BRANCH_NAME}.md`**，否则读原 PRD。

---

## 完成标志

- `.enriched-prd-${BRANCH_NAME}.md` 文件存在且通过 enrich-decide 检查
- `.dev-mode` 里 `step_0_5_enrich: done` (可选字段)
- 进入 Stage 1
