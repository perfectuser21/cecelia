# Stage 3: Integrate — Push + PR

> push + 创建 PR。CI 监控和合并由 Stop Hook 自动控制。

---

## 3.0 Harness 模式

```bash
BRANCH_NAME=$(git rev-parse --abbrev-ref HEAD)
DEV_MODE_FILE=".dev-mode.${BRANCH_NAME}"
HARNESS_MODE=$(grep "^harness_mode:" "$DEV_MODE_FILE" 2>/dev/null | awk '{print $2}' || echo "false")
```

**harness_mode = true 时**：只需 push + 创建 PR，不监控 CI。
Stop Hook 0.5 通道检测到 step_2_code=done + PR 已创建 → 自动 exit 0。

---

## 3.1 会话恢复检测

```bash
BRANCH_NAME=$(git rev-parse --abbrev-ref HEAD)
EXISTING_PR=$(gh pr list --head "$BRANCH_NAME" --state all --json number,url,state -q '.[0]' 2>/dev/null)
# 已合并 → 跳到 Stage 4；已开 → 直接标记完成（Stop Hook 处理 CI）
```

---

## 3.2 提交 + 推送 + 创建 PR

```bash
git add -u
git commit -m "feat: <功能描述>

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>"
/usr/bin/git push -u origin HEAD

BASE_BRANCH="main"
gh pr create --base $BASE_BRANCH --title "feat: <功能描述>" --body "## Summary
- <主要改动>

## Test
- [x] 本地测试通过

## SYSTEM BEHAVIOR CHANGE
Before: N/A
After:  N/A

## UNIMPLEMENTED
- N/A

---
Generated by /dev workflow"
```

---

## 3.3 CI 失败修复（Stop Hook 触发时执行）

> 不需要主动轮询 CI。Stop Hook 会在 CI 失败时 exit 2 并告知原因。
> 收到 Stop Hook 的 CI 失败指令后，按以下步骤修复：

```
① 分析原因（gh run view --log-failed）
② 本地复现（npm test / bash scripts/local-precheck.sh）
③ 修复代码
④ 本地验证通过 → push
```
