# Step 0.7: Decision Query — 查 Alex 历史决策作为 autonomous 约束

> 仅 autonomous_mode=true 激活。在 00.5-enrich 之后、01-spec 之前执行。
> 防止 AI 在 brainstorming 阶段自己瞎想设计决策。

## 0. 激活条件

```bash
BRANCH_NAME=$(git rev-parse --abbrev-ref HEAD)
DEV_MODE_FILE=".dev-mode.${BRANCH_NAME}"
AUTONOMOUS_MODE=$(grep "^autonomous_mode:" "$DEV_MODE_FILE" 2>/dev/null | awk '{print $2}' || echo "false")
[[ "$AUTONOMOUS_MODE" != "true" ]] && exit 0
```

## 1. 调用 Brain 决策匹配 API

```bash
PRD_FILE=".enriched-prd-${BRANCH_NAME}.md"
[[ ! -f "$PRD_FILE" ]] && PRD_FILE=".raw-prd-${BRANCH_NAME}.md"
[[ ! -f "$PRD_FILE" ]] && { echo "no PRD found"; exit 1; }

PRD_CONTENT=$(cat "$PRD_FILE")
RESULT=$(curl -s --connect-timeout 3 --max-time 10 \
  -X POST "http://localhost:5221/api/brain/decisions/match" \
  -H "Content-Type: application/json" \
  -d "$(jq -n --arg prd "$PRD_CONTENT" '{prd: $prd}')" 2>/dev/null || echo '{}')
```

## 2. 解析 + 处理

解析返回结果，区分三种情形：

- `matched` 非空 → 写 `.decisions-${BRANCH_NAME}.yaml`，包含每条匹配决策
- `missing_critical` 非空 → **暂停 autonomous**，创建 Brain task 等 Alex 决策
- `missing_routine` 非空 → 合理默认 + plan 文件标注 "auto-default: X, no decision found"

```bash
MATCHED=$(echo "$RESULT" | jq -r '.matched // [] | length')
MISSING=$(echo "$RESULT" | jq -c '.missing // []')

# 分类 missing
missing_critical=$(echo "$MISSING" | jq -c '[.[] | select(.classification == "critical")]')
missing_routine=$(echo "$MISSING" | jq -c '[.[] | select(.classification == "routine")]')

# critical 缺失 → 暂停 autonomous，创 Brain task
critical_count=$(echo "$missing_critical" | jq 'length')
if [[ "$critical_count" -gt 0 ]]; then
  echo "[DECISION-QUERY] missing_critical: $critical_count topics, 暂停 autonomous"
  curl -s -X POST "http://localhost:5221/api/brain/tasks" \
    -H "Content-Type: application/json" \
    -d "$(jq -n \
      --argjson missing "$missing_critical" \
      '{type:"decision_needed",title:"autonomous_mode 等待 Alex 决策",meta:{missing_critical:$missing}}')" \
    > /dev/null
  exit 2
fi
```

## 3. 产出

`.decisions-${BRANCH_NAME}.yaml` 格式：

```yaml
matched:
  - topic: 数据库选型
    decision: 用 PostgreSQL + pgvector
    confidence: 0.95
    source_id: 123
missing_defaults:
  - topic: 日志输出路径
    default: /var/log/app/
    reason: no decision found, using project convention
paused_critical:
  - topic: 部署架构
    reason: critical decision missing, awaiting Alex
```

写入命令：

```bash
{
  echo "matched:"
  echo "$RESULT" | jq -r '.matched[]? | "  - topic: \(.decision_topic)\n    decision: \(.decision)\n    confidence: \(.confidence)\n    source_id: \(.source_id)"'
  echo "missing_defaults:"
  echo "$missing_routine" | jq -r '.[]? | "  - topic: \(.topic)\n    default: auto\n    reason: no decision found, using project convention"'
} > ".decisions-${BRANCH_NAME}.yaml"

echo "[DECISION-QUERY] wrote .decisions-${BRANCH_NAME}.yaml"
```

## 4. Stage 1 读取方式

`01-spec.md` autonomous 分支将读 `.decisions-${BRANCH_NAME}.yaml`，在 Task Card 实现方案引用决策源。

Stage 2 Implementer 也会读此文件，实现必须遵守决策约束。
