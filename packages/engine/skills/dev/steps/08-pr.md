# Step 8: 提交 PR

> 版本号 + commit + push + PR
> **创建 PR 后继续 Step 9 等待 CI，Stop Hook 控制完成条件**

**Task Checkpoint**: `TaskUpdate({ taskId: "8", status: "in_progress" })`

---

## 8.1 会话恢复检测

**先检测是否是恢复的会话**：

```bash
EXISTING_PR=$(gh pr list --head "$BRANCH_NAME" --state all --json number,url,state -q '.[0]' 2>/dev/null)

if [ ! -z "$EXISTING_PR" ]; then
  PR_STATE=$(echo "$EXISTING_PR" | jq -r '.state')

  if [ "$PR_STATE" = "MERGED" ]; then
    echo "✅ PR 已合并，跳到 cleanup"
  elif [ "$PR_STATE" = "OPEN" ]; then
    echo "✅ PR 已存在，跳到 CI"
  fi
fi
```

---

## 8.2 版本号更新

**检测是否为测试任务**：

```bash
IS_TEST=$(git config branch."$BRANCH_NAME".is-test 2>/dev/null)

if [ "$IS_TEST" = "true" ]; then
    echo "🧪 测试任务：跳过 CHANGELOG 和版本号更新"
    # 不更新 package.json 和 CHANGELOG.md
else
    # 正常更新版本号
fi
```

**正常任务：提交前必须更新版本号（semver）**：

| commit 类型 | 版本变化 |
|-------------|----------|
| fix: | patch (+0.0.1) |
| feat: | minor (+0.1.0) |
| feat!: / BREAKING: | major (+1.0.0) |

**版本号变化示例**：

| 当前版本 | commit 类型 | 新版本 |
|---------|-------------|--------|
| 8.16.0 | fix: | 8.16.1 |
| 8.16.1 | feat: | 8.17.0 |
| 8.17.0 | feat!: | 9.0.0 |

更新 `package.json` 和 `CHANGELOG.md`。

**测试任务**：跳过版本号和 CHANGELOG 更新，commit 消息使用 `test:` 前缀。

---

## 8.3 Squash Evidence Commit（新增）

**在提交前，检查并合并 evidence commit**：

```bash
# 如果最后一个 commit 是 evidence commit，自动合并到代码 commit
bash scripts/squash-evidence.sh
```

**说明**：
- 避免 SHA 不匹配问题
- 保持干净的 commit 历史
- 最终只有一个 commit（代码 + evidence）

---

## 8.4 提交

```bash
git add -A
git commit -m "feat: <功能描述>

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>"
```

**注意**：
- 如果 Step 7 (Verify) 已经暂存了 evidence，这里会一次性提交
- 如果没有暂存 evidence，这里会包含所有变更

---

## 8.5 推送

```bash
/usr/bin/git push -u origin HEAD
```

---

## 8.5 创建 PR

```bash
BASE_BRANCH=$(git branch -r | grep -q 'origin/develop' && echo "develop" || echo "main")
gh pr create --base $BASE_BRANCH --title "feat: <功能描述>" --body "## Summary
- <主要改动>

## Test
- [x] 本地测试通过

---
Generated by /dev workflow"
```

---

## 8.6 完成后

**标记步骤完成**：

```bash
BRANCH_NAME=$(git rev-parse --abbrev-ref HEAD 2>/dev/null || echo "")
DEV_MODE_FILE=".dev-mode.${BRANCH_NAME}"
[[ -f "$DEV_MODE_FILE" ]] || DEV_MODE_FILE=".dev-mode"
sed -i "s/^step_8_pr: pending/step_8_pr: done/" "$DEV_MODE_FILE"
echo "✅ Step 8 完成标记已写入 .dev-mode"
```

**Task Checkpoint**: `TaskUpdate({ taskId: "8", status: "completed" })`

**立即执行下一步**：读取 `skills/dev/steps/09-ci.md` 并继续

**PR 创建后，立即继续 Step 9 等待 CI 并合并**。

Stop Hook 完成条件：
1. PR 已创建 ✅（刚创建）
2. CI 通过？→ Step 9 检查
3. PR 已合并？→ Step 9 执行

**不要在 PR 创建后停止**，继续执行 Step 9。
