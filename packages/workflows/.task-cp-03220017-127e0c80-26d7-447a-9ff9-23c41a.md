# Task Card: brain-register 新增 task_type 时验证 skill 文件存在

## 需求（What & Why）
**功能描述**:
当向 Brain executor.js 的 skillMap 新增 task_type 条目时，需要同步验证对应的 `packages/workflows/skills/<skill-name>/SKILL.md` 文件是否存在。
若缺少该文件，Brain 在 dispatch 时会找到 skill 路径，但 Claude Code 无法加载对应 skill，导致派发失败。

**背景**:
来自知识反刍 Batch 1。当前 brain-register SKILL.md 的 B 类（Executor 任务类型）流程只引导改 executor.js，但未提示验证 SKILL.md 是否存在。

**不做什么**: 不修改现有的 check-executor-agents.mjs 和 check-skills-registry.mjs；不改动 executor.js 的 skillMap 内容。

## 成功标准

1. [ARTIFACT] `packages/engine/scripts/devgate/check-skill-file-exists.mjs` 已创建，读取 executor.js skillMap 中新增的 skill 条目，若对应 SKILL.md 不存在则 exit 1
2. [ARTIFACT] `packages/workflows/skills/brain-register/SKILL.md` B 类步骤中新增验证命令：检查 SKILL.md 存在性
3. [BEHAVIOR] 在 `.github/workflows/ci-l2-consistency.yml` brain-l2 job 中调用新脚本（diff-based，只检查 PR 新增条目）
4. [GATE] CI 全部通过

## 验收条件（DoD）

## 实现方案（Stage 2 探索后填写）

**要改的文件**:
1. 新建 `packages/engine/scripts/devgate/check-skill-file-exists.mjs` — diff-based CI 脚本
2. 修改 `packages/workflows/skills/brain-register/SKILL.md` — B 类步骤新增验证说明
3. 修改 `.github/workflows/ci-l2-consistency.yml` — brain-l2 job 新增步骤

**Scope 锚定**:
- diff-based 检查逻辑：`git diff origin/main -- packages/brain/src/executor.js` 解析新增行
- 只检查新增的 skill 路径（不影响现有 3 个缺失文件）
- soft warning（检查脚本输出警告但不阻断 CI）或 hard check（新增必须有 SKILL.md）
- 决策：**hard check 仅针对 PR 新增条目**，旧条目豁免
