# Task Card: 修复 Sprint Contract Gate — Evaluator 独立测试计划强制执行

## 需求（What & Why）

**功能描述**: 修复 spec_review 的 Evaluator，使其必须独立生成测试计划（independent_test_plans > 0），不再允许直接 PASS 而跳过合约校验，同时修复 reviewer_model 字段记录不准确的问题。

**背景**: spec_review 的 independent_test_plans 在 12/18 次任务中为 0，说明 Evaluator 没有独立工作就直接 PASS，合约机制形同虚设。CRG（Contract Review Gate）的 prompt 不够严格，导致有 issues 时也未能触发更严格判断。

**不做什么**:
- 不修改 spec_review 之外的其他 skill
- 不改变 spec_review 的调用接口和输出数据结构（只修复内容质量）
- 不改变 DoD 验收框架本身

## 成功标准

1. [BEHAVIOR] Evaluator 先读需求描述独立生成测试计划，independent_test_plans 字段在每次 /dev 执行时均 > 0
2. [BEHAVIOR] CRG prompt 加严后，当存在 issues 时能触发更严格判断而非直接 PASS
3. [BEHAVIOR] reviewer_model 字段正确记录实际使用的模型名称
4. [GATE] CI 全部通过
