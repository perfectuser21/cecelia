# PRD: AI Coding 意图扩展 + CTO Review 两阶段前置审查集成（Engine 侧）

## 背景

AI coding 存在意图损耗问题：用户说1，AI交付0.3。需要两个前置环节集成到 dev workflow：
1. Codex A（意图扩展）：Step1后自动触发，沿项目链路补全PRD，生成 enriched PRD
2. Codex B（CTO review）：Step2代码写完后自动触发，读 enriched PRD + DoD + diff，PASS才允许push

## 本 PR 范围（Engine 侧）

- 不改动 packages/brain/（由 Brain Agent 负责）
- 只改 Engine/Workflow 侧：devloop-check.sh + 两个 step 文件

## 需要做的事

### 1. packages/engine/lib/devloop-check.sh
新增两个等待条件：
- 条件 1.5（PR检查之前）：检查 intent_expand_task_id，等待 intent_expand_status=completed
- 条件 2.5（CI检查之前）：检查 cto_review_task_id，等待 cto_review_status=pass

### 2. packages/workflows/skills/dev/steps/01-taskcard.md
Step 1 末尾新增 intent_expand 触发指令：
- 检查 .dev-mode 中是否有 brain_task_id
- 有则 POST Brain API /tasks/:id/request-intent-expand
- 把返回的 intent_expand_task_id 写入 .dev-mode

### 3. packages/workflows/skills/dev/steps/02-code.md
代码写完 push 之前新增 cto_review 触发指令：
- POST Brain API /tasks/:id/request-cto-review（带 PR number 或 diff）
- 把返回的 cto_review_task_id 写入 .dev-mode
- devloop-check.sh 条件 2.5 会等待结果

## 成功标准

- devloop-check.sh 在 intent_expand_status != completed 时阻塞
- devloop-check.sh 在 cto_review_status != pass 时阻塞
- 01-taskcard.md 包含 request-intent-expand 触发指令
- 02-code.md 包含 request-cto-review 触发指令
