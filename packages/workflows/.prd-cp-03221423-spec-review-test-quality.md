# PRD: spec-review SKILL.md 新增维度 D — DoD Test 字段可执行性验证

## 背景

当前 spec-review 审查 DoD 质量，但不检查 Test 字段是否是真实的行为验证命令。这导致即使 Test 字段写了泛化命令（如 `npm test`）或缺少断言，也能通过 spec-review 审查。

## 目标

在 SKILL.md 的审查维度中增加对 Test 字段可执行性的强制检查（blocker 级别），确保每条 DoD 的 Test 字段是真实的行为验证命令。

## 变更内容

- 将原维度 D「测试命令可执行性」重命名为维度 E
- 新增维度 D「Test 字段可执行性验证（强制 blocker）」
- FAIL 条件新增「Test 字段质量不达标（维度 D 任一 blocker）」
- version 从 1.0.0 更新为 1.1.0
- changelog 新增 1.1.0 条目

## 成功标准

- SKILL.md 包含维度 D（Test 字段可执行性验证）和维度 E（测试命令可执行性）两个独立章节
- FAIL 条件包含 Test 字段质量不达标项
- version 为 1.1.0
