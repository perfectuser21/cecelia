# PRD: Codex 独立审查加固

## 背景

spec_review 和 code_review_gate 任务目前走常规 cecelia-bridge 10-slot 池，与编码任务竞争槽位，导致审查任务被背压阻塞。

## 目标

为审查类任务（spec_review / code_review_gate）建立独立执行路径，不占用常规 cecelia-bridge 槽位。

## 方案

1. 新增 `triggerLocalCodexExec` 函数，使用本机 `codex-bin exec` 执行审查任务
2. 独立 2-slot 池（`/tmp/codex-review-locks`），与 cecelia-bridge 10-slot 池完全隔离
3. 路由逻辑：spec_review / code_review_gate → triggerLocalCodexExec
4. cecelia-run.sh 槽位从 12 调整为 10（2 个槽位移交独立池）
5. SKILL.md 更新：从 PR diff 改为 git diff（本机执行时无 PR）

## 成功标准

- spec_review / code_review_gate 任务不再出现在 /tmp/cecelia-locks 中
- 审查任务完成后自动回调 Brain execution-callback
- 常规编码任务不受影响
