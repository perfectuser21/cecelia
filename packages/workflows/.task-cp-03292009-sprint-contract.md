# Task Card: 升级 /dev Sprint Contract 机制

## 需求（What & Why）

**功能描述**: 为 /dev Pipeline 的两个审查 Gate 添加 Sprint Contract（GAN 式双向对抗协商）和 Evaluator Calibration（few-shot 定锚），解决"自我认证"问题。

**背景**:
- Anthropic Harness Design 揭示了 Sprint Contract 的价值：Generator 和 Evaluator 必须对同一件事达成共识才能开始写代码
- 当前 spec_review 是单向审查（主 agent 写 DoD+Test，spec_review 只做检查），存在"礼貌性通过"风险（circular self-certification）
- 当前 code_review_gate 没有 calibration examples，Evaluator 判断标准缺乏基准，容易漂移
- 修复后：spec_review 能独立写测试方案并与主 agent 比对；code_review_gate 有 3 个定锚样例

**不做什么**:
- 不改变 /dev 整体流程（4 Stage Pipeline 不变）
- 不改变 spec_review 的现有维度 A-F 的检查逻辑
- 不改变 code_review_gate 的现有维度 A-H 的判断规则
- 不改变 01-spec.md 的 spec_review 重试逻辑（无限重试仍然有效）
- 不改变 Brain API 调用方式
- 不改变 seal 文件写入机制
