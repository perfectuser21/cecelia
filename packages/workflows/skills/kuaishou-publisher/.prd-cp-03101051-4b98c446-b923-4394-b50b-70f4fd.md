## PRD - 快手发布 API 方案（Cookie + HTTP 直接调用）

**需求来源**: Brain 自动调度 — [P0 KR1] 快手发布 API 接通（OAuth + 图文上传）
**参考实现**: packages/workflows/skills/weibo-publisher/scripts/publish-weibo-api.cjs
**功能描述**: 新增 `publish-kuaishou-api.cjs`，对标微博新 API 方案——通过 CDP 提取浏览器 Cookie，再通过 HTTP 直接调用快手创作者中心 API 上传图片并发布图文，避免 CDP 浏览器 UI 自动化的脆弱性。

**涉及文件**:
- 新增: `packages/workflows/skills/kuaishou-publisher/scripts/publish-kuaishou-api.cjs`
- 新增: `packages/workflows/skills/kuaishou-publisher/scripts/__tests__/publish-kuaishou-api.test.cjs`
- 修改: `packages/workflows/skills/kuaishou-publisher/SKILL.md`（记录新方案）

## 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| CDP 连接失败（Windows PC 关机/Chrome 未开） | 打印 `[CDP_ERROR]`，exit 1 | P1 | kuaishou_cdp_error |
| 会话 Cookie 缺失（未登录快手创作者中心） | 打印 `[SESSION_EXPIRED]`，exit 2 | P1 | kuaishou_session_expired |
| 图片上传失败（HTTP 4xx/5xx） | 3 次重试，全败则 exit 1 | P1 | kuaishou_upload_failed |
| 发布接口失败 | 打印错误详情，exit 1 | P1 | kuaishou_publish_failed |
| 内容目录无图片 | 打印使用说明，exit 1 | P3 | kuaishou_no_images |
