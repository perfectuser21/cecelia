# Task Card: 添加 Scope 层级（Project → Scope → Initiative）

## 需求（What & Why）
**功能描述**: 在 Project 和 Initiative 之间加入 Scope 层级。Scope 代表 2-3 天的功能边界分组，每个 Scope 下挂 3-7 个 Initiative（1-2 小时 pipeline）。采用行业 Shape Up 方法论的 Scope 概念 + SPIDR 拆分技术。

**背景**: 当前 Project（1周）直接拆解到 Initiative（1-2小时），跨度太大导致拆解粒度不够。业界最佳实践是加一层中间缓冲（Scope），让拆解分两步走：Project → 3-4 个 Scope → 每个 Scope 3-7 个 Initiative。

**不做什么**:
- 不改现有 Project/Initiative 的行为
- 不改 tasks 表结构
- 不改 goals 表结构
- 不改现有 API 端点的行为（只扩展）

## 成功标准
1. [ARTIFACT] projects 表支持 type='scope'，通过 parent_id 串联 Project→Scope→Initiative
2. [BEHAVIOR] Brain API create-project 能创建 type='scope' 的记录
3. [ARTIFACT] decomp SKILL.md 包含 Project→Scope 和 Scope→Initiative 拆解模板
4. [ARTIFACT] decomp-check SKILL.md 包含 Scope 层审查标准
5. [BEHAVIOR] decomposition-checker.js 能识别 Scope 层并触发下一步拆解
6. [GATE] CI 全部通过
