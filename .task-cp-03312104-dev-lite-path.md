# Task Card: /dev 流程 Lite 路径分流

## 需求（What & Why）
**功能描述**: 在 /dev 流程中实现小任务 vs 大任务分流。5 条 Lite 条件全部满足时走轻量路径（跳过 Planner subagent + Sprint Contract，主 agent 直接写 Task Card），否则走完整 Sprint Contract 路径。

**背景**: 当前所有任务无论规模大小都走完整 Sprint Contract 路径，导致 fix:/chore:/docs: 等小任务也需要经过 Planner subagent 和 Sprint Contract，增加不必要的流程开销。通过 5 条条件判断任务规模，对小任务提供轻量路径以提升开发效率。

**不做什么**:
- 不改变完整路径（FULL）的任何现有逻辑
- 不改 sprint-contract-loop.sh
- CI 门禁对轻量路径不豁免（L1-L4 CI checks 不绕过）

## 成功标准

1. [ARTIFACT] /dev 入口脚本新增 Lite 路径判断逻辑，输出 LITE 或 FULL 模式标记
2. [BEHAVIOR] 5 条 Lite 条件全满足时，/dev 跳过 Planner subagent 和 Sprint Contract，主 agent 直接写 Task Card（Test 字段直接填写，不是 TODO）
3. [BEHAVIOR] verify-step.sh 对 LITE 模式豁免 Sprint Contract seal 检查，但仍验证 Task Card 基础格式
4. [BEHAVIOR] devloop-check.sh 对 LITE 模式跳过 Sprint Contract 相关条件
5. [BEHAVIOR] Sprint Report 对 LITE 模式使用 25 分制，对 FULL 模式使用原有满分制，并在报告中标注 LITE/FULL
6. [GATE] CI 全部通过
