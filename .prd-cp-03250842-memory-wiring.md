# PRD: 记忆系统 PR7 — 接入断链

## 背景

PR6 实现了 suggestion-cycle.js，PR5 实现了 recordExpectedReward，但两者都没有接入运行主循环：
- tick.js 从未调用 runSuggestionCycle → desires→suggestions 路径永远不触发
- executor.js 派发任务时未调用 recordExpectedReward → rpe_signal 永远为 null

## 改动

### 1. tick.js 接入 runSuggestionCycle（10.18 节）

在 tick.js 的 10.x 区块末尾，fire-and-forget 调用 runSuggestionCycle：
- 频率：每 tick（5min），由 runSuggestionCycle 内部通过 getActiveDesiresForSuggestion 自动过滤
- 位置：10.17 之后，11 之前

### 2. executor.js 接入 recordExpectedReward

在 triggerCeceliaRun 中，`await updateTaskRunInfo(task.id, runId, 'triggered')` 之后，
fire-and-forget 调用 recordExpectedReward(task.id, taskType, skill)。
- 失败不阻塞任务派发（catch 静默）

## 成功标准

- tick.js import runSuggestionCycle 并在 10.18 节调用
- executor.js import recordExpectedReward 并在任务派发后 fire-and-forget 调用
- 现有 tick/executor 逻辑无破坏
