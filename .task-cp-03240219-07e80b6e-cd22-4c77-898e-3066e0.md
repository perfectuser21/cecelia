# Task Card: OKR PR11 — 清理旧表残留收尾

## 需求（What & Why）
**功能描述**: 清除 PR10 合并后仍残留的 4 处旧表（goals/projects）实际查询，使旧表可安全删除。
**背景**: 已有 objectives/key_results/visions/okr_projects/okr_scopes/okr_initiatives 7 张新 OKR 表替代旧表，但代码中仍有 4 处旧表读写路径未清理。
**不做什么**: 不删除旧表本身（schema 变更另立 migration）；不动 notion-full-sync.js（该文件是 Notion 旧数据同步专用，单独处理）。

- `grep -rn "FROM goals\|FROM projects\|UPDATE goals\|UPDATE projects\|INSERT INTO goals\|INSERT INTO projects" packages/brain/src/*.js packages/brain/src/services/*.js` 不返回生产代码中的查询（不含注释行）
- actions.js updateGoal 旧表 fallback 已替换为 throw Error
- planner.js projCheck 只查新表（okr_projects/okr_scopes/okr_initiatives）
- memory-service.js getDetail 中 FROM projects / FROM goals 改为查新表
- project-compare.js 两处 FROM projects 改为查 okr_projects（UNION 新表）

## 实现方案

**要改的文件**:
1. `packages/brain/src/actions.js` — 删除 lines 509-522（UPDATE goals SET fallback），改为 `throw new Error('Goal not found in any OKR table (objectives/key_results/visions)')`
2. `packages/brain/src/planner.js` — line 1191-1202：删除 projects 表查询，只保留新表 UNION 查询，移除 repo_path 约束
3. `packages/brain/src/services/memory-service.js` — `FROM projects WHERE id = $1` 改为查 okr_projects UNION okr_scopes UNION okr_initiatives；`FROM goals WHERE id = $1` 改为查 objectives UNION key_results
4. `packages/brain/src/project-compare.js` — `FROM projects p LEFT JOIN key_results` 改为 `FROM okr_projects p LEFT JOIN key_results`；`FROM projects WHERE id = ANY($1::uuid[])` 改为查 okr_projects UNION okr_scopes UNION okr_initiatives

**新表字段映射**:
- projects.name → okr_projects.title（AS name）
- projects.type → 硬编码 'project' AS type
- okr_projects 有 kr_id（同旧 projects），可直接 JOIN key_results
- description 字段：新表无此列，用 NULL AS description 替代
