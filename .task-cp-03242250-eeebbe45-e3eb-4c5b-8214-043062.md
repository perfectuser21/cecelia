# Task Card: 修复 self_drive_health 链路探针故障

## 需求（What & Why）

**功能描述**: 修复 `probeSelfDriveHealth` 探针持续失败的根本原因，恢复 self-drive 引擎正常运行，并补齐探针的判定逻辑使其对"LLM失败"和"无需行动"两种情况有准确区分。

**背景**:
- 根本原因 1（数据问题）: DB 中 `profile-anthropic` 的 `thalamus` 被意外修改为 `codex/gpt-5.4-mini`（高级模型），codex team OAuth 账号触发 quota 限制，每次 self-drive cycle 都报 `cycle_error`。
- 根本原因 2（探针逻辑）: `probeSelfDriveHealth` 只检查 `tasks_created > 0`，导致系统 LLM 正常运行但判断"无需行动"（`no_action`）时也被误判为失败。
- 影响: 探针失败 → 自动创建修复任务 → 循环产生无效任务。

**不做什么**: 不修改 codex 账号 quota 配置；不改变 self-drive 的核心 LLM 分析逻辑；不修改其他 agents 的模型配置（mouth/cortex 等由其他探针负责）。

## 成功标准

1. [ARTIFACT] migration `192_fix_thalamus_model.sql` 已创建，执行后 `profile-anthropic` 的 `thalamus` 重置为 `{provider: "anthropic", model: "claude-haiku-4-5-20251001"}`
2. [BEHAVIOR] `probeSelfDriveHealth` 在有 `cycle_complete` 或 `no_action` 事件时返回 `ok: true`，在全部为 `cycle_error` 时返回 `ok: false`
3. [ARTIFACT] 测试文件 `capability-probe-highlevel.test.js` 新增 self_drive_health 探针逻辑单测（覆盖成功/失败场景）
4. [GATE] CI 全部通过
