# Task Card: 实现 PR 自动合并闭环

## 需求（What & Why）
**功能描述**: 在 stop-dev.sh 和 devloop-check.sh 中，当 CI 全部通过 + Stage 4 Ship 已完成 + Learning 验证通过时，真正执行 `gh pr merge` 而不是仅输出 JSON 建议。
**背景**: 当前逻辑在所有条件满足后仍然输出 JSON 建议 + exit 2，导致 cecelia-run.sh 无限循环 sleep 2s 重试。
**不做什么**: 不修改 cecelia-run.sh 的循环逻辑，只修复 stop-dev.sh 和 devloop-check.sh 的合并执行。

## 成功标准
1. [BEHAVIOR] stop-dev.sh 在 CI 通过 + Stage 4 完成 + Learning 验证通过时，执行 gh pr merge --squash --delete-branch
2. [BEHAVIOR] devloop-check.sh 同样执行真正的 gh pr merge
3. [BEHAVIOR] 合并成功后 exit 0 / return 0
4. [BEHAVIOR] 合并失败后 exit 1（非 exit 2），避免无限循环
5. [GATE] CI 全部通过

## 验收条件（DoD）

- [ ] [ARTIFACT] stop-dev.sh 包含 gh pr merge 执行逻辑
  Test: manual: node -e "const c=require('fs').readFileSync('packages/engine/hooks/stop-dev.sh','utf8');if(!c.includes('gh pr merge'))process.exit(1)"

- [ ] [ARTIFACT] devloop-check.sh 包含 gh pr merge 执行逻辑
  Test: manual: node -e "const c=require('fs').readFileSync('packages/engine/lib/devloop-check.sh','utf8');if(!c.includes('gh pr merge'))process.exit(1)"

- [ ] [ARTIFACT] 合并失败时 exit 1 而非 exit 2
  Test: manual: node -e "const c=require('fs').readFileSync('packages/engine/hooks/stop-dev.sh','utf8');const lines=c.split('\n');const mergeIdx=lines.findIndex(l=>l.includes('gh pr merge'));const after=lines.slice(mergeIdx,mergeIdx+10).join('\n');if(!after.includes('exit 1'))process.exit(1)"

- [ ] [ARTIFACT] 合并成功后有 cleanup 逻辑
  Test: manual: node -e "const c=require('fs').readFileSync('packages/engine/hooks/stop-dev.sh','utf8');if(!c.includes('force_cleanup_worktree'))process.exit(1)"
