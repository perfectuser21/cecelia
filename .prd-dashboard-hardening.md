## PRD - CeceliaOverview 接入 Hardening Status

**需求来源**: Brain v1.7.0 上线了 6 个稳定性护栏，但前端 CeceliaOverview 仪表盘完全没有反映这些功能。操作者无法看到警觉等级、隔离区、待审批操作等关键信息。

**功能描述**: 在 CeceliaOverview 页面增加 Brain Hardening 状态面板，实时展示：
1. **Alertness 警觉等级** — 当前等级（NORMAL/ALERT/EMERGENCY/COMA）+ 分数 + 手动覆盖按钮
2. **Quarantine 隔离区** — 隔离任务数量 + 点击展开列表
3. **Pending Actions 待审批** — 待审批危险操作数量 + 审批/拒绝按钮
4. **Overall Status** — 整体健康状态（ok/warn/critical）作为顶部指示灯

**涉及文件**:
- `frontend/src/api/brain.api.ts` — 新增 API 方法（getHardeningStatus, getAlertness, getQuarantinedTasks, getPendingActions, approveAction, rejectAction, overrideAlertness）
- `frontend/src/features/core/brain/pages/CeceliaOverview.tsx` — 增加 Hardening 状态区域
- `frontend/src/features/core/brain/components/HardeningPanel.tsx` — 新组件：护栏状态面板
- `frontend/src/features/core/brain/components/AlertnessIndicator.tsx` — 新组件：警觉等级指示器

**成功标准**:
1. CeceliaOverview 页面显示 overall_status 指示灯（ok=绿/warn=黄/critical=红）
2. 显示当前 Alertness 等级和分数，可手动覆盖
3. 显示 Quarantine 任务数和 Pending Actions 数
4. 数据来源于 `GET /api/brain/hardening/status` 和 `GET /api/brain/alertness`
5. 所有数据自动轮询刷新（30s 间隔）
6. 无回归：现有 tick 状态、集群信息、任务列表不受影响

**非目标**:
- 不做独立的 Quarantine 管理页面（只显示数量）
- 不做独立的 Pending Actions 审批页面（只显示数量 + 简单操作）
- 不做 Failure Classification 分析图表
- 不做 Event Backlog 实时曲线
