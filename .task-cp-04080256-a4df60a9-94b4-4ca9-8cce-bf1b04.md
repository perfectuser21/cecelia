# Task Card: Auth失败账号自动剔除 — 防止级联quarantine

## 需求（What & Why）
**功能描述**: 在账号选择逻辑中加入 auth 失败熔断机制，当某账号 API key 失效（401）时，自动临时排除该账号 2 小时，新任务路由到可用账号。

**背景**:
根因分析显示，24h失败任务中 60% 是 auth 失败（186个quarantined）。account3 API key 在2026-04-06~07 期间失效，但 `selectBestAccount` 没有 auth 失败剔除机制，导致新任务持续派到失效账号形成级联失败。对比：spending cap 已有完整的 `markSpendingCap`/`isSpendingCapped` 机制，auth 失败缺少同等保护。

**不做什么**: 不修改 quarantine 逻辑、不触碰 `quarantine.js`；不自动重试 auth 失败的任务（auth 可能是凭据永久失效，需人工介入）

## 成功标准

- [ ] [ARTIFACT] `account-usage.js` 新增 `markAuthFailure`/`isAuthFailed`/`loadAuthFailuresFromDB` 三个函数
- [ ] [ARTIFACT] DB migration 添加 `is_auth_failed`/`auth_fail_resets_at` 列到 `account_usage_cache`
- [ ] [BEHAVIOR] `selectBestAccount` 调用时，`isAuthFailed` 账号被跳过，日志显示 `AUTH_FAILED` 标记
  Test: manual:node -e "const {selectBestAccount,markAuthFailure}=await import('./packages/brain/src/account-usage.js'); markAuthFailure('account3'); const r=await selectBestAccount(); if(r&&r.accountId==='account3')process.exit(1); console.log('✅ auth-failed account skipped, selected:', r?.accountId)"
- [ ] [BEHAVIOR] `execution.js` auth 失败时自动调用 `markAuthFailure(dispatched_account)`，日志包含 `[auth-circuit-breaker]`
  Test: manual:node -e "const f=require('fs').readFileSync('./packages/brain/src/routes/execution.js','utf8'); if(!f.includes('auth-circuit-breaker'))process.exit(1); console.log('✅ auth-circuit-breaker found')"

## 实现方案

**要改的文件**:
- `packages/brain/src/account-usage.js` — 添加 `_authFailureMap`、`markAuthFailure`、`isAuthFailed`、`loadAuthFailuresFromDB`；更新 `selectBestAccount` 添加 auth 失败检查
- `packages/brain/src/routes/execution.js` — auth failure_class 路径添加 `markAuthFailure` 调用
- `packages/brain/migrations/` — 新增 migration 添加 `is_auth_failed`/`auth_fail_resets_at` 列

**受影响函数**:
- `selectBestAccount()` — 新增 `isAuthFailed` 检查，日志新增 `AUTH_FAILED` 标记
- `isAccountEligibleForTier()` — 加入 auth 失败检查（或在 mapped 数据里加）
- execution.js `classifyFailure` → auth 路径 → `markAuthFailure`

**不改什么**: quarantine.js、self-drive.js、tick.js
