## 背景

诊断发现 Sprint Contract Gate 已形同虚设：
- CRG：12次审查中11次 stats 全零（{blocker:0,warning:0,info:0}），100% PASS
- spec_review：18次审查中12次 independent_test_plans 数组为空（Evaluator 没独立工作就直接通过）
- 两个 seal 文件的 `reviewer_model` 字段均为 `undefined`（字段从未被写入）

根本原因：缺少"必须有所发现"的最低压力规则，Evaluator 可以零观察通过。

## 修复范围

### 修复1：CRG (code-review-gate/SKILL.md v1.6.0 → v1.7.0)
- 添加 `reviewer_model` 字段到 seal 输出 JSON
- 添加规则：stats.blocker + stats.warning + stats.info = 0 时，必须补充至少1条 info 级观察，否则 exit 2

### 修复2：spec_review (spec-review/SKILL.md v1.5.0 → v1.6.0)
- 添加 `reviewer_model` 字段到 seal 输出 JSON
- 添加规则：Task Card 含 DoD 条目时，independent_test_plans.length 必须 > 0，否则 exit 2

### 修复3：Stage 1 caller (01-spec.md v3.1.0 → v3.2.0)
- Sprint Contract Gate PASS 后，额外验证 plans.length > 0；若为0且 Task Card 含 DoD → 回调 spec_review（视为 FAIL 重试）

### 修复4：Engine 版本 bump
- 13.68.0 → 13.69.0（6个文件）
