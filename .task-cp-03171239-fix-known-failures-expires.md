# Task Card: known-failures expiry 检查真正生效

## 需求（What & Why）

**功能描述**: 让 `check-rci-health.mjs` 的 known-failures expiry 检查真正有牙齿——当存在无 `expires` 字段的条目时从警告升级为错误。

**背景**:
- `check-rci-health.mjs` 第 88 行起有 known-failures expiry 检查逻辑
- `ci/known-failures.json` 里的条目已有 `expires` 字段（现有 2 条：`test:flaky:integration` expires 2026-04-01，`stop-hook-router-tests` expires 2026-04-15）
- 但 `checkKnownFailuresExpiry()` 函数在 `noExpiry`（缺少 expires 字段）情况下只输出 `⚠️` 警告，不设置 `hasErrors = true`
- 结果：expiry 检查代码存在，但"无 expires 字段"这个路径从未真正阻塞

**不做什么**:
- 不修改 `regression-contract.yaml`（该文件本身没有 known-failures 区块）
- 不修改 `ci/known-failures.json`（已有完整 expires 字段）
- 不修改 CI 的 continue-on-error（`ci-l1-process.yml` 中 RCI Health Check 步骤已无 continue-on-error）

## 成功标准

1. [ARTIFACT] `check-rci-health.mjs` 中 `noExpiry.length > 0` 时设置 `hasErrors = true` 并输出 `❌` 而非 `⚠️`
2. [BEHAVIOR] 当 `ci/known-failures.json` 有缺少 `expires` 字段的条目时，`node scripts/devgate/check-rci-health.mjs` 以 exit code 1 退出
3. [BEHAVIOR] 当所有条目都有 `expires` 字段时，检查仍然通过（不误报）
4. [GATE] CI 全部通过

## 验收条件（DoD）

- [x] [ARTIFACT] `check-rci-health.mjs` noExpiry 判断分支输出 `❌` 并设置 `hasErrors = true`
  Test: manual:bash -c "grep -A8 'noExpiry.length > 0' packages/engine/scripts/devgate/check-rci-health.mjs | grep 'hasErrors = true'"

- [x] [BEHAVIOR] 模拟缺少 expires 的条目，脚本 exit 1
  Test: manual:bash packages/engine/scripts/devgate/test-known-failures-expiry.sh

- [x] [BEHAVIOR] 当前 known-failures.json（2 条均有 expires）检查仍然通过
  Test: manual:bash -c "cd packages/engine && node scripts/devgate/check-rci-health.mjs --check known-failures 2>&1 | grep 'PASS'"

- [x] [GATE] 所有现有测试通过
  Test: manual:bash -c "cd packages/engine && node scripts/devgate/check-rci-health.mjs 2>&1 | tail -5"

## 实现方案

**要改的文件**:
- `packages/engine/scripts/devgate/check-rci-health.mjs` — 将 noExpiry 判断从 `⚠️` 警告改为 `❌` 错误并设置 `hasErrors = true`

**Scope 锚定**: 仅修改 `checkKnownFailuresExpiry()` 函数返回值处理逻辑（约第 210-215 行），不改其他逻辑。
