# Enriched PRD: 稳定 harness pipeline — 修复 84% verdict 丢失

## 背景（量化）
最近 50 个 harness_evaluate 任务：
- PASS: 4 (8%) / FAIL: 1 (2%) / verdict_timeout: 2 (4%)
- **result=null: 42 (84%)** ← 主要问题
- Agent 侧输出正常（/tmp/cecelia-*.log JSON 完整）
- DB 侧 status=completed，result=NULL，payload.last_run_result=NULL
- callback_queue 表有完整 result_json（表明 result 没进入 tasks 表）

## 问题（精准根因）
`cecelia-run.sh` 在调用 webhook **之前** 抢先 PATCH `/api/brain/action/update-task` 把 status 改为 completed。之后 `callback-processor` UPDATE 守卫 `WHERE status='in_progress'` 就失效了 → agent 的 result 从未写入 tasks.result / payload.last_run_result。

路径：
- `packages/brain/scripts/cecelia-run.sh:675-681` 抢跑 PATCH（只改 status，不带 result）
- `packages/brain/src/actions.js:341-345` updateTask 不处理 result 字段
- `packages/brain/src/callback-processor.js:188` + `routes/execution.js:279` 守卫 `WHERE status='in_progress'` 在已 completed 时 no-op

## 成功标准
- verdict_timeout + result=null 比例从 84% 降到 < 20%
- 验证: `curl 'localhost:5221/api/brain/tasks?task_type=harness_evaluate&limit=50'` result=null 或 verdict_timeout 占 < 10/50
- 连续 48h 不再出现 harness_evaluate completed 但 payload.last_run_result IS NULL 的记录
- 至少一条完整 Planner→Contract→Generator→Evaluator→PASS→merge 链路成功

## 方案（唯一）
移除 cecelia-run.sh 抢跑 PATCH，让 callback-processor/execution-callback 成为唯一 status 变更入口。同时把守卫从 `status='in_progress'` 改为 `status != 'completed'`（白名单更精确：`IN ('in_progress','queued','dispatched')`）。

## 方案细节
**文件 1**: `packages/brain/scripts/cecelia-run.sh:675-681` 删除如下块:
```bash
local task_status="failed"
[[ $exit_code -eq 0 ]] && task_status="completed"
curl -s -X POST "$CORE_API/api/brain/action/update-task" \
  -H "Content-Type: application/json" \
  -d "{\"task_id\":\"$TASK_ID\",\"status\":\"$task_status\",\"idempotency_key\":\"cecelia-run-${TASK_ID}-${CHECKPOINT_ID}\"}" \
  --max-time 5 >/dev/null 2>&1 || true
```
保留第 670-673 的 runs/$CHECKPOINT_ID/status PATCH（那写的是 cecelia_runs 不是 tasks）。

**文件 2**: `packages/brain/src/callback-processor.js:188` 和 `packages/brain/src/routes/execution.js:279` 守卫从:
```
WHERE id = $1 AND status = 'in_progress'
```
改为:
```
WHERE id = $1 AND status IN ('in_progress', 'queued', 'dispatched')
```

**文件 3**: `packages/brain/src/__tests__/callback-processor.test.js` 新增 test case:
构造 task status='in_progress' + callback with result={"type":"result","result":"{\"verdict\":\"PASS\"}"}, 断言 UPDATE 后 `tasks.result->>'verdict'='PASS'` 且 `payload->'last_run_result'` 存在。
