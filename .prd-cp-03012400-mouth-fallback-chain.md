# PRD: 嘴巴降级链 Sonnet → Haiku → MiniMax M2.5

## 背景

当前 mouth LLM 调用无降级：Sonnet 账号额度满时直接报错，Cecelia 回复「恍神」。
实验证明降级顺序应为：Sonnet（最佳）→ Haiku（稳，不会出戏）→ MiniMax M2.5（兜底）。

注：Haiku 虽然会偶尔出戏，但在已知 Sonnet 不可用的场景下比 MiniMax 更自然。

## 实现方案

### 1. model_profiles 加 fallbacks 字段

```json
{
  "mouth": {
    "model": "claude-sonnet-4-6",
    "provider": "anthropic",
    "fallbacks": [
      { "model": "claude-haiku-4-5-20251001", "provider": "anthropic" },
      { "model": "MiniMax-M2.5", "provider": "minimax" }
    ]
  }
}
```

### 2. account-usage.js 增加 selectBestAccountForHaiku()

Haiku 不占用 sonnet 配额，账号选择时只过滤 `five_hour_pct >= 80`，不过滤 `sevenDaySonnetPct`。

### 3. callLLM() 实现 fallback 逻辑

- 主模型失败（抛异常）→ 依次尝试 fallbacks
- 每次失败记录 warn 日志
- 所有选项都失败 → 抛出最后一个错误
- 返回值增加 `attempted_fallback: boolean`

## 范围

- 仅修改 mouth agent 的降级行为
- 其他 agent（thalamus/cortex/reflection）保持原有逻辑不变
