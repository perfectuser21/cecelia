# Task Card: Stage 2 DoD 本地强制验证

## 需求（What & Why）
**功能描述**: verify-step.sh 的 Stage 2 验证必须逐条执行 Task Card 中的 DoD Test 命令，不能只跑 npm test 就标记通过。
**背景**: 当前 verify-step.sh 的 verify_step2() 只检查文件改动和跑 Engine npm test，不执行 DoD Test 命令。AI 可以标记 step_2_code: done 但实际没验证 DoD。02-code.md 的 2.3.3 也只有伪代码注释没有真脚本。
**不做什么**: 不改 CI 端的 dod-execution-gate.sh，只改本地 verify-step.sh 和 02-code.md。

## 成功标准

1. [BEHAVIOR] verify-step.sh step2 在 Gate 1 npm test 之后，额外逐条执行 Task Card 中所有 [BEHAVIOR] 条目的 Test 命令
2. [BEHAVIOR] manual: 类型的 Test 命令被直接执行，失败时 verify-step 返回 exit 1
3. [BEHAVIOR] contract: 和 tests/ 类型的 Test 命令被正确处理（contract 跳过，tests/ 检查文件存在）
4. [ARTIFACT] 02-code.md 的 2.3.3 段落有真实的 verify-step.sh 调用说明，不再是伪代码
5. [GATE] 所有现有测试通过

## 验收条件（DoD）

- [x] [BEHAVIOR] verify-step.sh 包含 DoD 逐条执行逻辑（解析 Task Card [BEHAVIOR] 条目并执行 Test 命令）
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/verify-step.sh','utf8');if(!c.includes('IN_BEHAVIOR'))process.exit(1)"

- [x] [BEHAVIOR] manual: 类型 Test 命令失败时 verify-step 返回失败
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/verify-step.sh','utf8');if(!c.includes('FAILED_ITEMS'))process.exit(1)"

- [x] [BEHAVIOR] contract: 类型 Test 被标记为 DEFERRED 跳过
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/verify-step.sh','utf8');if(!c.includes('DEFERRED'))process.exit(1)"

- [x] [ARTIFACT] 02-code.md 的 2.3.3 包含 verify-step.sh 调用而非伪代码注释
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/skills/dev/steps/02-code.md','utf8');if(!c.includes('verify-step.sh'))process.exit(1)"

- [x] [GATE] 所有现有测试通过
  Test: manual:bash -c "cd packages/engine && npm test 2>&1 | tail -5"
