# Task Card: Pipeline Patrol 巡航模块

## 需求（What & Why）
**功能描述**: 实现 Pipeline Patrol 巡航模块（`packages/brain/src/pipeline-patrol.js`），作为 Brain tick 的新周期性模块，扫描所有活跃的 `.dev-mode` 文件，检测卡住的开发 pipeline，自动创建诊断/接管任务。

**背景**: 当前 Brain tick 有 zombie-sweep 和 health-monitor 等周期性模块，但缺少对 dev pipeline 本身的监控。当某个 agent 的 pipeline 在某个 stage 卡住（例如 CI 等待超时、代码阶段长时间无进展），没有自动检测和修复机制。Pipeline Patrol 填补这个空白。

**不做什么**:
- 不修改 devloop-check.sh 或 stop-dev.sh
- 不进入 pipeline 内部干预执行流程
- 不修改 .dev-mode 文件内容
- 只做外围观察和任务创建

## 成功标准
> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [ARTIFACT] `packages/brain/src/pipeline-patrol.js` 存在且 export `runPipelinePatrol(pool)` 函数
2. [BEHAVIOR] `runPipelinePatrol` 扫描主仓库和所有 worktree 的 `.dev-mode.*` 文件，返回诊断结果
3. [BEHAVIOR] 检测卡住的 pipeline（Stage 1/2 = 20min, Stage 3 = 90min, Stage 4 = 15min）并创建 `pipeline_rescue` 类型任务
4. [BEHAVIOR] 检测进程已死但 pipeline 未完成的情况，创建接管任务
5. [ARTIFACT] tick.js 导入并周期性调用 `runPipelinePatrol`（与 zombie-sweep 同级）
6. [GATE] 所有现有测试通过
