## PRD - Brain 派发任务时注入相关 Learning 上下文

**需求来源**: Brain 自动调度任务（agent_ops 领域）
**功能描述**: Brain 在派发任务（dispatch）时，从 learnings 表中检索与当前任务相关的历史 Learning，注入到 task 的 context/payload 中，供 agent 执行时参考，避免重复踩坑。
**涉及文件**:
- `packages/brain/src/executor.js` — dispatch 逻辑，注入 learning context
- `packages/brain/src/learning-retriever.js` — 新文件，关键词检索 learnings
- `packages/brain/src/migrations/` — 如需迁移（learnings 表已存在）
- `packages/brain/__tests__/learning-retriever.test.js` — 单元测试

**非目标**:
- 不做向量语义搜索（仅关键词匹配）
- 不修改 learnings 表结构（migration 151 已有 learning_type 字段）
- 不影响 agent 执行协议（仅丰富 context，不改参数格式）

## 成功标准

- dispatch 时调用 keywordSearchLearnings 获取相关 learning
- 注入的 learning 按 learning_type 分组展示
- 仅注入高相关性（score > 0.3）的 learning，最多 3 条
- 不影响现有 dispatch 性能（< 200ms overhead）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| learnings 表查询失败 | 降级：跳过注入，继续正常 dispatch | P3（仅日志）| learning_retrieval_failed |
| 检索超时（> 200ms）| 降级：跳过注入，记录慢查询 | P2 | learning_retrieval_timeout |
