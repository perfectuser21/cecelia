# Task Card: 系统韧性加固 — 自动重启 + OOM 防护 + vitest --related

## PRD

### 背景
2026-03-21 凌晨 kernel panic（AMCC OOM），重启后 PostgreSQL 17 和 Brain 均未自动拉起：
- Brain LaunchDaemon 处于 disabled 状态
- PG17 只有 ~/Library/LaunchAgents（用户级），SSH 重启后不触发
- slot-allocator 无内存维度，并发 dev 任务叠加 vitest → OOM → kernel panic 循环

### 目标
1. Brain LaunchDaemon（`/Library/LaunchDaemons/com.cecelia.brain.plist`）永久 enabled + KeepAlive
2. PG17 系统级 LaunchDaemon（`/Library/LaunchDaemons/homebrew.mxcl.postgresql@17.plist`）开机自启
3. Brain slot-allocator 新增内存压力维度：可用内存 < 600MB 时停止派发 dev 类任务
4. 提供一键修复脚本 `scripts/setup-autostart.sh`（文档化 + 可重放）

## 成功标准

- 机器重启后 PG17 + Brain 自动拉起，无需手动干预
- 可用内存低于阈值时 backpressure.active=true，dev 任务停止派发
- 有专门的 DoD 测试验证以上两点
