# PRD: DoD Execution Gate

> 详细内容见 Task Card: `.task-cp-03161230-dod-execution-gate.md`

## 功能描述

在 CI L1 中新增 `dod-execution-gate.sh` 脚本，实际执行 Task Card 中所有 `[BEHAVIOR]` 条目后跟的 `Test:` 命令，让 CI 不仅验证格式，还验证行为是否真实工作。

## 背景

当前 CI L1 的 `check-dod-mapping.cjs` 只检查 DoD Test 字段的格式（非空、有正确前缀），从不执行命令。开发者可以写出格式正确但逻辑永远失败的 Test，CI 也会绿灯放行。新功能可能根本没有工作。

## 成功标准

1. `packages/engine/scripts/devgate/dod-execution-gate.sh` 文件存在且可执行
2. CI L1 `dod-check` job 新增执行 `dod-execution-gate.sh` 的 step
3. 脚本对 `manual:bash` 类 Test 实际执行并验证结果
4. `manual:curl`、`manual:chrome` 类标记为 DEFERRED（跳过但不报错）
5. `Test: tests/` 类检查文件是否存在
6. 任何执行失败时 exit 1
