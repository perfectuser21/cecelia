# Task Card: 修复 KR 进度信号 — verifier FK 断裂 + 进度采集失效

## 需求（What & Why）

**功能描述**：多个 KR 长期保持 0% 进度，Dashboard 看板无法反映任务实际进展。

**根本原因（已排查）**：
PR #1475 将 `kr-verifier.js` 的 JOIN 从 `goals` 改为 `key_results`，但没有同步更新
`kr_verifiers.kr_id` 外键指向。造成两类断链：
1. **7 个 verifier 指向 archived key_results**：`kr-verifier.js` 过滤条件
   `AND g.status IN ('active', 'in_progress')` 将它们全部跳过，进度永远不更新。
2. **8 个 verifier 指向 goals-only IDs（不在 key_results 表中）**：
   `UPDATE key_results WHERE id = kr_id` 匹配 0 行，静默失败。
3. **7 个活跃 key_results 没有任何 verifier 链接到它们** → `key_results.progress = 0`。
4. Dashboard 读取 `key_results.progress` → 全部显示 0%。

**不做什么**：
- 不改 `kr-verifier.js` 代码（PR #1475 后逻辑已正确）
- 不修复 `okr_projects.kr_id = NULL` 问题（下一个 PR）
- 不修改 Dashboard 前端

## 成功标准

1. migration 183 文件存在，包含 FK 变更 + verifier 重新链接 + 进度回填
2. 迁移后 7 个活跃 key_results 各有 1 个启用的 verifier 指向它们
3. 迁移后 key_results.progress 反映 verifier 的 current_value/threshold（不再全 0）
4. CI 全部通过
