# PRD：重构 cecelia-routes 降低函数复杂度

## 背景

Brain ComplexityScanner 扫描 `packages/brain/src/cecelia-routes.js`，发现：
- `inferStepProgress` 圈复杂度：15（阈值：10）
- `buildCheckpoints` 圈复杂度：13（阈值：10）

注：`mapStatus` 函数已通过 PR #1353 修复扫描器误报，当前复杂度为 2（正常），无需修改。

## 目标

将 `inferStepProgress` 和 `buildCheckpoints` 的圈复杂度降至 10 以下，同时保持功能不变。

## 重构策略

### inferStepProgress（15 → ≤10）
- 提取 `parseStepIdx(stepStr, total)` 辅助函数：解析并验证步骤索引
- 将 failed/cancelled 分支逻辑提取到独立函数 `buildFailedProgress(stepStr, total)`
- 将 in_progress 分支逻辑提取到独立函数 `buildRunningProgress(stepStr, total)`

### buildCheckpoints（13 → ≤10）
- 提取 `resolveCheckpointStatus(status, stepNum, currentStepIdx)` 辅助函数
- 将 map 回调内的条件逻辑转移到辅助函数

## 成功标准
- `inferStepProgress` 复杂度 ≤10
- `buildCheckpoints` 复杂度 ≤10
- npm test 全部通过
