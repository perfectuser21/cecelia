# Task Card: YAML 验证器（启动时检查格式）（PR4）

## 需求（What & Why）
**功能描述**: 新增 content-type-validator.js，在 Brain 启动时（selfcheck 阶段）对 content-types/ 目录下所有 YAML 文件做格式校验，有问题时打印 WARN 日志而不阻断启动。
**背景**: PR1/PR3 已建立 YAML 注册表 + Pipeline 集成。现在需要一个轻量验证器，确保新增的 YAML 文件格式正确，避免 Pipeline 运行时才发现配置错误。
**不做什么**: 不引入 JSON Schema 库，不阻断启动，不修改 YAML 文件本身，不重复 registry.js 的 validateConfig（验证器独立实现，更细粒度）。

## 成功标准
> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [ARTIFACT] 新增 content-type-validator.js，export validateContentType + validateAllContentTypes
2. [BEHAVIOR] validateContentType 对缺少 content_type/images.count/template.generate_prompt 的配置返回含错误信息的 errors 数组
3. [BEHAVIOR] validateAllContentTypes 读取所有 YAML 并汇总校验结果
4. [BEHAVIOR] selfcheck.js 调用 validateAllContentTypes()，有错误时打印 [WARN] 日志（不影响 allPassed）
5. [GATE] 新增测试覆盖 validator，CI 全部通过

## 实现方案
**要改的文件**:
- 新增 packages/brain/src/content-types/content-type-validator.js
  - validateContentType(config): 检查 content_type/images.count/template.generate_prompt，返回 { valid, errors }
  - validateAllContentTypes(): 调用 listContentTypes + getContentType + validateContentType，汇总
- 修改 packages/brain/src/selfcheck.js
  - import validateAllContentTypes
  - runSelfCheck 中调用，有 errors 时 console.warn('[WARN] Content type validation...')
- 新增 packages/brain/src/__tests__/content-type-validator.test.js

**Scope 锚定**: 仅修改 selfcheck.js + 新增 validator.js + 新增测试
