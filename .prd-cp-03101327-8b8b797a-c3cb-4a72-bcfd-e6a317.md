# PRD - feat: 对比报告 Notion 导出

## 背景

PR #783 已实现 `POST /api/brain/projects/compare/report` 端点，返回包含各项目优劣势的结构化报告（JSON/Markdown）。Initiative 描述要求"可通过 Brain API 触发推送到 Notion 或飞书"，该功能尚未实现。

## 具体需求

### 1. Notion 导出

在 `POST /api/brain/projects/compare/report` 的请求体中新增可选字段：
```json
{
  "project_ids": [...],
  "format": "json" | "markdown",
  "export": "notion"
}
```

当 `export=notion` 时：
1. 生成对比报告（已有逻辑）
2. 将 Markdown 报告推送到 Notion（创建新 Page）
3. 返回字段中包含 `notion_url`

### 2. 实现要求
- 复用 `~/.credentials/notion.env` 中已有的 Notion API Token
- 在 `packages/brain/src/project-compare.js` 中添加 `exportToNotion(report)` 函数
- Notion Page 标题：`项目对比报告 - {generated_at}`
- Page 内容：Markdown 转换为 Notion Blocks
- 若 Notion export 失败，不影响主响应（降级返回原报告 + error 字段）

### 3. 成功标准

POST 请求带 `export: "notion"` 时，响应中包含 `notion_url` 字段。

## 不在本次 PR 范围
- 飞书推送（可作为后续 PR）
- 定时自动推送
- 历史报告存档

## 成功标准

- POST /api/brain/projects/compare/report 带 export:notion 时返回含 notion_url 的响应
- Notion 导出失败时降级返回原报告 + error 字段，不影响主流程
- 新增 exportToNotion 函数的单元测试
