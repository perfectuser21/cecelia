# Task Card: PR Review Fail-Closed + Stop Hook 孤儿无限打回

## 需求（What & Why）

**功能描述**：修复两个安全漏洞，将两处 fail-open 行为改为 fail-closed。

**背景**：
1. `pr-review.yml` 调用 OpenRouter API 失败时（网络超时、key 失效、503 等），`curl` fallback 到错误字符串，`detect-review-issues.js` 未检测到 🔴 → job 通过 → PR 在零审查情况下合并。需要：API 失败 = job 失败（fail-closed）+ 最多重试 3 次。
2. `stop-dev.sh` 检测到 `.dev-lock` 存在但 `.dev-mode` 丢失时，重试计数超过 5 次后执行 `exit 0` 放行 → cleanup 全跳。需要：删除整个 `_ORPHAN_COUNT -gt 5` if 块（含内部 exit 0），孤儿情况无上限永远 `exit 2`。

**不做什么**：
- 不改变 DeepSeek 的 review prompt 内容
- 不改变 stop-dev.sh 其他正常流程逻辑（cleanup_done 等路径不变）
- 不改变 detect-review-issues.js 的 🔴 检测逻辑

---

## 成功标准

1. [ARTIFACT] `pr-review.yml` 加入重试循环（`MAX_RETRY=3` + `RETRY_COUNT`），最多重试 3 次
2. [ARTIFACT] `pr-review.yml` 所有重试耗尽后检测 API 错误（无 `.choices` → `API_ERROR=true` → `exit 1`）
3. [ARTIFACT] `stop-dev.sh` 删除整个 `_ORPHAN_COUNT -gt 5` if 块（含内部 exit 0），孤儿情况无上限永远 `exit 2`

---
