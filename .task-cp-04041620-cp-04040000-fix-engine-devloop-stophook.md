# Task Card: 修复 Engine P0/P1 Bug — devloop-check + stop-dev 死代码

## 背景

Engine Stop Hook 流水线存在两处 bug：
1. **P0（devloop-check.sh）**：`step_4_ship == done` 时，正确调用了 `_mark_cleanup_done` 写入 `cleanup_done: true`，但紧接着输出 `status: "blocked"` + `return 2`，导致 Stop Hook 永远无法放行（cleanup_done 被写入后本应在下次检测时 return 0，但这段代码路径使下次检测也永远看不到）。
2. **P1（stop-dev.sh）**：会话隔离代码从 `.dev-mode` 文件读取 `branch/tty/session_id`，但 `.dev-mode` 文件中根本没有这些字段（它们在 `.dev-lock` 中），导致死代码永远判假，徒增误判风险。

## 改动清单

### 改动 1：`packages/engine/lib/devloop-check.sh` 第 244-247 行
**修复**：`step_4_status == "done"` 已调用 `_mark_cleanup_done` 后，改输出 `status: done` 并 `return 0`。

**之前**（有问题）：
```bash
_mark_cleanup_done "$dev_mode_file"
_devloop_jq -n '{"status":"blocked","reason":"Stage 4 Ship 已完成，cleanup_done 已标记，等待下次检查退出","action":"..."}'
return 2
```

**之后**（正确）：
```bash
_mark_cleanup_done "$dev_mode_file"
_devloop_jq -n '{"status":"done","reason":"Stage 4 Ship 已完成，cleanup_done 已标记，工作流结束"}'
return 0
```

### 改动 2：`packages/engine/hooks/stop-dev.sh` 第 116-128 行
**删除**：从 `.dev-mode` 读取 `branch/tty/session_id` 做会话隔离的死代码（L117-126），保留 `BRANCH_NAME` 赋值（L128），但改为直接从 `.dev-lock` 已读取的 `_lb` 变量获取。
