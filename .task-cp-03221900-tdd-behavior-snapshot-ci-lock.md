# Task Card: dev pipeline 补三个硬锁单元

## 需求（What & Why）
**功能描述**: 在 Stage 2 流程中补三个强制（exit 1）单元，弥补自动化编码路径的关键缺口。
**背景**: 当前 /dev pipeline 存在执行漂移问题——AI 执行 literal instruction 时会丢失"不能删什么"的 behavioral intent（如 TDD 在 Codex→subagent 迁移中被无声删除）。CI 镜像检查用 `|| true` 收尾导致失败被忽略。
**不做什么**: 不改 Stage 1/3/4，不改 devloop-check.sh，不改 stop hook。

## 成功标准

1. [ARTIFACT] `02-code.md` 新增 Step 2.0（行为快照，含 exit 1 硬锁）
2. [ARTIFACT] `02-code.md` 新增 Step 2.1.5（TDD先行，红灯→绿灯强制流程）
3. [BEHAVIOR] `verify-step.sh` 能识别并执行 `[PRESERVE]` 类型 DoD 条目
4. [BEHAVIOR] `02-code.md` 2.3.3 + 2.3.6 失败时 exit 1（不再 `|| true`）
5. [GATE] CI 全部通过，Engine 版本 bump
