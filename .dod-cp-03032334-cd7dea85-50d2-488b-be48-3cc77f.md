## 回滚计划

如果部署后发现问题：

1. **回滚 commit**：`git revert <commit-hash>`
2. **环境变量清空**：`unset TASK_TIMEOUT_OVERRIDES`
3. **重启 Brain**：`pm2 restart brain`

## 验证步骤（手动测试）

### 验证 1：超时配置生效

```bash
# 1. 启动 Brain（设置环境变量）
export TASK_TIMEOUT_OVERRIDES='{"initiative_plan":120}'
pm2 restart brain

# 2. 查看日志确认配置生效
tail -f /tmp/brain.log | grep -i timeout
# 应该看到：[tick] Timeout config: initiative_plan=120min, default=60min

# 3. 创建 initiative_plan 任务并等待 70 分钟
curl -X POST http://localhost:5221/api/brain/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "title": "测试超时配置",
    "task_type": "initiative_plan",
    "description": "测试任务",
    "priority": "P1",
    "project_id": "test-project-id",
    "goal_id": "test-goal-id"
  }'

# 4. 等待 70 分钟后检查任务状态（应该仍在 in_progress，未超时）
curl -s http://localhost:5221/api/brain/tasks/<task_id> | jq '.status'
# 预期：in_progress（而非 failed）
```

### 验证 2：Orphan 重试机制

```bash
# 1. 创建 initiative_plan 任务并派发
curl -X POST http://localhost:5221/api/brain/tasks/dispatch/<task_id>

# 2. 等待任务开始执行（status=in_progress）
curl -s http://localhost:5221/api/brain/tasks/<task_id> | jq '.status'

# 3. 强制重启 Brain（模拟崩溃）
pm2 restart brain

# 4. 等待 Brain 启动完成，检查任务状态
curl -s http://localhost:5221/api/brain/tasks/<task_id> | jq '.status'
# 预期：queued（被重新入队）

# 5. 检查日志
tail -f /tmp/brain.log | grep -i "orphan retry"
# 应该看到：[executor] orphan retry: task_id=xxx, type=initiative_plan
```

### 验证 3：日志增强

```bash
# 1. 创建并派发 initiative_plan 任务
curl -X POST http://localhost:5221/api/brain/tasks/dispatch/<task_id>

# 2. 实时查看日志
tail -f /tmp/brain.log

# 3. 验证启动日志
# 应该看到：[initiative_plan] start: initiative_id=xxx, kr_id=xxx

# 4. 如果任务超时，验证超时日志
# 应该看到：[initiative_plan] timeout: elapsed=XXm, limit=120m
```
