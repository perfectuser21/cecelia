# Task Card: 修复 brain-unit CI OOM — 去掉 continue-on-error，改 maxForks=1

## 需求（What & Why）
**功能描述**: brain-unit CI job 设置了 `continue-on-error: true` 导致 CI Gate 形同虚设——即使单元测试失败也不阻止 PR 合并。根本原因是 vitest 用了 2 个 fork 并发，内存消耗超出 ubuntu-latest 7GB 限制。
**背景**: OOM 是因为 `maxForks: 2` + `NODE_OPTIONS=3072MB`，改为 `maxForks: 1` + `NODE_OPTIONS=4096MB` 后串行运行，内存峰值大幅下降，可以移除 continue-on-error。
**不做什么**: 不拆分测试文件，不更换 runner，不改集成测试配置。

## 成功标准
> brain-unit CI job 在正常测试通过时返回 success（而非 continue-on-error 掩盖的 failure）

## 验收条件（DoD）

- [x] [BEHAVIOR] vitest.config.js 中 maxForks 改为 1
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/vitest.config.js','utf8');if(!c.includes('maxForks: 1'))process.exit(1);console.log('ok')"

- [x] [BEHAVIOR] ci.yml brain-unit job 不含 continue-on-error
  Test: manual:node -e "const c=require('fs').readFileSync('.github/workflows/ci.yml','utf8');if(c.includes('continue-on-error'))process.exit(1);console.log('ok')"

- [x] [ARTIFACT] ci.yml brain-unit NODE_OPTIONS 改为 4096
  Test: manual:node -e "const c=require('fs').readFileSync('.github/workflows/ci.yml','utf8');if(!c.includes('4096'))process.exit(1);console.log('ok')"

## 实现方案
**要改的文件**:
- `packages/brain/vitest.config.js` — poolOptions.forks.maxForks: 2 → 1
- `.github/workflows/ci.yml` — 删除 continue-on-error + 注释，NODE_OPTIONS 3072→4096，--maxWorkers=2→--maxForks=1

**不改什么**: 集成测试、排除列表、其他 CI job
