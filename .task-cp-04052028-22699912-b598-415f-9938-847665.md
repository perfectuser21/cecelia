# Task Card: 修复 brain-unit CI OOM — 去掉 continue-on-error，改 maxForks=1

## 需求（What & Why）
**功能描述**: brain-unit CI job 设置了 `continue-on-error: true` 导致 CI Gate 形同虚设——即使测试失败也不阻止 PR 合并。根本原因是 V8 堆碎片问题：vitest 跑完所有测试后 worker 崩溃（但测试本身全部通过）。
**背景**: `npm test` 脚本已有智能 OOM 处理：worker 崩溃但无测试失败时退出 0。CI 应采用同样策略，这样既能移除 continue-on-error，又能正确处理 worker 崩溃。
**不做什么**: 不拆分测试文件，不更换 runner，不修改各 Brain 模块定时器。

## 成功标准
> 移除 continue-on-error 且 CI gate 有效：真实测试失败会阻断 PR，OOM worker 崩溃不会

## 实现方案
**要改的文件**:
- `packages/brain/vitest.config.js` — poolOptions.forks.maxForks: 2 → 1
- `.github/workflows/ci.yml` — 删除 continue-on-error，NODE_OPTIONS 3072→6144，添加 OOM 智能处理脚本

**OOM 智能处理原理**:
- vitest worker 崩溃（V8堆碎片）时 exit code != 0
- 但所有测试本身已通过，输出中有 "Tests N passed"，无 "Tests N failed"
- 此时视为成功（exit 0），与 package.json test 脚本保持一致
- 真实测试失败（有 "failed" 输出）时仍返回非零 → CI gate 有效

**不改什么**: 集成测试、其他 CI job、Brain 模块定时器代码
