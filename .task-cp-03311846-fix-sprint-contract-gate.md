# Task Card: 修复 Sprint Contract Gate 检查缺失

## 需求（What & Why）

**功能描述**: verify-step.sh 的 step1 验证在判断 `step_1_spec: done` 之前，必须检查四个 Sprint Contract seal 文件均存在且内容有效，缺少任一文件或内容不合规则则 exit 1 拒绝通过。

**背景**: 当前 verify_step1() 只校验 DoD 中 Test 字段的格式是否合规，不验证 Sprint Contract 是否真实运行。主 agent 可以直接手填 Test 字段，绕过 Generator + Evaluator 对抗流程，导致 Sprint Contract 机制形同虚设。

**不做什么**:
- 不修改 Sprint Contract Loop 脚本（sprint-contract-loop.sh）的逻辑
- 不修改 Planner / Generator / Evaluator subagent 本身的产出格式
- 不修改 step2 及以后的验证步骤
- 不改变 DoD Test 字段格式校验的现有逻辑

## 成功标准

1. [ARTIFACT] `packages/engine/hooks/verify-step.sh` 中 `verify_step1()` 包含对四个 seal 文件的存在性与内容校验逻辑
2. [BEHAVIOR] 缺少 `.dev-gate-planner.{branch}` 时，`verify-step.sh step1` 以非零状态码退出
3. [BEHAVIOR] 缺少 `.dev-gate-generator-sprint.{branch}` 时，`verify-step.sh step1` 以非零状态码退出
4. [BEHAVIOR] `.dev-gate-spec.{branch}` 中 `verdict` 不为 `PASS` 或 `divergence_count` 为 0 时，`verify-step.sh step1` 以非零状态码退出（橡皮图章检测）
5. [BEHAVIOR] `.sprint-contract-state.{branch}` 中 `round` 为 0 时，`verify-step.sh step1` 以非零状态码退出
6. [BEHAVIOR] 四个文件均存在且内容合规的正常流程下，`verify-step.sh step1` 正常通过（exit 0）
7. [GATE] CI 全部通过
