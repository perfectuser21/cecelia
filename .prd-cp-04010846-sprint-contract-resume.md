# PRD: Sprint Contract 断点续跑

## 背景
上下文压缩发生在 Sprint Contract 两轮之间时，.sprint-contract-state 记录了 round，但 agent 上下文丢失不知道已到哪一轮，重新开始导致重复对抗。

## 目标
sprint-contract-loop.sh 每轮结束后写 checkpoint，恢复时读 .sprint-contract-state.{branch} 自动判断已完成几轮、从正确轮次继续，不重复已完成的轮次。支持 --resume 标志或自动检测。

## 成功标准

- sprint-contract-loop.sh 支持 --resume 标志从上次中断轮次继续
- 压缩重启后 agent 调用 sprint-contract-loop.sh --resume 即可续跑
- CI 通过

## 涉及文件
- packages/engine/scripts/devgate/sprint-contract-loop.sh
- packages/engine/skills/dev/steps/01-spec.md（Resume 说明）
