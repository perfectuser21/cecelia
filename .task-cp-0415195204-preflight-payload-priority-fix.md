# Task Card: pre-flight 读 payload.prd_summary + priority normalize

## 需求（What & Why）

**功能描述**: 修复 `packages/brain/src/pre-flight-check.js` 的两个漏洞，使 autonomous 创建的 dev 任务不再被误判 cancel：
1. `descContent` 加 `task.payload?.prd_summary` fallback（创建器把 PRD 实际写入 payload，而非 description/prd_content）
2. `priority` 在做 valid 校验前，先把常见非标值（normal/medium/low/high/urgent/critical）规范化到 P0/P1/P2

**背景**: 最近一周 autonomous /dev 通道产出为 0（最后合并 2026-04-08）。排查发现近 50 个 dev 任务 54% cancel 率，其中 P0 "选题池 v1"（payload.prd_summary 1237 字符）和 "Langfuse 接入"（priority=normal）均被 pre-flight 误拒。

**不做什么**:
- 不改 tick.js 调度逻辑
- 不改任务创建 API schema（POST /tasks 入口规范化留给后续 PR）
- 不回填已 cancel 的任务状态

## 实现方案

**要改的文件**:
- `packages/brain/src/pre-flight-check.js` — descContent fallback + priority normalize
- `packages/brain/src/__tests__/pre-flight-check.test.js` — 补 4 个单测

**受影响函数**: `preFlightCheck(task)`

**改动点**:
1. `descContent` 表达式改为 `task.description || task.prd_content || task.payload?.prd_summary`
2. 新增 `PRIORITY_NORMALIZE_MAP`（lowercase → P0/P1/P2），在 validPriorities 校验前规范化 `task.priority`

**不改什么**:
- 其他 check（title/skill/placeholder）不动
- 函数签名不变
- `getPreFlightStats` 不动
