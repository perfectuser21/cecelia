# Task Card: Hook-CI 耦合重构 R12 — verify-step.sh 调用 check-version-sync.sh

## 需求（What & Why）

**功能描述**: 在 `packages/engine/hooks/verify-step.sh` 的 Gate 0d（周边一致性扫描）末尾，追加调用 `packages/engine/ci/scripts/check-version-sync.sh` 的逻辑。当 PR 改动了 `packages/engine/` 下的版本文件时，自动执行全量版本同步检查，消除"本地通过但 CI 失败"的场景。

**背景**: 当前 Gate 0d 只做同目录 sibling 文件的版本一致性扫描，不会调用 CI 中使用的 `check-version-sync.sh` 脚本。这导致开发者在本地通过 verify-step.sh 之后，push 到 CI 时还会因版本文件不同步而失败，形成不必要的调试循环。

**不做什么**: 不改动其他文件；不修改 CI 脚本本身；不改动 check-version-sync.sh 的逻辑。

## 成功标准

1. `verify-step.sh` Gate 0d 在改动 `packages/engine/` 版本文件时调用 `check-version-sync.sh`
2. `jq` 不存在时优雅跳过（输出警告，不报错、不阻断构建）
3. Engine 版本 5 文件同步 bump（package.json / package-lock.json / VERSION / .hook-core-version / regression-contract.yaml）
4. CI L1-L4 全部通过
