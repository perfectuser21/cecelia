## PRD

### 背景

Cecelia 系统的 CI L1/L3 门禁目前只保证前端代码能编译，不保证功能在浏览器里真实可用。每次前端功能合并后，在浏览器手测时频繁发现 bug，导致多轮修复循环。

### 目标

在 L1 CI 加 `frontend-browser-dod-check` job：当 PR 包含前端文件改动时，强制要求 DoD 中含浏览器验证关键词（`manual:chrome:` / `localhost:5211`），否则拒绝合并。

## 成功标准

1. 前端 PR 缺少浏览器验证步骤时，L1 CI 以清晰错误拒绝合并
2. 前端 PR 含 `manual:chrome:` 或 `localhost:5211` 时，检查通过
3. 非前端 PR 不受影响，job 静默跳过
