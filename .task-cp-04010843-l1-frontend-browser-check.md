## PRD

### 背景

Cecelia 系统的 CI L1/L3 门禁目前只保证前端代码能编译（TypeCheck + Build），不保证功能在浏览器里真实可用。每次前端功能合并后，在浏览器手测时频繁发现 bug，导致多轮修复。

### 问题根源

没有强制要求：开发者在提交前必须在浏览器里验证过主流程。

### 目标

在 L1 CI 加一个新检查 job：当 PR 包含前端文件改动（`apps/dashboard/` 或 `apps/api/`）时，强制要求 DoD 中至少有一个 `[BEHAVIOR]` 条目，其 `Test:` 字段包含浏览器验证关键词（`manual:chrome:` 或 `localhost:5211` 或 `localhost:5210`）。没有则 CI 拒绝合并。

### 成功标准

1. 前端 PR 缺少浏览器验证步骤时，L1 CI 以清晰错误拒绝
2. 前端 PR 含浏览器验证步骤时，检查通过
3. 非前端 PR 不受影响
