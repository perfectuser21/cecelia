# PRD: 修复 callCodexHeadless OAuth 账号轮换

## 背景
`callCodexHeadless` 在调用 `codex exec` 时，强制注入了 `OPENAI_API_KEY` 环境变量，导致 codex CLI 使用直接 API 计费而非 OAuth 订阅账号（team1/team2），造成 API 额度意外消耗。

## 问题
```js
env: {
  ...process.env,
  CODEX_API_KEY: apiKey,     // 覆盖 OAuth token
  OPENAI_API_KEY: apiKey,    // 覆盖 OAuth token
}
```

## 目标
1. 移除 `OPENAI_API_KEY` 和 `CODEX_API_KEY` 的强制注入
2. 改用 `CODEX_HOME=~/.codex-teamX` 轮换 OAuth 账号
3. 支持 team1、team2 轮换（round-robin 或按使用情况）
4. 如果所有 team 账号不可用，fallback 到 API key（保持向后兼容）

## 验收
- `codex exec` 调用时使用 team OAuth 账号
- 不再消耗直接 OpenAI API 额度
- team1/team2 轮换正常
