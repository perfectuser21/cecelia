# PRD: 优化 L4 CI — 并行 shard + 删除 Phase 2

## 背景

L4 CI (`ci-l4-runtime.yml`) 当前存在两个性能问题：

1. **串行分片**：3 个 shard 通过 `for` 循环顺序执行，实际上 Mac mini 已有 4 个 runner 实例（runner-1/2/3/4）可以并行处理
2. **双重运行**：Phase 1（无 coverage 检测失败）+ Phase 2（带 coverage 检测门禁）= 同样的 382 个测试文件跑两次

## 目标

将 L4 单次运行时间从 60-90min 压缩到 ~20min：
1. 将串行 `for SHARD in 1/3 2/3 3/3` 改为 GitHub matrix 并行（3 个 job 同时跑）
2. 删除 Phase 2（"Run Brain coverage thresholds"）整个 step

## 成功标准

- `brain-integration` job 使用 `strategy.matrix.shard` 并行运行 3 个分片
- Phase 2 "Run Brain coverage thresholds" step 已删除
- CI 配置文件语法有效，L4 仍能正确检测新增测试失败
- baseline 容忍机制保留（`ci/brain-integration-baseline.txt`）
- GoldenPath E2E step 保留
