# PRD: crystallize 执行闭环 — 补全3个断线

## 背景
crystallize 任务类型引入了能力蒸馏4步流水线，但执行层有3个断线：
playwright-runner.sh 不回传 Brain，execution.js 不调用 advanceCrystallizeStage，
executor.js 不为 crystallize 使用 playwright-runner.sh runner 模式。

## 目标

### 缺口1：playwright-runner.sh 回传
- crystallize 子任务完成后 POST /api/brain/execution-callback（status=AI Done / AI Failed）
- findings 包含 script_path（forge阶段）或 verify_passed（verify阶段）

### 缺口2：execution.js pipeline 推进
- execution-callback 收到 crystallize_* 子任务完成时，调用 advanceCrystallizeStage
- 逻辑与 content-pipeline 的 advanceContentPipeline 调用块对称

### 缺口3：executor.js runner 模式
- triggerCodexBridge 对 crystallize 类型使用 runner 模式（playwright-runner.sh）
- 传入 --task-id 参数，与 codex_dev 用 runner.sh 的方式对称

- [ ] 任务完成
