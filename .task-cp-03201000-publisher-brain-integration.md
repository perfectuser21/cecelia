# Task Card: feat(publisher): 多平台发布调度器 v1

## 需求（What & Why）

**功能描述**: 将 Brain 与 zenithjoy 现有的 8 个平台发布脚本打通，实现 Brain 可以派发 `content_publish` 类型任务，自动路由到对应平台发布者（Claude Code skill），并通过 `content_publish_jobs` 表追踪发布任务状态。

**背景**: zenithjoy 已有 8 个平台发布脚本（抖音/快手/小红书/头条/微博/视频号/知乎/公众号），多个已测试通过，但与 Cecelia Brain 完全孤立——没有任务队列、没有状态追踪、无法实现每日自动发布。本 PR 是「发布流程集成（端到端）」Initiative 的第一块基石。

**不做什么**:
- 不实际运行发布脚本（那是 publisher skill 的职责）
- 不修改 zenithjoy 的发布脚本
- 不实现发布任务的自动触发调度（只做 Brain 的接入层）
- 不修改 `publish_results` 表（已存在，保留）

## 成功标准

1. [ARTIFACT] `content_publish_jobs` 表存在于 PostgreSQL（migration 163 执行成功）
2. [BEHAVIOR] `task_type=content_publish` + `payload.platform=douyin` 在 `getSkillForTaskType` 中返回 `/douyin-publisher`
3. [BEHAVIOR] `GET /api/brain/publish-jobs` 返回 HTTP 200 和 `{ success: true, jobs: [] }`
4. [BEHAVIOR] `POST /api/brain/publish-jobs` 创建 job 记录并返回 201
5. [GATE] CI 全部通过
