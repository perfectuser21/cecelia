# Task Card: 修复 KR 进度自动更新失效

## 需求（What & Why）
**功能描述**: 修复 `kr-progress.js` 的 fallback 进度更新链路，该链路从未真正写入 `key_results.progress`。
**背景**: OKR 进度 0% 问题。PR #2007 修复了读取侧（context.js 改读 `key_results.progress`），但写入侧的 fallback 链路（`updateKrProgress`）仍有 bug：计算了 progress 但只 UPDATE 了 `objectives.updated_at`，既是错误的表，也没有写 `progress` 字段。
**不做什么**: 不改 kr-verifier.js（主链路，工作正常）；不改 okr-tick.js。

## 根本原因（2 个 bug）

**Bug 1 — kr-progress.js:56**  
```sql
UPDATE objectives SET updated_at = NOW() WHERE id = $1
```
- 错误的表：应是 `key_results`（`okr_projects.kr_id → key_results.id`）
- 缺少字段：没有 `progress = $2`

**Bug 2 — kr-progress.js:72-75（syncAllKrProgress）**  
```sql
SELECT id FROM objectives WHERE ... UNION ALL SELECT id FROM key_results WHERE ...
```
- `objectives` 没有 `kr_id`，传入 `updateKrProgress` 永远查不到 projects → progress 恒为 0
- 还会覆盖 kr-verifier 已写入的进度（fallback 不应覆盖已有 verifier 的 KR）

## 成功标准

> [BEHAVIOR] `updateKrProgress` 正确写入 `key_results.progress`
> [BEHAVIOR] `syncAllKrProgress` 只查 `key_results`，跳过已有 verifier 的 KR
