# PRD: 接通 Proactive Mouth 主动沟通系统

## 背景

proactive-mouth.js 已实现（v1.170.0），但三个触发点尚未接通：
1. notifier.js 只有 Webhook 发送，没有 Open API 直接发私信
2. execution-callback 任务完成时没有调 notifyTaskCompletion
3. 欲望系统没有触发 expressDesire

现有基础设施：
- FEISHU_APP_ID + FEISHU_APP_SECRET 已配置
- routes.js 已有 tenant_access_token 获取逻辑
- person_models 表已有 Alex（open_id: ou_b472f4ce7a9bf227b9f6ad4e557fb687）

## 目标

让 Cecelia 能真正主动发飞书消息给 Alex，无需 Webhook。

## 功能需求

### F1: notifier.js 支持 Open API 发私信

当 FEISHU_BOT_WEBHOOK 未配置时，自动降级到 Open API 直接发私信：
- 获取 tenant_access_token（用 FEISHU_APP_ID + FEISHU_APP_SECRET）
- 从 person_models 查 Alex 的 open_id（或用环境变量 FEISHU_OWNER_OPEN_ID）
- POST /im/v1/messages?receive_id_type=open_id 发私信

### F2: execution-callback 接通任务完成通知

在 Brain 的 execution-callback 路由中，任务完成（status=completed）时：
- fire-and-forget 调用 notifyTaskCompletion(pool, callLLM, task)
- 包含任务标题、结果摘要

### F3: tick 里欲望驱动主动消息

在 tick.js 的 "表达欲望" 阶段，若有未表达的欲望且不是 explore 类型：
- fire-and-forget 调用 expressDesire(pool, callLLM, desireContent)
- 限制频率：只在 Alex 超过 2 小时未互动时才主动出声

## 成功标准

- Caramel 完成任务后，Alex 的飞书收到消息
- Cecelia 有欲望且 Alex 沉默 2h+ 时，飞书收到主动问候
