# Task Card: Cecelia Knowledge & Documentation System

## 需求（What & Why）

**功能描述**：在现有 Dashboard (apps/dashboard) 中构建统一文档系统，包含5个子模块：Dev Log、Decision Registry、Design Vault、Daily Diary、Knowledge Map。

**背景**：Cecelia 系统缺乏统一知识沉淀层。决策、PR记录、学习、设计方案散落在数据库/对话/文件中，每次新对话都从零开始认知。

**不做什么**：
- 不做历史 PR 数据回填（dev_records 从实施后开始积累）
- 不引入新的 UI 组件库（用已有 Tailwind + lucide-react）
- 不做完整 Markdown 编辑器（只读+备注）

## 成功标准

- 用户打开 `/knowledge/dev-log` 能看到历史 PR 完整记录页面正常渲染
- 用户打开 `/knowledge/diary` 能看到每日日报时间线页面正常渲染
- `dev_records`、`design_docs`、`user_annotations` 三张表已创建
- PR 合并后回调自动写入 dev_records（pr-callback-handler.js 扩展）
- 用户添加的标注（user_annotations）持久保存
- Brain API 3组新端点响应正常：`/api/brain/dev-records`、`/api/brain/design-docs`、`/api/brain/user-annotations`
- 每日 15:00 UTC（23:00 上海）tick 自动生成日报到 design_docs
- Brain CI + Workspace CI 全部通过
