# Task Card: Brain 侧强制 Harness worktree 分支使用 cp-* 规约

## 需求（What & Why）

**问题**：Harness `harness_task` Generator 跑在 Docker 容器里，经常在错误分支提交代码（`harness-v2/task-<uuid>`），CI `branch-naming` check 挂掉，每次都需要人工 rename。

**根因**：`packages/brain/src/harness-worktree.js` 在 `git clone` 后执行 `git checkout -b harness-v2/task-<shortid>`，产出不符合 `hooks/branch-protect.sh` 正则 `^cp-[0-9]{8,10}-[a-z0-9][a-z0-9_-]*$` 的分支名。SKILL.md 只是对 LLM 的"建议"，不可靠。

**方案**：分支命名强制收归 Brain — Brain 建 clone 时就直接 `checkout -b cp-MMDDHHMM-ws-<shortid>`，Generator 收到 worktree 已在合规分支。顺带让 Brain 在 spawn Generator 之前尝试 `rebase origin/main`，让 Generator 从最新 main 出发。

## 成功标准

1. [BEHAVIOR] `ensureHarnessWorktree` 产出分支名符合 `^cp-[0-9]{8,10}-[a-z0-9][a-z0-9_-]*$`
2. [BEHAVIOR] 分支名含上海时区 MMDDHHMM
3. [BEHAVIOR] `harness-v2/task-` 不再作为分支名出现（仅保留为目录路径前缀）
4. [BEHAVIOR] clone 完成后尝试 `fetch origin main` + `rebase origin/main`；rebase 冲突时不抛出，log 后 `rebase --abort`
5. [BEHAVIOR] 现存 worktree 若持有旧 `harness-v2/` 分支，复用时自动 `checkout -B` 到新 cp-* 分支
6. [GATE] 所有 harness-worktree 相关单测通过
