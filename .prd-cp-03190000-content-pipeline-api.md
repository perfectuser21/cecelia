# PRD: Brain Content Pipeline API 路由

## 背景

content-pipeline-orchestrator 和 tick 集成已就绪，但 Dashboard 缺少调用 Brain 的 API 端点。

## 成功标准

1. GET /api/brain/content-types — 返回所有已注册内容类型列表
2. POST /api/brain/pipelines — 创建 content-pipeline 任务（body: keyword, content_type, priority?）
3. GET /api/brain/pipelines — 分页列出 content-pipeline 任务（含状态）
4. server.js 注册新路由
5. CI 全部通过
