# Task Card: CI Scope 深度修复 — routing-map 补全 + ci-evolution-check 扩展扫描范围

## 需求（What & Why）

**功能描述**：
1. 在 `ci/routing-map.yml` 末尾新增 4 个子系统条目（devgate-core / ci-helpers / ci-configuration / github-workflows），涵盖 `scripts/devgate/`、`scripts/` 辅助脚本、`ci/` 目录和 `.github/workflows/`。
2. 扩展 `scripts/ci-evolution-check.mjs`，在现有 packages/* 和 apps/* 扫描基础上，追加对上述 3 个目录的注册校验（Check 4 / Check 5 / Check 6），确保向这些路径新增文件时 Evolution Gate 能检测到遗漏注册。

**背景**：
当前 CI Evolution Gate 只保护 `packages/*` 和 `apps/*` 子目录。`scripts/devgate/`、`ci/`、`.github/workflows/` 等关键路径缺乏覆盖，修改时不会触发门禁校验，存在注册遗漏风险。

**不做什么**：
- 不修改现有测试逻辑
- 不修改 CI yml 工作流文件本身
- 不改动 packages/* 或 apps/* 现有扫描逻辑
- 不升级 Engine 版本（本次改动不属于 Engine skills/hooks）

## 成功标准

1. [ARTIFACT] `ci/routing-map.yml` 新增 4 个条目：devgate-core / ci-helpers / ci-configuration / github-workflows
2. [ARTIFACT] `scripts/ci-evolution-check.mjs` 新增扫描逻辑，覆盖 `scripts/devgate/`、`ci/`、`.github/workflows/`
3. [BEHAVIOR] `node scripts/ci-evolution-check.mjs` 运行 exit 0，输出中可见新注册的子系统
4. [GATE] CI L1-L4 全部通过
