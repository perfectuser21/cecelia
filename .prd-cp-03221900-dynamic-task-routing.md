# PRD: 动态任务类型路由配置

## 背景

B类非编码 Codex 任务（strategy_session/knowledge/suggestion_plan/scope_plan/project_plan）的路由参数目前 hardcoded 在代码里，每次调整都需发 PR。

## 分层策略（铁律，不动）

- A类（dev/cecelia_run）：hardcoded US
- Coding pathway B类（code_review/decomp_review/initiative_plan/initiative_verify/arch_review/architecture_design/architecture_scan）：hardcoded US + 本机Codex
- 其余 Codex B类：DB 动态配置 ← 本需求

## 目标

把「其余 Codex B类」任务的 location/executor/skill 存入 DB，前台可直接调整，Brain 动态读取，无需重启。

## 成功标准

### 后端
- 新建 `task_type_configs` 表（migration 172）
- 种入5个动态类型：strategy_session/suggestion_plan/knowledge/scope_plan/project_plan
- executor.js 对动态类型改为读内存缓存（启动时从DB加载）
- LOCATION_MAP 中动态类型条目注释标注（不删除，作为 fallback）
- API：GET /api/brain/task-type-configs 和 PUT /api/brain/task-type-configs/:task_type
- PUT 后立即刷新内存缓存

### 前端
- 新页面 /task-type-configs（系统Hub下）
- 表格展示动态类型的 location/executor/skill
- 支持行内编辑 + 保存
- 保存后 Brain 立即生效（无需重启）
