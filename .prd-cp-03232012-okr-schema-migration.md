# PRD: OKR 体系统一 PR1 — 数据迁移补全

## 目标

添加 migration 179，将旧 `goals`/`projects` 表中尚未迁移的数据补入新的 OKR 层级表（visions/objectives/okr_projects/okr_scopes/okr_initiatives），并更新 selfcheck.js schema version 到 179。

## 背景

旧表（goals + projects）使用 type 字段区分层级，新表为每层独立表。大部分数据已通过其他机制迁入新表，migration 179 用 ON CONFLICT DO NOTHING 确保幂等补全剩余缺失行。

## 范围

- `packages/brain/migrations/179_okr_data_migration.sql`（新增）
- `packages/brain/src/selfcheck.js`（EXPECTED_SCHEMA_VERSION 178 → 179）

## 成功标准

- migration 文件存在
- 所有旧数据在新表中有对应行（ON CONFLICT DO NOTHING 幂等）
- selfcheck.js schema version = 179
- CI 通过
