# Task Card: autonomous 顺化 Day 1 — B1 + B2 + F2

## 需求

Day 1 止血包。22 个症状通过 Plan Agent 统筹成 6 Epic / 19 PR，本 PR 做三个独立且低成本的改动，合并一次 engine version bump 以省 CI 周期。

## 三个子改动

### B1 — stop-dev.sh self-heal 孪生 bug（line 53）
PR #2373 修了跨 session orphan 隔离的不对称；self-heal 块是同一 bug 的孪生兄弟——外层 `if [[ -n "${CLAUDE_SESSION_ID:-}" ]]; then` 导致 headless/nested 场景下自愈永远不触发。
修复：去掉该外层门控，内层三条所有权验证（owner_session / session_id / 无标识 + main HEAD == branch）自己把关；空 sid 场景下第三条自动生效。写 dev-lock 时 session_id 空值留空不伪造。

### B2 — 逃生 env `CECELIA_STOP_HOOK_BYPASS=1`
用户被 Stop Hook 反复 block 时没有显式逃生通道。在 stop-dev.sh 最顶部加 bypass 检查：`[[ "${CECELIA_STOP_HOOK_BYPASS:-}" == "1" ]]` 则 log + exit 0。

### F2 — 01-spec.md §0.2.5 Self-Review 补 Type consistency（Step 4）
Superpowers 官方 writing-plans Self-Review 有 4 项，我们砍到 3 项（漏了 Type consistency）。这是 Explore agent 今天深度对比发现的 Med 缺口。补第 4 步"跨 task 类型一致性扫描"。
