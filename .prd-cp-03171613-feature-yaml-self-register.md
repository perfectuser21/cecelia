# PRD: feature.yaml 自注册机制 — 消除 CI 5 处手动同步

## 背景

每加一个新功能，需要手动触碰 5 个地方：
1. `feature-registry.yml` — 登记功能
2. `regression-contract.yaml` — 登记合约
3. 版本号多文件 — 手动 bump
4. 手动跑 `generate-path-views.sh` — 同步派生文档
5. Learning 文件

从 124 个 Learning 文件分析，版本/registry 相关失败占 ~70% 的 CI 失败来源。

## 目标

每个功能目录放一个 `feature.yaml`，CI 扫描它自动完成验证。新功能加进来不需要改任何中央脚本。

## 本次 PR 范围

- 新建 `packages/engine/schemas/feature-yaml.schema.json`（JSON Schema 定义）
- 新建 `packages/engine/scripts/scan-feature-yamls.mjs`（扫描+验证器）
- 新建 `packages/engine/skills/{dev,audit,assurance,qa}/feature.yaml`（4 个 proof of concept）
- 新建 `packages/engine/hooks/pre-commit-path-views.sh`（自动触发 generate-path-views.sh）
- 修改 CI L2 `engine-l2` job 新增 `feature-yaml-lint` 步骤
