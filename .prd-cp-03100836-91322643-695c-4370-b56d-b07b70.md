## PRD - fix(selfcheck): 版本检查改为 >= 防止 migration 崩溃循环

**需求来源**: Brain 自动调度任务
**功能描述**: selfcheck.js 目前用精确匹配（=== EXPECTED_SCHEMA_VERSION），只要 DB 有更新的 migration，Brain 就拒绝启动进入崩溃循环。改为 >= 检查：DB 版本 >= 预期版本即通过。
**涉及文件**: packages/brain/src/selfcheck.js
**非目标**: 不修改 migration 文件、不改 EXPECTED_SCHEMA_VERSION 值

## 成功标准

- parseInt(dbVersion) >= parseInt(EXPECTED_SCHEMA_VERSION) 时 selfcheck 通过
- DB 有更新 migration（版本号大于预期）时 Brain 不再崩溃循环
- 现有测试（DB 版本 = 预期）继续通过
- 新增测试（DB 版本 > 预期）验证通过

## 异常处理

N/A — 逻辑变更，无新 IO 操作
