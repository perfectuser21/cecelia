# Task Card: 扩展 check-coverage-completeness 覆盖 Brain 代码

## 背景

`check-coverage-completeness.mjs` 目前只扫描 Engine 自身（hooks/devgate/src），不覆盖 Brain。Brain 有 151 个 src/*.js 文件，其中 13 个无对应测试，但 CI 对此一无所知。

Brain 测试结构与 Engine 不同：
- Engine：`tests/SCRIPT.test.ts`
- Brain：`src/__tests__/MODULE*.test.js`（支持前缀匹配）

## 实现范围

修改两个文件：
1. `packages/engine/scripts/devgate/check-coverage-completeness.mjs` — 新增 exported `checkBrainCoverage(brainRoot)` 函数和 Check 4 输出
2. `packages/engine/tests/devgate/check-coverage-completeness.test.ts` — 补充 Check 4 单元测试

新增 Check 4：Brain src 覆盖率扫描
- 扫描 `{brainRoot}/src/*.js`（排除 `.d.js`、`*.test.js`）
- 匹配规则：`{brainRoot}/src/__tests__/{basename}*.test.js` 存在则视为有测试（前缀匹配）
- `HIGH_RISK_BRAIN_MODULES = new Set(['tick','thalamus','executor','cortex','planner'])`
- 高风险模块缺测试 → `missingRequired`（hasErrors = true，exit 1）
- 其余模块缺测试 → `missingOptional`（hasWarnings = true，不 exit 1）
- 输出章节标题包含 "Brain src 覆盖" 字样

## 成功标准

- 脚本输出含 "Brain src 覆盖" 章节
- executor.js 通过前缀匹配识别为有测试
- 所有高风险模块有测试时 exit 0
- 高风险模块缺测试时 exit 1
