# Task Card: conversation-digest — Brain 自动读取 Claude Code 对话日志，提炼 decisions/ideas/questions 写入 DB

## 需求（What & Why）

**功能描述**: 用户与 Claude Code 的规划对话目前完全没有被捕获。对话日志以 `.jsonl` 格式机械写入 `~/.claude-account1/projects/{slug}/`，Brain 在同一台机器可直接读取。通过 Brain 后台异步分析，提炼 decisions/ideas/open_questions/tensions 写入 DB，不依赖 LLM 记忆或 prompt 注入。

**背景**: before-coding 的讨论（规划、判断、决策）是最有价值的上下文，但目前完全丢失。唯一可靠的解法是把机械写入的日志文件作为 artifact，由 Brain 独立处理。

**不做什么**: 不做实时捕获、不分析 tool use 输出、不处理非 cecelia 项目的日志。

**核心设计**:

conversation_log_cursors 表追踪处理进度（file_path + last_line_processed），
conversation-digest.js 读取 ~/.claude-account1/projects/ 下的 .jsonl 文件，
提取 human/assistant 消息对，满足阈值（≥8条 human 消息或最后消息>30分钟前）时
调用 Cortex 分析，结果写入 conversation_captures + decisions 表。
