# Phase 8.3：基础设施 — dev_reviews 表 + PR 行数阈值 API 化

## 背景

Phase 8.1（PR #2456）建立了两个新机制但落地不完整：

1. **Structured Review Block 规范**：5 个自审点（B-4/B-5/B-6/SDD-2/SDD-3）输出 markdown 格式 review block，**但只写进 design doc**——没有结构化存储、无法查询、无法追踪质量分趋势
2. **PR 行数双阈值**（软 200 / 硬 400）：**硬编码在 proxy.md 和 B-2 subagent prompt**——harness-planner 不读、capacity-budget API 不暴露，两处规则存在漂移风险

用户 2026-04-20 明确要求："Structured Review Block 规范落地到 Brain（Phase 8.3），替代'打分'口头约定"。

## 目标

**两件基础设施**：

1. `dev_reviews` 表 + API：把 proxy 生成的 markdown review block 解析入库，Dashboard 可查
2. PR LOC 阈值 API 化：capacity-budget 暴露阈值，harness-planner + proxy 都从这里读

## 成功标准

### 1. dev_reviews 表（Brain）

- **迁移脚本**：`packages/brain/migrations/XXX_dev_reviews.sql`（表 schema）
- **表字段**：
  - `id` SERIAL PRIMARY KEY
  - `pr_number` INTEGER
  - `branch` TEXT
  - `point_code` TEXT NOT NULL — 'B-4' / 'B-5' / 'B-6' / 'SDD-2' / 'SDD-3'
  - `decision` TEXT NOT NULL — APPROVE / REQUEST_CHANGES / PASS_WITH_CONCERNS
  - `confidence` TEXT NOT NULL — HIGH / MEDIUM / LOW
  - `quality_score` INTEGER NOT NULL — 0-10
  - `risks` JSONB — 数组 of {risk: text, impact: text}
  - `anchors_user_words` TEXT — 引用
  - `anchors_code` TEXT
  - `anchors_okr` TEXT
  - `next_step` TEXT
  - `raw_markdown` TEXT — 原始 markdown block
  - `created_at` TIMESTAMP DEFAULT NOW()
- **索引**：`(pr_number)`, `(point_code)`, `(created_at)`

### 2. Brain API

- `POST /api/brain/dev-reviews`
  - body: `{ pr_number?, branch, point_code, decision, confidence, quality_score, risks, anchors_*, next_step, raw_markdown }`
  - 返回：`{ id, created_at }`
- `GET /api/brain/dev-reviews?pr=N&point=B-5&limit=20`
  - 返回：数组 of review 记录
- `GET /api/brain/dev-reviews/stats`
  - 返回：按 point_code 聚合，`{ point_code, avg_quality: number, count: number, low_confidence_rate: number }`

### 3. Review Block 解析器

- `packages/brain/src/review-parser.js`：输入 markdown block → 输出结构化 JSON（匹配上述字段）
- 处理缺失字段 fallback（不抛异常，填 null）
- 单元测试覆盖：完整 block / 缺字段 / 格式错乱

### 4. capacity-budget API 暴露阈值

- 现有 `GET /api/brain/capacity-budget` 返回增加字段：
  ```json
  {
    "pr_loc_threshold": {
      "soft": 200,
      "hard": 400,
      "source": "industry-aligned-smartbear-microsoft-2006"
    },
    ...
  }
  ```
- 阈值定义在一个 SSOT 常量文件 `packages/brain/src/constants/pr-thresholds.js`
- 不读硬编码其他地方

### 5. harness-planner 引用阈值

- `packages/workflows/skills/harness-planner/` 里计算 workstream LOC 大小时读 capacity-budget 的 `pr_loc_threshold`
- 若 API 不可达 → fallback 200/400（代码里注明 fallback 是 API down 时临时用）

### 6. proxy.md B-2 改成读 API

- `packages/engine/skills/dev/steps/autonomous-research-proxy.md` 的 B-2 scope decomposition 规则：
  - 改成 "`curl localhost:5221/api/brain/capacity-budget` 拿 `pr_loc_threshold`"
  - Appendix A.B-2 prompt 同步改

### 7. 端到端测试

- Brain 侧：migration + API + parser 单测
- Engine 侧：`tests/integration/review-block-e2e.test.ts` — 跑完整流程：生成 block → POST → 解析入库 → GET 验证

### 8. 版本 bump

- Brain：minor（新表 + 新 API）patch 自动 bump
- Engine：18.1.1 → 18.2.0（minor，proxy B-2 引用机制变）
- Brain migration 走 schema-version 自检

## 不做

- 不做 Phase 8.2 的 10 个交互点
- 不做 Dashboard 可视化（dev_reviews 展示层是后续 PR）
