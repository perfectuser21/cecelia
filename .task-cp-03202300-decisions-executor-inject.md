# Task Card: executor 派发前注入 decisions 摘要

## 需求（What & Why）

**功能描述**: executor.js 派发任务时，自动查询 decisions 表中的活跃决策，按 category 分组汇总成简洁摘要，注入到 agent 的 prompt 中。

**背景**: decisions 表已有 13 条用户决策（执行器选择、优先级、产品方向等），但 agent 启动后对这些决策一无所知。每次对话用户都要重新说一遍。

**不做什么**: 不做决策的自动提取（写入端），只做读取端的注入。

## 成功标准

1. [ARTIFACT] 新增 `packages/brain/src/decisions-context.js` 模块
2. [BEHAVIOR] `preparePrompt()` 返回的 prompt 前自动拼接 decisions 摘要
3. [BEHAVIOR] 按 category 分组，user 和 cecelia 决策分开标注
4. [BEHAVIOR] 摘要不超过 500 字
5. [BEHAVIOR] 无活跃决策时不注入（不浪费 token）
6. [GATE] CI 全部通过
