# Task Card: 修复 content-pipeline 成功率崩溃（当前24%）

## 需求（What & Why）

**功能描述**：
修复导致所有 content-pipeline 任务失败的代码 Bug，恢复内容流水线正常运行。

**背景**：

诊断结果（2026-03-26）：
- Brain 近 1 小时成功率跌至 24%
- 根本原因：**代码 Bug，占 100%**（非任务拆分/Scanner/环境问题）
- 具体 Bug：`content-pipeline-orchestrator.js` 中 `_parsePipelineParams` 使用错误字段名 `payload?.keyword`，实际字段为 `payload?.pipeline_keyword`
- 连锁效应：
  1. keyword 回落到完整 title（如 `[内容流水线] 决策负担减半 2026-03-26`）
  2. `executeGenerate` 基于错误 keyword 生成极短内容（copy.md 仅 170 字符 < 300，article.md 仅 290 字符 < 1000）
  3. `executeReview` 因长度检查失败，review_passed = false
  4. 4次 review 全部失败，触发 MAX_REVIEW_RETRY=3，pipeline 标记 failed
- 受影响任务：10 个 content-pipeline（2026-03-26），全部失败

**失败根因分类**：
- 代码缺陷：100%
- 任务拆分不合理：0%
- Scanner 误判：0%
- 环境/资源问题：0%

**不做什么**：
- 不重跑历史失败任务（修复代码后新任务会自动运行正确）
- 不修改 MAX_REVIEW_RETRY（逻辑正确，是喂进去的 keyword 错误）
- 不修改 review 的质量标准（标准本身合理）

## 成功标准

1. [ARTIFACT] `_parsePipelineParams` 字段名修复为 `pipeline.payload?.pipeline_keyword || pipeline.payload?.keyword || pipeline.title`
2. [BEHAVIOR] `executeGenerate` 生成的 copy.md ≥ 300 字符、article.md ≥ 1000 字符（确保通过 review 检查）
3. [GATE] CI 全部通过
