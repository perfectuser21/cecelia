# Task Card: Brain memory-search 端点

## 需求（What & Why）

**功能描述**: 在 Brain routes.js 新增 `POST /api/brain/memory-search` 端点。接收 query 文本，调用 OpenAI embedding 生成向量，在 memory_stream 表做 pgvector 相似度搜索，返回最相关的记忆条目。

**背景**: Claude Code 对话时存在上下文盲点——Cecelia 大脑里有 7847 条 memory_stream 记录（含 embedding），但 Claude Code 无法查询。现有 `/search-similar` 端点搜索的是 tasks 表，不是 memory_stream。需要一个专用端点让 Claude Code 能语义检索 Cecelia 记忆。

**不做什么**: 不修改现有 `/search-similar` 端点；不改 memory_stream 表结构；不添加写入逻辑。

## 实现方案

**要改的文件**: `packages/brain/src/routes.js`（在 `/search-similar` 端点附近新增）

**依赖**: `generateEmbedding`（已有 `./openai-client.js`），`pool`（已有 `./db.js`）

**核心 SQL**:
```sql
SELECT content, memory_type, importance, summary,
       1 - (embedding <=> $1::vector) AS similarity
FROM memory_stream
WHERE embedding IS NOT NULL
  AND status = 'active'
ORDER BY embedding <=> $1::vector
LIMIT $2
```
