### 实现方案摘要

1. **迁移文件**: `packages/brain/migrations/115_system_reports.sql`
   - 表: `system_reports (id UUID PK, type TEXT, content JSONB, metadata JSONB, created_at TIMESTAMPTZ)`
   - 索引: `(type, created_at DESC)`

2. **tick.js**:
   - 添加 `_lastReportTime` 内存变量（类似 `_lastGoalEvalTime`）
   - 添加 `REPORT_INTERVAL_MS = 48 * 60 * 60 * 1000`
   - 添加 `check48hReport(pool)` 函数：检查时间差 → 生成 mock 简报 → 写入 DB
   - 在 executeTick() 末尾（10.x 区段）添加 fire-and-forget 调用

3. **routes.js**:
   - `GET /api/brain/reports` → 查询 system_reports 表，支持 ?type=&limit= 参数
   - `POST /api/brain/reports/trigger` → 立即触发 check48hReport()

4. **generate48hReport(pool)**:
   - 查询近 48h 任务统计（completed/failed/queued count）
   - 查询近 48h 告警事件（cecelia_events）
   - 组装 mock 简报 JSON
   - INSERT INTO system_reports
