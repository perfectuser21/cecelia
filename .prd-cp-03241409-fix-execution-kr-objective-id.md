# PRD: execution.js KR 断链修复 — strategy_session INSERT key_results 补 objective_id

## 问题

`packages/brain/src/routes/execution.js:1013` 的 `strategy_session` 回调 INSERT INTO key_results 时没有传 `objective_id`，导致创建的 KR 成为孤岛。

## 目标

- 从 task 的 `goal_id` 字段找到对应的 `objectives.id`（利用 migration 181 的 goals→objectives 同步：goals(area_okr).id = objectives.id）
- 将 `objective_id` 传入 key_results INSERT
- 当 goal_id 为 null 或不对应有效 objective 时，记录 WARN 日志（保持可插入，objective_id = null）

## 成功标准

1. strategy_session 完成后创建的 KR 带有 objective_id（当 task 有有效 goal_id 时）
2. 无 goal_id 或无效 goal_id 时，KR 插入成功并记录 WARN
3. CI 通过
