## PRD - Cortex insight → action 自动闭合机制

**需求来源**: Brain 自动调度（盲点来源：Cortex Insight 2026-03-11，id: 90be8b92）
**功能描述**: cortex.js 在记录 insight 后，自动分析内容是否涉及代码级修复，若是则自动生成 dev task 并写入 tasks 表，防止 insight 永远积累在 learnings 表但 applied=false
**涉及文件**:
- `packages/brain/src/cortex.js` — 记录 insight 后触发分析
- `packages/brain/src/insight-action-bridge.js` — 新文件：insight→task 桥接逻辑
- `packages/brain/src/__tests__/insight-action-bridge.test.js` — 集成测试

## 成功标准

- cortex_insight 类型的 learning 创建后，若内容含代码修复信号，自动生成 dev task
- 已有 task 的 insight 不重复触发（去重）
- applied=true 标记正确更新
- integration test 验证 insight→task 链路

**非目标**:
- 不做 UI 展示
- 不修改 thalamus 路由
- 不处理非 cortex_insight 类型

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| 生成 task 失败 | 记录错误，不影响 insight 写入 | P2 | insight_task_create_failed |
| 重复创建检测失败 | 降级：跳过创建，记录日志 | P3 | insight_dedup_check_failed |
| 内容分析 LLM 超时 | 跳过本次，applied 保持 false | P2 | insight_analysis_timeout |
