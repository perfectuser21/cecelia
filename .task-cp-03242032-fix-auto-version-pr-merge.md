# Task Card: fix(ci): Auto Version workflow — gh pr merge auto 失败修复

## 需求（What & Why）

**功能描述**：
Auto Version workflow（`.github/workflows/auto-version.yml`）在 push to main 后触发，
创建版本 PR 成功，但 `gh pr merge --auto --squash` 失败（exit code 1），导致 workflow 整体失败。

**根本原因分析**（调查结果）：
1. 日志显示 `git push -u origin` 成功，但没有任何 `gh pr create` 的输出（没有打印 PR URL）
2. 直接跳到 exit code 1，说明 `gh pr create` 本身失败了
3. 失败原因：`--body` 包含多行 commit message（中文 + 换行 + 特殊字符），
   在 bash `$(...)` 展开时导致 `gh pr create` 参数解析出错，
   或者前一次失败的同版本 run 已留下 PR，重复创建报错
4. 脚本没有对 `gh pr create` 的失败做任何处理（无 `|| true`、无 fallback）
5. `gh pr merge --auto` 本身也存在问题：PR 刚创建时 CI 还未开始，
   而 `--auto` 在没有 required checks 通过前有时会报错（取决于仓库规则）

**修复方案**（采用方案A + 健壮性修复）：
- 修复 `gh pr create`：改为用 heredoc 传入 `--body`，避免多行字符串展开问题
- 增加 PR 已存在的处理：`gh pr create || gh pr view ...`（idempotent 设计）
- `gh pr merge` 改为 `|| true`：auto-merge 请求即使失败也不应让整个 workflow 失败
  （PR 已创建就算成功，人工可以后续合并）
- 增加详细日志：打印 PR URL、merge 结果，方便排查

**不做什么**：
- 不改变版本 bump 逻辑（读取 commit message → semver bump）
- 不改变触发条件（push to main + packages/brain/**）

## 成功标准

1. Auto Version workflow 在 Brain 版本变化时能成功执行（不 exit 1）
2. 版本 PR 能被创建（已存在时不报错，幂等）
3. `gh pr merge --auto` 失败时 workflow 仍以 exit 0 结束（不阻断后续流程）
4. workflow 打印清晰的 PR URL 和操作结果

## 实现方案

**要改的文件**：`.github/workflows/auto-version.yml`（仅最后的 "Create version bump PR" step）

**Scope 锚定**：只改 `Create version bump PR` step 中的 PR 创建和合并命令，
不改 bump 逻辑、不改触发条件、不改文件更新逻辑。

**具体改动**：
1. `gh pr create --body "..."` → 改为 heredoc 传入 body（避免多行展开问题）
2. 增加 `gh pr create || gh pr view --json url -q .url` fallback（幂等）
3. `gh pr merge ... --auto` → 改为 `gh pr merge ... --auto || echo "auto-merge not available, PR created: $PR_URL"`
