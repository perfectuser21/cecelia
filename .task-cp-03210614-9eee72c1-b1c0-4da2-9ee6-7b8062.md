# Task Card: KR3 内容流水线闭合 — export → content_publish 任务创建

## 需求（What & Why）

**功能描述**：在 `content-pipeline-orchestrator.js` 的 `advanceContentPipeline` 中，当 `content-export` 子任务完成后，自动从 manifest 的 `platforms` 字段读取目标平台列表，为每个平台创建 `content_publish` 任务，最后再将 pipeline 标记为 `completed`。

**背景**：当前 `content-export` 完成后直接将 pipeline 标记 `completed`，但 `content_publish` 任务从未被创建。`executor.js` 已有完整的 8 平台路由逻辑（content_publish + platform → /douyin-publisher 等），缺少"上游触发"。KR3 要求选题→文案→素材→发布完整闭合。

**不做什么**：
- 不修改各平台 publisher skill
- 不改 executor.js 的路由逻辑（已完备）
- 不等待 publish 任务完成再 complete pipeline（fire-and-forget 模式）

## 成功标准

1. [ARTIFACT] `advanceContentPipeline` 在 `content-export` 完成时，为 manifest 中的每个 platform 创建 `content_publish` 任务
2. [BEHAVIOR] 8 个平台（douyin/kuaishou/xiaohongshu/toutiao/weibo/zhihu/wechat/shipinhao）的 `content_publish` 任务能正确路由到各自 publisher skill
3. [GATE] 所有现有测试通过，新增测试覆盖闭合逻辑
