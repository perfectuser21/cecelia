# Task Card: 内容生成+自动发布链路失效根因修复

## 需求（What & Why）

**功能描述**: 修复内容生成流水线的三个根本原因，恢复每日 ≥5 条内容产出能力，推进 KR 进度从 1% 到 30%+。

**背景**: Scanner 检测到内容生成 KR（65b4142d）和自动发布 KR（4b4d2262）均在 1% 进度停滞。诊断发现三个代码 Bug 导致：
1. `topic-selection-scheduler.js` 使用了已取消的 Objective ID（`fedab43c`）作为 `CONTENT_KR_GOAL_ID`，导致所有 content-pipeline 父任务被排除在 tick 的 `allGoalIds` 查询之外
2. Liveness probe 把内容流水线父任务（无对应进程）识别为"僵死"并 quarantine，导致子任务在父任务被杀后自我取消
3. 上述两个 bug 叠加后，12 条 content-image-review 任务全部完成但 ZERO content-export 任务被创建，最终阶段从未触达

**不做什么**: 不修改内容生成逻辑本身；不修改发布平台集成；不重构流水线架构

## 成功标准

> [ARTIFACT] `topic-selection-scheduler.js` 中 `CONTENT_KR_GOAL_ID` 更新为活跃 KR ID `65b4142d-242b-457d-abfa-c0c38037f1e9`
> [BEHAVIOR] 内容流水线父任务不再被 liveness probe 错误杀死
> [BEHAVIOR] content-export 任务能够被创建（流水线最终阶段可触达）
