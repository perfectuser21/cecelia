## PRD - [配额轮转-PR1] DB schema: tasks 表新增 quota_exhausted 状态 + migration

**需求来源**: Brain 自动调度任务
**功能描述**: 配额耗尽时任务被标记为 failed，计入 failure_count，触发隔离，污染真实失败率。需要独立状态区分"业务失败"和"资源限制"。为 tasks 表新增 quota_exhausted 作为独立任务状态，不计入 failure_count，不触发 FAILURE_THRESHOLD 隔离机制。
**涉及文件**:
- `packages/brain/migrations/NNN_add_quota_exhausted.sql` — 新增 migration
- `packages/brain/src/executor.js` — failure_count 逻辑排除 quota_exhausted
- `packages/brain/src/selfcheck.js` — 更新 EXPECTED_SCHEMA_VERSION
- `DEFINITION.md` — 同步 schema_version（两处）
- `packages/brain/__tests__/` — 新增单测

**成功标准**: 见下方

## 成功标准

- [ ] migration 执行后 tasks 表支持 status=quota_exhausted
- [ ] quota_exhausted 状态不增加 failure_count（executor.js 中 failure_count 逻辑排除该状态）
- [ ] 单测：quota_exhausted 状态转换不触发隔离逻辑
- [ ] facts-check.mjs 通过（schema_version 已同步）

**非目标**:
- 不实现配额检测逻辑本身（那是后续 PR）
- 不修改 Brain API 响应格式
