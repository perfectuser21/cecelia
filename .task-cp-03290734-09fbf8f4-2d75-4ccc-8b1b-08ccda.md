# Task Card: DoD Test 字段强度升级

## 需求（What & Why）

**功能描述**：升级 `check-dod-mapping.cjs` 的 Test 字段强度检查，并更新 `DOD-TEMPLATE.md` 模板示例。

**背景**：
- `check-fake-dod-tests.cjs` 能检测 ls/cat/true/exit 0 等弱测试，但 `check-dod-mapping.cjs` 的 `detectFakeTest` 函数缺少这些模式
- `isInlineCommand` 正则未覆盖弱命令，导致 `manual:ls src/` 落入 evidence 文件路径，错误信息不明确
- `curl localhost:*` 等 CI 环境中必然失败的命令未被拦截
- `DOD-TEMPLATE.md` 仍使用旧格式 `manual:code-review`（证据ID），不是可执行命令

**不做什么**：
- 不重构 `check-fake-dod-tests.cjs`（保留独立，仅增强 `check-dod-mapping.cjs`）
- 不改变 CI workflow 文件
- 不修改 `check-manual-cmd-whitelist.cjs`

## 成功标准

- `check-dod-mapping.cjs` 能拦截 ls/cat/true/exit 0 等弱测试命令
- `check-dod-mapping.cjs` 能拦截 `manual:curl localhost:*`（CI无服务器环境）
- `DOD-TEMPLATE.md` 中所有旧格式证据ID示例替换为可执行命令格式

## 实现方案

**要改的文件**：
1. `packages/engine/scripts/devgate/check-dod-mapping.cjs` — 增强 `detectFakeTest` + 扩展 `isInlineCommand` 正则
2. `packages/engine/templates/DOD-TEMPLATE.md` — 更新示例
3. `packages/engine/tests/ci/check-dod-mapping.test.ts` — 新建测试文件（feat: PR 需要测试文件）

**核心改动**：
- `detectFakeTest`: 添加弱命令模式（纯列目录、恒真命令等）+ CI anti-pattern（curl localhost/127.0.0.1）
- `isInlineCommand` 正则: 添加已知弱命令以触发 detectFakeTest 路径而非 evidence 路径
- `DOD-TEMPLATE.md`: 替换旧格式 `manual:code-review` 等为 `node -e "..."` 可执行命令

**Scope 锚定**：`packages/engine/scripts/devgate/` + `packages/engine/templates/` + `packages/engine/tests/ci/`
