# Task Card: 注册5个 content-* task_type 到 Brain

## 需求（What & Why）

**功能描述**: 在 Brain executor skillMap 和 task-router VALID_TASK_TYPES/SKILL_WHITELIST 中注册5个新的 task_type，使 Brain 能正确派发内容工厂 Pipeline 任务。

**背景**: 内容工厂 Pipeline 需要5个新 task_type（content-pipeline / content-research / content-generate / content-review / content-export）。Brain executor 目前不认识这些类型，会报 "unknown task_type" 错误或 fallback 到 /dev。

**不做什么**: 不实现实际内容工厂业务逻辑（那在 zenithjoy 仓库），只在 Brain 层做路由注册。

## 成功标准

1. [ARTIFACT] `executor.js` skillMap 包含5个新 task_type 条目
2. [ARTIFACT] `task-router.js` VALID_TASK_TYPES 包含5个新 task_type
3. [ARTIFACT] `task-router.js` SKILL_WHITELIST 包含5个新 task_type 映射
4. [BEHAVIOR] Brain 收到 content-pipeline task 时不再报 "unknown task_type"，正确路由到 /content-creator
5. [GATE] CI 全部通过（brain-ci.yml）

## 实现方案

**要改的文件**:
1. `packages/brain/src/executor.js` — `skillMap` 中新增5个 task_type 条目
2. `packages/brain/src/task-router.js` — `VALID_TASK_TYPES` 和 `SKILL_WHITELIST` 各新增5个条目

**Scope 锚定**: 只在两个文件的已有数组/对象字面量中追加条目，不改其他逻辑。
