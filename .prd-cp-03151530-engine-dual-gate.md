# PRD: Engine 双重 Gate 状态机 — Script Gate 层加深

## 背景

/dev 工作流每步之间应有双重 Gate（Script Gate + Subagent Gate）验证。当前 Script Gate 层过弱：`branch-protect.sh` 只检查 DoD 文件存在，不检查内容质量；`check-dod-mapping.cjs` 也缺少条目数和类型的最低要求。

## 目标

加深两个 Script Gate 的检查深度，使其能够在机器层面强制 DoD 质量要求。

## 成功标准

1. `branch-protect.sh` 写 Task Card 时：DoD 条目数 < 3 → 拒绝（exit 2）
2. `branch-protect.sh` 写 Task Card 时：无 `[BEHAVIOR]` 条目 → 拒绝（exit 2）
3. `check-dod-mapping.cjs` CI 阶段：DoD 条目数 < 3 → exit 1
4. `check-dod-mapping.cjs` CI 阶段：无 `[BEHAVIOR]` 条目 → exit 1

## 不做

- 不改 stop-dev.sh 步骤状态机
- 不改 Skill steps/*.md（那是 LLM 层）
- 不改其他现有检查逻辑
