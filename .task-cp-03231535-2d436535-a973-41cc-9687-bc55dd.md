# Task Card: CI Gate — manual: 命令白名单本地前置拦截

## 需求（What & Why）

**功能描述**：在 verify-step.sh 中增加 `manual:` 命令白名单前置检查，同时提供共享脚本 `check-manual-cmd-whitelist.cjs` 和 CI L1 job，使本地就能拦截 CI 不允许的 `manual:` 命令（如 grep/ls/cat），而不是等到 CI L1 才发现报错。

**背景**：CI L1 已有白名单检查机制（仅允许 node/npm/curl/bash/psql），但开发者只有 push 后才知道自己写的 `manual:grep` 这类 DoD Test 不合规。需要在本地 verify-step.sh 验证 Task Card 时就给出明确错误提示。

**不做什么**：不修改现有 fake-dod-tests 检查逻辑；不变更白名单内容（node/npm/curl/bash/psql 为既有规范）。

## 成功标准

- verify-step.sh 在 Task Card 含 `manual:grep` 等非白名单命令时，step1 验证失败并给出清晰提示
- `check-manual-cmd-whitelist.cjs` 作为独立共享脚本，既被 verify-step.sh 调用，也被 CI L1 job 调用
- CI L1 新增 `manual-cmd-whitelist-check` job，stage >= 2 触发
- 测试文件 `tests/devgate/check-manual-cmd-whitelist.test.ts` 覆盖率 ≥ 60%
