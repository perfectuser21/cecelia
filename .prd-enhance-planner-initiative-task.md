## PRD - 增强 Planner：识别有 Initiative 无 Task 的 KR 并自动派发

**需求来源**: Brain 自动调度任务
**功能描述**:
增强 packages/brain/src/planner.js，使其能识别「有 Initiative 无 Task」的 KR 并自动生成拆解任务。

当前问题：KR4 有 72 个 Initiative，但 7 天内 0 个 Task 派发。
根因：Planner 只关注有 queued 任务的项目，忽略了「有 Initiative 无 Task」的情况。

**涉及文件**:
- `packages/brain/src/planner.js` - 主要修改文件

**具体需求**:

### 1. 增强 scoreKRs()
新增判断：KR 下有 active Initiative 但无 queued Task → score += 15

### 2. 增强 selectTargetProject()
无 queued Task 时，优先选择 Initiative 项目（即有 initiative 子项目但无 queued task 的项目）

### 3. 新增 generateInitiativePlanTask()
自动生成 initiative_plan 任务来拆解 Initiative，生成对应 Task

**成功标准**:
- Planner 能识别「有 Initiative 无 Task」的 KR
- 自动生成 initiative_plan 类型任务
- 下一个 tick 后 KR4 有新任务进入队列
- 所有相关测试通过

**非目标**:
- 不修改 tick.js、executor.js 等其他 Brain 文件
- 不修改数据库 schema
- 不改变现有任务派发逻辑，只新增对「有 Initiative 无 Task」的处理
