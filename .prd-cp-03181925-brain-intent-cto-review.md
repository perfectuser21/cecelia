# PRD: Brain 意图扩展 + CTO Review 两阶段前置审查系统

## 背景
AI coding 存在意图损耗问题：用户说1，AI 交0.3。需要两个前置审查环节：
1. Codex A（意图扩展）：task 进来 → 查 project→KR→OKR→Vision 链路 → 补全 PRD → 写 enriched-prd
2. Codex B（CTO review）：代码写完后 → 读 enriched PRD + DoD + 核心 diff → 整体判断 PASS/FAIL

## 需要做的事

### Brain 侧
1. 新增 `intent_expand` task_type：路由到 us（本机执行），skill `/intent-expand`
2. 新增 `cto_review` task_type：路由到 xian（西安 Codex），skill `/cto-review`
3. 注册到 5 个文件：task-router.js、executor.js、model-registry.js、routes/tasks.js、DEFINITION.md
4. 新增 API 端点：
   - `POST /api/brain/tasks/:id/request-intent-expand`：创建 intent_expand 子任务，沿 project→KR→OKR→Vision 链查 context
   - `POST /api/brain/tasks/:id/request-cto-review`：创建 cto_review 子任务，接收 pr_number 参数

## 成功标准

- intent_expand 和 cto_review 已注册到所有 5 个文件
- 两个 API 端点可用，返回 `{success: true, task_id: "..."}`
- DEFINITION.md task_types 表格包含两个新类型
- 所有现有测试通过

## 不做什么
- 不新增 DB migration（enriched_prd 写入 tasks.metadata，review_result 是已有字段）
- 不修改 CI 门禁规则
- Engine 侧触发逻辑（Step1/Step2后触发）属于后续任务
