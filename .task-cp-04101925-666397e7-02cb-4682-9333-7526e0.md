# Task Card: proactiveTokenCheck 不清除 API 401 设置的 auth_failed

## 需求（What & Why）
**功能描述**: `proactiveTokenCheck` 当 token 文件有效时会清除所有 auth_failed 熔断，但 auth_failed 可能是 Anthropic API 返回 401 设置的（source=api_error），而不是 token 过期导致的。对于 api_error 的熔断不应被清除，否则同一账号会不断被派发、不断 401。

**背景**: 区分 auth_failed 的设置来源：`api_error`（API 401）vs `token_expired`（token 文件过期），proactiveTokenCheck 只清除 token_expired 来源的熔断。

**不做什么**: 不改变退避计数、不改变 DB schema、不改变 spending cap 逻辑。

## 成功标准

> [BEHAVIOR] api_error 设置的 auth_failed 在 proactiveTokenCheck 发现 token 有效时不被清除

## 实现方案
**要改的文件**: 
- `packages/brain/src/account-usage.js`（核心改动）
- `packages/brain/src/__tests__/account-usage-proactive.test.js` 或新增测试文件（扩展单测）

**受影响函数**:
- `markAuthFailure(accountId, resetTimeISO, source)` — 新增 source 参数
- `proactiveTokenCheck()` — 清除逻辑改为判断 source
- `proactiveTokenCheck()` 内 token 过期时的 `markAuthFailure` 调用 — 传 `"token_expired"`

**不改什么**: `execution.js` 中的调用不需要显式传 source（使用默认 `"api_error"` 即可）；DB schema 不变。
