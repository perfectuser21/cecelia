# Task Card: feat(brain): system_registry 表 + /api/brain/registry 接口

## 需求（What & Why）

**功能描述**: 在 Brain 加一张 `system_registry` 表，记录系统里所有东西（skill/cron/api/machine/integration）的位置和状态。提供 GET 查询接口和 exists 检查接口。

**背景**: Claude 创建任何东西前先查这里，彻底解决孤岛和重复问题。当前系统里 skill/cron 等分散注册，没有统一索引，导致重复创建和孤岛。

**不做什么**: 不做 UI 可视化；不修改现有模块的注册逻辑（只提供表结构和查询接口）。

## 实现方案

**要改的文件**:
1. `packages/brain/migrations/197_system_registry.sql` — 新建表
2. `packages/brain/src/routes/registry.js` — 新建路由
3. `packages/brain/server.js` — 挂载路由
4. `packages/brain/src/selfcheck.js` — 更新 EXPECTED_SCHEMA_VERSION 到 197

**表结构**:
```sql
CREATE TABLE system_registry (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  type TEXT NOT NULL CHECK (type IN ('skill','cron','api','machine','integration','other')),
  location TEXT,
  status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active','inactive','deprecated')),
  description TEXT,
  metadata JSONB DEFAULT '{}',
  registered_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE UNIQUE INDEX system_registry_name_type_idx ON system_registry (name, type);
```

**接口设计**:
- GET /api/brain/registry — 列表查询（?type=、?status=、?q= 关键词搜索、?limit=、?offset=）
- GET /api/brain/registry/exists — 检查存在性（?name=X&type=Y）
- POST /api/brain/registry — upsert（name+type 为唯一键）
- PATCH /api/brain/registry/:id — 更新 status/location/description/metadata
