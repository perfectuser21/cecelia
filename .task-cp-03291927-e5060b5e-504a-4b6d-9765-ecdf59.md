# Task Card: Planner→Generator→Evaluator 链路内容注入 + divergence 下限

## 需求（What & Why）

**功能描述**: 强化 Planner→Generator→Evaluator 三阶段链路的真实有效性，通过两个改动实现：
1. 主 agent 在 spawn Generator subagent 和 Evaluator(spec_review) subagent 时，直接将 Planner 产出的 Task Card + Sprint Contract 内容嵌入 prompt，而非让 subagent 自己读文件；
2. spec_review subagent 完成后，Sprint Contract Gate 检查 divergence_count >= 1，若为 0 则认为 Evaluator 未真实挑战，直接 exit 2 要求重跑。

**背景**: 当前链路中 subagent 依赖自行读文件获取上下文，导致内容感知不可靠；Evaluator 可能零分歧通过，无法保证其真实审查了 Generator 的输出。

**不做什么**:
- 不改变三阶段流程的整体结构
- 不修改 Stage 0/1/3/4 的逻辑
- 不改变 divergence_count 的计算方式，只在 Gate 层加下限检查

## 成功标准

1. [ARTIFACT] `packages/engine/skills/dev/steps/02-code.md` 的 Generator prompt 模板含 `{SPRINT_CONTRACT_CONTENT}` 占位符
2. [ARTIFACT] `packages/workflows/skills/spec-review/SKILL.md` 含"由调用方注入"说明以及 divergence_count == 0 的 exit 2 规则
3. [BEHAVIOR] Generator subagent 被 spawn 时，其 prompt 已含 Sprint Contract 实际内容（通过占位符替换注入）
4. [BEHAVIOR] spec_review subagent 完成后，若 divergence_count == 0，Sprint Contract Gate 以 exit 2 终止提示重跑
5. [GATE] CI 全部通过
