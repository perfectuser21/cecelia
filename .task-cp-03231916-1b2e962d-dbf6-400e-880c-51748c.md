# Task Card: OKR 数据迁移 — 旧 goals/projects 搬入新 okr_* 表

## 需求（What & Why）

**功能描述**:
写数据迁移 SQL（migration 179），把旧表（goals/projects）的数据完整搬入新建的 7 张独立 OKR 层级表。

**背景**:
- 旧体系：goals（type=vision/area_okr）+ projects（type=project/scope/initiative）存 OKR 数据
- 新体系（migration 177 已建好表结构）：visions/objectives/key_results/okr_projects/okr_scopes/okr_initiatives
- migration 177 已部分迁移活跃数据（1条 vision→visions，7条 area_okr→key_results 误放）
- 本 PR 目标：完整迁移所有数据，ON CONFLICT DO NOTHING 保证幂等性

**实际数据量（已探索）**:
- goals.type=vision: 4 条 → visions
- goals.type=area_okr: 164 条 → objectives
- projects.type=project: 48 条 → okr_projects
- projects.type=scope: 24 条 → okr_scopes
- projects.type=initiative: 152 条 → okr_initiatives

**不做什么**:
- 不删旧表（留给 PR5）
- 不修复 migration 177 的 area_okr→key_results 误迁（留给后续 PR）
- 不修改 tasks 表 FK（PRD 明确不需要）

## 实现方案

**要改的文件**:
- `packages/brain/migrations/179_okr_data_migration.sql`（新建）

**列映射规则**:
- goals → visions: id→id, title→title, area_id→area_id, owner_role→owner_role, target_date→end_date, metadata→metadata, custom_props→custom_props
- goals → objectives: id→id, title→title, parent_id→vision_id（当 parent 在 visions 中，否则 NULL）, area_id, owner_role, target_date→end_date, metadata, custom_props
- projects → okr_projects: id→id, name→title, status→status, area_id, owner_role, deadline→end_date, metadata, custom_props; kr_id=NULL（goal_id 指向 area_okr，area_okr 在 objectives 不在 key_results）
- projects → okr_scopes: id→id, name→title, parent_id→project_id（当 parent.type='project'）, area_id, owner_role, deadline→end_date
- projects → okr_initiatives: id→id, name→title, parent_id→scope_id（当 parent.type='scope'，否则 NULL）, area_id, owner_role, deadline→end_date

**FK 边界处理**:
- objectives.vision_id: 仅当 parent_id 存在于 visions 时设值，否则 NULL（FK nullable）
- okr_initiatives.scope_id: 仅当 parent.type='scope' 时设值；parent.type='project' 的 105 条 initiative scope_id=NULL
- 旧表添加 COMMENT ON TABLE ... IS 'DEPRECATED: ...'
