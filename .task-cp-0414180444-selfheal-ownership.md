# Task Card: self-heal over-heal 修复 — session 所有权验证

## 需求
v16.7.0 的 self-heal 无差别愈合所有活跃 worktree 的 orphan，后台 Harness 任务的 dev-mode 会被当前 session 误愈 → 接手别人的任务。加所有权验证。

## 成功标准
- v16.8.0：self-heal 只愈合"能证明属于当前 session"的 dev-mode
- 证明条件（任一满足即愈合）：
  1. dev-mode 含 `owner_session: $CLAUDE_SESSION_ID`
  2. dev-mode 含 `session_id: $CLAUDE_SESSION_ID`（兼容老格式）
  3. dev-mode 无 owner/session 标识 且 worktree 的 HEAD 与 `git -C main branch --show-current` 匹配（同 shell 创建）
- 任一不满足 → 跳过（orphan 检测走 T2 session_id 隔离）
- 00-worktree-auto.md: /dev 创建 dev-mode 时写 `owner_session: $CLAUDE_SESSION_ID`
- 3 场景测试：own-session / other-session / no-owner + HEAD 匹配
