# PRD — fix(brain): 修复 circuit_breaker.status 在 HALF_OPEN 时显示 all_closed 的误导性问题

## 背景
Brain API degraded 诊断任务（任务ID: 1cb3596c-fa9b-4d52-a721-89b6c6f46614）发现：
`cecelia-run` 熔断器处于 HALF_OPEN（恢复中）状态时，`GET /api/brain/health` 仍返回
`circuit_breaker.status: "all_closed"`，使运维人员和 CURRENT_STATE.md 脚本无法准确判断系统恢复状态。

## 问题
- `openBreakers` 只过滤 OPEN 状态，HALF_OPEN 不被包含
- 导致 `circuit_breaker.status: "all_closed"` 和 `states.cecelia-run.state: "HALF_OPEN"` 矛盾
- `open: []` 为空但实际有 HALF_OPEN 熔断器，运维人员无法看出系统正在恢复

## 修复范围
- `packages/brain/src/routes/goals.js`（仅此一个文件）
- 添加 `halfOpenBreakers` 检测
- 更新 `circuit_breaker.status` 逻辑：all_closed / recovering / has_open
- 添加 `circuit_breaker.half_open` 数组

## 成功标准
当 cecelia-run 处于 HALF_OPEN 时，health API 返回 `circuit_breaker.status: "recovering"` 和 `half_open: ["cecelia-run"]`
