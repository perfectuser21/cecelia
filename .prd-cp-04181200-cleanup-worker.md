# PRD: R4 — 孤儿 worktree 自动清理 worker

## 背景
用 Agent() tool + isolation:"worktree" spawn 的 sub-agent 做 /dev 完事后就退出（创完 PR + 挂 auto-merge），**没有人负责 PR 合并后调 cleanup.sh**。积累 3 个孤儿 worktree（agent-a6a30b0d R1#2397 / agent-ac87b750 memory-health#2396 / agent-af0765fb R2#2398，均已 merged 但 worktree 还在）。标准 /dev 的 Stop Hook + devloop-check 在 PR merge 时会 cleanup，Agent() spawn 跑完就结束，Stop Hook 不触发。

## 目标
独立 orphan worktree 清理 worker：周期扫描 → 检测 PR 已 merged → 满足安全守卫 → 清理 worktree + 本地 branch。

## 范围

### 1. packages/brain/scripts/cleanup-merged-worktrees.sh（新）
- 扫描白名单 glob：`.claude/worktrees/agent-*`、`~/worktrees/cecelia/*`
- 对每个 worktree 做 guard 检查 + 清理
- 纯 bash，可独立运行，支持 DRY_RUN=1

### 2. 安全守卫 Guard A~E（clean 前必须全通过）
- **A**：无 uncommitted changes（`git status --porcelain` 空）
- **B**：无 unpushed commits（`origin/<branch>..HEAD` count = 0）
- **C**：PR merged_at > GRACE_SECONDS（默认 3600s，避免竞态）
- **D**：branch 不是 main/master/develop
- **E**：dev-mode 文件里无 `human_hold: true`

### 3. 清理动作
1. `git worktree unlock`
2. `git worktree remove --force` 失败则 `rm -rf + worktree prune`
3. `git branch -D <branch>`

### 4. 集成
packages/brain/src/cleanup-worker.js 调 shell，tick.js 每 10min 触发一次（`CECELIA_CLEANUP_WORKER_INTERVAL_MS` 可覆盖），`MINIMAL_MODE` 跳过。

### 5. 测试
- packages/brain/src/__tests__/cleanup-worker.test.js：mock exec 验证 5 场景
- DRY_RUN=1 冒烟脚本

### 6. Engine 14.17.2 → 14.17.3（feat，新增 Brain worker）+ feature-registry changelog

## 不做
- 不清理其他 session 的 worktree（白名单严格限定）
- 不清未 merged PR
- 不关未 merged PR
- 不做 Dashboard UI
- 不做 GitHub webhook

## 成功标准

- 脚本 DRY_RUN=1 跑 repo 所有 worktree，每个输出 [skip reason] 或 [would-cleanup]，无误清
- 当前 worktree（PR 未 merged）必然 skip
- merged 但 < 1h 的 worktree 报 grace period skip
- vitest 5 test 全 pass
- Engine 版本六文件同步 14.17.3
