# Task Card: 补充 INFRA 能力元数据，减少白名单依赖

## 需求（What & Why）

**功能描述**:
PR #1463 已为 capability-scanner 添加了 `INFRA_DEPLOYED_CAPABILITIES` 白名单（9个能力）和 `BRAIN_ALWAYS_ACTIVE` 集合。但这些能力在 `capabilities` 表中仍缺少 `related_skills`/`key_tables` 元数据，导致扫描器完全依赖 JS 硬编码白名单。

对其中 5 个有间接证据的能力，可通过补充 DB 元数据使其无需依赖白名单即可被自主检测：
- `branch-protection-hooks` → `related_skills=['dev']`（dev 任务执行 = branch hooks 运行）
- `ci-devgate-quality` → `related_skills=['dev']`（dev PR 流经 CI = DevGate 有效）
- `brain-deployment` → `related_skills=['dev']`（dev 合并 = Brain 部署触发）
- `cecelia-dashboard` → `key_tables=['tasks']`（Dashboard 读取 tasks 表）
- `zenithjoy-dashboard` → `key_tables=['tasks']`（ZenithJoy Dashboard 读取 tasks 表）

**背景**:
当前 Brain 运行旧代码（4:57AM CST 启动，PR #1463 在 21:05 合并），导致扫描结果仍显示 9 个孤岛（evidence=[]）。重启后这 9 个将变 active，但 DB 元数据缺失问题仍存在。

**不做什么**: 不修改 scanner 核心逻辑；不处理纯基础设施能力（cloudflare/nas/tailscale/vpn）—— 这4个确实无 DB 可查证据，白名单是正确方案。
