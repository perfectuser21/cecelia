# PRD: 修复 Engine 测试漂移

## 背景

Engine 本地运行 `npm test` 有 46 个失败测试。经分析分三类：
1. **macOS vs Linux 平台差异**（~32个）：`stat -c %a` 是 Linux 语法，macOS 不支持 → CI 上通过，本地失败（伪问题）
2. **ci-timeout.test.ts 引用已删除的 `engine-ci.yml`**（6个）：CI 架构重构拆成 l1/l2/l3/l4，但测试没更新
3. **stop-hook-state-integrity 行为漂移**（3个）：stop-dev.sh 增加了 session 预检查，改变了损坏 .dev-mode 场景的退出码

## 目标

修复第 2 类（ci-timeout.test.ts）和第 3 类（stop-hook-state-integrity），让 Engine 测试在本地也能正确反映真实行为。

## 范围

- `packages/engine/tests/workflows/ci-timeout.test.ts` — 更新为引用新 CI 文件（ci-l1-process.yml、ci-l3-code.yml 等）
- `packages/engine/tests/hooks/stop-hook-state-integrity.test.ts` — 修复或更新断言（反映 session 预检查后的实际行为）

## 成功标准

- `npx vitest run tests/workflows/ci-timeout.test.ts` → 全部通过
- `npx vitest run tests/hooks/stop-hook-state-integrity.test.ts` → 全部通过（或正确跳过 macOS 不兼容测试）
- Engine 本地 `npm test` 失败数从 46 减少到 ≤ 32（仅剩 macOS 平台差异伪失败）
