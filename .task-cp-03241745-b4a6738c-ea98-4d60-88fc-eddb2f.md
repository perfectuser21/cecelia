# Task Card: CI Scope 修复 — scripts/ci 进 routing-map + workspace 加 L4

## 需求（What & Why）

**功能描述**：
修复 CI 盲区：scripts/ 和 ci/ 目录变更不触发任何 CI；workspace 没有 L4 集成测试。

**背景**：
- 当前 `ci/routing-map.yml` 没有 `scripts` 和 `ci` 条目，改 CI 脚本本身不被 CI 检查
- `workspace`（apps/api + apps/dashboard）只有 [l1, l3]，缺少 L4 集成测试
- `ci-evolution-check.mjs` 只检测 packages/ 和 apps/ 下未注册子目录，不检查 scripts/ci 是否注册

**不做什么**：
- 不动 hook 文件
- 不动 baseline
- workspace L4 只写最小集成测试，不做完整 E2E

## 成功标准

- apps/api 能通过代理调用 Brain GET /api/brain/tasks，测试通过
- scripts/ 和 ci/ 变更触发 L2 检查（routing-map 注册 + L2 change detection 扩展）
- ci-evolution-check.mjs 能检测 scripts/ci 未注册情况
