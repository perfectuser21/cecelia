# Task Card: 内容工厂 — 内容类型绑定 notebook_id + Pipeline 步骤详情视图

## 需求（What & Why）
**功能描述**:
A. 内容类型配置新增 `notebook_id` 字段，新建 Pipeline 时选择内容类型后自动带入，无需手动填写；同时提供 ContentTypeConfigPage 供编辑。
B. Pipeline 卡片可展开，显示每个 stage 的真实状态（状态/耗时/失败原因），而非仅靠整体 status 颜色判断。

**背景**: 内容工厂 Pipeline 体验缺失两个关键功能：notebook_id 每次须手动录入（易遗忘/填错），以及 stage 详情不可见（无法诊断失败原因）。

**不做什么**:
- 不改 YAML 验证逻辑（`REQUIRED_FIELDS`）中增加 notebook_id 为必填
- 不新增 DB 迁移（notebook_id 存在 config JSONB 中，无需新列）
- 不修改 Brain orchestrator 逻辑

## 成功标准

1. [ARTIFACT] `solo-company-case.yaml` 含 `notebook_id` 字段
2. [BEHAVIOR] GET `/api/brain/content-types/solo-company-case/config` 返回含 `notebook_id` 的 config
3. [BEHAVIOR] ContentTypeConfigPage 可显示/编辑各内容类型的 `notebook_id`，PUT 保存成功
4. [BEHAVIOR] ContentFactory 选择内容类型后，`notebook_id` 字段自动填入来自该类型配置的值
5. [BEHAVIOR] GET `/api/brain/pipelines/:id/stages` 返回含 `summary` 字段（失败原因）
6. [BEHAVIOR] Pipeline 卡片展开后显示 stage 列表（状态 + 耗时）
7. [GATE] CI 全部通过
