## PRD - 修复 quarantined 任务过滤 + 默认启用自动派发

**需求来源**: 欲望驱动任务 b413eea6-b41c-48d6-8089-bf41c411c048

**功能描述**:
修复两个关键 bug：
1. `planner.js` 缺少 quarantined 状态过滤，导致隔离任务被重复选中
2. `tick.js` 默认未启用自动派发，导致队列空转

**涉及文件**:
- `packages/brain/src/planner.js` - 添加 quarantined 状态过滤
- `packages/brain/src/tick.js` - 默认启用 auto_dispatch

**成功标准**:
1. `planner.js` 的 `planNextTask()` 查询中加入 `!= 'quarantined'` 条件
2. `tick.js` 的 `runTick()` 默认调用 `dispatchNextTask()`（移除 disabled 注释）
3. CI 通过
4. 本地测试：隔离任务不被选中，自动派发正常工作

**非目标**:
- 不改变其他调度逻辑
- 不修改数据库 schema
- 不涉及前端

### 异常处理 & 报警规范

N/A - 纯逻辑修复，不涉及新的外部 IO

### 技术细节

**planner.js 修改**（约 2-3 行）：
```javascript
// 在 planNextTask() 的 SQL 查询 WHERE 子句中加入：
AND status != 'quarantined'
```

**tick.js 修改**（约 1-2 行）：
```javascript
// 在 runTick() 中移除 auto_dispatch 的 disabled 注释，确保默认调用：
await dispatchNextTask();
```
