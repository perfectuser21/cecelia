# PRD: Step 3 Push 后自动注册 3 个审查任务

## 背景

/dev Step 3（03-prci.md）在 push 成功后直接创建 PR，缺少审查任务注册环节。devloop-check.sh 的条件 2.5/2.6/2.7 已经实现了对 cto_review、code_quality_review、prd_coverage_audit 三个审查状态的检测，但 03-prci.md 中没有注册这些任务的逻辑，导致审查流程无法启动。

## 目标

在 Step 3 push 成功后、创建 PR 之前，自动向 Brain API 注册 3 个审查任务（cto_review、code_quality_review、prd_coverage_audit），并将返回的 task_id 写入 .dev-mode 文件。

## 成功标准

1. 03-prci.md 在 push（8.5）和 PR 创建（8.6）之间包含审查任务注册逻辑
2. 注册 3 种任务类型：cto_review、code_quality_review、prd_coverage_audit
3. 每个任务注册后将 task_id 和 status 写入 .dev-mode.{branch} 文件
4. Brain 不可用时跳过注册并打印警告，不阻塞主流程
5. 存在测试验证 03-prci.md 包含审查注册逻辑
