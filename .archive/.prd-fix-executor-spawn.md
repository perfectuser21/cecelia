## PRD - 修复 Cecelia Docker 任务执行问题

**需求来源**: Docker 部署后任务无法自动执行，Node.js spawn() 在 Alpine 容器中报 ENOENT 错误

**功能描述**:
修复 brain/src/executor.js 中的进程启动逻辑，使其能在 Alpine Linux 容器中正确执行 bash 脚本（cecelia-run）。当前问题是 Node.js spawn() 直接调用脚本路径时不识别 shebang，导致 ENOENT 错误。

**涉及文件**:
- `brain/src/executor.js` - 修改 spawn() 调用，明确使用 bash 执行脚本

**成功标准**:
1. 修改 executor.js 第 320 行的 spawn 调用
2. 从 `spawn(CECELIA_RUN_PATH, [task.id, runId, promptFile], {...})`
3. 改为 `spawn('bash', [CECELIA_RUN_PATH, task.id, runId, promptFile], {...})`
4. Docker 容器重启后，任务能自动派发并执行
5. 检查 `/tmp/cecelia-*.log` 中有 claude 执行日志
6. 熔断器保持 CLOSED 状态

**非目标**:
- 不修改 cecelia-run 脚本本身
- 不改变 Docker Compose 配置（临时 sed patch 已移除）
- 不涉及其他 executor 逻辑优化
