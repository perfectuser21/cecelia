# Task Card: /dev Verifier Subagent 状态机

## 需求（What & Why）

**功能描述**: 给 /dev 的每个关键步骤加独立 Verifier Subagent，实现状态机模式——每步完成后由独立 subagent 检查质量，FAIL 则主 agent 修复直到 PASS，确保每步输出质量有保证不传播错误。

**背景**: 当前 /dev 的问题：
1. 探索完后直接锁死 Task Card scope，不允许 DoD 因探索发现而加深
2. 主 agent 自己测试自己写的代码（确认偏差）
3. Task Card 和 Learning 只做格式检查，不做质量检查

**不做什么**: 不改 Stop Hook 逻辑；不改 PR/CI 流程；不引入新的外部依赖
