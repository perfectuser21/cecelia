# Task Card: 内容生成v1 - AI每日产出引擎

## 需求（What & Why）
**功能描述**: 修复内容生成管道的两个核心阻塞，新增每日产出统计 API，使系统能够可靠地每日产出 ≥3 条内容。
**背景**: 昨日（2026-04-05）topic-selection-scheduler 已成功运行并创建了 7 条 content-pipeline 任务，全部完成。但两个问题阻塞可靠性：(1) 无 notebook_id 时 executeResearch 直接失败而非降级 LLM；(2) 递归任务"AI内容生成每日任务"无 content_type 导致 notebook_id 缺失。
**不做什么**: 不改动 NotebookLM 主流程；不改动 6 阶段 Pipeline 架构；不改动发布流程。

## 成功标准

> [BEHAVIOR] content-research 阶段在无 notebook_id 时使用 LLM 研究降级而非返回 success:false
> [BEHAVIOR] orchestrator 在 content_type 缺失时默认 solo-company-case
> [ARTIFACT] GET /api/brain/pipelines/daily-stats 返回当日完成的 pipeline 数量

## 验收条件（DoD）

- [x] [BEHAVIOR] executeResearch: 无 notebook_id 时调用 LLM 完成 research，不返回 {success:false}
  Test: tests/content-pipeline-research-fallback.test.ts

- [x] [BEHAVIOR] orchestrator: content_type 为 null 时默认 'solo-company-case'
  Test: tests/content-pipeline-research-fallback.test.ts

- [x] [ARTIFACT] GET /api/brain/pipelines/daily-stats 接口存在并返回 {date, completed, in_progress, failed}
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/routes/content-pipeline.js','utf8');if(!c.includes('daily-stats'))process.exit(1)"

## 实现方案

**要改的文件**:
- `packages/brain/src/content-pipeline-executors.js` — executeResearch 添加 LLM research fallback
- `packages/brain/src/content-pipeline-orchestrator.js` — _parsePipelineParams 默认 content_type
- `packages/brain/src/routes/content-pipeline.js` — 添加 GET /daily-stats 路由
- `packages/brain/src/__tests__/content-pipeline-research-fallback.test.ts` — 新增测试

**受影响函数/API**:
- `executeResearch()` — executors.js
- `_parsePipelineParams()` — orchestrator.js
- 新增 `GET /api/brain/pipelines/daily-stats`

**不改什么**: NotebookLM 主流程代码不动；其余 5 个阶段 executor 不变；task-router.js 不变。
