# Task Card: War Room 改为 Summary + Area 详情页架构

## 需求（What & Why）
**功能描述**: 重构 War Room 为两层页面：总览页展示各 Area 卡片汇总，详情页展示单个 Area 下完整 OBJ→KR→Task 三列布局。

**背景**: 原单页把所有 Area 的数据混在一起，随着 OBJ 数量增长会拥挤且难以聚焦。按 Area 分层可以更清晰地对应到 Cecelia/ZenithJoy/Investment 三个业务领域。

**不做什么**:
- 不改 API 后端（纯前端重构）
- 不改导航菜单结构（War Room 入口不变）
- 不修改 apps/api 以外的其他模块

## 成功标准

- /gtd/warroom 渲染 Vision 横幅 + 三个 Area 卡片（Cecelia/ZenithJoy/Investment）
- 每张 Area 卡片显示活跃 OBJ 数、进行中 Task 数、KR 整体进度百分比，点击跳转 /gtd/warroom/:areaId
- /gtd/warroom/:areaId 渲染该 Area 下 OBJ→KR→Task 三列独立滚动布局
- 旧的混合单页布局代码已删除
- 现有测试通过（更新为新架构的数据逻辑）

## 实现方案

**要改的文件**:
1. `apps/api/features/gtd/pages/GTDWarRoom.tsx` — 改为总览页（Vision + Area 卡片）
2. `apps/api/features/gtd/pages/GTDWarRoomArea.tsx` — 新建详情页（三列布局）
3. `apps/api/features/gtd/index.ts` — 注册新路由 + 新组件
4. `apps/api/features/gtd/__tests__/warroom.test.ts` — 更新测试逻辑

**数据结构**（来自现有 /api/tasks/full-tree?view=okr）:
```
tree: [
  { type: 'vision', children: [
    { type: 'area', id: 'xxx', title: 'Cecelia', children: [
      { type: 'objective', status: 'active'/'in_progress', children: [
        { type: 'key_result', progress: 60, ... }
      ]}
    ]}
  ]}
]
```

**Area 卡片数据聚合**（前端计算）:
- `activeObjCount` = area.children.filter(obj => obj.status==='active'||'in_progress').length
- `krProgress` = 所有非 completed KR 的 progress 平均值
- `inProgressTaskCount` = 从全局 tasks 中过滤（task 无 area 字段时，按 KR ID 关联；无法关联时展示全部数量）

**路由设计**:
- `/gtd/warroom` → GTDWarRoom（总览）
- `/gtd/warroom/:areaId` → GTDWarRoomArea（详情，areaId 为 OKR tree 中 area 节点的 id）

**Scope 锚定**: 仅 apps/api/features/gtd/，不触碰其他 feature 或 apps/dashboard
