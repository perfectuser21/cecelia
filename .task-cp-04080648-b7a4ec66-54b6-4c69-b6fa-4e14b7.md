# Task Card: Harness v3.1 — contract 对抗 + 广谱验证修正

## 需求（What & Why）
**功能描述**: 修正 v3.0 引入的错误设计，恢复官方论文定义的正确 Harness 流程
**背景**: 当前 sprint-contract-proposer v3.0 将合同格式改为"行为描述+硬阈值"（无验证命令），sprint-evaluator v4.0 改为"独立广谱验证者"（自主设计测试）。但官方论文明确：合同应含验证命令，Evaluator 是机械执行器，GAN 对抗发生在 contract 协商阶段（挑战验证命令严格性）
**不做什么**: 不改 sprint-planner（已正确）；不改 sprint-generator（已正确）；不改 harness-* 系列（另一套实验性实现）

## 成功标准

- [ ] [ARTIFACT] sprint-contract-proposer 合同草案格式包含 `验证命令` bash 代码块
  Test: manual:node -e "const c=require('fs').readFileSync('packages/workflows/skills/sprint-contract-proposer/SKILL.md','utf8');if(!c.includes('验证命令'))process.exit(1)"

- [ ] [BEHAVIOR] sprint-contract-reviewer 审查重点是验证命令严格性（而非"行为描述可验证性"）
  Test: manual:node -e "const c=require('fs').readFileSync('packages/workflows/skills/sprint-contract-reviewer/SKILL.md','utf8');if(!c.includes('命令够不够严格')||!c.includes('验证命令'))process.exit(1)"

- [ ] [BEHAVIOR] sprint-evaluator 是机械执行器（读合同命令直接执行，不自主设计测试）
  Test: manual:node -e "const c=require('fs').readFileSync('packages/workflows/skills/sprint-evaluator/SKILL.md','utf8');if(!c.includes('机械执行')||c.includes('自主设计'))process.exit(1)"

## 实现方案
**要改的文件**:
- `packages/workflows/skills/sprint-contract-proposer/SKILL.md`
- `packages/workflows/skills/sprint-contract-reviewer/SKILL.md`
- `packages/workflows/skills/sprint-evaluator/SKILL.md`

**核心变更**:
1. Proposer: 合同格式从"行为描述+硬阈值" → "行为描述+硬阈值+验证命令（bash）"，命令覆盖 curl/npm test/psql/playwright
2. Reviewer: 审查从"行为描述是否可验证" → "验证命令是否足够严格（能否检测出错误实现）"
3. Evaluator: 从"独立广谱验证者（自主设计）" → "机械执行器（读合同命令执行）"

**不改什么**: sprint-planner / sprint-generator / sprint-report / harness-* 系列
