# PRD: 飞书群聊身份识别修复

## 背景

当前所有群聊成员都被识别为 '用户' + relationship='guest'，原因：
1. `contact/v3/users/{openId}` API 返回 `code=99991672`（权限不足），Feishu 机器人没有读取联系人的 scope
2. feishu_users 表完全为空（0 行），staleCache 无数据可回退
3. `getGroupMembers` API 可以工作但不写入 feishu_users，只返回名字数组
4. Mode A 群消息写 memory_stream 时 `memory_type='observation'` 违反 CHECK 约束（只允许 short/mid/long），所有群聊记忆写入失败

## 修复目标

### Fix 1: getGroupMembers 写入 feishu_users
- `getGroupMembers` 调飞书成员列表时，将每个成员的 `member_id`（open_id）、`name`、`relationship` 写入 `feishu_users`
- Alex 徐啸 识别为 owner，其余成员默认 colleague（工作群场景）
- 写入后 staleCache 可正常生效

### Fix 2: 调用顺序修正
- 群消息处理时，先调 `getGroupMembers`（预填 feishu_users），再调 `getFeishuUserName`（此时 staleCache 有数据）
- 保证 senderName 和 relationship 正确

### Fix 3: memory_type 修正
- 将 Mode A 非 @mention 路径中 `memory_type: 'observation'` 改为 `memory_type: 'short'`
- 修复所有群聊记忆写入失败的问题

## 非目标

- 不修改联系人 API 权限（需要 Feishu 应用配置，不是代码问题）
- 不修改 schema（feishu_users 表结构不变）
