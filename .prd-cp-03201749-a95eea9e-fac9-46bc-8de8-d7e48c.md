# PRD: [SelfDrive] 修正 Scanner 对内嵌能力的误判

## 功能描述

修正 capability-scanner.js 中的内嵌能力识别逻辑，确保 Brain 内嵌运行的能力（意识层能力、架构性能力、Engine hooks 等）不被误判为孤岛。

## 背景

- 当前 Scanner 将部分内嵌在 Brain 进程中的能力误判为孤岛
- migration 094 定义了 12 个 Cecelia 意识能力，但 Scanner 的 BRAIN_ALWAYS_ACTIVE 列表不完整
- Engine 层的 branch-protection、ci-devgate 等能力也需要特殊识别
- 这是后续孤岛激活决策的基础，必须准确

## 成功标准

- [ ] 修正 capability-scanner.js 中的 BRAIN_ALWAYS_ACTIVE 列表
- [ ] 所有 scope='cecelia' 的意识能力标记为 active 而非 island
- [ ] Engine 层核心能力（hooks、devgate）正确识别为非孤岛
- [ ] 所有现有测试通过

## 验收标准

- [ ] BRAIN_ALWAYS_ACTIVE 包含完整的 Cecelia 意识能力列表
- [ ] 新增 Engine 层能力识别机制
- [ ] 测试验证意识能力不被标记为 island
- [ ] CI 全部通过

## 实现方案

**要改的文件**: packages/brain/src/capability-scanner.js
**Scope 锚定**: Brain 能力扫描逻辑修正
