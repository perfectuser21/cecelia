### 2. `upsertLearning(title, content, category, triggerEvent, pool?)` — 纯 DB 操作，不含 embedding
**位置**：`learning.js` 新增导出函数（与现有 `recordLearning` 共存）

**逻辑**（精确 title 匹配）：
```
BEGIN
  existing = SELECT id, frequency_count FROM learnings WHERE title = $title LIMIT 1
  IF existing:
    UPDATE learnings
      SET frequency_count = COALESCE(frequency_count, 1) + 1,
          last_reinforced_at = NOW()
      WHERE id = existing.id
    RETURN { id: existing.id, upserted: false }
  ELSE:
    INSERT INTO learnings (title, category, content, trigger_event, frequency_count, last_reinforced_at)
      VALUES (..., 1, NOW())
    RETURN { id: new.id, upserted: true }
END
```

**为什么用 title 而非 content_hash**：
- thalamus/orchestrator-chat/quarantine 三处写入场景的 content 是 LLM 生成的文字，每次略有不同，content_hash 无法匹配；但 title 是事件标识符（如 "隔离分析：task-xxx"），语义唯一
- `recordLearning()` 已用 content_hash，不动它；upsertLearning 补充 title-based 去重

当前：BATCH_SIZE=10，单次处理后退出，下次等 1h

改为：
```javascript
export async function backfillLearningEmbeddings(dbPool) {
  if (!process.env.OPENAI_API_KEY) return { processed: 0, failed: 0 };
  const BATCH_SIZE = 50;
  const DELAY_MS = 200;
  let totalProcessed = 0, totalFailed = 0;
  let hasMore = true;

  while (hasMore) {
    // SELECT LIMIT 50 WHERE embedding IS NULL
    // IF rows.length === 0: hasMore = false, break

}
```
