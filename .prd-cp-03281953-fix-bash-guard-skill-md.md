# PRD: 修复 bash-guard 过激拦截 SKILL.md

## 背景

`~/.claude/skills/` 下的 26 个 skills 已从 symlink 转为独立真实目录，不再属于 git 追踪范围。bash-guard.sh 中"规则 4"是基于旧 symlink 假设的拦截逻辑，现已过时，需要移除。

## 目标

删除 `packages/engine/hooks/bash-guard.sh` 中的 `SKILL_PATH_PATTERN` 相关拦截代码块（规则 4，约 38 行），使 skills 可以自由编辑而无需走 /dev。

## 成功标准

- `bash-guard.sh` 中不再包含 `SKILL_PATH_PATTERN` 字符串
- 修改后 `bash-guard.sh` 语法检查通过（`bash -n`）
- 其他拦截规则（凭据泄露、git push main、HK 部署防护等）不受影响
