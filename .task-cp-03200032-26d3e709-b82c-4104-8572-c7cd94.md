# Task Card: 多媒体素材组装器 — 修复 SVG/PNG 渲染管道 + 平台格式规范化

## 需求（What & Why）

**功能描述**：
将 `content-pipeline-executors.js` 中内联的 `generateCards()` 函数（约 150 行）抽取为独立模块 `media-assembler.js`，同时修复其在 ESM 环境中使用 `require()` 加载 `@resvg/resvg-js` 导致的运行时崩溃。

**背景**：
`executeExport` 阶段调用内联 `generateCards()` 生成卡片图片。当前实现用 CommonJS `require('@resvg/resvg-js')` 加载渲染库，但整个 `packages/brain/src/` 是 ESM 模块（`"type": "module"`），`require` 在 ESM 中不可用，导致每次 export 时 resvg 渲染直接报错，卡片全部生成失败，`manifest.json` 中 `image_set.status` 始终为 `"failed"`，下游发布器拿不到任何图片素材。

**不做什么**：
- 不改变 SVG 视觉风格（THEMES/ACCENTS/卡片布局保持原样）
- 不修改 Pipeline 状态机（orchestrator.js 不动）
- 不新增图片上传逻辑（仅生成本地文件）

## 成功标准

1. [ARTIFACT] `packages/brain/src/media-assembler.js` 存在并导出 `assembleMedia` 函数
2. [ARTIFACT] `generateCards` 内联函数已从 `content-pipeline-executors.js` 中移除
3. [ARTIFACT] `media-assembler.js` 使用 dynamic `import()` 而非 `require()` 加载 `@resvg/resvg-js`
4. [BEHAVIOR] `executeExport` 改为调用 `assembleMedia`，manifest 含 `cover_wechat` 字段
5. [GATE] CI 全部通过
