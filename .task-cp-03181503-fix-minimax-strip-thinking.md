# Task Card: fix MiniMax 推理模型 <think> 标签导致空响应

## 需求（What & Why）
**功能描述**: 修复 `stripThinking` 函数，当 MiniMax 推理模型把所有响应内容包在 `<think>` 标签里时，降级提取 think 内容作为兜底，而不是抛出 "MiniMax returned empty content" 错误。
**背景**: MiniMax-M2.5-highspeed 是推理模型，有时会把所有内容（包括最终答案）包在 `<think>` 标签里。`stripThinking` 剥离后内容变空，`callMiniMaxAPI` 抛出错误，丘脑调用失败，Brain tick 停止。
**不做什么**: 不改模型配置、不改 thalamus.js、不改其他地方。

## 成功标准

1. [ARTIFACT] `stripThinking` 在剥离后内容为空时，降级返回 `<think>` 内容
2. [BEHAVIOR] MiniMax 推理模型返回全 `<think>` 格式时不再抛出 "empty content" 错误
3. [GATE] Brain 单测通过，CI 通过
