# Task Card: fix-dod-state-sync

## PRD

### 背景
verify-step.sh Gate 2 跑 DoD Test 通过后未写回 `[x]`，导致状态机（Task Card）与 CI DoD 完整性检查不同步。

### 功能描述
1. **verify-step.sh Gate 2**：每条 DoD `[BEHAVIOR]/[ARTIFACT]/[GATE]` Test 通过后，自动将 Task Card 对应行从 `- [ ]` 更新为 `- [x]`
2. **devloop-check.sh**：在 step_2_code 完成后、PR 创建前（条件 2.7），加入 DoD 完整性检查——Task Card 中仍有未勾选的 `[BEHAVIOR]/[ARTIFACT]/[GATE]` 条目时阻止继续

### 技术方案
- `verify-step.sh`：Gate 2 循环中加 `LINE_NUM` 行计数器 + `CURRENT_DOD_LINE` 跟踪条目行号；Test 通过时调用 `_mark_dod_done()` 用 `sed -i` 写回 `[x]`（macOS/Linux 兼容）
- `devloop-check.sh`：条件 2.5 之后插入条件 2.7，grep 统计 Task Card 中 `^\s*-\s*\[ \]\s*\[(BEHAVIOR|ARTIFACT|GATE)\]` 的未勾选数量，大于 0 则 exit 2 阻止

## 成功标准

- verify-step.sh Gate 2 中包含 `_mark_dod_done` 函数，Test 通过后用 `sed -i` 写回 `[x]`
- devloop-check.sh 包含条件 2.7 DoD 完整性检查，有未勾选 DoD 条目时输出 `dod_completeness` blocked 并 return 2
