# Task Card: 修复 BrainModelsPage handleSaveOrgan API 参数名

## 需求（What & Why）

**功能描述**：`BrainModelsPage.tsx` 的 `handleSaveOrgan` 函数发送 PATCH 请求时，
body 使用了 `{ agentId, model, provider }`，但后端 `/api/brain/model-profiles/active/agent`
要求 `{ agent_id, model_id, provider }`，导致点击保存时报错 `agent_id and model_id are required`。

**修复方案**：将 `handleSaveOrgan` 中 fetch body 改为 `{ agent_id: organId, model_id: modelId, provider }`。

**影响范围**：`apps/dashboard/src/pages/brain-models/BrainModelsPage.tsx` 第 128 行

## 成功标准

1. [BEHAVIOR] handleSaveOrgan fetch body 使用 agent_id 和 model_id
2. [BEHAVIOR] 不存在旧的 camelCase 参数名 agentId 和 model: modelId
3. [GATE] CI 全部通过
