# PRD: Cecelia P2P 对话查询能力——派任务+回调回飞书

**Task ID**: 0c74a4dd-bbcb-44cf-a03d-4ee075f5056a
**Branch**: cp-03231615-p2p-query-dispatch
**Priority**: P1

## 问题

Cecelia 在飞书 P2P 对话里无法真正"查东西"。当用户问 "zenithjoy dashboard 现在有什么" 时，她只会生成"我去查一下"的空话，不会真正派出探查任务，结果永远回不来。

## 根因

1. `thalamus.js`：信息性查询被分类为 `handle_chat`，不触发 `create_task`
2. `ops.js`：创建任务后没有注册 `task_interest` 订阅，任务完成时不会回调发飞书
3. `execution.js`：task_interest 回调不携带 findings，通知内容为空

## 需要实现的能力

1. 信息性查询 → thalamus 派出 `explore` 任务
2. P2P 流程：立即回复"正在查，马上给你～"；注册 task_interest 订阅
3. explore 任务完成后，把 findings 通过飞书发回给用户

## 成功标准

- 用户问 "我们 zenithjoy dashboard 现在有什么" → Cecelia 回复 "正在查，马上给你～"
- Brain 实际创建一个 `task_type='explore'` 的任务并派发
- 任务完成后，Cecelia 通过飞书 P2P 发回结果（含 findings 摘要）

## 改动文件

1. `packages/brain/src/thalamus.js` — 添加规则 7
2. `packages/brain/src/routes/ops.js` — P2P handler 注册 task_interest
3. `packages/brain/src/routes/execution.js` — task_interest 回调携带 findings
