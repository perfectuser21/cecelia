# PRD: fix MiniMax 推理模型 <think> 标签导致空响应

## 背景
MiniMax-M2.5-highspeed 是推理模型，有时把所有内容包在 <think> 标签里。
stripThinking 剥离后内容变空，callMiniMaxAPI 抛出 "MiniMax returned empty content" 错误，丘脑调用失败。

## 目标
修复 stripThinking 函数：剥离 <think> 后内容为空时，降级提取 think 内容作为兜底。

## 成功标准

1. [ARTIFACT] stripThinking 处理纯 <think> 响应时返回 think 内容而非空字符串
2. [BEHAVIOR] callMiniMaxAPI 不再因 think-only 响应抛出 empty content 错误
3. [GATE] Brain 测试通过
