# PRD: stop.sh Stage 4 完整完成条件

## 背景

PR #1730 修复了 p0 阶段提前 exit 0 的问题，但新的 "CI pass → exit 0" 仍然太早。
packages/quality/hooks/stop.sh 在 CI 全绿后直接退出，跳过了 Stage 4 的所有检查。

## 问题

CI 通过后缺少：
1. cleanup_done: true 检查（唯一合法出口）
2. step_4_ship: done 检查（Learning 是否写完推上去）
3. PR 是否已合并的检查
4. 合并后写 cleanup_done: true 触发最终退出

## 目标

CI 通过后的完整流程：
```
CI 通过
  → 检查 cleanup_done: true → exit 0（唯一出口）
  → 检查 step_4_ship: done
      → pending: exit 2（提示写 Learning + 合并）
      → done + PR 未合并: 执行合并 → exit 2
      → done + PR 已合并: 写 cleanup_done: true → exit 2（下次检测到 cleanup_done 则 exit 0）
```

## 成功标准

- CI pass 后不再直接 exit 0
- cleanup_done: true 是唯一 exit 0 出口
- step_4_ship: pending 时 exit 2 并提示写 Learning
- PR 合并后 + step_4_ship done → 写 cleanup_done: true
- 参考：packages/engine/lib/devloop-check.sh 条件 5/6 逻辑（SSOT）

## 修改范围

`packages/quality/hooks/stop.sh` 第 281-292 行（CI pass 的 exit 0 处）
