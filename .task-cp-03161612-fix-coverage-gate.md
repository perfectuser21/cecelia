# Task Card: 修复 test-coverage-required gate 误拦截非 feat commit

## 需求（What & Why）
**功能描述**: `test-coverage-required` job 在 push-to-main 场景下 `PR_TITLE` 为空，`[CONFIG]` 豁免失效，且脚本缺少 commit type 检查，导致 `docs:`/`chore:` 类型 commit 也被错误拦截要求测试文件。

**背景**: PR #982 新增了 `test-coverage-required` gate。PR #983（`docs:` commit）合并到 main 时触发了 L3 CI，gate 误拦截，L3 Code Gate 失败。

**根本原因**:
1. 脚本用 `PR_TITLE` 判断 `[CONFIG]` 豁免 — push-to-main 时 `PR_TITLE` 是空字符串
2. 脚本内部没有检查 commit type，任何触发 L3 的 commit 都会被强制要求测试文件

**不做什么**: 不修改其他 gate 逻辑

## 成功标准

1. [ARTIFACT] `ci-l3-code.yml` 中 `test-coverage-required` 脚本新增 commit message `[CONFIG]` 检查
2. [ARTIFACT] `test-coverage-required` 脚本新增非 `feat:` commit type 时直接 skip
3. [BEHAVIOR] `docs:` 类型 push 到 main 不再被 test-coverage-required 拦截
4. [GATE] CI 全部通过

## 验收条件（DoD）

- [x] [ARTIFACT] ci-l3-code.yml test-coverage-required 脚本含 commit message [CONFIG] 检查
  Test: manual:node -e "const fs=require('fs');const c=fs.readFileSync('.github/workflows/ci-l3-code.yml','utf8');if(!c.includes('COMMIT_MSG'))throw new Error('no COMMIT_MSG');console.log('PASS')"

- [x] [ARTIFACT] 脚本含非 feat commit type skip 逻辑
  Test: manual:node -e "const fs=require('fs');const c=fs.readFileSync('.github/workflows/ci-l3-code.yml','utf8');if(!c.includes('feat'))throw new Error('no feat check');console.log('PASS')"

- [x] [BEHAVIOR] push 到 main 的 docs: commit 中 [CONFIG] 检查通过（静态验证脚本逻辑）
  Test: manual:node -e "const fs=require('fs');const c=fs.readFileSync('.github/workflows/ci-l3-code.yml','utf8');if(!c.includes('grep -q'))throw new Error('no grep check');console.log('PASS')"

- [x] [GATE] 所有现有测试通过
  Test: manual:bash -c "cd packages/engine && npm test 2>&1 | tail -5"

## 实现方案
**要改的文件**: `.github/workflows/ci-l3-code.yml`
**Scope 锚定**: `test-coverage-required` job 的 `Check for test file changes` step，在 `[CONFIG]` 检查后、获取 CHANGED 文件前，增加 commit message `[CONFIG]` 检查 + commit type 非 `feat:` 时 skip
