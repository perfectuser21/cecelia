# Task Card: Feishu 消息统一经过丘脑路由

## 需求（What & Why）

**功能描述**：Feishu 消息进来时，先构造 USER_MESSAGE 事件交给丘脑处理，由丘脑决定是直接回复、调 handleChat 还是派发任务。

**背景**：
- 丘脑设计目标是 Gateway（所有信息统一路由）
- 但 Feishu 消息历史上直接调 `handleChat`，绕过了丘脑
- PR #1062 已给丘脑注入 Skills 地图 + 任务队列感知，现在丘脑有能力做路由决策
- 丘脑 thalamus.js 已有 USER_MESSAGE 事件类型和对应路由 prompt（line 506）

**不做什么**：
- 不改变 handleChat 本身的逻辑
- 不修改丘脑的 LLM prompt（只利用现有能力）
- 不改变群聊/私聊的消息获取逻辑

## 成功标准

1. [BEHAVIOR] Feishu 私聊消息进来时，先调用 thalamusProcessEvent(USER_MESSAGE 事件)，再根据返回决定后续动作
2. [BEHAVIOR] 丘脑返回 handle_chat 时，走原有 handleChat 逻辑（兜底）
3. [BEHAVIOR] 丘脑返回 dispatch_task 时，创建 Brain 任务而不调 handleChat
4. [BEHAVIOR] 丘脑调用失败时，fallback 到原 handleChat（不影响正常对话）
5. [BEHAVIOR] 群聊 @mention 消息同样先过丘脑
6. [GATE] Brain unit tests 通过，无现有测试破坏
