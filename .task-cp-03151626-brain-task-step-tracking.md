# Task Card: Brain PATCH custom_props + stop-dev.sh 步骤状态回写

## 需求（What & Why）

**功能描述**:
1. Brain PATCH `/api/brain/tasks/:id` 新增 `custom_props` JSONB merge 支持
2. `stop-dev.sh` 在每步完成时调用 Brain API，将 `/dev` 进度写入任务的 `custom_props`

**背景**: 状态机双 Gate（Script Gate + Subagent Gate）已在 PR #963 完成。但 Brain 不知道每个运行中的 `/dev` 任务当前在哪一步，Dashboard 无法显示实时进度。`custom_props` 字段是 Brain tasks 表里已有的 JSONB 列，可以存任意结构化数据。

**不做什么**: 不改 Dashboard 前端（那是 Task B / 5f0abbab 的事）；不改 `task_id` 写入逻辑（`.dev-mode` 中已有 `task_id:` 字段由 `fetch-task-prd.sh` 写入）。

## 成功标准

1. [ARTIFACT] `routes.js` PATCH endpoint 支持 `custom_props` 字段，合法 JSONB merge 请求返回 200 + 更新后 custom_props
2. [BEHAVIOR] 调用 `PATCH /api/brain/tasks/:id` 传 `{custom_props: {dev_step: 2}}` 后，`GET /api/brain/tasks/:id` 的 `custom_props` 字段包含 `dev_step: 2`（已有 key 保留，新 key 追加）
3. [GATE] Brain CI 全部通过

## 验收条件（DoD）

- [ ] [ARTIFACT] `packages/brain/src/routes.js` PATCH `/tasks/:task_id` 支持 `custom_props` 字段（jsonb merge，非替换），有效载荷示例 `{"custom_props": {"dev_step": 2, "pr_url": "https://..."}}`
  Test: manual:bash -c "grep -c 'custom_props' packages/brain/src/routes.js"

- [ ] [BEHAVIOR] `stop-dev.sh` 检测到步骤状态变化（`step_N_xxx: done` 写入 `.dev-mode`）时，若 `.dev-mode` 中有 `task_id`，调用 Brain PATCH API 更新 `custom_props.dev_step`
  Test: manual:bash -c "grep -c 'custom_props' packages/engine/hooks/stop-dev.sh"

- [ ] [ARTIFACT] Brain routes.js PATCH 同时支持 status 和 custom_props，两者均为可选（不再强制 status 必须存在），但至少有一个非空
  Test: manual:bash -c "grep -c 'custom_props\\|status' packages/brain/src/routes.js"

- [ ] [GATE] Brain 单元测试通过（baseline 容忍机制）
  Test: manual:bash -c "cd packages/brain && npm test 2>&1 | tail -3"

## 实现方案

**要改的文件**:
1. `packages/brain/src/routes.js` (lines 943-1047): 改 PATCH handler，允许 `custom_props` 字段（jsonb merge）
2. `packages/engine/hooks/stop-dev.sh`: 在步骤状态更新 `.dev-mode` 时同步调用 Brain PATCH API

**Scope 锚定**:
- routes.js: 仅修改 PATCH `/tasks/:task_id` handler，约 20 行改动
- stop-dev.sh: 新增一个 helper function `report_step_to_brain()`, 在 `.dev-mode` 更新位置调用，约 15 行新增

**custom_props 写入格式**:
```json
{
  "dev_step": 2,
  "dev_step_name": "code",
  "pr_url": "https://github.com/...",
  "branch": "cp-xxx",
  "updated_at": "2026-03-15T10:00:00+08:00"
}
```
