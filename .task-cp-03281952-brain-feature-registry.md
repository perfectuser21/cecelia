# Task Card: CI L2 扩展 — Brain 新文件强制登记 brain.md

## 需求（What & Why）

**功能描述**: 扩展 CI L2 的监控范围，当 PR 中有新增的 `packages/brain/src/*.js` 文件时，要求同步更新 `.agent-knowledge/brain.md`，否则 CI 失败。

**背景**: 当前 CI L2 的 Impact Check 只监控 `packages/engine/` 的核心文件变更，`packages/brain/src/` 的新模块上线没有强制登记要求，造成 Brain 功能孤岛（如 `self-drive.js`、`dopamine.js`、`pipeline-patrol.js` 上线后没有文档更新机制）。`.agent-knowledge/brain.md` 是 Brain 模块的知识文件，是 Claude 感知 Brain 状态的 SSOT，需要随新模块同步更新。

**不做什么**:
- 不修改现有的 Engine Impact Check 逻辑（只检查 feature-registry.yml）
- 不要求修改现有的 Brain 文件（只检测新增文件，`A` 状态）
- 不检测 `packages/brain/src/` 下修改现有文件的情况

## 成功标准

1. [ARTIFACT] `.github/workflows/ci-l2-consistency.yml` 新增 `brain-new-files-check` job
2. [BEHAVIOR] PR 中有新增 `packages/brain/src/*.js` 文件但未更新 `.agent-knowledge/brain.md` 时，CI 失败并给出明确提示
3. [BEHAVIOR] PR 中无新增 Brain 文件时，job 直接通过（跳过）
4. [GATE] CI L1/L2 全部通过

## 实现方案

**要改的文件**:
- `.github/workflows/ci-l2-consistency.yml`

**具体变更**:
1. 在 `new-files-need-rci-check` job 之后，添加新 job `brain-new-files-check`
2. job 条件：`if: github.event_name == 'pull_request'`
3. 逻辑：
   - 获取 PR 新增文件：`git diff --name-only --diff-filter=A origin/$BASE_REF...HEAD`
   - 过滤 `packages/brain/src/*.js` 文件
   - 若有新 Brain js 文件，检查变更文件中是否包含 `.agent-knowledge/brain.md`
   - 未更新 → `exit 1` 并给出提示
4. 更新 `l2-passed` gate job 的 `needs` 数组，加入 `brain-new-files-check`

**Scope 锚定**: 只改 `.github/workflows/ci-l2-consistency.yml` 一个文件
