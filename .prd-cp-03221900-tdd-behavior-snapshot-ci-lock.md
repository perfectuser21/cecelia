# PRD: dev pipeline 补三个硬锁单元

## 背景
当前 /dev pipeline 存在执行漂移问题——AI 执行 literal instruction 时会丢失"不能删什么"的 behavioral intent（如 TDD 在 Codex→subagent 迁移中被无声删除）。CI 镜像检查用 `|| true` 收尾导致失败被忽略。

## 目标
在 Stage 2 流程中补三个强制（exit 1）单元：
1. Step 2.0 行为快照 [PRESERVE]：探索前，扫描涉及模块的现有行为写入 Task Card [PRESERVE] 条目
2. Step 2.1.5 TDD先行：探索后、实现前，先写失败测试（红灯确认），再写实现（绿灯确认）
3. CI镜像强制：修复 2.3.3 + 2.3.6 的 || true 和缺失 exit 1

## 成功标准
- 02-code.md 新增 Step 2.0（行为快照，含 exit 1 硬锁）
- 02-code.md 新增 Step 2.1.5（TDD先行，红灯→绿灯强制流程）
- verify-step.sh 能识别并执行 [PRESERVE] 类型 DoD 条目
- 02-code.md 2.3.3 + 2.3.6 失败时 exit 1（不再 || true）
- Engine 版本 bump（patch）
