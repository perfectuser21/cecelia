# Task Card: Pipeline Security Bug Fix（4个 P0/P1 安全漏洞修复）

## 需求（What & Why）

**功能描述**: 修复 /dev 工作流中4个安全漏洞，防止 seal 校验绕过、不安全降级、并发竞争、合并时机误判

**背景**:
- 当前 `devloop-check.sh` 的 seal 文件验证仅检查文件存在性，seal 内容缺失或无效时不阻断（P0）
- `stop-dev.sh` 在 `devloop-check.sh` 未加载时会降级使用旧逻辑，可绕过新安全机制（P0）
- `lock-utils.sh` 在 `flock` 不可用时静默返回 0，掩盖并发安全问题（P1）
- `devloop-check.sh` 在合并 PR 前不重新验证 CI 和 mergeable 状态，可能在 CI 失败后仍执行合并（P1）

**不做什么**: 不改变 /dev 工作流的正常成功路径；不修改 CI 配置；不改 Brain 代码

## 成功标准

1. [BEHAVIOR] devloop-check.sh 检测到 seal 文件存在但 verdict 字段为空时，以阻断状态退出（不放行）
2. [BEHAVIOR] devloop-check.sh 检测到 seal 文件存在但 divergence_count 字段缺失或为 null 时，以阻断状态退出
3. [BEHAVIOR] stop-dev.sh 在 devloop-check.sh 未加载时，以 exit 2 拒绝降级而非执行旧逻辑
4. [BEHAVIOR] lock-utils.sh 在 flock 不可用时，acquire_dev_mode_lock 返回 1（失败）而非 0（成功）
5. [BEHAVIOR] devloop-check.sh 在执行 gh pr merge 前，重新检查 PR 的 mergeable 状态和 CI 结论，不符合条件时阻断
6. [GATE] CI 全部通过
