## PRD - 任务派发效果验证与监控

**需求来源**: Brain 自动调度任务（dev 类型）

**功能描述**:
已完成派发瓶颈修复（权重系统 + 任务清理），需要验证效果并添加监控能力：
1. 添加派发效果监控端点 GET /api/brain/dispatch/effectiveness
2. 分析并修复 initiative_plan 任务被 canceled 的原因，添加保护机制
3. 添加任务清理审计日志，提供审计查询端点 GET /api/brain/cleanup/audit

**涉及文件**:
- `packages/brain/src/server.js`（添加新 API 端点）
- `packages/brain/src/task-cleanup.js`（添加审计日志 + initiative_plan 保护）
- `packages/brain/src/task-router.js`（检查 initiative_plan 派发条件）
- `packages/brain/tests/`（单元测试）
- `packages/brain/package.json`（版本号）

**成功标准**:
- GET /api/brain/dispatch/effectiveness 可用，返回完整统计数据
  - canceled 任务统计（按 task_type 分组）
  - initiative_plan 失败率
  - 平均等待时长（按 priority 分组）
  - 权重系统生效验证
- initiative_plan canceled 率 < 10%
- 任务清理有审计记录，可追溯
- P0 任务平均等待时长 < 10 分钟
- Brain CI 通过，单元测试覆盖率 >= 80%

**非目标**:
- 不修改权重系统核心逻辑（上一个 PR 已完成）
- 不添加前端 UI（仅后端 API）
- 不修改 PostgreSQL schema（使用现有表结构记录审计日志）
