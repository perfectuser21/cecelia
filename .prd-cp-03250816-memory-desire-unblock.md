# PRD: 记忆系统 PR6 — Desire Unblock

## 背景

当前 Desire 系统已可从反思生成 desire 记录，但 suggestion pipeline（suggestion-triage → suggestion-dispatcher）在评分和生成建议时不感知当前欲望状态。导致 Brain 推送的建议与自身欲望脱节——即"有想法但说不出来"的阻塞状态。

## 目标

新建 `suggestion-cycle.js` 模块，将 active desire 注入 suggestion pipeline：

1. `getActiveDesiresForSuggestion(dbPool)` — 查询 status=pending、urgency≥7 的 desire，取前 5 条
2. `buildSuggestionPrompt(context, desires)` — 构建含 desire 上下文的 suggestion prompt
3. `runSuggestionCycle(dbPool)` — 编排上述两步，生成并写入新 suggestion

## 成功标准

- `getActiveDesiresForSuggestion` 正确查询 desires 表（pending + urgency≥7）
- `buildSuggestionPrompt` 在有 desire 时注入欲望上下文，无 desire 时返回纯上下文
- `runSuggestionCycle` 成功创建 suggestion 记录
- 测试覆盖有/无 desire 的 prompt 构建场景
