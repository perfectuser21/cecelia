# PRD: Pipeline Integrity Gate

## 任务 ID
5e8ce6ef-c5d6-4e7f-9d95-7a7eec9cb0f0

## 背景

Pipeline 安全属性是整个 CI/CD 系统的基础，但目前没有测试验证 pipeline 自身不被削弱。需要一个 meta-test 作为 CI 门禁的守护者，确保任何对 pipeline 文件的修改都不会引入安全漏洞（如 fail-open、N次后放行等）。

## 目标

新增 `packages/engine/tests/pipeline-integrity.test.ts`，作为 meta-test 验证 pipeline 自身的安全不变量。

## 成功标准

- pipeline-integrity.test.ts 文件存在且包含 4 个 describe 块
- known-failures.json 中包含 4 条 known-failing 条目
- 未被 known-failures 标记的测试全部通过
- Engine L1 CI 通过

## 实现范围

1. 新建 `packages/engine/tests/pipeline-integrity.test.ts`
2. 修改 `packages/engine/ci/known-failures.json` 添加 4 条 known-failing 条目
3. Engine 版本 13.64.0 → 13.66.0（5 个文件同步 bump）
