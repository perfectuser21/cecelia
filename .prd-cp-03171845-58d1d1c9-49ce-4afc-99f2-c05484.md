# PRD - fix(tick): initTickLoop 启动时调用 syncOrphanTasksOnStartup，修复重启孤儿静默丢失

## 背景
Brain 重启后，所有 in_progress 任务会静默变成孤儿（找不到对应进程），但这些任务从未被 requeue。系统在 executor.js 中已实现 `syncOrphanTasksOnStartup()`，也已 import 到 tick.js，但 `initTickLoop()` 从未调用它。

## 目标
在 `initTickLoop()` 内，tick loop 启动前，调用 `await syncOrphanTasksOnStartup()` 并将结果写入启动日志。

## 成功标准
- [ ] `initTickLoop()` 在 startTickLoop/enableTick 之前调用 `syncOrphanTasksOnStartup()`
- [ ] 孤儿任务（进程已消失 + watchdog_retry_count < 2 + 无 error_message）变为 queued
- [ ] OOM/被 kill 类孤儿（error_message 不为空）仍走 failed
- [ ] 启动日志输出 `[initTickLoop] startup-sync: orphans_found=X requeued=X rebuilt=X failed=X`
- [ ] 现有 `executor-startup-sync.test.js` 全通
- [ ] 新增测试：模拟 Brain 重启场景，验证 initTickLoop 触发了 syncOrphanTasksOnStartup
