# Task Card: fix execution.js sprint_contract_propose verdict 纯文本正则提取

## 需求（What & Why）
**功能描述**: execution.js 的 sprint_contract_propose 完成回调中，verdict 提取依赖 `extractVerdictFromResult` 函数，但该函数只能匹配 `"verdict":"PROPOSED"` 的 JSON 格式。当 cecelia-run 传来的是纯文本（如仅含 `PROPOSED` 单词）时，提取失败 → `proposeVerdict === null` → GAN 守卫拦截 → Reviewer 永远不派发。

**背景**: 所有 sprint_contract_propose completed 任务的 DB result 字段均为 null，证实提取从未成功过。  

**不做什么**: 不改 skill 输出格式，不改 cecelia-run.sh，只在 execution.js 加 fallback。

## 成功标准

- [x] [BEHAVIOR] sprint_contract_propose: extractVerdictFromResult 失败后，fallback 能从纯文本中提取 PROPOSED 单词
  Test: manual:node -e "const f=require('fs').readFileSync('packages/brain/src/routes/execution.js','utf8');if(!f.includes('\\\\bPROPOSED\\\\b'))process.exit(1);console.log('OK: PROPOSED fallback present')"

- [x] [BEHAVIOR] sprint_contract_review: verdict 提取路径覆盖 result.result 纯文本字符串（Claude SDK JSON 格式）
  Test: manual:node -e "const f=require('fs').readFileSync('packages/brain/src/routes/execution.js','utf8');const hasReviewFallback=f.includes('result?.result')&&(f.includes('APPROVED')||f.includes('REVISION'));if(!hasReviewFallback)process.exit(1);console.log('OK: review fallback present')"

- [x] [ARTIFACT] execution.js 中 sprint_contract_propose 处理块存在 fallback 提取逻辑
  Test: manual:node -e "const f=require('fs').readFileSync('packages/brain/src/routes/execution.js','utf8');if(!f.includes('proposeRawText'))process.exit(1);console.log('OK: proposeRawText variable present')"

## 实现方案
**要改的文件**: `packages/brain/src/routes/execution.js`

**Line ~1721-1725**: `sprint_contract_propose` 处理块：
- 在 `extractVerdictFromResult` 调用后，加 fallback：提取 rawText（优先 result.result，否则 result as string），用 `/\bPROPOSED\b/i` 匹配

**Line ~1757-1767**: `sprint_contract_review` 处理块：
- 检查是否覆盖 Claude SDK JSON 格式（`result?.result` 字段），若 reviewResultRaw 为空时加入 `result.result` 兜底

**不改什么**: extractVerdictFromResult 函数本身不改（其他地方也在用）
