# Task Card: 记忆系统 PR5 — Dopamine RPE（奖赏预测误差）

## 需求（What & Why）

**功能描述**：在 `packages/brain/src/dopamine.js` 中新增 RPE（Reward Prediction Error）计算机制。
RPE = 实际奖赏 - 期望奖赏（历史同类任务均值）。
RPE > 0 → 超出预期，增强学习；RPE < 0 → 低于预期，降低同类任务优先级调整系数。

**背景**：当前 dopamine.js 只记录实际奖赏（intensity），无期望基线。引入 RPE 是强化学习的核心机制，使 Brain 能区分"完成一件容易的事"和"完成一件难事"的不同价值，实现更精准的自驱学习。

**不做什么**：不修改 tasks 表 schema；不改 Self-Drive 调度逻辑；不更改已有 `recordReward` 函数签名。

## 成功标准

1. [ARTIFACT] `packages/brain/src/dopamine.js` 新增 `computeExpectedReward(taskType)`、`recordRPE(taskId, taskType, actualIntensity)` 和 `getRPEHistory(hours)` 三个导出函数
2. [BEHAVIOR] `scanAndReward` 在完成任务后自动调用 `recordRPE`，写入 `event_type='rpe_signal'` 事件到 `cecelia_events`，payload 包含 `{ task_id, task_type, actual, expected, rpe }` 字段

## 实现方案

**要改的文件**：
- `packages/brain/src/dopamine.js` — 新增 3 个函数 + 修改 `scanAndReward`
- `packages/brain/src/__tests__/dopamine.test.js` — 新增 RPE 相关测试

**Scope 锚定**：仅 dopamine.js 及其测试文件，不触碰其他模块。

**RPE 计算逻辑**：
```
computeExpectedReward(taskType):
  SELECT AVG(payload->>'actual') FROM cecelia_events
  WHERE event_type='rpe_signal' AND payload->>'task_type'=taskType
  限最近 30 条；无记录返回 0.5

recordRPE(taskId, taskType, actualIntensity):
  expected = await computeExpectedReward(taskType)
  rpe = actualIntensity - expected
  INSERT cecelia_events (event_type='rpe_signal', source='dopamine', payload={task_id, task_type, actual, expected, rpe})
  if rpe < 0: 读 brain_config.rpe_adjustments, 累加 rpe（下限 -3.0），写回

getRPEHistory(hours=24):
  SELECT FROM cecelia_events WHERE event_type='rpe_signal' AND created_at >= NOW()-INTERVAL
```
