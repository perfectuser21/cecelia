# PRD: worktree 活跃锁 — 防止 GC 误删活跃 session 的 worktree

## Task 信息

- **Task ID**: a5a35116-293d-4cd4-8c32-3a102565060c
- **Feature ID**: N/A
- **Order**: 0
- **来源**: Brain 数据库

## 功能描述

`worktree-gc.sh` 在删除 worktree 之前，只检查"有没有未提交改动"。如果一个 /dev session 正在运行，但改动都已 commit+push，worktree 是干净的，GC 就会把它删掉，导致正在运行的 session 崩溃。

本 PRD 引入活跃锁文件机制：在 worktree 内创建 `.dev-session-active`，GC 删除前检查这个文件。

**涉及文件**：
- `packages/engine/scripts/worktree-gc.sh` — 删除 worktree 前检查锁文件
- `packages/engine/scripts/stop-dev.sh` — session 启动时创建锁，结束时删除锁
- `.gitignore` — 将 `.dev-session-active` 加入忽略列表

## 成功标准

- [ ] `worktree-gc.sh` 在 worktree 目录中存在 `.dev-session-active` 文件时，跳过删除并输出 WARN

- [ ] `stop-dev.sh` 在 session 正常结束（`cleanup_done` 触发）时删除 `.dev-session-active`

### 异常处理

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| GC 遇到活跃锁 | 跳过删除，输出 WARN 日志 | P3 | worktree_gc_skip_active |
| 锁文件创建失败 | 降级继续运行（不阻塞 session 启动） | P3 | session_lock_create_fail |
