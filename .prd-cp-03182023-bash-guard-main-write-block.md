# PRD: bash-guard.sh 补漏 main 分支 Bash 写文件拦截

## 背景
bash-guard.sh 规则 3 只拦截特定代码扩展名的重定向写入，但 `git show <ref>:path > packages/xxx` 类命令可以绕过，直接在 main 分支覆盖源码。

## 目标
在规则 3 之后新增规则 3b：在 main/develop 分支时，拦截任何 Bash 重定向写入 packages/、apps/、scripts/、hooks/ 目录的操作。

## 成功标准
- bash-guard.sh 包含规则 3b
- main 分支时 `git show ... > packages/x` 被 exit 2 拦截
- cp-* 分支时放行
- engine CI 通过
