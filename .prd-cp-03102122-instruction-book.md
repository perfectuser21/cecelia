# PRD：Instruction Book — 用户层说明书体系

## 背景

Cecelia 现有文档体系（docs/current、docs/gaps）只服务于系统内部。
缺少面向使用者的"操作说明"：系统现在有什么能力、新功能怎么用、有什么命令/skill。
这是文档四层体系的最后一块缺口。

## 目标

1. 建立 `docs/instruction-book/` 目录作为用户层说明书
2. 在 `/dev` Step 6 代码写完后自动生成一条 instruction entry
3. 用明确的结构区分 skills 说明 vs features 说明

## 成功标准

1. `docs/instruction-book/README.md` 存在，描述 instruction book 结构
2. `docs/instruction-book/skills/` 和 `docs/instruction-book/features/` 目录存在
3. `packages/engine/skills/dev/steps/06-code.md` 包含"Instruction Book Update"步骤
4. 至少 3 个已有 skill/feature 有对应的 instruction entry（作为示例）
5. 每个 entry 格式统一：What / Trigger / How to use / Output / Added in

## 非目标

- 不写 runbook（运维操作手册是另一层）
- 不自动检查 instruction 是否"完整"（太主观）
- 不改 CI gate（这次只建基础设施）
