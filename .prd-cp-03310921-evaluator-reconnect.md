# PRD：修复对抗网络两个缺陷

## 背景
/dev Pipeline 存在两个未修复的缺陷，影响对抗网络的有效性。

## 需求

### 1. Sprint Contract 无限收敛（01-spec.md）
当前 MAX_ROUNDS=3，3轮后强制 FAIL。应改为无限收敛：
- 去掉 MAX_ROUNDS=3 上限
- 改为死循环检测：连续 2 轮分歧列表完全相同（无任何变化）→ 判定死循环 → 注册 P1 任务 → FAIL

### 2. 独立 Evaluator 接回 Stage 2（02-code.md）
playwright-evaluator.sh 脚本存在但断开。应在 Stage 2 重新接入：
- 位置：Generator 自验证之后、CRG 之前
- 调用：bash playwright-evaluator.sh <TASK_CARD_PATH> <BRANCH>
- FAIL → 打回 Generator 改代码（无次数上限）
- PASS → 继续 CRG

## 成功标准
- 01-spec.md Sprint Contract Gate 不再有固定轮数上限
- 02-code.md Stage 2 有独立 Evaluator 步骤（调用 playwright-evaluator.sh）
- 两处修改均有对应 DoD Test 验证
