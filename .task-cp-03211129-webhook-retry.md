# Task Card: cecelia-run webhook 回调重试机制

## 需求（What & Why）
**功能描述**: cecelia-run.sh 的 send_webhook 函数用 `|| true` 吞掉所有 curl 错误，导致 webhook 回调失败时 Brain 收不到状态更新，task 永久卡在 in_progress 状态。
**背景**: 这是 P0 级 bug。Brain 依赖 execution-callback 来更新 task 状态，回调丢失 = task 永远不会完成或失败。
**不做什么**: 不改 Brain 侧逻辑，只修复 cecelia-run.sh 的回调可靠性。

## 成功标准

1. [BEHAVIOR] send_webhook 失败时自动重试，最多 3 次，指数退避（2s/4s/8s）
2. [BEHAVIOR] 3 次全部失败后，payload 写入 /tmp/cecelia-callback-queue/<task_id>.json 供后续恢复
3. [BEHAVIOR] cleanup trap 中也调用 send_webhook，确保异常退出也有回调
4. [GATE] 不破坏现有正常流程（curl 成功时不重试、不延迟）

## 验收条件（DoD）

- [x] [BEHAVIOR] send_webhook 函数包含重试循环，最多 3 次，间隔 2s/4s/8s
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/scripts/cecelia-run.sh','utf8');if(!c.includes('retry') || !c.includes('sleep 2') || !c.includes('sleep 4') || !c.includes('sleep 8'))process.exit(1)"

- [x] [BEHAVIOR] 全部失败后写入 /tmp/cecelia-callback-queue/<task_id>.json
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/scripts/cecelia-run.sh','utf8');if(!c.includes('cecelia-callback-queue') || !c.includes('.json'))process.exit(1)"

- [x] [BEHAVIOR] cleanup trap 中调用 send_webhook 确保异常退出也有回调
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/scripts/cecelia-run.sh','utf8');const cleanup=c.substring(c.indexOf('cleanup()'),c.indexOf('trap cleanup'));if(!cleanup.includes('send_webhook'))process.exit(1)"

- [x] [GATE] 脚本语法正确且不破坏现有结构
  Test: manual:bash -n packages/brain/scripts/cecelia-run.sh

## 实现方案
**要改的文件**: packages/brain/scripts/cecelia-run.sh
**Scope 锚定**: 只改 send_webhook 函数体 + cleanup 函数体，不动其他逻辑
