# Task Card: DoD Test 字段强度升级

## 需求（What & Why）

**功能描述**: 升级 `check-dod-mapping.cjs` 的 Test 字段强度检查，并更新 `DOD-TEMPLATE.md` 模板，确保每条 DoD Test 字段都是可执行的 node/curl 命令（exit 0/1），禁止模糊的弱测试。

**背景**:
- 当前 `detectFakeTest()` 缺少 printf、ls、cat、true、exit 0、standalone grep 等弱测试模式
- `validateBehaviorTestStrength()` 只对 [BEHAVIOR] 条目生效，其他条目没有保护
- 模板仍然展示旧的 evidence-id 占位格式，误导 AI 写无法执行的占位符

**不做什么**: 不修改 `check-fake-dod-tests.cjs`，不修改 CI 流程，不改动 `check-manual-cmd-whitelist.cjs`。

## 成功标准

- 升级后的 `check-dod-mapping.cjs` 能拦截 printf/ls/cat/true/exit 0/standalone grep 弱测试
- `DOD-TEMPLATE.md` 展示标准可执行 Test 命令写法示例
- 现有合法 Test 格式（node/curl/bash/tests/contract）不受影响
