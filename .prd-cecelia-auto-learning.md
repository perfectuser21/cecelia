## PRD - Cecelia 自动学习回路

**需求来源**: 闭合 Cecelia 的学习回路。任务完成/失败时自动产生 learning，反刍消化后更新 self-model，让 Cecelia 从自己的行为中学习成长。

**功能描述**: 修改 execution-callback 端点，在任务完成/失败时自动写入 learnings 表，让 Cecelia 从任务执行结果中自动学习，而不是完全依赖用户手动输入。

**涉及文件**:
- `packages/brain/src/execution-callback.js` （或相应的 routes.js 文件中的 /execution-callback 端点）
- 单元测试文件
- 可能涉及数据库表 learnings

**成功标准**:
1. 任务 status=completed 时自动创建 category='execution_result' 的 learning 记录
2. 任务 status=failed 时自动创建 category='failure_pattern' 的 learning 记录
3. 只记录 task_type IN ('dev', 'feature', 'research')，跳过 code_review 等高频低价值类型
4. 利用 content_hash 自动去重，避免重复内容
5. 每日上限 50 条，防止过度学习
6. 不影响现有 execution-callback 的任务状态更新和 WebSocket 广播
7. 单元测试覆盖所有场景

**非目标**:
- 不修改现有的手动 learning 输入机制
- 不改变 learnings 表结构
- 不影响反刍消化 (rumination) 流程
