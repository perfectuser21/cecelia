# PRD: Cecelia Knowledge & Documentation System

## 背景

Cecelia 系统缺乏统一知识沉淀层。决策、设计、PR记录、学习全部散落在数据库/对话/文件中，用户和 AI 都无法通过单一入口访问历史信息。每次新对话都从零开始，导致认知断层。

## 目标

在现有 Dashboard (apps/dashboard) 中构建一套完整的文档系统，实现：
- 用户在前台可浏览、标注、备注所有信息
- Cecelia 自动写入决策、设计、PR记录、日报
- 人机共享统一知识库，系统"记忆"不随对话消失

## 核心模块

1. **Dev Log** — 每个 PR 一条记录，完整生命周期
2. **Decision Registry** — 所有重要决策，带原因、时间
3. **Design Vault** — 调研报告、架构方案、技术评估存档
4. **Daily Diary** — 每晚自动生成日报
5. **Knowledge Map** — 所有主题的索引入口

## 成功标准

- 用户打开 `/knowledge/dev-log` 能看到历史 PR 完整记录页面正常渲染
- 用户打开 `/knowledge/diary` 能看到每日日报时间线页面正常渲染
- `dev_records`、`design_docs`、`user_annotations` 三张表已创建
- PR 合并后回调自动写入 dev_records
- 用户添加的标注持久保存
- Brain API 3组新端点响应正常
- 每日 15:00 UTC（23:00 上海）tick 自动生成日报
- Brain CI + Workspace CI 全部通过

## 不做什么

- 不做历史 PR 数据回填
- 不引入新的 UI 组件库
- 不做完整 Markdown 编辑器
