# PRD: 修复 heartbeat 垃圾循环（路由错误 + 防重复漏洞）

## 背景
dept_heartbeat 任务不在 executor.js 的 US_ONLY_TYPES 集合中，被错误路由到 Codex Bridge 执行失败。同时 dept-heartbeat.js 防重复检查只看 status IN ('queued', 'in_progress')，不看 'quarantined'，导致隔离后立刻创建新任务。一晚上产生 40+ 条垃圾 heartbeat 任务。

## 成功标准
1. dept_heartbeat 任务在美国本机执行，不再路由到 Codex Bridge
2. quarantined 状态的 heartbeat 任务被视为"已存在"，不会重复创建
3. 现有 quarantined heartbeat 任务被清理为 canceled

## 修复方案
1. 在 executor.js 的 US_ONLY_TYPES 集合中加入 'dept_heartbeat'
2. 在 dept-heartbeat.js 的防重复 SQL 中加入 'quarantined' 状态
3. 创建清理 SQL 脚本
