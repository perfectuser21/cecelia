# Task Card: 内容飞轮 I3 — 发布后数据回收

## 需求（What & Why）
**功能描述**: content_publish 任务完成 4 小时后，自动触发对应平台的数据 scraper，将阅读/点赞/播放等指标写入 pipeline_publish_stats 表，并通过 API 接口对外暴露。
**背景**: 内容发布后没有自动回收各平台数据的机制，导致内容飞轮缺少效果反馈环节。现有 scraper 脚本已就绪但未与 pipeline 关联。
**不做什么**: 不实现 scraper 脚本本身（已有）；不做实时数据同步（仅 4h 延迟单次采集）；不做各平台账号登录管理；不做数据可视化。

## 成功标准
> [ARTIFACT] = 产出物（文件/API/配置）
> [BEHAVIOR] = 运行时行为（系统如何响应输入）
> [GATE] = 门禁（CI/测试/代码质量）

1. [ARTIFACT] pipeline_publish_stats 表的 migration SQL 文件存在
2. [ARTIFACT] 发布后数据回收模块（触发器逻辑）文件存在
3. [BEHAVIOR] content_publish 任务完成 4 小时后，对应平台 scraper 被自动调用，结果写入 pipeline_publish_stats 表
4. [BEHAVIOR] GET /api/brain/pipelines/:id/stats 接口返回该 pipeline 各平台的数据汇总（views/likes/comments/shares）
5. [GATE] CI 全部通过
