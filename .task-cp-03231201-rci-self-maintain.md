# Task Card: RCI 自维护闭环——增/删/改全自动强制

## 需求（What & Why）

**功能描述**: 建立 RCI 完整自维护机制，彻底消除人工维护需求。三个闭环：
- **闭环1（删/改）**: `check-rci-stale-refs.cjs` — 扫描 regression-contract.yaml 中所有 `file:` / `test: tests/` 引用是否指向真实存在的文件，有悬空引用即 CI 拒绝合并
- **闭环2（增）**: `detect-priority.cjs` 新增文件路径自动识别 — 改动 hooks/*.sh / lib/devloop-check.sh 等核心文件时自动升级为 P0/P1，触发已有 `require-rci-update-if-p0p1.sh` 强制要求更新 RCI
- **闭环3（CI接入）**: ci-l1-process.yml 新增 stale refs 检查步骤，每次 PR 都跑

**背景**: 现有 RCI 186条，但没有机制防止条目变成死条目（引用的文件被删除/重命名）；P0/P1 判断依赖人工标注，实际几乎不触发，导致 RCI 只增不减不改。

**不做什么**: 不做"新增能力时自动生成 RCI 条目"（需要 AI 理解语义，超出自动化范围）。

## 成功标准

1. [ARTIFACT] `check-rci-stale-refs.cjs` 存在且能扫描出 regression-contract.yaml 中引用不存在文件的条目
2. [BEHAVIOR] 当 regression-contract.yaml 有悬空 file 引用时，`check-rci-stale-refs.cjs` 返回非零退出码
3. [BEHAVIOR] `detect-priority.cjs` 当改动文件包含 `hooks/verify-step.sh` 等核心路径时，自动返回 P0
4. [GATE] CI L1 新增 stale refs 检查步骤，当前 regression-contract.yaml 全部通过（186条无悬空）
