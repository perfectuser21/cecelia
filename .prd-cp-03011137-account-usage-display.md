# PRD: Account Usage 显示增强

## 目标
在 Cecelia Dashboard 的 Live Monitor 页面，为三个 Claude Max 账号显示完整的用量数据。

## 背景
当前 Dashboard 只显示 5h 用量和 7d 用量，缺少 7d Sonnet 专属用量，且字段含义不清晰。
用户在 Anthropic 网站看到的数据包括：
- CURRENT SESSION（= five_hour，当前5小时窗口）
- CURRENT WAKE ALL MODELS（= seven_day）
- CURRENT WAKE SONNET ONLY（= seven_day_sonnet）

## 需求

### 1. DB Migration（packages/brain/migrations/）
给 account_usage_cache 表新增列：
- seven_day_sonnet_pct DOUBLE PRECISION NOT NULL DEFAULT 0
- seven_day_resets_at TIMESTAMPTZ（7天的重置时间）

### 2. account-usage.js（packages/brain/src/）
- upsertCache：解析并存储 seven_day_sonnet_pct 和 seven_day_resets_at
- 返回对象带上这两个新字段
- 缓存 TTL 从 10 分钟改为 3 分钟

### 3. Dashboard（apps/dashboard/src/pages/live-monitor/LiveMonitorPage.tsx）
AccUsageRings 组件每个账号显示：
- Ring 主值：five_hour_pct（当前限速最直接指标）
- 行1："5h: 35% ↺14:59"
- 行2："7d: 48% ↺3/6"
- 行3："son: 69%"

## 成功标准
- Dashboard 显示三行数据，含重置时间
- 数字与 Anthropic 网站数据一致（缓存 3min 内）
