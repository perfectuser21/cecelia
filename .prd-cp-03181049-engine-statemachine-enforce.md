# PRD: Engine 状态机代码层强制（Step 验证门禁）

## 背景

/dev Verifier Subagent 状态机从 PR #962 起就存在于 02-code.md 提示词里，
但从未在 Engine 脚本层实现过。AI 可以自报 step_N: done 而不接受任何代码验证。
devloop-check.sh 只检查 PR/CI/step_10/step_11，对 Step 1-9 完全盲目。

## 目标

在 branch-protect.sh（PreToolUse Hook）里拦截 .dev-mode 写入，
对关键步骤运行真实验证脚本，使状态机真正存在于代码层。

## 成功标准

- verify-step.sh 存在且覆盖 step1/step2/step4 三个验证
- branch-protect.sh 在 .dev-mode 写入时调用 verify-step.sh
- 验证失败时阻止写入并输出具体错误信息
- 所有现有测试通过

## 不做什么

- 不改变 devloop-check.sh 现有逻辑
- 不改动 02-code.md 提示词
- 不要求每条 DoD 都有 Verifier Subagent（提示词层的事）
