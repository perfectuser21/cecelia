# Task Card: 升级 /dev Sprint Contract 机制：双向协商 DoD + Evaluator Calibration

## 需求（What & Why）
**功能描述**: 升级 /dev 的三个核心组件，实现 GAN 式对抗验证循环：
1. code_review_gate：加入 Evaluator Calibration（few-shot 锚定示例），强化 blocker 零容忍裁决
2. spec_review：加入双向协商机制（Sprint Contract），subagent 独立写测试方案后与主 agent 比对
3. packages/engine/skills/dev/steps/01-spec.md：Stage 1 末尾加 Sprint Contract Gate（硬门禁）

**背景**: 解决「自我认证」问题——主 agent 写 DoD Test 字段后自己审查，存在主观偏差。Sprint Contract 要求 Generator 和 Evaluator 必须对同一件事达成共识，才能开始写代码。

**不做什么**: 不改 /dev 整体流程结构，不改 CI 配置，不新增工具脚本，不改 Stage 2-4。

## 成功标准

- [ARTIFACT] code_review_gate SKILL.md 包含 Evaluator Calibration 章节，有 3 个锚定示例（明确 FAIL + 明确 PASS + 边界案例）
- [ARTIFACT] spec_review SKILL.md 包含双向协商机制（Sprint Contract）章节，描述 subagent 独立测试方案生成 + 比对逻辑
- [ARTIFACT] packages/engine/skills/dev/steps/01-spec.md Stage 1 末尾有 Sprint Contract Gate 硬门禁说明
- [BEHAVIOR] spec_review 输出格式包含 independent_test_plans 字段，与主 agent Test 字段比对结果
- [GATE] CI 全部通过
