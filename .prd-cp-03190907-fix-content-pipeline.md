# PRD: 修复内容工厂 Pipeline 列表加载错误

## 背景

GET /api/brain/pipelines 报错 `column "failed_at" does not exist`，导致前台内容工厂页面 Pipeline 列表无法加载。

## 问题根因

`packages/brain/src/routes/content-pipeline.js` 的 GET 查询 SELECT 了 `failed_at` 字段，但 tasks 表中该列不存在。tasks 表只有 `completed_at`，失败时由 orchestrator 写 `status='failed', completed_at=NOW()`。

## 修复范围

移除 GET /api/brain/pipelines SQL 中的 `failed_at` 字段。

## 成功标准

1. GET /api/brain/pipelines 返回正常 JSON 数组，不报 SQL 错误
2. 前台内容工厂页面 Pipeline 列表正常加载
3. CI 全部通过
