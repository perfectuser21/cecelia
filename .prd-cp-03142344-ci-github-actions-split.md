# PRD：CI 分流 — L1 轻量 job 迁移到 GitHub Actions

## 背景

香港 VPS（4核 8GB）运行 6 个 cecelia self-hosted runner，CPU load 长期在 11-12（满载 3 倍）。
分析发现 L1/L2/L3 的大量 job 不需要 npm install，完全可以跑在 GitHub-hosted runner 上，
腾出 HK VPS CPU 给真正需要本地环境的重任务。

仓库为公开仓库（将来私有化），GitHub Pro 用户有 3000 分钟/月配额。
轻量 job 每个 10-25s，按 25 PR/天估算约 1500 分钟/月，在配额内。

## 目标

将 ci-l1-process.yml 中不需要 npm install 的轻量 job 迁移到 `ubuntu-latest`，
减少 HK VPS CPU 压力，同时不影响 CI 功能正确性。

## 改动范围

**文件**：`.github/workflows/ci-l1-process.yml`（在 cecelia 仓库）

**改动**：以下 job 的 `runs-on` 从 `[self-hosted, Linux, hk-vps]` 改为 `ubuntu-latest`：

1. `changes`（Detect Changes）
2. `verify-dev-workflow`（Branch Name Check）
3. `cleanup-check`（Cleanup Check）
4. `check-prd`（PRD Success Criteria Check）
5. `check-learning`（Learning Format Gate）
6. `required-paths-check`（Required Dev Paths Check）
7. `ci-config-audit`（CI Config Audit）
8. `dev-health-check`（Dev Health Gate）
9. `l1-passed`（L1 Process Gate Passed）

**保留 hk-vps 的 job**（需要 npm install）：
- `dod-check`
- `quality-meta-tests`
- `engine-l1`

## 成功标准

- [ ] 改动后 L1 CI 正常通过
- [ ] 轻量 job 在 ubuntu-latest 上运行
- [ ] npm 重任务仍在 hk-vps 上运行
