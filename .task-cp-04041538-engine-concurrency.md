# Task Card: Engine 并发加固三件事

## 需求（What & Why）
**功能描述**: 为多 agent 并发场景加固 engine，消灭双拷贝漂移、修复 headless 会话匹配、补 macOS 锁 fallback
**背景**: 系统将来必然跑并发，当前有三处可靠性缺口会在并发下放大
**不做什么**: 不改 devloop-check.sh 条件逻辑，不动 Brain 侧调度

## 成功标准

- `hooks/` 根目录改为指向 `packages/engine/hooks/` 的目录软链接，两份合一
- stop-dev.sh `_session_matches` 的 headless fallback 去掉 `lock_session 必须为空` 的错误约束
- stop-dev.sh 并发锁在 macOS（无 flock）下用 `mkdir` 原子锁替代

## 验收条件（DoD）

- [x] [ARTIFACT] hooks/ 是指向 packages/engine/hooks/ 的软链接
  Test: manual:node -e "const fs=require('fs');const s=fs.lstatSync('hooks');if(s.isSymbolicLink()&&fs.readlinkSync('hooks')==='packages/engine/hooks')console.log('OK');else process.exit(1)"

- [x] [BEHAVIOR] stop-dev.sh headless fallback 不再要求 lock_session 为空
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/stop-dev.sh','utf8');if(c.includes('-z \"\$lock_session\"'))process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] stop-dev.sh 含 mkdir 原子锁 fallback
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/stop-dev.sh','utf8');if(!c.includes('mkdir'))process.exit(1);console.log('OK')"

## 实现方案
**要改的文件**:
1. `hooks/` — `git rm -r` 后创建 symlink `hooks -> packages/engine/hooks`
2. `packages/engine/hooks/stop-dev.sh:34-36` — 删除 `&& -z "$lock_session"` 约束
3. `packages/engine/hooks/stop-dev.sh:79-84` — 加 else 分支用 mkdir 实现锁

**不改什么**: devloop-check.sh 逻辑、CI 配置、其他 hook 文件
