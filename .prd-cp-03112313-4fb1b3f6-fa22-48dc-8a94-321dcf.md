# PRD - tick.js 连接 billing pause 熔断 — dispatch 检查 + quota_exhausted requeue

## 需求来源
Brain 自动调度任务（dev 类型）

## 功能描述
tick.js 已导入 getBillingPause()，但 Step 7 派发循环从未调用它。setBillingPause 在 execution-callback（billing_cap 失败时）被正确设置，但无任何代码消费这个暂停信号。结果：配额耗尽后系统仍持续派发 → 批量 quota_exhausted 失败。

## 成功标准

- tick.js dispatchNextTask 调用前检查 getBillingPause()
- 若 pause.active=true：log + 跳过本次派发
- billing pause 自动过期后，自动将 quota_exhausted 任务 requeue 回 queued
- Brain API GET /api/brain/billing-pause 返回当前暂停状态
- 单元测试：billing pause active 时零派发；inactive 时正常派发

## 技术路径

### 1. tick.js - dispatch 前检查（核心，约 10 行）
```js
const pause = getBillingPause();
if (pause.active) { log + return { dispatched: 0, reason: 'billing_pause' }; }
```

### 2. tick.js - 暂停过期后 requeue quota_exhausted
```sql
UPDATE tasks SET status='queued' WHERE status='quota_exhausted'
```
（getBillingPause 过期后下一 tick 触发）

### 3. routes.js - GET /api/brain/billing-pause 可观测端点

## 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| billing_pause active | 跳过派发，记录日志 | P2 | billing_pause_dispatch_skip |
| quota_exhausted requeue 失败 | 日志记录，下次 tick 重试 | P2 | quota_exhausted_requeue_error |
