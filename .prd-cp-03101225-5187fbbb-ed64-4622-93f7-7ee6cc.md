# PRD - 修复 slot-allocator detectUserSessions headless/headed 误判

## 需求来源

Brain 自动调度（agent_ops）— 2026-03-10 夜间派发积压根因排查

## 功能描述

macOS 上 claude 子进程覆盖进程标题，导致 ps -o args= 只显示 `claude`（不含 -p）。
detectUserSessions() 用 `/ -p /.test(args)` 判断 headless，将无头任务进程误归类为 headed。

**修复**：通过 PPID 链接识别 headless 进程。
cecelia-run bash 脚本的 args 包含 `CECELIA_HEADLESS=true`，
对每个 claude 进程，检查其父进程（PPID）的 args，
若含 `CECELIA_HEADLESS=true` 则归类为 headless。

## 涉及文件

- `packages/brain/src/slot-allocator.js`（核心修复）
- `packages/brain/src/__tests__/slot-allocator.test.js`（新增测试）

## 成功标准

- headless 进程不再被计入 user.used
- 4 个无头任务运行时 user.mode = "absent"
- taskPool.budget 恢复正常，dispatchAllowed = true
- 单元测试覆盖 PPID 检测路径

## 非目标

- 不修改 platform-utils.js（listProcessesWithPpid 已存在，直接复用）
- 不修改 Linux 路径（Linux 上 ps args 包含完整参数，原有检测已正确）

## 异常处理

N/A - 无 IO 操作（只是在已有进程列表上做查找）
