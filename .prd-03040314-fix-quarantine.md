## PRD - 修复 quarantine.js 三处逻辑错误

**需求来源**: Cecelia Brain 反思 - initiative_plan 管道枯竭问题

**功能描述**:
修复 `packages/brain/src/quarantine.js` 中三处逻辑错误，恢复 initiative_plan 任务的自动释放流程。

**根因分析**:
1. **释放逻辑缺失**: `releaseFromQuarantine` 只更新状态为 'queued'，未恢复原有 priority/assignee/queued_at
2. **KR_ID 覆盖错误**: `reprioritizeTask` 错误地用 `task.kr_id` 覆盖了 `task.initiative_id`，导致派发失败
3. **活跃检测不完整**: `fetchActiveQuarantine` 只检测 `failure_count` 和 `last_failure_at`，未检测 `ready_for_release`

**涉及文件**:
- `packages/brain/src/quarantine.js`（3 处修复）
- `packages/brain/tests/quarantine.test.js`（新增测试用例）

**成功标准**:
1. ✅ `releaseFromQuarantine` 恢复任务原有 priority/assignee/queued_at
2. ✅ `reprioritizeTask` 保留 `task.initiative_id`，只更新 priority
3. ✅ `fetchActiveQuarantine` 返回所有活跃隔离任务（包括 ready_for_release=true）
4. ✅ 所有测试通过（包括新增测试）
5. ✅ CI 通过

**非目标**:
- 不修改隔离策略本身（熔断阈值、升级规则等）
- 不修改 Brain 其他模块
- 不改变 API 接口

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| 释放任务时 DB 更新失败 | 抛出异常，记录日志 | P1 | quarantine_release_failed |
| 获取隔离任务时 DB 查询失败 | 返回空数组，记录日志 | P1 | quarantine_fetch_failed |
| 重新分配优先级时更新失败 | 抛出异常，记录日志 | P1 | quarantine_reprioritize_failed |
