# Task Card: 记忆系统修复 — 对话写入 memory_stream（带情绪标签+显著性评分）

## 需求（What & Why）

**功能描述**:
Cecelia 的对话记忆管道缺陷：写入 memory_stream 的对话记录无 salience_score/emotion_tag，
且 memory-retriever 从 cecelia_events（无 embedding）读取对话历史，无法语义检索。

**背景**: 对话是信息密度最高的输入。纠正/决策类对话（RPE 大）应有更高 salience，
优先进入语义检索层，使 Cecelia 能"记住"重要的用户反馈。

**不做什么**:
- 不删除 cecelia_events recordChatEvent 调用（保留向后兼容）
- 不修改其他 source_type 的 memory_stream 写入逻辑
- 不修改 memory_stream 表的现有约束和索引

## 成功标准

1. [ARTIFACT] memory_stream 表新增 `salience_score FLOAT` 和 `emotion_tag TEXT` 两列（迁移文件 186）
2. [ARTIFACT] selfcheck.js EXPECTED_SCHEMA_VERSION 更新为 '186'
3. [BEHAVIOR] orchestrator-chat.js 的 memory_stream INSERT 改用 source_type='conversation_turn' 并携带 salience_score/emotion_tag
4. [BEHAVIOR] salience_score：纠正类→0.9，决策类→0.8，疑问类→0.6，普通→0.3

7. [BEHAVIOR] memory-retriever loadConversationHistory 主路径改用 memory_stream（source_type='conversation_turn'），空时 fallback cecelia_events
8. [GATE] CI 全部通过

### D2 — computeSalience 函数
在 `packages/brain/src/orchestrator-chat.js` 新增并导出：
```js
export function computeSalience(message) {
  const text = message || '';
  if (/不对|错误|不是|纠正|改一下|重新|你搞错|说错/.test(text)) return 0.9;
  if (/决定|确认|就这样|采纳|方案确定|好的就/.test(text)) return 0.8;
  if (/？|\?|吗|为什么|怎么|什么/.test(text)) return 0.6;
  return 0.3;
}
```

const msResult = await pool.query(`

if (msResult.rows.length > 0) {
