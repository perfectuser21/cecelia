# PRD: 修复 L4 CI PostgreSQL 连接失败

## 背景

L4 CI (Brain Integration Tests) 在 `node src/migrate.js` 步骤报 `ECONNREFUSED`。
根因：动态计算的 `DB_PORT` 环境变量未可靠传递到 migrate 步骤，且缺少 PostgreSQL 就绪检查。

## 成功标准

1. [BEHAVIOR] migrate 步骤之前必须有 `pg_isready` 循环等待，确认 PostgreSQL 可连接后才执行
2. [BEHAVIOR] migrate 步骤必须显式设置 `DB_PORT` 环境变量，不依赖隐式 GITHUB_ENV 传递
3. [BEHAVIOR] 如果 `DB_PORT` 为空，必须从 `GITHUB_RUN_ID` 和 `shard_num` 重新计算（防御性回退）
4. 清理脚本的 `ipcs` 解析兼容 macOS 和 Linux 格式
5. 测试步骤同样显式传递 `DB_PORT`

## 范围

- `.github/workflows/ci-l4-runtime.yml`

## 风险

- 低：只修改 CI 配置，不影响生产代码
