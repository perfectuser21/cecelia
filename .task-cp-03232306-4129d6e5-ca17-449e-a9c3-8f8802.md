# Task Card: execution.js strategy_session KR 补 objective_id

## 需求（What & Why）
**功能描述**: 修复 `packages/brain/src/routes/execution.js` 的 strategy_session 回调，INSERT INTO key_results 时必须提供 objective_id，否则创建的 KR 成为孤岛（没有挂到任何 Objective 下）。

**背景**: OKR 迁移后，key_results.objective_id 是关联到 objectives 表的外键。strategy_session 产生的 KR 需要挂到 Objective，才能在 OKR 体系中正常显示和管理。

**不做什么**: 不重构整个 strategy_session 处理流程，只补 objective_id 关联逻辑。

## 成功标准

- strategy_session 创建的 key_results 记录必须有 objective_id（不能为 null）
- 需要在 INSERT 前先查找或创建关联的 objective
- 若输出 JSON 携带 objective_id 则优先用；否则按优先级 fallback

## 实现方案

**要改的文件**: `packages/brain/src/routes/execution.js`（仅此文件）

**Scope 锚定**: 第 1006-1018 行（`for (const kr of krs)` 循环内的 INSERT）

**逻辑**：
1. 先检查 `parsedOutput.objective_id`（策略会议 JSON 顶层携带时优先用）
2. 若无，检查每个 `kr.objective_id`
3. 若仍无，从 task 的 payload.objective_id 查（需扩展 ssTaskResult 查询）
4. 若仍无，INSERT INTO objectives（title 取自 meeting_summary 前80字或 task title）得到 id
5. 将 objectiveId 传入 INSERT INTO key_results
