# PRD - feat(cortex): 添加 LLM 调用详细计时与 prompt token 估算日志

## 背景

Cortex 频繁超时但缺乏可观测性——不知道是 prompt 太长（token超限）、模型响应慢（quota），还是网络问题。

## 目标

在 cortex.js callLLM 调用前后记录关键指标，输出到 cortex_log/decision_log，为根因分析提供数据基础。

## 成功标准

- [ ] 每次 Cortex 调用记录：prompt 估算 token 数、响应耗时(ms)、是否超时、超时类型
- [ ] 数据存入 cortex_executions 表或 decision_log.metadata
- [ ] /api/brain/cortex/stats 端点返回最近 24h 的超时率和平均耗时
- [ ] 单元测试覆盖计时逻辑

## 技术要点

- 用字符数 ÷ 4 快速估算 token（无需调 tokenizer API）
- 记录字段：prompt_tokens_est, response_ms, timed_out, error_type
- 不影响现有 Cortex 调用路径性能
