# PRD: 修复 consolidation.js 查询 Bug

## 问题一：tasks 查询用了不存在的 failed_at 字段

`gatherTodayData` 里查 tasks：
```sql
COALESCE(completed_at, failed_at) AS ended_at
WHERE (completed_at >= $1::date OR failed_at >= $1::date)
```

但 tasks 表没有 `failed_at` 字段（只有 `completed_at` 和 `updated_at`）。
失败任务应用 `updated_at` 代替。

## 问题二：memory_stream source_type 过滤遗漏实际数据

当前过滤：`'chat', 'task_reflection', 'conversation_insight', 'failure_record', 'user_fact'`

但数据库实际存在的类型：
- `feishu_chat`（130条/天）
- `orchestrator_chat`（38条/天）
- `narrative`（情绪叙事）

这三类才是 Cecelia 每日活动的主体，当前过滤导致 memories 永远为空。

## 修复方案

1. tasks 查询：`failed_at` → `updated_at`，保留 completed 和 failed 两种 status
2. memory_stream 过滤：追加 `feishu_chat`, `orchestrator_chat`, `narrative`
3. 更新测试：调整 mock 数据中的 source_type 和 tasks 查询期望
