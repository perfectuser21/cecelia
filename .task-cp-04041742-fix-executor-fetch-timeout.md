# Task Card: fix(executor): trigger-cecelia fetch 添加 AbortSignal.timeout

## 需求（What & Why）

**功能描述**: 为 executor.js 中的 `/trigger-cecelia` HTTP 请求添加 30 秒超时。
**背景**: Brain tick 调用 `triggerCeceliaRun` → `fetch(EXECUTOR_BRIDGE_URL/trigger-cecelia)`，该 fetch 没有设置任何超时或 AbortSignal。当 cecelia-bridge 不响应时（无 session 可用、bridge 重启等），Promise 永不 resolve，`_tickRunning=true` 永久保持，后续所有 tick 请求被跳过，事件循环被阻塞，导致 Brain HTTP 完全无响应。

**不做什么**: 不改 tick 调度逻辑，不改 bridge 协议，不修改其他 fetch 调用。

## 成功标准

> [BEHAVIOR] fetch 超时后，tick 能正常完成并释放 `_tickRunning`

## 验收条件（DoD）

- [x] [BEHAVIOR] executor.js 的 /trigger-cecelia fetch 包含 signal: AbortSignal.timeout(30000)
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/executor.js','utf8');if(!c.includes('AbortSignal.timeout(30000)'))process.exit(1);console.log('ok')"

- [x] [BEHAVIOR] AbortError 被捕获并作为 executor 错误处理（不 crash Brain）
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/executor.js','utf8');if(!c.includes('AbortError'))process.exit(1);console.log('ok')"

- [x] [ARTIFACT] TICK_AUTO_RECOVER_MINUTES 默认值从 5 提升到 60（避免频繁自动恢复触发同样问题）
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/tick.js','utf8');if(!c.includes(\"'60'\"))process.exit(1);console.log('ok')"

## 实现方案

**要改的文件**:
- `packages/brain/src/executor.js` line ~2729: 添加 `signal: AbortSignal.timeout(30000)`，在 catch 块处理 AbortError
- `packages/brain/src/tick.js` line ~78: `TICK_AUTO_RECOVER_MINUTES` 默认值从 '5' 改为 '60'

**受影响函数/API**: `triggerCeceliaRun()` in executor.js, `initTickLoop()` in tick.js
**不改什么**: bridge 协议、tick 调度频率、其他 fetch 调用
