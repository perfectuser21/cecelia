# Task Card: Brain 部署流程可观测性补齐

## PRD

### 背景
Brain deploy webhook (`POST /api/brain/deploy`) 是 fire-and-forget 设计：
返回 202 后异步执行 `deploy-local.sh`，但没有任何状态追踪机制。
GitHub Actions `deploy.yml` 只验证 HTTP 响应码，无法知道部署是否真正成功。

### 目标
补齐最小可用的部署可观测性：
1. Brain 记录最近一次部署的状态（running/success/failed）
2. 暴露 `GET /api/brain/deploy/status` 端点供 CI 查询
3. `deploy.yml` 在触发 webhook 后 poll status 确认部署完成

### 范围
- `packages/brain/src/routes/ops.js` — 加状态追踪 + GET 端点
- `.github/workflows/deploy.yml` — 加 poll-for-deploy-success 步骤
- `packages/brain/src/__tests__/` — 加 deploy status 测试

---

## 成功标准
- `GET /api/brain/deploy/status` 返回 `{status, version, started_at, finished_at, elapsed_ms}`
- `POST /api/brain/deploy` 成功后，status 变为 `success`
- `POST /api/brain/deploy` 失败后，status 变为 `failed` 并含 `error` 字段
- `deploy.yml` 在 webhook 后 poll status（最多 300s），超时则 fail
- 新增单元测试验证 status 状态流转
