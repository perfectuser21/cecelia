# Task Card: decomp-checker type 匹配修正（area_okr 识别）

## 需求（What & Why）
**功能描述**: decomposition-checker.js 的 Check A/B/C 只搜索 `type IN ('area_kr', 'kr')`，但 goals 表中实际的 KR 数据 type 为 `area_okr`，导致 decomp-checker 完全找不到可拆目标。

**背景**: goals 表有 10 个 in_progress 的 area_okr（均有具体可量化指标，实质是 KR 级别），但 decomp-checker 的三个 Check 都只搜索 area_kr/kr 类型，完全忽略 area_okr。系统其他模块（kr-progress.js、planner.js、similarity.js、validate-okr-structure.js）已经正确包含 area_okr 作为 KR 类型。

**不做什么**: 不修改 goals 表数据本身的 type 字段，只修复 decomp-checker 的查询条件。

## 成功标准
1. [BEHAVIOR] decomp-checker Check A 查询条件包含 area_okr 类型
2. [BEHAVIOR] decomp-checker Check B 查询条件包含 area_okr 类型
3. [BEHAVIOR] decomp-checker Check C 查询条件包含 area_okr 类型
4. [BEHAVIOR] decomp-checker Check D 的子 KR 搜索条件包含 area_okr 类型
5. [GATE] 所有现有测试通过
