# PRD: 扫描 Learning 提炼工程规则 → 写入 Brain 知识层

## 背景
docs/learnings/ 下积累了约 146 条工程经验（每次 PR 失败/修复的根本原因 + 预防措施），但这些知识没有被系统化利用。Brain 在决策时无法引用这些教训，导致同类错误反复出现。

## 目标
将 learning 提炼为结构化可执行规则，写入 Brain 知识层，使 Brain 能在派发任务时自动参考。

## 技术要点
1. **扫描**：读取 `docs/learnings/*.md` 所有文件，提取 `### 下次预防` 章节内容
2. **提炼**：每条 `- [ ]` checklist 项生成一个规则条目写入 knowledge 表
3. **写入**：直接 INSERT 到 knowledge 表（upsert，幂等）
4. **验证**：GET /api/brain/knowledge?type=learning_rule 能返回所有规则

## 实现方案
- 新脚本 `packages/brain/scripts/distill-learnings.js`
- 新路由 `packages/brain/src/routes/knowledge.js`
- 注册到 `packages/brain/server.js`
- 新 migration 155：(type, sub_area) 唯一索引

## 成功标准
- [ ] ≥ 25 条规则被提炼并写入 Brain knowledge 层
- [ ] GET /api/brain/knowledge?type=learning_rule 返回规则列表
- [ ] 规则写入幂等（重复运行不重复插入）
- [ ] CI 通过
