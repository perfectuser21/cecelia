# Task Card: fix(ci): L3 覆盖率门禁扩展 fix/refactor + L4 task-lifecycle E2E

## 需求（What & Why）

**功能描述**:
1. `ci-l3-code.yml` 覆盖率检查扩展到 fix/refactor PR：目前 `check-changed-coverage.cjs` Gate 3（变更行覆盖率）对 fix/refactor PR 在有覆盖率数据时会跳过。需扩展：fix/refactor PR 在变更行存在时，Gate 3 必须强制执行（有覆盖率数据则检查，无数据则 warn-only）。
2. 新增 L4 task-lifecycle E2E：在 `packages/brain/scripts/task-lifecycle-check.sh` 新增完整链路测试（创建任务 → queued → in_progress → completed），并加入 `ci-l4-runtime.yml` shard 1/4。

**背景**: CI 质量保障增强，fix/refactor PR 也需要测试覆盖背书；L4 任务生命周期 E2E 保障核心业务流不被破坏。

**不做什么**: 不修改 Gate 1（feat 必须有新增测试文件）；不修改 Gate 2（测试 import 源码）；不修复 PATCH 响应的 emitEvent bug（E2E 通过 GET 验证状态）。

3. [ARTIFACT] `packages/brain/scripts/task-lifecycle-check.sh` 存在且可执行
4. [ARTIFACT] `ci-l4-runtime.yml` shard 1/4 包含 task-lifecycle-check.sh 调用
5. [GATE] CI 全部通过
