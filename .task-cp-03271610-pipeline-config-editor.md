# Task Card: Content Pipeline Config Editor — 双侧对比 + Provider 选择器

## 需求（What & Why）

**功能描述**：
1. **后端**：`POST /content-types/test-step` 接受 `provider` 字段，默认改为 `'anthropic'`（无头 bridge），不再硬编码 `anthropic-api`
2. **前端**（zenithjoy dashboard）：ContentTypeConfigPage 改为双侧对比布局：
   - 左侧：已保存的 prompt（只读）+ "测试当前"按钮 + 测试结果
   - 右侧：可编辑的 prompt + "测试修改"按钮 + 测试结果 + 保存按钮
   - 顶部工具栏：Provider 下拉选择（无头/Codex 默认、Anthropic API、OpenAI）+ 模型选择

**背景**：
- 上一个 PR (#1614) 已添加 test-step API，但 provider 硬编码为 `anthropic-api`（直接 API），不是无头 bridge
- 员工在前端配置 prompt 时，需要能看到"当前保存的 prompt"和"修改后的 prompt"对比，才能安心修改

**不做什么**：不改变 pipeline 本身的执行逻辑，不改变 DB schema

4. [GATE] CI 全部通过
