# Task Card: 记忆系统 PR5 — Dopamine RPE（奖赏预测误差）

## 需求（What & Why）

**功能描述**:
当前 dopamine.js 只记录实际奖赏，没有 RPE（Reward Prediction Error）机制。
RPE = actual - expected，是多巴胺神经科学的核心：
- RPE > 0：比预期好，正向学习信号
- RPE < 0：比预期差，负向学习信号

**背景**: 有了 RPE，Cecelia 能区分"确实做得好"和"只是运气好"，
使 Self-Drive 的任务选择更精准（不只看绝对奖赏，看相对预期偏差）。

**不做什么**:
- 不修改现有 recordReward 的签名
- 不修改 habit_patterns 逻辑
- 不在 tick.js 或 decision-executor.js 中接入（那是后续 PR 的工作）
