# PRD: Janitor 杀孤儿后回报 Brain 触发重调度

## 背景

Janitor v4.0 已能精准识别并 kill claude 孤儿进程（5条件算法），但 kill 之后没有任何回报。
被 kill 的 claude 进程对应的 Brain 任务会永远挂在 `in_progress` 状态，tick 不会重试，形成调度死锁。

## 目标

Janitor `kill_if_claude_orphan` 函数 kill 进程后，实现闭环：

1. 扫描所有 worktree 根目录的 `.dev-lock.*` 文件，找到与该 pid 关联的分支名
2. 查询 Brain API 找到 `in_progress` 状态下分支匹配的任务（通过 custom_props.branch 或 title 匹配）
3. PATCH 该任务为 `failed`，写入 `custom_props.fail_reason = "orphan_killed_by_janitor"`
4. Brain tick 下轮重新 dispatch 这个任务

## 约束

- 回报 Brain 失败（Brain 不可达）时：只打 log，不影响 kill 流程（fire-and-forget）
- .dev-lock 文件格式：`branch: cp-xxx` 第一或第二行
- Brain PATCH API：`PATCH /api/brain/tasks/{id}` body: `{"status":"failed","custom_props":{"fail_reason":"orphan_killed_by_janitor"}}`

## 成功标准

- janitor.sh 包含 `orphan_killed_by_janitor` 字符串
- `kill_if_claude_orphan` 函数在 kill 后调用 `notify_brain_orphan_killed` 函数
- `notify_brain_orphan_killed` 函数能从 `.dev-lock.*` 读取分支名并 PATCH Brain API
