**关键迁移规则**:
- `JOIN project_kr_links pkl ON pkl.project_id = op.id` → 删除（直接用 op.kr_id）
- `SELECT kr_id FROM project_kr_links WHERE project_id = $1` → `SELECT kr_id FROM okr_projects WHERE id = $1`
- `FROM project_kr_links pkl JOIN okr_projects op ON op.id = pkl.project_id WHERE pkl.kr_id = $1` → `FROM okr_projects op WHERE op.kr_id = $1`
- `LEFT JOIN goals g ON t.goal_id = g.id` → 删除（goals无domain列，COALESCE默认zenithjoy）
- `LEFT JOIN projects p ON ...` → `LEFT JOIN okr_projects p ON ...`（p.name → p.title）
- goal_evaluations FK: DROP CONSTRAINT goal_evaluations_goal_id_fkey → ADD CONSTRAINT REFERENCES key_results(id) ON DELETE CASCADE
- migration 顺序: ALTER goal_evaluations FK → DROP project_kr_links → DROP goals → DROP projects
