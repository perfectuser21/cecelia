# Task Card: AI Coding 意图扩展 + CTO Review 两阶段前置审查集成

## 需求（What & Why）

**功能描述**：
在 Engine/dev workflow 中集成两个前置审查环节：
1. **intent_expand（Step1后）**：触发 Brain API，Codex A 沿项目链路补全 PRD，生成 enriched PRD
2. **cto_review（Step2后）**：触发 Brain API，Codex B 审阅 enriched PRD + DoD + diff，PASS 才允许 push

**背景**：
AI coding 存在意图损耗问题（用户说1，AI交付0.3）。通过在 devloop 中增加等待条件，确保每次开发都经过意图扩展和 CTO 审查。

**不做什么**：
- 不改动 packages/brain/ 代码（由 Brain Agent 单独实现）
- 不改动 CI 配置（CI 保持不变作为硬门禁）
- 不实现 intent_expand / cto_review 的具体 AI 逻辑（由 Brain 侧实现）

## 成功标准

1. [ARTIFACT] `packages/engine/lib/devloop-check.sh` 新增条件 1.5（检查 intent_expand 完成）和条件 2.5（检查 cto_review PASS）
2. [ARTIFACT] `packages/workflows/skills/dev/steps/01-taskcard.md` 在 Step 1 末尾新增触发 intent_expand 的指令
3. [ARTIFACT] `packages/workflows/skills/dev/steps/02-code.md` 在代码完成、push 前新增触发 cto_review 的指令
4. [BEHAVIOR] 当 .dev-mode 含 `intent_expand_task_id` 且 status != completed 时，devloop-check.sh 退出码 1（阻塞）
5. [BEHAVIOR] 当 .dev-mode 含 `cto_review_task_id` 且 review_result != PASS 时，devloop-check.sh 退出码 1（阻塞）
6. [GATE] CI 全部通过
