# Task Card: Drain 自杀修复 + tick watchdog

## 需求（What & Why）
**功能描述**: 修复 tick.js 中 drain 完成后调用 disableTick() 导致系统完全停摆的致命 bug，降低自动恢复等待时间，并增加 tick watchdog 机制。

**背景**: 当前 drain 完成后（第 2738-2745 行）会调用 `disableTick()`，直接关掉 tick loop，系统完全停摆无法自动恢复。TICK_AUTO_RECOVER_MINUTES=30 太长，且运行期间没有自动恢复机制，只有启动时检查。

**不做什么**: 不改变 drain 的基本语义（暂停派发等现有任务完成），不改变手动 enable/disable 的行为。

## 成功标准
> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [BEHAVIOR] drain 完成后不再调用 disableTick()，改为降速 + 自动恢复
2. [BEHAVIOR] TICK_AUTO_RECOVER_MINUTES 从 30 改为 5
3. [BEHAVIOR] tick watchdog 每 5 分钟检查 tick 状态，非手动 disable 自动恢复
4. [BEHAVIOR] disableTick() 写入 source 字段区分 manual/drain/alertness
5. [GATE] CI 全部通过
