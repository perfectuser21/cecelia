# Task Card: Seal 文件 commit 到分支 — 防止上下文压缩后状态丢失

## 需求（What & Why）

**功能描述**: 在 Sprint Contract 完成后（sprint-contract-loop.sh exit 0 之前）自动 git add 所有 `.dev-gate-*.{branch}` 和 `.sprint-contract-state.{branch}` 文件，使它们随 PR commit 进分支，git worktree add 后能自动恢复。

**背景**: 上下文压缩后 worktree 消失，seal 文件（.dev-gate-spec/planner/crg/generator-sprint/lite + .sprint-contract-state）全部丢失，导致 devloop-check 无法继续、Sprint Report 评分归零。这些文件不在 .gitignore 里，原本应该被 commit 到分支。

**不做什么**: 不修改 seal 文件的内容格式、不影响 sprint-contract-loop.sh 的逻辑判断、不修改 .gitignore。

## 成功标准

1. [ARTIFACT] sprint-contract-loop.sh 在 exit 0 前自动执行 `git add` 所有 `.dev-gate-*.{branch}` 和 `.sprint-contract-state.{branch}` 文件
2. [BEHAVIOR] git add 命令只在 blocker_count == 0 时执行（收敛后才 commit，避免中间状态）
3. [ARTIFACT] 04-ship.md push 前步骤包含 seal 文件 commit 说明（文档同步）
4. [GATE] CI 全部通过
