# PRD: 质检报告上传 Brain dev-logs API

## 背景
`packages/engine/skills/dev/scripts/generate-report.sh` 生成 JSON 质检报告但只存本地 `.dev-runs/`。Brain 有 `dev_execution_logs` 表和 `/api/brain/dev-logs` POST 端点，但从未被调用，导致质检数据无法集中分析。

## 目标
在 generate-report.sh 生成 JSON 报告后，自动上传到 Brain dev-logs API，实现质检数据集中存储。

## 成功标准

- generate-report.sh 末尾有 curl POST 调用 Brain dev-logs API
- 上传是非阻塞的（curl --max-time 5），失败只输出 warning 不 exit 1
- payload 包含 task_id、run_id、phase、status、metadata（质检详情）
- Brain 不可用时不影响 cleanup 流程
