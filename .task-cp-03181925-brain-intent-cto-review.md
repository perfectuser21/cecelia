# Task Card: Brain 意图扩展 + CTO Review 两阶段前置审查系统

## 需求（What & Why）

**功能描述**：
在 AI coding 流程中新增两个前置审查环节：
1. `intent_expand`（意图扩展）：任务进来后，沿 project→KR→OKR→Vision 链路补全 PRD，把丰富的上下文写入 `task.metadata.enriched_prd`
2. `cto_review`（CTO 整体审查）：代码写完后，独立审查 enriched PRD + DoD + 核心 diff，判断 PASS/FAIL，结果写入 `task.review_result`

**背景**：
AI coding 存在意图损耗问题：用户说1，AI 交0.3。两个前置环节可以：
- intent_expand：把战略上下文补充进来，让 AI 真正理解"为什么做"
- cto_review：独立视角整体判断，减少 CI 机器检查无法覆盖的质量盲区

**不做什么**：
- 不新增 DB migration（enriched_prd 写入 tasks.metadata，review_result 是已有字段）
- 不修改 CI 门禁规则
- Engine 侧触发逻辑（Step1/Step2后触发）属于后续任务，本次只做 Brain 侧
