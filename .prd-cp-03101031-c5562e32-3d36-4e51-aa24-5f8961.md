## PRD - 小红书发布集成（N8N flow 接通 Node.js 脚本）

**需求来源**: Brain 自动调度任务 — 小红书发布集成（Playwright 自动化 + 图文发布）
**功能描述**: 已有 Playwright CDP 脚本 (`publish-xiaohongshu-image.cjs`)，但 N8N flow 的 `SSH-小红书` 节点仍调用不存在的 Python 脚本。需要将 N8N flow 更新为调用新 Node.js 脚本，完成完整的发布流程集成。

**涉及文件**:
- `packages/workflows/n8n/workflows/media/flow-内容发布.json` — 更新 SSH 命令 + 准备节点 + 解析节点
- `packages/workflows/skills/xiaohongshu-publisher/SKILL.md` — 更新状态为生产就绪

**成功标准**:
## 成功标准
- N8N flow `SSH-小红书` 节点调用 `publish-xiaohongshu-image.cjs` 而非 Python 脚本
- N8N webhook 支持 `contentDir` 参数（可选），当提供时直接用于发布
- `解析-小红书` 节点优先解析 JSON（`{"success": true/false}`），兼顾关键词匹配
- SKILL.md 状态更新为生产就绪
- 现有单元测试全部通过

**非目标**:
- 不修改 Windows PC 端 CDP 配置
- 不修改 `publish-xiaohongshu-image.cjs` 核心发布逻辑
- 不实现图片自动同步到 Windows PC（图片预置是使用者责任）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| contentDir 不存在 | SSH 命令输出错误 JSON，解析节点返回 success=false | P2 | xhs_publish_dir_not_found |
| Node.js 脚本 exit 非 0 | 解析节点输出 success=false，飞书通知 | P1 | xhs_publish_failed |
| 无图片文件 | 脚本提前报错，解析节点捕获 | P2 | xhs_publish_no_images |
