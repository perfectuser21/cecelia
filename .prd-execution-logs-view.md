## PRD - 执行日志查看页面

**需求来源**: Cecelia 系统任务 - 需要一个页面来查看任务执行的详细日志

**功能描述**:
在 Cecelia Core 前端添加一个新的页面，允许用户查看任务执行的详细日志。目前系统虽然有 ExecutionStatus.tsx 显示 N8N workflow 的执行状态，但缺少一个专门查看 Cecelia Brain 派发的任务（dev/automation/qa/audit 等）的执行日志页面。

**核心需求**:
1. 显示所有任务的执行记录列表（从 tasks 表）
2. 显示每个任务的详细日志（从 metadata.log_file 字段或 /tmp 日志文件）
3. 支持按状态筛选（queued/running/completed/failed）
4. 支持按任务类型筛选（dev/automation/qa/audit）
5. 实时更新运行中任务的日志（WebSocket 或轮询）
6. 显示任务执行时间线和 checkpoint 信息

**涉及文件**:
- 新增：`frontend/src/pages/ExecutionLogsPage.tsx` - 主页面组件
- 新增：`frontend/src/api/logs.ts` - API 调用封装
- 修改：`frontend/src/App.tsx` - 添加路由
- 修改：`brain/src/routes.js` - 添加日志查询 API 端点
  - `GET /api/brain/tasks/:taskId/logs` - 获取单个任务的日志
  - `GET /api/brain/tasks/:taskId/checkpoints` - 获取任务的 checkpoint 信息
  - `GET /api/brain/tasks` - 获取任务列表（可能已存在，需要验证）

**技术方案**:
1. 后端 API：
   - 从 PostgreSQL tasks 表读取任务列表和状态
   - 从 metadata.log_file 字段获取日志文件路径
   - 读取 /tmp 目录下的日志文件内容（如 /tmp/cecelia-*.log）
   - 支持分页和实时流式读取（tail -f 效果）

2. 前端页面：
   - 左侧：任务列表（带筛选）
   - 右侧：日志详情（带时间戳、高亮、自动滚动）
   - 使用 lucide-react 图标保持一致性
   - 使用现有的 Tailwind CSS 样式
   - 参考 ExecutionStatus.tsx 的设计风格

3. 实时更新：
   - 优先使用现有的 WebSocket 机制（websocket.js）
   - 或使用 30 秒轮询作为降级方案

**成功标准**:
1. 能够查看所有任务的执行记录
2. 能够查看单个任务的完整日志（包括 stdout/stderr）
3. 运行中的任务日志能够实时更新
4. 页面响应式设计，支持暗色模式
5. 日志内容支持搜索和过滤
6. 能够下载日志文件

**非目标**:
1. 不修改日志格式或日志存储机制
2. 不处理历史日志的归档和清理（这是 Brain 的职责）
3. 不实现日志分析和统计（这是另一个功能）
4. 不处理敏感信息过滤（假设日志已经处理过敏感信息）

**参考**:
- 现有页面：`ExecutionStatus.tsx` - 用于 N8N workflow 状态展示
- WebSocket 实现：`brain/src/websocket.js`
- 任务执行器：`brain/src/executor.js`
