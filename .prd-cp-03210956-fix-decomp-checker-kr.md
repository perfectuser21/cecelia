# PRD: 修复 decomposition-checker KR 识别错误

## 问题
decomposition-checker.js 中三处 SQL 查询使用错误的 type (`area_okr`) 和 status (`pending`) 条件，无法匹配数据库中实际的 KR 数据（`type = 'area_kr'`, `status = 'ready'`）。

## 目标
修正 Check A/B/C/D 的 SQL 查询条件，使 OKR 拆解飞轮能自动运转。

## 成功标准
1. Check A/B/C 的 SQL 使用 `type IN ('area_kr', 'kr')` 替代 `type = 'area_okr'`
2. Check A 的 status 条件扩展为 `IN ('pending', 'ready')`
3. Check D 的子 KR 查询兼容 `area_kr` 类型
4. 所有现有测试通过

## 改动范围
- `packages/brain/src/decomposition-checker.js` — 修正 4 处 SQL 查询的 type/status 条件
