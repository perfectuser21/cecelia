# Task Card: P0 修复 — RCA→auto-fix 无限重试，熔断机制失效

## 需求（What & Why）

**功能描述**: 修复两处导致 auto-fix 死循环的 bug：
1. `capability-probe.js` 使用 `probe_${f.name}_${Date.now()}` 作为 signature，Date.now() 使每次 signature 唯一，`dispatchToDevSkill` 的 3 次熔断永远无法触发。
2. `dispatchToDevSkill` 的 guard 只查 `status='failed'` 的任务，不查 `queued/in_progress`，导致正在修复中仍会重复创建任务。

**背景**: Brain probe 失败时会触发 auto-fix，但由于 signature 不稳定 + guard 不完整，每次 probe 失败都会创建新任务，形成死循环。

**不做什么**: 不改动 shouldAutoFix 逻辑，不改动 MAX_AUTO_FIX_ATTEMPTS 常量，不改动 generateFixPrd。

## 成功标准

1. [ARTIFACT] capability-probe.js signature 不含 Date.now()，使用稳定的 `probe_${f.name}`
2. [ARTIFACT] dispatchToDevSkill guard 同时检查 queued/in_progress 状态
3. [BEHAVIOR] 同一 probe 失败多次 → 只创建一个 auto-fix 任务，不重复
4. [GATE] CI 全部通过
