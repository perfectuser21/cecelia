# Task Card: DoD Test 字段强度升级

## 需求（What & Why）
**功能描述**: 升级 DoD [BEHAVIOR] 条目的 Test 字段格式规范，要求每条 Test 字段必须是可执行命令（node/curl），返回明确的 pass/fail（exit 0/1）。在 check-dod-mapping.cjs 中增加 Test 字段断言强度检查，拦截弱测试。

**背景**: 当前 check-dod-mapping.cjs 已有基础的假测试检测（echo/grep|wc-l/test -f），但缺少对"看似可执行但无断言"命令的拦截（如 `node -e "console.log('ok')"`）。需要升级为更严格的强度检查：manual: 中的 node/bash 命令必须含退出码断言逻辑（process.exit、if/then、|| exit）。同时需更新现有测试用例，将无断言的 `console.log('ok')` 替换为有断言的命令。

**不做什么**: 不改变 tests/ 和 contract: 格式的验证逻辑；不修改 CI 白名单本身。

## 成功标准

- check-dod-mapping.cjs 新增 validateAssertionStrength 函数，拦截 manual: 中无断言的 node/bash 命令
- 弱测试模式（echo/grep wc-l/无断言 node 命令）被 CI 拦截并给出明确修复提示
- 检查输出包含标准可执行 Test 写法示例，引导开发者写强测试
