# Task Card: Content Export NAS 上传 + export_path 回写

## 需求（What & Why）
**功能描述**: content-export 阶段完成后，自动将本地输出内容上传至 NAS，并将 NAS 路径写回 pipeline 的 export_path 字段，前端 PipelineOutputPage 可展示导出路径。
**背景**: 目前 export 执行完只保存在本地 /perfect21/zenithjoy/content-output/{keyword}/，pipeline 的 export_path 字段为 null，前端无法展示导出路径，内容无法持久归档到 NAS。
**不做什么**: 不修改 export 本身的内容生成逻辑；不改动 NAS 目录结构规范之外的存储路径；不处理 NAS 鉴权配置（已有 nas-fetch-content.sh 可参考）。

## 成功标准
> [ARTIFACT] = 产出物（文件/API/配置）
> [BEHAVIOR] = 运行时行为（系统如何响应输入）
> [GATE] = 门禁（CI/测试/代码质量）

1. [ARTIFACT] pipeline 执行完成后，NAS `{NAS_BASE}/{pipeline_id}/` 目录下存在 article/、cards/、images/、manifest.json
2. [BEHAVIOR] `GET /api/brain/pipelines/:id/output` 返回的 export_path 字段为非 null 的 NAS 路径字符串
3. [BEHAVIOR] 前端 PipelineOutputPage 右侧面板展示 export_path 值
4. [GATE] CI 全部通过
