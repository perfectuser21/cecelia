# PRD: 内容注入 + Evaluator divergence 下限

## 背景

Planner→Generator→Evaluator 链路需要保证信息真实传递。当前实现中 subagent 依赖文件路径自行读取内容，存在读不到或读错的风险。同时 divergence_count = 0 意味着 Evaluator 与主 agent 方案完全一致，可能表示 Evaluator 没有独立思考。

## 功能描述

1. **内容注入**：主 agent spawn Generator subagent 和 spec_review(Evaluator) subagent 时，把 Planner 产出的 Task Card + Sprint Contract 内容直接嵌入 prompt，不是让 subagent 自己去读文件路径。

2. **Evaluator divergence 下限**：Sprint Contract Gate 检查 divergence_count >= 1；为 0 说明 Evaluator 没有真实挑战主 agent，直接 exit 2 要求重跑。

## 成功标准

- 02-code.md Generator prompt 模板明确说明内容必须直接嵌入（不传文件路径）
- spec-review SKILL.md Sprint Contract Gate 包含 divergence_count = 0 → exit 2 逻辑
