# Task Card: Brain 部署脚本支持 launchd 模式

## 需求（What & Why）

**功能描述**：补齐 Brain 部署脚本（brain-deploy.sh、brain-reload.sh）对 launchd 部署模式的支持，使 CI webhook 触发的自动部署在当前 Mac mini 环境下真正可用。

**背景**：
- Brain 在这台 Mac mini 上通过 **launchd** 管理（`com.cecelia.brain.plist`），不用 Docker
- 现有 brain-deploy.sh 全程依赖 Docker（docker build、docker compose up -d），无法在无 Docker 环境下工作
- brain-deploy.sh 中 `/home/xx/` 路径硬编码，在 macOS（`/Users/administrator/`）上无效
- deploy webhook（ops.js POST /api/brain/deploy）→ deploy-local.sh → brain-deploy.sh 这条链路，在 Mac mini 上因步骤 1（docker build）失败而断裂
- PR #1316 已加入 deploy webhook + CI 轮询，但缺少底层部署能力支撑

**不做什么**：
- 不改动 CI 工作流（deploy.yml）
- 不改动 ops.js deploy webhook 逻辑
- 不添加 Docker Compose 方式的部署（Docker 方式本身已完整）

## 成功标准

1. [ARTIFACT] `scripts/brain-deploy.sh` 新增 launchd 模式：Docker 不可用时自动切换，跳过镜像构建、使用 `launchctl kickstart -k` 重启 Brain
2. [ARTIFACT] `scripts/brain-reload.sh` 新增 launchd 模式：Docker 不可用时用 launchctl 替代 docker compose restart
3. [BEHAVIOR] `bash scripts/brain-deploy.sh --dry-run` 在当前 Mac mini 环境输出 launchd 模式路径（非 Docker 路径）
4. [GATE] 所有现有测试通过

## 验收条件（DoD）

## 实现方案

**要改的文件**：
- `scripts/brain-deploy.sh` — 加 launchd 模式检测 + 重启逻辑 + 路径修复
- `scripts/brain-reload.sh` — 加 launchd fallback

**Scope 锚定**：
- 纯 bash 脚本改动，无 JS 代码改动
- launchd 重启：`launchctl kickstart -k gui/$(id -u)/com.cecelia.brain`
- launchd plist 路径：`~/Library/LaunchAgents/com.cecelia.brain.plist`
- 模式检测：`docker info >/dev/null 2>&1 && docker inspect cecelia-node-brain >/dev/null 2>&1`
- migration 直接运行：`cd $BRAIN_DIR && node src/migrate.js`
- selfcheck 直接运行：`cd $BRAIN_DIR && node src/selfcheck.js`
