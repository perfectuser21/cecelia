# PRD: 修复 cto_review execution-callback 断链

## 背景

架构巡检发现 cto_review pipeline 存在致命断链：execution-callback 中没有处理 task_type=cto_review 完成时将审查结果写入 review_result 列的逻辑，导致 devloop-check.sh 条件 2.5 永远阻塞。

## 目标

在 execution.js 的 execution-callback 中增加 cto_review 完成时的链式处理，将审查结论写入 cto_review 任务自身和父任务的 review_result 字段。

## 成功标准

- execution-callback 收到 task_type=cto_review && status=completed 时，正确将 decision+summary 写入 review_result 列
- devloop-check.sh 条件 2.5 能正确读到 review_result 并判断 PASS/FAIL
- 补充单元测试覆盖 cto_review callback 的 PASS/FAIL/无 parent_task_id 场景
- 所有 brain 现有测试继续通过
