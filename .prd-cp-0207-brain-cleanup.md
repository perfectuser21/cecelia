## PRD - Brain 旧模块清理

**需求来源**: 用户审查发现 Brain 38 个文件中有 ~10 个是三层大脑架构（2/6 上午设计）之前的旧代码，存在职责重叠、死代码、放错位置等问题

**功能描述**: 清理 brain/src/ 中被三层架构取代的旧模块，消除矛盾

**要删除的文件**（完全死的或被取代的）:
1. `prd-queue.js` — 完全孤儿，无任何 import
2. `planner-llm.js` — 直接调 Anthropic API 拆 task，被秋米 /okr 取代
3. `perception.js` — 旧感知层，1/31 迁移残留，tick.js 已直接做资源检测
4. `orchestrator.js` — 旧编排器，1/31 迁移残留，tick.js + thalamus.js 已取代
5. `self-diagnosis.js` — 旧诊断，1/31 迁移残留，cortex.js L2 已取代
6. `decomposer.js` — TRD 纯代码分解，被秋米 /okr 取代
7. `hk-bridge.js` — 独立可执行文件，不该在 brain/src/ 里
8. `minimax-executor.js` — 独立可执行文件，不该在 brain/src/ 里

**要修改的文件**:
1. `routes.js` — 清除被删模块的 import 和对应路由
2. `planner.js` — 删除 autoGenerateTask、KR_STRATEGIES、getFallbackTasks（保留 scoreKRs、planNextTask 的 KR 选择逻辑）
3. `decision.js` — 检查是否被 tick.js 引用，如果只被 routes.js 用则删除

**要更新的文件**:
1. `DEFINITION.md` — 同步架构变更
2. `LEARNINGS.md` — 记录清理经验

**涉及文件**: brain/src/ 下约 10 个文件删除/修改，routes.js 清理引用

**成功标准**:
1. 所有 658 个现有测试仍然通过
2. Brain 容器正常启动并通过 selfcheck
3. facts-check.mjs 通过
4. 无死 import（无引用被删文件的 import 语句）

**非目标**:
- 不重构三层大脑架构本身
- 不修改 tick.js / thalamus.js / cortex.js / executor.js 等核心模块的逻辑
- 不改动数据库 schema
- templates.js 暂时保留（planner.js 还用到部分功能，后续再处理）
