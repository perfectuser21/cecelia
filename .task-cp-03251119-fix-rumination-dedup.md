# Task Card: P0 Rumination→Desire 死循环修复

## 目标

修复 rumination.js 无去重导致 Desire 模块反复创建任务的 P0 死循环。

## 成功标准

1. rumination.js 写入 suggestions 表前先检查 content_hash，24h 内同 hash 不重复写入
2. 重复洞察打印 `[rumination] skipped duplicate insight` 日志
3. 单元测试覆盖三场景（首次写入/重复跳过/24h 外不触发），CI 全部通过

- [x] **[BEHAVIOR]** 重复洞察触发跳过，打印 `[rumination] skipped duplicate insight (content_hash=...)`
- [x] **[ARTIFACT]** `packages/brain/src/__tests__/rumination-dedup.test.js` 覆盖三个场景：首次写入/重复跳过/24h 外不触发
- [x] **[GATE]** CI 全部通过

## 实现方案

1. 在 rumination.js 顶部 import `crypto` from `'node:crypto'`
2. 写入 suggestions 前：
   - 计算 `content_hash = crypto.createHash('sha256').update(mainInsight).digest('hex')`
   - 查询 `SELECT id FROM suggestions WHERE content_hash = $1 AND created_at > NOW() - INTERVAL '24 hours' LIMIT 1`
   - 若有记录 → 跳过，log skipped
   - 若无记录 → INSERT，并附带 content_hash 字段
3. suggestions 表需要有 content_hash 列（若无则 ALTER TABLE 添加）
4. 新增测试文件
