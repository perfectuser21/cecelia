# PRD: 动态CI体系重构第一层 — /dev 专属健康保护

## 背景

/dev 是 Cecelia 系统的 meta 层——所有代码变更、所有自主任务都通过 /dev 完成。
但 /dev 自身没有任何专属的健康检查：Stop Hook 坏了、`.dev-lock` 丢了、branch-protect 失效了，
整个自主系统立即瘫痪，且没有任何 CI 能发现这个问题。

同时 CI 是静态写死的，随着 Cecelia 功能增长，CI 检测范围无法自动跟上。

## 本次范围（第一层）

**最紧急**：/dev 自身的健康保护，确保核心机制不能静默失效。

### 问题 1：.dev-lock 在 context 恢复后丢失
- 现象：context 压缩重载后，`.dev-lock.<branch>` 不存在 → Stop Hook exit 0 → 工作流失去约束
- 修复：Step 00 worktree 检测时，若 `.dev-mode.<branch>` 存在但 `.dev-lock.<branch>` 不存在，自动重建

### 问题 2：没有 CI 检测 /dev 自身健康
- 现象：Stop Hook、branch-protect、.dev-lock 逻辑改坏了，没有任何自动检测
- 修复：新增 `packages/engine/scripts/test-dev-health.sh` + engine-ci 新 job "Dev Health Gate"

### 问题 3：CI 全量跑，不感知变更
- 现象：改一行注释也跑全部 L1/L2/L3/L4，浪费且不扩展
- 修复（本次部分）：Detect Changes 输出真正驱动 job 的 if 条件

## 成功标准

- `.dev-lock` 丢失时 Step 00 自动重建，Stop Hook 恢复 exit 2 约束
- `test-dev-health.sh` 能验证 Stop Hook 在有/无 .dev-lock 时的行为
- engine-ci.yml 新增 "Dev Health Gate" job，在 engine 相关变更时自动触发
- 现有所有 CI 通过
