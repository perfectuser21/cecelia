# Task Card: 删除 /dev 技能文档废弃的多 Feature 支持章节

## 需求（What & Why）
**功能描述**: 从技能文档中删除已废弃的"多 Feature 支持（可选）"章节，并在删除位置添加一行说明注释。

**背景**: 多 PR 编排功能已移到 Initiative 层（/architect skill）。/dev 层不再负责多 feature 状态机和 `/dev continue` 命令。保留此章节会造成误导，开发者可能错误地以为 /dev 支持 multi-feature 状态机。

**不做什么**:
- 不修改 `## 完成度检查` 及其后的内容
- 不修改其他 steps/*.md 文件
- 不更改 Engine 的任何逻辑代码

## 成功标准

1. [ARTIFACT] `packages/engine/skills/dev/SKILL.md` 中 `## 多 Feature 支持（可选）` 章节已删除
2. [ARTIFACT] 删除位置有注释行 `> 多 PR 编排在 Initiative 层，参考 /architect skill。`
3. [BEHAVIOR] Engine 版本同步检查通过（5 个版本文件一致）
4. [ARTIFACT] `packages/engine/features/feature-registry.yml` 新增 changelog 条目
5. [GATE] CI 全部通过
