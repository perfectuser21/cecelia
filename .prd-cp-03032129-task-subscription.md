# PRD: 对话驱动任务订阅 + 欲望闭环

## 背景

当前 execution-callback 对所有任务完成都发飞书通知，太吵。
正确设计：Alex 在对话中问到某个任务 → Brain 记录关注 → 任务完成时才通知 Alex。

同时，欲望系统表达 (expression.js) 后缺乏 Alex 回应的闭环 —— Alex 的回复进了 orchestrator-chat 但未通知欲望系统。

## 功能需求

### F1: 对话驱动任务订阅

- 当 Alex 在飞书/对话中问："那个任务怎么样了"、"XX 完成了吗" 等
- orchestrator-chat 检测到任务关注意图 → 查找匹配任务 → 存入 working_memory
- Key 格式：`task_interest:<task_id>`
- TTL：48 小时（task_id 维度）+ 通用关键词兜底 1 小时
- execution-callback 任务完成时：检查 working_memory → 有订阅才通知

### F2: 欲望系统闭环

- Cecelia 表达欲望（发飞书消息）后，Alex 回复时
- orchestrator-chat 检测到 Alex 有回复 → 查找近期 'expressed' 状态的 desires
- 将这些 desires 更新为 'acknowledged' 状态
- 意义：记录欲望得到了回应，影响未来同类欲望的权重

## 成功标准

1. Alex 问"那个任务完成了吗" → 对应任务完成时收到飞书通知
2. Alex 没问过的任务完成 → 不发飞书（安静）
3. Cecelia 发飞书表达欲望 → Alex 回复 → desires 表出现 'acknowledged' 状态记录
4. 所有现有 CI 测试通过

## 不在范围

- 跨对话任务订阅持久化（当前仅 working_memory TTL 范围内）
- 欲望表达后不关联 pending_conversations（不加 migration）
- 推送所有任务完成通知（已废弃行为）
