# PRD — 修复 learnings 向量化管道

## 背景
learnings 表 558 条记录全部 embedding=null，原因：
1. `generateLearningEmbeddingAsync` 静默吞掉所有错误（catch (_err) {}），失败了完全看不到
2. 没有 backfill 机制，历史数据永远无法补全
3. memory_stream 过期记录（68k 条）未清理，污染向量搜索空间

## 要做的事
1. 修复 embedding-service.js：加日志，失败后写入重试队列（working_memory key=embedding_retry_queue）
2. 新增 backfill 函数：Brain 启动时自动给 embedding=null 的 learnings 补向量（分批，每批10条）
3. 新增 DB migration：清理 expires_at < NOW() 的 memory_stream 记录

## 成功标准
- [ ] 新写入 learning 失败时有日志，写入重试队列
- [ ] Brain 启动时自动 backfill（每批10条，有速率控制）
- [ ] DB migration 存在，清理过期 memory_stream
- [ ] 有测试覆盖 backfill 和重试逻辑
