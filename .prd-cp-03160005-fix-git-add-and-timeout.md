# PRD: git add -A 改 -u + Stop Hook 90 分钟超时保护

## 背景

当前 /dev 工作流存在两个问题：
1. `03-prci.md` 中使用 `git add -A`，会意外添加未追踪文件（如 .env、临时文件等）到 commit，存在安全风险。
2. `devloop-check.sh` 中 Stop Hook 在 CI pending 状态下会无限阻止会话结束，若 CI 真的卡死（如 runner 挂了），agent 无法退出。

## 目标

1. 将 `03-prci.md` 中所有 `git add -A` 改为 `git add -u`，只暂存已追踪文件的修改。
2. 在 `devloop-check.sh` 的 CI pending 检测前加入 90 分钟全局超时保护，超时后允许退出并提示手动检查。

## 成功标准

- `03-prci.md` 中 Step 8.4 和 Step 9.2b ⑤ 的 `git add -A` 均已改为 `git add -u`
- `devloop-check.sh` 在读取 `started:` 字段后计算已用时间，超过 5400 秒（90 分钟）则 exit 0
- Engine 版本从 12.81.0 bump 到 12.82.0，7 个版本文件全部同步
- CI 通过
