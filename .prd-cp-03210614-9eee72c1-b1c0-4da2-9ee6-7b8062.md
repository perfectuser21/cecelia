# PRD: KR3 内容流水线闭合 — export → content_publish 任务创建

## 背景
content-export 完成后直接将 pipeline 标记 completed，但 content_publish 任务从未被创建。
executor.js 已有完整的 8 平台路由逻辑，缺少"上游触发"。

## 功能描述
在 content-pipeline-orchestrator.js 的 advanceContentPipeline 中，当 content-export 子任务完成后，
自动为每个发布平台创建 content_publish 任务（fire-and-forget），再将 pipeline 标记为 completed。

## 成功标准

- [ ] content-export 完成时为 8 个平台创建 content_publish 任务（含幂等保护）
- [ ] content_publish 任务 payload 含 platform 字段
- [ ] 现有测试通过，新增测试覆盖闭合逻辑
