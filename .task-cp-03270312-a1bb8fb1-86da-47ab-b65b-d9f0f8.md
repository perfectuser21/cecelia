# Task Card: Dashboard 双升级 — Inline 编辑 + 作战室页面

## 需求（What & Why）

**功能描述**：两个独立功能
1. **前台 Inline 编辑**：OKR/KR/Project 页面支持点击直接编辑标题，像 Notion 一样，不跳转表单
2. **作战室页面**：新路由 `/warroom`，实时展示当前聚焦 OKR、进行中任务、各层级状态

**背景**：Brain 已有完整数据 + PATCH 端点，但前台只能只读查看，修改需要绕道 API 或 DB。作战室是"我们在往哪跑"的仪表盘，让用户随时感知系统状态。

**不做什么**：
- 不做富文本/Markdown 编辑（只做纯文本 title inline edit）
- 不做 description 大段编辑（scope 过大）
- 不做 Initiative 单独页面（无该页面，略过）
- 不做拖拽排序、批量编辑

## 成功标准

- [ ] OKRPage 的 Objective 标题点击后可直接编辑并保存到 Brain（PATCH /api/brain/goals/:id）
- [ ] OKRPage 的 KR 标题同样支持 inline 编辑
- [ ] ProjectDetail 的项目名称支持 inline 编辑
- [ ] 新路由 `/warroom` 存在并可访问，展示当前聚焦 OKR + 进行中任务 + 统计数字
- [ ] 导航栏有"作战室"入口（execution navGroup 下，order 0）

4. `frontend/src/features/core/planning/pages/WarRoomPage.tsx`（新建）
   - 轮询 30s：`/api/brain/okr/current` + `/api/brain/tasks?status=in_progress&limit=20`
   - 展示：当前聚焦 Objective（取第一条 active）、进行中任务列表、统计（待办/进行中/今日完成）

5. `frontend/src/features/core/planning/index.ts`（修改）
   - 新增路由 `{ path: '/warroom', component: 'WarRoomPage', navItem: { label: '作战室', icon: 'Swords', group: 'planning', order: 0 } }`

- Brain 后端无需改动（PATCH goals/projects 已支持 title 更新）
