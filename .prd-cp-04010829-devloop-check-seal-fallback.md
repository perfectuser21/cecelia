# PRD: devloop-check 从 seal 文件反推 step 状态

## 背景

`.dev-mode` 文件消失时（会话压缩、worktree GC 等场景），`devloop-check.sh` 的直接执行入口完全失明——输出 `NO_ACTIVE_SESSION` 并退出，无法告知 agent 当前处于哪个 stage。

但 seal 文件已经 commit 进分支（P0 已修），可以从 seal 文件反推步骤状态。

## 目标

当 `.dev-mode` 不存在时，`devloop_check_main()` 从 seal 文件反推状态：
- `.dev-gate-planner.{branch}` 或 `.dev-gate-lite.{branch}` 存在 → step_1 = done
- `.dev-gate-crg.{branch}` 存在 → step_2 = done
- PR 已在远端创建（via `gh pr list`）→ step_3 = done

输出明确的"当前推断 stage + 缺什么 + 下一步"，不误报 step_1 pending。

## 涉及文件

- `packages/engine/lib/devloop-check.sh`（修改 `devloop_check_main()` 函数）

## 成功标准

- `.dev-mode` 不存在时 `devloop-check.sh` 仍能输出正确 stage 推断
- 有 seal 文件时不误报 step_1 pending
- CI 通过
