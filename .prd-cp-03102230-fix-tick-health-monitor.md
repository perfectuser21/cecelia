# PRD: Brain 重启后 tick_enabled=false 超时自动恢复

## 功能描述

Brain 重启时，若 working_memory 中 tick_enabled=false 且距离 disabled_at 超过 30 分钟，自动 enable 并写入 P1 告警事件。

## 需求来源

`initTickLoop()` 启动时读取 working_memory 中的 tick_enabled：
- 如果是 false → 不启动，记录日志，就此停下
- 问题：tick 被禁用的原因可能早已消失（如熔断解除），但 Brain 不知道，每次重启都保持 disabled

实际发生（2026-03-10）：Tick 在 21:01 启动后 ~20 分钟内又被禁用（enabled=false），
需要人工 `curl -X POST localhost:5221/api/brain/tick/enable` 才能恢复。

## 成功标准

1. `disableTick()` 额外存储 `disabled_at` 时间戳
2. `initTickLoop()` 在发现 disabled 时：
   - 若 `disabled_at` 不存在或超过 30 分钟：自动 enable + 写 cecelia_events P1 警告
   - 若 `disabled_at` < 30 分钟：保持 disabled（可能是刚刚因为熔断等被正常禁用）
3. 阈值通过常量 `TICK_AUTO_RECOVER_MINUTES = 30` 控制

## 范围

`packages/brain/src/tick.js`：
1. `disableTick()` → 存储 `{ enabled: false, disabled_at: ISO_STRING }`
2. `initTickLoop()` → 检查 disabled_at，超过阈值时 auto-enable + emit P1 事件
3. 新增常量 `TICK_AUTO_RECOVER_MINUTES = 30`
4. 新增单元测试 `tick-health-monitor.test.js`
