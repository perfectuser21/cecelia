# Task Card: CI P1 — coverage-delta 改为阻塞 + 只对 feat: PR 跑

## 需求（What & Why）

1. **coverage-delta 改为阻塞**：L3 Gate 里 coverage-delta 失败目前只输出警告不阻止合并。需要改为硬门禁：brain 有改动且 coverage-delta 失败时 L3 Gate 失败。vitest.config.js 已有阈值（75/75/80/75）。

2. **coverage-delta 只对 feat: PR 跑**：fix:/chore:/refactor: PR 会触发 brain-unit + coverage-delta 两次完整测试，重复约 25 分钟。coverage-delta 只在 feat: PR 时有意义。

**不做什么**: 不修改 vitest.config.js 阈值，不改 Brain 代码。

## 成功标准

1. [ARTIFACT] `coverage-delta` job 的 `if:` 条件包含 `commit_type == 'feat'` 检查
2. [ARTIFACT] L3 Gate 中 `coverage-delta` 失败时设置 `FAILED=true`（阻塞合并）
3. [GATE] CI 全部通过（[CONFIG] 标签豁免测试覆盖检查）

## 验收条件（DoD）

- [x] [ARTIFACT] coverage-delta job 的 if: 条件包含 feat commit type 检查
  Test: manual:bash -c "grep -c 'commit_type.*feat' .github/workflows/ci-l3-code.yml"

- [x] [ARTIFACT] L3 Gate 中 coverage-delta 失败时设置 FAILED=true
  Test: manual:bash -c "node -e \"const fs=require('fs'); const yml=fs.readFileSync('.github/workflows/ci-l3-code.yml','utf8'); if(yml.includes('Coverage Delta Check') && yml.includes('FAILED=true'))console.log('ok'); else process.exit(1);\""

- [x] [BEHAVIOR] 修改后 coverage-delta 变为阻塞（L3 gate 非阻塞注释已删除）
  Test: manual:bash -c "node -e \"const fs=require('fs'); const yml=fs.readFileSync('.github/workflows/ci-l3-code.yml','utf8'); if(!yml.includes('非阻塞，覆盖率报告供参考'))console.log('ok: blocking now'); else process.exit(1);\""

- [x] [GATE] 所有现有测试通过
  Test: manual:bash -c "git log --oneline -1"

## 实现方案

**要改的文件**: `.github/workflows/ci-l3-code.yml`

**改动点 1 — coverage-delta job if: 条件，加 commit_type == 'feat' 检查**

**改动点 2 — L3 Gate 中 coverage-delta 改为阻塞（feat brain PR 时）**
