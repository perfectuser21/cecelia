# PRD：修复秋米新版本 INSERT 失败

## 问题

用户说"写一个新的一个版本"后，右侧聊天显示"v6已生成，请在左侧查看"，但左侧什么都没有。

Brain 容器日志：
```
[autumnrice/chat] new version parse failed: column "updated_at" of relation "pending_actions" does not exist
```

根本原因：routes.js 的 INSERT 语句包含 `updated_at` 列，但 `pending_actions` 表没有此列（DB schema 确认）。

## 修复

文件：`packages/brain/src/routes.js`，L1806 附近

**当前（错误）：**
```sql
INSERT INTO pending_actions (action_type, context, params, status, comments, created_at, updated_at)
VALUES ($1, $2::jsonb, $3::jsonb, 'pending_approval', '[]'::jsonb, NOW(), NOW())
```

**修复后：**
```sql
INSERT INTO pending_actions (action_type, context, params, status, comments, created_at)
VALUES ($1, $2::jsonb, $3::jsonb, 'pending_approval', '[]'::jsonb, NOW())
```

## 验收

- 用户说"写一个新的一个版本"后，左侧出现新 Tab（V2/V3...）
- Brain 日志：`new version created: <id> (<N> initiatives)`
- 无 `updated_at` 错误
