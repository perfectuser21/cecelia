## DoD - 修复 Linux→macOS 全量兼容性问题

### 验收清单

- [x] platform-utils.js 新增 `getAvailableMemoryMB()` 函数，macOS 用 vm_stat free+inactive，Linux 用 os.freemem()
  Test: manual:bash -c "grep -c 'getAvailableMemoryMB' packages/brain/src/platform-utils.js"

- [x] executor.js 内存计算改用 `getAvailableMemoryMB()`，不再直接调 os.freemem()
  Test: manual:bash -c "grep -c 'getAvailableMemoryMB' packages/brain/src/executor.js"

- [x] routes.js VPS 监控内存字段改用 `getAvailableMemoryMB()`
  Test: manual:bash -c "grep -c 'getAvailableMemoryMB' packages/brain/src/routes.js"

- [x] healing.js 删除本地 `isProcessAlive()`，改用 platform-utils 的 `processExists()`
  Test: manual:bash -c "grep -c 'processExists' packages/brain/src/alertness/healing.js"

- [x] healing.js `pgrep -f "claude.*-p"` 改为使用 platform-utils 的 `countClaudeProcesses()` 或等效平台安全方案
  Test: manual:bash -c "grep -c 'countClaudeProcesses\|processExists' packages/brain/src/alertness/healing.js"

- [x] `getAvailableMemoryMB()` 导出到 platform-utils 公共接口
  Test: manual:bash -c "grep -c 'export function getAvailableMemoryMB' packages/brain/src/platform-utils.js"

- [x] macOS 上 `getAvailableMemoryMB()` 返回值 > 1000MB（vm_stat 包含 inactive）
  Test: manual:bash -c "node -e \"import('./packages/brain/src/platform-utils.js').then(m => { const v = m.getAvailableMemoryMB(); console.log(v); process.exit(v > 1000 ? 0 : 1) })\""
