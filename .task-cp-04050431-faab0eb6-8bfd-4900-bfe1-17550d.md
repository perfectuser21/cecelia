# Task Card: 修复内容发布链路 — escalation 取消 Bug + KR verifier SQL 错表

## 需求（What & Why）
**功能描述**: 修复两个阻断内容自动发布链路的 Bug，使 KR 进度能正确反映实际执行状态。
**背景**:
1. `escalation.js`的 `cancelPendingTasks` 保护了 `content-publish`（连字符）但实际 task_type 是 `content_publish`（下划线），导致每次 escalation 触发时所有发布任务被取消
2. KR verifier SQL 查 `content_publish_jobs` 旧表（只有 2 条 2026-03-19 的 running 记录），应查 `tasks` 表中 `task_type='content_publish'` 的完成记录
3. 两个 in_progress 的 pipeline（行业跟进能力、经营日清能力）停滞：经营日清能力有 export 完成但父 pipeline 未推进到 completed

**不做什么**: 不改 publisher skill 逻辑；不改 pipeline orchestrator 核心流程；不修改历史已取消的任务

## 成功标准
- escalation 触发时 content_publish(下划线) 任务不再被取消
- KR verifier 查正确的表（tasks 表）
