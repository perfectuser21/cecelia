# Task Card: Brain dispatch-now 端点 + devloop-check 90 分钟超时修复

## 需求（What & Why）

**功能描述**:
1. 新增 Brain API 端点 `POST /api/brain/dispatch-now`，不经过 tick loop 直接派发任务执行
2. 修复 devloop-check.sh 中 90 分钟 CI 超时返回 done 的 bug（应返回 blocked）

**背景**:
- /dev 的 Codex 审查（cto_review/code_quality/prd_audit）注册后依赖 tick loop 派发，但调度器默认关闭，导致审查任务永远不执行
- dispatch-now 让 /dev 主 agent 能"注册+立即触发"，不依赖调度器状态
- 90 分钟超时 bug：CI pending 超 90 分钟时 devloop-check 返回 done，导致未完成的工作流无声结束

**不做什么**:
- 不改 tick loop 本身
- 不改调度器默认开关状态
- 不改 Codex 审查 Skill 内容

## 成功标准

1. [BEHAVIOR] POST /api/brain/dispatch-now 能接收 task_id，直接执行任务（不依赖 tick loop）
2. [BEHAVIOR] devloop-check.sh 90 分钟超时返回 blocked（不返回 done），并创建 Brain P0 诊断任务
3. [GATE] CI L1-L4 全通过
