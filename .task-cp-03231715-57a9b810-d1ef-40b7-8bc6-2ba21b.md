# Task Card: KR 优先级收敛与依赖排序引擎

## 需求（What & Why）

**功能描述**：
为 Brain 增加一个自动收敛层，对当前所有活跃 KR 按价值、依赖关系、进度和阻塞程度进行综合评分，
输出 top 3 主线焦点，并给出暂停/降级建议。避免 20 个项目同时摊薄资源。

**背景**：
当前 7 个 active KR 全部处于 P1，进度均为 0%（仅"管家闭环"有 13%）。
Brain SelfDrive 每 tick 都在向所有 KR 分配资源，但缺乏收敛机制导致没有一条主线能快速完成。
需要一个收敛引擎输出优先聚焦哪 3 个 KR，其余暂停，为数据闭环、自动发布、Dashboard 提供统一调度依据。

**不做什么**：
- 不修改 KR 状态（只读推荐，不自动 pause KR）
- 不引入新的 DB 表（在现有 `key_results`、`okr_projects`、`tasks` 表上查询）
- 不修改 SelfDrive 调度逻辑（本期仅提供 API，集成由后续任务完成）

## 成功标准

1. `GET /api/brain/kr/convergence` 接口可调用，返回含 top3 + pause_candidates 的 JSON
2. 评分算法综合 4 个维度：进度权重(35%)、任务活跃度(30%)、项目密度(20%)、指标动量(15%)
3. top3 结果可解释：每个 KR 附带 score、rank、reason 字段
4. pause_candidates 包含低分 KR 及 suggestion 字段（"暂停" 或 "降级"）
5. 单元测试覆盖评分逻辑（含边界：0 个 KR、全部 0 分、并列分）

## 实现方案

**要改的文件**：
1. `packages/brain/src/kr-convergence.js`（新建）— 评分引擎核心
2. `packages/brain/src/routes/kr-convergence.js`（新建）— API 路由
3. `packages/brain/server.js`（修改）— 注册路由
4. `packages/brain/src/__tests__/kr-convergence.test.js`（新建）— 单元测试

**评分公式**：
```
score = progress_score * 0.35
      + activity_score * 0.30
      + project_density_score * 0.20
      + metric_momentum_score * 0.15
```

- `progress_score`：`kr.progress / 100`（0-1）
- `activity_score`：该 KR 关联 projects 下 active/in_progress tasks 数量，归一化到 0-1
- `project_density_score`：该 KR 关联 okr_projects 数量，归一化到 0-1
- `metric_momentum_score`：`metadata.metric_current > 0` 则 1，否则 0

**API 响应结构**：
```json
{
  "computed_at": "ISO8601",
  "active_kr_count": 7,
  "top3": [
    { "id": "...", "title": "...", "score": 0.85, "rank": 1, "reason": "...", "progress": 13 }
  ],
  "pause_candidates": [
    { "id": "...", "title": "...", "score": 0.05, "suggestion": "暂停", "reason": "..." }
  ],
  "all_ranked": [ ... ]
}
```

**Scope 锚定**：Brain 后端，不触碰前端/Engine/Workflows。
