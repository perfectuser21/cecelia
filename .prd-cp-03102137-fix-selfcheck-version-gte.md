# PRD: selfcheck 版本检查改为 >= 防止 migration 崩溃循环

## 背景

selfcheck.js 目前使用精确匹配（`===`）检查 DB schema version。这导致：
- 每次有新 migration 合并后，旧的 Brain 实例立刻拒绝启动
- Brain crash → LaunchDaemon 重启 → 仍然失败 → 无限崩溃循环
- 人工干预才能恢复（修改 EXPECTED_SCHEMA_VERSION 或回滚 migration）

2026-03-10 实际发生：migration 142 在 main 上运行后，Brain 拒绝启动，需要手动 sed 修改代码才能恢复。

## 目标

1. **消灭 migration 崩溃循环**：DB version >= expected 即通过，向前兼容
2. **提交 dirty 改动**：main 上 EXPECTED_SCHEMA_VERSION 有未提交的 sed 改动（141→142），需要正式提交到 PR 里

## 范围

- `packages/brain/src/selfcheck.js`：
  1. `EXPECTED_SCHEMA_VERSION` 更新为 '142'
  2. 版本比较逻辑：`parseInt(maxVer) >= parseInt(EXPECTED_SCHEMA_VERSION)` 替代 `maxVer === EXPECTED_SCHEMA_VERSION`
  3. 错误信息更新：反映"版本过旧"而非"版本不匹配"
- `packages/brain/src/__tests__/selfcheck.test.js`：
  1. 更新 EXPECTED_SCHEMA_VERSION 硬编码测试 ('141' → '142')
  2. 新增测试：DB 版本高于 expected 时应 PASS（验证 >= 逻辑）

## 验收标准

- Brain 在 DB 版本 = expected 时 PASS（现有行为保持）
- Brain 在 DB 版本 > expected 时 PASS（新行为）
- Brain 在 DB 版本 < expected 时 FAIL（保持严格的向下检查）
- 所有现有 selfcheck 测试通过
- facts-check.mjs 通过
