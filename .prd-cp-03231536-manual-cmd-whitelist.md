# PRD: CI Gate — manual: 命令白名单本地前置拦截

## 背景

`manual:` 命令只允许特定工具在 CI ubuntu-latest 上运行（node/npm/curl/bash/psql）。
当前问题：本地 verify-step.sh 不检查命令是否在白名单内，只有到 CI L1 执行时才失败。
这导致"本地通过 → CI 失败"的循环，浪费 CI 时间。

## 目标

本地就能拦截非白名单 `manual:` 命令，与 CI 白名单保持一致。

## 成功标准

1. 新增共享脚本 `packages/engine/scripts/devgate/check-manual-cmd-whitelist.cjs`
2. verify-step.sh 在执行 DoD Test 前先调用白名单检查
3. CI L1 新增 `manual-cmd-whitelist-check` job（stage 2+）
4. 测试文件覆盖率 ≥ 60%

## 白名单定义

允许：`node` / `npm` / `npx` / `curl` / `bash` / `psql`
拒绝：其他所有顶层命令（grep / ls / cat / find / sed / awk 等）
