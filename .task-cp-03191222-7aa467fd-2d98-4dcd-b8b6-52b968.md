# Task Card: 定时发布内容队列管理器（cron + 内容日历）

## 需求（What & Why）

**功能描述**：在 NAS 内容目录中放置 `schedule.json`，cron 每 5 分钟扫描，到时自动调用 N8N Webhook `content-publish` 发布内容到指定平台，发布后写入 `published_at` 防重复，并发飞书通知。

**背景**：三平台发布目前只能即时发布，缺少按计划定时发布能力。

**不做什么**：
- 不修改 N8N 工作流内部逻辑
- 不新增数据库表
- 不支持跨天跨平台批量调度（只调用已有 content-publish webhook）

## 成功标准

1. [ARTIFACT] `packages/workflows/scripts/schedule-publisher.sh` 存在且可执行
2. [ARTIFACT] `packages/workflows/scripts/validate-schedule-json.sh` 存在，能验证 schedule.json 格式
3. [ARTIFACT] `packages/workflows/scripts/setup-publisher-cron.sh` 存在，能注册 cron
4. [BEHAVIOR] schedule-publisher.sh 扫描 NAS，对 publishAt 在 5 分钟内、published_at 为空的条目触发 N8N Webhook
5. [BEHAVIOR] 已触发的条目写入 published_at 时间戳，不会重复触发
6. [BEHAVIOR] 触发时通过 FEISHU_BOT_WEBHOOK 发飞书通知
7. [GATE] CI 全部通过
