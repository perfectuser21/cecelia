# Task Card: quickcheck.sh — monorepo ROOT_NM 修复

## 需求（What & Why）
**功能描述**: PR #2025 修复不完整。ENGINE_NM 指向空的 packages/engine/node_modules/（目录存在但无 .bin），-d 检查假阳性，PATH 注入无效，vitest 仍然找不到。
**背景**: npm workspaces monorepo 把所有二进制提升到根目录 node_modules/.bin/，包级目录是空的。
**不做什么**: 不改测试逻辑，不改 ESLint 候选路径（已正确）。

## 验收条件（DoD）

- [x] [ARTIFACT] quickcheck.sh 含 ROOT_NM 变量指向根 node_modules
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/quickcheck.sh','utf8');if(!c.includes('ROOT_NM'))process.exit(1)"

- [x] [BEHAVIOR] 存在性检查改为 -x ROOT_NM/.bin/vitest，不再用 -d ENGINE_NM
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/quickcheck.sh','utf8');if(c.includes('[[ ! -d \"\$ENGINE_NM\"'))process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] 主仓库环境下 ROOT_NM/.bin/vitest 可执行
  Test: manual:bash -c "ROOT_NM=/Users/administrator/perfect21/cecelia/node_modules && [[ -x \"\$ROOT_NM/.bin/vitest\" ]] && echo OK"

- [x] [BEHAVIOR] worktree 环境下 ROOT_NM 正确指向主仓库根 node_modules
  Test: manual:bash -c "cd /Users/administrator/worktrees/cecelia/0b18c6c6-98ff-4f1d-b772-629854-2 && REPO=\$(git rev-parse --show-toplevel) && GCD=\$(git rev-parse --git-common-dir) && MAIN=\$(dirname \"\$(cd \"\$REPO\" && cd \"\$GCD\" && pwd)\") && ROOT_NM=\"\$MAIN/node_modules\" && [[ -x \"\$ROOT_NM/.bin/vitest\" ]] && echo OK"

## 实现方案
**文件**: scripts/quickcheck.sh
**改动**:
1. MAIN_REPO_ROOT 之后加 `ROOT_NM="$MAIN_REPO_ROOT/node_modules"`
2. `[[ ! -d "$ENGINE_NM" ]]` → `[[ ! -x "$ROOT_NM/.bin/vitest" ]]`
3. `[[ ! -d "$BRAIN_NM" ]]` → `[[ ! -x "$ROOT_NM/.bin/vitest" ]]`
4. 所有 `PATH="$ENGINE_NM/.bin:$PATH"` → `PATH="$ROOT_NM/.bin:$PATH"`
5. TypeCheck 存在性检查：`[[ -d "$ENGINE_NM" ]]` → `[[ -d "$ROOT_NM" ]]`
