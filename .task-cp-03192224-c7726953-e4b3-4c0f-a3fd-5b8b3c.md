# Task Card: Learning 表同步修复 — distill-learnings.js 写入 learnings 表

## 需求（What & Why）
**功能描述**: 修改 `packages/brain/scripts/distill-learnings.js`，在写入 knowledge 表的同时，也写入 learnings 表（title, learning_type, content, metadata, source_branch）。

**背景**: 现状下 distill-learnings.js 只写 knowledge 表（type='learning_rule'），而 learning-retriever.js 读取的是 learnings 表。两张表不通，导致 100+ 篇高质量 Learning 永远不会被注入到下次 /dev 任务。字段映射：name→title，learning_rule→best_practice（learning_type），sub_area→metadata.source_file，filename（去 .md）→source_branch。

**不做什么**: 不修改 learning-retriever.js，不修改 knowledge 表写入逻辑（保持向后兼容），不处理 embedding。

## 成功标准
> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [ARTIFACT] distill-learnings.js 修改后同时写入两张表（knowledge + learnings）
2. [BEHAVIOR] 运行 `node packages/brain/scripts/distill-learnings.js --dry-run` 显示 learnings 表写入预览，无报错
3. [BEHAVIOR] 实际运行后 `SELECT COUNT(*) FROM learnings WHERE learning_type='best_practice'` 结果 ≥ 25
4. [GATE] CI 全部通过
