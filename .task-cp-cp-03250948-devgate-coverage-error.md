# Task Card: Devgate 覆盖率检查 warning→error 升级

## 需求（What & Why）

**功能描述**: 将 `check-coverage-completeness.mjs` 中 devgate 脚本覆盖率检查从 warning 级别升级为 error 级别（CI 阻断），并为高风险脚本补充最小测试。

**背景**: Engine 有 13 个 devgate 脚本，其中 `check-rci-stale-refs`、`scan-rci-coverage`、`check-changed-coverage` 已有测试，其余 10 个无测试。当前检查仅为 warning，CI 不阻断，导致高风险脚本无测试保护。

**方案**: 采用白名单机制 — 高风险脚本（`HIGH_RISK_DEVGATE_SCRIPTS`）缺测试时 exit 1；其余脚本缺测试时保持 warning（不阻断）。高风险脚本：`check-dod-mapping`、`scan-rci-coverage`、`check-coverage-completeness`、`check-rci-stale-refs`、`check-changed-coverage`。

**注**: 以下 DoD 中引用的测试文件（`check-coverage-completeness.test.ts`、`check-dod-mapping.test.ts`）是本 Task 需要新建的文件。

**不做什么**: 不为纯 CLI 脚本（无 exports）添加假测试；不修改其他 devgate 脚本。

## 成功标准

- `check-coverage-completeness.mjs` 运行后，高风险 devgate 脚本缺失测试时 exit 1
- `check-dod-mapping.cjs` 和 `check-coverage-completeness.mjs` 均有对应测试文件
- 现有低风险脚本缺测试时显示 ⚠️ warning，不阻断 CI
