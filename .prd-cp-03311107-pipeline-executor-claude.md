# PRD: 接通 content pipeline executor 的 Claude 调用

## 背景

content-pipeline-executors.js 中有 4 个阶段没有真正调用 Claude CLI：
- executeCopywriting：用本地模板拼文案
- executeCopyReview：用静态关键词规则
- executeGenerate：只标记状态
- executeImageReview：只检查文件存在

配置页面（ContentTypeConfigPage）设置的 generate_prompt、review_prompt、image_prompt、image_review_prompt 全部被忽略。

## 目标

将上述 4 个函数改为真正调用 Claude CLI，读取配置中的 prompt 字段执行内容生成和审核。

## 不做什么

- 不修改 executeResearch（已接通 notebooklm）
- 不修改 executeExport
- 不改 orchestrator 调度逻辑
- 不修改数据库 schema
- 不接入真实图片渲染 API（executeGenerate 只生成文字描述）

## 成功标准

- executeCopywriting 使用 generate_prompt 调用 Claude CLI，产出真实文案（rerun 时注入 review_feedback）
- executeCopyReview 使用 review_prompt 调用 Claude CLI，返回每条 rule 的 passed/score/comment
- executeGenerate 使用 image_prompt 调用 Claude CLI，产出 cards/image-descriptions.md
- executeImageReview 使用 image_review_prompt 调用 Claude CLI，返回 {review_passed, issues}

## 实现范围

文件：`packages/brain/src/content-pipeline-executors.js`
