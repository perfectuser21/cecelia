# PRD - [配额轮转 PR1] DB schema — tasks 表新增 quota_exhausted 状态 + migration

**需求来源**: Brain 自动调度任务（agent_ops 领域）
**功能描述**: 配额耗尽的任务当前被标记为 failed，导致 failure_count 增加，最终触发隔离，污染失败率指标。需要新增独立状态 quota_exhausted 使其不计入失败阈值。
**涉及文件**:
- `packages/brain/src/migrations/` — 新增 migration SQL
- `packages/brain/src/selfcheck.js` — EXPECTED_SCHEMA_VERSION 升级
- `packages/brain/src/planner.js` — 不把 quota_exhausted 计入 failure_count
- `packages/brain/src/executor.js` — 不把 quota_exhausted 触发隔离
- `packages/brain/__tests__/` — 单测

**成功标准**:
- tasks.status 枚举包含 'quota_exhausted'
- 该状态的任务 failure_count 不增加
- migration 文件存在且编号连续

**非目标**: 不修改 Brain 其他调度逻辑，不改变 failed 状态行为

## 成功标准

- [ ] tasks.status CHECK 约束新增 'quota_exhausted' 枚举值
- [ ] selfcheck.js EXPECTED_SCHEMA_VERSION 同步升级
- [ ] planner.js / executor.js 不把 quota_exhausted 状态计入 failure_count，不触发隔离
- [ ] 单测：quota_exhausted 状态任务不触发 FAILURE_THRESHOLD 隔离逻辑

### 异常处理 & 报警规范

N/A - 无 IO 操作（schema migration + 逻辑条件修改）
