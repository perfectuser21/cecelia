# PRD: 修复 BrainModelsPage handleSaveOrgan API 参数名

## 背景

BrainModelsPage.tsx 的 handleSaveOrgan 函数保存 organ 模型时，
发送 PATCH 请求 body 使用 camelCase 命名（`agentId`, `model`），
但后端接口要求 snake_case（`agent_id`, `model_id`），
导致点击保存按钮时报错 `agent_id and model_id are required`。

## 修复

将 `handleSaveOrgan` fetch body 由：
```
{ agentId: organId, model: modelId, provider }
```
改为：
```
{ agent_id: organId, model_id: modelId, provider }
```

## 成功标准

- handleSaveOrgan fetch body 使用 agent_id 和 model_id
- 不存在旧的 agentId 键名
- 不存在 model: modelId 旧写法
- CI 全部通过
