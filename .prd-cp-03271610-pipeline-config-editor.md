# PRD: Content Pipeline Config Editor — 双侧对比 + Provider 选择器

## 背景

test-step API（PR #1614）已上线，但 provider 硬编码为 anthropic-api，前端缺少双侧对比和 Provider 选择功能。

## 目标

1. 后端：test-step 默认使用 `anthropic`（无头 bridge），支持前端传入 provider 覆盖
2. 前端（zenithjoy）：双侧对比布局 + Provider/模型选择器

## 成功标准

- test-step 不传 provider 时使用 `anthropic`（无头 bridge）
- ContentTypeConfigPage 双栏布局：左=已保存(只读)，右=可编辑，各有独立测试按钮
- Provider 选择器：无头/Codex（默认）、Anthropic API 直连、OpenAI
- CI 全部通过
