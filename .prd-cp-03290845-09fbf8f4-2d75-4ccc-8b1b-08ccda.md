# PRD: DoD Test 字段升级 — 从模糊描述升级为可执行命令

## 背景

`check-dod-mapping.cjs` 在 `GITHUB_ACTIONS=true` 时实际执行 `manual:` 命令。L1 CI 无 PostgreSQL、无 Brain 服务、无 node_modules，导致 `curl localhost/psql/npm test` 在 CI 中静默失败但错误信息不清晰。同时 `DOD-TEMPLATE.md` 使用旧格式 evidence ID（如 `manual:code-review`），在当前校验器下直接失败。

## 功能描述

1. `check-dod-mapping.cjs` 新增 `detectCiIncompatibleCommand` 函数，预先检测并拦截在 L1 CI 中必然失败的命令（curl localhost、psql、npm test），给出友好替代建议
2. `DOD-TEMPLATE.md` 模板从旧格式 evidence ID 升级为 CI 兼容的可执行命令示例
3. 新增测试文件验证 CI 不兼容命令被正确拦截

## 成功标准

- check-dod-mapping.cjs 包含 detectCiIncompatibleCommand 函数
- manual:curl localhost:... → exit 1，给出 node -e 替代建议
- manual:psql ... → exit 1
- manual:npm test → exit 1
- DOD-TEMPLATE.md 无旧格式 manual:code-review / manual:git-check 等 evidence ID
