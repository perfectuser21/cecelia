# PRD: model-registry.js 更新 - 根据 Benchmark 结果修正模型路由

## 背景

2026-02-23 完成了 7 模型 × 5 维度全面 Benchmark（结果存储在 brain/src/data/model-benchmark.json）。

关键发现：
1. **MiniMax-M2.1 是纯推理链（reasoning-only）模型**：响应仅包含 `<think>` 块，strip 后输出为空，无法用于正常 completion 任务
2. **MiniMax-M2.5-highspeed** 是非编程任务最佳 MiniMax：商业分析/意图路由只需 28-63s，仅用少量 thinking tokens
3. **MiniMax-M2** 是 Coding Plan 订阅中的专用模型，质量略好于 M2.5-highspeed

## 需求

### 1. 修复 thalamus agent 默认模型（高优先级）

**问题**：thalamus `recommended_model` 当前是 `MiniMax-M2.1`，但 M2.1 是纯推理链模型，无法产生正常输出
**修复**：改为 `MiniMax-M2.5-highspeed`，同时从 `allowed_models` 中移除 `MiniMax-M2.1` 和 `MiniMax-M2.1-highspeed`

### 2. 废弃 MiniMax-M2.1 系列

在 MODELS 中给以下两个模型加上 `deprecated: true`：
- `MiniMax-M2.1`
- `MiniMax-M2.1-highspeed`

### 3. 新增 MiniMax-M2

在 MODELS 列表中新增：
- id: `MiniMax-M2`，name: `M2 (Coding Plan)`，provider: `minimax`，tier: `premium`

## 验收标准

1. `thalamus.recommended_model === 'MiniMax-M2.5-highspeed'`
2. `thalamus.allowed_models` 不包含 `MiniMax-M2.1` 或 `MiniMax-M2.1-highspeed`
3. `MODELS` 中 M2.1 和 M2.1-highspeed 有 `deprecated: true`
4. `MODELS` 中存在 `MiniMax-M2` 条目
5. 所有现有测试通过
