## PRD - User Profile Context Injection（用户画像注入回复上下文）

**需求来源**: User Memory/Action 三件套 PR3 — 存记忆(PR1) → 听指令(PR2) → 因人而异(PR3)
**功能描述**:
  在 Cecelia 对话时，自动加载 user_memory 表中的用户画像，
  将已知用户事实注入到 LLM system prompt 中，
  使 Cecelia 能结合用户背景生成更个性化的回复。

**涉及文件**:
  - brain/src/user-profile.js（修改：新增 getUserProfileContext 函数）
  - brain/src/orchestrator-chat.js（修改：在 LLM 调用前注入 profile context）
  - brain/src/__tests__/user-profile.test.js（新建：getUserProfileContext 单元测试）
  - brain/src/__tests__/orchestrator-chat.test.js（修改：验证 profile context 注入）

**成功标准**:
  1. getUserProfileContext(userId) 从 user_memory 表读取事实，返回格式化字符串
  2. orchestrator-chat.js 在每次回复时调用此函数，将 profile 注入到对话上下文
  3. 当 user_memory 为空时，函数返回空字符串，不影响正常流程
  4. 全部测试通过（含 mock）

**非目标**:
  - 不修改 user_memory 表结构（PR1 已完成）
  - 不修改 extractAndSaveUserFacts（PR1 已完成）
  - 不做 UI 展示（属于 workspace）
  - 不在 context 注入中包含敏感/隐私过滤（后续 PR 再做）
