# PRD: CI M1 — L1 加 gitleaks secrets 扫描

## 背景

仓库目前无任何 secrets 扫描，硬编码 API key / token 可能被意外提交到 git，造成安全泄露风险。

## 目标

在 L1 Process Gate 新增 `gitleaks/gitleaks-action@v2` job，自动扫描每次 PR 提交中是否包含敏感信息（API keys、passwords、tokens 等）。

## 范围

仅修改 `.github/workflows/ci-l1-process.yml`：
1. 新增 `secrets-scan` job（运行 gitleaks/gitleaks-action@v2）
2. `l1-passed` gate 将 `secrets-scan` 加入 needs 并检查结果

## 成功标准

1. [ARTIFACT] ci-l1-process.yml 包含 `gitleaks/gitleaks-action@v2` job 定义
2. [ARTIFACT] `l1-passed` job 的 needs 数组包含 `secrets-scan`
3. [GATE] CI 全部通过（本 PR 使用 `[CONFIG]` 标签豁免测试覆盖率检查）

## 非目标

- 不修改 L2/L3/L4
- 不添加 .gitleaks.toml 白名单配置（保持默认规则即可）
