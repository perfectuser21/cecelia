# Task Card: CI Gate — 假 DoD Test 检测

## 需求（What & Why）

**功能描述**：扩展 verify-step.sh 假测试检测正则（加入 grep|wc、wc -l、printf），并在 CI L1 新增 `fake-dod-test-check` job，通过共享脚本 `check-fake-dod-tests.cjs` 统一检测。

**背景**：
- verify-step.sh 已有 echo/ls/cat 检测，但缺 grep|wc、wc -l 等常见伪模式
- CI 完全没有假测试检测，假测试可以混入并通过 CI
- 设计原则：本地 verify-step 检测 = CI 检测，两者共用同一脚本

**不做什么**：不改变正常 `manual:node`/`npm test`/`contract:` 的执行逻辑。

## 成功标准

1. 共享脚本 check-fake-dod-tests.cjs 能检测所有假测试模式并 exit 1
2. verify-step.sh 调用共享脚本（或扩展自身正则）覆盖 grep|wc 模式
3. CI L1 fake-dod-test-check job 在 stage 2+ 触发，发现假测试则 blocked

## 验收条件（DoD）

- [x] [BEHAVIOR] check-fake-dod-tests.cjs 脚本存在且可执行
  Test: manual:node -e "require('fs').accessSync('packages/engine/scripts/devgate/check-fake-dod-tests.cjs');console.log('ok: check-fake-dod-tests.cjs exists')"

- [x] [BEHAVIOR] verify-step.sh 假测试正则包含 wc 模式
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/verify-step.sh','utf8');if(!c.includes('wc'))process.exit(1);console.log('ok: wc pattern present')"

- [x] [BEHAVIOR] CI L1 包含 fake-dod-test-check job
  Test: manual:node -e "const c=require('fs').readFileSync('.github/workflows/ci-l1-process.yml','utf8');if(!c.includes('fake-dod-test-check'))process.exit(1);console.log('ok: fake-dod-test-check job present')"

- [x] [PRESERVE] 原有 echo/ls/cat 假测试检测不变
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/verify-step.sh','utf8');if(!c.includes('echo |ls'))process.exit(1);console.log('ok: legacy patterns preserved')"

- [x] [GATE] 所有现有测试通过
  Test: manual:node -e "require('child_process').execSync('npm test --workspace=packages/engine --silent 2>&1 | tail -5',{stdio:'inherit',cwd:process.cwd()})"

## 实现方案

**要改/创建的文件**：
1. `packages/engine/scripts/devgate/check-fake-dod-tests.cjs` — 新建，共享检测脚本
2. `packages/engine/hooks/verify-step.sh` — 扩展假测试正则 + 调用共享脚本
3. `.github/workflows/ci-l1-process.yml` — 新增 fake-dod-test-check job

**Scope**：`packages/engine/`, `.github/`
