# Task Card: 修正 Scanner 对 Brain 内嵌能力的识别规则

## 需求（What & Why）
**功能描述**: 更新 `capability-scanner.js` 的扫描逻辑，使其能正确识别已内嵌在 Brain 进程中运行的能力（熔断保护、看门狗资源监控、情绪感知、自愈与免疫等），避免将其误标为孤岛（island）。

**背景**: 当前 Scanner 只检查两类活动迹象：
1. `related_skills` 在 `run_events`/`tasks` 表中的使用记录
2. `key_tables` 是否有数据

Brain 内嵌能力（如 `circuit-breaker-protection`、`watchdog-resource-monitor`、`emotion-perception`、`self-healing` 等）直接跑在 Brain 进程内，不走外部 skill 调用，部分也没有配置 `key_tables`。导致 Scanner 把这些核心保护机制误标为孤岛，影响后续决策准确性。

**不做什么**: 不修改数据库结构，不修改 capability 的注册数据，不修改 probe 系统。

## 成功标准

1. [ARTIFACT] `capability-scanner.js` 新增 `BRAIN_EMBEDDED_SOURCES` 常量，映射 capability ID 到 cecelia_events 的 source 名称
2. [ARTIFACT] `capability-scanner.js` 新增 `BRAIN_ALWAYS_ACTIVE` 白名单集合，覆盖 `scope='cecelia'` 且无 skill/table 活动的架构性能力
3. [BEHAVIOR] 被白名单覆盖的 capability 扫描结果状态为 `active`，evidence 包含 `brain_embedded:true`
4. [BEHAVIOR] 配置了 `cecelia_events` source 的 capability 能根据 90 天内的事件记录正确判定活跃状态
5. [GATE] CI 全部通过
