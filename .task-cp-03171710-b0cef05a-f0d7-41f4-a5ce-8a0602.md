# Task Card — fix(ci): 修复 CI 审计发现的 5 个问题

## PRD

### 背景
Brain 自动调度的 CI 审计任务，修复 5 个已发现的 CI 配置问题。

### 问题清单

**P0**
1. ci-l3-code.yml — coverage-delta 条件依赖 brain-unit.result==success，brain-unit 被 skip 时 coverage-delta 也 skip，feat+brain 改动可能绕过覆盖率检查

**P1**
2. ci-l3-code.yml — TEST_PATTERNS 缺 .mts 扩展名（.test.mts / .spec.mts），feat PR 使用 mts 测试文件会被误判无覆盖
3. ci-l3-code.yml — [CONFIG] 豁免路径 ^packages/engine/ 太宽，改 engine docs 也能豁免 test coverage

**P2**
4. ci-l2-consistency.yml — engine-l2 version check 豁免列表缺 release: 前缀
5. ci-l1-process.yml — changes job 失败时 engine output 为空，engine-l1/dev-health 检查被跳过（误报通过）

### 修复方案
1. coverage-delta if 条件：去掉 brain-unit.result 依赖，改为仅检查 should_run_l3 + commit_type + brain changes
2. TEST_PATTERNS 加 .mts 支持
3. [CONFIG] 豁免路径改为 ^packages/engine/(hooks|skills|scripts|config|devgate) 精确匹配
4. engine-l2 if 条件加 !startsWith(pr.title, release:)
5. L1 gate 在检查 engine changes 前先验证 changes.result == success

## 成功标准

- [ ] coverage-delta if 条件不含 brain-unit.result（仅检查 should_run_l3 + commit_type + brain）
- [ ] TEST_PATTERNS 包含 .test.mts 和 .spec.mts
- [ ] [CONFIG] 豁免路径使用精确的 packages/engine 子目录模式（非宽泛 ^packages/engine/）
- [ ] engine-l2 version check if 条件包含 release: 豁免
- [ ] L1 gate 在检查 engine 输出前先验证 changes.result == success
