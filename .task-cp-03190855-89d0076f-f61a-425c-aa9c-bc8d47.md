# Task Card: Brain Content Pipeline 基础编排

## 需求（What & Why）
**功能描述**: 让 Brain tick loop 能识别并调度 content-research / content-generate / content-review / content-export 四种 task_type。在 task-router.js 中补全 LOCATION_MAP 路由注册，修复 tick.js 中 content-pipeline 父任务阻塞子任务派发的 bug，确保 content-pipeline-orchestrator.js 的 orchestrateContentPipelines 在 tick 中被调用。

**背景**: content-pipeline 子任务（content-research 等）与父任务（content-pipeline）共享 project_id 时，selectNextDispatchableTask 会因"同一 project 已有 in_progress 任务"而阻塞子任务派发。LOCATION_MAP 也缺少四种子任务类型，getValidTaskTypes() 无法返回它们。

**不做什么**: 不做 Publisher 集成，不做 Dashboard UI，不做 Human Approve 节点。

## 成功标准

1. [ARTIFACT] task-router.js LOCATION_MAP 包含 content-research/content-generate/content-review/content-export 四种类型（→ 'us'）
2. [ARTIFACT] selectNextDispatchableTask 不因 content-pipeline(in_progress) 阻塞子任务派发
3. [BEHAVIOR] tick.js 在每次 tick 中调用 orchestrateContentPipelines(pool)
4. [BEHAVIOR] content-pipeline 状态机正确推进（queued→in_progress→completed）
5. [GATE] CI 全部通过
