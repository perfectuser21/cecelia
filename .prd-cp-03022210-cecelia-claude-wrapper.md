# PRD: cecelia-claude 统一无头调用 Wrapper

## 背景

Cecelia 有两条无头调用路径：

1. `/trigger-cecelia`（任务执行，cecelia-run）：通过 `selectBestAccount()` 自动选择账号，设置 `CLAUDE_CONFIG_DIR=/home/xx/.claude-accountX`
2. `/llm-call`（内部 LLM 调用，cecelia-bridge.cjs）：直接使用 `process.env` (HOME=/home/xx) 里的 `~/.claude` credentials，**无账号评估**

问题：account3 spending-capped 时，`/llm-call` 仍然使用 account3（因为 `~/.claude/.current-account` 是"3"），导致 thalamus/memory/reflection 等内部 LLM 调用全部超时。

## 目标

创建一个统一的 `/home/xx/bin/cecelia-claude` wrapper 脚本：

- 所有无头 claude 调用都通过此脚本
- 脚本自动从 Brain API 获取最佳账号（`/api/brain/account/best`）
- 设置正确的 `CLAUDE_CONFIG_DIR`
- 支持 `CECELIA_CREDENTIALS` env var 强制覆盖（兼容现有路径）
- fallback：Brain API 不可用时使用 `~/.claude/.current-account`

## 变更范围

### 1. 新建 `/home/xx/bin/cecelia-claude`（wrapper 脚本）

```bash
#!/bin/bash
# cecelia-claude: 统一无头 Claude 调用入口
# 自动选择最佳账号，设置 CLAUDE_CONFIG_DIR
```

### 2. 新增 Brain API `/api/brain/account/best`（GET）

返回当前最佳可用账号 ID（调用 selectBestAccount 逻辑的轻量版）。

响应：`{ "account": "2", "reason": "lowest_usage" }`

### 3. 更新 `cecelia-bridge.cjs`（`/llm-call` 路径）

将 `spawn(claudeBin, ...)` 改为使用 `cecelia-claude` wrapper（通过环境变量 `CECELIA_CLAUDE_BIN` 或默认路径 `/home/xx/bin/cecelia-claude`）。

### 4. 更新 `packages/brain/src/routes.js`（新增 account/best 端点）

## 成功标准

1. thalamus/memory/reflection 在 account3 spending-capped 时自动切换到 account2
2. 现有 `/trigger-cecelia` 路径不受影响
3. Brain 正常运行时 `/api/brain/account/best` 响应 < 500ms
4. Brain 不可用时 wrapper fallback 到 `~/.claude/.current-account`
