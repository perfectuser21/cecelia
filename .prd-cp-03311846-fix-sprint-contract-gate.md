# PRD: 修复 Sprint Contract Gate 检查缺失

## 背景

当前 `verify_step1()` 只校验 DoD Test 字段格式，不验证 Sprint Contract 是否真实运行。主 agent 可以直接手填 Test 字段，绕过 Generator + Evaluator 对抗流程，导致 Sprint Contract 机制形同虚设。

## 目标

修复 `packages/engine/hooks/verify-step.sh` 的 `verify_step1()` 函数，使其在允许 `step_1_spec: done` 之前必须检查 Sprint Contract 四个 seal 文件均存在且内容有效。

## 范围

- `packages/engine/hooks/verify-step.sh`
- `packages/engine/tests/hooks/verify-step.test.ts`

## 成功标准

1. 缺少 `.dev-gate-generator-sprint.{branch}` 时，step1 验证 exit 1
2. `.dev-gate-spec.{branch}` 中 verdict != PASS 时，step1 验证 exit 1
3. `.dev-gate-spec.{branch}` 中 divergence_count = 0 时，step1 验证 exit 1（橡皮图章检测）
4. `.sprint-contract-state.{branch}` 中 round = 0 时，step1 验证 exit 1
5. 四个文件均存在且内容合规时，step1 验证 exit 0
