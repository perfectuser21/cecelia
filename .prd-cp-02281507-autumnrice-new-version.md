# PRD：秋米直接写入新版本到左侧 Tab

## 背景

用户在 OkrReviewPage 与秋米对话，说"写一个新版本"时，秋米当前把完整 Initiative 列表写在聊天消息里，用户需要手动确认才能写入数据库。用户希望秋米直接写入数据库，左侧 Tab 自动出现新版本。

## 目标

当用户说"写一个新版本"等触发词时：
1. 秋米直接生成新的 Initiative 列表并存入数据库（创建新 pending_action 记录）
2. 左侧 Tab 自动出现新版本（V2/V3...）并切换过去
3. 右侧聊天只显示简短确认消息（"已在左侧展示新版本，请查看"）

## 涉及文件

- `packages/brain/src/routes.js`：autumnrice/chat 端点
- `apps/api/features/inbox/pages/OkrReviewPage.tsx`：前端版本切换逻辑

## 成功标准

1. 用户说"写一个新版本" → 秋米回复简短确认，左侧出现新 V2 Tab
2. 新 Tab 显示新生成的 Initiative 列表（可编辑）
3. 聊天里不显示完整 Initiative 列表
4. 后端 Brain 测试通过
