# PRD - fix(brain): execution-callback result=null 时补 fallback error_message

## 背景
当 cecelia-run 崩溃或被信号杀死时，它可能仍向 /api/brain/execution-callback 发送回调，但 result 字段为 null。此时 routes.js ~line 2819 的 errorMessage 构建逻辑：
```
errorMessage = String(result || status)
```
在 result=null 时只写入 status 字符串（如 "failed"），完全没有上下文。

## 目标
当 result 为 null 且 status 表示失败时，构建含完整上下文的 fallback errorMessage。

## 功能描述
- execution-callback 端点：result=null + status=failed 时，errorMessage 包含：task_id、timestamp、"callback received but result was null"、exit_code（若有）
- result=null 且 stderr 不为空时：errorMessage 包含 stderr 后 300 字符
- result 有值时：原有逻辑不变
- 单测：mock request，验证 result=null 时 error_message 包含预期上下文

## 涉及文件
- packages/brain/src/routes.js（execution-callback 端点，~line 2703）
- packages/brain/__tests__/（新增单测）

## 成功标准
- [ ] result=null + status=failed 时，errorMessage 包含 task_id、timestamp、"callback received but result was null"、exit_code
- [ ] result=null 且 stderr 不为空时，errorMessage 包含 stderr 后 300 字符
- [ ] result 有值时原有逻辑不变
- [ ] 单测通过

## 非目标
- 不修改其他端点
- 不修改 cecelia-run 的发送逻辑
