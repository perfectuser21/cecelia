# PRD: 三平台发布结果回调 + 飞书通知统一化

## 背景
三平台发布结果目前各自处理（小红书有飞书通知，抖音/快手无统一回调）。本 PR 建立统一的结果追踪和飞书通知机制。

## 目标
所有平台发布完成后，统一写入发布结果（成功/失败 + 作品 ID），并推送飞书 Bot 通知。

## 技术方案
每个发布脚本完成后输出标准 JSON：success / platform / contentType / workId / url / error。N8N 解析结果 → 成功推送飞书「发布成功+链接」，失败推送「失败+错误原因」。写入 Brain PostgreSQL publish_results 表。

## 成功标准
- [ ] 成功发布 → 飞书收到成功通知，含作品链接
- [ ] 失败发布 → 飞书收到失败通知，含错误信息
- [ ] publish_results 表有记录写入
- [ ] 三平台（抖音/快手/小红书）均覆盖
