# PRD：Cortex Insight → Dev Task 自动闭合机制

## 背景

Cortex 皮层在分析事件后，会把洞察写入 `learnings` 表（category=`cortex_insight`，`applied=false`）。
但这些 insight 只是"观察记录"，从未转化为可执行的行动。

盲点来源：Cortex Insight 2026-03-11（id: 90be8b92）
self_model 连续 10 条重复洞察是观察者惰性的量化证据——insight 写入后没有自动触发修复任务的机制，导致洞察积累但零行动。

## 问题

- cortex_insight 类型的 learning 创建后 `applied=false` 永远不变
- 没有机制把 insight 转化为可执行的 dev task

## 目标

1. 在 `cortex.js` `recordLearnings()` 记录 insight 后，自动分析是否涉及代码级修复
2. 若包含代码修复信号：自动生成对应 dev task（写入 tasks 表），设置 `learning_id` 关联
3. 同一 insight 已有对应 task 时不重复创建（去重）
4. insight 触发 task 后标记 `applied=true`

## 实现范围

- **Migration 150**: `tasks` 表添加 `learning_id UUID REFERENCES learnings(id)` 列
- **actions.js** `createTask()`: 支持 `learning_id` 参数，写入 INSERT
- **cortex.js** `autoCreateTaskFromInsight()`: 新函数，分析 insight 内容并创建 task
- **cortex.js** `recordLearnings()`: 插入 learning 后调用 `autoCreateTaskFromInsight()`
- **selfcheck.js**: EXPECTED_SCHEMA_VERSION → `'150'`
- **integration test**: 覆盖 insight→task 链路
