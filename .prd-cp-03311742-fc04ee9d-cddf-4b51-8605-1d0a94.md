# PRD: fix(escalation): cancel_pending 排除 content pipeline 子任务

## 背景
`cancelPendingTasks` 会取消所有 queued/pending 任务，但排除列表缺少 content-copywriting/content-copy-review 等子类型，导致 content pipeline 子任务被 escalation 误取消，pipeline 永久卡在 in_progress。

## 功能描述
在 `cancelPendingTasks` 的 `AND task_type NOT IN` 排除列表中加入以下子类型：
- content-pipeline
- content-research
- content-copywriting
- content-copy-review
- content-generate
- content-image-review
- content-export

受影响文件：`packages/brain/src/alertness/escalation.js`

## 成功标准
- cancelPendingTasks SQL 排除列表包含全部 7 个 content pipeline 类型
- content pipeline 相关任务在 escalation emergency_brake 时不被取消
- 原有 research 和 suggestion_plan 排除保持不变
