## PRD - Stop Hook 重试上限 15→30 + 失败后通知 Brain API

**需求来源**: 用户请求：stop-dev.sh 重试上限太低，且失败时没有通知 Brain
**功能描述**: 修改 packages/engine/hooks/stop-dev.sh，将最大重试次数从 15 提高到 30，并在 30 次重试全部失败后通过 curl 调用 Brain API 将对应 task 状态标记为 failed
**涉及文件**:
- packages/engine/hooks/stop-dev.sh（核心改动）
- packages/engine/package.json（version bump）
- packages/engine/package-lock.json（同步）
- 根 package-lock.json（packages/engine 条目同步）
- packages/engine/VERSION
- packages/engine/.hook-core-version
- packages/engine/regression-contract.yaml（S6 changelog 条目）
- packages/engine/features/feature-registry.yml（changelog 条目）
- packages/engine/docs/paths/（generate-path-views.sh 重新生成）

**成功标准**:
- stop-dev.sh 中重试上限从 15 改为 30
- 所有错误消息中的 "15" 改为 "30"
- 30 次失败后，若 .dev-mode 中有 task_id，调用 PATCH /api/brain/tasks/{id} 设置 status=failed
- Engine 版本正确 bump（6 个文件同步）
- feature-registry.yml 有新 changelog 条目
- CI 通过

**非目标**: 不修改其他重试逻辑、不改变 stop hook 的其他行为

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| Brain API 调用失败 | `|| true` 忽略，不影响 stop hook 流程 | P3 | stop_hook_brain_notify_failed |
| task_id 为空 | 跳过 Brain 通知 | P3 | - |
| curl 超时 | --max-time 5 限制，忽略 | P3 | - |

## 成功标准

1. stop-dev.sh 重试上限从 15 改为 30
2. 失败后通过 Brain API 通知任务失败（task_id 存在时）
3. Engine 版本 6 个文件同步 bump
4. feature-registry.yml 新增 changelog 条目
5. PR title 含 [CONFIG] tag
6. CI 全绿
