# PRD: 知识感知闭环

## 背景
Claude 跨对话没有持久感知：不知道当前 OKR、不知道最近做了什么、不知道有哪些 Brain API 可查。
表和数据都已存在，缺的是"最后一公里"——注入和汇总接口。

## 目标
打通 Claude ↔ Cecelia 知识感知闭环，让 Claude 每次开口就能感知当前状态。

## 要做的 4 件事

### F1: GET /api/brain/context — 汇总接口
新建 `packages/brain/src/routes/context.js`，在 `server.js` 注册为 `/api/brain/context`。
返回 JSON：
- `okr`：当前活跃 OKR 名称 + 完成度
- `recent_prs`：最近 5 个 dev_records（PR标题 + branch + merged_at）
- `active_tasks`：进行中任务（title + task_type + priority）
- `recent_decisions`：最近 5 条 decisions（title + category）
- `summary_text`：以上内容的纯文本摘要（供 Claude 直接阅读）

### F2: GET /api/brain/okr/current — OKR 进度接口
在现有 OKR 路由中添加 `/current` 端点（或新建路由文件）。
查询 `areas → objectives → key_results → okr_projects` 4层，
返回树形结构 + 每层完成度百分比。

### F3: .claude/CLAUDE.md 工具地图
在 `packages/brain/.claude/CLAUDE.md`（或项目根 `.claude/CLAUDE.md`）中新增一节：
```
## Brain 知识查询工具（Claude 可直接调用）
- GET localhost:5221/api/brain/context        → 当前状态全景摘要（推荐先查这个）
- GET localhost:5221/api/brain/okr/current    → OKR 层级 + 进度
- GET localhost:5221/api/brain/tasks?status=in_progress → 进行中任务
- GET localhost:5221/api/brain/dev-records?limit=10 → 最近PR记录
- GET localhost:5221/api/brain/design-docs?type=diary → 日报
- GET localhost:5221/api/brain/decisions?status=active → 有效决策
- GET localhost:5221/api/brain/knowledge/search?q=xxx → 搜索知识库（语义）
```

### F4: Stop Hook 触发 session summary
在 `packages/engine/hooks/stop.sh` 末尾（非 dev 模式时），调用：
```bash
curl -s -X POST localhost:5221/api/brain/conversation-summary \
  -H "Content-Type: application/json" \
  -d '{"trigger":"session_end"}' > /dev/null 2>&1 &
```
同时在 Brain 新增 `POST /api/brain/conversation-summary` 端点，
内部调用现有的 `conversation-consolidator.js`。

## 成功标准
- `curl localhost:5221/api/brain/context` 返回包含 okr/recent_prs/active_tasks 的 JSON
- `curl localhost:5221/api/brain/okr/current` 返回 OKR 树形结构
- `.claude/CLAUDE.md` 包含 Brain 工具地图章节
- `packages/engine/hooks/stop.sh` 末尾有 conversation-summary 调用
- `POST /api/brain/conversation-summary` 端点存在并返回 200
