# PRD：delivery_type 字段 + PR 行为声明机制

## 背景

当前系统最大的 gap：AI 完成一个任务，无法区分"交付的是文档"还是"交付的是系统行为变化"。
CI 只检查代码格式，不检查"系统现在会怎么跑"。
导致 AI 可以通过改一个文件、写一段文档来"看起来完成"，而实际系统行为没有改变。

## 目标

1. 每个 Task/Initiative 必须声明 `delivery_type`，明确交付的是什么
2. `behavior-change` 类型必须有 runtime evidence（代码 + 测试 + 运行日志）
3. PR 描述必须包含 `SYSTEM BEHAVIOR CHANGE` 和 `UNIMPLEMENTED` 两段
4. CI 对 `behavior-change` 类型强制检查 evidence

## 功能描述

### P0：delivery_type 字段

tasks 表新增 `delivery_type` 字段，取值：
- `doc-only`：只改文档
- `code-only`：只改代码（无行为变化）
- `doc+code`：文档 + 代码
- `behavior-change`：系统行为改变（最严格）

默认值：`code-only`

### P0：PR 行为声明模板

/dev Step 8 (PR) 创建 PR 时，body 必须包含：

```
## SYSTEM BEHAVIOR CHANGE
Before: [系统之前怎么跑]
After:  [系统现在怎么跑]
Log:    [关键日志事件]

## UNIMPLEMENTED
- [本 PR 故意不做的事]
```

### P1：behavior-change CI 检查

当 delivery_type=behavior-change 时，CI 额外检查：
- 有对应的测试文件
- 有 runtime log 关键词（通过 grep 验证）

## 成功标准

1. tasks 表有 delivery_type 字段，Brain API 支持读写
2. /dev Step 8 PR body 模板包含 SYSTEM BEHAVIOR CHANGE + UNIMPLEMENTED
3. delivery_type=behavior-change 的 CI 检查脚本存在并可运行
4. 本 PR 自身使用新模板格式（dogfooding）
