# PRD: 选题决策闭环 v1

## 背景

现有 `topic-selection-scheduler.js` 每日 AI 生成选题后，直接创建 `content-pipeline` tasks，但从未向 `content_topics`（候选库）写入数据。`content_topics` 表结构已存在，但与调度流程断联。

## 目标

实现选题决策闭环，让 AI 选题结果进入候选库，并自动采纳 top 5，同时提供 API 查询接口。

## 范围

1. Migration 209：`content_topics` 新增 `source` 字段
2. `topic-selection-scheduler.js`：在创建 tasks 前写入 `content_topics` 候选库，并自动采纳 top 5
3. `content-pipeline.js`：新增 `GET /topics` 和 `GET /topics/today` 两个查询路由
4. 新增单元测试 `topic-selection-candidate-pool.test.ts`

## 成功标准

- Migration 文件存在且格式正确
- `topic-selection-scheduler.js` 在生成选题后写入 `content_topics`，source 为 `ai_daily_selection`
- 自动采纳 top 5（status=adopted）
- `GET /topics` 和 `GET /topics/today` API 可用
- 单元测试覆盖核心行为
