# Task Card: 修复 quickcheck.sh — worktree 里测试静默跳过

## 需求（What & Why）
**功能描述**: quickcheck.sh 在 git worktree 环境下找不到 node_modules，静默 exit 0 跳过所有测试，导致 pre-push hook 形同虚设。
**背景**: /dev 在 worktree 里开发，push 时 pre-push hook 触发 quickcheck.sh，但 worktree 目录里没有 node_modules（依赖在主仓库），所有测试检查被跳过，代码未经测试就 push 出去。
**不做什么**: 不修改测试逻辑本身，不改变 quickcheck.sh 的检查项，只修复路径解析。

## 成功标准

> [BEHAVIOR] 在 worktree 中执行 quickcheck.sh，能正确找到主仓库的 node_modules 并运行测试
> [ARTIFACT] quickcheck.sh 包含 MAIN_REPO_ROOT 变量和 fallback 路径逻辑

## 验收条件（DoD）

- [x] [ARTIFACT] scripts/quickcheck.sh 包含 `git rev-parse --git-common-dir` 逻辑，计算出 MAIN_REPO_ROOT
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/quickcheck.sh','utf8');if(!c.includes('git-common-dir'))process.exit(1)"

- [x] [BEHAVIOR] 在主仓库运行时 MAIN_REPO_ROOT 等于 REPO_ROOT，行为不变
  Test: manual:bash -c "cd /Users/administrator/perfect21/cecelia && MAIN=$(git rev-parse --git-common-dir | xargs dirname) && REPO=$(git rev-parse --show-toplevel) && [ \"$MAIN\" = \"$REPO\" ] && echo OK"

- [x] [BEHAVIOR] 在 worktree 运行时 MAIN_REPO_ROOT 正确指向主仓库
  Test: manual:bash -c "cd /Users/administrator/worktrees/cecelia/fix-quickcheck-worktree-nodemoules && MAIN=$(git rev-parse --git-common-dir | xargs dirname) && echo $MAIN && [ \"$MAIN\" = \"/Users/administrator/perfect21/cecelia\" ] && echo OK"

- [x] [ARTIFACT] node_modules 检查改为先查 REPO_ROOT，fallback 到 MAIN_REPO_ROOT
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/quickcheck.sh','utf8');if(!c.includes('MAIN_REPO_ROOT'))process.exit(1)"

## 实现方案
**要改的文件**: `scripts/quickcheck.sh`
**核心改动**:
1. 在 REPO_ROOT 定义之后，加入 MAIN_REPO_ROOT 计算逻辑
2. ENGINE_CHANGED/BRAIN_CHANGED 的 node_modules 检查改用 fallback 变量
3. npx/node 命令的 node_modules 路径改用 fallback 变量
**不改什么**: 测试逻辑、检查项、退出码语义
