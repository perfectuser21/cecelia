# Task Card: 需求语义解析基础框架

## 需求（What & Why）

**功能描述**: 基于 thalamus 三层架构，建立能够解析中英文需求描述的混合解析器，采用正则+LLM补全的模式，提供快速关键词提取和实体识别能力。

**背景**: 为了实现全自动开发接管，系统需要在 thalamus 架构内解析用户自然语言需求。遵循 fact-extractor 的设计模式：正则快速提取（零延迟、零成本）+ LLM 并行补全（fire-and-forget）+ 反哺进化机制。

**技术选型**:
- 基础解析：正则表达式（零延迟、零成本）
- 智能补全：Haiku LLM（fire-and-forget 并行调用）
- 中英文分词：Natural 库（已有依赖）
- 反哺学习：learned_patterns 表存储进化模式
- 架构集成：遵循 thalamus "LLM只下指令"原则

**不做什么**:
- 不绕过 thalamus 架构做直接解析
- 不使用阻塞式 LLM 调用
- 不实现独立的智能决策逻辑
