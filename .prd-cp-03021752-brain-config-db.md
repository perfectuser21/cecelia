# PRD：大脑架构 DB 配置化 + React Flow 可视化编辑

## 背景

当前 `brain-manifest.js` 将大脑模块/连接全部 hardcode 在 JS 文件中，且 `SuperBrain.tsx` 中还有三处重复的硬编码（NODE_LAYOUT、CONNECTION_PATH、BROKEN_CONNECTIONS_SET）。用户无法在不改代码的情况下调整大脑架构的可视化布局。

## 目标

把架构配置迁移到 PostgreSQL，前端用 React Flow 展示，支持拖拽节点，保存到 DB。改动立即生效，不需要重新部署代码。

## 范围（最小可行方案）

### 后端

1. **新增 migration**：创建 `brain_nodes` 和 `brain_connections` 表
   - `brain_nodes(id, block_id, label, nature, pos_x, pos_y, created_at)`
   - `brain_connections(id, from_node, to_node, is_broken, color, created_at)`

2. **seed 数据**：把 `brain-manifest.js` 中现有的 15 个节点 + 21 条连接写入 DB

3. **API 端点**：
   - `GET /api/brain/architecture` → 返回 nodes + connections
   - `PATCH /api/brain/architecture/nodes/:id` → 更新单节点位置（x/y）
   - `PATCH /api/brain/architecture/connections/:id` → 更新连接属性（is_broken）

### 前端

4. **安装 React Flow**：`@xyflow/react`

5. **改造 SuperBrain.tsx 的 Level 2 视图**：
   - 从 `/api/brain/architecture` 获取数据
   - 用 React Flow 渲染（替代现有自绘 SVG）
   - 拖拽节点后调用 PATCH API 保存位置
   - 断路连接标红色虚线（从 `is_broken` 字段读取）

## 成功标准

- 用户在前端拖动节点，刷新后位置保持不变（已保存到 DB）
- 新增/断开连接不需要改代码
- SuperBrain Level 2 视图使用 React Flow，效果与之前相当

## 不在范围

- 添加/删除节点的 UI（只做拖拽位置保存）
- 修改 Level 1 视图（健康徽章/Issues 面板保持不变）
- 修改 Signal Panel（保持不变）
