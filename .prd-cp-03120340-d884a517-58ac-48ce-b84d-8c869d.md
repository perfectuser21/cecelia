# PRD - tick.js step 6.5 梯度 requeue

## 背景

billing_pause 过期后，当前 step 6.5 一次性将所有 quota_exhausted 任务 requeue，同 tick 全部涌入 queue，导致派发雪崩。需要改为梯度释放。

## 目标

tick.js step 6.5 改为：
1. 每次最多释放 MAX_REQUEUE_PER_TICK（默认 2）个 quota_exhausted 任务
2. 按 priority ASC（P0 先）、created_at ASC 排序选取
3. 若剩余 quota_exhausted 任务 > 0，下一个 tick 继续释放（自然流控）

## 技术要点

- 改 UPDATE 语句：加 ORDER BY priority ASC, created_at ASC LIMIT MAX_REQUEUE_PER_TICK
- PostgreSQL 的 UPDATE...ORDER BY 需要子查询：
  `UPDATE tasks SET status='queued' WHERE id IN (SELECT id FROM tasks WHERE status='quota_exhausted' ORDER BY priority ASC, created_at ASC LIMIT 2)`
- MAX_REQUEUE_PER_TICK = 2（与 Burst Limiter 协调）

## 成功标准

- [ ] billing_pause 过期后，每 tick 最多 requeue 2 个 quota_exhausted 任务
- [ ] P0 任务先于 P1/P2 被 requeue
- [ ] MAX_REQUEUE_PER_TICK 可通过常量调整
- [ ] 单测：插入 6 个 quota_exhausted 任务，验证第一 tick 只 requeue 2 个
