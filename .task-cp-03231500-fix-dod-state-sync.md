# Task Card: 修复状态机与 CI DoD 同步漏洞

## 需求（What & Why）

**功能描述**：修复 verify-step.sh Gate 2 跑 DoD Test 通过后未写回 `[x]` 的 bug，同时在 devloop-check.sh 状态机中增加与 CI 对应的 DoD 完整性检查。

**背景**：
- verify-step.sh Gate 2 执行 DoD Test 命令，全部通过后 exit 0
- 但 Test 通过后 `[ ]` 从未被改写为 `[x]`，task card 永远保留 `[ ]`
- devloop-check.sh 状态机判定 step_2_code=done 只看 verify-step exit code
- CI (check-dod-mapping.cjs) 检查 `[ ]` 是否存在 → 卡住
- **设计原则**：状态机 = CI 镜像，通过状态机必然通过 CI

**不做什么**：不改变 Gate 2 的测试逻辑，不改 Test 命令格式规则。

## 成功标准

1. verify-step.sh Gate 2 每条 Test 通过后自动将对应 `[ ]` 改为 `[x]`
2. devloop-check.sh 在 step_2_code done 检查中新增：task card 无残留 `[ ]` DoD 条目
3. CI check-dod-mapping.cjs 对此类 PR 不再卡住（状态机提前拦截）

## 验收条件（DoD）

- [x] [BEHAVIOR] verify-step.sh Gate 2 Test 通过后自动标 `[x]`
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/verify-step.sh','utf8');if(!c.includes('sed') || !c.includes('[x]'))process.exit(1);console.log('sed [x] write-back present')"

- [x] [BEHAVIOR] devloop-check.sh 新增 DoD 完整性检查（无残留 `[ ]`）
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/lib/devloop-check.sh','utf8');if(!c.includes('dod_complete') && !c.includes('DoD.*\\\\[ \\\\]') && !c.includes('unchecked'))process.exit(1);console.log('devloop DoD check present')"

- [x] [PRESERVE] verify-step.sh Gate 2 失败逻辑不变（Test 失败仍 exit 1）
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/verify-step.sh','utf8');if(!c.includes('Gate 2 失败'))process.exit(1);console.log('Gate 2 fail path preserved')"

- [x] [GATE] 所有现有测试通过
  Test: manual:node -e "require('child_process').execSync('npm test --workspace=packages/engine --silent 2>&1 | tail -5',{stdio:'inherit',cwd:process.cwd()})"

## 实现方案

**要改的文件**：
1. `packages/engine/hooks/verify-step.sh` — Gate 2 成功分支加 sed 写回 `[x]`
2. `packages/engine/lib/devloop-check.sh` — step_2_code done 后加 DoD 完整性条件

**Scope**：`packages/engine/`
