# PRD: Hook-CI 耦合重构 R12 — verify-step.sh 调用 check-version-sync.sh

## 背景

当前 verify-step.sh Gate 0d 只做同目录 sibling 文件的版本一致性扫描，不调用 CI 中使用的 check-version-sync.sh 脚本。开发者在本地通过 verify-step.sh 之后，push 到 CI 时还会因版本文件不同步而失败，形成不必要的调试循环。

## 目标

在 verify-step.sh Gate 0d 末尾追加调用 check-version-sync.sh 的逻辑，消除"本地通过但 CI 失败"的场景。

## 成功标准

- verify-step.sh Gate 0d 在改动 Engine 版本文件时调用 check-version-sync.sh
- jq 不存在时优雅跳过（不报错）
- Engine 版本 5 文件同步 bump
- CI L1-L4 通过
