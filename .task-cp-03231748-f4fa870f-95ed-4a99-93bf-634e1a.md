# Task Card: OKR 飞轮 Brain 侧 — okr-closer.js + tick 集成

## 需求（What & Why）

**功能描述**：为新 OKR 表（okr_initiatives / okr_scopes / okr_projects）实现完成检测飞轮，使其与旧系统（initiative-closer.js for projects 表）一致。

**背景**：
- 层级结构：key_results → okr_projects → okr_scopes → okr_initiatives → tasks（via okr_initiative_id）
- 旧系统（projects 表）已有 initiative-closer.js，新 okr_* 表尚无对应实现
- 新表状态字段：okr_initiatives.status（'pending'/'in_progress'/'completed'），okr_scopes/okr_projects.status（'planning'/'in_progress'/'completed'）

**不做什么**：
- 不修改旧 initiative-closer.js
- 不修改数据库 schema
- 不做 KR 进度回算（超出本任务 scope）

## 成功标准

- okr-closer.js 文件存在且导出三个检测函数
- tick.js 调用 okr-closer.js 的三个函数（在现有 initiative-closer 调用区域后面）
- task-router.js 包含 okr_initiative_plan / okr_scope_plan / okr_project_plan 路由到 /decomp
- thalamus.js ACTION_WHITELIST 添加三个新 task_type
- __tests__/okr-closer.test.js 覆盖三个函数主流程

## 实现方案

**要改的文件**：
- `packages/brain/src/okr-closer.js`（新建）
- `packages/brain/src/tick.js`（添加三次 try/catch 调用）
- `packages/brain/src/task-router.js`（VALID_TASK_TYPES / SKILL_WHITELIST / LOCATION_MAP / TASK_REQUIREMENTS）

**Scope 锚定**：
- okr-closer.js 逻辑：查 okr_initiatives.status IN ('in_progress','active') → 检查关联 tasks（via okr_initiative_id）全部完成 → UPDATE status='completed' → 创建 okr_scope_plan 任务
- okr_scopes：查 status IN ('in_progress','planning') → 子 initiatives 全完成 → UPDATE → 创建 okr_project_plan 任务
- okr_projects：查 status IN ('in_progress','planning') → 子 scopes 全完成 → UPDATE
- 新 task_type：okr_initiative_plan / okr_scope_plan / okr_project_plan，路由到 /decomp，location=xian（general B类）
