## PRD - Dashboard 48h 系统简报展示和端到端测试

**需求来源**: Brain 自动调度任务 - Initiative「48h简报自动化调度集成」第三个 PR
**功能描述**: 在 Dashboard 中展示系统简报，并添加 Brain API 端点；完成端到端测试验证
**涉及文件**:
- `apps/dashboard/src/pages/Reports.tsx` - 简报列表页（新增）
- `apps/dashboard/src/pages/ReportDetail.tsx` - 简报详情页（新增）
- `apps/dashboard/src/router.tsx` - 路由配置（修改）
- `packages/brain/src/server.js` - 添加 API 端点（修改）

**成功标准**:
1. Brain API 端点实现：GET /api/brain/reports 和 GET /api/brain/reports/:id
2. Dashboard 简报列表页展示生成时间、报告类型、推送状态
3. Dashboard 简报详情页展示完整内容（KR 进度、任务统计、系统健康、异常和风险）
4. 端到端测试全部通过（手动触发 tick → 验证 system_reports 表有记录 → 访问 Dashboard 查看）
5. CI 通过，PR 合并

**非目标**:
- 不实现简报编辑功能
- 不实现简报手动创建
- 不修改简报生成逻辑

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| system_reports 表不存在 | 返回空列表，不报错 | P3 | reports_table_missing |
| 简报 ID 不存在 | 返回 404 | P3 | report_not_found |
| 数据库查询失败 | 返回 500 + 错误信息 | P2 | reports_db_error |
| 前端 API 请求失败 | 显示错误状态，提供重试 | P3 | reports_fetch_error |
