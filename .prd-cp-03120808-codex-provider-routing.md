# PRD: codex-bridge /run 端点 + executor provider 路由

## 背景

西安 Mac mini 有 5 个 Codex 账号（team1-team5），通过 codex-bridge（:3458）管理。
目前 codex-bridge 只有 `/execute`（通用）和 `/execute-review`（代码审查）两个端点。
executor.js 无法将 `codex_qa` 或 `provider=codex` 的任务路由到西安执行。

用户需要：任务创建时可以通过 `provider` 字段自由指定执行器：
- `provider: 'claude'` → 本地 cecelia-bridge（Claude Code）
- `provider: 'minimax'` → 本地 cecelia-bridge（MiniMax API）
- `provider: 'codex'` → 西安 codex-bridge（Codex CLI + team1-5 账号）

## 目标

1. 给 codex-bridge 加 `/run` 端点（接受 skill + task 参数的通用接口）
2. executor.js 在 `location=xian` 或 `provider=codex` 时调用西安 codex-bridge `/run`
3. task-router.js 将 `codex_qa` 路由到 `location=xian`
4. DB migration：tasks.location CHECK 加入 'xian'

## 成功标准

- `/run` 端点在西安 codex-bridge 上可接受 POST 请求并异步执行
- executor.js `triggerCodexBridge()` 函数向 100.86.57.69:3458/run 发送请求
- `task_type=codex_qa` 的任务 location 为 'xian'，executor 路由到 codex-bridge
- DB migration 146 添加 'xian' 到 location CHECK 约束
- 现有测试通过

## 范围

**修改文件**：
- `infrastructure/scripts/codex-bridge/codex-bridge.cjs`（西安，添加 /run）
- `packages/brain/src/executor.js`（添加 triggerCodexBridge + provider routing）
- `packages/brain/src/task-router.js`（codex_qa → xian）
- `packages/brain/migrations/146_location_xian.sql`（DB migration）
- `packages/brain/src/selfcheck.js`（EXPECTED_SCHEMA_VERSION → 148）
- 相关测试文件

**不修改**：routes.js（不是核心路由层），dashboard（不在本 PR 范围）
