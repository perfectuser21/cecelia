# PRD: 社媒热点数据接入接口

## 背景

Project #4（数据采集完整化，数据闭环 KR，进度 0%）当前 planning。该 project 是内容选题、发布等所有下游能力的基础依赖，必须首先启动。

HK VPS 上已运行 8 个平台采集器（douyin/kuaishou/xiaohongshu/toutiao/weibo/zhihu/channels/wechat），采集数据存入 TimescaleDB，但 Brain 的内容生成系统目前未与 TimescaleDB 打通，数据闭环 KR 进度为 0%。

## 功能描述

在 Brain 中新增一个 HTTP API 端点（`GET /api/brain/social/trending`），用于查询各平台最近的热门内容数据，使内容生成系统能够消费来自 TimescaleDB 的社媒采集数据。

## 成功标准

- `packages/brain/src/routes/social-trending.js` 文件存在，并在 `src/routes.js` 中注册
- `GET /api/brain/social/trending` 端点返回 HTTP 200，响应体为 JSON 数组，支持 platform 过滤和 limit 参数
- 当 TimescaleDB 不可达时，降级返回空数组（不报错）
- CI 全部通过（含 social-trending.test.ts 单元测试）

## 不做什么

- 不修改现有采集器逻辑
- 不改动 content_topics 表结构
- 不实现内容生成系统对热点数据的自动消费

## Scope

`packages/brain/` — 新增路由文件和单元测试，修改 routes.js 注册路由
