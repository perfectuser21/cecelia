# Task Card: brain-unit diff coverage gate

## 需求（What & Why）

**功能描述**: 在 CI 中加入 diff coverage 门禁 — PR 新增行的测试覆盖率 < 80% 则 CI fail，迫使开发者为新代码路径补测试。

**背景**: 现有 CI 只检查"有没有测试文件"（L3 门禁），不检查"测试是否覆盖新增代码行"。每次修复 `execution.js` 的 callback 分支只加了 mock 单测，集成测试从没跟上，导致修一修就坏了。

**不做什么**: 不改 vitest.config.js（已有 coverage 配置），不改 brain-unit 现有逻辑，不要求 100% 全量覆盖（只查 PR diff 新增行）。

## 成功标准

- [ARTIFACT] `.github/workflows/ci.yml` 含 `brain-diff-coverage` 新 job
- [BEHAVIOR] 新 job 在 PR brain 代码变动时运行 vitest --coverage + diff-cover，对新增行覆盖率 < 80% 时 exit 1
- [BEHAVIOR] `ci-passed` 将 brain-diff-coverage 纳入门禁

## 实现方案

**要改的文件**: `.github/workflows/ci.yml`（仅此一个文件）

**新增 job**: `brain-diff-coverage`
- needs: [changes, brain-unit]
- if: brain changes AND brain-unit passed
- steps: checkout (fetch-depth:0) → node setup → npm ci → vitest --coverage → diff-cover
- timeout: 35min（coverage 比纯测试慢 ~50%）

**ci-passed 变更**: needs 列表加 brain-diff-coverage，check 函数加一行

**不改**: vitest.config.js（已有 coverage.reporter=['lcov'] + reportsDirectory='./coverage'）
