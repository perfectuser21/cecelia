# PRD: 飞书增强 v1 —— 滚动上下文 + 多用户识别 + 群聊支持

## 背景

当前飞书 Bot 每次对话都是无记忆的（`handleChat(text, {}, [])`），且只能识别 owner（Alex）一个用户，无法区分苏彦卿、于瑾等团队成员，也不支持群聊。

## 目标

让 Cecelia 在飞书中有完整的对话体验，能记住上下文、认识团队成员、在群聊中响应 @mention。

## 功能

### 1. 滚动对话上下文

- 新建 `feishu_conversations` 表存储每用户的对话历史
- 飞书消息触发时，先查询该 open_id 最近 10 轮对话（20条 user+assistant 消息）
- 将历史作为 `messages[]` 传入 `handleChat`
- 回复后写入该轮对话（user message + assistant reply）

### 2. 多用户识别

- 新建 `feishu_users` 表缓存 open_id → 用户名/信息
- 通过飞书 `GET /open-apis/contact/v3/users/{user_id}?user_id_type=open_id` 获取用户名
- 缓存 TTL: 24小时
- 将 `sender_name` 注入到 `context`，`handleChat` 用 `context.sender_name || 'Alex'` 写记忆
- 团队映射（owner 识别）：Alex(徐啸) 的 open_id → user_id='owner'，其他人 → 'guest'

### 3. Source 注入

- `context.source = 'feishu'` 传入 handleChat
- memory_stream 写入时标注来源为 `feishu`

### 4. 群聊 @mention 支持

- 允许 `chat_type === 'group'` 的消息
- 仅当 `message.mentions` 中包含 Bot 的 open_id 时才回复
- 回复到群聊时用 `chat_id` 而非 `open_id` 作为 receive_id
- 去掉消息文本中的 `@Cecelia` 前缀再传给 handleChat

### 5. 修改 handleChat 支持 sender_name

- `handleChat` 接收 `context.sender_name` 和 `context.source`
- memory_stream 写入时用 `sender_name`（而非硬编码 'Alex'）
- `extractAndSaveUserFacts` 只对 user_id='owner' 调用
