# PRD: Content Pipeline 配置可视化 — 前端编辑器 + Executor 注入

## 背景

让 ZenithJoy 员工在 Dashboard 前端编辑内容 pipeline 每步的 prompt/规则/参数。
本 PR 实现 Dashboard 编辑器（PR3）和 Executor 注入（PR4）。
依赖 PR #1611（已合并）提供的 DB 表和 CRUD API。

## 成功标准

- `/content/config` 路由可访问，展示所有内容类型的 prompt/规则编辑器
- 用户修改 prompt/规则并点击保存，后端 `content_type_configs` 表更新成功
- `executeCopyReview` 执行时优先使用 DB 中的 `copy_rules.brand_keywords`/`banned_words`，无 DB 配置时 fallback 到 YAML 默认值
