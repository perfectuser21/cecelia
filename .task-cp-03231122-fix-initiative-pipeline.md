# Task Card: 修复 Initiative Coding Pathway 全链路闭环

## 需求（What & Why）

**功能描述**: 修复 OKR→KR→Project→Scope→Initiative→Task 全链路中的 4 个断点，使整个 pipeline 能够自动闭环运转。

**背景**: 
当前 Initiative Pipeline 存在以下问题：
1. `actions.js` 创建 initiative 时直接设为 `active`，绕过 pending 队列，容量控制（MAX_ACTIVE_INITIATIVES）失效
2. KR 层缺少完成检测（checkKRCompletion）和下一个 KR 激活逻辑（activateNextKRs）
3. `tick.js` 没有调用 KR 完成检测函数
4. `task-router.js` 中 `scope_plan`/`project_plan` task_type 没有路由映射，被创建后永久卡在队列

**不做什么**: 不改 /dev SKILL.md，不改 initiative_verify 逻辑，不重构 planner.js

## 成功标准

- [ARTIFACT] actions.js 中 createInitiative 初始 status 为 `pending`
- [ARTIFACT] 新增 packages/brain/src/kr-completion.js，包含 checkKRCompletion 和 activateNextKRs 函数
- [BEHAVIOR] tick.js Section 0.7 调用 checkKRCompletion，Section 0.11 调用 activateNextKRs
- [ARTIFACT] task-router.js 中 scope_plan 和 project_plan 路由到 /decomp
- [GATE] CI 全部通过
