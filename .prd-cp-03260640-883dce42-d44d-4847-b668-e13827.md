# PRD: feat(capture): Capture Digestion Pipeline v1

## 背景
captures 表（收件箱）已建好，用户可通过 Dashboard QuickCapture 输入。现在需要建立消化管道：AI 分析原始 capture → 拆解为原子事件 → 用户确认路由到目标表。

## 目标
1. 建立 capture_atoms 表 — 存放 AI 拆解后的原子事件
2. Brain tick 中增加 capture-digestion job — 扫描 inbox captures，调 LLM 拆解
3. /inbox 页面增加 Review 步骤 — 用户确认每个 atom 的类型和路由
4. 确认后写入目标表（notes/knowledge/content_topics/tasks/decisions/events）

## 成功标准
- capture_atoms 表存在，含 capture_id/content/target_type/target_subtype/suggested_area_id/status/routed_to_table/routed_to_id/confidence
- events 表存在，含 name/date/event_type/location/people/area_id
- Brain digestion job 能将 inbox capture 拆解为 atoms
- /inbox 页面显示 pending_review atoms
- confirm 操作后根据 target_type 写入对应目标表（notes/tasks/decisions/events 等）
- PATCH /api/brain/capture-atoms/:id 支持 confirm/dismiss 操作
- 所有现有测试通过，不破坏现有功能
