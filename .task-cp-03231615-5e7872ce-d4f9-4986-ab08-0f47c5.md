# Task Card: CI Gate — 新增 hook/lib 文件必须有 RCI 条目

## 需求（What & Why）
**功能描述**: 当 PR 新增 `packages/engine/hooks/` 或 `packages/engine/scripts/devgate/` 下的文件时，CI L2 检查该文件是否已在 `regression-contract.yaml` 中有对应 RCI 条目。

**背景**: 新能力（hook/devgate 工具）悄悄滑入生产而无 RCI 覆盖，导致回归测试漏洞。此检查作为防护门禁，强制要求新文件必须声明其 RCI 条目。

**不做什么**: 不检查已有文件（只看新增）；不检查 hooks/ 和 devgate/ 之外的目录；不修复已有文件的 RCI 缺失问题。

## 成功标准

1. 新增 `packages/engine/scripts/devgate/check-new-files-need-rci.cjs`，导出 `scanMissingRci` 函数，接受 changedFiles 数组和 contractContent 字符串参数
2. CI L2 (`ci-l2-consistency.yml`) 的 `engine-l2` job 包含 `new-files-need-rci-check` step，检测 hooks/ 和 scripts/devgate/ 下新增文件
3. 三重降级保护：scanMissingRci 空数组输入返回空数组（降级保护 1）；scanMissingRci 非目标路径返回空数组（降级保护 3）；scanMissingRci 已有 RCI 的文件返回空数组（降级保护 2）；scanMissingRci 缺少 RCI 的新文件返回该文件路径
4. 测试文件存在且格式合法（packages/engine/tests/devgate/check-new-files-need-rci.test.ts）
