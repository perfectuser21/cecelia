# Task Card: Engine Dev 线升级 — 新增 code-quality + prd-audit Codex 门禁

## 需求（What & Why）
**功能描述**: 将 /dev 流程中的代码质量检查从"SKILL.md 纸上谈兵的 subagent prompt 建议"升级为"西安 Codex 异步执行 + devloop-check.sh 硬门禁"。新增 2 个 Engine Skill（code-quality、prd-audit），更新 devloop-check.sh 条件顺序使所有 Codex 审查在 CI 之前完成。

**背景**: 当前 devloop-check.sh 已有 3 个 Codex 异步检查点（intent-expand、cto-review、PR review），但代码质量审查（垃圾清理+简洁性+DRY+规范）和 PRD 覆盖审计仍然只是 SKILL.md 中的 prompt 建议，不影响闭环判断。需要将它们升级为与 cto-review 同等地位的硬门禁。

**不做什么**:
- 不删除 engine/skills 下的旧 audit/qa/assurance（另起任务处理）
- 不修改 Brain 的 executor/task-router（只新增 task_type 注册）
- 不改 CI workflow 文件
- 不改 stop-dev.sh（它只 source devloop-check.sh）

## 成功标准
1. [ARTIFACT] `packages/engine/skills/code-quality/SKILL.md` 存在，包含垃圾清理+简洁性+DRY+规范 4 维度审查 prompt

3. [BEHAVIOR] devloop-check.sh 新增条件 2.6（code_quality）和 2.7（prd_audit），在 CI 检查（条件 3）之前
