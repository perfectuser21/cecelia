## PRD - Cecelia 嘴巴添加 call_brain_api 工具 + brain-manifest 注入

**需求来源**: Cecelia 嘴巴目前只有 4 个硬编码 action（create_task/cancel_task/save_note/update_user_profile），无法主动查询 Brain API（如查找飞书群 chat_id、查看当前任务队列、触发 tick 等），也不知道自身有哪些能力。

**功能描述**:
1. 在 `orchestrator-chat.js` 的 Claude API 调用中添加 `call_brain_api(path, method, params)` 工具，让嘴巴能动态调用任意 Brain API 端点（GET/POST）
2. 将 `brain-manifest.generated.json` 的能力摘要注入到 system prompt（通过 `buildUnifiedSystemPrompt`），让嘴巴知道自己有哪些能力，何时该委托 Brain

**涉及文件**:
- `packages/brain/src/orchestrator-chat.js` — 主改动：添加工具 + manifest 注入
- `packages/brain/src/__tests__/orchestrator-chat.test.js` — 补充测试

**成功标准**:
- Cecelia 被问"你能不能在群里打招呼"时，能调用 call_brain_api 查询飞书群列表，而不是说"我没有这个能力"
- Cecelia 被问"你有哪些能力"时，能基于 brain-manifest 给出准确回答
- call_brain_api 调用失败时，静默返回错误信息给 LLM（不影响主对话）
- brain-manifest 注入失败时，静默降级（不影响对话）

**非目标**:
- 不修改 brain-manifest.generated.json 本身
- 不改变 thalamus/cortex 逻辑
- 不添加其他工具（call_brain_api 是唯一新工具）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 |
|----------|----------|----------|
| call_brain_api 网络超时 | 返回 {error: "timeout"} 给 LLM | P3 |
| Brain API 返回 4xx/5xx | 返回 {error: "API 错误: xxx"} 给 LLM | P3 |
| brain-manifest 文件不存在 | 静默降级，不注入 | P3 |
| brain-manifest JSON 解析失败 | 静默降级，不注入 | P3 |
