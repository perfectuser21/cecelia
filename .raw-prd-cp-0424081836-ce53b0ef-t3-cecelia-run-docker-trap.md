# T3 — cecelia-run.sh docker 模式 worktree remove no-op 修复

## Goal
`cecelia-run.sh:L409` cleanup() trap 在 docker 容器内跑 `git worktree remove` 必失败（容器内无 worktree metadata）→ 导致 host 侧留"半清理"worktree。改为仅 host 模式执行 remove，docker 模式写 flag 文件到共享 volume。

## 背景
Phase B2 forensic 发现的次要 bug：cecelia-run 在 Brain docker 容器内跑（harness dispatch 等场景），exit trap 里 `git worktree remove` 容器内 .git 没 worktree metadata → 必 fail → host 侧 worktree 残留，最终靠 zombie-cleaner/sweep 清（兜底但有延迟）。

## Tasks
1. 在 `cecelia-run.sh` 加 docker 检测：`[[ -f /.dockerenv ]]` 为 docker 模式
2. 修 cleanup() trap（约 L409）：
   - host 模式 → 保留原 `git worktree remove --force` 逻辑
   - docker 模式 → 写 flag `/tmp/cecelia-worktree-cleanup-flags/<basename>.flag` 内容为 worktree path
3. 加简单 bash 测试（`packages/brain/scripts/test-cecelia-run-docker-trap.sh`），覆盖 host vs docker 模式分支

## 成功标准
- host 模式 behavior 完全不变（原逻辑路径保留）
- docker 模式不再调 `git worktree remove`（改为 touch flag 文件）
- 新 bash 测试 pass（3 cases：host / docker / flag 路径生成）

## 不做
- 不实施 host 侧 flag 消费（#2572 zombie-cleaner 目前已能清 orphan，24h 观察后决定要不要加消费逻辑）
- 不改 Dockerfile（用 `/.dockerenv` 内建检测，无需改 entrypoint）
- 不重构 cleanup() 其它部分（杀进程 / release_lock 逻辑不动）
