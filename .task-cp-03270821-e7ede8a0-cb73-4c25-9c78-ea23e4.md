# Task Card: 文档+聊天分栏界面（Notion AI 风格）

## 需求（What & Why）

**功能描述**：在 Cecelia Dashboard 建一个 Notion AI 风格的分栏界面：
- 左边：文档内容（markdown 渲染+编辑，通过 Brain design_docs API 存取）
- 右边：Claude 聊天窗口（支持模型选择）
- Claude 可通过特殊标记更新左边文档内容
- 记忆模式：文档全文 + 最近 10 条对话历史

**背景**：Brain 已有 design_docs API（CRUD），orchestrator-chat 已有聊天链路。
本任务将二者组合成可交互的编辑界面，放在 `/knowledge` 知识库模块下。

**不做什么**：不改 Cecelia 意识链路（orchestrator-chat），不做复杂 WebSocket 推送，
文档更新通过解析 Claude 回复中的 `<doc_update>` 标记实现客户端即时更新。

## 成功标准

- 打开 `/docs/:id` 页面，左右分栏显示（文档内容 + 聊天窗口）
- 右边输入问题，Claude 回复显示在右侧，若含 `<doc_update>` 标记则更新左边文档
- 模型选择下拉框（Haiku/Sonnet/Opus）切换生效
- 文档内容通过 Brain design_docs API 读写持久化
- 新增 `/docs/list` 文档列表页（从 design_docs 读取）

## 实现方案

**要改的文件**：
1. `packages/brain/src/routes/doc-chat.js`（新建）— 轻量 doc-chat 端点，支持 model 参数
2. `packages/brain/src/server.js`（修改）— 注册 `/api/brain/doc-chat` 路由
3. `apps/api/features/knowledge/pages/DocEditor.tsx`（新建）— 分栏界面
4. `apps/api/features/knowledge/pages/DocList.tsx`（新建）— 文档列表
5. `apps/api/features/knowledge/index.ts`（修改）— 注册新路由和组件

**技术方案**：
- Brain doc-chat 端点：接受 `{ message, messages[], docContent, docId, model }`，
  用 callLLM 发送请求（系统提示包含文档全文），解析 `<doc_update>` 标记，
  返回 `{ success, reply, docContent? }`
- 前端 DocEditor：左右 50/50 布局，左侧 markdown 预览+编辑切换，
  右侧聊天 + 模型选择器，聊天发送后解析响应更新文档
- 文档保存：docContent 变更时 PUT /api/brain/design-docs/:id

**Scope 锚定**：knowledge feature 下新增 2 个页面 + Brain 1 个新路由
