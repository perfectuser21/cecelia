# Task Card: 封堵孤岛 — /dev 强制注册 + 集成测试门禁

## 需求（What & Why）

**功能描述**：
新功能通过 `/dev` 开发后，必须：
1. 在 `docs/instruction-book/` 更新说明（防止功能无文档成为孤岛）
2. 包含集成测试（防止功能无测试验证成为孤岛）

两项要求都通过 CI 门禁强制执行，不可绕过。

**背景**：
capability-scanner 每 6 小时扫描孤岛（从未调用或无测试覆盖的能力），Magic Moment 期间构建了大量模块，但部分功能没有文档、没有集成测试，导致 scanner 将其标记为孤岛。根本原因是没有门禁强制要求开发者注册新功能。

**不做什么**：
- 不修改 capability-scanner 逻辑
- 不要求每个 PR 都在 system_registry 表插入数据（太重）
- 不修改现有 L3 unit test 检查逻辑

## 成功标准

1. [ARTIFACT] `ci-l1-process.yml` 新增 `instruction-book-required` job
2. [ARTIFACT] `ci-l3-code.yml` 新增 Brain `feat:` PR 集成测试检查
3. [BEHAVIOR] `feat:` PR 未更新 instruction-book → L1 失败，提示补充文档
   Test: manual:node -e "require('fs').readFileSync('.github/workflows/ci-l1-process.yml','utf8').includes('instruction-book-required')||process.exit(1)"
4. [BEHAVIOR] Brain `feat:` PR 无集成测试 → L3 失败，提示补充集成测试
   Test: manual:node -e "require('fs').readFileSync('.github/workflows/ci-l3-code.yml','utf8').includes('integration-test-required')||process.exit(1)"
5. [ARTIFACT] 本 PR 自身满足两个门禁（附带 instruction-book 条目 + 集成测试）
6. [GATE] 所有现有测试通过
   Test: manual:node -e "require('fs').accessSync('packages/brain/src/__tests__/integration/system-registry-api.integration.test.js')"

## 实现方案

**要改的文件**：
1. `.github/workflows/ci-l1-process.yml` — 新增 `instruction-book-required` job
2. `.github/workflows/ci-l3-code.yml` — 新增 `integration-test-required` job，接入最终 gate
3. `docs/instruction-book/features/dev-registration-gate.md` — 新建说明书条目
4. `packages/brain/src/__tests__/integration/system-registry-api.integration.test.js` — 新建集成测试

**Scope 锚定**：
- 改 CI YAML：新增 job，不修改现有 job
- 新建文档：不修改已有文档
- 新建测试：测试 `/api/brain/registry` 路由逻辑（mock DB）

**门禁逻辑**：
- instruction-book-required：只对 `feat:` PR 执行；`[SKIP-DOCS]` 或 `[CONFIG]` 标签豁免
- integration-test-required：只对 `feat:` PR + Brain 代码变更执行；`[CONFIG]` 标签豁免
