## 背景

当前 /dev 流程中，同一个 AI 写 PRD→DoD→Test→Code，所有验证自引用。CI 只检查格式（Test 字段非空），不检查语义（Test 是否真的验证了 PRD 承诺）。

典型失败模式：PRD 说"自动生成 X"，DoD 只验"X 文件存在"，代码什么都没做也能通过。

## 方案

两刀切：

### 刀一：PRD 承诺分类 + DoD Test 类型强制匹配（Step 5）

PRD 每条成功标准标注类型，DoD Test 类型必须匹配：

| PRD 承诺类型 | DoD Test 最低要求 |
|---|---|
| [ARTIFACT] 产出物存在 | grep/ls/file-exists OK |
| [BEHAVIOR] 运行时行为 | tests/*.test.ts 或 manual:curl+断言 必须 |
| [GATE] 门禁通过 | contract:RCI-ID OK |

核心约束：BEHAVIOR 类不允许用 grep/ls/file-exists 验证。

### 刀二：Step 7.5 独立审计员（PRD→代码语义对比）

Step 7.4 代码审查后新增 Step 7.5，一个独立 subagent 做 PRD 对比审计：
- 读 PRD 成功标准
- 读代码 diff
- 读 DoD+Test
- 逐条判断：MATCH / DOWNGRADED / MISSING

### 刀三：check-dod-mapping.cjs 增强

检测 BEHAVIOR 类条目使用了弱测试（grep/ls/file-exists）时报错。

2. [ARTIFACT] Step 7 (07-verify.md) 包含 Step 7.5 独立审计员 subagent 流程（PRD→代码语义对比）
