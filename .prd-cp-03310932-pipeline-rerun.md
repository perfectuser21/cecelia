# PRD: content-pipeline /run 支持重新生成已完成的 pipeline

## 问题
POST /api/brain/pipelines/:id/run 对 status=completed 的 pipeline 返回 400。
用户点"重新生成"按钮会触发失败。

## 改动
`packages/brain/src/routes/content-pipeline.js`：
- 移除 `if (pipeline.status === 'completed') return 400` 的限制
- 改为：completed 状态时先重置 status='queued'，然后正常触发

## DoD
- [ ] POST /run 对 completed pipeline 返回 202（不再 400）
  Test: contract:build-passes

## 成功标准
- [x] 重新生成按钮不再触发 400 失败
- [x] completed 状态 pipeline 重置为 queued 后可重新生成
