# Task Card: 知识感知闭环 — context汇总接口 + OKR接口 + CLAUDE.md工具地图 + Stop Hook summary

## 需求（What & Why）

**功能描述**:
打通 Claude 跨对话感知断层，提供 4 项能力：
1. `GET /api/brain/context` — 汇总接口，一次返回 OKR状态 + 活跃任务 + 近期决策
2. `GET /api/brain/okr/current` — 当前 OKR 进度（objectives + key_results，带进度百分比）
3. `.claude/CLAUDE.md` 新增 Brain API 目录章节，让 Claude 在任何对话中都能快速定位接口
4. Stop Hook 对话结束时触发 conversation-consolidator，写入本次 session summary

**背景**: Cecelia 跨对话感知断层问题：Claude 每次启动都需要重新探索系统状态，缺少汇总入口。conversation-consolidator 现在依赖 tick 被动触发（30min 空闲），Stop Hook 主动触发可保证每次对话都留有记录。

**不做什么**:
- 不改 conversation-consolidator 的 30min 空闲逻辑
- 不做 GitHub PR 数据（需要 gh cli，CI 环境无法测试）
- 不改前端 Dashboard

## 成功标准

- `GET /api/brain/context` 接口存在，返回包含 `okr`、`tasks`、`decisions` 字段的 JSON 对象
- `GET /api/brain/okr/current` 接口存在，返回带进度的 OKR 数据（objectives + key_results）
- `.claude/CLAUDE.md` 新增 `## Brain API 目录` 章节，列出核心接口

- CI 全部通过
