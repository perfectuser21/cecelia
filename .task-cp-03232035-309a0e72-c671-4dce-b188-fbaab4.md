## 实现方案

**Migration 184** (`packages/brain/migrations/184_write_path_okr_tables.sql`):
- DROP FK: tasks.project_id_fkey, tasks.goal_id_fkey, project_kr_links.project_id_fkey, project_kr_links.kr_id_fkey, project_repos.project_id_fkey
- ADD to okr_projects: sequence_order INTEGER, current_phase VARCHAR(50), time_budget_days INTEGER

**列映射关键规则**:
- goals.type='mission'/'area_okr' → objectives (title, priority, status, description)
- goals.type='area_kr'/'global_kr' → key_results (title, priority, progress, status, description)
- goals.type='vision' → visions (title, status, description)
- projects.type='initiative' → okr_initiatives (title=name, scope_id=parent_id)
- projects.type='scope' → okr_scopes (title=name, project_id=parent_id)
- projects.type='project' → okr_projects (title=name, description, status)
- pause_kr/activate_kr → UPDATE key_results (active KRs are in key_results per planner.js 读操作)
- adjust_priority → UPDATE okr_projects SET sequence_order
- update_roadmap → UPDATE okr_projects SET current_phase
- executePlanAdjustment → UPDATE okr_projects SET time_budget_days, end_date
- initiative APPROVED → UPDATE okr_initiatives SET status='completed'

**selfcheck.js**: EXPECTED_SCHEMA_VERSION = '184'
