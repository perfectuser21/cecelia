# PRD: 修复 runner.sh $CODEX_HOME） 变量边界 bug

## 背景
runner.sh 在 bash 3.2 + LANG=en_US.UTF-8 + set -u 环境（西安 Mac mini M4）下，
全角括号 `）`（UTF-8: \xef\xbc\x89）被 bash 3.2 当作变量名的一部分，
导致展开不存在的变量 `CODEX_HOME）`，触发 unbound variable 报错。
team1 quota 超额后，account rotation 因崩溃无法切换到 team2-5。

## 需求
修复三处 `$CODEX_HOME）` → `${CODEX_HOME}）`，加花括号明确变量边界。

## 成功标准
- 三处 echo 语句使用 ${CODEX_HOME}） 格式
- bash 3.2 + UTF-8 locale 下不报 unbound variable
- 所有现有测试通过
