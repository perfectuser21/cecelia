# Task Card: 统一孤儿任务恢复逻辑（删除 startup-recovery.js DB 段）

## 需求（What & Why）
**功能描述**: 删除 startup-recovery.js 中的 DB 孤儿任务恢复段，统一由 executor.js::syncOrphanTasksOnStartup 负责完整恢复逻辑。

**背景**: 两条并行孤儿处理路径存在优先级冲突。路径A（startup-recovery.js）无条件将 in_progress 重置为 queued，先于路径B（executor.js::syncOrphanTasksOnStartup）运行，导致智能路径等到的是空集，等于空转。

**不做什么**: 不修改 syncOrphanTasksOnStartup 的核心逻辑；不修改 executor.js 导出。

## 成功标准

1. [ARTIFACT] startup-recovery.js 中无 UPDATE tasks 语句（pool 参数移除）
2. [ARTIFACT] server.js 显式调用 syncOrphanTasksOnStartup（启动序列正确）
3. [ARTIFACT] tick.js 中 initTickLoop 不再内部调用 syncOrphanTasksOnStartup（避免双调用）
4. [BEHAVIOR] Brain 重启后，in_progress 孤儿任务经由 syncOrphanTasksOnStartup 处理（进程检测→区分重试vs失败）
5. [GATE] CI 全部通过
