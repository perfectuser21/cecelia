# Task Card: P1 Worktree 路径持久化

## 背景

Worktree 默认创建在 `{主仓库}/.claude/worktrees/` 下，系统重启或上下文压缩后会话不会丢失 worktree（因为不在 /tmp），但如果主仓库目录本身有变动，或者用户希望 worktree 存放在独立的持久目录，需要一个可配置的路径机制。

任务目标：将 `worktree-manage.sh` 的默认路径改为 `~/worktrees/`（持久目录，跨会话存在），并同步更新文档和安全检查逻辑。

## 涉及文件

- `packages/engine/skills/dev/scripts/worktree-manage.sh`（路径逻辑核心）
- `packages/engine/skills/dev/steps/00-worktree-auto.md`（文档更新）
- `packages/engine/skills/dev/scripts/worktree-gc.sh`（安全检查适配）

## 成功标准

- worktree-manage.sh 的 `generate_worktree_path` 使用 `~/worktrees/{project-name}/{task-name}` 路径
- 支持环境变量 `WORKTREE_BASE` 覆盖（默认 `~/worktrees`）
- 00-worktree-auto.md 中记录 WORKTREE_BASE 的说明和用法
- worktree-gc.sh 的路径安全检查支持 `~/worktrees/` 前缀
