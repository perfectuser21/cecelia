# DoD - [SelfDrive] 验证并激活多平台内容发布

## 完成标准

### 1. 发布系统状态验证
- [ ] 检查八大平台发布工具的存在性和完整性
- [ ] 验证每个发布工具的核心脚本和配置文件
- [ ] 确认发布工具的依赖关系完整性

### 2. 依赖关系分析
- [ ] 分析每个发布工具与外部系统的依赖
- [ ] 确认是否存在循环依赖或阻塞依赖
- [ ] 验证发布系统的独立性

### 3. 功能完整性测试
- [ ] 测试发布工具的基本功能（模拟发布）
- [ ] 验证配置文件的有效性
- [ ] 检查必要的认证和授权机制

### 4. 激活配置
- [ ] 如确认无依赖问题，配置 stage=4 激活
- [ ] 更新相关配置文件
- [ ] 生成激活确认文档

## 测试用例

### Test 1: 发布工具完整性检查
```bash
# 验证所有发布工具 skill 存在
for skill in douyin-publisher kuaishou-publisher toutiao-publisher wechat-publisher weibo-publisher xiaohongshu-publisher zhihu-publisher; do
  [[ -f ~/.claude/skills/$skill/SKILL.md ]] || exit 1
done
```

### Test 2: 脚本文件验证
```bash
# 验证核心发布脚本存在
find ~/.claude/skills/ -name "*publisher*" -type d | while read dir; do
  [[ -f "$dir/SKILL.md" ]] || exit 1
done
```

### Test 3: 依赖关系检查
```bash
# 检查是否存在循环依赖或外部阻塞依赖
grep -r "requires\|depends" ~/.claude/skills/*publisher* || echo "无外部依赖"
```

### Test 4: 配置有效性验证
```bash
# 验证发布工具配置格式正确
for publisher in ~/.claude/skills/*publisher*/SKILL.md; do
  grep -q "触发词" "$publisher" || exit 1
done
```

### Test 5: 激活状态确认
```bash
# 验证 stage=4 配置生效（激活后执行）
# 这个测试在激活配置后执行
node -e "console.log('Multi-platform publisher activated to stage=4')"
```

## 验收标准
1. 所有发布工具完整可用
2. 无阻塞性外部依赖
3. 基础功能测试通过
4. 如满足条件，成功激活到 stage=4
