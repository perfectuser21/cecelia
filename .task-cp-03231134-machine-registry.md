# Task Card: 机器注册表 + Capability Tags 动态路由

Branch: cp-03231134-machine-registry
PRD: .prd-cp-03231134-machine-registry.md

## 目标

用 MACHINE_REGISTRY + TASK_REQUIREMENTS（tags）替换 task-router.js 中的静态 LOCATION_MAP，实现基于机器能力的动态路由，不再锁地点。

## 实现范围

### task-router.js
- 新增 `TASK_REQUIREMENTS` 常量（task_type → tags[]），替换 `LOCATION_MAP`
- 导出 `getTaskRequirements(taskType)` 函数

### executor.js
- 新增 `MACHINE_REGISTRY` 常量（3台机器 + tags）
- 新增 `selectBestMachine(requiredTags)` 函数（tags过滤 + 负载均衡）
- 更新 `triggerCeceliaRun()` 使用 tags 路由替代 location 路由
- 移除 HK MiniMax 路由路径

## 成功标准

1. `getTaskRequirements('dev')` 返回 `['has_git']`，`getTaskRequirements('research')` 返回 `['general']`
2. `triggerCeceliaRun()` 对 `dev` 任务路由到 US M4，对 `research` 任务路由到 xian-m4 或 xian-m1
3. HK MiniMax 路由路径完全移除（不再有 location === 'hk' 分支）
