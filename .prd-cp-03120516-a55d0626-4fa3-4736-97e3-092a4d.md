## PRD - migration: brain_state 表新增 global_quota_cooldown_until 字段

**需求来源**: Brain 自动调度任务
**功能描述**: billing_pause 机制废弃后，全局 quota 熔断无状态持久化。新建 migration，在 brain_state 表增加 global_quota_cooldown_until TIMESTAMPTZ 字段，用于记录全局配额冷却截止时间，防止各账号配额耗尽时独立撞墙批量失败。
**涉及文件**:
- `packages/brain/migrations/` — 新增 migration SQL
- `packages/brain/src/selfcheck.js` — 更新 EXPECTED_SCHEMA_VERSION
- `packages/brain/src/` — 新增 setter/getter 函数（setGlobalQuotaCooldown / isGlobalQuotaCooling）

**成功标准**:
- migration 成功执行，brain_state 表含 global_quota_cooldown_until 字段
- 新增 setter/getter：setGlobalQuotaCooldown(durationMs)、isGlobalQuotaCooling()
- selfcheck.js EXPECTED_SCHEMA_VERSION 同步更新
- facts-check.mjs 通过

**非目标**: 不修改 dispatcher 调用逻辑（只提供数据层支持）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| brain_state 表不存在 | migration 自动创建（IF NOT EXISTS） | P3 | N/A |
| DB 查询失败（isGlobalQuotaCooling） | 返回 false，不阻断调度 | P2 | quota_cooldown_check_failed |
| DB 写入失败（setGlobalQuotaCooldown） | 抛出错误，由调用方处理 | P1 | quota_cooldown_set_failed |
