# Task Card: 修复 heartbeat 垃圾循环（路由错误 + 防重复漏洞）

## 需求（What & Why）
**功能描述**: 修复 dept_heartbeat 任务的两个 bug：路由错误和防重复漏洞
**背景**: dept_heartbeat 不在 executor.js 的 US_ONLY_TYPES 集合中，被错误路由到 Codex Bridge 执行失败。同时 dept-heartbeat.js 防重复检查不包含 'quarantined' 状态，导致隔离后立刻创建新任务。一晚上产生 40+ 条垃圾 heartbeat 任务。

## 成功标准
1. dept_heartbeat 任务在美国本机执行，不再路由到 Codex Bridge
2. quarantined 状态的 heartbeat 任务被视为"已存在"，不会重复创建
3. 现有 quarantined heartbeat 任务被清理为 canceled

## 验收条件（DoD）

- [x] [ARTIFACT] executor.js 的 US_ONLY_TYPES 包含 'dept_heartbeat'
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/executor.js','utf8');if(!c.includes(\"'dept_heartbeat'\"))process.exit(1)"

- [x] [BEHAVIOR] dept-heartbeat.js 防重复 SQL 包含 quarantined 状态
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/dept-heartbeat.js','utf8');if(!c.includes('quarantined'))process.exit(1)"

- [x] [ARTIFACT] 清理 SQL 脚本存在于 scripts/ 目录
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/cleanup-quarantined-heartbeat.sql','utf8');if(!c.includes('canceled'))process.exit(1)"

- [x] [GATE] 所有现有测试通过
  Test: manual:bash -c "npm test 2>&1 | tail -5"

## 实现方案
**要改的文件**:
- `packages/brain/src/executor.js` — US_ONLY_TYPES 加入 'dept_heartbeat'
- `packages/brain/src/dept-heartbeat.js` — 防重复 SQL 加入 'quarantined'
- `scripts/cleanup-quarantined-heartbeat.sql` — 清理脚本

**Scope 锚定**: 仅修改路由集合和 SQL 查询条件，不改动业务逻辑
