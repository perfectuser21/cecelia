# Task Card: 修复 Coding 通道路由——信任 task-router.js 作为 SSOT

## 需求（What & Why）

**功能描述**：`executor.js` 的路由逻辑当前使用 `US_ONLY_TYPES` 白名单决定任务去哪跑。白名单不完整，`initiative_plan`/`scope_plan`/`project_plan` 等 Claude Code 任务（task-router.js 中已标注 location='us'）因为不在白名单里，被错误路由到西安 Codex Bridge，导致 "Not inside a trusted directory" 错误。

**根本原因**：两套路由表不同步——task-router.js 的 LOCATION_MAP 是 SSOT，executor.js 的 US_ONLY_TYPES 是副本且已过时。

**修法**：删除 `US_ONLY_TYPES` 白名单，executor.js 直接调用 `getTaskLocation()` 的返回值决定路由：
- location='us' → US cecelia-bridge（Claude Code）
- location='xian' → 西安 Codex Bridge
- location='hk' → HK MiniMax

**不做什么**：不动 `codex_dev`/`codex_qa`/`codex_test_gen`/`pr_review` 的位置（US 机器暂无 Codex bridge，需单独任务搭建）

## 成功标准

1. [BEHAVIOR] executor.js 删除 US_ONLY_TYPES，改用 getTaskLocation() 路由
2. [BEHAVIOR] initiative_plan 路由到 US（不再派到西安 Codex Bridge）
3. [GATE] 所有现有测试通过
