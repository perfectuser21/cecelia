## 根本原因

vitest.config.js 中 isolate: false + pool: forks 导致同一进程复用模块缓存。
real-DB 测试文件（如 desire-system.test.js）用 vi.resetModules() 加载真实 db.js，
之后 focus.test.js 中的静态 vi.mock('../db.js') 无法覆盖已缓存的真实模块。

### 修复模式

静态 vi.mock（错） -> beforeAll + vi.resetModules() + vi.doMock + dynamic import（对）
