# T1 — zombie-sweep 双通道 safety net（UUID + mtime）

## Goal
修 `packages/brain/src/zombie-sweep.js::sweepStaleWorktrees` 第二层 safety net 失效问题：现有按 `payload->>'branch'` 匹配 in_progress task，但 Brain 多数 task 不塞 branch → 30-50% orphan 漏检。加双通道：保留 branch match + 新增 UUID match（复用 zombie-cleaner `findTaskIdForWorktree`）+ mtime 短路（复用 `isWorktreeActive`）。

## 背景
Phase B2 forensic 发现：zombie-cleaner.js（已修 #2572）+ zombie-sweep.js 是两层 safety net。第一层已修，第二层仍按 payload.branch SQL 匹配，但大部分 task payload 不含 branch 字段 → 漏检。本 PR 让两层都基于 `.dev-mode*` 文件作为单一事实源。

## Tasks
1. 改 `packages/brain/src/zombie-sweep.js:sweepStaleWorktrees`：
   - 保留原 branch match SQL 作 Channel 1
   - 新增从同一 DB query 提取 `inProgressTaskIds` Set
   - 循环里加 Channel 2：`findTaskIdForWorktree(wt.path)` + `inProgressTaskIds.has(taskId)` + `isWorktreeActive(wt.path)` 双重短路
2. 从 `./zombie-cleaner.js` import `findTaskIdForWorktree` 和 `isWorktreeActive`（同级模块，已合入 #2572 导出）
3. 测试：加 2 cases 覆盖 Channel 2（payload.branch 缺失但 .dev-mode UUID 命中 / mtime fresh 短路）
