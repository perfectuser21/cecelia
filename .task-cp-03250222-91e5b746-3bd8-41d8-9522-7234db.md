# Task Card: conversation-digest — Brain 自动读取 Claude Code 对话日志

## 需求（What & Why）
**功能描述**: Brain 自动扫描 ~/.claude-account1/projects/{slug}/ 下的 .jsonl 对话日志，提炼 decisions/ideas/questions/tensions/summary，异步写入 DB。

**背景**: 用户与 Claude Code 的规划对话目前完全没有被捕获。对话日志以 .jsonl 格式写在 ~/.claude-account1/projects/{slug}/ 下，Brain 在同一台机器可以直接读取。不依赖 LLM 记忆（prompt 注入），而是机械地读取 artifact，由 Brain 异步分析。

**不做什么**: 不实时拦截对话；不修改 Claude Code 的写日志行为；不在对话中注入上下文（那是 Memory 系统的工作）。

## 成功标准

1. [ARTIFACT] 新增 DB 表 conversation_log_cursors（tracking 哪些文件/行已处理）
2. [ARTIFACT] 新增 packages/brain/src/conversation-digest.js 模块（scanLogDirectory/readNewMessages/shouldTriggerDigest/analyzeWithLLM/persistDigest）
3. [BEHAVIOR] Brain tick 每轮检测新日志内容，发现满足阈值的 session 后触发分析
4. [ARTIFACT] 分析结果写入 conversation_captures（含 ideas/open_questions/tensions 新列）+ decisions 表
5. [ARTIFACT] selfcheck.js EXPECTED_SCHEMA_VERSION = 196

### 关键设计

2. migration 196 同时扩展 conversation_captures：新增 TEXT[] 列 ideas/open_questions/tensions
3. scanLogDirectory：扫描 ~/.claude-account1/projects/ 下所有 .jsonl 文件

5. shouldTriggerDigest：≥8条 human 消息 OR 最后消息超过30分钟前
6. analyzeWithLLM：用 callLLM('thalamus', prompt) 提炼 5 个维度
7. persistDigest：写 conversation_captures + decisions 表（去重）
8. tick.js：import 后在主 tick 函数中 Promise.resolve().then(() => runConversationDigest()) 非阻塞调用
