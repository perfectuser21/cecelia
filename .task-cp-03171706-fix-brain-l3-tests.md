# Task Card: 修复 Brain L3 测试失败 — startup-sync + blocks

## 需求（What & Why）

CI L3 每次 push main 都失败，原因是 `coverage-baseline` job 的 `npm run test:coverage` 跑了两个问题测试。

**问题1：`startup-sync.test.js`（3 个失败）**
PR #1025（`统一孤儿任务恢复逻辑`）改了孤儿处理逻辑：
- 旧行为：孤儿任务 → 直接 status=failed
- 新行为：`watchdog_retry_count < 2` → requeue；`>= 2` 才 fail

但测试仍按旧行为写，任务 `watchdog_retry_count=0` 走了 requeue 路径，断言 `orphans_fixed=1` 失败（实际是 `requeued=1`）。

**问题2：`blocks.test.js`（17 个失败）**
纯集成测试，直接 `fetch('http://localhost:5221/api/brain/...')`。
`brain-unit` job 已经用 `--exclude='src/__tests__/blocks.test.js'` 排除。
但 `coverage-baseline` 和 `coverage-delta` job 用 `npm run test:coverage` 没有排除，CI 里 Brain 没跑，全部 `fetch failed`。

## 成功标准

1. [BEHAVIOR] CI L3 `coverage-baseline` job 在 push main 后通过
2. [ARTIFACT] `startup-sync.test.js` 3 个失败测试修复，反映新 requeue 行为
3. [ARTIFACT] `blocks.test.js` 加 skip guard，CI 单元测试中跳过

## 实现方案

**修改文件**：
1. `packages/brain/src/__tests__/startup-sync.test.js`
   - Test 1 `'should mark orphan task as failed when no matching process'`：payload 加 `watchdog_retry_count: 2`
   - Test 2 `'should handle tasks without run_id as orphans'`：payload 加 `watchdog_retry_count: 2`
   - Test 3 `'should handle mixed alive and orphan tasks'`：orphan-2 的 payload 加 `watchdog_retry_count: 2`
2. `packages/brain/src/__tests__/blocks.test.js`
   - 顶部加 skip guard：`const canRunBlocks = !!process.env.BRAIN_INTEGRATION; const describeBlocks = canRunBlocks ? describe : describe.skip;`
   - 将 `describe('Blocks API', ...)` 改为 `describeBlocks('Blocks API', ...)`
