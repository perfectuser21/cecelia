# PRD: cecelia-run webhook 回调重试机制

## 背景
cecelia-run.sh 的 send_webhook 函数用 `|| true` 吞掉所有 curl 错误，导致 webhook 回调失败时 Brain 收不到状态更新，task 永久卡在 in_progress。

## 目标
1. send_webhook 失败时自动重试，最多 3 次，指数退避（2s/4s/8s）
2. 全部失败后写入本地失败队列文件供后续恢复
3. cleanup trap 中也调用 send_webhook，确保异常退出也有回调

## 成功标准
- send_webhook 回调失败时自动重试 3 次
- 重试全部失败后 payload 持久化到 /tmp/cecelia-callback-queue/
- 异常退出时 cleanup 函数发送回调
