## Task 1: DB Migration 234 — topics 表
**文件**: `packages/brain/migrations/234_topics_pool.sql`
**内容**:
```sql
CREATE TABLE topics (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  angle TEXT,
  priority INTEGER DEFAULT 50,
  status VARCHAR(20) NOT NULL DEFAULT 'draft'
    CHECK (status IN ('draft', '已通过', '已发布', '已废弃')),
  target_platforms JSONB DEFAULT '[]',
  scheduled_date DATE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_topics_status ON topics(status);
CREATE INDEX IF NOT EXISTS idx_topics_priority ON topics(priority DESC);
INSERT INTO schema_version (version, description, applied_at)
VALUES ('234', 'topics 表 - 主理人选题池', NOW())
ON CONFLICT (version) DO NOTHING;
```
**测试命令**: `node -e "require('fs').accessSync('packages/brain/migrations/234_topics_pool.sql')"`
**预期**: 文件存在，无报错

## Task 2: Brain topics routes
**文件**: `packages/brain/src/routes/topics.js`
**接口**:
- GET /api/brain/topics?status=已通过&limit=20&offset=0 → {topics: [], total: N}
- POST /api/brain/topics body: {title, angle?, priority?, status?, target_platforms?, scheduled_date?} → {ok:true, topic:{...}}
- PATCH /api/brain/topics/:id body: 任意字段 → {ok:true, topic:{...}}
- DELETE /api/brain/topics/:id → {ok:true}
- GET /api/brain/topics/rhythm → {daily_limit: 1}
- PATCH /api/brain/topics/rhythm body: {daily_limit: N} → {ok:true, daily_limit: N}

rhythm 存储在 Brain DB 的 key_value 表（若有），否则用 topics_rhythm_config 表
**测试命令**: `curl -s http://localhost:5221/api/brain/topics | node -e "const d=require('fs').readFileSync('/dev/stdin','utf8');const r=JSON.parse(d);if(!Array.isArray(r.topics))process.exit(1)"`
