# PRD: 记忆系统 PR9 — 改进 computeSalience 多维度评分

## 背景

当前 computeSalience 只有 4 行正则（0.3/0.6/0.8/0.9）：
- 所有普通对话返回 0.3
- rumination 过滤 salience_score >= 0.7，只有纠正（0.9）和决定（0.8）类消息进入反刍

问题：大量有价值对话（洞察、计划、情绪表达、长消息）被设定为 0.3，完全被 rumination 忽略。

## 改进方案

多维度评分，每个维度独立贡献，最终取最高值（非叠加，避免误报膨胀）：

| 维度 | 权重 | 触发条件 |
|------|------|----------|
| 纠正类 | 0.90 | 不对/错了/纠正/改一下等 |
| 决定类 | 0.85 | 决定/确认/就这样/采纳等 |
| 洞察类 | 0.80 | 发现/理解/明白/原来/关键等 |
| 情绪强烈 | 0.75 | 感谢/担心/焦虑/开心/很好等 |
| 计划类 | 0.70 | 下一步/计划/安排/接下来等 |
| 疑问类 | 0.55 | 问号/吗/为什么/怎么/什么 |
| 长消息 | 0.55 | 消息长度 >= 50 字 |
| 基础分 | 0.30 | 其余 |

向后兼容：最低仍为 0.3，最高为 0.9，不破坏现有测试。

## 成功标准

- computeSalience 洞察类返回 >= 0.75
- computeSalience 计划类返回 >= 0.65
- computeSalience 长消息（>=50字）返回 >= 0.50
- 纠正类仍返回 0.9，决定类仍返回 0.85
- 普通短消息仍返回 0.3
