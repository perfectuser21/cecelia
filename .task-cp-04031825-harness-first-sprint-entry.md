# Task Card: Harness v2.0 第一个 dev task 未路由到 sprint 流程

## 需求（What & Why）
**功能描述**: 当 /architect M2 注册 dev task（harness_mode: true）并被 dispatcher 派发时，executor 的 preparePrompt 应将其路由到 sprint_generate 流程，而非普通 dev task 流程。同时 execution callback 中 dev task（harness_mode）完成后应触发创建 sprint_evaluate。
**背景**: Harness v2.0 中，第一个 dev task 带有 harness_mode: true，但 executor.js 的条件判断只匹配 sprint_generate/sprint_fix，导致完全忽略 harness_mode 标记。execution.js 同理。
**不做什么**: 不修改 harness 的其他流程（evaluate/fix 循环）。

## 成功标准
> [BEHAVIOR] dev task（harness_mode: true）被路由到 sprint harness 流程
