# Task Card: content-pipeline 读取 YAML 配置（PR3）

## 需求（What & Why）
**功能描述**: 让 content-pipeline-orchestrator.js 在创建子任务时，通过 getContentType() 读取 YAML 配置中的 generate_prompt、images.count、review_rules，而不是硬编码。
**背景**: PR1 已建立 content-type-registry.js + solo-company-case.yaml。现在让 Pipeline 实际使用这些配置，实现内容类型配置与 Pipeline 代码的解耦。
**不做什么**: 不修改 content-research/content-export 的逻辑，不改数据库 schema，不做 Dashboard UI。

## 成功标准
> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [ARTIFACT] content-pipeline-orchestrator.js import getContentType，在 orchestrateContentPipelines 中读取 YAML 配置
2. [BEHAVIOR] content-generate 子任务的 description 包含 YAML template.generate_prompt 内容（而非硬编码）
3. [BEHAVIOR] content-generate 子任务的 payload 包含 images_count（来自 YAML images.count）
4. [BEHAVIOR] content-review 子任务的 payload 包含 review_rules（来自 YAML）
5. [BEHAVIOR] content_type 不在注册表时，pipeline 标记为 failed 并打印错误（而非静默继续）
6. [GATE] CI 全部通过
