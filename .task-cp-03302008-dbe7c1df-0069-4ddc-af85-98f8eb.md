# Task Card: 接通 content pipeline executor 的 Claude 调用

## 需求（What & Why）
**功能描述**: content-pipeline-executors.js 的四个阶段（copywriting/copy-review/generate/image-review）改为调用 callLLM，使用 typeConfig 中配置的 prompt，让内容真正由 AI 生成，而不是本地模板拼接。

**背景**: 当前 executor 中 copywriting 用静态模板生成文案，copy-review 只做字符串长度检查，generate 不生成任何内容，image-review 只检查文件是否存在。配置页面设置的 prompt 从未被用到。

**价值**: 让内容工厂产出真实 AI 生成内容；prompt 可通过配置页面热更新，无需改代码。

---

## 验收清单

- [x] executeCopywriting 调用 callLLM 生成真实文案，使用 typeConfig.template.generate_prompt
- [x] 第二次 rerun 时将 previous_feedback 注入 prompt
- [x] executeCopyReview 调用 callLLM 审查文案，返回 rule_scores 逐条评分
- [x] executeGenerate 调用 callLLM 生成卡片内容描述，写入 cards/llm-card-content.json
- [x] executeImageReview 调用 callLLM 审核内容质量，返回具体意见
- [x] LLM 调用失败时降级到原有本地逻辑，不中断 pipeline
- [x] CI 通过，测试覆盖新的 callLLM mock 路径
