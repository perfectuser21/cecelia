# Task Card: 接通 content pipeline executor 的 Claude 调用

## 需求（What & Why）
**功能描述**: content-pipeline-executors.js 的四个阶段（copywriting/copy-review/generate/image-review）改为调用 callLLM，使用 typeConfig 中配置的 prompt，让内容真正由 AI 生成，而不是本地模板拼接。

**背景**: 当前 executor 中 copywriting 用静态模板生成文案，copy-review 只做字符串长度检查，generate 不生成任何内容，image-review 只检查文件是否存在。配置页面设置的 prompt 从未被用到。

**价值**: 让内容工厂产出真实 AI 生成内容；prompt 可通过配置页面热更新，无需改代码。

## 成功标准

- callLLM 已接入 executor：executeCopywriting/executeCopyReview/executeGenerate/executeImageReview 四个阶段均通过 callLLM 调用 Claude
- executeCopywriting 调用 callLLM 生成真实文案，生成结果写入 cards/copy.md 和 article/article.md，不是静态模板拼接
- executeCopyReview 返回 rule_scores 逐条评分（每条规则的评分），而非仅 issues 数组；找不到目录时返回 review_passed=false
- executeCopywriting 第二次 rerun 时，若携带 previous_feedback，prompt 中包含该 feedback 内容
- executeGenerate 调用 callLLM 生成卡片内容描述，写入 llm-card-content.json；返回 image_count 和 image_style 字段
- executeImageReview 调用 callLLM 审核内容质量，返回具体意见和质量分
- callLLM 失败时各 executor 降级到原有静态逻辑，不中断 pipeline；executeCopywriting 降级路径不报错（success: true）
- prompt 修改后，下次执行立即生效（无缓存）
