# PRD: 清理 prd/task 残留文件 + 自动代谢机制

## 背景
main 分支上积累了 641 个 `.prd-*.md` + `.task-*.md` 文件，每天还在涨。CI 只在 PR review 期间需要这些文件，合并进 main 后毫无价值。

## 功能描述
1. 删除 main 上所有 `.prd-*.md` + `.task-*.md` 文件
2. 新增 GitHub Actions workflow 在每次 push 到 main 后自动清理这类文件
3. Stage 4 Ship cleanup 中同时清理 `.dev-seal.<branch>` 和 `.dev-mode.<branch>` 残留

## 成功标准
- main 上的 `.prd-*.md` / `.task-*.md` 文件全部删除
- 新增 `cleanup-merged-artifacts.yml`，每次 push 到 main 后自动清理
- Stage 4 cleanup 包含 dev-seal 和 dev-mode 清理指令

## DoD
- [x] main 上不再有 `.prd-*.md` / `.task-*.md` 文件
- [x] `.github/workflows/cleanup-merged-artifacts.yml` 存在且有效
- [x] cleanup workflow 触发条件为 push 到 main
- [x] workflow 含自动 commit 逻辑
- [x] 04-ship.md 的 cleanup 步骤包含 dev-seal 和 dev-mode 清理指令
