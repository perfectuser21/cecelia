# PRD - fix(routes): execution-callback 从 req.body.stderr 提取错误并写入 tasks.error_message

## 背景
execution-callback 接收 cecelia-run 的 AI Failed 回调时，result=null，错误详情在 req.body.stderr 中，但当前代码从不读取 stderr 字段，也不写入 tasks.error_message 列。导致 auto-learning 每次读取 error_message 拿到 NULL，返回 No details available，反刍系统空转。

## 目标
修复后，AI Failed 回调到达时，tasks.error_message 必须被写入真实错误内容。

## 验收标准
- [ ] execution-callback 在 status==AI Failed 时，从 req.body.stderr 提取并写入 tasks.error_message
- [ ] errorMsg 提取优先级：result.error > result.stderr > body.stderr > body.failure_class > status
- [ ] SQL: UPDATE tasks SET error_message=$1 WHERE id=$2 在事务中执行
- [ ] 不影响 success 回调路径

## 技术要点
- 文件：packages/brain/src/routes.js 的 /execution-callback handler（约 2717 行）
- 现有 errorMsg 变量仅用于 classifyFailure，需额外写 UPDATE tasks SET error_message
- 注意：exec 在事务 client 中，确保 error_message 与 status 原子更新

## 成功标准
- [ ] 任务完成
