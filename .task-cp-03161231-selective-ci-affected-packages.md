# Task Card: 基于依赖图的选择性 CI

## 需求（What & Why）

**功能描述**：创建 `packages/engine/scripts/affected-packages.js`，基于 git diff 和包依赖图计算受影响的包集合，并在 CI workflows 中使用该脚本统一管理选择性执行逻辑。

**背景**：
- 现有 L2/L3/L4 CI 已有 `changes` job，但各 workflow 独立实现了差异检测逻辑（重复代码），且 `quality-meta-tests` 等 L1 job 全量跑
- 各包之间没有 `workspace:*` 依赖，但存在逻辑上的影响关系（如 `packages/engine/` 改动影响 engine 测试）
- 需要一个统一脚本来标准化"哪些文件改动影响哪些包"的逻辑

**不做什么**：
- 不改变现有 CI 的检测结果（只是统一实现，输出应与现有逻辑一致）
- 不引入 workspace 依赖关系（各包 package.json 不变）
- 不修改 L1 always-on 的流程检查类 job（branch check, PRD check 等必须全量跑）

## 成功标准

1. [ARTIFACT] `packages/engine/scripts/affected-packages.js` 存在且可执行
2. [ARTIFACT] 脚本接受改动文件列表参数，输出 JSON 数组（如 `["brain","engine"]`）
3. [BEHAVIOR] 改动 `packages/brain/src/tick.js` → 输出包含 `"brain"`
4. [BEHAVIOR] 改动 `packages/engine/scripts/foo.js` → 输出包含 `"engine"`
5. [BEHAVIOR] 改动根目录文件（如 `package.json`）→ 输出所有包（兜底）
6. [GATE] CI 全部通过

## 实现方案

**要改的文件**：
- `packages/engine/scripts/affected-packages.js`（新建）
- `packages/engine/feature-registry.yml`（更新 changelog）
- Engine 版本 bump 相关 7 个文件

**包映射规则**：
```
packages/brain/     → brain
packages/engine/    → engine
packages/quality/   → quality
packages/workflows/ → workflows
apps/api/           → api
apps/dashboard/     → dashboard
.github/workflows/  → (所有包，因为影响所有 CI)
根目录文件          → (所有包，兜底)
```
