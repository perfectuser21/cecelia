# PRD: suggestion-dispatcher 绕过丘脑直接建任务修复

## 问题

`packages/brain/src/suggestion-dispatcher.js:88` 直接执行 `INSERT INTO tasks`，
绕过丘脑（Thalamus）创建任务，违反「任务创建权统一收归丘脑」架构规则。

这是 Rumination→Desire 自循环的最后一个断口：反刍写入 suggestions 表 →
suggestion-dispatcher 直接 INSERT tasks → 无丘脑去重/速率限制 → 可能重复创建。

## 修复方案

1. **thalamus.js**：新增 `SUGGESTION_READY` 事件类型和 L0 快速路由处理器
2. **suggestion-dispatcher.js**：移除直接 INSERT，改为发送 `SUGGESTION_READY` 事件给丘脑

调用链：suggestion-dispatcher → thalamus.processEvent(SUGGESTION_READY) → createTask（actions.js，含 dedup）

suggestion.status 更新（'in_progress'）仍由 dispatcher 负责，不是任务创建权的一部分。

## 成功标准

1. [ARTIFACT] suggestion-dispatcher.js 不包含直接 `INSERT INTO tasks`
2. [BEHAVIOR] suggestion-dispatcher 通过 thalamus.processEvent 创建任务，trigger_source='suggestion_dispatcher'
3. [GATE] Brain CI L1-L4 通过
