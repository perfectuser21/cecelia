## PRD（需求）

### 背景
Cecelia 系统需要一个能让 Codex 自动扫描覆盖率低的模块并生成测试的机制。
当前 Codex 已有 `codex_qa`（免疫检查）和 `codex_dev`（全流程开发）两种任务类型，
但缺乏专门的测试生成能力。

### 功能描述
新增 `codex_test_gen` 任务类型，让 Brain 能够调度 Codex 自动：
1. 扫描指定模块或整个仓库的测试覆盖率
2. 识别覆盖率低（< 阈值）的模块
3. 为目标模块生成对应的单元测试

SKILL.md 定义 Codex 执行时的行为规范。

### 技术范围
- 新增 `packages/engine/skills/codex-test-gen/SKILL.md`
- `packages/brain/src/task-router.js`：VALID_TASK_TYPES + SKILL_WHITELIST + LOCATION_MAP
- `packages/brain/src/role-registry.js`：CTO task_types 添加 `codex_test_gen`
- `packages/brain/src/executor.js`：getModelForTask 添加 null 返回（Codex CLI 自选模型）
- `packages/brain/src/model-registry.js`：添加 codex_test_gen agent 条目
- `packages/engine/features/feature-registry.yml`：changelog 条目
- Engine version bump（5 文件）
