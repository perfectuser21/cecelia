# PRD: DoD Test 字段升级 — 从模糊描述升级为可执行命令

## 背景

DoD `[BEHAVIOR]` 条目的 Test 字段格式规范存在执行盲区：
- `check-dod-mapping.cjs` 的 `detectFakeTest` 缺少 ls/cat/true/exit 0 弱命令检测
- `manual:ls src/` 类命令落入 evidence-ID 路径，错误信息不明确
- CI 反模式（curl localhost）未被拦截，会导致 CI 必然失败
- `DOD-TEMPLATE.md` 仍使用旧格式 `manual:code-review` 证据ID

## 功能描述

升级 DoD Test 字段的格式规范和检查脚本，确保每条 Test 字段是可执行的命令，返回明确 pass/fail（exit 0/1）。

## 成功标准

- `check-dod-mapping.cjs` 能拦截弱测试命令（纯列目录、恒真命令）
- `check-dod-mapping.cjs` 能拦截 CI 反模式（curl localhost/127.0.0.1）
- `DOD-TEMPLATE.md` 所有旧格式证据ID示例替换为可执行命令
- 新增单元测试覆盖新增检测逻辑
