# PRD - fix(cortex): 实现 CECELIA_CORTEX_TIMEOUT_MS 专属超时 — 恢复皮层战略能力

**需求来源**: Brain 自动调度任务（agent_ops 领域）
**功能描述**: Cortex（L2 皮层）在 LLM 调用中超时 121.9s 后被 kill（exit 143），战略思考能力离线。根本原因：`callCortexLLM()` 使用 `CECELIA_BRIDGE_TIMEOUT_MS || 120000`（120s），而 Opus 在高并发下 p99 响应时间高达 147s，超过阈值。
**涉及文件**: `packages/brain/src/cortex.js`
**非目标**: 不修改 Bridge/Thalamus 的超时配置

### 异常处理 & 报警规范

N/A - 纯超时参数调整，无新增 IO 操作

## 成功标准
- [ ] `callCortexLLM()` 使用独立的 `CECELIA_CORTEX_TIMEOUT_MS`（默认 300000ms）
- [ ] 向下兼容 `CECELIA_BRIDGE_TIMEOUT_MS`（fallback）
- [ ] 单元测试覆盖超时参数注入
- [ ] Opus 147s 慢响应不再触发 cortex 超时
