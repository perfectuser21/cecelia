# PRD: Pipeline Rerun 幂等检查修复

## 问题描述

`content-pipeline-orchestrator.js` 中 `_startOnePipeline` 函数的幂等检查包含了 `completed` 状态：

```sql
AND status IN ('queued', 'in_progress', 'completed')
```

当 pipeline 重跑时，系统找到旧的已完成 content-research 子任务，误判为"已在飞"而跳过创建新子任务，导致 pipeline 永远卡在 `in_progress`。

## 修复方案

将幂等检查的 `status IN` 子句改为只包含活跃状态：

```sql
AND status IN ('queued', 'in_progress')
```

## 成功标准

- 修改后，已完成的 content-research 子任务不会阻止 pipeline 重新启动新的子任务
- 已有活跃（queued/in_progress）子任务的 pipeline 仍正确跳过，避免重复创建
