# Task Card: Generator 独立 subagent — Stage 2 写代码从主 agent 拆出

## 需求（What & Why）

**功能描述**: 把 /dev Stage 2 的代码编写阶段从主 agent 内部拆成独立 Generator subagent。主 agent 在探索代码后，构建一个"干净 prompt"（只含 Sprint Contract + CLAUDE.md 编码规范 + 代码库上下文文件），spawn Generator subagent 完成实际写代码。主 agent 变成纯编排者。

**背景**: 当前 Stage 2 主 agent 带着大量 brain context（OKR、任务历史、调度信息）直接写代码，导致 context 污染。Generator subagent 是一个"隔离写代码环境"，只看 spec 和代码，不知道外部调度逻辑，代码质量更纯粹。

**不做什么**:
- 不改变 Stop Hook 机制（.dev-mode 写入权限仍归主 agent）
- 不改变 code_review_gate 环节（仍由主 agent 触发）
- 不改变 verify-step.sh 的调用时机（Stage 2 完成时）
- 不改变 CI 流程

## 成功标准

- Stage 2 由主 agent spawn Generator subagent 完成代码编写
- Generator subagent 的 prompt 只含 spec + CLAUDE.md + 代码库上下文，不含 Brain context / OKR / 任务历史
- Stop Hook 仍然正常工作（主 agent 控制 .dev-mode，Generator 禁止写入）
