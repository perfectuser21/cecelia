# Task Card: 重设计 /dev 对抗网络（Planner/Generator/Evaluator 三角独立）

## 需求（What & Why）
**功能描述**: 修复 /dev Sprint Contract Gate 的三个根本性缺陷，使 Planner/Generator/Evaluator 真正独立，对标 Anthropic 官方对抗网络设计
**背景**: 当前 Planner 写含 Test 字段的 DoD，Evaluator 收到含答案的 Task Card 后"假装不看"独立提案，Sprint Contract 本质上是主 agent 左手打右手，形同虚设
**不做什么**: 不改变 Stage 2 Generator 写代码流程；不引入新的 skill 文件；不改变 CI 流程

## 成功标准

1. [ARTIFACT] planner-prompt.md 模板中 DoD 条目只有 `Test: TODO`，无任何非 TODO 的具体 Test 命令（[GATE] 条目除外由 Orchestrator 添加）
2. [BEHAVIOR] 01-spec.md Sprint Contract Gate 流程：Orchestrator 将 Task Card Test 字段剥离后分别传给 Generator subagent 和 Evaluator subagent，两者互不知道对方提案
3. [BEHAVIOR] 比对发现分歧时，Orchestrator 将对方提案展示给各自，N 轮迭代直到收敛或超限
4. [GATE] CI 全部通过
