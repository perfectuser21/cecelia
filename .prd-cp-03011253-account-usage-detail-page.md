# PRD: Account Usage 详情页

## 背景
LiveMonitorPage 中的 AccUsageRings 组件显示三个账号的用量圆环，当前三行文字太密集。
用户需要：外层只看关键状态，点击跳转到独立详情页查看完整数据。

## 目标
1. **AccUsageRings（外层，紧凑）**：只显示 5h 用量 + 重置倒计时，点击跳转详情页
2. **AccountUsagePage（详情页）**：新路由，显示三账号 × 三指标完整数据，带颜色区分

## 需求

### 外层（AccUsageRings 改造）
- 每个账号只显示一个圆环（ring），ring 颜色反映 five_hour_pct 状态
- ring 下只显示一行：`39% ↺14:59`（5h 用量 + 5h 重置倒计时）
- 整个区域可点击，`cursor: pointer`，点击跳转 `/account-usage`
- 可加一个小提示（如右上角小图标或底部文字 "详情 →"）

### 详情页（AccountUsagePage 新建）
路由：`/account-usage`

**布局**：
- 页面标题：「账号用量」+ 最后刷新时间
- 三列（account1 / account2 / account3），每列显示：
  - 账号名
  - 🔵 5h: 39% ↺ 07:00（蓝色 `#58a6ff`）
  - 🟡 7d: 50% ↺ 3/6（金黄 `#d29922`）
  - 🟣 son: 72% ↺ 3/6（紫色 `#bc8cff`）
  - 用量 ≥ 80% 时该行显示红色警告
- 底部「强制刷新」按钮（调用 POST /api/brain/account-usage/refresh）
- 深色主题，与 LiveMonitorPage 风格一致

**颜色规范**：
- 5h 指标：`#58a6ff`（蓝）
- 7d 指标：`#d29922`（金黄）
- 7d-sonnet 指标：`#bc8cff`（紫）
- 用量 ≥ 80%：`#f85149`（红色警告，替换对应行颜色）

## 技术约束
- 只改 dashboard（`apps/dashboard/src/`）
- 不改后端 API
- 路由用现有 react-router-dom 方案
- 状态管理：本地 useState（无需 global store）
