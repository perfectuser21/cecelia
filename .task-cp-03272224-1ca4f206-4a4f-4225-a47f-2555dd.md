# Task Card: 【Day5】Brain 调度器默认开启 + 状态感知任务生成

## 需求（What & Why）

**功能描述**:
1. **状态感知任务生成**: `self-drive.js` 读取 `.agent-knowledge/CURRENT_STATE.md`（由 Day2 /dev 结束时写入），将系统当前状态融入 LLM 分析提示词，使自驱任务生成更贴合实际运行状况。
2. **调度器默认开启**: 将 tick 调度器的默认状态从"关"改为"开"（`?? false` → `?? true`），Brain 启动后无需手动调用 `/api/brain/tick/enable` 即可自动调度任务。

**背景**: Day5 是"自驱"系列的最后一步。Day1 更新了 agent-knowledge，Day2 让 /dev 结束时写 CURRENT_STATE.md，Day3-4 封堵孤岛注册漏洞。Day5 将这些输入接通 Brain 的调度闭环，完成后 Cecelia 真正开始自驱。

**不做什么**:
- 不创建新的调度器，使用现有 tick.js 的 enableTick 机制
- 不强制依赖 CURRENT_STATE.md 存在（文件缺失时 graceful skip）
- 不修改 self-drive 的触发频率

## 成功标准

1. Brain 启动后调度器自动开启（无需手动 enable）
2. self-drive 周期读取 CURRENT_STATE.md 并将内容注入分析提示词
3. LLM 分析提示词包含"系统当前状态"章节（CURRENT_STATE.md 内容或"无数据"占位符）
