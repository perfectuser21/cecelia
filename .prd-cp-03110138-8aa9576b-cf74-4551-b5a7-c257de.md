# PRD - auto-learning: effectiveResult=null 时从 DB error_message 回填

## 背景

auto-learning.js 的 processExecutionAutoLearning() 已经查询了 tasks.error_message 字段（SELECT task_type, title, error_message FROM tasks WHERE id = $1），但没有将 taskRow.error_message 传递给 handleTaskFailedLearning()。

当 effectiveResult=null（全空 callback 场景）时，extractTaskSummary(null) 直接返回 "No details available"，忽略了 DB 层已存好的有意义的 error_message。

## 目标

修改 processExecutionAutoLearning 和 handleTaskFailedLearning，在 effectiveResult=null 时使用 taskRow.error_message 作为 fallback，让 auto-learning 的失败摘要有实际内容。

## 成功标准

- [ ] processExecutionAutoLearning 将 taskRow.error_message 传给 handleTaskFailedLearning 作为第五个参数 dbErrorMessage
- [ ] handleTaskFailedLearning 接收 dbErrorMessage 参数，在 extractTaskSummary(result) 返回 "No details available" 时用 dbErrorMessage 替代
- [ ] 新增测试：mock DB 返回带 error_message 的行，验证 learning.content 包含 error_message 内容
- [ ] 全空 callback 场景的 learning record 不再出现 "No details available"
