# PRD: P0/P1 Brain 架构问题修复

## 背景

Brain tick 每次执行时报 `column g.priority does not exist` 错误，导致 `compareGoalProgress()`
完全失效。与此同时，Rumination 产出的洞察会不断重复创建相同任务（没有对 `failed` 状态的去重），
auto-fix 模块也缺乏重试次数上限，可能陷入无限循环。

## 问题清单

### P0: decision.js ORDER BY g.priority 报错

**文件**: `packages/brain/src/decision.js:133`

**现象**: `compareGoalProgress()` 中的 UNION 子查询仅 SELECT `id, title, status, progress, created_at`，
但 ORDER BY 使用 `g.priority`，该列不存在于子查询 → 每次 tick 报错。

**根因**: OKR 迁移后改了 UNION 查询的 SELECT 列表，但忘记加 `priority` 列。

**修复**: 在 UNION 内两个 SELECT 中都加上 `priority`，并在外层 SELECT 加 `g.priority`。

### P1: Rumination→Desire 自循环（重复创建任务）

**文件**: `packages/brain/src/actions.js` (createTask dedup 逻辑)

**现象**: Rumination 洞察中的 `[ACTION:]` 标记触发丘脑创建 research 任务。
`createTask` 的去重仅覆盖 `queued/in_progress` + 24h 内 `completed`，
不覆盖 `failed` 任务 → 任务失败后下次反刍又创建相同任务，无限循环。

**修复**: 扩展 createTask 去重逻辑，对 `trigger_source IN ('rumination', 'cortex', 'auto_fix')`
的任务，同时检查 72h 内 `failed` 的同标题任务。

### P1: RCA→auto-fix 死循环（无重试上限）

**文件**: `packages/brain/src/auto-fix.js` (dispatchToDevSkill)

**现象**: 任务失败 → Cortex RCA → auto-fix 创建修复任务 → 修复任务失败 →
下次 RCA 再创建 → 无限循环，`failed` 状态不被去重。

**修复**: `dispatchToDevSkill` 调用前检查同 signature 的 auto-fix 任务失败次数，
超过 `MAX_AUTO_FIX_ATTEMPTS = 3` 时拒绝创建并记录警告。

## 成功标准

- Brain tick 不再出现 `column g.priority does not exist` 错误
- Rumination 相同洞察在 72h 内不重复创建 failed 过的 research 任务
- auto-fix 对同一错误签名最多重试 3 次，超限后停止并记录日志
