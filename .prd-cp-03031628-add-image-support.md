# PRD: Cecelia 图片支持（飞书 + Dashboard）

## 背景

Cecelia 目前只能处理文字和语音消息，无法接收和理解图片内容。用户在飞书群/私信发图片，或在 Dashboard 聊天发图片，Cecelia 完全看不到。本功能让 Cecelia 具备视觉理解能力，能「看」到图片后再回复。

## 目标

让用户可以在飞书（私信/群聊）和 Dashboard 聊天界面给 Cecelia 发图片，Cecelia 能看到图片内容并作出回应。

## 功能需求

### 1. llm-caller.js 多模态支持

- `callLLM(agentId, prompt, options)` 中，`prompt` 支持传入 content array
- content array 格式：`[{ type: 'text', text: '...' }, { type: 'image', source: { type: 'base64', media_type: 'image/jpeg', data: '...' } }]`
- 当 prompt 是 string 时，保持向后兼容（转为 `[{ type: 'text', text: prompt }]`）
- 透传到 Anthropic API 的 `messages[0].content`

### 2. 飞书图片接收（routes.js）

- 检测 `msgType === 'image'`（不再只处理 text/audio）
- 从消息 content 中取 `image_key`
- 调用飞书 API 下载图片：`GET https://open.feishu.cn/open-apis/im/v1/messages/{message_id}/resources/{image_key}?type=image`
- 转成 base64，构建多模态 content array
- 文字部分：`"[图片消息]"` + 如有 caption 则附上
- 同样走 handleChat 或 Mode A 决策链路

### 3. Dashboard 图片上传（前端）

- 聊天输入框旁加「📎」上传按钮（`<input type="file" accept="image/*">`）
- 选图后显示缩略图预览（可点击移除）
- 发送时把图片转成 base64 随消息一起发送
- API 请求 body 新增 `image_base64` 和 `image_media_type` 字段

### 4. orchestrator-chat.js 图片支持

- 接收请求 body 中的 `image_base64` 和 `image_media_type`
- 若存在，将 prompt 转为多模态 content array（text + image）
- 调用 `callLLM` 时传入 content array

## 成功标准

- 飞书私信发图片 → Cecelia 能描述图片内容后回复
- 飞书群 @Cecelia 发图片 → Cecelia 能看图回复
- Dashboard 聊天上传图片 → Cecelia 能看图回复
- 纯文字消息（不带图片）行为不变

## 影响文件

- `packages/brain/src/llm-caller.js`（多模态支持）
- `packages/brain/src/routes.js`（飞书图片消息处理）
- `packages/brain/src/orchestrator-chat.js`（Dashboard 图片接收）
- `apps/dashboard/src/`（图片上传 UI）
- `apps/api/`（API 透传 image 参数）
