## PRD - RNA KR 进度自动回写闭环

**需求来源**: desire_system 欲望驱动任务 - "为 Cecelia 添加任务执行结果自动回写 KR 进度的闭环"

**功能描述**:
实现 RNA (Recursive Next Action) 学习闭环的核心机制：每次任务执行完成后，自动将结果回写到对应 KR 的进度中，形成"行动 → 学习 → 进度更新"的闭环。

当前问题：RNA KR 停在 0% 进度，因为虽然有 34 次分析和洞察，但缺少实际行动和进度反馈机制。

**涉及文件**:
- `packages/brain/src/task-completion-handler.js` (新建) - 任务完成后处理逻辑
- `packages/brain/src/kr-progress-sync.js` (新建) - KR 进度计算和更新逻辑
- `packages/brain/src/server.js` - 添加回调端点
- `packages/brain/src/executor.js` - 集成任务完成回调
- `packages/brain/tests/kr-progress-sync.test.js` (新建) - 单元测试

**成功标准**:
1. Task 完成时自动触发 KR 进度计算
2. KR 进度基于关联 Tasks 的完成率计算（completed tasks / total tasks）
3. 进度更新写入 PostgreSQL `goals` 表的 `progress` 字段
4. Brain API 返回更新后的 KR 进度
5. 单元测试覆盖核心逻辑
6. CI 通过所有检查

**非目标**:
- 不改变现有任务派发逻辑
- 不涉及前端 UI 更新（仅后端逻辑）
- 不处理 KR 拆分或创建（只更新现有 KR 进度）

**RNA 闭环验证**:
完成后，RNA KR 将从 0% → 基于实际完成任务数的真实进度，证明"行动产生学习"的闭环有效运作。
