# Task Card: CI M1 — L1 加 gitleaks secrets 扫描

## 需求

在 `ci-l1-process.yml` 新增 `secrets-scan` job，使用 `gitleaks/gitleaks-action@v2` 自动检测每次 PR 中是否包含硬编码 secrets（API key、token、password 等）。

## 技术方案

1. 在 `l1-passed` job 之前插入新 job `secrets-scan`
2. `secrets-scan` 使用 `gitleaks/gitleaks-action@v2`，需要 `GITHUB_TOKEN` env
3. `l1-passed` 的 `needs` 数组加入 `secrets-scan`
4. `l1-passed` gate 脚本检查 `secrets-scan` 结果

## DoD 验收清单

- [ ] D1: [ARTIFACT] ci-l1-process.yml 包含 `gitleaks/gitleaks-action@v2`
  - `manual:bash -c "grep -c 'gitleaks/gitleaks-action' .github/workflows/ci-l1-process.yml"`
- [ ] D2: [ARTIFACT] l1-passed job 的 needs 包含 `secrets-scan`
  - `manual:bash -c "grep -c 'secrets-scan' .github/workflows/ci-l1-process.yml"`
- [ ] D3: [BEHAVIOR] ci-l1-process.yml YAML 语法正确可解析
  - `manual:bash -c "python3 -c \"import yaml; yaml.safe_load(open('.github/workflows/ci-l1-process.yml'))\" && echo OK"`
- [ ] D4: [GATE] CI 全部通过
  - `gate:ci-passed`

## 成功标准

1. [ARTIFACT] `ci-l1-process.yml` 包含 `gitleaks/gitleaks-action@v2` job 定义
2. [ARTIFACT] `l1-passed` job 的 needs 数组包含 `secrets-scan`
3. [GATE] CI 全部通过（本 PR 使用 `[CONFIG]` 标签豁免测试覆盖率检查）
