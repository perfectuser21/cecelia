## PRD - Dashboard 白屏修复 + 自动部署

**需求来源**: 用户反馈 http://perfect21:5211 显示白屏，且每次代码合并后需要手动重建 dist
**功能描述**:
1. 更新 APP_VERSION 强制清除用户浏览器的 Service Worker 旧缓存，修复白屏
2. 新增 InstructionBook 页面到正确目录（apps/api/features/knowledge/），修复 PR #779 的目录错误
3. 新增自动重建脚本：git pull 后自动重建 apps/dashboard dist

**涉及文件**:
- `apps/dashboard/src/main.tsx` — 更新 APP_VERSION
- `apps/api/features/knowledge/pages/InstructionBook.tsx` — 新建（从 frontend/src 移植）
- `apps/api/features/knowledge/index.ts` — 注册路由
- `apps/api/features/knowledge/pages/KnowledgeHome.tsx` — 添加入口卡片
- `scripts/rebuild-dashboard.sh` — 新建自动重建脚本

## 成功标准

- APP_VERSION 更新为新日期值，用户刷新即可清除旧 SW 缓存
- InstructionBook 页面在 /knowledge/instruction-book 路由可访问
- rebuild-dashboard.sh 脚本可执行，重建完成后 dist 更新

**非目标**: 不设置 git hook（CLAUDE.md 约束代码改动需走 /dev）
