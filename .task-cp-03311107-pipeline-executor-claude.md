# Task Card: 接通 content pipeline executor 的 Claude 调用

## 需求（What & Why）

**功能描述**：将 `content-pipeline-executors.js` 中 4 个未接通 Claude 的阶段，改为真正调用 Claude CLI 执行内容生成和审核。

**背景**：
- `executeCopywriting` 用本地模板拼文案，不调用 Claude
- `executeCopyReview` 用静态关键词规则，不调用 Claude
- `executeGenerate` 只标记状态，不生成任何内容
- `executeImageReview` 只检查文件存不存在
- 配置页面设置的 `generate_prompt`、`review_prompt`、`image_prompt`、`image_review_prompt` 全部被忽略

**不做什么**：
- 不修改 `executeResearch`（已接通 notebooklm，保持不变）
- 不修改 `executeExport`
- 不改 orchestrator 调度逻辑
- 不修改数据库 schema
- `executeGenerate` 和 `executeImageReview` 本次接通 Claude 文本调用，不接入真实图片渲染 API（渲染仍由 executeExport 的 resvg 负责）

## 成功标准

- `executeCopywriting` 使用配置的 `generate_prompt` 调用 Claude CLI，产出真实文案文件（不是模板拼接），rerun 时注入上次审核反馈
- `executeCopyReview` 使用配置的 `review_prompt` 调用 Claude CLI，返回每条 `review_rule` 的评分对象和 pass/fail
- `executeGenerate` 使用配置的 `image_prompt` 调用 Claude CLI，产出图片描述文件 `cards/image-descriptions.md`
- `executeImageReview` 使用配置的 `image_review_prompt` 调用 Claude CLI，返回 `{review_passed, issues}` 结构

## 实现方案

**要改的文件**：`packages/brain/src/content-pipeline-executors.js`

**调用方式**：使用 `execSync('claude -p "..." --output-format json', {timeout: 120000})`

**executeCopywriting 设计**：
1. 读 `typeConfig.template.generate_prompt`，替换 `{keyword}` 和 `{findings}`
2. 若 `task.payload.review_feedback` 存在（rerun 场景），append 到 prompt："上一次审核反馈：{review_feedback}，请在此基础上改进。"
3. 调用 `claude -p "<prompt>" --output-format text`
4. 把输出写入 `cards/copy.md` 和 `article/article.md`（按 `---` 分割或智能解析）

**executeCopyReview 设计**：
1. 读 `typeConfig.template.review_prompt`，注入文案内容
2. 注入 `review_rules` 列表，要求 Claude 对每条规则给出 `{id, passed, score, comment}`
3. 调用 `claude -p "<prompt>" --output-format json`
4. 解析返回 `{review_passed, rule_results, issues}`
5. 任意 blocking 规则 `passed=false` → `review_passed=false`

**executeGenerate 设计**：
1. 读 `typeConfig.template.image_prompt`，注入文案内容
2. 调用 `claude -p "<prompt>" --output-format text`
3. 把输出写入 `cards/image-descriptions.md`

**executeImageReview 设计**：
1. 读 `typeConfig.template.image_review_prompt`（或 fallback 到 review_prompt）
2. 注入图片描述文件内容
3. 调用 `claude -p "<prompt>" --output-format json`
4. 解析 `{review_passed, issues}`
