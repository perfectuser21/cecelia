# Task Card: exit_code 分流 — OOM 降并发重试 vs 代码错误阻塞

## 需求（What & Why）

**功能描述**：在 dev 任务失败分类器中，对 exit_code=137（OOM/被强制 kill）和 exit_code=1（代码错误）实施不同的重试策略：
- exit_code=137 → 新增 `resource_killed` 类别，延迟重试（15min 起），附带 reduce_concurrency 提示
- exit_code=1 → code_error + needs_blocked → 不重试，直接标记 `blocked` 等待人工修复

**背景**：Cortex Insight（learning_id: d5432407）发现历史任务（3c3efc10、844576b3）两种失败被同等对待，OOM 直接放弃（实际可缓解），代码错误无感知标记（实际需人工介入）。

**不做什么**：不改变文本模式匹配的 code_error 行为（仍可重试）；不改变 quarantine.js；不改变 transient/auth/resource 文本匹配。

## 成功标准

- exit_code=137 传入时，`classifyDevFailure` 返回 class='resource_killed'，retryable=true，reduce_concurrency=true，delay≥15min
- exit_code=1 传入时，`classifyDevFailure` 返回 class='code_error'，retryable=false，needs_blocked=true
- execution.js 收到 needs_blocked=true 时，将任务 UPDATE status='blocked'（blocked_reason='code_error'，无 blocked_until）
- 现有文本匹配 code_error（TypeScript error 等）retryable 行为不变
