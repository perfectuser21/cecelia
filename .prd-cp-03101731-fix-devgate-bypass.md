# PRD：A+ 强制开发证据方案（DevGate 系统性漏洞修复）

## 背景

当前 /dev 工作流存在系统性绕过漏洞。根本原因不是某个脚本写错，而是：

> **开发证据（PRD/DoD/LEARNINGS）是必需品，但系统把它当成了可选附件。**

具体问题链：
1. `.prd-*.md` / `.dod-*.md` 在 `.gitignore` 中 → CI 永远看不到
2. CI 检查脚本全部有"文件不存在 → 跳过（exit 0）"逻辑
3. `ci-passed` gate 只判断 `failure/cancelled`，`skipped` 被放过
4. 结果：只要建一个 `cp-XXXXXXXX-xxx` 分支名，不提交任何证据，CI 全绿

## 目标

将 /dev 从"制度约束"升级为"工程化约束"：
- **制度约束**：有流程，但可以绕过
- **工程化约束**：系统级别无法绕过

## 功能描述

### P0：核心修复

**1. 移除 `.gitignore` 对 PRD/DoD 的忽略**

`.prd-*.md` 和 `.dod-*.md` 改为 git 追踪文件（可审计、可 CI 读取）。
`.dev-mode*` 和 `.dev-lock*` 等运行时状态文件仍在 gitignore 中。

**2. CI 检查脚本：缺失必需文件 → 直接 fail（不再 skip）**

- `check-dod-mapping.cjs`：CI 环境找不到 DoD → `exit 1`（不是 `exit 0`）
- `check-prd.sh`：找不到 PRD 文件 → `exit 1`（不是 `exit 0`）
- `check-learning.sh`：LEARNINGS.md 不存在 → `exit 1`；本 PR 没有新增 LEARNINGS 内容 → `exit 1`

**3. `ci-passed` gate 改为只接受 `success`**

所有依赖 job 的结果判断改为 `!= 'success'`，消除 `skipped` 漏洞。

### P1：Learning 规则明确化

- 走 /dev 的 PR 必须修改 `docs/LEARNINGS.md`
- 允许唯一例外：`[SKIP-LEARNING]` 标签 + PR body 说明原因（纯文档/配置修复）

### P2：基于路径触发强制 dev 检查

对高风险改动路径（CI workflow、hooks、scripts、brain 核心逻辑），在 DevGate 中自动要求 PRD/DoD 存在。
配置文件：`packages/engine/config/required-dev-paths.yml`

## 成功标准

- 1. 创建一个 `cp-XXXXXXXX-test` 格式分支，不提交任何 PRD/DoD/LEARNINGS，推 PR → DevGate CI 必须失败（当前会通过）
- 2. `.prd-*.md` 和 `.dod-*.md` 文件可以被 `git add` 并出现在 PR diff 中
- 3. `ci-passed` job 的逻辑中，任何 job result 不是 `success` 均导致 gate 失败
- 4. `check-prd.sh` 在找不到 PRD 文件时 exit 1
- 5. `check-dod-mapping.cjs` 在 CI 环境找不到 DoD 时 exit 1
- 6. `check-learning.sh` 在 PR 没有新增 LEARNINGS 内容时 exit 1
- 7. P2 路径触发规则配置文件存在并被 devgate.yml 引用

## 不在范围内

- 工件目录规范化（散落根目录的问题，后续 P2+ 处理）
- 本地 branch-protect.sh 的改动（已经有本地保护，这次只加固 CI）
