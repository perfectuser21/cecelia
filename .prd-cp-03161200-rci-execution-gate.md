# PRD: RCI Execution Gate — CI 实际执行 regression-contract test_command

## 背景

`packages/quality/contracts/cecelia-quality.regression-contract.yaml` 中每个 RCI 条目都有 `test_command`（如 `bash tests/test-gateway-http.sh`），但 CI 从来不执行这些命令，回归契约形同虚设。

## 目标

1. 新建脚本 `packages/engine/scripts/devgate/rci-execution-gate.sh`
2. 脚本扫描 `packages/quality/contracts/cecelia-quality.regression-contract.yaml`，提取 `priority: P0` 条目并执行 `test_command`
3. 将脚本接入 `ci-l1-process.yml` 的 `dod-check` job 尾部

## 成功标准

- `rci-execution-gate.sh` 文件存在于 `packages/engine/scripts/devgate/`
- 脚本解析 cecelia-quality RCI，只执行 P0 条目
- 执行前检查 test_file 是否存在，不存在则 DEFERRED（而非 FAIL）
- 需要运行时服务的命令（启动后台进程等）也输出 DEFERRED: requires runtime
- 输出格式：`✅ C-DB-INIT-001: pass` 或 `❌ C-GATEWAY-HTTP-001: failed` 或 `⏭️  C-DB-INIT-001: DEFERRED`
- CI `dod-check` job 新增 "Run RCI Execution Gate (P0 contracts)" step
- Engine 版本 bump 到 12.87.0
