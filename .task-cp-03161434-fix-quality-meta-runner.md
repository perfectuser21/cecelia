# Task Card: 修复 quality-meta-tests job CI 性能问题

## 需求（What & Why）

**功能描述**: 将 `ci-l1-process.yml` 中 `quality-meta-tests` job 的 `runs-on` 从 HK VPS 改为 `ubuntu-latest`。

**背景**: 该 job 当前耗时 8 分钟，主要原因是 `runs-on: [self-hosted, Linux, hk-vps]` 放到了 HK VPS，有约 5 分钟的排队等待时间。注意：`npm ci --ignore-scripts` 步骤实际上是必需的（`check-dod-mapping.cjs` 依赖 `js-yaml`），因此保留此步骤，只迁移 runner。

**不做什么**: 不修改测试脚本，不删除 npm ci（js-yaml 是必要依赖）。

## 成功标准

1. [ARTIFACT] `quality-meta-tests` job 的 `runs-on` 改为 `ubuntu-latest`（不再是 hk-vps）
2. [BEHAVIOR] 该 job 在 ubuntu-latest 上能正常运行并通过测试
3. [ARTIFACT] Engine 版本 bump 到 12.89.0（5 个文件同步）
4. [ARTIFACT] `feature-registry.yml` 包含 12.89.0 changelog 条目
5. [GATE] CI L1 全部通过

## 验收条件（DoD）

- [x] [ARTIFACT] `quality-meta-tests` job 的 `runs-on` 是 `ubuntu-latest`（不含 hk-vps）
  Test: manual:bash -c "grep -A 3 'quality-meta-tests:' .github/workflows/ci-l1-process.yml | grep 'ubuntu-latest'"

- [x] [ARTIFACT] `quality-meta-tests` job 不再依赖 hk-vps runner
  Test: manual:node -e "const fs=require('fs');const c=fs.readFileSync('.github/workflows/ci-l1-process.yml','utf8');const m=c.match(/quality-meta-tests:[\s\S]{0,200}/);if(m&&m[0].includes('hk-vps'))throw new Error('FAIL: hk-vps still in job');console.log('PASS')"

- [x] [BEHAVIOR] Engine 版本文件全部同步到 12.89.0
  Test: manual:node -e "const fs=require('fs');['packages/engine/VERSION','packages/engine/.hook-core-version'].forEach(f=>{const v=fs.readFileSync(f,'utf8').trim();if(v!=='12.89.0')throw new Error(f+' wrong: '+v)});console.log('PASS')"

- [x] [ARTIFACT] `feature-registry.yml` 包含 12.89.0 条目
  Test: manual:node -e "const fs=require('fs');const c=fs.readFileSync('packages/engine/features/feature-registry.yml','utf8');if(!c.includes('12.89.0'))throw new Error('FAIL');console.log('PASS')"

- [x] [GATE] 所有现有测试通过
  Test: manual:bash -c "node packages/engine/scripts/devgate/check-dod-mapping.cjs 2>&1 | tail -5"

## 实现方案

**要改的文件**:
1. `.github/workflows/ci-l1-process.yml` — 两处修改（runs-on + 删除 npm ci step）
2. `packages/engine/VERSION` — bump 12.88.0 → 12.89.0
3. `packages/engine/package.json` — bump 版本号
4. `packages/engine/package-lock.json` — 同步版本号（两处）
5. `packages/engine/.hook-core-version` — bump 版本号
6. `packages/engine/regression-contract.yaml` — bump 版本号
7. `packages/engine/features/feature-registry.yml` — 添加 changelog 条目

**Scope 锚定**: CI 配置修改 + Engine 版本 bump，无逻辑变更
