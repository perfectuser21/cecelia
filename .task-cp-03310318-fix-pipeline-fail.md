# Task Card: 修复 content pipeline orchestrator 忽略阶段失败

## 分支
cp-03310318-62989b2d-0467-4dcb-801e-fd6db8

## 问题描述
`STAGE_HANDLER_MAP` 中 `content-research`、`content-copywriting`、`content-generate`、`content-export`
四个非审核阶段忽略 `taskStatus` 参数（用 `_s` 丢弃），导致即使阶段返回 `{success:false}` 
（即 `taskStatus === 'failed'`），pipeline 仍会推进到下一阶段，最终显示 `completed` 但无实际产出。

## 根因
```js
// 当前（有 bug）：
'content-research': (ctx, _s, _f, db) => _handleResearchComplete(ctx, db),
// _s = taskStatus 被丢弃，无论成功失败都执行 _handleResearchComplete
```

## 修复方案
1. 新增 `_markPipelineFailed(ctx, db)` 辅助函数
2. 在 `STAGE_HANDLER_MAP` 中为 4 个非审核阶段加入 `taskStatus` 检查：
   当 `status === 'failed'` 时调用 `_markPipelineFailed`，停止 pipeline
