# PRD: 修复 Brain API 图片扫描 — 改为前缀匹配

## 背景

Brain API 的 `GET /api/brain/pipelines/:id/output` 路由中，图片扫描逻辑使用精确文件名匹配（如 `dankoe-01.png`），但实际生成的图片文件名带有描述词后缀（如 `dankoe-01-failures.png`），导致 `image_urls` 字段始终返回空数组。

## 目标

改为前缀扫描：读取目录后过滤以 `{topic}-` 开头且以 `.png` 结尾的文件，封面文件名包含 `-cover.`，卡片文件为其余文件按字母排序。

## 成功标准

- 当目录中存在 `dankoe-01-failures.png` 时，API 响应的 `image_urls` 包含该图片（type: card）
- 当目录中存在 `dankoe-cover.png` 时，API 响应的 `image_urls` 包含该图片（type: cover）
- 不改变输出数据结构（`image_urls` 数组格式、`images` 兼容字段）
- 所有现有测试通过
