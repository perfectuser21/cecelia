# Task Card: Pipeline Rerun 幂等检查修复

## 需求（What & Why）

**功能描述**: 修复 `_startOnePipeline` 函数的幂等检查逻辑，使 `completed` 状态的旧子任务不再阻止 pipeline 重跑。

**背景**: 当用户对一个已完成的 pipeline 触发 rerun 时，`_startOnePipeline` 的幂等查询包含了 `completed` 状态，导致找到旧的已完成 content-research 子任务，误判为"已在飞"而跳过创建新子任务。结果 pipeline 被标记为 `in_progress` 但永远无法推进，进入死锁状态。

**不做什么**: 不改动 advanceContentPipeline 中的其他查询逻辑，不修改 pipeline 重跑的触发机制，不改变幂等检查的其他字段。

## 成功标准

> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [ARTIFACT] `content-pipeline-orchestrator.js` 中幂等检查的 SQL 不再包含 `completed` 状态
2. [BEHAVIOR] 已有 completed 子任务的 pipeline 触发重跑时，能成功创建新的 content-research 子任务
3. [GATE] CI 全部通过

## 验收条件（DoD）

  Test: manual:node -e "const fs=require('fs'); const src=fs.readFileSync('packages/brain/src/content-pipeline-orchestrator.js','utf8'); const m=src.match(/status IN \([^)]+\)/g); const idempotentCheck=m&&m.find(s=>s.includes('content-research')||src.indexOf(s)>src.indexOf('_startOnePipeline')); if(!m||m.some(s=>s.includes('completed')&&src.substring(0,src.indexOf(s)).includes('_startOnePipeline')))throw new Error('幂等检查仍含 completed'); console.log('PASS: 幂等检查不含 completed');"

## 实现方案

**要改的文件**: `packages/brain/src/content-pipeline-orchestrator.js`

**Scope 锚定**: 第 116 行，将 `AND status IN ('queued', 'in_progress', 'completed')` 改为 `AND status IN ('queued', 'in_progress')`
