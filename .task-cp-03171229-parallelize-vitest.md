# Task Card: L3 vitest 串行→分组并行提速

## 需求（What & Why）

**功能描述**: 修改 `packages/engine/vitest.config.ts`，将 17 个无 shell 依赖的测试文件设为可并行执行，39 个有 execSync/exec/spawn/child_process 依赖的测试文件保持串行（或受控并发），从而降低 L3 CI 耗时。

**背景**: 当前 `fileParallelism: false` 导致 56 个测试文件全部串行运行，L3 CI 耗时 10+ 分钟。经分析，17 个测试文件（ci/known-failures*.test.ts、ci/scan-rci-coverage.test.ts、dev/checklist.test.ts、dev/dev-step-files.test.ts、dev/step-expectations.test.ts、devgate/check-changed-coverage.test.ts、devgate/l2b-check.test.ts、devgate/snapshot-prd-dod.test.ts、scripts/cleanup.test.ts、scripts/stop-cleanup-bugfixes.test.ts、skills/language-rule.test.ts、stop-hook-flow-test.test.ts、utils/mathUtils.test.ts、workflow-guard-2.test.ts、workflow-guard-3.test.ts、workflows/ci-timeout.test.ts）不依赖 shell 命令，可以安全并行。

**不做什么**:
- 不删除任何测试
- 不修改测试逻辑（只改 vitest 配置）
- 不修改有 shell 依赖的测试的执行方式（保持串行或限制并发）

## 实现方案

**要改的文件**: `packages/engine/vitest.config.ts`

**分析结果**:
- CLEAN（可并行）17 个文件：ci/known-failures-expiry、ci/known-failures、ci/scan-rci-coverage、dev/checklist、dev/dev-step-files、dev/step-expectations、devgate/check-changed-coverage、devgate/l2b-check、devgate/snapshot-prd-dod、scripts/cleanup、scripts/stop-cleanup-bugfixes、skills/language-rule、stop-hook-flow-test、utils/mathUtils、workflow-guard-2、workflow-guard-3、workflows/ci-timeout
- SHELL（需串行）39 个文件：其余所有 hooks/ 测试以及有 execSync/spawn 的文件

**方案**: 使用 vitest `projects` 配置（vitest 3.x workspace）：
1. `parallel-group`：include CLEAN 文件列表，`fileParallelism: true`
2. `serial-group`：include 其余文件，`fileParallelism: false`
