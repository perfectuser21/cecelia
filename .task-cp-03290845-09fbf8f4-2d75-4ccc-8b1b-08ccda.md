# Task Card: DoD Test 字段升级 — 从模糊描述升级为可执行命令

## 需求（What & Why）

**功能描述**：
升级 DoD [BEHAVIOR] 条目的 Test 字段格式规范和检查脚本。要求：
1. `check-dod-mapping.cjs` 新增 CI 不兼容命令检测，拦截 `curl localhost/psql/npm test` 等在 L1 CI 中必然失败的命令
2. 弱测试（echo/grep wc-l 等）已被 detectFakeTest 拦截，本次补充 CI 不兼容模式检测
3. `DOD-TEMPLATE.md` 模板从旧格式 evidence ID（`manual:code-review`）升级为可执行命令

**背景**：
- `check-dod-mapping.cjs` 在 `GITHUB_ACTIONS=true` 时实际执行 `manual:` 命令
- L1 CI 无 PostgreSQL、无 Brain 服务、无 node_modules，导致 `curl localhost/psql/npm test` 静默失败，但错误信息不清晰
- `DOD-TEMPLATE.md` 模板使用 `manual:code-review` 等旧格式，在当前校验器下会因找不到 evidence 文件而直接失败

**不做什么**：
- 不修改已有的 `detectFakeTest()` 逻辑
- 不修改 `validateBehaviorTestStrength()` 逻辑
- 不修改 CI workflow 文件
- 不删除 `manual:<EVIDENCE_ID>` 旧格式支持（向后兼容）

## 成功标准

1. [ARTIFACT] `check-dod-mapping.cjs` 包含 `detectCiIncompatibleCommand` 函数
2. [BEHAVIOR] `manual:curl localhost:5221/...` → check-dod-mapping.cjs 返回 exit 1 并给出友好提示
3. [BEHAVIOR] `manual:psql cecelia -c "..."` → check-dod-mapping.cjs 返回 exit 1
4. [BEHAVIOR] `DOD-TEMPLATE.md` 中 Test 字段为 CI 兼容的可执行命令格式（无旧式 `manual:code-review`）
5. [GATE] CI 全部通过

## 实现方案

**要改的文件**：
- `packages/engine/scripts/devgate/check-dod-mapping.cjs`：新增 `detectCiIncompatibleCommand(cmd)` 函数，在 inline command 路径中调用
- `packages/engine/templates/DOD-TEMPLATE.md`：更新 Test 字段示例为 CI 兼容命令
- `packages/engine/tests/devgate/check-dod-mapping-ci-incompatible.test.ts`（新文件）：新增 CI 不兼容命令检测的测试

**Scope 锚定**：
- `detectCiIncompatibleCommand` 在 `validateTestRef` 的 `isInlineCommand=true` 路径中，在 `detectFakeTest` 检查之后调用
- 阻断模式：`curl (localhost|127\.0\.0\.1|\$BRAIN_URL|127\.0\.|0\.0\.0\.0)`、`psql `、`npm (test|run test)`、`npx vitest`
- 提示用 `node -e "require('fs').readFileSync/accessSync"` 替代
