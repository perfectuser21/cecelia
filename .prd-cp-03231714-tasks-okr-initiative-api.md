# PRD: tasks API 支持 okr_initiative_id 字段读写

## 背景
PR #1441 添加了 `tasks.okr_initiative_id` 列（UUID FK → okr_initiatives），但
POST/PATCH `/api/brain/tasks` 端点没有暴露此字段的写入能力，导致 Task 无法通过
API 与 OKR Initiative 关联，KR 进度自动回算闭环断链。

## 问题定位
- `packages/brain/src/routes/task-tasks.js` POST `/` — INSERT 不含 okr_initiative_id
- `packages/brain/src/routes/task-tasks.js` PATCH `/:id` — 只接受 status/priority/title
- GET `/:id` 已做 `SELECT *`，能返回该字段（无需改）

## 需要改动
1. POST `/` — 接受 `okr_initiative_id` 参数，写入 INSERT
2. PATCH `/:id` — 接受 `okr_initiative_id` 参数，写入 UPDATE

## 验收标准
- POST 创建 task 时传 okr_initiative_id，GET 查回来字段非 null
- PATCH 更新 okr_initiative_id，GET 查回来字段已更新
- 无 okr_initiative_id 时行为向后兼容（仍然正常创建/更新）
