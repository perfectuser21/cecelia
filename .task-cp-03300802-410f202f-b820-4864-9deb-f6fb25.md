# Task Card: PipelineOutputPage 重构为作品主页（四 Tab）

## 需求（What & Why）

**功能描述**：将内容工厂 Pipeline 详情页重构为完整的内容作品主页，采用深色风格（#07050f 背景、紫色渐变标题），展示四个维度的数据。

**背景**：当前内容工厂 `/content-factory` 仅有触发表单和 Pipeline 列表，点击 Pipeline 后没有详情页。需要为每条 Pipeline 创建完整的作品主页，让创作者能一眼看清楚内容生产成果、发布状态、数据表现。

**四 Tab 结构**：
1. **Summary** - 整体表现汇总（总曝光、总互动、平台数、互动率）
2. **生成记录** - 内容产出（封面图、卡片图片、文章/卡片文案预览）+ Pipeline 执行阶段
3. **发布记录** - 8 个平台各自的发布状态（已发布/定时/未发布/失败）—— 暂时 mock
4. **数据记录** - 各平台数据（播放/点赞/评论/收藏）—— 暂时 mock 占位

**数据来源**：
- 生成记录：Brain API `/api/brain/pipelines/:id/output` + `/api/brain/pipelines/:id/stages`
- 发布记录：Brain API（platform_publishes 表，暂时 mock 如无数据）
- 数据记录：各平台回填（暂时 mock 结构占位）

**不做的事**：不新建后端表、不改 Brain API、只改前端页面

## 成功标准

- 访问 `/content-factory/:id` 时，展示带四个 Tab 的作品详情页
- 生成记录 Tab 正确展示文章文案、卡片文案、Pipeline 执行阶段信息
- 发布记录和数据记录 Tab 有 mock 占位内容（8 平台结构）
- ContentFactory 列表页中的 Pipeline 条目可点击跳转到详情页

## 技术方案

**修改文件**：
1. 新建 `apps/api/features/content/pages/PipelineOutputPage.tsx` — 四 Tab 作品详情页
2. 新建 `apps/api/features/content/pages/PipelineOutputPage.test.tsx` — 测试文件
3. 修改 `apps/api/features/knowledge/index.ts` — 注册新路由 `/content-factory/:id`
4. 修改 `apps/api/features/content/pages/ContentFactory.tsx` — 列表项可点击

**路由**：`/content-factory/:id` → `PipelineOutputPage`

**设计规范**：
- 背景色：`#07050f`（深紫黑）
- 标题：紫色渐变（`from-purple-400 to-purple-200`）
- Tab 激活：紫色下划线
- 卡片边框：`border-purple-900/30`
