# Task Card: Pipeline 步骤详情 Input/Prompt/Output 三栏视图

## 需求（What & Why）
**功能描述**: 重新设计 Harness Pipeline 详情页，将当前"阶段概览+文件折叠"替换为"串行步骤列表+三栏钻取视图"。每步可展开查看 Input/Prompt/Output。
**背景**: 当前详情页只展示阶段状态和文件内容，无法回溯每步 AI 实际收到了什么 prompt、输入了什么、产出了什么。
**不做什么**: 不修改 executor.js preparePrompt 本身，不改 DB schema，不做 Markdown 渲染/语法高亮。

## 成功标准

- [BEHAVIOR] API 返回 steps 串行数组，包含每步的 input/prompt/output 内容
- [BEHAVIOR] 前端步骤列表可展开显示三栏视图
- [ARTIFACT] 单元测试验证 buildSteps 逻辑

## 验收条件（DoD）

## 实现方案

**要改的文件**:
- `packages/brain/src/routes/harness.js`：新增 `buildSteps()` 函数 + `rebuildPrompt()` 函数 + `fetchFileFromBranch()` 函数，pipeline-detail 端点新增 steps 返回
- `apps/dashboard/src/pages/harness-pipeline/HarnessPipelineDetailPage.tsx`：重写主体区域，新增 StepList + StepDetailPanel 组件

**受影响函数/API**:
- `GET /api/brain/harness/pipeline-detail` — 响应新增 steps 字段
- `buildStages()` — 保留不动（阶段时间线概览仍需要）
- `buildGanRounds()` — 保留不动（steps 构建参考其逻辑）

**不改什么**:
- `packages/brain/src/executor.js` — 只作为参考，不修改
- DB schema — 不新增表或字段
