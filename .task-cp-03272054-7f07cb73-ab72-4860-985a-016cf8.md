## 实现方案

### 要改的文件
1. **新建** `packages/brain/src/daily-publish-scheduler.js` — 每日调度器
2. **新建** `packages/brain/src/publish-monitor.js` — 队列监控 + 重试
3. **修改** `packages/brain/src/tick.js` — 集成两个新模块
4. **修改** `packages/brain/src/routes/publish-jobs.js` — 增加 /stats 端点
5. **修改** `packages/brain/server.js` — 确认路由注册

### Scope 锚定

**daily-publish-scheduler.js** 核心逻辑：
```
triggerDailyPublish(pool):
  1. 检查 working_memory 是否今日已触发（防重复）
  2. 查询 content_publish_jobs WHERE status='pending' ORDER BY created_at
  3. 优先级平台: ['douyin', 'xiaohongshu', 'wechat', 'kuaishou', 'weibo', 'toutiao']
  4. 为每个 pending job 创建 content_publish task（跳过今日已有的）
  5. 写入 working_memory 记录今日触发时间 + 创建数量
  6. 返回 {created, skipped, platforms}
```

**publish-monitor.js** 核心逻辑：
```
monitorPublishQueue(pool):
  1. 查询 failed content_publish tasks WHERE retry_count < 3
  2. 重置为 queued + retry_count+1（让 tick 重新派发）
  3. 统计今日 content_publish tasks 状态（queued/in_progress/completed/failed）
  4. 写入 working_memory key='daily_publish_stats'
  5. 返回 {retried, stats}
```

**GET /api/brain/publish/stats**：
```
从 working_memory 读取 daily_publish_stats
+ 实时查询今日 content_publish tasks 状态
返回 {date, coverage, success_rate, platforms: [...]}
```

**tick.js 集成点**：
- 在 nightly-tick 附近的 09:00 CST 检查块中调用 triggerDailyPublish
- 每 tick 在 content pipeline 之后调用 monitorPublishQueue
