## PRD - feat(learning): OAuth 401 失败 Learning 聚合去重

**需求来源**: Brain 自动调度任务
**功能描述**: 在写入 learning 时，检测是否存在同类 failure（相同 error_type + 相同 platform/category），若存在则合并（increment count）而非新增记录，解决 241 条积压中大量重复 OAuth 401 记录的问题
**涉及文件**:
- `packages/brain/src/actions.js` — recordLearning() 添加 upsert 逻辑
- `packages/brain/migrations/` — 新增 occurrence_count 字段 migration
- `packages/brain/src/__tests__/` — 单元测试
**非目标**: 不修改 Cortex 读取 learning 的逻辑；不删除现有 learning 记录（仅聚合）

## 成功标准
- 新 learning 写入时检查：同 error_type + 同 source_category 的 learning 若 created_at > now()-24h 则合并
- 合并时 learning.occurrence_count +1，updated_at = now()
- learnings 表新增 occurrence_count INTEGER DEFAULT 1 字段（migration）
- 针对现有积压执行一次历史聚合（按 error_type 分组，合并为代表性记录）
- 聚合后 unprocessed learning 数量预计从 241 降至 < 30
- 单元测试覆盖聚合逻辑

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| migration 执行失败 | 回滚，记录错误日志 | P1 | learning_migration_failed |
| upsert 查询失败 | 降级为 insert（保证不丢 learning） | P2 | learning_upsert_failed |
| 历史聚合脚本失败 | 记录日志，不影响正常运行 | P2 | learning_backfill_failed |
