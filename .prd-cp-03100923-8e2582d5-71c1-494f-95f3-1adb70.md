## PRD - 实现 POST /api/brain/projects/compare/report 接口

**需求来源**: Brain 自动调度任务（task_type: dev）
**功能描述**: 在 Brain API 中新增跨项目对比报告生成接口，支持 JSON 和 Markdown 两种输出格式，包含评分、优劣势分析和对比摘要
**涉及文件**:
- `packages/brain/src/routes.js` — 新增路由
- `packages/brain/src/project-compare.js` — 新建逻辑文件（< 150 行）
- `packages/brain/tests/project-compare.test.js` — 单元测试

**非目标**: 不实现导出到文件、不实现 PDF 格式、不改动现有路由

## 成功标准

1. `POST /api/brain/projects/compare/report` 传入 2 个有效 project_id，返回 200 含完整 JSON 报告
2. `format=markdown` 返回 markdown 字段，格式正确
3. 传入不存在的 project_id 返回 400
4. 传入少于 2 个 project_id 返回 400
5. 有单元测试覆盖正常路径和错误路径
6. 不破坏现有 Brain CI（L3 brain-unit 通过）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| project_ids 少于 2 个 | 返回 400 + 错误消息 | P3 | compare_report_invalid_input |
| project_id 不存在于数据库 | 返回 400 + 错误消息 | P3 | compare_report_project_not_found |
| 数据库查询异常 | 返回 500 + 记录日志 | P1 | compare_report_db_error |

### 端点规范

**请求**:
```
POST /api/brain/projects/compare/report
Content-Type: application/json

{
  "project_ids": ["<uuid>", "<uuid>"],
  "format": "json" | "markdown",      // 默认 json
  "include_tasks": true | false        // 默认 false
}
```

**响应（JSON 格式）**:
```json
{
  "generated_at": "ISO8601",
  "format": "json",
  "projects": [
    {
      "id": "<uuid>",
      "name": "项目名称",
      "type": "initiative",
      "status": "active",
      "task_stats": {
        "total": 10,
        "completed": 6,
        "in_progress": 2,
        "queued": 2,
        "completion_rate": 0.6
      },
      "strengths": ["完成率高", "无阻塞任务"],
      "weaknesses": ["缺乏 P0 任务"],
      "score": 75
    }
  ],
  "summary": "对比摘要文字",
  "markdown": "..."   // 仅当 format=markdown 时返回
}
```

**评分维度（总分 100）**:
- 完成率（0-40 分）：`completion_rate * 40`
- 无阻塞任务（0-20 分）：无 failed/quarantined 任务得满分
- 有 P0 任务且进行中（0-20 分）
- 最近 7 天活跃度（0-20 分）：有 updated_at 在 7 天内的任务得满分
