# Task Card: 启动每日AI内容生成调度引擎

## 需求（What & Why）

**功能描述**：接通每日AI内容生成调度引擎的最后一公里——注册 content-pipeline API 路由、创建 `topic_selection_log` 数据库迁移、补全选题手动触发接口，使整条 选题→生成→质检→入库→发布队列 流程完整可运行。

**背景**：
- `topic-selection-scheduler.js`、`topic-selector.js`、`content-pipeline-orchestrator.js`、`content-pipeline-executors.js` 均已实现
- tick.js 已调用 `triggerDailyTopicSelection` + `orchestrateContentPipelines` + `executeQueuedContentTasks`
- 但 `content-pipeline.js` 路由**从未注册**到 `routes.js`，导致 `/api/brain/pipelines` 等端点 404
- `topic_selection_log` 表不存在，历史选题去重数据丢失（7 日避重失效）

**不做什么**：
- 不改动任何 executor 业务逻辑（research/copywriting/review/generate/export）
- 不新增内容类型（YAML 已够用）
- 不改动 tick 调用时机
- routes.js 不修改（server.js 已直接注册 content-pipeline 路由）

## 成功标准

- `GET /api/brain/content-types` 返回已注册内容类型列表（不再 404）
- `GET /api/brain/pipelines` 返回最近 pipeline 任务列表（不再 404）
- `POST /api/brain/pipelines/trigger-topics` 可手动触发今日选题生成
- migration 203 执行后 `topic_selection_log` 表存在
- 所有新增文件通过 CI L1-L4
