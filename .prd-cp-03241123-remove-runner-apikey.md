# PRD: runner.sh 删除 API Key 注入，强制 OAuth 模式

## 背景

`packages/engine/runners/codex/runner.sh` 当前版本（v2.4.1）在启动时会从
`~/.credentials/openai.env` 加载 `OPENAI_API_KEY`，并将其导出为 `CODEX_API_KEY`。

这导致 Codex 进入 **API Key 模式**（调用 `https://api.openai.com/v1/responses`），
而不是用户要求的 **ChatGPT OAuth 包月模式**（`chatgpt.com/backend-api/...`）。

API Key 模式在西安机器通过 SOCKS5 代理时会失败，且消耗 API 配额。

## 目标

删除 runner.sh 中的 API Key 注入代码块（lines 103-125），确保所有 Codex runner
始终使用 ChatGPT OAuth 包月模式，**永远不走 API Key 模式**。

## 范围

- `packages/engine/runners/codex/runner.sh`：删除 API Key 注入逻辑
- 版本 bump：v2.4.1 → v2.5.0（功能性改动）

## 成功标准

- runner.sh 中不再包含 `CODEX_API_KEY` 加载逻辑
- runner.sh 中不再包含对 `~/.credentials/openai.env` 的读取
- runner.sh 版本号更新为 v2.5.0
- 若环境变量 `CODEX_API_KEY` 已被外部设置，runner 不干预（不清除，也不设置）
