# PRD: Brain 部署流程可观测性补齐

## 背景
Brain deploy webhook (`POST /api/brain/deploy`) 是 fire-and-forget 设计：
返回 202 后异步执行 `deploy-local.sh`，无状态追踪机制。
GitHub Actions `deploy.yml` 只验证 HTTP 响应码，无法知道部署是否真正成功。

## 目标
1. Brain 内存中记录最近一次部署的状态（idle/running/success/failed）
2. 暴露 `GET /api/brain/deploy/status` 端点供 CI 查询
3. `deploy.yml` 在触发 webhook 后 poll status 确认部署完成

## 范围
- `packages/brain/src/routes/ops.js`
- `.github/workflows/deploy.yml`
- `packages/brain/src/__tests__/deploy-status.test.js`

## 成功标准
- GET /api/brain/deploy/status 返回含 status 字段的 JSON
- POST /api/brain/deploy 成功后 status 变为 success
- POST /api/brain/deploy 失败后 status 变为 failed
- deploy.yml 在 webhook 后 poll status 最多 300s
