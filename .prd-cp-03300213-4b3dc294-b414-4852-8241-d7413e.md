# PRD: 内容工厂 — 内容类型绑定 notebook_id + Pipeline 步骤详情视图

## 背景
内容工厂 Pipeline 有两个核心体验缺失：
1. 每次新建 Pipeline 需要手动填 NotebookLM ID，应改为从内容类型配置自动带入
2. Pipeline 只显示整体状态色块，看不到每个阶段的输入/输出/提示词/失败原因

## 需求

### A. 内容类型绑定 notebook_id（ZenithJoy 前端 + Cecelia 后端）
- solo-company-case 等内容类型配置里加 notebook_id 字段
- ContentTypeConfigPage 加 notebook_id 编辑入口
- 新建 Pipeline 时选内容类型后自动从配置带入 notebook_id

### B. Pipeline 步骤详情视图（ZenithJoy 前端）
- Pipeline 卡片点击展开，显示每个 stage 的真实状态（调用 /stages API）
- 每个 stage 显示：状态/耗时/失败原因
- StageProgress 改为使用真实 stage 数据而非仅靠 pipeline 整体 status 推断

## 成功标准
- [ ] ContentTypeConfigPage 可编辑 notebook_id
- [ ] 新建 Pipeline 表单选内容类型后自动填入 notebook_id
- [ ] Pipeline 卡片展开后显示真实 stage 状态
- [ ] solo-company-case 配置有 notebook_id 字段（后端）
