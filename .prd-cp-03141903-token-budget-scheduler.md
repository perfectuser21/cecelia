## PRD - 动态 Token 预算调度器

**需求来源**: 用户反馈上周 Claude 3账号在断续运行中提前耗尽，Codex 5账号 rate limit 命中，导致系统无法持续运行。需要一个动态调度系统，根据实时预算状态调整并发策略。

**功能描述**:

新增 `token-budget-planner.js` 模块，并改造 `slot-allocator.js` 实现动态槽位分配：

1. **预算规划器（token-budget-planner.js）**
   - 读取 account-usage.js 提供的各账号 5h/7day 使用率和 reset 时间
   - 计算每个 Claude 账号的"安全燃烧速率"：`remaining_budget × 0.70 ÷ hours_to_reset`
   - 70% 给自主任务，30% 永久保留给用户手动使用
   - 输出：`{ claude_safe_slots, codex_safe_slots, minimax_slots, total_safe }`

2. **动态 slot-allocator 改造**
   - task_pool 不再固定 5，改为每次 tick 前从 planner 读取
   - 三池动态分配：claude_pool + codex_pool + minimax_pool

3. **任务降级矩阵**
   - `dev` / `code_review`：Claude 充足 → Claude；Claude 紧张 → Codex；Claude 耗尽 → Codex
   - `architecture_design` / `strategy_session` / `initiative_plan`：只用 Claude，不降级，排队等待
   - `codex_dev` / `codex_qa`：始终走 Codex
   - `explore` / `research` / `talk`：始终走 MiniMax（不消耗 Claude）

**涉及文件**:
- `packages/brain/src/token-budget-planner.js`（新建）
- `packages/brain/src/slot-allocator.js`（改造）
- `packages/brain/src/tick.js`（集成 planner 调用）
- `packages/brain/src/task-router.js`（加 executor_affinity 字段）
- `packages/brain/tests/`（新增单元测试）

**成功标准**:

## 成功标准

- token-budget-planner.js 能正确计算各账号安全燃烧速率
- slot-allocator 的 task_pool 槽位数量随 Claude 账号剩余配额动态变化
- Claude 配额 < 20% 时，dev 任务自动路由到 Codex（降级）
- architecture/strategy 任务在 Claude 配额不足时保持排队，不降级
- 30% 用户预留逻辑生效（user_pool 预算不被自主任务占用）
- 全量 npm test 无回归

**非目标**:
- 不做跨账号 token 共享（账号独立计算）
- 不做 MiniMax 账号配额追踪（暂时视为无限）
- 不做 UI 看板（Brain API 已有 /api/brain/resources 端点可查）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| account-usage 接口超时 | 降级：使用上次缓存值，保守估计 50% 剩余 | P2 | budget_fetch_timeout |
| 所有 Claude 账号耗尽 | 只派 Codex + MiniMax 任务，Claude 任务排队 | P1 | claude_budget_exhausted |
| Codex bridge 不可用 | dev 任务回退 Claude，记录 warn | P2 | codex_bridge_unavailable |
| planner 计算异常 | 回退静态配置（task_pool=3），记录错误 | P1 | budget_planner_error |
