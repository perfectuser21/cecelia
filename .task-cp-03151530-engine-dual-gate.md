# Task Card: Engine 双重 Gate 状态机

## 需求（What & Why）

**功能描述**：在 /dev 工作流的每个步骤之间，实现 Script Gate（脚本验证）强制执行。具体：加深 branch-protect.sh 的 DoD 质量检查，加深 check-dod-mapping.cjs 的 CI 层深度检查。

**背景**：当前 branch-protect.sh 只检查 DoD 文件是否存在，不检查内容质量。导致 Claude 可以写 1 条 DoD 条目、全是 ARTIFACT、描述只有 3 个字，也能通过门禁。check-dod-mapping.cjs 也缺少条目数和类型的最低要求。这让整个双重 Gate 状态机的 Script Gate 层形同虚设。

**不做什么**：不改 stop-dev.sh 的步骤状态机逻辑（那是更大的架构改动）；不改 Skill steps/*.md（那是 LLM 层，本次只做 Script Gate 层）。

## 成功标准

1. [ARTIFACT] branch-protect.sh 新增 DoD 深度检查：条目数 < 3 → exit 2 拒绝写入
2. [ARTIFACT] branch-protect.sh 新增：没有 `[BEHAVIOR]` 标签条目 → exit 2 拒绝写入
3. [ARTIFACT] branch-protect.sh 新增：条目描述 < 20 字符 → exit 2 拒绝写入
4. [BEHAVIOR] 写一个只有 1 条 ARTIFACT 条目的 DoD 文件 → hook 实际拦截，不让写入
5. [BEHAVIOR] 写一个有 3 条条目但全是 ARTIFACT 的 DoD → hook 实际拦截
6. [ARTIFACT] check-dod-mapping.cjs 新增：条目数 < 3 → exit 1
7. [ARTIFACT] check-dod-mapping.cjs 新增：无 `[BEHAVIOR]` 条目 → exit 1
8. [GATE] 所有现有 Engine CI 检查继续通过
