## PRD - 实现 Brain tick 48h 简报定时检查与 cortex 生成逻辑

**需求来源**: Brain 自动调度任务
**功能描述**: 在 tick.js 中添加 48h 简报定时检查，超过时间后触发 cortex.js 生成系统简报，并通过 API 端点支持手动触发和查询
**涉及文件**:
- `packages/brain/src/tick.js` - 添加 48h 定时检查逻辑
- `packages/brain/src/cortex.js` - 添加 generateSystemReport() 函数
- `packages/brain/src/server.js` - 添加 API 端点
- `packages/brain/migrations/` - 创建 system_reports 表
**成功标准**:
1. tick.js 每次执行都检查 48h 条件
2. cortex 能成功生成包含完整信息的简报
3. 简报内容包含：时间范围、任务统计、资源状态、OKR 进展
4. 数据库正确存储简报
5. GET /api/brain/reports/latest 返回最新简报
6. POST /api/brain/reports/generate 手动触发成功
7. 单元测试覆盖核心逻辑
**非目标**: 不做推送通知、不做简报 UI 展示页面

### 具体需求

#### 1. tick.js 定时检查逻辑
- 在 tick.js 中添加 48h 简报检查逻辑
- 从数据库读取 last_report_time（存储在 system_reports 表）
- 判断距离上次简报是否超过 48h（48 * 60 * 60 * 1000 ms）
- 超过时间后触发 cortex 生成简报

#### 2. cortex 简报生成功能
- 在 cortex.js 中添加 generateSystemReport() 函数
- 收集过去 48h 的关键数据：完成的任务数量和类型、失败/隔离的任务、警觉等级变化、资源使用情况、OKR 进展
- 使用 Sonnet 生成简洁的简报（中文，Markdown 格式）
- 返回生成的简报内容和时间戳

#### 3. 数据库支持
- 创建 system_reports 表
- 字段：id, report_type, content, stats, generated_at, created_at
- 存储简报内容、生成时间、统计数据

#### 4. API 端点
- POST /api/brain/reports/generate - 手动触发生成简报
- GET /api/brain/reports/latest - 获取最新简报

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| LLM 调用失败 | 重试1次，仍失败则记录错误并跳过本次简报 | P2 | report_generation_failed |
| 数据库查询失败 | 记录错误，降级为空统计 | P2 | report_db_error |
| tick 中简报触发异常 | 捕获异常，不影响主 tick 流程 | P2 | report_tick_error |
