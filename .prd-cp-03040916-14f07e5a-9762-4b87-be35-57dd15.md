## PRD - 实现 Brain tick 48h 简报定时检查与 cortex 生成逻辑

**需求来源**: Brain 自动调度任务，Initiative「48h简报自动化调度集成」首个开发 PR
**功能描述**: 在 tick.js 中添加 48h 简报定时检查，在 cortex.js 中实现 generateSystemReport() 函数
**涉及文件**:
- `packages/brain/src/tick.js` - 添加 48h 检查逻辑
- `packages/brain/src/cortex.js` - 添加 generateSystemReport() 函数
- `packages/brain/src/server.js` - 新增 /api/brain/reports/* 路由
- `packages/brain/src/db/migrations/` - 新增 system_reports 表迁移
**成功标准**: tick 每次执行检查 48h 条件，cortex 成功生成中文 Markdown 简报，数据库正确存储，API 端点可用
**非目标**: 不实现简报推送（飞书）、不实现 Dashboard 展示、不实现简报订阅功能

### 具体需求

#### 1. tick.js 定时检查逻辑
- 在 tick.js 中添加 48h 简报检查逻辑
- 从 system_reports 表读取 last_report_time
- 判断距离上次简报是否超过 48h（48 * 60 * 60 * 1000 ms）
- 超过时间后触发 cortex 生成简报

#### 2. cortex 简报生成功能
- 在 cortex.js 中添加 generateSystemReport() 函数
- 收集过去 48h 的关键数据：完成的任务数量和类型、失败/隔离的任务、警觉等级变化、资源使用情况、OKR 进展
- 使用 Sonnet 生成简洁的简报（中文，Markdown 格式）
- 返回生成的简报内容和时间戳

#### 3. 数据库支持
- 创建 system_reports 表
- 字段：id, content, report_type, period_start, period_end, stats_json, created_at

#### 4. API 端点
- POST /api/brain/reports/generate - 手动触发
- GET /api/brain/reports/latest - 获取最新简报

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| cortex 生成简报失败 | 记录错误，跳过本次（下次 tick 重试） | P2 | report_generation_failed |
| 数据库写入失败 | 记录错误日志，不影响主流程 | P2 | report_db_write_failed |
| 数据收集部分失败 | 用可用数据生成简报，标注缺失 | P3 | report_data_partial |
