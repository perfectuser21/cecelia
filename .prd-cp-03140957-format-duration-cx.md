# PRD: format_duration_ms 亚秒边界修正

## 需求来源

- `/dev --task-id 337c1602-b2ce-484a-850d-c1e81ac5f4a4`
- 原始任务目标：提供 `packages/engine/lib/format-duration.sh`，把毫秒值格式化为人类可读字符串

## 背景

- `format-duration.sh` 的主体能力已经在 `main` 中存在
- 当前实现对 `1ms` 到 `99ms` 的非零输入会输出 `0s`
- 这会把真实的非零耗时错误压平，丢失亚秒信息

## 功能描述

- 保持现有天/小时/分钟/秒格式不变
- 修正 `format_duration_ms()` 的亚秒边界：任何 `>0ms && <100ms` 的输入都至少显示 `0.1s`
- 补齐 shell 与 TypeScript 测试，覆盖 `<100ms` 非零输入

## 涉及文件

- `packages/engine/lib/format-duration.sh`
- `packages/engine/tests/format-duration.test.sh`
- `packages/engine/tests/format-duration.test.ts`

## 成功标准

- [BEHAVIOR] `format_duration_ms 50` 返回 `0.1s`
- [BEHAVIOR] `format_duration_ms 1` 返回 `0.1s`
- [BEHAVIOR] 现有示例输出保持兼容：`500ms -> 0.5s`、`65000ms -> 1m 5s`
- [GATE] shell 与 TypeScript 测试通过

## 非目标

- 不改动分钟以上的格式规则
- 不引入新的命令行入口或依赖
