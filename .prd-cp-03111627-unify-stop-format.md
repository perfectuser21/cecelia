# PRD: 统一 Stop Hook 状态文件格式

## 背景

Stop hook (`stop-dev.sh`) 目前支持两种状态文件格式：
1. **旧格式**（无后缀）：`.dev-mode`、`.dev-lock`、`.dev-sentinel`
2. **新格式**（per-branch 后缀）：`.dev-lock.<branch>`、`.dev-mode.<branch>`、`.dev-sentinel.<branch>`

旧格式不支持并行多个 /dev 会话，且与新格式共存导致 ~200 行兼容代码。
更严重的是，`stop-dev.sh` 的 D6-1 预检只扫描 `.dev-lock.*` 文件，
当会话使用旧格式（只有 `.dev-mode` 无 `.dev-lock.*`）时，预检直接 `exit 0`，
导致会话过早退出。

## 目标

删除所有旧格式兼容代码，只保留 per-branch 格式，大幅简化 `stop-dev.sh`。

## 范围

### 需要修改的文件
1. `packages/engine/hooks/stop-dev.sh` — 删除旧格式兼容代码（~200 行）
2. `packages/engine/hooks/stop.sh` — 删除旧格式 fallback（lines 49-52）
3. `hooks/stop-dev.sh` — 根目录副本，同步更新
4. `hooks/stop.sh` — 根目录副本，同步更新
5. `packages/engine/skills/dev/steps/03-branch.md` — 清理旧格式引用

### 不需要修改的文件
- Brain 代码
- CI 配置
- 其他 hooks

## 成功标准

1. `stop-dev.sh` 只识别 per-branch 格式（`.dev-lock.<branch>`、`.dev-mode.<branch>`、`.dev-sentinel.<branch>`）
2. 旧格式状态文件（`.dev-mode`、`.dev-lock`、`.dev-sentinel` 无后缀）被完全忽略
3. 并行多个 /dev 会话正常工作（每个会话有独立的 per-branch 状态文件）
4. `stop-dev.sh` 行数减少 150+ 行
5. `hooks/` 根目录副本与 `packages/engine/hooks/` 保持同步
