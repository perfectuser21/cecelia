## 目标

扩展 model-registry.js + llm-caller.js，支持4类执行路径，每个 agent 支持 default/optional/fallback 三档配置。

## 4类 Provider

1. `anthropic` — Claude Code 无头（已有，确认模型列表）
2. `anthropic-api` — Anthropic 直接 API（已有，不变）
3. `openai` — OpenAI 直接 API（已有，补充注册 gpt-5.4、gpt-5.4-pro、gpt-5.4-nano、o3）
4. `codex` — Codex 无头（缺失，需新增）：gpt-5.4、gpt-5.4-mini、gpt-5.3-codex、gpt-5.2、gpt-5.1-codex-max、gpt-5.1-codex-mini

## 改动范围

### model-registry.js
- 新增 codex provider 的模型注册
- 补充 OpenAI API 缺失模型
- 每个 agent 新增 default_model/fallback_model 字段（可选，向后兼容）

### llm-caller.js
- 新增 callCodexHeadless(prompt, model, options) — 调 `codex exec --model <model>` subprocess
- provider=codex 时走 Codex 无头路径
- 三档降级逻辑：default(无头) → optional → fallback(API)

## 成功标准
- reflection agent 可通过 API 切换到 codex provider（gpt-5.4-mini），不再调 Opus
- llm-caller 支持 provider=codex 走无头 subprocess
- 现有 anthropic/openai/minimax 路径不受影响
