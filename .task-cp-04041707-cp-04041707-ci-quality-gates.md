# Task Card: CI 质量门禁补齐

## 需求（What & Why）
**功能描述**: 在 CI 里加两个主动拦截检查，防止幽灵引用和孤岛引用持续积累
**背景**: V13→V14 重设计时删掉了全部 devgate 体系，现在每次 /dev PR 都可能带入死引用，只能靠人工 audit sprint 清理
**不做什么**: 不恢复 V13 的 32-job CI，不引入重型静态分析工具

## 成功标准

- regression-contract.yaml 里引用不存在测试文件时 CI 自动失败
- /dev Stage 4 有完工清理检查脚本可调用
- engine-tests job 包含两个新步骤

## 验收条件（DoD）

- [x] [ARTIFACT] `packages/engine/ci/scripts/check-contract-refs.sh` 存在且可执行
  Test: manual:node -e "const fs=require('fs');const s=fs.statSync('packages/engine/ci/scripts/check-contract-refs.sh');if(!s.isFile())process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] check-contract-refs.sh 对不存在的测试文件引用返回非零退出码
  Test: manual:node -e "const {execSync}=require('child_process');try{execSync('bash packages/engine/ci/scripts/check-contract-refs.sh',{stdio:'pipe'})}catch(e){console.log('OK — correctly fails on missing refs')}"

- [x] [ARTIFACT] `packages/engine/skills/dev/scripts/check-cleanup.sh` 存在且可执行
  Test: manual:node -e "const fs=require('fs');const s=fs.statSync('packages/engine/skills/dev/scripts/check-cleanup.sh');if(!s.isFile())process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] ci.yml engine-tests job 包含 Contract Refs Check 步骤
  Test: manual:node -e "const c=require('fs').readFileSync('.github/workflows/ci.yml','utf8');if(!c.includes('check-contract-refs'))process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] 04-ship.md 包含完工清理检查调用
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/skills/dev/steps/04-ship.md','utf8');if(!c.includes('check-cleanup'))process.exit(1);console.log('OK')"

## 实现方案

**要改的文件**:
1. 新建 `packages/engine/ci/scripts/check-contract-refs.sh` — 扫描 regression-contract.yaml 里的 `test: "tests/..."` 路径，逐一验证文件存在
2. 修改 `.github/workflows/ci.yml` — engine-tests job 里加 "Contract Refs Check" 步骤
3. 新建 `packages/engine/skills/dev/scripts/check-cleanup.sh` — 检查孤岛引用（.sh 文件里 source/bash 调用的路径是否存在）
4. 修改 `packages/engine/skills/dev/steps/04-ship.md` — Stage 4 加一步调用 check-cleanup.sh
