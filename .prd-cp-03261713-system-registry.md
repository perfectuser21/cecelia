# PRD: system_registry 表 + /api/brain/registry 接口

## 背景

Cecelia 系统现在有 73 个 skill、多个 registry 文件、cron job、API 端点、机器节点，但没有统一的"这个系统里有什么"的查询入口。导致 Claude 每次创建新东西都可能产生重复（今天已发生：重复创建了 skill-index.md）。

## 目标

在 Brain 加一张 `system_registry` 表，作为系统全局目录（类似 ClawHub）。任何东西创建前先查这里，创建后登记进来。

## 成功标准

1. Brain DB 有 `system_registry` 表，含 type/name/location/description/status/depends_on/metadata 字段
2. `GET /api/brain/registry` 支持 `?type=skill` 等过滤查询
3. `GET /api/brain/registry/exists?type=skill&name=xxx` 返回 `{exists: bool, item?}`
4. `POST /api/brain/registry` 登记新条目
5. `PATCH /api/brain/registry/:id` 更新状态
6. 初始数据：把现有 skill（从 .agent-knowledge/skills-index.md 读取）批量写入
