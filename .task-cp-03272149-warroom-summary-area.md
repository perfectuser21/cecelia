# Task Card: War Room 改为 Summary + Area 详情页架构

## 需求（What & Why）

**功能描述**：将 `/gtd/warroom` 页面重构为两层架构：
1. 总览页（`/gtd/warroom`）：Vision 横幅 + Area 卡片列表（Cecelia/ZenithJoy/Investment 等有活跃 OBJ 的 Area），每张卡片显示活跃 OBJ 数、进行中任务数、KR 整体进度，点击跳转详情页。
2. Area 详情页（`/gtd/warroom/:area`）：该 Area 下的 OBJ → KR → 进行中 Task，三列横向布局，各列独立滚动。

**背景**：现有 War Room 把所有数据混在一起（所有 Area 的 OBJ 全部混列），信噪比低，无法聚焦到单个业务线。

**不做什么**：不改后端 API；不改其他 GTD 页面；不重构公共组件；不添加编辑功能。

## 实现方案

**要改的文件**：
- `apps/api/features/gtd/pages/GTDWarRoom.tsx`（重写为总览页）
- `apps/api/features/gtd/pages/GTDWarRoomArea.tsx`（新建 Area 详情页）
- `apps/api/features/gtd/index.ts`（新增路由和组件）
- `apps/api/features/gtd/__tests__/warroom-area.test.ts`（新建单元测试）

**API**：
- 总览页：`GET /api/tasks/full-tree?view=okr` → Vision→Area→OBJ→KR 树
- 详情页：`GET /api/tasks/full-tree?view=okr` + area slug 过滤，`GET /api/brain/tasks?status=in_progress&limit=20`

**Area Slug 规则**：`area.id`（URL-safe UUID 不够可读，改用 area.name.toLowerCase().replace(/\s+/g, '-').replace(/[^a-z0-9-]/g, '')` → "Cecelia" → "cecelia"，"ZenithJoy" → "zenith-joy"）

**路由**：`/gtd/warroom/:area` — React Router `useParams()` 取 `area` 参数，匹配 full-tree 中 area slug
