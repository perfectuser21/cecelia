# PRD: 修复 023 migration idx_run_events_task_id 不幂等

## 背景

smoke.test.js 在 CI 中启动真实 Brain server 时，server migrate 失败导致进程退出：

```
[FAIL] 023_add_run_events_observability_v1.1.sql: relation "idx_run_events_task_id" already exists
[smoke] Server exited with code 1
```

## 问题

`023_add_run_events_observability_v1.1.sql` 中所有 `CREATE INDEX` 语句都没有 `IF NOT EXISTS` 保护。CI 数据库（PostgreSQL service）在跑多次测试后已存在这些索引，重复执行 migration 时报错，server 启动失败。

## 目标

为 023 migration 中所有 `CREATE INDEX` / `CREATE UNIQUE INDEX` 语句加上 `IF NOT EXISTS`，使 migration 幂等，可以被安全地重复执行。

## 成功标准

1. [BEHAVIOR] 023 migration 可重复执行，不因 "already exists" 报错
2. [ARTIFACT] 023 migration 所有 CREATE INDEX 均含 IF NOT EXISTS
3. [GATE] CI L3 smoke.test.js 通过
