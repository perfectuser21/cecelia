# Task Card: Capture Atom Review UI

## 需求（What & Why）

**功能描述**: 在 /gtd/inbox 页面增加 Atom Review 面板，让用户审阅 Brain Digestion Job 拆解出的 capture_atoms。用户可以确认（confirm）或驳回（dismiss）每个 atom，按 target_type 筛选，查看 AI 置信度和分类理由。

**背景**: Capture Digestion 系统的前三个 PR 已完成：
- PR1: captures 表 + QuickCapture UI
- PR2: capture_atoms + life_events 表 + CRUD API
- PR3: Brain Digestion Job（AI 拆解 inbox captures 为 atoms）

现在缺少最后一环：用户审阅界面。capture_atoms 状态为 pending_review 的记录需要人工确认后才能路由到目标表。

**不做什么**:
- 不做 routing 逻辑（confirm 后写入目标表的逻辑留给后续 PR）
- 不改 Brain Digestion Job
- 不改数据库 schema

## 成功标准

- GTDInbox 页面新增 "Atom Review" Tab/面板，展示 pending_review 状态的 capture_atoms
- 每个 atom 卡片显示 content、target_type、target_subtype、confidence 百分比、ai_reason
- 支持按 target_type 筛选（note/knowledge/content/task/decision/event + 全部）
- 每个 atom 有 confirm 和 dismiss 两个操作按钮，调用 PATCH /api/capture-atoms/:id
- 操作后自动刷新列表
- 空状态提示"暂无待审阅的 Atom"
- 现有的 QuickCapture + Inbox 列表功能不受影响
