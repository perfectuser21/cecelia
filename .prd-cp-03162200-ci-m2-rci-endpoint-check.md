# PRD: CI M2 — 新 Brain API Endpoint 必须有 RCI 回归契约

## 背景

仓库中 `regression-contract.yaml` 目前为空（core: [], golden_paths: []），而 `packages/brain/src/routes.js` 有 264 个路由端点。新功能 PR 添加新路由时，没有任何机制强制要求同步补充回归契约条目，导致 API 覆盖率持续恶化。

## 目标

在 L2 Consistency Gate（brain-l2 job）新增检查：如果 PR 在 `packages/brain/src/routes*` 中添加了新的 `router.get/post/patch/delete/put` 路由，必须同时在 `regression-contract.yaml` 中添加对应的契约条目（`ci:` 或 `hooks:` 或 `workflow:` 节下）。

## 范围

1. 新脚本：`scripts/devgate/check-new-api-endpoints.mjs`
2. L2 brain-l2 job 调用该脚本

## 成功标准

1. [ARTIFACT] `scripts/devgate/check-new-api-endpoints.mjs` 文件存在
2. [ARTIFACT] `ci-l2-consistency.yml` 的 brain-l2 job 调用 check-new-api-endpoints.mjs
3. [BEHAVIOR] 脚本在无新路由 diff 时可执行并退出码为 0

## 非目标

- 不对现有 264 个路由进行追溯检查（仅检查 PR 新增的路由）
- 不修改 regression-contract.yaml 的现有结构
