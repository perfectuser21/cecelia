# Task Card: execution.js KR 断链修复 — strategy_session INSERT key_results 补 objective_id

## 需求（What & Why）

**功能描述**: `strategy_session` 任务完成后，执行回调解析 KR JSON 并写入 `key_results` 表，但插入时没有传 `objective_id`，导致创建的 KR 成为孤岛（没有挂到任何 Objective 下）。

**背景**: OKR 表结构迁移（PR1-PR9）已完成，新表 `key_results.objective_id REFERENCES objectives(id)`。没有 `objective_id` 的 KR 在 OKR 树中不可见，调度器无法通过 Objective → KR 路径发现它们。

**根本原因**: 
- `execution.js:1013` INSERT INTO key_results 只传了 `title/status/owner_role/metadata`，没有 `objective_id`
- 任务的 `goal_id` 字段（若有）指向关联的 Objective，可用来关联

**不做什么**: 
- 不改 strategy_session skill 的输出格式
- 不强制要求 objective_id NOT NULL（保持 nullable）
- 不修改 execution.js 其他部分

## 成功标准

1. [BEHAVIOR] strategy_session 任务完成后，创建的 key_results 记录带有 objective_id（当任务有 goal_id 时）
2. [BEHAVIOR] 当任务没有 goal_id 时，key_results 插入成功（objective_id = null），并记录 WARN 日志
3. [GATE] CI 全部通过（L1-L4）

   ```js
   let krObjectiveId = null;
   if (ssTaskRow.goal_id) {
     const objCheck = await pool.query('SELECT id FROM objectives WHERE id = $1', [ssTaskRow.goal_id]);
     if (objCheck.rows.length > 0) krObjectiveId = ssTaskRow.goal_id;
   }
   if (!krObjectiveId) {
     console.warn('[execution-callback] strategy_session KR 无关联 Objective — 将成为孤岛');
   }
   ```
