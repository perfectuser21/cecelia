# Task Card: CI Gate — manual: 命令白名单本地前置拦截

## 目标

在 verify-step.sh 本地执行时，拦截 `manual:` 命令中使用非白名单工具的测试，
防止用户在本地通过后到 CI 才发现命令不允许（当前 CI 白名单：node/npm/curl/bash/psql）。

## 成功标准

1. 新增 `check-manual-cmd-whitelist.cjs`，导出 `scanManualViolations(content)`
2. `verify-step.sh` 集成白名单检查，本地即可拦截非白名单 manual: 命令
3. CI L1 新增 `manual-cmd-whitelist-check` job（stage 2+）
4. 测试文件覆盖率 ≥ 60%

## 实现方案

### 新增脚本：check-manual-cmd-whitelist.cjs

- 扫描 Task Card 中所有 `Test: manual:<cmd>` 行
- 提取 `<cmd>` 的第一个单词（顶层命令）
- 只允许：`node` / `npm` / `npx` / `curl` / `bash` / `psql`
- 拒绝：`grep` / `ls` / `cat` / `find` / `sed` / `awk` 等非白名单命令
- 导出 `scanManualViolations()` 函数供测试 import（参照 check-fake-dod-tests.cjs 模式）
- Exit 0 = 通过，Exit 1 = 违规

### verify-step.sh 扩展

- 在已有假测试检查之后，新增 `manual:` 命令白名单检查
- 调用 `check-manual-cmd-whitelist.cjs` 检测 Task Card
- 给出清晰错误：`请用 manual:node -e "..." 替代 manual:<forbidden-cmd>`

### CI L1 新增 job

- `manual-cmd-whitelist-check` job
- 在 stage 2+ 触发（同 fake-dod-test-check）
- 使用共享脚本 check-manual-cmd-whitelist.cjs
- 三重降级保护（无 .dev-mode / 无字段 / 无文件 → 跳过）
