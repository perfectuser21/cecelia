# Task Card: 重构 executeCopywriting（圈复杂度 26 → 10 以下）

## 需求（What & Why）
**功能描述**: 重构 `packages/brain/src/content-pipeline-executors.js` 中的 `executeCopywriting` 函数，将圈复杂度从 26 降至 10 以下。
**背景**: 代码复杂度扫描触发自动调度任务。高复杂度函数难以维护、测试和理解。
**不做什么**: 不改变函数对外 API 签名和返回值结构；不改其他 executor 函数。

## 实现方案

### 要改的文件
- `packages/brain/src/content-pipeline-executors.js`（第 198-299 行）

### 重构策略：提取 5 个子函数

| 子函数 | 职责 | 消除的复杂度来源 |
|--------|------|----------------|
| `_filterTopFindings(findings)` | findings 过滤 + 降级逻辑 | 2 个 if 分支 |
| `_buildCopywritingPrompt(keyword, top, typeConfig, previousFeedback)` | 构建 LLM prompt 字符串 | 1 个 if 分支 + 字符串拼接 |
| `_executeLLMPath(keyword, top, typeConfig, previousFeedback, dir)` | LLM 调用 + 文件写入 | try/catch + 正则匹配 + fallback |
| `_buildStaticSocialCopy(keyword, top)` | 静态社交文案模板 | map 回调中的三元表达式 |
| `_buildStaticArticle(keyword, top)` | 静态公众号长文模板 | map 回调中的三元表达式 |

重构后 `executeCopywriting` 主体：setup → 加载 findings → filter → LLM path or static path → return，复杂度约 4。
