## PRD - 补全 execution-callback 错误捕获：静默失败转为有据可查

**需求来源**: Brain 自动调度（Desire ID: 84e2b426-4660-4b90-8357-443a13b03fc0）
**功能描述**: execution-callback 接口接收 Agent 执行结果时，错误被静默吞掉（catch 后只打 log，不持久化）。导致任务 error_message 字段长期为 null，Cortex 反思时看不到任何失败细节，RCA 变成「看不见病灶的手术」。
**涉及文件**:
- packages/brain/src/routes.js（execution-callback 路由）
## 成功标准

- Agent 以非零 exit code 完成时，error_message 不为 null
- blocked_detail 包含 { exit_code, stderr_tail, timestamp } 结构
- callback handler 无静默 catch
- 集成测试：构造失败 callback，断言 DB 字段被正确填充
**非目标**: 不修改任务调度逻辑，不重构 callback 整体结构，只补全错误字段写入

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| exit code 非零 | 写 error_message + blocked_detail | P2 | agent_execution_failed |
| stderr 为空但 exit 非零 | blocked_detail.stderr_tail = "" | P3 | agent_exit_nonzero_no_stderr |
