# Task Card: /dev 加入 TDD 两阶段探索 + 任务评分

## 需求（What & Why）

**功能描述**: 给 /dev workflow 加入两项机制：
1. TDD 两阶段探索：把原来"先看代码再写DoD"改为"浅扫接口→Test Designer锁定测试→深度探索实现"
2. 任务评分：Step 1 输出置信度自评，Step 5 输出四维执行质量分

**背景**:
- 当前 /dev 先探索代码再写 DoD 存在 accommodation bias（倾向写容易通过的测试）
- 任务完成后只有 completed/failed，看不到质量，无法优化

**不做什么**: 不改 Brain DB schema、不改 Dashboard 显示、不改 Stop Hook 逻辑

## 成功标准

1. [ARTIFACT] 02-code.md 有明确三段式：浅扫（≤5文件）→ Test Designer Subagent → 深度探索
2. [ARTIFACT] Test Designer Subagent prompt 明确禁止读实现代码，要求测试必须当前 FAIL
3. [ARTIFACT] 01-taskcard.md Step 1 末尾有置信度输出块（写入 .dev-mode）
4. [ARTIFACT] 05-clean.md 有四维评分计算 + 输出格式 + Brain API PATCH 写入
5. [GATE] CI 全部通过
