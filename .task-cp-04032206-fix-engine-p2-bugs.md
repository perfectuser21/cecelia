# Task Card: Engine P2 Bug 修复（2 个验证真实 Bug）

## 需求（What & Why）
**功能描述**: 修复 2 个 P2 级真实 bug（另外 3 个"P2"经验证均为假报告，不改）
**背景**: 多 agent 扫描后逐行验证，P2 中只有这 2 个确实存在
**不做什么**: 不改 bash-guard.sh（--message 已处理）、_mark_cleanup_done flock（顺序正确）、affected-packages.js（Node.js stderr 是同步流）

## 修复清单

### Bug 1（P2）: devloop-check.sh — jq 缺少错误处理导致 stderr 噪音
- **文件**: `packages/engine/lib/devloop-check.sh:171-173`
- **问题**: `jq -r '.[0].status // "unknown"'` 等无 `2>/dev/null` 和 fallback，`gh run list` 返回非 JSON 时 jq 报错到 stderr，且 `ci_status` 为空触发 `*` 分支误报"CI 状态未知"
- **修复**: 加 `2>/dev/null || echo "fallback_value"`

### Bug 2（P2）: stop.sh — 后台 curl 缺 disown，SIGHUP 可能杀死通知
- **文件**: `packages/engine/hooks/stop.sh:79-81`
- **问题**: `curl ... &` 启动后台进程但未 `disown`，shell 收到 SIGHUP 时会向所有子进程发送 SIGHUP，可能在 curl 完成前杀死它，导致 Brain 会话结束通知丢失
- **修复**: 添加 `disown $! 2>/dev/null || true`
