## 背景

`pipeline-trace.sh` 当前每个 Stage 只输出一行摘要，用户无法从中看到 pipeline 执行的细节：
- Stage 1 有 spec seal 的 timestamp、reviewer、summary 但不显示
- Stage 2 有 generator 的 files_modified 列表、build_status 但不显示
- Stage 2 有 CRG seal 的 reviewer 但不显示
- Stage 3 无 PR number（只显示完整 URL）
- Stage 4 无 Learning RCA 摘录
- Planner seal 完全未显示

---

## 成功标准

运行 `pipeline-trace.sh <branch>` 后，每个 Stage 下显示 2-4 行详细字段，包含以下信息：

- **Stage 0**: started 时间（已有）
- **Stage 1**: spec verdict + divergence_count（已有）+ **timestamp + reviewer + summary 首行**
- **Stage 2**: crg verdict（已有）+ **crg reviewer + build_status + files_modified 数量/列表**
- **Stage 3**: PR URL（已有）+ **PR #number 提取** + CI status（已有）
- **Stage 4**: learning path（已有）+ cleanup（已有）+ **Learning RCA 首行摘录**
- **Planner**: 新增一行显示 planner seal status 和 sealed_by
