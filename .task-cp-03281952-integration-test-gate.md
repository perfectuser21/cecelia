# Task Card: CI L3 集成测试门禁 — Brain↔Engine↔API 跨模块验证

## 需求（What & Why）

**功能描述**：
1. 在现有集成测试（`apps/api/src/__tests__/brain-api-integration.test.ts`）基础上，补充 Brain API 健康检查 + 关键端点存在性契约测试，以及 Engine Hook → Brain API 联动契约测试（验证 session-start.sh 调用的 API 格式）。
2. 在 CI L3 (`ci-l3-code.yml`) 增加一个新 check：PR 改动了 `packages/brain/src/` 或 `apps/api/` 时，对应目录必须有测试文件变动。Brain 改动但无测试 → warning（不阻塞，输出明确提示）。

**背景**：
- 现有只有单元测试，改 A 不知道 B 坏了
- `apps/api/`、`apps/dashboard/` 完全没有测试扫描
- Brain 有 10/151 模块无测试
- CI L3 只要求 `feat:` PR 有 `*.test.ts`，单元测试就够，没有跨模块要求

**不做什么**：
- 不强制要求 dashboard 测试（复杂度高，这次只加 warning）
- 不依赖本地 Brain 服务运行（测试必须在 CI ubuntu-latest 通过）
- 不做 apps/api/ 的复杂端到端测试

## 成功标准

- `apps/api/src/__tests__/brain-api-integration.test.ts` 补充了 Brain API 契约测试（session-start 联动、context 端点存在性）
- `packages/brain/src/__tests__/integration/` 下新增 `brain-endpoint-contracts.test.ts`，验证关键端点的请求/响应格式契约（用 mock，不依赖真实服务）
- CI L3 新增 `brain-test-coverage-warning` job：改动了 `packages/brain/src/` 但无测试文件变动时输出 warning（exit 0 不阻塞）
- 所有新测试在 CI ubuntu-latest 无需真实服务即可通过
