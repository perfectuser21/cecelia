# PRD: 修复 Sprint Contract Gate — Evaluator 独立测试计划强制执行

## 背景

spec_review 的 independent_test_plans 在 12/18 次任务中为 0，Evaluator 没有独立工作就直接 PASS，合约机制形同虚设。

根因分析：
1. SKILL.md 中 `divergence_count == 0 → exit 2` 是 shell 语法，Agent subagent 无法真正执行 exit 2，导致检查形同虚设
2. 主 agent 在验证 PASS 时不检查 `independent_test_plans.length > 0`
3. Task Card 传入 spec_review 时包含 Test 字段，Evaluator 可直接复制而非独立生成
4. seal 文件无 `reviewer_model` 字段

## 修复目标

1. Evaluator 先读需求描述（不看 Generator 的测试计划），独立生成测试计划
2. 再与 DoD 比对，差异超阈值 = FAIL
3. CRG 换用更严格的 prompt，有 issues 必须触发更严格判断
4. reviewer_model 字段必须正确记录

## 成功标准

- [BEHAVIOR] spec_review 的 independent_test_plans 在每次 /dev 执行时均 > 0
- [BEHAVIOR] divergence_count = 0 时 verdict = FAIL（不再允许 exit 2 形式）
- [BEHAVIOR] reviewer_model 字段在 seal 文件中正确记录
- [GATE] CI 全部通过
