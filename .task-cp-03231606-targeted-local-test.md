# Task Card: 02-code.md 本地测试改为精准测试（去全量，去重复）

## 需求（What & Why）

**功能描述**: 修改 `packages/engine/skills/dev/steps/02-code.md`，将两处 `npm test`（全量）合并为一处精准测试（仅跑改动相关文件）。

**背景**: 当前 Stage 2 在 2.3.1 和 2.3.7 各跑一次全量 `npm test`。Brain 有 100+ 个测试，全量运行会导致 Mac mini 内存不足（OOM），机器崩溃重启。传统团队做法是本地只跑与改动相关的测试文件，全量回归交给 CI（GitHub Actions ubuntu-latest）。

**不做什么**: 不修改 CI 流程；不修改 TDD 红绿灯阶段（那个跑单个文件，是合理的）。

2. [ARTIFACT] `02-code.md` 的 2.3.7 节删除重复的 `npm test` 全量，不再独立运行全量测试

## 实现方案

**文件**: `packages/engine/skills/dev/steps/02-code.md`

**改动一：2.3.1 精准测试**

将原来的：
```bash
if [[ "$HAS_TEST" == "yes" ]]; then
    npm test
fi
```

改为：查找与改动相关的测试文件，只跑这些文件。推导逻辑：
- 改动文件 `packages/brain/src/foo.js` → 找 `packages/brain/src/__tests__/foo.test.js`
- 找到 → `npx vitest run <file>`
- 找不到 → 跳过（全量由 CI 负责）

**改动二：2.3.7 删除重复 npm test**

2.3.7 原来的「跑 npm test」步骤已被 2.3.1 精准测试覆盖，删除该行，只保留 check-learning 和 check-dod-mapping 两步。
