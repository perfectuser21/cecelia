# PRD: 记忆蒸馏文档层 Layer 2（SOUL/SELF_MODEL/USER_PROFILE/WORLD_STATE）

## 背景

Cecelia 当前记忆系统缺失 Layer 2（蒸馏文档层）。每次对话靠向量检索召回原始记录，
无任何保证核心身份的锚点。OpenClaw 用 SOUL.md 解决这个问题：核心性格/价值观每次
必注入，不管数据库出什么问题她的身份都不会丢。

## 目标

新增 4 个蒸馏文档，按以下策略注入：
- **SOUL**：核心性格/价值观/行为准则，永不过期，每次必注入（预算外）
- **SELF_MODEL**：当前能力边界/最新洞察摘要，从 learnings 提炼，每周更新
- **USER_PROFILE**：用户是谁/偏好/沟通风格，从 user_profile_facts 提炼，每次对话后更新
- **WORLD_STATE**：活跃 OKR/项目状态/关键事实，每天更新

## 技术实现

1. Migration 175: distilled_docs 表（type, content, updated_at, version, generated_by）
2. distilled-docs.js: getDoc(type) + seedSoul() + refreshSelfModel() + refreshUserProfile() + refreshWorldState()
3. buildMemoryContext 修改: 在 block 最前面固定注入 SOUL + SELF_MODEL（预算外），chat 模式追加 USER_PROFILE + WORLD_STATE
4. server.js: 启动时 seed SOUL（若不存在），WORLD_STATE 每天更新 cron
5. selfcheck.js: EXPECTED_SCHEMA_VERSION → '175'
6. DEFINITION.md: schema_version → 175

## 成功标准

1. distilled_docs 表存在，可写入 4 种 type（SOUL/SELF_MODEL/USER_PROFILE/WORLD_STATE）
2. Brain 启动时自动 seed SOUL 文档（若不存在）
3. buildMemoryContext 返回的 block 在任何模式下都包含 SOUL 内容
4. chat 模式下 block 额外包含 SELF_MODEL + USER_PROFILE + WORLD_STATE
5. selfcheck.js EXPECTED_SCHEMA_VERSION = '175'，migrations/175 存在
6. 所有现有测试继续通过（无回归）
