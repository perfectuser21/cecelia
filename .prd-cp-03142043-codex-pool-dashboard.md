# PRD：Codex 并发池 + budgetState Dashboard 可视化

## 背景

PR #950 合并了动态 Token 预算调度器，实现了 Claude Code 的 7 天预算感知和 Codex 降级路由。但有两个下游缺口：
1. slot-allocator 里没有 Codex 专属并发上限，大量任务降级时可能同时打爆 Codex
2. budgetState（abundant/moderate/tight/critical）只在 API 层可见，Dashboard 没有展示

## 目标

1. **Codex 并发池**：在 slot-allocator 中新增 Codex 专属槽位计数，限制同时运行的 Codex 任务数，防止 quota exceeded 风暴
2. **budgetState Dashboard 可视化**：在 Codex Dashboard 或 Brain Status 页面展示当前预算状态和 Pool C 缩放比例

## 需求

### Codex 并发池（Brain）

- 新增 `MAX_CODEX_CONCURRENT = 3`（可配置，与 3 个账号对应）
- `slot-allocator.js` 查询当前 `in_progress` 的 codex 任务数，若达上限则 Codex 不可用
- `getSlotStatus()` API 返回新增字段：`codexRunning`、`codexMax`、`codexAvailable`
- `calculateSlotBudget()` 返回 `codexAvailable: boolean`

### budgetState Dashboard 可视化（Dashboard）

- `GET /api/brain/resources` 已返回 `budgetState` 对象（state、pool_c_scale、avg_remaining_pct、account_count）
- 在 Dashboard 的 Codex 或 Resources 区域展示：
  - 当前状态 badge：🟢 abundant / 🟡 moderate / 🟠 tight / 🔴 critical
  - Pool C 缩放：例如 "70% (moderate)"
  - Codex 并发：例如 "2/3 running"

## 成功标准

- slot-allocator 在 3 个 codex 任务都在跑时，`codexAvailable` 返回 false，新任务等待而不是第 4 个并发开跑
- Dashboard 页面能看到 budgetState badge（4 种颜色状态之一）
- 所有相关单元测试通过
