# PRD: OKR 统一 PR3 — 业务代码迁移 goals/projects → 新 OKR 层级表

## 背景

Migration 179 完成数据迁移，Migration 181 添加双向同步触发器。PR1461/1462 已迁移 tick/routes 层。仍有大量业务文件直接查询旧 goals/projects 表，需迁移到新 OKR 层级表（objectives / key_results / okr_projects / okr_scopes / okr_initiatives）。

## 目标

将以下业务文件的 SELECT 查询从旧 goals/projects 表迁移到新 OKR 层级表：
- `kr-completion.js`：goals WHERE type='area_okr' → objectives
- `kr-progress.js`：goals WHERE type IN → objectives / key_results
- 其他可安全迁移的业务文件

## 约束

- IDs 一致：objectives.id == goals.id（同 UUID），跨表关联仍然有效
- UPDATE/INSERT 保留在旧表（触发器自动同步到新表）
- 不迁移需要 priority/progress/repo_path 等旧表专有字段的文件
- status 值映射：goals.status='in_progress' → objectives.status='active'（查询需用 IN ('active','in_progress')）

## 成功标准

- kr-completion.js、kr-progress.js 的 SELECT 查询切换到新表
- 现有测试全部通过
- CI 全部通过
