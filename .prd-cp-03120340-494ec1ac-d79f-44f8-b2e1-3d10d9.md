## PRD - Cortex insight → action 自动闭合机制

**需求来源**: Brain 自动调度任务（task_type=dev）
**功能描述**: insight 在 learnings 表积累（category=cortex_insight）但 applied=false 永远不变，没有机制把 insight 转化为可执行的 dev task。需要在 cortex.js 记录 insight 后，自动分析是否涉及代码级修复，若是则自动生成对应 dev task（写入 tasks 表），设置 learning_id 关联。
**涉及文件**:
- `packages/brain/src/cortex.js` — 记录 insight 后触发自动分析
- `packages/brain/src/actions.js` — createTask 用于创建修复任务
- `packages/brain/src/db/` 或 migrations — 如需 learning_id 字段
- `packages/brain/__tests__/` — integration test

**成功标准**: cortex_insight 类型的 learning 创建后，若内容含代码修复信号，72h 内自动生成 dev task；已有 task 的 insight 不重复触发；applied=true 标记正确更新；integration test 验证 insight→task 链路。

**非目标**: 不修改 insight 的分类逻辑；不影响现有 cortex 决策流程；不处理非 cortex_insight 类型的 learning。

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| createTask 失败 | 记录日志，下次 tick 重试 | P2 | cortex_insight_task_create_failed |
| insight 内容为空 | 跳过，不创建 task | P3 | cortex_insight_empty_skip |
| 重复检测误判 | 以 learning_id 为 dedup key | P3 | cortex_insight_dedup_skip |
