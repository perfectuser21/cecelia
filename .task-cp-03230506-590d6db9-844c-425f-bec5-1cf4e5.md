# Task Card: OKR体系统一 PR2 — Brain核心代码切新表

## 需求（What & Why）

**功能描述**：将 Brain 核心调度代码（tick.js、task-router.js）中对旧表（goals/projects/project_kr_links）的 SQL 引用替换为新 OKR 表（key_results/okr_projects/okr_scopes/okr_initiatives）。

**背景**：PR1 已完成数据迁移（key_results 14条，okr_projects 56条，okr_initiatives 160条），新旧表 UUID 相同（迁移时保留了 ID），因此切换后 tasks.goal_id 仍能正确关联。okr-closer.js 已使用新表，无需修改。

**关键发现**：
- key_results 有 7 条 active（与 goals.type='area_okr' UUID 重叠）、7 条 archived
- key_results 没有 priority/progress 字段（goals 有），返回时需 null 兜底
- okr_projects 用 title（非 name），查询需 AS name 别名
- task-router.js diagnoseKR 中 initiatives 通过 okr_scopes 中转查询

**不做什么**：不修改 tasks.goal_id 列（向后兼容），不修改 okr-closer.js（已完成）。

## 成功标准

- tick.js 两处 goals 查询切换到 key_results，状态过滤适配新值（active）
- task-router.js diagnoseKR 三处 SQL 切换到新表（key_results/okr_projects/okr_initiatives）
- Brain 服务启动无数据库错误
- CI 全部通过

## 实现方案

**tick.js 改动（2处）**：
1. 行 2160：`goals WHERE type='area_okr' AND status IN ('ready','in_progress','decomposing')` → `key_results WHERE status IN ('active','in_progress','decomposing')`
2. 行 2177：`goals WHERE status NOT IN ('completed','cancelled','canceled')` → `key_results WHERE status NOT IN ('completed','cancelled','archived')`

**task-router.js 改动（diagnoseKR 函数）**：
1. KR 查询（行 604）：`goals WHERE id=$1`（含 priority/progress）→ `key_results WHERE id=$1`（无这两列）
2. Projects 查询（行 617-623）：`projects JOIN project_kr_links` → `okr_projects WHERE kr_id=$1 + title AS name`
3. Initiatives 查询（行 633-641）：`projects WHERE parent_id=$1 AND type='initiative'` → `okr_initiatives JOIN okr_scopes WHERE scope.project_id=$1 + title AS name + okr_initiative_id 计数`
4. 任务计数（行 635-638）：`project_id=i.id` → `okr_initiative_id=i.id`
5. 返回值（行 757-758）：`kr_priority/kr_progress` 用 null 兜底
