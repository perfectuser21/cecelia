# Task Card: Content-Type 配置从 YAML 迁移到数据库

## 需求（What & Why）
**功能描述**: 将 content-type 配置从 YAML 文件迁移到数据库（content_type_configs 表），并提供 CRUD API，让前端可以编辑配置。
**背景**: 当前 content-type 配置硬编码在 YAML 文件中，修改需要改代码。迁移到 DB 后前端可以直接编辑。
**不做什么**: 不删除现有 YAML 文件（保留作为兜底）。不改动 content-pipeline-orchestrator 或其他消费方。

## 成功标准
- content_type_configs 表通过 migration 201 创建，selfcheck EXPECTED_SCHEMA_VERSION 更新为 201
- getContentType() 和 listContentTypes() 改为 DB 优先、YAML 兜底
- 提供 GET/PUT content-types/:type/config 和 POST content-types/seed 三个 API 端点
- 所有现有测试继续通过，新增测试覆盖 DB 优先逻辑和 fallback 逻辑
