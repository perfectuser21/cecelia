# Task Card: F4 — 1:1 复刻 Superpowers 交互点

## 需求
用户诉求：所有 /dev 用的 skill 都应该是 superpowers 的 skill（只优化不替代），所有 7-8 轮交互点都要 1:1 复刻（只是 agent 代答）。
审计发现：skill 覆盖仅 50%（7/14），交互点替代 78% — 未达 1:1。

## 实现方案（5 个子改动一个 PR）

### ① 02-code.md §2.3/2.4 — 引 receiving-code-review
加 ARCHITECTURE_ISSUE 升级分支。Reviewer 发现架构问题 → 报 `STATUS: ARCHITECTURE_ISSUE` → controller 派 architect-reviewer（不让 Implementer 直接改）。

### ② 02-code.md §2.3/2.4 派遣前 prompt — 引 requesting-code-review
Review 请求规范化 5 项：commit SHA / PLAN_OR_REQUIREMENTS / DESCRIPTION / FILES_CHANGED / KNOWN_LIMITATIONS。缺任一 = BLOCKED。

### ③ 02-code.md §2.5 BLOCKED 链 v2 — 引 executing-plans + dispatching-parallel-agents
重新设计 BLOCKED 升级：1 自修 / 2 Spec Reviewer / 3 dispatching-parallel-agents / 4 Brain task 人介入。
取代之前"3 次直接派 systematic-debugging"的设计（后者期望人类已识别失败，不适合自动）。

### ④ 01-spec.md §0.2.5 加 Step 5 Critical Gap Abort — 引 executing-plans
Self-Review 发现 PRD 矛盾 / 核心文件缺失 / DoD Test 语法错 / decision 冲突 → 写 `autonomous_aborted: true` + abort_reason → 创 Brain task 让人介入。

### ⑤ 04-ship.md finish-branch — 引 finishing-a-development-branch
Discard 路径加 typed-confirm。autonomous 模式下不读 stdin 直接 abort + Brain task（安全底线）。

### ⑥ autonomous-research-proxy.md — 新表"交互点替代矩阵 v2"
17 个 superpowers 交互点 + 我们的 Tier 映射 + 状态（完全替代 / 部分替代 / 刻意丢弃 / 保留给人）。

### ⑦ Engine 版本 14.16.0 → 14.17.0（minor）
6 文件同步 + feature-registry changelog。
