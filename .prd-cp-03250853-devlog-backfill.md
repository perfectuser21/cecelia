# PRD: DevLog 数据接通 — dev_records 历史回填 + PR merge 自动写入

## 背景
`dev_records` 表存在但完全为空，DevLog 页面显示空白。
Brain tasks 表中有大量有 `pr_url`/`pr_title`/`branch` 的已完成任务，
但没有机制把这些信息写入 `dev_records`。

## 目标
1. **历史回填**：批量把 tasks 表中有 pr_url 的记录写入 dev_records
2. **自动写入**：修改 pr-callback-handler.js，PR merge 时自动创建/更新 dev_records 条目

## 不做什么
- 不改 dev_records 表结构
- 不回填 prd_content / dod_items（太大，不值得）
- 不改前端（DevLog 页面已有）

## 成功标准
- `SELECT count(*) FROM dev_records` > 50（历史数据）
- pr-callback-handler.js 在找到 task 后写入 dev_records
- DevLog API (`GET /api/brain/dev-records`) 返回有数据
