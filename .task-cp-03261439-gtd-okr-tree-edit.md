# Task: GTD OKR Tree 完整重建：Area分组 + 全链路树 + 编辑功能

## 背景
GTD 页面数据断路问题：
1. areas 表有重复行（Cecelia×2，ZenithJoy×2），前端显示混乱
2. GTDOkr 只展示 Objective→KR 两层，缺少 Area 分组和 Project→Scope→Initiative 下层
3. GTDProjects 的 projects.js 把所有 okr_projects 映射成 type='initiative'，导致页面空白
4. DatabaseView 只读，无法编辑任何字段

## 改动范围
- `apps/api/src/task-system/full-tree.js`（新建）— 返回 Area→Obj→KR→Project→Scope→Initiative 全树
- `apps/api/src/task-system/routes.js` — 挂载 /full-tree
- `apps/api/features/gtd/pages/GTDOkr.tsx` — 使用全树，Area 作为分组头
- `apps/api/features/gtd/components/DatabaseView.tsx` — 加 inline 编辑（status/priority inline select）
- `apps/api/src/task-system/projects.js` — 修复 type 映射（okr_projects→'project'，okr_initiatives→'initiative'）
- DB migration（内嵌在 full-tree.js 启动时）— 归档重复 areas

## 成功标准
- /gtd/okr 显示：[Cecelia] 和 [ZenithJoy] 两个 Area 分组，各含 Objective→KR 展开树
- 点击 KR 可继续展开 Project→Scope→Initiative
- 点击 status/priority 可 inline 修改
- /gtd/projects 显示 124 条 Project（不再是空白）
