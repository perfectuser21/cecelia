# PRD: crystallize task type — 替代 codex_playwright

## 背景

`codex_playwright` 和 `playwright-auto` 只有2步（探索+验证），缺少 Scope 和 Register，是不完整的原型。新的 `crystallize` task type 是完整4步流水线：Scope → Forge → Verify → Register。

## 目标

1. 新建 `crystallize` task type（含5个子类型：crystallize_scope/forge/verify/register）
2. 替代 `codex_playwright` + `playwright-auto`，从所有文件中移除旧类型
3. playwright-auto skill 重命名为 playwright

## 成功标准

- crystallize task type 可通过 Brain API 创建，不报 constraint violation
- crystallize-orchestrator.js 实现4步状态机
- codex_playwright 从所有文件中完全移除
- CI 全部通过
