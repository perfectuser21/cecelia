# Task Card: 知识文档系统完善 — 补全author字段+新建conversation_captures+放开design_docs类型

## 需求（What & Why）

**功能描述**: 知识文档系统多张表缺少author/made_by字段，无法区分人类决策和系统决策。同时缺少对话捕获表，导致Claude Code会话内容无法持久化。

**背景**: 审计发现decisions表10,539条记录无author字段（355条/天是机器决策），design_docs type被硬锁死5种类型不够用，conversation_captures表缺失导致会话记忆为零。

**不做什么**: 不回填历史数据，不改动现有记录内容，不修改前端页面。

## 成功标准

1. [ARTIFACT] decisions表有 author/made_by/priority/area/alternatives/decided_at 字段
2. [ARTIFACT] learnings/knowledge/dev_records/ideas 各表有 author/made_by 字段
3. [ARTIFACT] conversation_captures表存在，含完整索引
4. [ARTIFACT] conversation-captures路由文件包含 GET 和 POST handler
5. [ARTIFACT] strategic-decisions路由支持 made_by/author 过滤查询
6. [ARTIFACT] design_docs migration包含 meeting/strategy/roadmap 等新类型
7. [ARTIFACT] selfcheck.js EXPECTED_SCHEMA_VERSION = '194'
8. [GATE] CI 全部通过
