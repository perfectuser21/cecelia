# Task Card: P2P 对话查询能力——派任务+回调回飞书

## 需求（What & Why）

**功能描述**：Cecelia 在飞书 P2P 对话里无法真正"查东西"。当用户问"zenithjoy dashboard 现在有什么"时，她只会生成"我去查一下"的空话，不会真正派出探查任务，结果永远回不来。

需要实现的能力：
1. orchestrator-chat.js P2P 流程里，mouth 判断需要查询时，能通过 `dispatch_query_task` 信号 dispatch 一个 exploratory/research 任务
2. 任务完成后通过 execution-callback 把结果发回原 P2P 对话（调飞书 send_message API 回复用户）
3. 任务派发时先发一条"正在查，稍等"给用户，避免无声等待

**背景**：Brain 任务是 Cecelia 的手脚，但 P2P 对话路径上，任务创建（create_task）和任务回调没有关联到原始对话。需要在任务 payload 里存入 p2p_callback 元数据（open_id），execution-callback 完成后读取并发飞书消息。

**不做什么**：
- 不做群聊回调（仅 P2P）
- 不改 thalamus LLM 路由器逻辑（不改 thalamusProcessEvent）
- 不做流式回复

## 成功标准

- 飞书 P2P 问"zenithjoy dashboard 现在有什么"时，orchestrator-chat.js 通过 dispatch_query_task 信号派出探查任务，MOUTH_SYSTEM_PROMPT 引导 LLM 正确识别查询意图
- Cecelia 立即发出"正在查，马上给你"作为 reply，ops.js handleChat 携带 conversation_id: openId 传递 P2P 上下文
- thalamus.js observeChat 处理 dispatch_query_task，在 tasks 表创建 research 任务，payload 含 p2p_callback 和 open_id 字段
- 任务完成后 execution.js 检查 p2p_callback，调用 sendFeishuToOpenId 把结果发回飞书给用户（openId 为空时优雅降级返回 false）
