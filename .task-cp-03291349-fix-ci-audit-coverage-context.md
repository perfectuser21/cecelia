# Task Card: 修复3个P0地基缺口 — CI audit + coverage + /context SQL

## 需求（What & Why）

**功能描述**：修复 /repo-audit 发现的3个虚假绿缺陷，让 CI 门禁真实有效。

**背景**：
- /repo-audit 2026-03-29 得分 72/120，发现3个 CI 虚假绿
- 虚假绿比真实红更危险：表面健康但实际无效
- 这是地基阶段 KR4（/repo-audit 72→90）的核心修复 PR

**具体问题**：
1. `no_audit_in_ci`：L3 brain-l3 job 没有 npm audit，14个 HIGH 漏洞对 CI 不可见
2. `no_coverage_gate`：Brain 单元测试在 CI 中以 `--no coverage` 运行（step 名"no coverage"），vitest.config.js 定义的75%阈值从不执行；coverage-delta job 有 `continue-on-error: true` 导致阈值永不生效
3. `/context SQL bug`：`decisions` 表实际列为 `id/ts/trigger/input_summary/llm_output_json/action_result_json/status`，但 context.js 查询 `SELECT title, category, status, created_at FROM decisions`，导致 `GET /api/brain/context` 500错误

**不做什么**：
- 不改 enforce_admins（GitHub UI 设置，不在代码里）
- 不改测试本身，只改 CI 结构
- 不动 Brain 核心逻辑

## 成功标准

1. [ARTIFACT] `.github/workflows/ci-l3-code.yml` 中 brain-l3 job 有 `npm audit --audit-level=high`（根目录执行）
2. [BEHAVIOR] `GET /api/brain/context` 不再返回 500 错误
   Test: manual:curl -s localhost:5221/api/brain/context | node -e "const d=JSON.parse(require('fs').readFileSync('/dev/stdin','utf8'));if(!d.success)process.exit(1)"
3. [ARTIFACT] `coverage-delta` job 中的 coverage run 步骤不再有无条件 `continue-on-error: true`（阈值违反应该失败）
4. [GATE] CI 全部通过

## 实现方案

1. `.github/workflows/ci-l3-code.yml`

   - coverage-delta job：移除 `continue-on-error: true`，改为 OOM 智能检测（有报告才查阈值）
2. `packages/brain/src/routes/context.js`
