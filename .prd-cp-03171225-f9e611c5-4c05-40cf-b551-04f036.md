# PRD: fix(executor): syncOrphanTasksOnStartup 孤儿改 requeue

## 背景
Brain 重启时，executor.js 的 syncOrphanTasksOnStartup 将所有 in_progress 任务无条件标记为 failed，导致静默数据丢失。

## 目标
修复 syncOrphanTasksOnStartup 逻辑：
- 可重试孤儿（Brain 重启中断）→ 改为 queued
- 真实失败孤儿（exit_code 非零、retry_count >= max_retries）→ 保持 failed

## 成功标准
- [ ] Brain 重启后，之前 in_progress 的任务出现在 queued 状态
- [ ] retry_count >= max_retries 的任务仍保持 failed
- [ ] 单元测试：startup-sync 路径 requeue 行为有断言
