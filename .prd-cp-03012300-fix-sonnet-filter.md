# PRD: 修复 selectBestAccount 遗漏 sevenDaySonnetPct 过滤

## 问题

account1 的 `seven_day_sonnet_pct=100%`（Sonnet 周额度耗尽），但 `five_hour_pct=6%`，
当前 .filter() 只检查 `a.pct < USAGE_THRESHOLD && !a.extraUsed`，
导致 account1 被选中后 LLM 调用返回 exit code 143（SIGTERM），
Cecelia 始终回复"（此刻有些恍神，稍后再聊）"。

## 修复

在 `.map()` 里补 `sevenDaySonnetPct` 字段，在 `.filter()` 里加 `&& a.sevenDaySonnetPct < 100`。
