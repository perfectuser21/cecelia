# PRD: 修复 rumination 链路 — NotebookLM 响应清洗

## 背景
rumination 链路故障：NotebookLM CLI 切换 notebook 时返回前缀噪音文本，
被误判为有效洞察写入 memory_stream，导致 dedup 机制失效、垃圾洞察持续累积。

## 功能描述
在 `rumination.js` 中添加 `cleanNotebookResponse()` 函数，
在 `digestLearnings()` 中清洗 NotebookLM 响应，提取真实 Answer 内容，
噪音前缀导致内容为空时 fallback 到 callLLM。

## 成功标准
- NotebookLM 响应中的 "Different notebook specified" 前缀被清洗，不写入 memory_stream
- 清洗后洞察长度 < 50 时，fallback 到 callLLM 确保生成真实洞察
- cecelia_events 中 rumination_output 记录正确写入，dedup 机制恢复工作
