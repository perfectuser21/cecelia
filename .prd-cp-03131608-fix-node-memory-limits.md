# PRD：修复 Node.js 内存管理配置缺口

## 背景

Brain 从 Linux VPS 迁移到 macOS Mac mini M4（16GB 统一内存）后，出现三波 OOM 崩机。
根因之一：各 package.json 的 test script 从未设置 NODE_OPTIONS 内存上限，
依赖外部环境变量（只在 CI workflow YAML 里设置了），本地 npm test 裸跑会 OOM。

## 目标

将内存限制固化到配置文件中，跨平台（macOS/Linux）、跨环境（本地/CI）都有保障。

## 修改范围（纯配置，不改逻辑代码）

### L1（阻塞性，必修）
- `packages/brain/package.json`：test/test:coverage script 加 NODE_OPTIONS=3072MB

### L2（功能性，建议修）
- `packages/brain/Dockerfile`：CMD 加 --max-old-space-size=3072
- `docker-compose.yml`：node-brain service 加 deploy.resources.limits.memory: 4G
- `packages/engine/package.json`：test/test:coverage script 加 NODE_OPTIONS=2048MB
- `packages/quality/package.json`：test script 加 NODE_OPTIONS=2048MB
- `apps/api/package.json`：test script 加 NODE_OPTIONS=2048MB
- `docker-compose.yml`：frontend service 加 deploy.resources.limits.memory: 256M

## 验收标准

- packages/brain 本地 npm test 能正常运行，不 OOM
- 所有修改文件语法正确
