# Task Card: fix(executor): worktree_creation_failed 未知错误根因修复

## 需求（What & Why）
**功能描述**: 修复 `cecelia-run.sh` 中 `worktree_creation_failed: unknown error` 永远显示 unknown error 的问题，并增加详细错误日志便于诊断。
**背景**: 至少 5 个 dev 任务以 `failure_class=env_setup / stderr=worktree_creation_failed: unknown error` 失败。根因是 use-after-delete bug：第 427 行 `rm -f "$wt_stderr_log"` 删除临时文件后，第 431 行再尝试读取同一文件，文件已不存在，`cat` 失败，始终落入 `echo 'unknown error'` 分支。
**不做什么**: 不修改 worktree-manage.sh 逻辑，不修改 executor.js，不修改 startup-recovery.js（无需改动）。

## 成功标准

1. [ARTIFACT] `cecelia-run.sh` 中 use-after-delete bug 已修复：先保存 stderr 内容到变量，再删除临时文件
2. [BEHAVIOR] worktree 创建失败时，`err_log` 中写入的是真实 stderr 内容而非 `unknown error`
3. [GATE] CI 全部通过
