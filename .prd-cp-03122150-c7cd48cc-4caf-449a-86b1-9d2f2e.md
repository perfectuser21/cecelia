# PRD - fix(profile): cortex 切换为 anthropic-api provider + Sonnet fallback

**需求来源**: Brain 自动调度
**功能描述**: 将 DB profile-anthropic 中 cortex.provider 更新为 anthropic-api，并在 model-profile.js FALLBACK_PROFILE 中为 cortex 添加 fallbacks
**涉及文件**:
- `packages/brain/src/model-profile.js` - 更新 FALLBACK_PROFILE.cortex.fallbacks
- DB migration - 更新 profile-anthropic 中 cortex.provider 为 anthropic-api
**非目标**: 不修改其他 profile 配置，不影响其他 agent 的调用路径

## 背景
当前 DB 中 profile-anthropic 的 cortex 配置为 provider=anthropic（走 bridge），
即使修复了 bridge cap，直连 anthropic-api 仍然更快（无 bridge 中间层）。
FALLBACK_PROFILE 已经正确配置了 anthropic-api，但 DB profile 落后。

## 目标
1. 将 DB profile-anthropic 中 cortex.provider 更新为 anthropic-api
2. 在 model-profile.js FALLBACK_PROFILE 中为 cortex 添加 fallbacks: [{provider: anthropic-api, model: claude-sonnet-4-6}]
3. 验证 callAnthropicAPI 路径被正确触发

## 成功标准
- [ ] profile-anthropic 中 cortex.provider = anthropic-api（migration 验证）
- [ ] FALLBACK_PROFILE 包含 cortex fallback 到 Sonnet（代码验证）
- [ ] 测试验证 callAnthropicAPI 被调用（非 callClaudeViaBridge）

## 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| migration 执行失败 | 回滚，保持原 provider 配置 | P1 | profile_migration_failed |
| anthropic-api 调用失败 | fallback 到 bridge | P2 | cortex_api_fallback |
