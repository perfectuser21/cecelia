# Task Card: Learning 格式模板修正 + Brain Task 回调修复

## 需求（What & Why）

**功能描述**: 两处 /dev 流程缺陷修复：
1. `04-learning.md` 的记录模板格式与 CI `check-learning.sh` 实际要求不一致——模板用 `**粗体**`，CI 要求 `### 根本原因` / `### 下次预防` / `- [ ]`，导致按模板写的 Learning 直接被 CI 拒绝（PR #968 踩坑）
2. `update-task-status.sh` 用 PATCH API 标记 dev task completed 无法生效——Brain 对 dev task 要求带 `pr_url` 才能变为 `completed`，否则变为 `completed_no_pr`，导致队列堆积

**背景**: PR #968 今天踩了 Learning 格式问题；Brain 队列中已合并的任务无法自动标记完成。

**不做什么**: 不改 check-learning.sh 本身，不改 Brain routes.js 逻辑。

## 成功标准

1. [ARTIFACT] 04-learning.md 的「记录模板」章节使用 `### 根本原因` / `### 下次预防` / `- [ ]` 格式
2. [ARTIFACT] 04-learning.md 的「上传反馈到 Brain」章节改为 execution-callback + pr_url 方式
3. [GATE] CI 全部通过
