## PRD - executor.js setBillingPause 写 cecelia_events 持久化

**需求来源**: Brain 自动调度任务
**功能描述**: PR #874 实现了 _billingPause in-memory 标记，但 Brain 重启后状态丢失。本 PR 让 setBillingPause 在设置 in-memory 状态的同时，写一条 billing_pause_set 事件到 cecelia_events，作为持久化来源。
**涉及文件**: packages/brain/src/executor.js, packages/brain/src/__tests__/executor-billing-pause.test.js（或相关测试文件）
**成功标准**: setBillingPause 调用后 cecelia_events 有 billing_pause_set 记录，poolRef 为 null 时降级只设 in-memory 不报错，写入失败时 warn 不阻塞
**非目标**: 不实现重启恢复读取逻辑（只写入，不读取）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| poolRef 为 null | 降级：只设 in-memory，不报错 | P3 | billing_pause_set_skipped |
| cecelia_events INSERT 失败 | fire-and-forget：打 warn 日志，不阻塞 | P3 | billing_pause_persist_failed |

## 成功标准

- [ ] setBillingPause 调用后，cecelia_events 有 billing_pause_set 记录
- [ ] poolRef 为 null 时降级：只设 in-memory，不报错
- [ ] 写入失败时打 warn 日志，不阻塞主流程（fire-and-forget）
- [ ] 单测：mock pool，验证 cecelia_events INSERT 被调用
- [ ] 现有 billing-pause 测试不能回归
