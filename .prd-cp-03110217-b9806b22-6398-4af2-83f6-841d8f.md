## PRD - executor 派发前增加 claude --version 健康探针（登录态异常暂停派发）

**需求来源**: Brain 自动调度任务，agent_ops 领域
**功能描述**: 在 executor.js 的 dispatchNextTask() 函数中，派发任务前先执行 `claude --version` 探针。若探针失败（exit code 非 0 或输出包含 "Not logged in"/"Please run /login" 等关键词），则暂停派发，写入 brain_events 告警事件，并在下一个 tick 自动重探，成功后恢复派发。
**涉及文件**:
- `packages/brain/src/executor.js` — 主要改动，添加探针逻辑
- `packages/brain/src/__tests__/executor.test.js` — 单测（mock claude --version 失败场景）
**成功标准**: 见下方

## 成功标准
- [ ] dispatchNextTask() 调用前有探针检查（runClaudeProbe 函数）
- [ ] 探针超时 5s，失败时 dispatch_paused=true，写 brain_events
- [ ] 登录态丢失时不派发新任务
- [ ] 下一个 tick 自动重探，成功后恢复派发（dispatch_paused=false）
- [ ] 有对应单测（mock claude --version 失败场景）

**非目标**: 不修改 tick 频率；不修改任务本身的错误处理逻辑

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| claude --version 超时（>5s） | 视为探针失败，暂停派发 | P1 | claude_probe_timeout |
| 输出含 Not logged in / Please run /login | 暂停派发，告警 | P1 | claude_auth_lost |
| exit code 非 0（其他原因） | 暂停派发，告警 | P1 | claude_probe_failed |
| 探针成功后恢复 | 清除 dispatch_paused，写恢复事件 | P3 | claude_probe_recovered |
