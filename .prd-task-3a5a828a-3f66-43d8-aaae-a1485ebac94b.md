## 功能描述

## 背景

billing_pause 全局熔断在 quarantine 路径存在缺口：

当 Claude 输出 BILLING_CAP 错误（out of usage / spending cap reached）时，quarantine.js 会：
- ✅ 调用 parseResetTime(errorStr) 解析配额重置时间
- ✅ 调用 blockTask() 阻塞**当前单个任务**
- ❌ 未调用 setBillingPause() → 全局 _billingPause 变量未被设置

结果：dispatcher 不感知到熔断，继续派发剩余 N-1 个排队任务，它们会立刻撞墙（BILLING_CAP）再被逐个 blockTask，O(n) 成本而非 O(1)。

**对比**：callbacks/routes.js 的 quota_exhausted 路径已正确调用 setBillingPause。quarantine 路径漏掉了这一步。

## 成功标准

- [ ] quarantine.js BILLING_CAP 处理块，在 blockTask 之后调用 setBillingPause(resetTime.toISOString(), 'billing_cap')
- [ ] setBillingPause 通过动态 import 从 executor.js 获取（避免循环依赖，与现有 blockTask 动态 import 模式一致）
- [ ] 单元测试：BILLING_CAP 触发 quarantine 处理时，getBillingPause() 返回 active=true
- [ ] 不破坏现有 callback quota_exhausted 路径（两路并存）

## 技术要点

**文件**：packages/brain/src/quarantine.js，约 line 1017-1028

**修改**：
```js
if (classification.class === FAILURE_CLASS.BILLING_CAP) {
  const { blockTask } = await import("./task-updater.js");
  const { setBillingPause } = await import("./executor.js"); // ← 新增
  const errorStr = String(task.payload?.error_details || "");
  const resetTime = parseResetTime(errorStr) || new Date(Date.now() + 2 * 60 * 60 * 1000);
  const r = await blockTask(taskId, { reason: "billing_cap", detail: errorStr.slice(0, 500), until: resetTime });
  setBillingPause(resetTime.toISOString(), "billing_cap"); // ← 新增：全局熔断
  console.log(`[quarantine] Task ${taskId} blocked + global billing pause SET until ${resetTime.toISOString()}`);
  return { blocked: true, reason: "billing_cap", blocked_until: resetTime.toISOString(), result: r };
}
```
