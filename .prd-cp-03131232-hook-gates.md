## PRD - Hook Gates 真锁实现（方案A）

**需求来源**: /dev 工作流大量"必须"要求只是 .md 文字，AI 可跳过；只有 branch-protect/stop-dev/bash-guard 是真正的 hook 锁。
**功能描述**: 在现有 3 个 hook 文件中加入 5 个真锁，将 .md 文字 gate 变为代码强制执行。
**涉及文件**:
- `hooks/bash-guard.sh` — 新增 git push 拦截 + git commit 格式拦截
- `hooks/stop-dev.sh` — 新增 Learning 内容验证（不只看 flag）
- `hooks/branch-protect.sh` — 新增 .prd-*.md 内容验证 + .dod-*.md 内容验证

**非目标**: 不改 Stop Hook 核心循环逻辑，不改 CI，不改 subagent 机制

## 成功标准

- `git push origin` 命令触发时，`local-precheck.sh` 自动运行，失败则 push 被阻止（exit 2）
- `git commit -m "random text"` 被阻止，要求 feat:/fix:/docs: 等 Conventional Commits 格式
- stop-dev.sh 在 STEP_10 flag=done 时额外运行 `check-learning.sh` 验证内容，内容不达标则重置 flag 并阻止合并
- Write `.prd-*.md` 时，若内容缺少 `## 成功标准` 章节则被 branch-protect.sh 阻止（exit 2）
- Write `.dod-*.md` 时，若内容缺少 `- [ ]` checkbox 则被 branch-protect.sh 阻止（exit 2）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| local-precheck.sh 未找到 | 放行（不阻止 push，打印警告） | P3 | hook_precheck_not_found |
| check-learning.sh 运行失败（非格式错误） | 降级：只检查 flag，打印警告 | P3 | hook_learning_check_error |
| commit message 无 -m 参数（interactive） | 放行（无法检测内容） | P3 | N/A |
