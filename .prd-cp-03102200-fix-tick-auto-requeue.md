# PRD: 超时任务自动 requeue，最多3次后隔离

## 功能描述

修改 `autoFailTimedOutTasks()` 行为：超时任务不再直接 fail，改为 requeue 重试，最多3次后隔离。

## 需求来源

`autoFailTimedOutTasks()` 在 tick.js 中处理超时（>60分钟）的 in_progress 任务：
- 调用 `handleTaskFailure()` → 递增 `failure_count`、检查是否应隔离（FAILURE_THRESHOLD=3）
- 若不应隔离：直接标记 `status = 'failed'`（问题！永远停在 failed，不重试）
- 若应隔离：标记为 quarantined（正确）

## 成功标准

超时任务行为改为：
- 1st timeout: failure_count=1, requeue（重新排队）
- 2nd timeout: failure_count=2, requeue（重新排队）
- 3rd timeout: failure_count=3, quarantine（隔离，不再重试）

## 范围

`packages/brain/src/tick.js` — `autoFailTimedOutTasks()` 函数：
1. "not quarantined" 分支：改为 `UPDATE tasks SET status='queued', started_at=NULL`
2. action 字段改为 `auto-requeue-timeout`
3. 新增单元测试 `tick-timeout-requeue.test.js`
