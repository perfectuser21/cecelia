# Task Card: 修复内容工厂 Pipeline 列表加载错误

## 需求（What & Why）

**功能描述**: 修复 GET /api/brain/pipelines 报错 `column "failed_at" does not exist`，让内容工厂前台页面能正常加载 Pipeline 列表。

**背景**: tasks 表只有 `completed_at`，`failed_at` 列根本不存在。GET 查询 SELECT 了不存在的列导致 500 错误，前台列表一直加载失败。Brain tick loop 编排（orchestrateContentPipelines）已经在 tick.js 中正确注册。

**不做什么**: 不加 failed_at 列到数据库（没必要），不改变 Pipeline 执行逻辑。

## 成功标准

1. [ARTIFACT] `content-pipeline.js` GET 查询不再包含 `failed_at` 字段
2. [BEHAVIOR] GET /api/brain/pipelines 返回正常 JSON 数组（无 SQL 错误）
3. [GATE] CI 全部通过

## 验收条件（DoD）
