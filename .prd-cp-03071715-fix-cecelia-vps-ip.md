# PRD: 修复 cecelia 仓库 n8n workflows 中硬编码的老 VPS IP

## 背景

美国 VPS 已从旧节点（146.190.52.84）迁移到新旧金山 SFO3 节点（134.199.234.147）。
cecelia 仓库的 n8n workflow JSON 文件中多处硬编码了老 IP，包括 SSH host 配置和 HTTP 执行器 URL。
老 VPS 销毁后这些 workflow 会失败，最关键的是 `unit-claude-code-异步执行.json` 中的执行器 API URL，
会导致 Cecelia Brain 无法派发 AI 任务。

## 目标

将 cecelia 仓库 `packages/workflows/n8n/` 中所有活跃 workflow 文件的老 VPS IP 替换为新 IP。

## 涉及文件（packages/workflows/n8n/ 下）

1. `workflows/tools/unit-workflow变更同步.json` — SSH host
2. `workflows/tools/unit-夜间备份.json` — SSH host
3. `workflows/cecelia/unit-claude-code-异步执行.json` — HTTP URL (port 9876)
4. `workflows/cecelia/flow-devgate-每夜推送.json` — SSH host
5. `workflows/media/unit-微博数据爬取.json` — SSH host
6. `workflows/media/unit-今日头条数据爬取.json` — SSH host
7. `workflows/media/unit-小红书数据爬取.json` — SSH host
8. `workflows/media/unit-头条ai测试数据爬取.json` — SSH host
9. `workflows/media/flow-内容发布.json` — SSH host
10. `workflows/media/unit-知乎数据爬取.json` — SSH host
11. `workflows/media/unit-快手数据爬取.json` — SSH host
12. `workflows/media/unit-公众号数据爬取.json` — SSH host
13. `workflows/media/unit-视频号数据爬取.json` — SSH host
14. `devgate-nightly-push.json` — SSH host

## 成功标准

- `packages/workflows/n8n/` 下不再出现 `146.190.52.84`（排除 archive/）
