# Task Card: KR3 冲刺 — 每日进度报告调度器 + 模块任务分解

## 需求（What & Why）

**功能描述**: 在 Brain 中添加每日 06:00 UTC KR3 进度报告调度器，并通过 Brain API 创建微信小程序剩余 7 个核心功能模块的 dev 任务。

**背景**: ZenithJoy KR3「微信小程序上线 — 基础功能可用，无重大bug」当前进度 25%，距 4 月中旬上线目标有 75% 缺口。采用夜间自驱引擎并行推进剩余模块，需要每日进度跟踪机制。

**不做什么**: 不实现小程序功能本身（由拆解出的 dev 任务完成）；不修改现有 tick 逻辑；不添加数据库表。

## 实现方案

**要改的文件**:
- 新增: `packages/brain/src/kr3-progress-scheduler.js`
- 修改: `packages/brain/src/tick.js`（添加 import + 调用）
- 修改: `.agent-knowledge/brain.md`（注册新模块）

**核心逻辑**:
- `isInKR3ReportWindow(now)`: UTC hour===6 && minute<5
- `scheduleKR3ProgressReport(pool)`: 幂等检查 → 查 KR3 key_result 进度 → 查 miniapp dev 任务状态 → console.log 进度摘要

**不改什么**: DB schema、其他 tick 步骤、nightly-tick.js
