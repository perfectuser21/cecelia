# Task Card: deploy webhook 非阻塞化 + 并发保护

## 需求（What & Why）

**功能描述**：
`POST /api/brain/deploy` webhook 目前使用 `execSync` 调用 deploy-local.sh，
导致 Node.js 事件循环在部署期间（最长 10 分钟）完全阻塞：
- 所有 Brain API 请求超时
- Tick 循环冻结
- 历史失败 exit 137（OOM kill）时 Brain 进程本身被杀，deployState 丢失

**背景**：
此任务属于 agent_ops 域，为后续自动发布/数据闭环/Dashboard 能力提供稳定交付基础。
PR #1316 已补齐 deployState + GET 端点 + CI poll，本次补齐最后一块：异步执行 + 并发保护。

**不做什么**：
- 不改 brain-deploy.sh / deploy-local.sh 脚本内容
- 不改 CI deploy.yml 轮询逻辑
- 不引入数据库持久化（deployState 仍 in-memory）

## 成功标准

1. `POST /api/brain/deploy` 立即返回 202，deploy 脚本异步执行，不阻塞事件循环
2. 并发保护：第二次 POST 在 running 时返回 409 Conflict
3. deployState 新增 `output_tail`（最后 20 行 stdout/stderr）供调试用
4. CI 全部通过
