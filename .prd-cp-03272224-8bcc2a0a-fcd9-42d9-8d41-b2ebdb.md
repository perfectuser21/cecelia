# PRD: 封堵孤岛 — /dev 强制注册 + 集成测试

## 背景

新功能通过 `/dev` 开发后，没有门禁强制要求开发者：
1. 更新 `docs/instruction-book/`（说明书）
2. 编写集成测试

导致能力扫描器（capability-scanner）将这些功能标记为"孤岛"。

## 目标

添加两道 CI 门禁，封堵孤岛产生的根源：
1. **L1 Instruction Book Gate**：`feat:` PR 必须更新 `docs/instruction-book/`
2. **L3 Integration Test Gate**：Brain `feat:` PR 必须包含集成测试文件

## 成功标准

- `feat:` PR 未更新 instruction-book → L1 CI 失败
- Brain `feat:` PR 无集成测试 → L3 CI 失败
- 本 PR 自身满足两道门禁（附带说明书条目 + 集成测试）
- `[SKIP-DOCS]` 标签可豁免 instruction-book 检查
- `[CONFIG]` 标签可豁免两道门禁

## 改动范围

- `.github/workflows/ci-l1-process.yml`：新增 `instruction-book-required` job
- `.github/workflows/ci-l3-code.yml`：新增 `integration-test-required` job
- `docs/instruction-book/features/dev-registration-gate.md`：新建说明书条目
- `packages/brain/src/__tests__/integration/system-registry-api.integration.test.js`：新建集成测试
