# Task Card: Janitor 杀孤儿后回报 Brain 触发重调度

## 需求摘要

Janitor v4.0 kill claude 孤儿进程后，必须通知 Brain 更新对应任务状态，使其可被重调度。

## 上下文

- Janitor 每 15 分钟运行一次 frequent 模式
- `kill_if_claude_orphan()` 识别并 kill 无头孤儿 claude 进程
- kill 后对应的 Brain 任务仍为 `in_progress`，永远不会被重调度
- 需要在 kill 后读取 `.dev-lock.*` 拿到分支名，PATCH Brain API

## 成功标准

- janitor.sh 新增 `notify_brain_orphan_killed(pid, branch)` 函数
- 函数逻辑：扫描所有 worktree 的 `.dev-lock.*` → 提取 branch → 查 Brain 找 in_progress 任务 → PATCH status=failed + fail_reason
- `kill_if_claude_orphan` kill 后调用该函数（fire-and-forget，失败只 log）
- 包含字符串 `orphan_killed_by_janitor`
