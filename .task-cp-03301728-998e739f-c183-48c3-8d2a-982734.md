# Task Card: 自动日报生成

## 需求（What & Why）

**功能描述**: Brain 每天 UTC 01:00 自动生成 ZenithJoy 内容日报，汇总昨日内容产出、各平台发布情况、数据回收和异常告警，并推送到飞书。

**背景**: 当前没有任何日报生成逻辑，需要人工查看各平台数据，效率低且容易遗漏异常。自动日报可以让团队每天早上（北京时间 09:00）第一时间掌握前一天的内容运营全貌。

**不做什么**:
- 不做日报的 Web UI 展示页面
- 不做历史日报的查询 API
- 不做多渠道推送（仅飞书，不含邮件/钉钉/微信）
- 不修改 content-pipeline 本身的逻辑
- 不做实时数据看板

## 成功标准

1. [ARTIFACT] `packages/brain/src/daily-report-generator.js` 文件存在，导出 `generateDailyReport` 函数
2. [ARTIFACT] `tick.js` 或 `daily-publish-scheduler.js` 中包含日报触发注册
3. [BEHAVIOR] 每天 UTC 01:00（北京时间 09:00）系统自动生成日报，写入 `working_memory`（key 格式为 `daily_report_{DATE}`），并推送到飞书 webhook
4. [PRESERVE] 现有 tick 循环、daily-publish-scheduler 的已有调度逻辑不受影响
5. [GATE] CI 全部通过
