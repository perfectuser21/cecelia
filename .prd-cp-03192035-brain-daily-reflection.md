# PRD: Brain 每日自检反思

## 背景

Brain tick loop 只做任务派发，不做自检。dev_execution_logs 和 learnings 数据积累了但从不分析。需要一个每日自检反思机制，自动发现反复出现的问题并创建修复任务。

## 目标

创建 `daily-reflection-scheduler.js`，每天定时分析执行日志和 learnings，发现反复问题时自动创建 self_fix 任务。

## 成功标准

- daily-reflection-scheduler.js 存在且导出 triggerDailyReflection 函数
- tick.js 中注册了 triggerDailyReflection 调用
- 触发窗口为 04:00 UTC（北京 12:00），与 contract-scan 的 03:00 错开
- 有去重机制：同一天不重复触发
- 分析最近 7 天 dev_execution_logs，聚合失败率和 TOP 错误
- 查询未应用的 learnings
- 同一 error_message 出现 >= 3 次时创建 self_fix 任务
- self_fix 任务包含足够上下文（错误摘要、出现次数、关联任务 ID）

## 技术方案

参考 `daily-review-scheduler.js` 模式：
1. `isInReflectionWindow(now)` — 时间窗口检查
2. `hasTodayReflection(pool)` — 去重
3. `analyzeExecutionLogs(pool)` — SQL 聚合分析
4. `getUnappliedLearnings(pool)` — 查询未应用 learnings
5. `createSelfFixTask(pool, errorPattern)` — 创建修复任务
6. `triggerDailyReflection(pool)` — 主入口，在 tick.js 中调用
