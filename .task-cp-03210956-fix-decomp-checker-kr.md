# Task Card: 修复 decomposition-checker KR 识别错误

## 需求（What & Why）
**功能描述**: 修复 decomposition-checker.js 中三处 SQL 查询的 type/status 匹配错误，使 OKR 拆解飞轮能自动运转。
**背景**: 数据库中可拆解的 KR 实际是 `type = 'area_kr'` + `status = 'ready'`，但代码查询的是 `type = 'area_okr'` + `status = 'pending'`，导致 Check A/B/C 全部无法匹配到真实 KR 数据。同时 Check D 的子 KR 查询也需要兼容 `area_kr` 类型。
**不做什么**: 不改变拆解任务的创建逻辑和描述内容，不改 planner 或 tick 逻辑。

## 成功标准

1. [BEHAVIOR] Check A 能匹配 `type IN ('area_kr', 'kr')` 且 `status IN ('pending', 'ready')` 的 KR
2. [BEHAVIOR] Check B 能匹配 `type IN ('area_kr', 'kr')` 且 `status IN ('ready', 'in_progress')` 的 KR
3. [BEHAVIOR] Check C 能匹配 `type IN ('area_kr', 'kr')` 且 `status IN ('ready', 'in_progress')` 的 KR
4. [BEHAVIOR] Check A 创建拆解任务后将 KR status 更新为 `decomposing`（已有此逻辑，确认保留）
5. [BEHAVIOR] Check D 的子 KR 查询兼容 `area_kr` 类型
6. [GATE] 所有现有测试通过
