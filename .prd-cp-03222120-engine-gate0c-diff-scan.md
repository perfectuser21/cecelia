# PRD: Gate 0c 改为只扫 diff 新增行

## 背景

verify-step.sh 的 Gate 0c 检查变更文件中是否含有 `console.log` / `debugger`。
当前实现对**整个文件**执行 grep，导致只要改动的文件中预存在任何 `console.log`（哪怕是生产结构化日志），Gate 0c 就会误报为"含调试垃圾代码"，阻止 push。

典型案例：`executor.js` 含 58 处 `console.log` / `console.warn` 结构化日志，任何涉及该文件的 PR 都会被误拦截。

## 目标

将 Gate 0c 改为只扫描 `git diff` 中**新增行**（`+` 开头），语义更准确：
- ✅ 禁止**新增** `console.log` / `debugger`
- ✅ 不影响文件中已有的结构化日志

## 成功标准

- [ ] verify-step.sh Gate 0c 使用 `git diff origin/main...HEAD` 只检查新增行
- [ ] 预存在的 console.log 不再触发误报
- [ ] 新增的 console.log 仍会被正确拦截
