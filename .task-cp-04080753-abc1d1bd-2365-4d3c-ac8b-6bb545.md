# Task Card: CI 绿灯率提升 — Deploy 稳定性三项修复

## 需求（What & Why）
**功能描述**: 修复导致 Deploy 工作流 62% 失败率的三个根本原因
**背景**:
1. Brain 用 execSync 阻塞事件循环 → launchctl kickstart -k 触发后新 Brain 启动 EADDRINUSE（FATAL: listen EADDRINUSE :::5221），造成部署失败
2. Safe Lane staging 返回 skipped_no_docker/no_env 状态，CI 不识别 → 300s 超时失败
3. packages/brain/__tests__/ 和 scripts/ 文件变更也触发全量 Brain 部署（不必要）
**不做什么**: 不修改 Staging 部署逻辑本身、不改 CI L1-L4 结构

## 成功标准
> [BEHAVIOR] Deploy 工作流不再因 EADDRINUSE 而失败
> [BEHAVIOR] Safe Lane staging 跳过时 CI 正常通过（不超时）
> [ARTIFACT] test/script-only 变更不触发 Brain 部署

## 验收条件（DoD）

- [x] [BEHAVIOR] ops.js 改用 detached spawn + 文件持久化，Brain 可自由处理 SIGTERM
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/routes/ops.js','utf8');if(!c.includes('detached: true'))process.exit(1);console.log('OK: detached spawn found')"

- [x] [BEHAVIOR] brain-deploy.sh 成功后写 /tmp/cecelia-deploy-status.json
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/brain-deploy.sh','utf8');if(!c.includes('cecelia-deploy-status.json'))process.exit(1);console.log('OK: status file write found')"

- [x] [BEHAVIOR] deploy.yml Safe Lane 轮询识别 skipped_* 状态为成功
  Test: manual:node -e "const c=require('fs').readFileSync('.github/workflows/deploy.yml','utf8');if(!c.includes('skipped_'))process.exit(1);console.log('OK: skipped_ handling found')"

- [x] [ARTIFACT] deploy-local.sh NEED_BRAIN 排除 __tests__/ 和 scripts/ 路径
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/deploy-local.sh','utf8');if(!c.includes('__tests__') && !c.includes('src/scripts'))process.exit(1);console.log('OK: exclusions found')"

## 实现方案
**要改的文件**:
- `packages/brain/src/routes/ops.js` — 核心: execSync → spawn(detached:true) + 文件状态
- `scripts/brain-deploy.sh` — 在 EXIT trap 写 /tmp/cecelia-deploy-status.json
- `.github/workflows/deploy.yml` — Safe Lane: 添加 skipped_* 识别
- `scripts/deploy-local.sh` — NEED_BRAIN 排除 __tests__/ 和 src/scripts/

**不改什么**: brain-build.sh / staging-deploy.sh / ci.yml
