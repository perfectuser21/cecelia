# PRD: Bridge 超时修复（CECELIA_BRIDGE_TIMEOUT_MS + degraded fallback）

## 需求来源

Brain 的 thalamus/cortex 通过 `cecelia-bridge` 调用 claude CLI（`/llm-call` endpoint）。
当前硬编码 90s 超时 → claude 进程被 SIGTERM → exit code 143 → bridge 返回 HTTP 500 `{ ok: false, error: "" }`。
调用方（`llm-caller.js`）无法区分：超时 vs API 错误 vs 权限问题 → 错误日志无信息 → 重试无意义。

## 功能描述

1. Bridge 超时时间通过环境变量 `CECELIA_BRIDGE_TIMEOUT_MS` 可配置（默认 120000ms）
2. Bridge 超时时返回 HTTP 200 + `{ ok: false, status: 'timeout', degraded: true, message, elapsed_ms }` 而非 HTTP 500
3. `llm-caller.js` 识别 `degraded: true` 响应，抛出带 `degraded=true` 属性的错误
4. `cortex.js` 超时时计入熔断器（`recordFailure`），并记录到任务 `error_message`

## 成功标准

- 超时不再产生空 error 的 HTTP 500
- 调用方可区分 timeout vs 其他错误
- env var 可调整超时上限
- 新增测试覆盖 degraded 响应处理逻辑
