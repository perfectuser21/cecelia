# PRD: tick 感知模块移到 dispatch 前面

## 背景
tick.js 的 executeTick() 中，感知/巡航模块（zombie sweep、Pipeline Patrol、health check、initiative-closer、kr-progress）位于 thalamus 处理之后。当 thalamus dispatch_task 直接 return 或 canDispatch=false 时，这些感知模块被跳过。

## 目标
把感知模块移到 alertness/cognition 评估之后、thalamus 之前，确保它们不受任何中间 return 影响。

## 成功标准
1. 所有感知模块在 thalamus 之前执行
2. canDispatch=false 时感知模块仍正常运行
3. 代码有"感知层"注释标记
4. CI 通过
