# Task Card: CI P0 修复：fix commit 不跳 L3 + coverage-delta 接上基线对比

## 需求（What & Why）

**功能描述**: 修复两个 P0 级 CI 问题：

1. **detect-commit-type 策略修正**：`fix:`/`refactor:`/`chore:` commit 目前跳过 L3 全部测试，这是高风险漏洞。新策略：只有 `docs:` 和 `style:` 可以跳过 L3，`fix:`/`refactor:`/`chore:` 必须跑 L3。

2. **coverage-delta 基线对比没连上**：`coverage-delta` job 调用 `davelosert/vitest-coverage-report-action@v2` 时缺少 `json-summary-compare-path`，且没有下载 `coverage-baseline` artifact 的步骤。需要在 `coverage-delta` job 里加：先 `download-artifact coverage-baseline`，再把 `json-summary-compare-path` 指向下载的文件。首次运行（无基线）应 `continue-on-error`。

**背景**: CTO 架构审查发现，`fix:` commit 跳 L3 导致 bug fix 不跑测试，覆盖率 delta 功能因缺少基线下载步骤实际上只展示绝对覆盖率而非 delta。

**不做什么**: 不修改覆盖率阈值（另一个任务），不改 Brain 代码。

## 成功标准

1. [ARTIFACT] `detect-commit-type` 中 `fix:`/`refactor:`/`chore:` 不再导致 `should_run_l3=false`
2. [ARTIFACT] `coverage-delta` job 新增 `actions/download-artifact` 步骤下载基线
3. [ARTIFACT] `coverage-delta` job 的 `vitest-coverage-report-action` 配置了 `json-summary-compare-path`
4. [GATE] CI 全部通过（[CONFIG] 标签豁免测试覆盖检查）

- [x] [GATE] 所有现有测试通过（CI 通过）

## 实现方案

**要改的文件**:
1. `.github/workflows/ci-l3-code.yml` — detect-commit-type + coverage-delta

**改动点**:
- `detect-commit-type` 的 skip_types: `fix|docs|style|refactor|chore|test` → `docs|style`
- `coverage-delta` job 添加 `actions/download-artifact@v4` 步骤（`continue-on-error: true`）
- `coverage-delta` 的 `vitest-coverage-report-action` 添加 `json-summary-compare-path`

**Scope 锚定**: 仅修改 `.github/workflows/ci-l3-code.yml`，不触碰 Brain 代码或测试文件
