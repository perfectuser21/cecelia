# Task: Skills 索引自动更新脚本

## 需求

`.agent-knowledge/skills-index.md` 当前是手动维护的静态文件。每次新增 Skill 时需要手动更新，容易遗漏。

`packages/workflows/skills/` 目录（仓库内）是 Skills 的真实来源（53个目录，每个含 SKILL.md）。需要一个脚本，从这个目录自动生成/更新 `skills-index.md`，并通过 CI 或本地手动触发保持同步。

实现方案：本地脚本（`scripts/generate-skills-index.mjs`），扫描 `packages/workflows/skills/`，读取每个 SKILL.md 的 frontmatter，生成结构化的 `skills-index.md`。新增 Skill 时只需跑一次脚本即可。

## 成功标准

- `scripts/generate-skills-index.mjs` 存在且可执行
- 运行脚本后 `.agent-knowledge/skills-index.md` 被重新生成（内容来自 packages/workflows/skills/）
- 脚本读取每个 SKILL.md 的 name、description 字段
- 生成的索引按类别分组，格式与现有 skills-index.md 一致
- 脚本有 --dry-run 参数（只打印不写文件）
