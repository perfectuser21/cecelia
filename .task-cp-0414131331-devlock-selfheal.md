# Task Card: Stop Hook dev-lock 自愈机制

## 需求
修复 `.dev-mode` 存在但 `.dev-lock` 丢失时 Stop Hook 永久 block 的场景。用当前 `CLAUDE_SESSION_ID` 自动重建 dev-lock。

## 成功标准
- stop-dev.sh 在 orphan 扫描前检测 dev-lock 缺失
- 检测条件：dev-mode 首行是 `dev` + dev-lock 不存在 + `CLAUDE_SESSION_ID` 非空
- 自愈：写新 dev-lock 含 session_id / tty / recreated_at / recovered:true
- 日志: stderr 输出 `[Stop Hook] dev-lock 自愈重建`
- 自愈后 Stop Hook 重新匹配 session 走正常 devloop_check 流程
