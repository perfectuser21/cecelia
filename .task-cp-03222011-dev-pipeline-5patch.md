# Task Card: /dev pipeline 5项补强

## 任务目标

补强 /dev pipeline 的5个系统性漏洞：CI失败无记录、Learning检查时机滞后、PRD格式无本地验证、并行PR覆盖无提示、code_review_gate无信息卫生维度。

## 成功标准

- CI失败时 .dev-incident-log.json 被写入，Stage 4 Learning 能读到真实 CI 失败次数
- Stage 1 自检本地运行 check-prd.sh，PRD bullet 格式错误在本地被拦截不等 CI
- 03-integrate.md 新增并行PR重叠文件检测警告（3.1.2b节），避免 squash merge 覆盖
- code-review-gate 新增维度H信息卫生（引用已删除功能/路径 = warning）

## 实现方案

**要改的文件**:
- `packages/engine/skills/dev/steps/01-spec.md`
- `packages/engine/skills/dev/steps/02-code.md`
- `packages/engine/skills/dev/steps/03-integrate.md`

- Engine 版本 6 文件（package.json, package-lock.json, VERSION, .hook-core-version, regression-contract.yaml, feature-registry.yml）

**Scope 锚定**: packages/engine/skills/dev/steps/ + packages/workflows/skills/code-review-gate/ — 纯文档修复，无代码层改动
