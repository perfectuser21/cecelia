# PRD: Add navigate_to_page Tool

## 问题描述 (Problem)

Voice commands like "去OKR页面" or "跳转到projects" are being interpreted as `open_detail` calls, which require specific item names. Users want to navigate to pages, not open specific items.

## 功能描述 (Goal)

Add a `navigate_to_page` tool to the OpenAI Realtime API so voice navigation commands work properly.

## 实现方案 (Changes)

1. Add `navigate_to_page` tool to tools array in `orchestrator_routes.py`
2. Implement handler in `execute_tool` function
3. Return action that frontend can intercept to navigate

## Pages
- okr: /okr
- projects: /projects
- tasks: /tasks
- orchestrator: /orchestrator
- planner: /planner
- brain: /brain
- home: /

## 成功标准 (Success Criteria)

1. "去OKR页面" → navigates to /okr
2. "打开项目" → navigates to /projects
3. Frontend receives navigation action and executes it
