# PRD: 修复任务路由编辑：UPSERT + 全C类可编辑

**Task ID**: 62dfe49a-874c-45b1-97f9-8fbac99393fb

## 问题

`task-type-config-cache.js` 的 `updateConfig` 只做 UPDATE，
task_type 不在 DB 里返回 null → API 404 → 前端无法保存。
导致 C类固定 Codex 任务（codex_dev/pr_review/content-pipeline 等）无法编辑。

## 方案

1. Backend: `updateConfig` 改为 UPSERT（INSERT ... ON CONFLICT DO UPDATE）
2. Frontend: 所有 C类任务 `editable: true`，DetailPanel 不依赖 dynamicConfig 是否存在

## 成功标准

- [ ] PUT 任意 C类 task_type 均返回 200（不再 404）
- [ ] 前端所有 C类任务（包括 codex_dev、content-pipeline 等）显示编辑按钮
- [ ] 切换机器保存后 Brain 缓存立即生效
