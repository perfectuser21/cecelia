# PRD：Cecelia 嘴巴感知飞书群 + 工具链修复

## 背景

用户通过飞书私聊问 Cecelia"能给群里发消息吗"，Cecelia 回答"不行"。
根因：
1. `buildManifestBlock` 只告知 API 端点存在，但 LLM 不知道实际 group_id 值，无法构造 POST 请求
2. 工具链结果注入后写着"不要再调用 call_brain_api 了"，导致 GET groups → POST send 的两步链路被截断

## 目标

Cecelia 在私聊中说"给群里发消息"时，能够直接发出去，不需要用户提供 group_id。

## 成功标准

1. `buildManifestBlock` 从 DB 查询已知飞书群，将 group_id 和最近活跃时间直接注入系统提示
2. 工具链结果注入提示改为允许继续调用（移除"不要再调用"的硬性限制）
3. 测试：D15 新增 2 个测试用例覆盖上述行为

## 非目标

- 不改变 call_brain_api 的执行逻辑
- 不修改飞书发送相关 API 端点
