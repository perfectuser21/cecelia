## PRD - 修复 Linux→macOS 全量兼容性问题

**需求来源**: 系统从 Linux 迁移到 Mac mini 后，逐个发现兼容坑导致 Cecelia 反复瘫痪。本次一次性找完所有问题统一修复。

**功能描述**: 修复 Brain 代码中所有假设 Linux 环境的代码，覆盖内存检测、Shell 命令、进程管理三大类。

**涉及文件**:
- `packages/brain/src/executor.js` — os.freemem() 修复
- `packages/brain/src/routes.js` — os.freemem() 修复（VPS 监控）
- `packages/brain/src/platform-utils.js` — 新增 getAvailableMemoryMB() 函数
- `packages/brain/src/alertness/healing.js` — pgrep -f + isProcessAlive() 统一
- `packages/brain/scripts/cecelia-run.sh`（如在 git 追踪）— stat -c 修复

**成功标准**:
## 成功标准
1. Brain watchdog 在 macOS 上显示 `free_mem_mb` ≥ 1000MB（不再是 66MB）
2. `mem_pressure` < 0.5（不再是 0.99）
3. `effectiveSlots > 0`，Brain 能正常派发任务
4. VPS 监控页面内存显示正确
5. healing.js 进程检测使用统一 platform-utils 函数

**非目标**:
- 不改 Linux 路径的行为
- 不重构其他无关代码
- 不动测试框架配置

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| vm_stat 命令失败 | fallback 到 os.freemem() | P3 | `macos_vmstat_fallback` |
| stat 命令格式不匹配 | 返回 null，不阻断流程 | P3 | N/A |
