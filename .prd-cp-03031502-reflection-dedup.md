# PRD - 反思去重机制（Reflection Deduplication）

**需求来源**: Cecelia 欲望系统反馈 - desire ID `3fcd1dc7-7678-4c38-84d6-2b82e265e9f1`

**问题背景**:
- 反思循环已成为执行瓶颈：8 轮相同诊断、0 行修复代码
- 系统把「精准诊断」当成了「完成任务」
- 相似的 memory_stream → 相似的 insight → 相似的 desire → 相似的 initiative_plan 任务（被去重拦截）
- 反思层只对 memory_stream 做去重，但**没有对生成的洞察本身做去重**

**功能描述**:
在 `packages/brain/src/desire/reflection.js` 中添加基于内容 hash 的幂等检查机制：

1. **洞察内容 hash 计算**：在生成新洞察后，计算内容的语义相似度 hash
2. **历史洞察检查**：查询最近 7 天的反思洞察（memory_stream 中 content 以 `[反思洞察]` 开头的记录）
3. **相似度检测**：使用 Jaccard similarity（token-based）检测新洞察与历史洞察的相似度
4. **去重决策**：
   - 如果相似度 > 0.75，跳过写入，输出日志 `[reflection] Insight skipped (duplicate)`
   - 否则正常写入 memory_stream

**涉及文件**:
- `packages/brain/src/desire/reflection.js` - 核心去重逻辑
- `packages/brain/src/__tests__/desire-system.test.js` - 添加去重测试用例

**成功标准**:
1. ✅ 反思层在写入洞察前检查历史重复
2. ✅ 相似度 > 0.75 的洞察被跳过，不写入 memory_stream
3. ✅ 日志输出包含去重决策信息
4. ✅ 测试覆盖：模拟重复洞察被成功拦截
5. ✅ 不影响正常的非重复洞察写入流程

**非目标**:
- ❌ 不修改 memory_stream 的去重逻辑（已有，保留）
- ❌ 不改动 desire-formation 层的去重（已有 initiative_plan 任务去重，保留）
- ❌ 不使用 embedding 向量相似度（成本高，token-based 够用）

**技术方案**:
```javascript
// 在 runReflection() 中，生成 insight 后、写入 memory_stream 前：

// 1. 查询最近 7 天的反思洞察
const { rows: recentInsights } = await pool.query(`
  SELECT content FROM memory_stream
  WHERE content LIKE '[反思洞察]%'
    AND created_at > NOW() - INTERVAL '7 days'
  ORDER BY created_at DESC
  LIMIT 20
`);

// 2. 计算 Jaccard 相似度
const newTokens = new Set(insight.toLowerCase().split(/\s+/).filter(t => t.length > 1));
let maxSimilarity = 0;

for (const old of recentInsights) {
  const oldContent = old.content.replace('[反思洞察] ', '');
  const oldTokens = new Set(oldContent.toLowerCase().split(/\s+/).filter(t => t.length > 1));

  let intersection = 0;
  for (const t of newTokens) { if (oldTokens.has(t)) intersection++; }
  const union = new Set([...newTokens, ...oldTokens]).size;
  const similarity = union > 0 ? intersection / union : 0;

  if (similarity > maxSimilarity) maxSimilarity = similarity;
}

// 3. 去重决策
if (maxSimilarity > 0.75) {
  console.log(`[reflection] Insight skipped (duplicate, similarity=${maxSimilarity.toFixed(2)})`);
  await pool.query(/* 重置 accumulator */);
  return { triggered: true, insight: null, skipped: 'duplicate', similarity: maxSimilarity };
}

// 4. 否则正常写入
```
