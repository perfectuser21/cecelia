# DoD - 仪表盘 UI/UX 设计

QA: docs/QA-DECISION-dashboard-ui-ux.md

## 验收标准

### 功能验收

#### 平台维度统计
- [ ] 显示至少 5 个平台的发布统计数据
      Test: manual:浏览器访问仪表盘页面，验证平台卡片数量
- [ ] 每个平台显示成功率百分比
      Test: manual:验证每个平台卡片显示成功率数字
- [ ] 平台卡片按成功率或活跃度排序
      Test: manual:验证卡片顺序符合排序逻辑

#### 时间维度分析
- [ ] 支持切换 24h/7d/30d 三种时间范围
      Test: manual:点击时间范围切换按钮，验证图表数据更新
- [ ] 显示时间序列折线图或柱状图
      Test: manual:验证图表显示趋势数据
- [ ] 图表交互：悬停显示详细数据点
      Test: manual:鼠标悬停在图表上，验证 tooltip 显示

#### 成功率维度
- [ ] 显示成功/失败/进行中任务的饼图或环形图
      Test: manual:验证图表显示三种状态占比
- [ ] 显示具体数量和百分比
      Test: manual:验证数字标签显示正确

#### 响应式布局
- [ ] 桌面端布局合理，信息密度适中
      Test: manual:1920x1080 分辨率下截图验证
- [ ] 移动端布局自适应，卡片堆叠
      Test: manual:375x667 分辨率下截图验证

#### 数据刷新
- [ ] 数据自动刷新，默认间隔 30 秒
      Test: manual:等待 30 秒验证数据更新
- [ ] 提供手动刷新按钮
      Test: manual:点击刷新按钮验证数据立即更新

#### 用户体验
- [ ] 加载时显示骨架屏或加载动画
      Test: manual:首次加载页面观察加载状态
- [ ] 数据为空时显示友好的空状态提示
      Test: manual:清空数据后验证空状态UI
- [ ] 支持浅色/深色主题切换
      Test: manual:切换主题验证颜色变化

### 技术验收
- [ ] 路由配置正确，可通过导航访问
      Test: manual:点击侧边栏导航进入仪表盘
- [ ] API 端点调用正常
      Test: manual:打开浏览器 DevTools Network 验证 API 请求
- [ ] 首屏加载时间 < 2 秒
      Test: manual:使用 Chrome Lighthouse 测试性能
- [ ] 图表渲染流畅，无卡顿
      Test: manual:观察图表切换和交互流畅度

### 代码质量
- [ ] 组件结构清晰，职责分离
      Test: contract:audit-pass
- [ ] 无 console.log 和未使用的 import
      Test: contract:audit-pass
- [ ] CSS 样式一致，符合现有设计系统
      Test: manual:代码审查验证样式规范

### 边界情况
- [ ] 成功率极值处理（0%或100%）
      Test: manual:mock API返回success_rate=0或100，验证UI显示正常
- [ ] 数据异常大值处理
      Test: manual:mock API返回超大数值（如999999），验证数字格式化
- [ ] 平台数量超预期处理
      Test: manual:mock API返回20个平台，验证布局不崩溃
