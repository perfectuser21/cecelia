# PRD: P0 修复 — RCA→auto-fix 无限重试，熔断机制失效

## 问题

`capability-probe.js` 在触发 auto-fix 时使用 `probe_${f.name}_${Date.now()}` 作为 signature，Date.now() 使每次 signature 唯一，导致 `dispatchToDevSkill` 的 3 次失败熔断永远无法触发。每次 probe 失败都创建新 auto-fix 任务，死循环。

同时 `dispatchToDevSkill` 的 guard 只查 `status=failed` 的任务，不查 `queued/in_progress`，导致正在修复中也会重复创建任务。

## 方案

1. `capability-probe.js:343` 改用稳定 signature：`probe_${f.name}`（去掉 Date.now()）
2. `auto-fix.js:dispatchToDevSkill` guard 新增：查同 signature 是否有 queued/in_progress 任务，有则跳过

## 成功标准

- [x] capability-probe.js signature 不含 Date.now()，使用稳定的 `probe_${f.name}`
- [x] dispatchToDevSkill guard 同时检查 queued/in_progress 状态
- [x] 同一 probe 失败多次 → 只创建一个 auto-fix 任务，不重复
- [x] CI 全部通过
