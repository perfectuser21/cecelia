# Task Card: 修复 L2 concurrency group — 去掉 run_number

## 需求（What & Why）

**功能描述**: 将 L2 CI 的 concurrency group 从 `l2-v2-${{ github.ref }}-${{ github.run_number }}` 改回 `l2-${{ github.ref }}`，保留 `cancel-in-progress: true`。

**背景**: 当初加 `run_number` 是为了绕过 pending 死锁（当时 `cancel-in-progress: false`），每次 run 有唯一 group 让它绕过锁。但现在 `cancel-in-progress: true`，加了 `run_number` 反而导致每次 run 都在自己的独立 group 中，同分支的旧 run 永远不会被取消，并发控制完全失效。

**不做什么**: 不改其他 CI 文件（L1/L3/L4）；不改并发控制以外的逻辑。

## 成功标准

1. [ARTIFACT] `ci-l2-consistency.yml` 的 concurrency group 不含 `run_number`
2. [ARTIFACT] `cancel-in-progress: true` 保留
3. [BEHAVIOR] 同分支两次 push 时，新的 L2 run 能取消旧的 run（不会同时跑两个）
4. [GATE] CI 全部通过

## 验收条件（DoD）

- [x] [ARTIFACT] concurrency group 改为 `l2-${{ github.ref }}`，去掉 `run_number`
  Test: manual:node -e "const f=require('fs').readFileSync('.github/workflows/ci-l2-consistency.yml','utf8'); if(f.includes('run_number'))throw new Error('run_number 未删除'); console.log('OK: run_number 已删除')"

- [x] [ARTIFACT] cancel-in-progress 仍为 true
  Test: manual:node -e "const f=require('fs').readFileSync('.github/workflows/ci-l2-consistency.yml','utf8'); if(!f.includes('cancel-in-progress: true'))throw new Error('cancel-in-progress 不是 true'); console.log('OK')"

- [x] [BEHAVIOR] group 名称包含 github.ref，保证同分支并发能互相取消
  Test: manual:node -e "const f=require('fs').readFileSync('.github/workflows/ci-l2-consistency.yml','utf8'); const m=f.match(/group:\\s*(.+)/); if(!m||!m[1].includes('github.ref'))throw new Error('group 缺少 github.ref'); console.log('OK: group =',m[1].trim())"

- [x] [GATE] 所有现有测试通过
  Test: manual:bash -c "cd packages/engine && npm test 2>&1 | tail -5"
