# Task Card: capability-probe 批次总失败数 ≥5 触发回滚

## 需求（What & Why）
**功能描述**: 在 capability-probe.js 的 runProbeCycle 中，当单次批次总失败数 ≥ 5 时立即触发 brain-rollback.sh，不需要等待连续3次失败。
**背景**: PR #1898 已实现"连续失败3次触发回滚"，但 PRD 还要求"同一批次总失败数 ≥ 5 触发回滚"这一更激进的保护机制。当系统大面积崩溃（5+探针同时失败）时需立即回滚。
**不做什么**: 不改变连续失败阈值（3次），不修改告警格式，不改其他文件

## 成功标准

## 验收条件（DoD）

- [x] [BEHAVIOR] capability-probe.js 中 runProbeCycle：当批次失败数 ≥ 5 时触发回滚，附带 P0 告警
  Test: manual:node -e "const fs=require('fs');const c=fs.readFileSync('packages/brain/src/capability-probe.js','utf8');if(!c.includes('ROLLBACK_BATCH_THRESHOLD'))process.exit(1);if(!c.includes('failures.length >= ROLLBACK_BATCH_THRESHOLD'))process.exit(1)"

- [x] [BEHAVIOR] 批次 ≥5 触发时附加回滚状态到飞书告警消息
  Test: manual:node -e "const fs=require('fs');const c=fs.readFileSync('packages/brain/src/capability-probe.js','utf8');if(!c.includes('batch_failures'))process.exit(1)"

  Test: manual:node -e "const fs=require('fs');const c=fs.readFileSync('packages/brain/src/__tests__/capability-probe-rollback.test.js','utf8');if(!c.includes('ROLLBACK_BATCH_THRESHOLD') && !c.includes('批次'))process.exit(1)"

## 实现方案
**要改的文件**:
- `packages/brain/src/capability-probe.js` — 添加 ROLLBACK_BATCH_THRESHOLD=5 常量 + runProbeCycle 批次检查
- `packages/brain/src/__tests__/capability-probe-rollback.test.js` — 添加批次 ≥5 测试

**受影响函数**: `runProbeCycle`（添加批次失败检查逻辑）
**不改什么**: checkConsecutiveFailures、executeRollback、ROLLBACK_CONSECUTIVE_THRESHOLD
