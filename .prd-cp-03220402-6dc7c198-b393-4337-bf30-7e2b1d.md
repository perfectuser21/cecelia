# PRD: dev pipeline 补三个硬锁单元

## 背景

dev pipeline 的 Stage 2 缺乏行为快照和 TDD 强制约束，导致改动可能在不知情的情况下破坏现有行为；CI 镜像检查因 `|| true` 而失去阻断力。

## 功能描述

在 `packages/engine/skills/dev/steps/02-code.md` 补三个强制（exit 1）单元：

1. **Step 2.0 行为快照 [PRESERVE]**：探索代码前，扫描涉及模块的现有行为并写入 Task Card `[PRESERVE]` 条目，verify-step.sh Gate 2 强制执行验证
2. **Step 2.1.5 TDD先行**：探索代码后、写实现前，必须先写失败测试（红灯确认），再写实现（绿灯确认）
3. **CI镜像强制**：修复 2.3.6 的 `|| true`，全部改为强制 `exit 1`；verify-step.sh 补全对 `[PRESERVE]` 条目的识别与执行

## 成功标准

- [PRESERVE] 条目类型在 verify-step.sh 中被识别和执行
- 02-code.md Step 2.0 存在且有 exit 1
- 02-code.md Step 2.1.5 存在 TDD 红灯→绿灯流程
- 2.3.6 无 `|| true`，失败 exit 1
- Engine 版本 bump（patch）
