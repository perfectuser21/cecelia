## PRD - feat(dashboard): ProjectCompare 接入 Brain KR 进度 + 实时刷新

**需求来源**: Brain 自动调度任务
**功能描述**: 在现有 ProjectCompare 页面中接入 Brain KR Compare API，展示 KR 进度区块，并实现每 30 秒自动刷新。
**涉及文件**: `apps/api/features/planning/pages/ProjectCompare.tsx`

## 成功标准

- KR 进度区块显示在每个对比卡片中，含 KR 数量、平均进度条、近 7 天速度
- 选择项目后每 30 秒自动刷新，组件卸载时定时器被清除
- 右上角显示刷新时间戳（"上次刷新：XX 秒前" 或 "刷新中..."）
- 当 Brain API 不可用时降级显示（不影响原有 task_stats 展示）

**非目标**:
- 不添加新路由或新页面
- 不修改 Brain API 端点
- 不修改其他组件

## API 规范

**端点**: `GET /api/brain/projects/compare?ids={id1},{id2}[,{id3},{id4}]`

**返回数据结构（每个项目）**:
```json
{
  "id": "...",
  "name": "...",
  "kr_progress": { "total": 3, "completed": 1, "in_progress": 2, "avg_progress": 42.5 },
  "task_velocity": { "completed_last_7d": 5, "created_last_7d": 8 },
  "trend": [ { "week": "2026-W10", "completed": 3, "created": 5 } ]
}
```

## KR 进度区块 UI

在现有对比卡片的任务统计下方新增「KR 进度」区块：
- KR 总数 / 已完成 / 进行中
- avg_progress 显示为进度条（0-100%）
- 速度：近 7 天完成 N 个任务

## 实时刷新

- 每 30 秒自动重新请求（仅在已选择 ≥ 2 个项目时触发）
- 右上角显示「上次刷新：XX 秒前」或「刷新中...」
- 组件卸载时清除定时器

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| Brain Compare API 不可用 | 降级显示（隐藏 KR 区块，保留原有 task_stats） | P3 | kr_compare_api_unavailable |
| 网络超时 | 静默忽略，保留上次数据 | P3 | kr_compare_timeout |
