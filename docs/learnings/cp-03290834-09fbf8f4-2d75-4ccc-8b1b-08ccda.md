## 任务概述
升级 check-dod-mapping.cjs 的 detectFakeTest() 函数，新增 printf/ls/cat/true/exit 0/standalone grep 等弱测试模式的拦截。同时更新 DOD-TEMPLATE.md 模板，展示标准可执行的 node -e 测试写法。

### 根本原因
现有 detectFakeTest() 函数只检测了 echo/grep wc-l/test -f/TODO 四种模式，缺少 printf/ls/cat/true/exit 0/standalone grep 等同样属于"无真实断言"的弱测试模式。

这些弱测试模式共同特征是：命令能成功执行（exit 0），但没有任何 assert 逻辑，无法真正验证功能。

因此这些弱测试能绕过 check-dod-mapping.cjs 的拦截，悄然进入 DoD 和 CI，让表面"通过"实际无效的测试污染质量体系。

同时发现 echo/printf/ls/cat 等检查若使用全文匹配（`\becho\b`），会误判 node -e "...echo..." 命令中引号内的字面量，需改为只匹配命令开头（`^\s*echo\s`）。
