# Learning: 重构 runDesireSystem 降低圈复杂度

**Branch**: cp-03221804-3c3efc10-166d-4826-a04b-de84ee
**Date**: 2026-03-23

## 变更摘要

重构 `packages/brain/src/desire/index.js` 中的 `runDesireSystem` 函数，将圈复杂度从 38 降低至 5。

## 根本原因

`runDesireSystem` 函数将六层欲望系统（感知/情绪/记忆/反思/形成/表达）的所有分支逻辑内联在单一函数中，包含：
- 6 个 try/catch 块
- 嵌套的 explore/act/follow_up 分支
- 每个分支中的内联 SQL 查询

导致圈复杂度高达 38，远超阈值 10。

## 解决方案

提取 7 个职责单一的子函数：

| 函数 | 职责 |
|------|------|
| `runPerceptionAndEmotion` | Layer 1+1.5 感知与情绪 |
| `runMemoryLayer` | Layer 2 记忆 |
| `runReflectionAndFormation` | Layer 3+4 反思与欲望形成 |
| `getExpressionCandidate` | Layer 5 表达候选（含错误处理） |
| `handleExploreDesire` | explore 类型欲望处理 |
| `handleActFollowupDesire` | act/follow_up 类型欲望处理 |
| `handleNotifyDesire` | Layer 6 通知类型表达 |

主函数 `runDesireSystem` 变为纯编排器，圈复杂度降至 5。

## 下次预防

- [ ] 单函数超过 50 行时主动检查圈复杂度
- [ ] 多个 try/catch + if/else 嵌套 = 提取子函数的信号
- [ ] 欲望系统扩展新类型时，按「新增独立处理函数 + 主函数加一行 if」模式添加
