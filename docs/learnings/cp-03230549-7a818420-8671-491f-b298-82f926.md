### 根本原因

routes 层（8个文件）仍然查询废弃的 `goals`/`projects` 表，而 migration 179 已将数据迁移到新 OKR 表体系（`objectives`/`key_results`/`okr_projects`/`okr_scopes`/`okr_initiatives`）。这导致 Brain API 在 PR2（tick/task-router）切新表之后，routes 层查询返回空数据。

### 关键技术决策

**层级映射**（已确认与 task-router.js PR2 一致）:
- `goals WHERE type='area_okr'` → `objectives`
- `goals WHERE type='area_kr'` → `key_results`
- `projects WHERE type='project'` → `okr_projects`（含 `kr_id` 外键）
- `projects WHERE type='scope'` → `okr_scopes`（含 `project_id` 外键）
- `projects WHERE type='initiative'` → `okr_initiatives`（含 `scope_id` 外键）
- `tasks.project_id` → `tasks.okr_initiative_id`

**测试 mock 陷阱**：`intent-match.js` 从单表查询（goals）变为三表查询（objectives + key_results + okr_projects），对应测试的 `pool.query.mockResolvedValueOnce` 也必须从 1 次增加到 3 次，否则第 2/3 次查询返回 undefined 报错。

**task-router-diagnose.js** 的 SQL 中用 `okr_initiative_id AS initiative_id` 作别名，测试 mock 的任务计数行字段名必须是 `initiative_id`（不是 `project_id`）。
