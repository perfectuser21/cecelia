# Learning: arch_review 每4小时定时执行 + 路由到西安 Codex

Branch: cp-03230753-57fb49ec-a804-4fc7-951a-d294d2
Date: 2026-03-23

## 功能概述

为 `arch_review` 任务类型添加定时调度逻辑（每4小时自动派发），并将路由从 `'us'` 改为 `'xian'`（西安 Codex agents）。

### 根本原因

arch_review 无定时触发，架构漂移只能靠手动发现。task-router.js 中 arch_review 路由到 `'us'`（美国本机），但 arch_review 是纯分析/审查类任务，应路由到西安 Codex agents 而非美国本机（本机 /dev 才是美国专属）。

### 实现

1. `daily-review-scheduler.js` — 新增4个函数：
   - `isInArchReviewWindow(now)` — UTC 小时 % 4 === 0 且分钟 < 5
   - `hasRecentArchReview(pool)` — 4小时内已有 arch_review 任务则去重
   - `hasCompletedDevTaskSinceLastArchReview(pool)` — Guard：上次 review 后至少1个 dev 任务完成
   - `triggerArchReview(pool, now)` — 调度入口，窗口检查→去重→guard→插入任务

2. `tick.js` — import `triggerArchReview`，在 tick 末尾 fire-and-forget 调用

3. `task-router.js` — `'arch_review': 'us'` → `'arch_review': 'xian'`
