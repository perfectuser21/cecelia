# Learning: fix(brain): actions.js catch-all 路由拦截 create-project/scope/initiative 端点

**分支**: cp-03240011-b7319ded-9ae3-41d7-98f0-dfb6fe
**日期**: 2026-03-24

### 根本原因

Express.js 路由按注册顺序匹配。`POST /action/:actionName`（catch-all）在第 494 行注册，早于具体路由 `create-project`（564 行）、`create-scope`（601 行）、`create-initiative`（527 行）。导致所有 `/action/*` 请求都被 catch-all 处理，具体专用处理器永远不可达。

decomp 工作流调用这三个端点时，走了通用 `handleAction`，由于 action name 不在白名单中而失败，导致整个拆解流程阻塞。

### 修复

将 `POST /action/:actionName` catch-all 移到所有具体 action 路由之后注册，恢复正确的路由优先级。

### 下次预防

- [ ] 新增 catch-all 路由时，必须在文件末尾（而非顶部）注册
- [ ] `actions.js` 的路由顺序：具体路由 → catch-all（注释已添加提示）
- [ ] Code Review 时检查：任何 `/:param` 形式的路由是否出现在同前缀的具体路由之前
