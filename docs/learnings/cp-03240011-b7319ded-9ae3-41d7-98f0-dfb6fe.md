# Learning: fix(brain): actions.js catch-all 路由拦截具体端点

**分支**: cp-03240011-b7319ded-9ae3-41d7-98f0-dfb6fe
**日期**: 2026-03-24

### 根本原因

Express.js 路由按注册顺序匹配，先注册的优先命中。`packages/brain/src/routes/actions.js` 中，catch-all 路由 `POST /action/:actionName` 在第 494 行注册，早于具体路由（`create-project` 564 行、`create-scope` 601 行、`create-initiative` 527 行）。所有 `/action/*` 请求都被 catch-all 的通用 `handleAction` 处理，三个具体端点的专用逻辑永远不可达。`handleAction` 中这些 action name 不在白名单，直接失败，导致 decomp 工作流（调用这三个端点）全部阻塞。

### 下次预防

- [ ] 新增 catch-all 路由（`/:param` 形式）时，必须在同前缀所有具体路由之后注册
- [ ] Code Review 时检查：Express 路由文件中 `/:param` 是否出现在具体路由之前
- [ ] `actions.js` 已添加注释提示 catch-all 必须在具体路由之后
