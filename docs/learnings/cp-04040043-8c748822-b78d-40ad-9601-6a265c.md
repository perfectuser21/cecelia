# Learning: 重构 notionBlockToDBRow（CC 24 → dispatch-table）

**Branch**: cp-04040043-8c748822-b78d-40ad-9601-6a265c
**Date**: 2026-04-04

### 根本原因

`notionBlockToDBRow` 使用 `switch` 语句处理 10 种 block 类型，每个 `case` 加 `||` 短路运算符累积了大量条件分支（CC=24），导致函数难以扩展和测试。

### 解决方案

用 **dispatch-table**（`BLOCK_TYPE_MAP` 对象）替换 `switch`：
- 每种类型对应 `{ dbType, getContent }` 条目
- 主函数压缩至 6 行（CC=2）
- 新增 block 类型只需在表中添加一行，无需修改主函数

### 下次预防

- [ ] 新增 block 类型处理时直接在 `BLOCK_TYPE_MAP` 中追加，不要往函数体加 `if/switch`
- [ ] `notion-sync.js` 其他高 CC 函数（如 `notionPageToKnowledgeRow`）同样可用 dispatch-table 模式优化
- [ ] 导出内部工具函数（`notionBlockToDBRow`）有利于单元测试，不必总是通过集成测试覆盖
