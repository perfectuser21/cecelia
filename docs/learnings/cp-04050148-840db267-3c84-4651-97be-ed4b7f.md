# Learning: content-export resvg 渲染失败 — require is not defined

**任务**: fix(brain): content-pipeline content-export 阶段执行失败排查与修复

### 根本原因

`content-pipeline-executors.js` 是 ES module（顶层使用 `import`/`export`），但 `renderPng()` 函数内部使用了 CommonJS 的 `require(resvgPath)` 来加载 `@resvg/resvg-js`。ES module 不提供全局 `require`，导致每次 `renderPng` 调用均抛出 `require is not defined`，所有卡片渲染失败（count=0），`generateCards` 返回 `false`，pipeline 被标记为 `failed`，陷入无限重试循环。

### 关键诊断点

1. **错误只在 stderr**：`console.error('[export] resvg 渲染失败')` 输出到 stderr（`brain-error.log`），而 stdout 只有 `[export] 共生成 0 张卡片`——排查时要同时看两个日志文件
2. **错误消息误导**：`executeExport` 对 `generateCards` 返回 false 的统一报错为"有效 findings 为 0（brand_relevance < 3）"，掩盖了实际是 resvg 渲染失败
3. **研究数据实际存在**：findings.json 数据完整，`brand_relevance=4`，不是 findings 问题

### 修复

- 在 `import` 区加入 `import { createRequire } from 'module'` + `const _require = createRequire(import.meta.url)`
- 将 `renderPng` 中 `require(resvgPath)` 改为 `_require(resvgPath)`
- 改善 `executeExport` 错误消息：区分「无调研数据」vs「resvg 渲染失败」两种场景

### 下次预防

- [ ] 新建 executor 文件时检查：若文件顶层有 `import` 语句，不能在函数体内使用 `require()`
- [ ] 若需要在 ES module 中加载 CJS 包，用 `createRequire(import.meta.url)` 创建 require
- [ ] 调试 content-export 失败时，先看 `brain-error.log` 而非只看 `brain.log`
