# Learning: 封堵孤岛 — /dev 强制注册 + 集成测试门禁

### 根本原因

新功能通过 `/dev` 开发后，没有任何门禁要求开发者：
1. 更新 `docs/instruction-book/`（新功能无文档 → 使用者不知道 → 功能被闲置）
2. 编写集成测试（无测试覆盖 → capability-scanner 扫描后标记为孤岛）

根本原因：CI 只检查"有没有测试文件"（test-coverage-required），但未区分单元测试 vs 集成测试，也未检查说明书更新。

### 下次预防

- [ ] 新增 Brain feat 功能时，必须同步创建 `docs/instruction-book/features/` 说明条目
- [ ] Brain feat PR 必须在 `packages/brain/src/__tests__/integration/` 新增或更新集成测试
- [ ] 使用 `[SKIP-DOCS]` 标签豁免内部/基础设施功能（无需用户文档）
- [ ] 使用 `[CONFIG]` 标签豁免 CI/Engine 配置类 PR（无需说明书和集成测试）

### 实现细节

**L1 instruction-book-required**：
- 触发条件：`feat:` PR + stage >= 2 + 无 `[CONFIG]`/`[SKIP-DOCS]` 标签
- 检查：`docs/instruction-book/` 下有文件变更
- 错误提示：明确指出在 features/ 或 skills/ 创建文档

**L3 integration-test-required**：
- 触发条件：`feat:` commit type + Brain 代码变更 + 无 `[CONFIG]` 标签
- 检查：`packages/brain/src/__tests__/integration/` 下有文件变更
- 错误提示：明确指出创建 `.integration.test.js` 文件

**本 PR 自身满足两道门禁**：
- 说明书：`docs/instruction-book/features/dev-registration-gate.md`
- 集成测试：`packages/brain/src/__tests__/integration/system-registry-api.integration.test.js`（13 个测试全部通过）
