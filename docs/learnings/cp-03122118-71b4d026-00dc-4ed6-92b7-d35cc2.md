## bridge timeout cap 语义错误（2026-03-12）

### 根本原因

`cecelia-bridge.cjs` 中 `BRIDGE_TIMEOUT_MS` 同时充当默认值和 Math.min 上限，导致调用方传入的更长超时（如 cortex 的 300s）被截断为默认值 120s。

```js
// 错误：上限和默认值是同一个变量
const timeoutMs = Math.min(timeout || BRIDGE_TIMEOUT_MS, BRIDGE_TIMEOUT_MS);
```

### 下次预防

- [ ] 设计 timeout 参数时，默认值和硬上限必须用不同的变量/常量表达
- [ ] 新增 bridge endpoint 时，unit test 必须覆盖"传入值超过默认值时不被截断"这个 case
- [ ] bridge 启动日志记录所有可配置的超时参数值，便于运维排查
