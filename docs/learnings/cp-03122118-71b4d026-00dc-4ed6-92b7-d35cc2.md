# Learning: bridge timeout cap 语义错误

## 问题根因

`cecelia-bridge.cjs` 中 `BRIDGE_TIMEOUT_MS` 既作默认值又作上限：

```js
// 错误：上限和默认值用同一个变量
const timeoutMs = Math.min(timeout || BRIDGE_TIMEOUT_MS, BRIDGE_TIMEOUT_MS);
```

导致调用方（cortex）传入的 300s 被截断为 120s，cortex 任务频繁因超时失败。

## 修复方案

```js
// 正确：上限固定为 600s，默认值仍是 BRIDGE_TIMEOUT_MS
const timeoutMs = Math.min(timeout || BRIDGE_TIMEOUT_MS, 600000);
```

- `timeout` 有值时直接使用（不超过 600s 上限）
- `timeout` 无值时 fallback 到 `BRIDGE_TIMEOUT_MS`（120s 默认值）

## 经验规则

**单一变量不应同时充当"默认值"和"上限"**。两个语义应该分开：
- 默认值 = `BRIDGE_TIMEOUT_MS`（可配置）
- 硬上限 = `600000`（常量，防止无限等待）

## 影响范围

- `packages/brain/scripts/cecelia-bridge.cjs` line 74
- 影响所有调用 `/llm-call` 接口并传入 `timeout` 字段的调用方（主要是 cortex）
