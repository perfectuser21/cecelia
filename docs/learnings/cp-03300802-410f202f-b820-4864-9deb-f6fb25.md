### 根本原因

1. **DoD 格式陷阱**：`**[BEHAVIOR]**` 的 Markdown bold 格式导致 check-dod-mapping.cjs 无法识别标签（正则 `/^\[([A-Z]+)\]/` 需要直接以 `[` 开头），应写成 `[BEHAVIOR]` 不加粗体。

2. **`accessSync` 不被识别为断言**：`node -e "require('fs').accessSync(...)"` 虽然会在文件不存在时抛出异常（exit 1），但 check-dod-mapping 的断言检查正则要求明确包含 `process.exit/throw/if` 等关键词，需改用 `readFileSync + if + process.exit(1)` 模式。

3. **`tests/` 路径仅指根级 tests/ 目录**：DoD 中 `Test: tests/xxx` 指的是项目根级 `tests/` 目录（Brain 集成测试），不是 `apps/api/features/` 下的单元测试。前端组件测试应使用 `manual:node -e "readFileSync...includes...process.exit(1)"` 形式。

4. **spec_review Gate seal 文件名**：code_review_gate 期望的 seal 文件是 `.dev-gate-crg.{BRANCH}`（crg = code review gate），不是 `.dev-gate-code.{BRANCH}`。subagent 写错文件名后主 agent 无法解除 gate 阻塞。
