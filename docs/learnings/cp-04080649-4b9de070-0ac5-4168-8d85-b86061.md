# Learning: 周报写入 system_reports 打通 Dashboard 数据闭环

### 根本原因

`weekly-report-generator.js` 已完整实现周报生成（8 板块），但只写 `working_memory`，Dashboard 读 `system_reports`，两个存储互不可见，导致周报无法在 Dashboard 展示。

### 解决方案

1. 在 `weekly-report-generator.js` 中新增 `saveWeeklyReportToSystemReports()`，周报生成后同时写入 `system_reports` 表（type=weekly_report）
2. `generateWeeklyReport()` 增加 `opts.force` 参数，允许手动触发跳过时间窗口
3. `system-reports.js` POST /generate 支持 `weekly_report` type，调用真实生成逻辑
4. `ReportDetailPage.tsx` 增加 `weekly_report` 渲染分支（数据概览卡片 + 平台明细 + 爆款话题 + 完整文本折叠）

### 下次预防

- [ ] 新增数据生成模块时，同步检查 Dashboard 读数据的表是否匹配
- [ ] 存储路径（working_memory vs system_reports）在设计阶段就对齐，避免两段代码各用各的
- [ ] `force` 参数要有明确文档注释，避免误用于生产自动触发
