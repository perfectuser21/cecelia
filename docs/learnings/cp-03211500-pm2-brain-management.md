# Learning: PM2 进程管理配置

分支: cp-03211500-pm2-brain-management
日期: 2026-03-21

### 根本原因

Brain 作为 Cecelia 核心服务，之前缺少进程管理器，崩溃后需要人工重启。日志文件没有轮转策略，长时间运行会占满磁盘空间。

### 下次预防

- [ ] 新服务上线时，同步配置进程管理（PM2 ecosystem）
- [ ] 日志轮转应作为服务部署的标准组件
- [ ] 磁盘监控脚本应集成到定期巡检中

### 关键决策

1. PM2 配置使用 CJS 格式（ecosystem.config.cjs），因为 Brain 使用 ESM（type: "module"）
2. 日志路径使用 /tmp/ 而非项目目录，避免 git 污染
3. 内存上限设为 1G，超出后自动重启
4. 日志轮转：单文件 50MB，保留 5 份，启用压缩
