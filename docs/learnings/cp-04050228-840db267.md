# Learning: content-pipeline 幂等检查遗漏 completed 状态

**分支**: cp-04050228-840db267-3c84-4651-97be-ed4b7f  
**日期**: 2026-04-05

### 根本原因

`_startOnePipeline` 的幂等检查 SQL 只包含 `status IN ('queued', 'in_progress')`，遗漏了 `'completed'` 状态。

当 pipeline 经历以下流程时会触发无限循环：
1. content-research 子任务完成（status = completed）
2. 后续阶段（如 content-export）失败
3. 父 pipeline 被标记为 failed
4. Brain 重启后 syncOrphanTasksOnStartup 将父 pipeline 重排队（queued）
5. `_startOnePipeline` 被触发，幂等检查找不到活跃的 research 任务
6. 创建新的 content-research 子任务 → 回到步骤 1
7. 无限循环，产生大量孤儿 research 任务

### 下次预防

- [ ] 幂等检查 SQL 在写注释时明确列出"为什么不包含某状态"，而不是假设 completed 不需要检查
- [ ] 每次修改 `_startOnePipeline` 时，验证所有 status 状态的处理逻辑
- [ ] 考虑增加单元测试覆盖 pipeline re-queue 后重启的幂等场景
