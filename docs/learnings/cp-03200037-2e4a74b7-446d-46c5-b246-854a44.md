# Learning: cross-ref-registry.yaml 跨文件引用声明（2026-03-20）

## 背景

创建 `scripts/cross-ref-registry.yaml`，声明 4 条核心跨文件引用规则（devmode-fields、task-types、skill-names、model-ids），使 SSOT 与消费者的依赖关系机器可读。

### 根本原因

创建纯声明性 YAML 文件时，仍需满足 CI 流程要求：
1. **Learning Format Gate（L1）**：所有走 `/dev` 的 PR 必须包含 `docs/learnings/<branch>.md`，无论改动多小。
2. **Test Coverage Required（L3）**：`feat:` 类型 PR 必须包含测试文件。declarative 配置文件也不例外——要么加测试，要么用 `chore:`/`docs:` 前缀。
3. **Brain task_type 约束（非阻塞）**：`code_quality_review` 和 `prd_coverage_audit` 类型虽在 `VALID_TASK_TYPES` 中声明，但 DB `tasks_task_type_check` 约束尚未同步，注册时报 422。已知问题，不影响主流程。

### 下次预防

- [ ] 纯声明性文件（YAML/JSON registry）使用 `chore:` 前缀而非 `feat:`，避免触发 Test Coverage Required Gate
- [ ] 每次 push 前检查 commit type：若用 `feat:`，必须同时准备测试文件
- [ ] Learning 文件在第一次 push 前就写好并加入 commit，不要等到 CI 失败后才补
- [ ] 向 Brain 注册 `code_quality_review` / `prd_coverage_audit` 失败时，检查 DB schema 是否同步了 task_type
