# Learning: Watchdog Crisis 模式误杀低 RSS 任务

**分支**: cp-04090407-a4df60a9-94b4-4ca9-8cce-bf1b04  
**日期**: 2026-04-09

### 根本原因

`watchdog.js` Crisis 模式（系统压力 ≥ 1.0）中：

```js
const killCount = Math.max(1, ...)  // 保证至少杀 1 个进程
```

`Math.max(1, ...)` 导致即使所有运行中任务的 RSS 都极低（如 2MB），系统压力偶发性 ≥1.0 时仍会强制杀掉 top RSS 的进程。

实际失败场景：
- 任务 `[SelfDrive] 任务失败率根因修复` 启动后约 60 秒内，系统压力 pressure=1.04
- Watchdog 检测到 RSS=2MB（刚启动），仍将其列为 top offender 并 kill
- 历史记录中 3 个 `dev` 类任务被 `Crisis: pressure=1.0X, RSS=2MB` 杀死

### 修复方案

新增 `CRISIS_KILL_MIN_RSS_MB = 20`（可通过环境变量覆盖）：
- Crisis 模式排序后，若 top offender 的 RSS < 20MB，则全部降级为 `warn`，不执行 kill
- 这保留了 Crisis 模式对高内存任务的保护作用，同时不会误杀刚启动的正常轻量任务

### 下次预防

- [ ] Crisis 模式 kill 逻辑改动时，需验证 "单个低RSS任务" 场景不会被误杀
- [ ] 新的 `watchdog-crisis-min-rss.test.js` 覆盖了边界值（RSS=19 不 kill，RSS=20 kill）
- [ ] 如果系统内存很小（< 4GB），可通过 `CRISIS_KILL_MIN_RSS_MB=10` 降低阈值
