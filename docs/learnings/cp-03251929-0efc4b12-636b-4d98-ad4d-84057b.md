# Learning: OKR 全树视图 — /tree API 深度扩展

**分支**: cp-03251929-0efc4b12-636b-4d98-ad4d-84057b
**日期**: 2026-03-26
**PR**: #1578

## 功能概述

扩展 `GET /api/brain/okr/tree` 从 3 层（Vision→Objectives→KRs）到完整 7 层（+Projects→Scopes→Initiatives→Tasks）；新增 `/okr/tree` 前端树组件，支持展开/收起和节点跳转。

### 根本原因

现有 `/tree` 端点仅查询到 KR 层（Vision→Objectives→KRs），无法展示完整 OKR 执行链路。
数据库中 `okr_projects / okr_scopes / okr_initiatives / tasks.okr_initiative_id` 四层表已就位，
但路由实现停留在旧三层设计，导致用户无法在单一视图中看到从 Vision 到 Task 的完整追踪关系。

### 实现要点

- **批量查询代替 N+1**：每层使用 `ANY($1)` 批量查询所有父节点 ID，再用对象 Map 聚合，避免嵌套循环的 N+1 问题
- **向后兼容**：保持原有 `tree[].objectives[].key_results` 结构，在 KR 上新增 `projects` 字段（非破坏性扩展）
- **任务字段精选**：tasks 查询指定字段（不用 SELECT *），避免暴露 prd_content 等大字段

### 下次预防

- [ ] 扩展现有嵌套查询时，先检查现有测试对响应结构的断言范围，确认扩展不破坏断言
- [ ] 多层嵌套批量查询时，注意任务表 tasks 的 `okr_initiative_id` 字段是可空的（通过 `WHERE okr_initiative_id = ANY($1)` 过滤 null）
- [ ] 前端树组件每层单独 useState，不要共用全局展开状态，避免重渲染问题
