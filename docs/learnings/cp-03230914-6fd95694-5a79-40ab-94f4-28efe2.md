### 根本原因

PR #1475 将 `kr-verifier.js` 的 JOIN 从 `goals` 改为 `key_results`，但遗漏了配套的数据库迁移：
- `kr_verifiers.kr_id` 的 FK 约束仍指向 `goals(id)`
- 7 个存活的 verifier 的 `kr_id` UUID 指向旧 goals 表（已被新 key_results 表替代）
- 结果：KR 进度信号卡在 0%，因为 kr-verifier.js 查不到任何 verifier

同批次还发现：
1. `tick-blocked-recovery.test.js` 6 个缺失 mock 导致测试超时（真实 LLM 调用）
2. 其中 `zombie-sweep.js` 未 mock 导致测试期间删除活跃 worktree（worktree 被 `git worktree remove --force` 删除，后续 269 个 test file 找不到）
3. `vi.resetModules()` 在 `isolate:true` 模式下会清空 mock 注册表，导致 mock 失效
