# Learning: content-export 竞态 + generateCards 无降级

**Branch**: cp-04042312-840db267-3c84-4651-97be-ed4b7f
**Date**: 2026-04-05

### 根本原因

1. **竞态：content-export 被 Codex 抢占**
   `tick.js` 的 `dispatchNextTask` 未排除 content-export 任务。`executeQueuedContentTasks`（本地 fire-and-forget）尚未标记任务 in_progress 时，dispatch 系统已将它派给西安 Codex → Codex usage limit 失败 → quarantined → pipeline 终止。

2. **generateCards 无降级**
   所有 findings 的 `brand_relevance < 3` 时（NotebookLM 返回低质量数据），`generateCards` 直接 return false → export 失败。而 `_filterTopFindings`（copywriting 使用）有相同场景的 fallback，export 没有。

### 下次预防

- [ ] 本地执行器管理的任务类型（content-export 等）必须在 `selectNextDispatchableTask` 中显式排除，防止 dispatch 系统双重派发
- [ ] 新增/改写阶段函数时，对比 copywriting 的 `_filterTopFindings` 是否有 fallback，保持一致性
- [ ] Content-export 被设计为本地 Node.js 执行（`executeExport`），但 `LOCATION_MAP['content-export'] = 'xian'` 的注释未更新，下次清理时一并修正
