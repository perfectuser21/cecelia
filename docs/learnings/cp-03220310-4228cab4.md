# Learning: 重构 cecelia-routes 降低复杂度

## 根本原因
inferStepProgress（复杂度 15）和 buildCheckpoints（复杂度 13）在同一函数体内
处理多种 status 分支，导致嵌套深、分支数多。

### 下次预防
- 函数内出现 4+ 个 if/else 且涉及多 status 判断时，优先提取状态判断到独立辅助函数
- 对 DEV_STEPS.map 内的 if/else 链，提取为 getXxxStatus(status, ...) 辅助函数
- [ ] 新增路由函数前先确认复杂度 ≤ 10（可用 scanner.analyzeComplexity 本地验证）
