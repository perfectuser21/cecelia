# Learning: Dashboard 3大模块 bug修复 + 集成测试

**分支**: cp-04100051-ceaea7ee-bf2d-4015-9f75-6e4b20

### 根本原因

1. **RoadmapPage SelfDrive 数据源错误**：`fetchAll` 中事件查询使用了 `event_type=task_dispatched`，但注释和功能设计均为 `cycle_complete`。导致 SelfDrive 面板显示任务派发事件而非大脑思考记录，20分钟演示中面板内容与标签不符。

2. **server-perf.test.ts 本地运行失败**：测试文件顶层 `import compression from 'compression'` 导致在未运行 `npm ci` 的本地环境中整个测试套件无法加载（ERR_MODULE_NOT_FOUND）。CI 中 `npm ci` 会安装 compression，但本地开发时无法快速运行测试。

3. **ViralAnalysisPage 无测试覆盖**：KR5 相关页面 ViralAnalysisPage 完全缺少测试文件，之前修复过3个阻断bug但未补充测试。

### 下次预防

- [ ] 新增 API 请求时，对照注释和功能设计确认参数值（event_type 等枚举值）而不仅看代码存在
- [ ] 测试文件中需要可选依赖时，使用 `vi.mock()` 而不是顶层 `import`，确保本地开发也能运行测试
- [ ] 每个新页面/修复页面，在同一 PR 中同步添加基础渲染测试（至少 5 个用例）
