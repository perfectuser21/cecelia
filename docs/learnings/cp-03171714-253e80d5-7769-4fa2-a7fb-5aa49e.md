# Learning: 修复 Brain L3 CI 测试失败（2026-03-17）

## 任务概述

修复 CI L3 每天挂掉的 20 个测试（startup-sync 3 个 + blocks.test.js 17 个）。

---

## Learning 1：测试没跟上业务逻辑变更导致长期 CI 失败

### 根本原因

PR #1025 修改了孤儿恢复逻辑（watchdog_retry_count < 2 → requeue，不再直接 fail），
但对应的 startup-sync.test.js 3 个测试用例仍验证旧行为（orphans_fixed=1、status=failed）。
业务逻辑 PR 合并后，没有同步更新测试，导致测试与实现永久不匹配。

### 下次预防

- [ ] 修改孤儿处理路径等状态机逻辑时，同 PR 同步更新对应测试断言
- [ ] PR description 中明确列出"测试变更"：旧行为 vs 新行为，reviewer 检查
- [ ] 如果测试会失败而逻辑正确，在 PR title 加 [TEST-UPDATE-PENDING] 标记，下次 PR 必须修

---

## Learning 2：集成测试混入单元测试套件导致 CI 失败

### 根本原因

blocks.test.js 是集成测试（需要真实 Brain API + PostgreSQL），
但被放在与单元测试相同的目录下，CI 运行 npm test 时一起跑，
导致 17 个测试因无法连接服务而失败。

### 下次预防

- [ ] 新建需要外部服务的测试时，必须加 describe.skipIf(!process.env.INTEGRATION_TESTS) 标记
- [ ] 集成测试文件命名约定：*.integration.test.js 或加注释说明运行方式
- [ ] INTEGRATION_TESTS=true npm test 作为本地完整测试命令，CI 只跑单元测试

---

## Learning 3：DoD BEHAVIOR test 不能用 npx vitest（CI 无 Brain 依赖）

### 根本原因

Task Card 的 BEHAVIOR DoD 条目写了 manual:bash -c "cd packages/brain && npx vitest run ..."，
但 DoD Verification Gate 运行时只安装 engine 依赖，没有 Brain 的 node_modules，
导致 npx vitest 找不到命令，exit code 1，CI Gate 失败。

### 下次预防

- [ ] manual: 命令只用 grep/ls/cat/node，禁止 npx vitest / npm test（参见 MEMORY.md 规则）
- [ ] BEHAVIOR 验证测试文件内容（断言存在）用 grep -q "expect(...)"，不用运行 vitest
- [ ] 写完 DoD 后本地运行 bash packages/engine/scripts/devgate/dod-execution-gate.sh 验证
