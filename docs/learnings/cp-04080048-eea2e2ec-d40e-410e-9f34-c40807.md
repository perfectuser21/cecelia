# Learning: KR进度排查 — kr-verifier 未写 current_value

**PR**: cp-04080048-eea2e2ec-d40e-410e-9f34-c40807  
**任务**: KR进度更新机制失效排查 — 0%进度不可信

## 现象

`/api/brain/okr/current` 接口所有 KR 的 `current_value` 永远为 `0.00`，
即使 `kr_verifiers` 表已采集到真实指标（如发布量=4）。

## 根本原因

`kr-verifier.js` 的 `UPDATE key_results` 语句只写了：
- `progress`（百分比）
- `metadata.metric_current`（JSON 字段）

**遗漏了** `current_value`（原始数值列）。

`kr_verifiers.current_value` 是 verifier 自己的追踪字段，
`key_results.current_value` 是 KR 对外暴露的原始值，两者没有同步。

## 修复

在 `kr-verifier.js` 的 UPDATE 语句中加入 `current_value = $2`，
参数由 `currentValue.toString()` 改为 `currentValue`（数值类型，直接写入 numeric 列）。
