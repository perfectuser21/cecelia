# Learning: extractVerdictFromResult 无法从 claude JSON output 提取 verdict

**分支**: cp-04080059-1746f39c-cda3-40bc-9c21-91678b
**日期**: 2026-04-08

### 根本原因

`extractVerdictFromResult` 的 `object` 分支用 `res?.result?.verdict` 读取嵌套 verdict。
但 claude `--output-format json` 输出的 `result` 字段是**字符串**而非对象（`{type, result: "...JSON or text...", ...}`），字符串上的 `.verdict` 永远是 `undefined`，导致 verdict 始终返回 null。

### 修复方案

在 `object` 分支，先取 `res.verdict` 或 `res.result.verdict`（仅当 result 是 object 时）；若两者都为 null，检查 `res.result` 是否为 string，若是则执行与 string 分支完全相同的 JSON.parse + regex 搜索。

### 下次预防

- [ ] 写 result 提取逻辑时，**明确区分** `res.result` 可能是 object 或 string 两种情况
- [ ] harness 回调链路测试时，用真实 `claude --output-format json` 输出格式（含 string result）做覆盖测试，而非只测 `{verdict: "..."}` 直接字段
- [ ] `extractVerdictFromResult` 类似"多格式解析"函数应有单元测试覆盖所有输入变体
