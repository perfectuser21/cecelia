# Learning: fix(engine): /dev Stage3 drift detection + spec_review 测试层检查

### 根本原因

Stage 3 push 前缺乏 scope 检查机制，改动文件可能悄悄超出 Task Card 声明的范围而不被察觉。

spec_review 只验证 Test 命令格式和可执行性，未检查测试层级（unit/integration/e2e）与被测行为是否匹配，导致 [BEHAVIOR] 条目用静态文件检查代替运行时验证的问题无法被发现。

这两个缺口独立存在：drift detection 解决实现范围漂移问题，维度 F 解决测试策略合理性问题。
