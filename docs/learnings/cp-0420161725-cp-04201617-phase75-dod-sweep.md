**日期**：2026-04-20

## 背景

Phase 8.3（PR #2463）在实施过程中撞到了一条死 DoD：

```
Test: manual:node -e "const fs=require('fs');const p=require('os').homedir()+'/.claude/skills/harness-report/SKILL.md';if(!fs.readFileSync(p,'utf8').includes('Initiative 级 Report'))process.exit(1);console.log('PASS')"
```

两处错：
1. 引用 `~/.claude/skills/harness-report/SKILL.md` — 这是**本地开发环境**的用户 global skill 路径，CI runner（ubuntu-latest）上不存在
2. 断言文件里含 "Initiative 级 Report" 字符串 — 但 repo 里 `packages/workflows/skills/harness-report/SKILL.md` 根本没有这段文字（DoD 合并时就是假的）

这条死 DoD 导致**无关 PR 的 CI 失败**（因为 DoD BEHAVIOR gate 会在每个含 DoD.md 的 PR 上重跑所有 node 命令）。

### 根本原因

1. **DoD 只在 PR 提交时跑一次，合并后不再 enforce**：合并时命令通过，但 AI 写的命令可能是"运气通过"（比如别的 PR 改了那个文件恰好让断言成立）。更不用说完全合理的 `~/.claude/skills/` 路径在 CI 环境不存在，AI 本地跑通但 CI 一跑就炸。
2. **历史 per-PR DoD/PRD 残留积累**：`.github/workflows/cleanup-merged-artifacts.yml` regex 漏抓 `.dod-` 前缀（只写了 `.prd-|.task-|DoD\.cp-|PRD\.cp-|TASK_CARD\.cp-`），导致 3 个 `.dod-cp-*.md` 遗留在仓库根。同时仓库累积了 152 个历史 per-PR 副本在 root / packages/brain / packages/quality / packages/workflows。
3. **没有 manual: 命令白名单回归测试**：`check-manual-cmd-whitelist.cjs` 和 `check-dod-mapping.cjs` 都有假测试检测，但没有单元测试验证它们能抓住边界 case — 一旦脚本逻辑退化（比如白名单误漏 prefix），不会有 CI 告警。

### 修复范围

**扫描报告**：全 repo 扫描 404 个含 `Test: manual:` 的 md 文件，共 460 个 `node -e` 命令：

- DEAD：175（38%）

死命令分布：
- **145 个（82%）**在历史 per-PR 残留（`.task-*` / `.dod-*` / `DoD.cp-*` / `sprints/*/task-card.md`）— 这些 PR 已合并，残留是无效状态
- **8 个（5%）**是设计文档/skill 模板里的示例（`"..."` 占位符）— false positive，不是真 DoD
- **22 个（12%）**是 `sprints/` 目录下早期迭代的 task-card.md — 留给 harness owner 单独清理

**本 PR 修复**：
1. `git rm` 152 个历史 per-PR 残留（不含 `docs/archive/` 归档）
2. `git rm` 9 个根 / packages/ 内 stale 的 `.dod.md` / `.prd.md` / `.prd.md.old` / `TASK.md`
3. 修 `cleanup-merged-artifacts.yml` regex：`^(\.prd-|\.task-|\.dod-|DoD\.cp-|PRD\.cp-|TASK_CARD\.cp-)` — 补 `.dod-` 前缀
4. 新增 `packages/engine/tests/dod/dod-manual-commands.test.ts`：11 个断言覆盖白名单（PASS `node/curl/bash/psql/npm/npx/playwright`，FAIL `grep/ls/cat/echo/printf/find/sed/awk`）
