# Learning: 记忆系统 PR5 — Dopamine RPE 奖赏预测误差

Branch: cp-03240631-8b7fcd20-6672-4b09-bdbb-52b84f
Date: 2026-03-24

### 根本原因

RPE 测试中 `mockQuery.mock.calls.find()` 条件只检查 `c.includes('rpe_signal')`，
而 `computeExpectedReward` 的 SELECT 查询同样包含 `'rpe_signal'` 字符串，
导致 find 返回了错误的调用（SELECT 而非 INSERT），
`insertCall[1][0]` 是 taskType 字符串而非 JSON payload，JSON.parse 失败。

### 下次预防

- [ ] 测试中 findCall 条件必须足够精准，同时匹配操作类型（INSERT/SELECT/UPDATE）和关键词
  正确写法：`c[0].includes('INSERT') && c[0].includes('rpe_signal')`
- [ ] 同一测试文件中有多个 query 用同一关键词时，find 条件必须加额外限定符
- [ ] 写测试前先列出该函数会触发的所有 pool.query 调用，确认 mock 顺序和 find 条件不冲突
