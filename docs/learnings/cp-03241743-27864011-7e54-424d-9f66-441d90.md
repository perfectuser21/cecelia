# Learning: 记忆系统 PR8 — 接入 runConversationConsolidator 到 tick

**Branch**: cp-03241743-27864011-7e54-424d-9f66-441d90
**Date**: 2026-03-25

## 功能

在 tick.js 中以每 5 分钟频率调用 `runConversationConsolidator()`，使 Brain 在用户对话空闲 ≥30 分钟时自动压缩对话内容写入 `memory_stream`。

### 根本原因

`conversation-consolidator.js` 已实现完整逻辑但未接入 tick 调度，导致对话总结永远不会自动运行。

tick.js 中其他模块（如 runPipelinePatrol）均通过相同的 import + state var + interval + call block 模式接入，
但 conversation-consolidator 模块在实现时缺少了这最后一步 tick 接入。

解决方式：在 executeTick() 的感知模块区域新增 5 分钟间隔调用块，与 runPipelinePatrol 模式完全一致。

### 修复

在 `tick.js` 中添加：
1. `import { runConversationConsolidator } from './conversation-consolidator.js'`
2. `_lastConversationConsolidatorTime = 0`（loop state）
3. `CONVERSATION_CONSOLIDATOR_INTERVAL_MS`（5min，env var 可覆盖）
4. `executeTick()` 中 pipeline patrol 之后的间隔检查调用块（fire-and-forget + catch）
