### 根本原因

`packages/brain/server.js` 中 `app.use('/api/brain', contentPipelineRoutes)` 挂载在第 166 行，
而 `app.use('/api/brain', brainRoutes)` 在第 177 行。
`contentPipelineRoutes` 包含 `router.get('/:id/stats', ...)` 路由，
Express 将 `/api/brain/publish-results/stats` 的 `publish-results` 解析为 `:id`，
在 `brainRoutes` 中的 `publishResultsRouter`'s `GET /publish-results/stats` 执行前触发，
导致 DB 报错 "invalid input syntax for type uuid: publish-results"。

### 修复方案

将 `app.use('/api/brain', brainRoutes)` 移至 `app.use('/api/brain', contentPipelineRoutes)` 之前，
确保更具体的路由（publish-results/stats）先被匹配。

### 下次预防

- [ ] 新增通配路由（`/:id/*` 模式）的路由文件挂载到 `/api/brain` 时，必须放在所有具体命名路由之后
- [ ] Brain server.js 新增路由挂载时，需检查是否存在 `/:id/xxx` 模式与现有路由冲突
- [ ] 考虑将 contentPipelineRoutes 的 `/:id/*` 路由挂载到 `/api/brain/pipelines` 而非 `/api/brain`（长期优化）
