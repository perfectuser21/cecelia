# Learning: CI Gate — 新增 hook/devgate 文件必须有 RCI 条目

**Branch**: cp-03231615-5e7872ce-d4f9-4986-ab08-0f47c5
**Date**: 2026-03-23
**PR**: #1446

### 根本原因

Task Card 中的 Test: 字段使用了 `packages/engine/tests/devgate/...` 格式路径，但 `check-dod-mapping.cjs` 的 Phase 3 正则只接受 `tests/` 前缀（不含 `packages/engine/`）。导致 CI DoD Verification Gate 在 Phase 3 报"测试文件不存在"，同时"未验证项检查"也认为是 `[ ]` 未勾选（因为 CI 读的是旧提交的 Task Card）。

两个独立问题叠加：
1. Task Card 中的 `Test:` 字段路径格式与 `check-dod-mapping.cjs` 解析规则不匹配
2. 本地更新的 Task Card（含 `[x]` + 正确 `manual:` 格式）未包含在首次 push 的提交里

### 下次预防

- [ ] 在写 Task Card 的 Test: 字段时，优先使用 `manual:node -e "..."` 格式，而非 `tests/path/to/file.ts` 格式，除非路径完全符合 `tests/` 前缀（相对于仓库根目录）
- [ ] Task Card 的所有 `[x]` 和 `Test:` 字段修改完成后，立即 `git add .task-*.md && git commit`，不要等到 version bump 提交时一起处理
- [ ] `manual:` 命令白名单：只允许 `node`/`npm`/`curl`/`bash`/`psql`。`grep`/`ls`/`cat` 均无效
- [ ] 第一次 push 前检查：`git diff --name-only HEAD~1..HEAD` 确认 Task Card 已包含在提交中
- [ ] 根目录 `package-lock.json` 同样需要随版本 bump 一起提交（CI 一致性检查可能依赖它）
