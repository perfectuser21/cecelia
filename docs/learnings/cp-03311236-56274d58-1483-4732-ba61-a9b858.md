# Learning: KR verifier SQL 查询错误导致进度全 0%

## 根本原因

Brain 的 `kr_verifiers` 表中，4 个 KR 的 SQL 查询指向了**空表或错误表**：

| KR | 错误表 | 真实数据在 |
|----|-------|----------|
| 自动发布 | `publish_results`（空） | `content_publish_jobs` |
| 内容生成 | `content_topics`（空） | `content_publish_jobs` |
| 系统稳定 | `self_healing_log`（空） | `alertness_metrics` |
| 算力全开 | tasks in-progress snapshot（公式错） | tasks 30天完成率 |

诊断过程：先从 OKR API 发现 `管家闭环` 40% 正确，其余 0%。然后直接运行各 verifier 的 SQL 查询，确认 `管家闭环` 的 `daily_logs` 有数据（12行），其他表为空或逻辑错误。

## 下次预防
