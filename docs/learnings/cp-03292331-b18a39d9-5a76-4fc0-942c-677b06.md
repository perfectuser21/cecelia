# Learning: Pipeline Security Bug Fix

## 任务概述

修复 /dev 工作流中4个 P0/P1 安全漏洞（Engine v13.65.1）

### 根本原因

1. **seal 文件仅检查存在性**：devloop-check.sh 条件 1.5 只检查 `.dev-gate-spec.<branch>` 是否存在，不检查 `verdict` 和 `divergence_count` 字段的有效性。攻击面：创建空 seal 文件或省略关键字段即可绕过 Gate。

2. **fallback 路径降级不安全**：stop-dev.sh 在 devloop-check.sh 未加载时执行旧内联逻辑，该逻辑缺少 seal 验证、divergence_count 门禁等新安全机制，相当于提供了一个无保护的降级通道。

3. **flock 不可用时静默成功**：lock-utils.sh `acquire_dev_mode_lock` 在 flock 缺失时返回 0（成功），调用方无法感知锁获取失败，可能导致并发写入 .dev-mode 造成数据损坏。

4. **PR 合并前无二次确认**：devloop-check.sh 直接执行 `gh pr merge`，不重新检查 PR mergeable 状态和最新 CI 结论，可能在 PR 产生冲突或 CI 重新运行失败的情况下强行合并。

### 下次预防

- [ ] 新增 Gate seal 文件时，默认模板应包含内容校验逻辑（不止文件存在性）
- [ ] 每次新增安全机制后，检查是否有 fallback/降级路径会绕过新机制
- [ ] flock 等系统工具调用的返回值处理：缺失时必须硬错误，不能静默跳过
- [ ] 有状态操作（合并 PR）前，总是重新验证前提条件而非信任之前的检查结果
