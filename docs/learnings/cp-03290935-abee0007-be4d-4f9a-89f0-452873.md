# Learning: Planner 独立 subagent — Stage 1 Task Card 生成从主 agent 拆出

**分支**: cp-03290935-abee0007-be4d-4f9a-89f0-452873
**日期**: 2026-03-29
**Engine 版本**: 13.58.0

---

### 根本原因

Stage 1 中主 agent 直接生成 Task Card，导致主 agent 的编码规范（CLAUDE.md）和 Brain 调度上下文（OKR/KR/Project 层级）污染了需求描述过程。Planner 应只知道"要做什么"（任务描述）和"系统有什么"（SYSTEM_MAP），不应知道"怎么写代码"。这与 Stage 2 的 Generator subagent 隔离原则相同，但 Stage 1 一直没有做对称的拆分。

---

### 解决方案

在 `01-spec.md` 1.2 节引入 Planner subagent 模式：
- 主 agent 只传：任务描述 + `docs/current/SYSTEM_MAP.md`
- 禁止传入：CLAUDE.md、Brain context、代码库细节
- Planner 只输出 WHAT（行为描述），不写 HOW（实现细节）
- 新建 `packages/engine/skills/dev/lib/planner-prompt.md` 作为 prompt 模板
