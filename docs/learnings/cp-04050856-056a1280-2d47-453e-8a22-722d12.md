# Learning: 小红书发布脚本两处修复

## 根本原因

1. **xian-mac 跳板 SSH 密钥失效**：美国 Mac mini (`~/.ssh/id_rsa`) 的公钥未加入 xian-mac 的 `authorized_keys`，导致 `scpImagesToWindows` 第一步就 ETIMEDOUT 或 Permission denied。US Mac mini 实际上可以通过 Tailscale 直连 `xian-pc`（Windows），无需跳板。

2. **图文 tab 文字变更**：小红书创作平台将 tab 文字从 `'图文'` 改为 `'上传图文'`，导致 Step 2 无法点击切换，页面停在「上传视频」tab，后续上传按钮坐标出现负值。

## 下次预防

- [ ] `scpImagesToWindows` 改为优先直连 `xian-pc`，降级才走 xian-mac 跳板（已修复）
- [ ] Step 2 tab 选择兼容 `['上传图文', '图文']` 多种文字（已修复）
- [ ] Tailscale relay 连接需要更长超时（`ConnectTimeout=30`，execSync timeout 45-60s）
- [ ] 发布前检查截图 `01-nav.png`，确认 tab 已切到「上传图文」再继续
