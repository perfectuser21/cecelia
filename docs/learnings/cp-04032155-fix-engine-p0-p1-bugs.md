## 修复内容

### Bug 1 (P0): check-version-sync.sh — exit 0 掩盖版本读取失败
- **根本原因**: `BASE_VERSION` 为空时写的是 warning + `exit 0`，导致 package.json 损坏时 CI 版本同步检查静默通过
- **修复**: 改为 `exit 1`，失败时正确阻止 CI

### Bug 2 (P1): devloop-check.sh — date fallback=0 引发误报
- **根本原因**: `_se=$(... || echo 0)` 在 date 解析失败时设为 `0`，导致 `$(( $(date +%s) - 0 ))` 返回当前 unix 时间戳（~17亿），永远满足 `> 5400`，每次都误报"CI 已 pending 90+ 分钟"
- **修复**: `_se` 初始化为空字符串，增加 `-n "$_se"` 卫语句跳过 date 解析失败的情况

### Bug 3 (P1): execution-logger.sh — sed `%→%%` 污染日志
- **根本原因**: `sed 's/%/%%/g'` 把所有 `%` 替换为 `%%`，但 `%` 在 sed 中无特殊含义，这是无效操作，且会破坏日志内容（`50%` → `50%%`）
- **修复**: 删除该 sed 命令

### Bug 4 (P1): fire-learnings-event.sh — awk 只读最后一个 `###` 节
- **根本原因**: 第一个 awk 每遇到 `###` 就重置 content，最终只有最后一节被保留。当 `### 下次预防` 不是最后一节时，预防措施丢失
- **修复**: 简化为直接全文 awk 搜索 `### 下次预防` 节，无需两段 awk
