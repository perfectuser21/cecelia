# Learning: 修复 probeEvolution 查询错误表（2026-03-20）

## Probe 查询表与写入表不一致导致误报故障

### 根本原因

`probeEvolution()` 在 `packages/brain/src/capability-probe.js` 中查询 `cecelia_events` 表（`event_type LIKE '%evolution%'`），但 `evolution-scanner.js` 的实际写入目标是 `component_evolutions` 表。两表不同，导致 Probe 永远返回 `ok: false`，即使 `component_evolutions` 7 天内有 147 条记录也视为故障。

### 下次预防

- [ ] 新增 Probe 检查点时，先通过数据库确认被监控数据实际存在哪张表（`\d+ <table>`），再写查询逻辑，不要依赖命名推断
- [ ] Probe 函数的 detail 字段应包含被查询的表名或字段名，方便快速诊断（如 `7d_pr_evolutions=147 last_date=2026-03-18`）
- [ ] 修复 Probe 误报后，验证方式：直接用 SQL 查询正确表，而非等待服务器重启后调用 API（服务器运行 main 分支代码，worktree 中的修改无法直接验证 BEHAVIOR DoD）
