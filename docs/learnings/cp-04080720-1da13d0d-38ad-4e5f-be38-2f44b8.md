## sprint-generator SKILL.md v3.1.0 — context 解析缺失导致 Generator 无法执行（2026-04-08）

### 根本原因

v3.0.0 重写专注于"不再自写合同"的概念，但忽略了 Generator session 实际执行时的三个必要环节：
1. **${SPRINT_DIR} 未定义**：Brain 通过提示词头部注入 `**sprint_dir**:`，但 SKILL.md 直接使用 `${SPRINT_DIR}` 变量，session 无法解析。
2. **跳过 worktree 创建**：Mode 1 直接写 `git add / git commit`，在主 repo 根目录执行会被 hook 阻止。
3. **Mode 2 极简**：仅一行"读 eval-round-N.md → 修复 → Push"，无法实际执行。

附带发现：`detect-review-issues.js` 对 DeepSeek 输出 "未发现严重问题（🔴）" 误判为严重问题（正则方向错误）。

### 下次预防
