# Learning: Self-Drive goal_id 自动挂载

**分支**: cp-03200035-92c35f6b-51d9-4252-bb5e-4c0918
**日期**: 2026-03-20

## 问题描述

Self-Drive 创建的所有任务 `goal_id = NULL`，Area 调度器（area-scheduler.js）通过 `tasks.goal_id → goals.domain` 判断业务线，NULL 全部归入 DEFAULT_LINE（zenithjoy）。

## 根本原因

`self-drive.js` 调用 `createTask()` 时未传 `goal_id`。`trigger_source='self_drive'` 在 `isSystemTask()` 白名单中，跳过了 goal_id 强制校验，所以没有报错，只是静默地产生了错误归属。

## 修复方案

1. LLM prompt 增加 `area` 输出字段，让 LLM 判断任务归属业务线（cecelia/zenithjoy/investment）
2. 新增 `getGoalIdForArea(area)` 查询 goals 表活跃 OKR
3. `createTask` 调用时传入 `goal_id`，找不到时 fallback null（向后兼容）

## 下次预防

- [ ] 新增 `trigger_source` 时，即使在 `isSystemTask()` 白名单中，也应检查是否需要 `goal_id`
- [ ] 自动任务（self_drive/auto_fix/cortex）创建时，应尽量传 `goal_id` 以确保 Area 归属正确
- [ ] 在 `createTask` 的 jsdoc 中标注：即使 goal_id 非必填（系统任务），传入也有助于 Area 调度
