# Learning: fix(bridge) per-request timeout 被 120s 上限截断

### 根本原因

`cecelia-bridge.js` 的 `/llm-call` 路由中，`Math.min(timeout || BRIDGE_TIMEOUT_MS, BRIDGE_TIMEOUT_MS)` 把第二参数也设为全局默认值，导致即使调用方显式传入更大的 timeout，也始终被截断到 120s。Cortex Opus 需要 300s，结果在 121s 时被 SIGTERM（exit 143）。

### 下次预防

- [ ] Bridge 新增超时相关逻辑时，区分「默认值」和「上限值」两个概念
- [ ] 引入 per-request 可配置超时时，上限用独立环境变量（`*_MAX_*`）而非复用默认值变量
- [ ] 代码 review 时对 `Math.min(x, SAME_VARIABLE)` 形式保持警惕——如果两参数用同一变量，通常意味着 per-request override 失效
