# Learning: 固化 CECELIA_MAX_SEATS=10 硬上限

**Branch**: cp-03181603-1db4c758-58f3-4202-8725-b88fdf
**Date**: 2026-03-18
**Task**: D2-hardcap — 硬编码 CECELIA_MAX_SEATS=10，防止高并发下资源耗尽

---

### 根本原因

1. **docker-compose.yml 用了 `${CECELIA_MAX_SEATS:-8}` 而非硬编码值**
   默认值 8 在没有 `.env` 文件时会被使用，导致生产容器可能跑 8 个 slot 而非预期的 10。
   应当在 compose 文件中直接写死，避免"环境变量未设置时默默降级"。

2. **.env.example 缺少 CECELIA_MAX_SEATS 的说明**
   新成员部署时不知道此变量存在，也不知道为何要设置，导致可能被遗漏。

3. **DEFINITION.md 的产能模型说明缺少 EFFECTIVE_MAX_SEATS 概念**
   文档只描述了 MAX_SEATS（物理上限），没有说明 `min(CECELIA_MAX_SEATS, PHYSICAL_CAPACITY)` 的双层模型。

---

### 下次预防

- [ ] 所有 docker-compose.yml 环境变量：硬编码关键限制值，不用 `${VAR:-默认}` 形式
- [ ] .env.example 每个变量必须有注释说明用途和默认值
- [ ] DEFINITION.md 产能公式区域，改变公式时同步更新（facts-check.mjs 强制校验）
- [ ] 双层容量模型（physical + budget cap）在测试中验证 `Math.min(cap, physical)` 行为
