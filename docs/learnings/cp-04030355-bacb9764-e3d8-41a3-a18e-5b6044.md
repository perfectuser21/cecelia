# Learning: rumination probe 链路故障诊断

## 根本原因

`shouldRunDaily()` 的 5 分钟触发窗口（UTC 18:00-18:05）过窄，在以下叠加场景下导致当天 synthesis_archive 无写入：

1. **路径 1 阻断**：`runRumination` 有 `isSystemIdle()` 守卫。当系统有 in_progress 任务时（如 4 个 executor 占用），路径 1 全天跳过。
2. **路径 2 窗口太窄**：`runSynthesisSchedulerIfNeeded` 仅在每天 UTC 18:00-18:05 触发。若 tick 在窗口期内错位（任何原因），当天写入机会彻底消失。

实测（2026-04-02 UTC）：synthesis_archive 整天无 period_start='2026-04-02' 条目，导致 `probeRumination` 在 April 3 00:45 UTC 之后持续失败，触发 P0 诊断任务。

## 修复方案

**扩宽 `shouldRunDaily()` 为 UTC 18:00~23:59（6 小时宽窗口）**：

```js
// Before: 5 分钟硬窗口
return now.getUTCHours() === DAILY_HOUR_UTC && now.getUTCMinutes() < DAILY_WINDOW_MIN;

// After: 6 小时宽窗口，幂等靠 hasTodaySynthesis
return now.getUTCHours() >= DAILY_HOUR_UTC;
```

`runDailySynthesis` 内部的 `hasTodaySynthesis()` 负责防重复，所以宽窗口不会重复调用。
