# Learning: 认证层故障排查 — auth 级联 + 路由双前缀 Bug

**分支**: cp-04080609-5248c043-0eb8-4d74-88b5-f862b8
**日期**: 2026-04-08

---

### 根本原因

1. **account3 OAuth token 过期静默级联**：token 过期后，所有派发到 account3 的任务批量返回 401，被标记为 `failure_class: auth`，隔离到 `quarantined` 状态。凭据刷新后没有自动恢复机制，13 个业务任务永久沉没。

2. **路由路径重复 Bug**：`infra-status.js` 定义 `router.get('/credentials/health', ...)` 但 `server.js` 已将其挂载在 `/api/brain/credentials`，导致实际路径是 `/api/brain/credentials/credentials/health`，API 返回 404。

3. **自驱恐慌级联**：auth 失败拉低成功率 → SelfDrive 创建大量诊断任务 → 诊断任务也因同一 auth 问题失败 → 恶性循环。该循环已通过 `auth_failed` 排除成功率计算部分缓解，但积压任务仍需恢复。

---

### 下次预防

- [ ] 路由路径定义时，`router.get` 的路径与挂载路径不要重复（`app.use('/api/brain/credentials', router)` + `router.get('/credentials/health')` = 404）
- [ ] 凭据恢复后（token 刷新），tick.js 会自动重排队 auth 隔离任务（`recoverAuthQuarantinedTasks`）
- [ ] pipeline_rescue 的 auth 失败（~159个）是僵尸 worktree 产生的，不需要恢复，SQL 层已排除
- [ ] `account_usage_cache.is_auth_failed` 是熔断状态，需要同时检查本地文件 + DB 两层才能确认凭据健康
