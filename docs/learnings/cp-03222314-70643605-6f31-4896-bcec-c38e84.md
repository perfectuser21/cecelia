## 背景

verify-step.sh Gate 2 跑 DoD Test 通过后未写回 `[x]`，导致状态机（Task Card）与 CI DoD 完整性检查不同步。

### 根本原因

`verify-step.sh` Gate 2 的 `while IFS= read -r line` 循环只执行 DoD Test 命令、统计通过/失败数，但没有回写结果到 Task Card。导致所有 Test 本地通过后，Task Card 中 `[BEHAVIOR]` 条目仍然是 `[ ]`，CI 的 `check-dod-mapping.cjs` 看到未勾选项报错。

同时 `devloop-check.sh` 也缺少对应的 DoD 完整性验证，无法在推进到 PR 创建前拦截未勾选的状态。

### 下次预防

- [ ] 每次新增 "执行某命令并判断结果" 的 Gate 逻辑时，同步考虑"结果要不要写回状态文件"
- [ ] `devloop-check.sh` 条件新增时检查与 verify-step.sh Gate 是否形成完整闭环（Gate 写 → devloop-check 校验）
- [ ] Engine hooks 改动必须同步检查 CI L2 版本一致性要求，及时 bump 5 个文件
