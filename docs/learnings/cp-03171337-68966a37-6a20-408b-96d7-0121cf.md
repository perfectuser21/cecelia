# Learning: 接入 check-coverage-completeness.mjs 到 CI

**日期**: 2026-03-17

---

## 任务背景

PR #1004 关闭时遗漏了将 `check-coverage-completeness.mjs` 接入 CI。脚本本身在 commit 69800a629 已加入 main，但 `ci-l2-consistency.yml` 的 `brain-l2` job 未调用它。

---

## 根本原因

PR 关闭时遗漏 CI 接入步骤是常见错误：脚本文件存在于仓库，但没有任何 CI job 实际调用它，导致守护规则形同虚设。

### 下次预防

- [ ] 新增 devgate 脚本时，必须同时检查是否已接入对应 CI job
- [ ] PR 描述中显式列出"新增脚本 X → 接入 CI job Y"，避免遗漏
- [ ] 提交前运行 `grep <script-name> .github/workflows/*.yml` 验证已接入

---

## 实现

变更极简：在 `brain-l2` job 末尾追加一个 step：

```yaml
- name: Check coverage completeness (source files must have tests)
  run: node packages/engine/scripts/devgate/check-coverage-completeness.mjs
```

脚本已存在，Node.js 环境已配置，无需额外依赖安装。

---

## 注意事项

- `brain-l2` job 仅在 `packages/brain/` 变更时触发，所以该守护规则只在 brain 变更时执行
- 若要让该检查在 engine 变更时也执行，可考虑在 `engine-l2` job 中同样添加此步骤（当前 PRD 只要求 brain-l2）
- 脚本运行结果为 PASS with warnings（hooks/src 覆盖率不完整），不影响 CI 通过
