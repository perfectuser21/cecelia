# Learning: harness verdict 纯文本提取 — 测试覆盖补全

## 任务
fix(brain): sprint_contract_propose verdict 纯文本提取覆盖

## 根本原因

PR #2084 修复了生产代码（`extractVerdictFromResult` + 纯文本 fallback），
但测试文件的 `simulateHarnessCallback` 仍使用旧逻辑：
```js
// 旧（只检查对象）
const proposeVerdict = (result !== null && typeof result === 'object')
  ? (result.verdict || result?.result?.verdict)
  : null;
```

同时，`extractVerdictFromResult` 接受 `validVerdicts` 参数但从不使用，
导致调用方以为可以过滤 verdict，实际并无效果。

## 修复

1. **`extractVerdictFromResult` 添加 `_check()` 辅助** — 实际过滤 validVerdicts
2. **`simulateHarnessCallback` 对齐生产代码** — 添加纯文本 fallback 路径
3. **4 个新测试** — 覆盖纯文本 PROPOSED/APPROVED 场景（plain string + Claude CLI JSON）
