# Learning: Scanner 误判 Brain 内嵌能力为孤岛

## 分支
`cp-03192015-c79db6c0-f912-4b98-8e35-8490d0`

### 根本原因

`capability-scanner.js` 只检查两类活动迹象：
1. `related_skills` 在 `run_events`/`tasks` 表中的调用记录
2. `key_tables` 是否有数据

Brain 内嵌能力（熔断保护、看门狗、情绪感知、自愈等）直接运行在 Brain 进程内，不走外部 skill 调用，部分意识类能力（migration-094 新增）也没有配置 `key_tables`。导致 Scanner 把这些核心保护机制误标为孤岛（island），影响后续自主决策的准确性。

### 下次预防

- [ ] 新增 Brain 内嵌模块时，在 `BRAIN_ALWAYS_ACTIVE` 或 `BRAIN_EMBEDDED_SOURCES` 中同步登记，避免被误判为孤岛
- [ ] `BRAIN_EMBEDDED_SOURCES` 中的 source 名称须与对应模块实际写入 `cecelia_events` 的 source 字段严格一致（可用 `grep "emit(" packages/brain/src/<module>.js` 确认）
- [ ] migration-094 等新增意识类能力注册时，若无 `related_skills` 和 `key_tables`，必须同步在 Scanner 白名单中登记
