# Learning: Scanner 新增 Brain 内嵌能力识别规则

## 任务概述
修正 `capability-scanner.js`，使其能正确识别已内嵌在 Brain 进程中运行的能力，避免将熔断保护、看门狗、情绪感知、自愈免疫等能力误标为孤岛（island）。

### 根本原因
Brain 内嵌能力（运行在 Brain 进程内部的模块）不通过外部 skill 调用，也不一定有配置 key_tables；部分意识类能力（如 emotion-perception、desire-formation）完全没有 related_skills 和 key_tables。Scanner 原有逻辑只检查这两类活动迹象，导致内嵌能力被错误标为孤岛，影响后续决策准确性。

### 下次预防
- [ ] 新增 Brain 内嵌模块时，同步检查 `BRAIN_ALWAYS_ACTIVE` 或 `BRAIN_EMBEDDED_SOURCES` 是否需要更新
- [ ] cecelia_events source 名称必须与对应模块实际 emit 时的 source 字段一致（circuit-breaker.js → 'circuit_breaker'）
- [ ] 意识类能力（scope='cecelia'，migration-094 新增）没有 related_skills/key_tables 是设计如此，不是遗漏
