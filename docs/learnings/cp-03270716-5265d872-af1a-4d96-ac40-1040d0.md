# Learning: War Room Summary + Area 详情页架构

**Branch**: cp-03270716-5265d872-af1a-4d96-ac40-1040d0
**PR**: feat(dashboard): War Room 改为 Summary + Area 详情页架构
**Date**: 2026-03-27

## 根本原因

本次重构将 War Room 从单页混合布局改为两层结构：
- 总览页 Area 卡片指标增加（进行中任务数 + KR 整体进度）
- Area 详情页从 2 列改为 3 列（OBJ | KR | Task）各列独立滚动

## 关键决策

1. **KR 整体进度计算**：`Math.round(doneKRs / totalKRs * 100)`，totalKRs=0 时 pct=0 避免除零
2. **进行中任务数**：总览页展示全局 in_progress 任务总数（非按 Area 过滤），各 Area 卡片显示相同数值，属于系统整体活跃度指标
3. **三列 KR 展开**：使用 `objectives.flatMap(obj => obj.children.map(...))` 将所有活跃 OBJ 的 KR 平铺展示，使用 `kr.id`（UUID）作为 key 保证唯一性
4. **并行数据获取**：`Promise.allSettled` 并行拉取 OKR tree + in_progress tasks，任一失败不影响另一列数据渲染
