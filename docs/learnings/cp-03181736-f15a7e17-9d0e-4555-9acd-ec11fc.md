# Learning: Brain 设置 CECELIA_BUDGET_SLOTS=7

## 任务概要

在 docker-compose.yml 中固化 `CECELIA_BUDGET_SLOTS=7`，通过预算层保护保留 3 个 slot 缓冲。

### 根本原因

历史资源耗尽事件：queue 积压峰值 11 个任务，而 `physical_capacity=10`、`budget_cap=null`，
系统满载时无任何缓冲，导致新任务无法调度、突发流量直接耗尽所有 slot。

### 下次预防

- [ ] 新部署环境必须同时设置 `CECELIA_MAX_SEATS`（物理上限）和 `CECELIA_BUDGET_SLOTS`（预算上限）
- [ ] `CECELIA_BUDGET_SLOTS` 应始终小于 `CECELIA_MAX_SEATS`，保留至少 2-3 个 slot 缓冲
- [ ] 代码侧（executor.js）已支持 `CECELIA_BUDGET_SLOTS` 环境变量，无需改代码，只改配置
- [ ] `GET /api/brain/capacity` 是验证配置是否生效的标准检查端点
