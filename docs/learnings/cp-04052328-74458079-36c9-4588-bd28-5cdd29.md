# Learning: 全平台数据采集端点完整化

### 根本原因

`platform_scrape.sh` 在 HK VPS 执行后将结果输出到 stdout 供 N8N 读取，但 Brain 缺少专用的
"爬虫结果写回"端点。导致：

1. `processPendingScraperTasks()` 永远只能写 placeholder（0 值），因为 `pipeline_publish_stats`
   始终为空
2. `topic-heat-scorer.js` 的 HAVING 过滤（`> 0`）将所有 placeholder 行排除，
   导致选题引擎从无历史高热话题参考
3. `weekly-report-generator.js` 的 ROI 计算（读 `content_analytics`）也因无真实数据而返回空

### 下次预防

- [ ] 新增平台爬虫时，同步确认数据写回链路（scraper → Brain API → DB）
- [ ] 任何新采集端点上线前，在 `platform_scrape.sh` 中加 BRAIN_API_URL 测试验证
- [ ] `processPendingScraperTasks` 类似的"检查+写占位"模式，要确保真实写入路径存在

### 修复

添加 `POST /api/brain/analytics/scrape-result` 端点，同时写入：
- `content_analytics`（供 ROI 计算）
- `pipeline_publish_stats`（供话题热度评分，条件：需提供 publishTaskId）

在 `platform_scrape.sh` 中新增 `BRAIN_API_URL` 环境变量支持，采集完成后自动 POST 结果到 Brain。
