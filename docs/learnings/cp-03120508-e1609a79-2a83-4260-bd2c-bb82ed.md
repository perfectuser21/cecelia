# Learning - quarantine BILLING_CAP 路径漏设全局熔断（2026-03-12）

**Branch**: cp-03120508-e1609a79-2a83-4260-bd2c-bb82ed
**PR**: #905

### 根本原因

billing_pause 全局熔断有两条触发路径，但只有一条调用了 `setBillingPause`：

| 路径 | 触发时机 | setBillingPause |
|------|---------|----------------|
| routes.js quota_exhausted 回调 | task 执行完毕后回调 | ✅ 已调用 |
| quarantine.js BILLING_CAP 分类 | handleTaskFailure 失败分类 | ❌ 漏调用 |

通过 quarantine 路径捕获 BILLING_CAP 时，只有当前任务被 `blockTask` 阻塞，全局 `_billingPause` 未设置。dispatcher 不感知熔断，继续派发剩余 N-1 个排队任务直至全部撞墙（O(n) 开销而非 O(1)）。
