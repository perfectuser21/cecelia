# Learning: CI Scope 深度修复 — routing-map 补全 + ci-evolution-check 扩展

## 根本原因

`ci-evolution-check.mjs` 只扫描 `packages/*` 和 `apps/*` 两个目录，对 `scripts/devgate`、`ci/`、`.github/workflows/` 这三个关键 CI 基础设施目录没有注册校验覆盖。同时 `ci/routing-map.yml` 也未记录这些路径，导致修改 DevGate 脚本或 GitHub Workflows 时 Evolution Gate 无法感知遗漏注册。

## 修复方案

1. `ci/routing-map.yml` 新增 4 个条目：`devgate-core`（scripts/devgate）、`ci-helpers`（scripts）、`ci-configuration`（ci）、`github-workflows`（.github/workflows）
2. `scripts/ci-evolution-check.mjs` 新增 Check 4，验证这三个基础设施路径已在 routing-map 注册

## 下次预防

- [ ] 添加新的 CI 基础设施目录时，必须同步更新 `ci/routing-map.yml`
- [ ] 修改 DevGate 脚本目录结构时，运行 `node scripts/ci-evolution-check.mjs` 验证
- [ ] `feat:` commit 的 PR 必须包含测试文件改动（CI L3 `Test Coverage Required` 强制要求）
- [ ] 新增 CI 工具脚本时，在 `scripts/__tests__/` 下同步添加对应测试
- [ ] `console.log` 在新增代码中会被 Gate 0c 误报为调试代码，CLI 输出脚本改用 `process.stdout.write` 或 `out` helper
