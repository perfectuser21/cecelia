# Learning: Codex 独立审查池 — DoD Test 命令 CI 兼容性（2026-03-22）

## 背景

PR #1324 实现 `triggerLocalCodexReview()` 函数，将 spec_review/code_review_gate 路由到本机 Codex CLI 并使用独立锁池。L1 CI 出现两个门禁失败。

### 根本原因

1. **ESM 模块不能用 `require()` 加载**：DoD Test 第一条使用 `const {triggerLocalCodexReview}=require('./packages/brain/src/executor.js')`，但 executor.js 是 ES Module（含 `import` 语句），Node.js 的 `require()` 加载 ESM 时会尝试解析所有 import，导致 `Cannot find package 'uuid'`（CI 环境没有安装 node_modules）。

2. **CI 环境没有 node_modules**：DoD [GATE] 条目使用 `cd packages/brain && npm test`，但 L1 CI 不安装 `packages/brain` 的依赖，npm test 无法执行。

3. **缺少 Learning 文件**：PR 没有包含 `docs/learnings/<branch>.md`，触发 Learning Format Gate 硬失败。

### 下次预防

- [ ] 凡是测试 JS 模块内容是否存在某函数/变量，优先用 `readFileSync` 读取源码字符串检查，而非 `require()/import()` 加载执行
- [ ] DoD 的 `[GATE] 所有现有测试通过` 条目禁止用 `npm test`，改用 `readFileSync` 静态检查或 `node --check` 语法验证
- [ ] 每次 /dev 任务完成后必须在第一次 push 前创建 `docs/learnings/<branch>.md`，包含根本原因 + 下次预防 + checklist
- [ ] 在 main 上有 ESM 的包（executor.js 等）写 DoD Test 时先确认文件头是否有 `import` 语句，有则不能用 `require()`
