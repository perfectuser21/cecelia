# Learning - Brain dispatch 注入历史 Learning 上下文

**Branch**: cp-03120415-54ec35c4-b6e6-44d8-bcb8-04517d
**Type**: best_practice

### 根本原因

Brain 在派发任务时，agent 只能看到 PRD 内容，没有历史踩坑经验，容易重复犯相同错误。

### 实现要点

- `learning-retriever.js` 独立模块，不修改 `learning.js` 的内部函数
- 打分维度：task_type 匹配(+10)、domain 匹配(+6)、title 关键词(最高+8)、failure_pattern 类别(+4)、时效性(+1~3)
- 满分 31 分，threshold = 0.3（即原始分 >= 9.3 才注入），避免注入不相关内容
- `learning_type` 字段（migration 151 新增）用于分组展示；NULL 时 fallback 到 `category`
- 降级策略：DB 失败时 catch 并返回 ''，不影响正常 dispatch

### DoD Test 命令注意事项

含单引号的 grep 正则（如 `grep -c '...\|\\'\\''...'`）在 shell 转义后会导致 bash 解析错误（unexpected EOF）。应改为只匹配关键字的简单模式，如 `grep -c 'catch'`。

### 下次预防

- [ ] DoD Test 命令含单引号时，使用简单关键词替代复杂正则（如 `grep -c 'catch'` 而非含转义的复合正则）
- [ ] 新建独立检索模块（learning-retriever）而非修改已有内部函数（保持 learning.js 内聚性）
- [ ] 打分归一化后用 threshold 过滤，避免未命中 task_type 的 learning 被注入（会增加无关噪音）
