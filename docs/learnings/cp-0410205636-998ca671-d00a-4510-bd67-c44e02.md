# Harness v4.0 完成报告 worktree 清理（2026-04-10）

## 根本原因

Harness v4.0 工作流完成后，多个 /dev 会话的临时状态文件（.dev-lock.*/.dev-mode.*）残留在 worktree 中。这些文件被 git 版本控制跟踪，导致 git 状态混乱。Stop Hook 试图验证这些文件时，会因为孤立的文件对（lock 有对应的 mode）而阻止会话结束。

## 下次预防

- [ ] 实现自动清理机制：/dev 会话结束时自动删除对应分支的 .dev-lock/mode 文件
- [ ] 改进 Stop Hook 的孤立文件检测：只验证当前会话的 lock/mode 对，忽略其他分支的残留文件
- [ ] 在 worktree 创建时执行 `git clean -fd .dev-*` 清理旧文件
- [ ] 定期审查 .gitignore 规则，确保临时文件不会被 track
