# Learning: 修复 Sprint Contract Gate — Evaluator 独立测试计划强制执行

## 根本原因

1. **exit 2 无效**：SKILL.md 中 `divergence_count == 0 → exit 2` 是 shell 语法，但 spec_review 是 Claude Agent subagent，不是 shell 进程，exit 2 无法传递退出码。Evaluator 读到这个指令但无法执行，直接忽略。
2. **主 agent 未验证 independent_test_plans**：主 agent 收到 `verdict: PASS` 就接受，没有检查 `independent_test_plans.length > 0`。
3. **Test 字段泄露**：Task Card 包含 Test 字段直接传给 Evaluator，Evaluator 直接复制主 agent 方案，导致 divergence_count 为 0，触发 exit 2，但 exit 2 无效……恶性循环。

## 下次预防

- [ ] SKILL.md 中的退出逻辑必须用 JSON verdict 字段，而非 shell exit 代码
- [ ] Evaluator subagent 的任何"强制失败"都必须通过 seal 文件 verdict=FAIL + issues 字段来传递
- [ ] 主 agent 在接受 PASS 前必须验证 seal 文件的 `independent_test_plans.length > 0`
- [ ] Task Card 传给审查 subagent 前必须 mask Test 字段（用 REDACTED 替代）
