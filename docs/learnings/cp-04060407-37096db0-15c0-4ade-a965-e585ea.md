# Learning: 内容生成管道 v1 破零

## 任务
[SelfDrive] [P1] 启动内容生成管道 v1 - 破零

## 根本原因

### 1. 调度窗口过窄（触发失败）
`topic-selection-scheduler.js` 的 `isInTriggerWindow` 只允许 UTC 01:00-01:05（5 分钟）内触发。Brain 因 CI/CD 重启或任何原因错过该窗口，当日就没有选题任务，内容生成 KR 归零。

### 2. content_type 缺少 notebook_id（研究阶段立即失败）
`ai-tools-review` 和 `ai-workflow-guide` 的 DB 配置没有 `notebook_id`，`executeResearch` 因此立即返回 `success: false`，所有使用这两个类型的 pipeline 在 research 阶段就终止。`AVAILABLE_CONTENT_TYPES` 包含了未就绪的类型。

### 3. 破损 recurring pipeline
`recurring_task 0d40dc61` 每日创建的 pipeline 没有 `content_type`，orchestrator 无法获取 `notebook_id`，research 必然失败。

## 修复方案

1. `isInTriggerWindow` 扩展为 UTC 01:00-20:00（catch-up 机制），`hasTodayTopics()` 已有去重保护
2. `AVAILABLE_CONTENT_TYPES` 只保留 `['solo-company-case']`（有 notebook_id 配置）
3. 禁用 recurring_task `0d40dc61`，改由 topic-selection-scheduler 接管

## 下次预防

- [ ] 新增内容类型时，必须先配置 `notebook_id` 才加入 `AVAILABLE_CONTENT_TYPES`
- [ ] 每日调度器只有 5 分钟窗口是脆弱设计，应优先用 `hasTodayTopics` 去重而非窄窗口
- [ ] recurring_task 创建时必须验证 payload 是否包含执行所需的字段（content_type 等）
