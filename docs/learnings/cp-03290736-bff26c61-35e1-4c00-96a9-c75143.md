### 根本原因

1. **DoD Test 路径解析陷阱**：check-dod-mapping.cjs 的 projectRoot 是 git 仓库根，而非 packages/engine/。因此 `tests/engine/playwright-evaluator.test.ts` 被解析为 `<repo-root>/tests/engine/...`（不存在），导致 L1 失败。正确做法是用 `manual:node -e` 直接验证脚本行为。

2. **regression-contract.yaml contract: 引用范围**：check-dod-mapping.cjs 只搜索 hooks/workflow/ci/export/n8n 几类合约，不搜索 skills 节之后的自定义 RCI。导致 `contract:PE-001` 验证失败。新增 RCI 后对应 DoD 仍需用 `manual:node -e` 文件内容检查。

3. **generate-path-views.sh 产物必须随 feature-registry.yml 一起提交**：修改 feature-registry.yml 后未立即 commit docs/paths/ 下的产物，引发 L2 drift gate 失败。

4. **`isRun` 等未使用变量**：脚本中声明但未使用的变量（`const isRun = args.includes('--run')`）会被 code-review-gate 标记为 blocker，需及时删除。

---
