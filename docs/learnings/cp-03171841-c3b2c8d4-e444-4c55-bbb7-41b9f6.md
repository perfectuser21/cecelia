# Learning: 修复 migration 023 CREATE INDEX 幂等性

**Branch**: cp-03171841-c3b2c8d4-e444-4c55-bbb7-41b9f6
**Date**: 2026-03-17

### 根本原因

`023_add_run_events_observability_v1.1.sql` 中所有 `CREATE INDEX` 语句缺少 `IF NOT EXISTS`。
CI DB 环境中若已运行过该 migration（或表已有相同索引名），重跑时 PostgreSQL 报 `already exists` 错误，导致 migrate 非零退出，smoke.test.js server 启动失败。

### 修复方法

给 18 个索引全部加 `IF NOT EXISTS`：
- `run_events` 表：15 个普通索引 + 1 个唯一索引
- `run_artifacts` 表：3 个普通索引

### 下次预防

- [ ] 新增 migration 时，所有 `CREATE INDEX`/`CREATE UNIQUE INDEX` 必须带 `IF NOT EXISTS`
- [ ] 所有 `CREATE TABLE` 已有 `IF NOT EXISTS`（已正确），索引同理
- [ ] `CREATE TABLE IF NOT EXISTS` + `CREATE INDEX IF NOT EXISTS` = 完全幂等 migration
