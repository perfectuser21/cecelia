# Learning: Dashboard KR5 Bug 修复 — /api/goals 404 + 静默错误

## 根本原因

### 问题 1: LiveMonitorPage 使用了未注册的 `/api/goals` 路由
- `apps/dashboard/vite.config.ts` 代理规则：`/api/brain/*` → Brain (5221)，`/api/*` → apps/api (5211)
- `apps/api/src/dashboard/server.ts` 中没有注册 `/api/goals` 路由（只有 `/api/brain`、`/api/tasks`、`/api/okr` 等）
- LiveMonitorPage 调用 `/api/goals?limit=200`，实际打到 apps/api，返回 404/HTML
- Brain 的 Goals 接口正确路径是 `/api/brain/goals`

### 问题 2: CollectionDashboard/AreaSlots 静默错误
- fetch 失败时用 `catch { /* 静默 */ }` 或 `console.error`
- loading 结束后 data 为 null，用户看到空白页，无法诊断问题

## 下次预防

- [ ] 新增 Dashboard 页面时，优先检查 vite.config.ts 代理规则，确认路由实际注册位置
- [ ] `/api/brain/*` 才走 Brain；通用数据路由走 apps/api 需要先确认路由已注册
- [ ] fetch 错误处理模板：`try { ... setError(null) } catch(e) { setError(...) }`，不允许空 catch
- [ ] 每次 bug 修复前先用 `grep -rn` 确认 API 路由在 server.ts 中已注册
