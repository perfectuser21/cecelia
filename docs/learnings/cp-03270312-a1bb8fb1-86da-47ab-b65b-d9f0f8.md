### 根本原因

1. **verify-step.sh symlink 路径 bug**：`hooks/` 是指向 `packages/engine/hooks` 的符号链接。`$(dirname "${BASH_SOURCE[0]}")` 返回逻辑路径 `hooks`，`hooks/../scripts/devgate` 在 shell 层面成立（内核跟随 symlink 解析），但 Node.js 使用词法路径解析，`hooks/../scripts/devgate/check-manual-cmd-whitelist.cjs` 被解析为不存在的 `/scripts/devgate/...`，导致 MODULE_NOT_FOUND。

2. **PRESERVE 测试先于实现存在**：Task Card 中的 DoD 指向 `tests/frontend/*.test.ts`，但文件未提前创建。Gate 2 在 verify-step 时检查文件存在，发现缺失而失败。

3. **OKRPage 字段名差异**：测试最初断言 `progress_pct`，但实际代码中 OKR 对象使用 `progress`（Workspace API 与 Brain API 字段命名不同）。
