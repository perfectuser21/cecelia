# Learning: fix(escalation): cancel_pending 排除 content pipeline 子任务

## 根本原因

`cancelPendingTasks` 的 `task_type NOT IN` 排除列表在 content pipeline 功能上线后未同步更新，仅包含 `research` 和 `suggestion_plan`，漏掉了所有 content pipeline 子任务类型（content-research / content-copywriting / content-copy-review / content-generate / content-image-review / content-export / content-pipeline）。当系统 alertness 升至 L2（emergency_brake）时，这些子任务会被误取消，导致 content pipeline 永久卡在 in_progress。

## 下次预防

- [ ] 新增 task_type 时，同步检查 `cancelPendingTasks` 排除列表是否需要更新
- [ ] 新增 pipeline 类功能时，在 escalation.js 注释中记录"此 task_type 需要排除"
- [ ] 考虑将排除列表提取为常量 `CANCEL_EXCLUDED_TYPES`，集中维护
