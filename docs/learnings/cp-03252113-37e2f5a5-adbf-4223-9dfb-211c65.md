# Learning: 修复 areas.js GET /:id 断路

## 任务背景
PR#1583 修复了 goals/projects/areas GET / 端点代理问题，但 areas.js GET /:id 仍查询不存在的旧 goals/projects 表。

## 根本原因
OKR 迁移 PR12 DROP 了 goals 和 projects 旧表，但 areas.js 的 `GET /:id` 路由中有如下查询未清理：
- `SELECT * FROM projects WHERE area_id = $1`（projects 表已不存在）
- `SELECT g.* FROM goals g JOIN projects p ON...`（goals/projects 表已不存在）

PR#1583 只修改了 `GET /` 的 ORDER BY 问题，未处理 `GET /:id` 中对旧表的依赖。

## 下次预防

- [ ] 每次 DROP 旧表时，必须搜索整个代码库所有对该表的引用（不只是路由文件的 GET /）
- [ ] 包括 GET /:id、POST、PATCH、DELETE 等所有方法
- [ ] 使用 `grep -r "FROM goals\|FROM projects" apps/` 全面检查后再合并
- [ ] PR 描述要明确列出"已检查以下文件无残留引用"
