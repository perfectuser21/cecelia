# Learning: 重构 runReflection 降低圈复杂度（12 → 9）

Branch: cp-03231530-964423f6-9ee8-486e-826e-ea54ba
Date: 2026-03-23

### 根本原因

`runReflection` 在 PR #1432 后仍有复杂度 12（阈值 10），原因是主函数内保留了 3 个独立的 try/catch 块（LLM调用、写入洞察、重置accumulator），每个 catch 各计 +1，加上原有的 if 判断导致超标。

### 修复方案

提取两个私有辅助函数：
1. `_callLLMForReflection(memories, accumulator)` — 封装 prompt 构建 + LLM 调用 + catch（返回 null）
2. `_persistInsightAndReset(pool, insight, accumulator)` — 封装写入洞察 + 重置 accumulator 各自的 try/catch

`runReflection` 主函数复杂度从 12 降至 9（CC=9，低于阈值 10）。

### 下次预防

- [ ] 每次重构降复杂度时，优先看 try/catch 块——每个 catch 计 +1，是最容易被忽视的复杂度来源
- [ ] 辅助函数提取原则：把"独立的错误处理语义"（降级行为不同的逻辑）封装为独立函数，不要为了行数凑合拆分
- [ ] 验证方法：用 `fn.match(/\bif\b|\bcatch\b|\bcase\b|\?\./g).length + 1` 快速估算 CC，无需工具
