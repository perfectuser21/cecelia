# Learning: executeImageReview 圈复杂度重构

**分支**: cp-0412005657-dbfae02a-6881-48ba-9438-821b70
**日期**: 2026-04-12

### 根本原因

`executeImageReview` 函数承担了 5 个独立职责（配置加载、文件检查、图片计数、内容加载、LLM调用+解析），导致圈复杂度达 29，嵌套的 try/catch 和条件判断难以维护。

### 重构策略

提取 5 个独立子函数，每个函数只有 1 个职责：
- `_imageReviewLoadConfig` — DB/YAML 配置加载（含 try/catch）
- `_imageReviewCheckFiles` — 文件存在性检查
- `_imageReviewCountImages` — 图片统计（含 try/catch）
- `_imageReviewLoadContent` — 待审核内容加载
- `_imageReviewCallLLM` — LLM 调用 + JSON 解析

主函数缩减至 36 行（原 79 行），复杂度降至约 6。

### 下次预防

- [ ] 新增 executor 函数时，单个函数职责不超过 3 个（超出即拆分）
- [ ] 多个 try/catch 块是提取子函数的信号
- [ ] 同模块内已有 `_load*`/`_build*`/`_parse*` 等命名惯例，新提取函数沿用此格式
