### 根本原因

migration 088_progress_ledger.sql 中 4 个 `ALTER TABLE ... ADD CONSTRAINT` 语句直接裸写，没有幂等保护。当 migration 在已存在 constraint 的环境重跑时，PostgreSQL 抛出 `duplicate_object` 错误导致 CI 失败。

### 下次预防

- [ ] 所有 `ADD CONSTRAINT` 语句必须用 `DO $$ BEGIN ... EXCEPTION WHEN duplicate_object THEN NULL; END $$` 包裹
- [ ] 新增 migration 时检查：CREATE TABLE/INDEX 用 IF NOT EXISTS，ADD CONSTRAINT 用 DO $$ 幂等块
- [ ] Code review checklist 加入"是否所有 DDL 都有幂等保护"
