## setBillingPause cecelia_events 持久化（2026-03-12）

### 根本原因

PR #874 实现的 `setBillingPause` 只在 in-memory `_billingPause` 变量中设置状态，Brain 进程重启后状态丢失，导致 billing pause 熔断保护失效。需要借助 `cecelia_events` 表作为持久化来源。

### 下次预防

- [x] 任何影响 Brain 全局调度行为的 in-memory 状态（billing pause、账号限流等），在设置时同步写一条事件到 cecelia_events（fire-and-forget 模式，不阻塞主流程）
- [x] 复用 `recordSessionEnd` 的 fire-and-forget 模式：`poolRef.query(...).catch(e => console.warn(...))`
- [x] poolRef 为 null 时降级（只设 in-memory），避免测试环境报错
- [x] 新增单测验证 pool.query 被调用，同时验证 poolRef=null 场景不抛错
