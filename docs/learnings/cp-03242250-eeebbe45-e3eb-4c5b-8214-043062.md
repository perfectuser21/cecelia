# Learning: 修复 self_drive_health 探针误判

## 任务
修复 `probeSelfDriveHealth` 探针持续失败导致的死循环

## 根本原因

### 根本原因 1：DB 数据错误
Migration 159 将 `profile-anthropic` 的 `thalamus` 设置为 `gpt-5.4-mini`（codex/openai），导致 codex team OAuth quota 耗尽时所有 self-drive cycle 都报 `cycle_error`。

### 根本原因 2：探针判定逻辑过窄
`probeSelfDriveHealth` 只检查 `tasks_created > 0`，导致 LLM 正常运行但判断无需行动（`no_action`）时被误判为失败，触发自动修复 → 循环创建无效任务。

## 修复方案
1. Migration 192 重置 `profile-anthropic` 的 `thalamus` 为 `anthropic/claude-haiku-4-5-20251001`
2. 改写探针查询：`cycle_complete` 或 `no_action` 均计为成功；只有 `cycle_error` 才算失败

## 下次预防

- [ ] 修改任何 model_profiles 的 thalamus 配置时，必须同步检查该 provider 的 quota 状态
- [ ] 探针判定逻辑应区分"系统健康但无需行动"和"系统故障"两种情况，不能用业务指标（tasks_created）判断系统存活
- [ ] 新探针添加时，单测必须覆盖 ok:true 和 ok:false 两种路径
