# Learning: MiniMax 路由清理 + content-* EXECUTOR_AFFINITY 补全

**Branch**: cp-03232346-cf877cca-dde0-4e9c-8ca9-8dba5f
**Date**: 2026-03-24

## 变更摘要

将 `token-budget-planner.js` 中 `explore/research/talk/data/dept_heartbeat` 5 个任务类型的 `primary` 执行器从 `'minimax'` 改为 `'codex'`，并补全 `content-*` 5 个缺失的 EXECUTOR_AFFINITY 条目。

### 根本原因

`task-router.js` 已将上述任务路由到西安 Codex（`primary: 'codex'`），但 `token-budget-planner.js` 的 `EXECUTOR_AFFINITY` 仍保留旧的 `primary: 'minimax'` 配置，导致两处路由逻辑不一致，预算计算器可能错误地把这些任务计入 MiniMax 配额。

同时 `content-*` 系列任务类型在 EXECUTOR_AFFINITY 中完全缺失，会导致 `getExecutorAffinity()` 回退到 DEFAULT_AFFINITY（`primary: 'claude'`），与实际路由（codex）不符。

两处配置（路由层 + 预算层）各自维护执行器映射，缺乏单一来源约束，是漂移的根本原因。

### 下次预防

- [ ] 新增任务类型时，必须同步更新 `task-router.js` 和 `token-budget-planner.js` 两处配置
- [ ] 添加 CI 检查：`task-router.js` 中的所有任务类型必须在 `EXECUTOR_AFFINITY` 中有对应定义
- [ ] 路由层（task-router）和预算层（token-budget-planner）的执行器定义应保持单一来源，避免双写漂移
