# Learning: Bridge /llm-call 500 — 无 accountId 时 CLAUDE_CONFIG_DIR 为空

## 根本原因

Bridge 的 `/llm-call` 端点在 `accountId` 未传时，直接用进程自身的环境变量 spawn `claude -p`。
Bridge 进程本身无 `CLAUDE_CONFIG_DIR`，导致 `claude -p` 报 "Not logged in · Please run /login" 并 exit 1，
Bridge 返回 `500 {"ok":false,"error":"exit code 1"}`。

触发条件：`llm-caller.js` 的 `callClaudeViaBridge()` 调用 `selectBestAccount()` 时返回 `null`
（所有账号超配额或 `selectBestAccount` 抛出异常），`accountId` 保持 `undefined`，
Bridge 不知道用哪个账号。

## 下次预防

- [ ] Bridge 新增任何 spawn 子进程的端点时，必须显式设置 `CLAUDE_CONFIG_DIR`（无论 accountId 是否存在）
- [ ] `callClaudeViaBridge()` 中 accountId 必须始终有值（有最佳账号时用最佳，无时用 fallback）
- [ ] Bridge 新增端点测试：必须覆盖"无 accountId"场景，验证 stdout 非空
- [ ] 诊断 500 首先看 stderr：`{"ok":false,"error":"exit code 1"}` 中 error 字段含实际 stderr，若为 "exit code N" 则 stderr 为空，说明 claude 连启动都未成功

## 修复

1. `cecelia-bridge.cjs` 无 accountId 时 fallback 到 `DEFAULT_CLAUDE_CONFIG_DIR`（env 或 `~/.claude-account1`）
2. `llm-caller.js` 初始化 `accountId = FALLBACK_ACCOUNT`，selectBestAccount 成功才覆盖
