# Learning: Deploy CI 诊断闭环确认（2026-04-06）

### 根本原因

Deploy CI 连续失败（2026-04-06 08:12~09:01 UTC）的根因已由 PR #1953 修复：

1. **concurrency 死锁**：deploy job 与 workflow 持有同名 `deploy-production` concurrency group → job 进入等待自己释放锁的死锁 → 0秒失败
2. **BRAIN_URL 无效**：secret 未设置时 fallback 指向不可达域名 → webhook 404
3. **shepherd 断链**：PR merge 后未回写 task.status=completed → OKR 进度不推进

### 下次预防

- [ ] 新增 workflow concurrency 配置时，检查 job-level 与 workflow-level 是否同名
- [ ] CI secret fallback 默认值使用 CI 网络可达的地址（`http://IP:PORT`，不用域名）
- [ ] 0秒失败的 job 优先排查 concurrency 死锁（不是权限问题）
