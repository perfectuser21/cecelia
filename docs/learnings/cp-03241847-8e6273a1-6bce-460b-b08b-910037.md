# Learning: Hook-CI 耦合重构 R12 — Gate 0d 版本同步检查

**Branch**: cp-03241847-8e6273a1-6bce-460b-b08b-910037
**Date**: 2026-03-25

## 问题描述

verify-step.sh Gate 0d 只做了同目录 sibling consistency check，但没有调用
check-version-sync.sh 做全面的 5 文件版本同步检查。
同时 check-version-sync.sh 硬依赖 `jq`，在没有 jq 的环境下直接报错退出。

### 根本原因

1. Gate 0d 最初设计为"轻量 sibling check"，未考虑调用专用的版本同步脚本
2. check-version-sync.sh 使用 `jq -r '.version'` 但未检查 jq 是否存在，导致非 CI 环境
   （本地开发机、某些 Docker 镜像）因缺少 jq 而报错退出

### 下次预防

- [ ] 新增 CI 工具脚本时，第一行检查依赖工具是否可用（`command -v jq &>/dev/null`）
- [ ] Gate 0d 等本地门禁应主动调用与 CI 同等的检查脚本，保证本地/CI 行为一致
- [ ] Engine hooks 变更时，记得同时更新 feature-registry.yml 和重新生成 path views
