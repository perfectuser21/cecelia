# Learning: active_goals=0 时的 tick 行为修复

### 根本原因

tick.js 的 KR 目标集合构建逻辑在 `!hasFocus && readyKrIds.length === 0` 时，会 fallback 到"查询所有非归档 KR"的全局兜底，导致对无效/过期目标派发任务。同时，该分支完全忽略了健康状态，即便 alertness >= ALERT 也继续派发，没有触发恢复模式。

### 下次预防

- [ ] 任何 tick fallback 逻辑（global_fallback、transition period 等）在写入时，同步检查是否需要 health gate
- [ ] `active_goals=0` 是一个异常信号，不是"正常可以继续运行"的状态 — 应先判断是否 health 异常，再决定是恢复还是静默跳过
- [ ] healing.js 的 `startRecovery` 在 tick.js 中仅导入了 `getRecoveryStatus`，新增 recovery 路径时需同步更新 import
