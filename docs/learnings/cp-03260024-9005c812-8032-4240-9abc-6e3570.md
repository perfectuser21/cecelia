# Learning: fix(gtd): full-tree 过滤 archived objectives

branch: cp-03260024-9005c812-8032-4240-9abc-6e3570
date: 2026-03-26

### 根本原因

`full-tree.js` 的 objectives 查询仅按 `area_id` 过滤，未排除 `status = 'archived'` 的条目。
已归档的 objectives 会以重复/过期节点形式出现在 GTD OKR 树中，破坏 Area → Objective → KR 的视图清洁度。
同时 Areas INNER JOIN 也未过滤 archived objectives，导致只有归档 objectives 的 area 仍会被选入结果集。
修复方式：在 JOIN 条件和 WHERE 子句两处都加 `AND o.status != 'archived'`。

### 下次预防

- [ ] 编写 OKR 层级树查询时，检查每层查询是否需要过滤 `status` 字段
- [ ] Areas JOIN objectives 时同步加 `AND o.status != 'archived'`，防止空/无效 area 被带入结果
- [ ] `status` 过滤应在 JOIN 条件或 WHERE 子句中明确声明，不依赖上层调用方过滤
