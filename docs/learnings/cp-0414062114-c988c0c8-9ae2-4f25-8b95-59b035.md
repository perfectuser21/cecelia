# KR3 配置检测器 decisions 表列名错误（2026-04-14）

### 根本原因

`kr3-config-checker.js`（PR #2329 合并时）查询语句使用了 `key`/`value` 列：

```sql
SELECT key, value FROM decisions WHERE key = ANY($1)
```

但 `decisions` 表实际 schema 不存在这两列，正确列名为：
- `topic`（varchar 200）— 决策主题，用作唯一标识符
- `decision`（text）— 决策内容，用作值存储

导致 `GET /api/brain/kr3/check-config` 始终返回 `"检测失败: column "key" does not exist"`，KR3 进度永远无法标记就绪。

### 下次预防

- [ ] 新增操作 `decisions` 表的模块时，先运行 `psql cecelia -c "\d decisions"` 确认实际列名，不靠记忆
- [ ] `decisions` 表的 `topic` 无唯一约束，写入时用 UPDATE（废弃旧记录）+ INSERT 两步法，不能用 `ON CONFLICT(topic)`
- [ ] PR 中新增 DB 查询模块，必须同步添加 mock 单元测试，在 CI 中验证列名正确性
