# Learning: tick.js billing pause 熔断连接

## billing pause 信号从未被消费（2026-03-11）

### 根本原因

`getBillingPause()` 在 tick.js 导入但 `dispatchNextTask` 中注释为"已废弃（v1.196.0）"，从未实际调用。execution-callback 收到 "AI Quota Exhausted" 时也从未调用 `setBillingPause`，导致 billing pause 信号从未被生产，也从未被消费。结果：配额耗尽后系统持续派发新任务，产生大量 quota_exhausted 失败。

### 下次预防

- [ ] 新增/修改熔断机制时，同时验证「生产侧」（setBillingPause 调用点）和「消费侧」（dispatchNextTask 检查点）两端是否都已连通
- [ ] 在 tick.js 添加函数时，若已有注释"废弃"但函数仍导出，须确认废弃是否有效（是否真的不再需要）或重新激活
- [ ] 新写 vitest 测试时，`beforeEach` 用 `vi.clearAllMocks()`，不用 `vi.resetAllMocks()`，防止清除模块级 mock 实现导致意外 TypeError
- [ ] tick() 主函数中调用 executor mock 函数时加可选链（`?.`），防止旧测试的 null mock 触发 TypeError
