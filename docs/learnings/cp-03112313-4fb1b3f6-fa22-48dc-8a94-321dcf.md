# Learning: tick.js billing pause 熔断连接

## 背景

`getBillingPause()` 已导入 tick.js 但从未在 `dispatchNextTask` 中调用，导致配额耗尽后系统持续派发 → 批量 `quota_exhausted` 失败任务堆积。

## 关键决策

### 1. vi.resetAllMocks() vs vi.clearAllMocks() 陷阱（再次确认）

`vi.resetAllMocks()` 在 `beforeEach` 中使用会清除所有 mock 的 `mockReturnValue`/`mockImplementation`，导致 `calculateSlotBudget` 返回 `undefined`，下游代码 `slotBudget.dispatchAllowed` 报 TypeError。

**修复**：`beforeEach` 改用 `vi.clearAllMocks()`，只对特定 mock 重新设置返回值。

### 2. 可选链防御 null mock

旧测试中 `getBillingPause: vi.fn().mockReturnValue(null)`（tick-blocked-recovery.test.js 等），我在 tick() 主函数中调用 `getBillingPause()?.active` 时加了可选链（`?.`），防止 null 导致 TypeError。

### 3. quota_exhausted requeue 位置选择

在 tick() Step 6.5（Step 7 dispatch 之前）执行 requeue。逻辑：
- billing pause 激活期间：任务留在 `quota_exhausted`（不 requeue）
- billing pause 过期后：下一个 tick 的 step 6.5 自动 requeue → queued → 正常派发

### 4. setBillingPause 调用点

在 routes.js execution-callback 中，`newStatus === 'quota_exhausted'` 时调用 `setBillingPause`：
- 尝试从 `result.quota_reset_at` 读取 reset 时间
- 默认 1 小时后（安全的保守值）

### 5. GET /api/brain/billing-pause 端点

支持 `?clear=true` 参数手动清除 billing pause，方便运维介入。
