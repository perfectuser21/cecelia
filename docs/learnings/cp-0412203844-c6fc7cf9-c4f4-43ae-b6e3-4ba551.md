# Learning: Bridge /llm-call 500 — DB profile 配置与代码 FALLBACK 不一致

## 根本原因

DB `profile-anthropic` 的 `thalamus.provider` 配置为 `'anthropic'`（Bridge/claude -p 路径），
而代码 `FALLBACK_PROFILE` 注释和内容均指示应使用 `'anthropic-api'`（直连 REST，稳定）。

当 `loadActiveProfile()` 从 DB 加载时，DB 值覆盖了代码默认值，导致 thalamus 走 Bridge，
Bridge 的 `claude -p` 在限流/配额不足时以 exit code 1 退出，Bridge 返回 HTTP 500。
`callClaudeViaBridge` 无重试直接抛出，整个内容审核任务失败。

## 下次预防

- [ ] DB profile 更新后必须对照 `FALLBACK_PROFILE` 检查一致性（两处都是 SSOT，容易分叉）
- [ ] `callClaudeViaBridge` 的 Bridge 500 重试已加入（`bridge500Retry`，最多2次，500ms/1000ms退避）
- [ ] thalamus profile 修改（provider 切换）应同步更新 FALLBACK_PROFILE 注释
- [ ] 迁移 233 修复 DB 中的 thalamus 配置，添加 fallbacks 作为第二道防线
