# Learning: stop-dev.sh fallback spec_review 超时检查遗漏

### 根本原因

PR #1296 在 stop-dev.sh fallback 路径中补齐了 code_review_gate 的 15 分钟超时，
但同步遗漏了 spec_review 的相同超时处理。

主路径（devloop-check.sh）通过 `_check_codex_review` 函数统一处理两者，
fallback 路径由于是手工维护，只同步了 code_review_gate，漏掉了 spec_review。

此外，测试 `pr-gate-phase1.test.ts > should pass when all items have Test fields`
依赖仓库根 `.prd.md` 内容进行 DoD→PRD 追溯检查，每次更新 `.prd.md` 后测试就可能失败。

### 下次预防

- [ ] 每次在 devloop-check.sh 添加新 Gate 检查时，同时检查 stop-dev.sh fallback 是否需要同步
- [ ] 对称原则：spec_review 和 code_review_gate 的处理应完全对称
- [ ] DoD→PRD 追溯相关的测试应使用独立的 TEST_DIR + .prd.md，不依赖仓库根 .prd.md
