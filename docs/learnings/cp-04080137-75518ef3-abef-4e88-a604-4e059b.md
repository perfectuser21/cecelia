# Learning: KR 进度采集链路修复

## 背景
KR2（AI私域线）verifier 持续报错 "inconsistent types deduced for parameter $2"，导致 verifier 无法更新 key_results.current_value。

### 根本原因
`kr-verifier.js` 中 UPDATE key_results 语句：

```sql
SET current_value = $2,
    metadata = ... || jsonb_build_object('metric_current', $2::text)
```

PostgreSQL 在解析 prepared statement 时，对 `$2` 有两个类型推断上下文：
1. `current_value = $2` → 列类型 `numeric(12,2)`，推断 $2 为 numeric
2. `$2::text` → 被显式 cast 到 text，PostgreSQL 认为 $2 的原始类型是 unknown

两个上下文冲突 → "inconsistent types deduced for parameter $2"。

注意：这个错误具有间歇性（KR1 有时成功有时失败），因为 node-postgres 的 prepared statement 缓存状态会影响类型推断行为。

### 下次预防
- [ ] 在 UPDATE 语句中，同一参数不能在一处用列类型推断、另一处用 `::text` 显式转换
- [ ] 正确写法：`($2::numeric)::text` — 先明确 $2 是 numeric，再 cast 到 text
- [ ] 写涉及同参数多次引用的 SQL 时，先 PREPARE + EXECUTE 手动测试
