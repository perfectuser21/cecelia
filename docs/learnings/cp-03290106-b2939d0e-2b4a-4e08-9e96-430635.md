## 背景
PR: fix(brain): 清理 goals 表 5 条 Test KR 占位符数据
Branch: cp-03290106-b2939d0e-2b4a-4e08-9e96-430635
Date: 2026-03-29

### 根本原因

**DoD [BEHAVIOR] 测试用了 `require('pg')` 直连数据库，但 CI 环境无 pg 模块也无数据库服务**。

数据迁移类任务的 [BEHAVIOR] 测试不能测「数据库里有没有数据」（需要运行时），
而应该测「迁移 SQL 文件本身的内容是否正确」（静态验证）。

### 下次预防

- [ ] 数据迁移 PR 的 [BEHAVIOR] DoD 测试必须使用 `node -e "require('fs').readFileSync(...)"` 验证 SQL 内容
- [ ] 禁止在 DoD Test 中使用 `require('pg')` 或 `psql` 等需要数据库连接的命令
- [ ] 迁移文件验证模板：
  ```
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/migrations/xxx.sql','utf8');if(!c.includes('EXPECTED_CONTENT'))process.exit(1)"
  ```
- [ ] 新增迁移时同步更新 `EXPECTED_SCHEMA_VERSION`（selfcheck.js）和 DEFINITION.md schema_version
