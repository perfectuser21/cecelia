# Learning: 重构 executeImageReview 圈复杂度 29→10

### 根本原因
executeImageReview 函数承担了多个职责（文件验证、图片计数、内容读取、LLM 调用、结果解析），导致圈复杂度达到 29，远超阈值 10。

### 解决方案
提取 5 个单职责子函数，主函数只做流程编排：
- validateImageFiles — 文件结构检查
- countCardImages — 图片计数
- loadContentForReview — 内容读取
- parseLLMReviewResult — LLM 响应解析
- callImageReviewLLM — LLM 调用封装

主函数用早返回消除嵌套，从 79 行降至 31 行。

- [ ] 函数职责超过 3 个时立即拆分
- [ ] 圈复杂度扫描告警阈值 15（而非等到 29 才处理）
- [ ] 新增 LLM 调用场景时，调用+解析必须分两个函数
