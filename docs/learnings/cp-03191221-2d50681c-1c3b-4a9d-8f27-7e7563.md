## 踩坑记录

### 根本原因

**DoD `$?` 展开陷阱**：在 Task Card 的 Test 命令中使用 `bash -c "...; [ $? -ne 0 ]"`（双引号包裹），当 `check-dod-mapping.cjs` 通过 Node.js execSync 运行时，外层 `/bin/sh` 会将 `$?` 展开为上一条命令的退出码（通常为 0），导致测试命令实际执行的是 `[ 0 -ne 0 ]`（永远失败），而不是检查 node 命令的真实退出码。

**PRD/DoD 双路径规则**：在 `packages/workflows/` 子树开发时，PRD 和 DoD 文件必须同时放置在 worktree 根目录和 `packages/workflows/` 两处，因为 branch-protect.sh 就近检测，从子目录走到根目录时会被 `packages/workflows/.prd.md`（旧任务残留）拦截。

### 下次预防

- [ ] DoD 负向退出码测试必须使用单引号包裹 bash -c 参数：`bash -c '...; [ $? -ne 0 ]'`（单引号防止外层 shell 展开 `$?`）
- [ ] 在 `packages/workflows/` 子树开发时，记住 PRD/DoD 要放两处：根目录 + `packages/workflows/`
- [ ] Learning 文件必须在第一次 push 前加入 commit，否则 CI Learning Format Gate 失败
