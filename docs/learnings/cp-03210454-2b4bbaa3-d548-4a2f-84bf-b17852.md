# Learning: codex_test_gen task_type 注册

**任务**: feat(engine): 新增 codex_test_gen task_type + SKILL.md — Codex 自动测试生成
**PR**: #1302
**日期**: 2026-03-21

---

### 根本原因

新增 Brain task_type 需要同步更新 5 处文件，且每处都有独立验证规则：
- `task-router.js`：VALID_TASK_TYPES + SKILL_WHITELIST + LOCATION_MAP（3 处）
- `role-registry.js`：CTO task_types
- `executor.js`：getModelForTask null 返回（Codex CLI 不需要 Brain 选模型）
- `model-registry.js`：AGENTS 数组
- `DEFINITION.md`：facts-check 追踪
- `brain-manifest.generated.json`：需重新生成

### 下次预防

- [ ] 新增 Codex 类 task_type 时，必须运行 `node packages/brain/scripts/generate-manifest.mjs` 更新 manifest
- [ ] `DEFINITION.md` 的 task_types 表必须包含新类型，否则 facts-check 失败阻断 push
- [ ] Codex 系列（codex_qa/codex_playwright）task_type 在 `executor.js` 中需要 `return null`（不让 Brain 选模型，由 Codex CLI 自行决定）
- [ ] `stash pop` 冲突后需手动验证版本文件（VERSION/package.json/.hook-core-version）没有被旧值覆盖
- [ ] DoD 的 `[BEHAVIOR]` 条目必须用 `node -e "require('fs').readFileSync(...)"` 验证源码逻辑，不能用 grep/curl/psql

### 收获

新增 Xian Codex 任务类型无需修改 `executor.js` 的路由逻辑（`US_ONLY_TYPES` 不包含的自动走 Codex Bridge），只需正确设置 `LOCATION_MAP` 中的值为 `xian`。
