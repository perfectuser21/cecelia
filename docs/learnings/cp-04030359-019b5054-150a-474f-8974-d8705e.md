## 数据闭环系统 — 全平台采集→周报（2026-04-03）

### 根本原因
`platform_scraper` 任务由 `post-publish-data-collector.js` 正确创建并入队，但 `task-router.js` LOCATION_MAP 和 `executor.js` skillMap 中均无该 task_type 条目，导致任务永久积压在 `queued` 状态，数据采集从未执行。周报模块因从未被需求驱动而未建立。

### 下次预防
- [ ] 新增任何会创建任务的模块（如 `*-data-collector.js`）时，必须同步在 LOCATION_MAP + skillMap 中注册对应 task_type，否则任务无路由
- [ ] 参考模式：`content_publish` → publisherSkillMap 动态路由；`platform_scraper` → skillMap + LOCATION_MAP 静态条目
- [ ] 周报/月报模块命名规范：`{period}-report-generator.js`，trigger 条件写 `is{Period}ReportTriggerWindow()`，幂等 key 写 `{period}_report_triggered_{KEY}`
