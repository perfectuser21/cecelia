# Learning: 执行日志上传 Brain — dev-logs 路由挂载 + generate-report.sh POST

## 根本原因

`packages/brain/src/routes/dev-logs.js` 文件已存在（含 POST/GET 路由），但从未在 `server.js` 中 `import` 和 `app.use()` 注册，导致 `/api/brain/dev-logs` 端点 404。

同时 `generate-report.sh` 生成 JSON 报告只存本地 `.dev-runs/`，没有回写 Brain。

## 变更内容

1. `packages/brain/server.js`：添加 `import devLogsRoutes` + `app.use('/api/brain/dev-logs', devLogsRoutes)`
2. `packages/engine/skills/dev/scripts/generate-report.sh`：末尾加 curl POST，读取 `DEV_TASK_ID` 环境变量；若未设置（interactive 模式）则静默跳过
3. `packages/workflows/skills/dev/scripts/generate-report.sh`：同步相同变更

## 下次预防

- [ ] 新增路由文件时，同步检查 `server.js` 是否已注册，否则端点永远不可达
- [ ] Brain API 添加新表后，验证对应端点在 server.js 中已挂载（`grep routes/dev-logs server.js`）
- [ ] `generate-report.sh` 使用 `DEV_TASK_ID` env var 区分有/无 Brain 环境，避免 interactive 模式报错
