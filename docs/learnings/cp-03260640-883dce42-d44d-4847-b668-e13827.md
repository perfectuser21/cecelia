### 根本原因

新增 Brain migration（199）后，`selfcheck.js` 中的 `EXPECTED_SCHEMA_VERSION` 和 `DEFINITION.md` 中的 `schema_version` 都需要同步更新，否则 push 时的 facts-check 会失败。

本次在第一次 commit（feat）中只写了 migration SQL 文件，但没有同步更新 `selfcheck.js`（hardcode 版本号）和 `DEFINITION.md`（文档中的 schema_version 字段），导致 push 时本地预检（bash-guard → local-precheck → facts-check）失败，不得不额外追加两次 fix commit。

根本原因是：migration 的版本同步涉及三处（migration 文件本身 + selfcheck.js + DEFINITION.md），但 /dev 流程中没有提醒一起更新，容易漏。
