## 重构 buildLearningContext 降低圈复杂度（2026-04-04）

### 根本原因

`buildLearningContext` 函数将 DB 查询、慢查询计时、评分过滤、结果格式化四个职责全部内联在一个 try 块中，导致圈复杂度累计到 21。复杂度扫描工具（含箭头函数内部分支）统计了函数内所有 decision point。

### 下次预防

- [ ] 函数超过 20 行 or 含 3+ 层嵌套时，立即拆子函数
- [ ] "查询+计时+处理+格式化"混在一起是拆分信号，各职责应有独立函数名
- [ ] DoD Test 验证函数行数上限，防止重构后再度膨胀
