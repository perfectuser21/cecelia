# Learning: Brain API output 端点扩展 — 文件扫描代替 URL 猜测

## 背景
ZenithJoy 内容产出详情页需要展示文章全文、卡片文案和实际图片。原 output 端点只返回猜测的图片 URL，图片无法显示。

### 根本原因
原实现用字符串拼接构造图片 URL（`{slug}-cover.png`），但实际文件名会因关键词含空格、特殊字符时处理方式不同而不匹配。

文章内容也未读取，前端只能显示空内容或占位符，无法呈现实际生产结果。

图片服务器（9998 端口）上确实存有 250+ 张图片，但访问需要精确文件名，猜测无法可靠工作。

### 下次预防
- [ ] 凡是依赖文件系统路径的 API，必须用 existsSync 验证后再返回 URL
- [ ] 不要基于关键词"猜测"文件名，实际扫描比猜测可靠
- [ ] content-output 目录命名包含日期前缀，扫描时需用模糊匹配而非精确匹配
