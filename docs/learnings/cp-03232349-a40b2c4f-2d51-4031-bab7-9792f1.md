## 04-ship.md 加入 4.4 Update Memory 步骤（2026-03-24）

### 根本原因

Stage 4 Ship 流程在清理（Clean）前缺少状态记忆同步步骤，导致每次任务完成后 state.md/roadmap.md 未更新，系统对自身状态的认知逐渐与现实脱节。

设计上 4-Stage Pipeline 只关注代码交付，未将记忆更新纳入 Ship 阶段的标准流程。这造成每次任务完成后，AI 在下次对话中需要重新探索已完成的状态，浪费时间且容易遗漏。

修复方式是在 4.5 Clean 之前插入 4.4 Update Memory 步骤，明确要求更新 state.md 和 roadmap.md。
