# Learning: BrainModelsPage handleSaveOrgan API 参数名修复

## 任务
修复 BrainModelsPage.tsx handleSaveOrgan 函数中 fetch body 参数名错误。

### 根本原因
前端 `handleSaveOrgan` 发送 body 时使用了 camelCase 命名（`agentId`, `model`），
而后端 API `/api/brain/model-profiles/active/agent` 使用 snake_case（`agent_id`, `model_id`）。
前后端命名约定不一致，前端开发时未对照后端 API 文档。

### 下次预防
- [ ] 前端调用 Brain API 时，先查阅 `packages/brain/src/routes/` 中对应路由的参数解构，确认命名约定
- [ ] Brain 后端统一使用 snake_case，前端 fetch body 也应使用 snake_case
- [ ] 添加 E2E 测试或集成测试，覆盖保存 organ 模型的完整流程
