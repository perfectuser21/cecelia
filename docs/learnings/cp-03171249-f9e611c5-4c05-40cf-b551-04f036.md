# Learning: syncOrphanTasksOnStartup 孤儿改 requeue

**分支**: cp-03171249-f9e611c5-4c05-40cf-b551-04f036
**日期**: 2026-03-17

## 背景
Brain 重启时，executor.js 的 `syncOrphanTasksOnStartup` 将所有 `in_progress` 任务无条件标记为 `failed`，导致合法任务被静默丢弃。

## 根本原因
原始代码对"进程消失"只有一种处理路径：标记 `failed`。未考虑进程消失的两种来源：
1. **Brain 重启中断**：进程因 Brain 进程退出而终止，任务本身没有错误
2. **真实失败**：任务因错误退出（有 error_message 或已耗尽重试）

## 解决方案
添加判断逻辑，基于以下字段区分：
- `retry_count < max_retries AND error_message IS NULL` → 可重试 → `queued`
- `retry_count >= max_retries OR error_message 非空` → 真实失败 → `failed`

max_retries 通过 JOIN `retry_config` 表动态获取，fallback 默认 3。

## 下次预防
- [ ] 任何将任务标记为 failed 的代码路径，先检查是否有重试余额
- [ ] Brain 重启场景（进程消失但无错误）与任务失败场景（进程有错误码）要明确区分
- [ ] executor.js 中新增的孤儿处理逻辑需要同步更新单元测试
