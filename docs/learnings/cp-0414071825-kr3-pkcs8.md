# Learning: KR3 私钥转换 + WX Pay 阻断点分析

## 根本原因

KR3 微信支付配置任务被重复派发（3次，每次401失败），根因是：
1. WX Pay 商户号（MCHID/V3_KEY/SERIAL_NO）需在微信商户平台人工注册，Agent 无法自动完成
2. 前序任务失败是因为尝试调用需要认证的外部 API 而没有有效凭据
3. 私钥已存在（PKCS#1 格式）但未转换为微信支付要求的 PKCS#8 格式

## 下次预防

- [ ] KR3 WX Pay 商户号属于"外部阻断"（需人工注册），Brain 派发此任务时应标记 `blocked_reason: pending_human_action`
- [ ] 私钥格式转换是唯一可自动化的步骤，通过 `--convert-key` 模式已完成
- [ ] 商户号就绪后：`bash scripts/kr3-setup-wx-pay.sh --mark-done`，KR3 即可进入灰度测试
- [ ] 避免对需要人工注册的外部服务重复派发 dev 任务
