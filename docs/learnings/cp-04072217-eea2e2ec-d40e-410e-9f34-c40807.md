## context.js OKR 进度恒为 0% — progress 列与 current_value 脱钩（2026-04-08）

### 根本原因

`key_results` 表有两个进度相关字段：
- `current_value`：手动设置，从未被 kr-verifier 自动更新
- `progress`：由 `kr-verifier.js` 定时计算并写入（实际进度来源）

`context.js` 用 `current_value / target_value` 计算 Objective 进度，
而 `okr/current` 用 `progress` 列。两者脱钩导致 context 接口永远返回 0%。

### 下次预防

- [ ] 新增进度 API 时，优先查看 `kr-verifier.js` 了解实际数据来源再写 SQL
- [ ] `progress` 是 KR 进度的 SSOT，`current_value/target_value` 是遗留设计，写 OKR 进度 SQL 时统一用 `progress` 列
- [ ] brain-unit tests 应覆盖 context.js 返回非零进度的断言
