# Learning: Generator subagent — Stage 2 写代码从主 agent 拆出

**Branch**: cp-03290734-e27ff28c-4f5f-4a01-ad30-5d3e0a
**Date**: 2026-03-29

## 背景

把 /dev Stage 2 的代码编写阶段从主 agent 内部拆成独立 Generator subagent，解决 context 污染问题。

### 根本原因

主 agent 带着大量 brain context（OKR、任务历史、调度信息）直接写代码，导致 context 污染。Generator subagent 是一个"隔离写代码环境"，只看 spec 和代码，不知道外部调度逻辑。

### 实现要点

1. Generator subagent 的 prompt 只包含三要素：Sprint Contract（Task Card）+ CLAUDE.md 编码规范 + 探索发现的代码库上下文
2. 明确的隔离约束：禁止调用 localhost:5221（Brain API），禁止读取 OKR/KR/决策历史，禁止修改 .dev-mode 文件
3. 主 agent 变成纯编排者，保留 2.3 自验证（verify-step.sh）和 2.4 code_review_gate
