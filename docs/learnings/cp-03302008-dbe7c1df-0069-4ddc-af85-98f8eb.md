---

### 根本原因

1. **预存在测试超时**：`tick-suggestion-integration.test.js` 中的测试调用 `executeTick()`，内部触发真实的 thalamus LLM bridge 调用，耗时 32–39 秒，超过 vitest 默认 30 秒超时。这是预存在问题，与本次改动无关，但阻断了 Gate 1。
2. **测试文件路径配置**：vitest 配置只 include `packages/brain/src/__tests__/`，把测试放在 `tests/` 根目录不会被执行，需放到 `packages/brain/src/__tests__/` 下才能被 Gate 1 捕获。
3. **DoD Test 字段格式**：verify-step.sh 用 `^\s+Test:` 匹配，字段必须是 `  Test:`（两个空格缩进），不能有 `-` 前缀。

### 下次预防

- [ ] 新增集成测试时检查 vitest include 路径配置，确认测试文件在正确目录下
- [ ] 改动会影响 tick 流程时，先跑一次 `tick-suggestion-integration.test.js` 确认是否有超时问题，若有则提前加 `{ timeout: 60000 }`
- [ ] Task Card DoD 的 Test 字段用 `  Test:`（两空格），不加 `- ` 前缀
- [ ] 写 `[BEHAVIOR]` DoD 时优先用 `packages/brain/src/__tests__/` 下的文件引用，而非根目录 `tests/`
