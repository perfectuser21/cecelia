### 根本原因

DoD Task Card 格式错误：使用了 `### [BEHAVIOR]` Markdown 标题格式，而 `check-dod-mapping.cjs` 的 `parseDodItems()` 只解析 `- [ ]`/`- [x]` checkbox 行。导致 CI DoD Verification Gate 报告"缺少 [BEHAVIOR] 条目"，即使文件内容完整。

同时：test 文件放置于 `packages/brain/src/__tests__/` 时触发了 "Brain New Files Must Update brain.md" L2 gate（`^packages/brain/src/.*\.js$` 正则包含 `__tests__` 子目录）。

### 下次预防

- [ ] DoD 条目只用 `- [x] [TYPE] 描述` checkbox 格式，禁止 `### [TYPE]` 标题格式
- [ ] `Test:` 字段缩进两个空格（`  Test:`），不加破折号，紧跟在 checkbox 行下面
- [ ] 新增 `packages/brain/src/**/*.test.js` 文件时，同步更新 `.agent-knowledge/brain.md`
- [ ] `feat:` PR 需要 `docs/instruction-book/features/` 条目（或 `[SKIP-DOCS]` 豁免）
