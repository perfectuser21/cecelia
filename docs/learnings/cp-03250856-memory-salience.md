# Learning: 记忆系统 PR9 — computeSalience 多维度评分

Branch: cp-03250856-memory-salience
Date: 2026-03-25

## 实现内容

`computeSalience` 从 4 行正则扩展为 8 个维度，取最高分：
- 纠正类：0.90（不变）
- 决定类：0.85（原 0.8，轻微提升）
- 洞察类：0.80（新增：发现/原来/理解/关键/规律/本质）
- 情绪强烈类：0.75（新增：担心/焦虑/开心/难受）
- 计划类：0.70（新增：下一步/计划/安排/打算）
- 疑问类：0.55（原 0.6，略降）
- 长消息（>=50字）：0.55（新增）
- 基础分：0.30（不变）

## 关键决策

取最高分而非叠加，避免"恰好含多个关键词"的普通消息被误判为高显著性。
保持 0.30 基础分，向后兼容现有逻辑。

## 根本原因

原实现只有纠正/决定/疑问三类，rumination 阈值 0.7 导致只有纠正和决定消息能进入反刍。
洞察、计划、情绪这类有实质内容的消息全部被设为 0.3 而跳过，memory_stream 的"高价值"段稀疏。
根本设计缺陷：salience 维度设计时未对齐 rumination 过滤器阈值，导致 5 类有价值消息类型被系统性忽略。

## 下次预防

- [ ] 新增维度时先写测试确认分布合理性，避免所有消息都 >= 0.7
