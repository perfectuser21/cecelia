# Learning: feat(engine): [CONFIG] 显式内容注入 — 主 agent 把 Planner 输出直接传入 Generator 和 Evaluator prompt

**Branch**: cp-03291925-eeb0d90d-6d24-4bd2-9313-14f015
**Date**: 2026-03-29

---

## 背景

在 /dev 工作流中，Generator 和 spec_review(Evaluator) subagent 原本依赖自主读取 Planner 产出文件（Task Card + Sprint Contract）。这导致文件路径错误或被忽略时，subagent 可能在没有 Planner 上下文的情况下运行，破坏三阶段强链路。

---

## 根本原因

Planner → Generator → Evaluator 的上下文传递依赖文件路径约定，而非物理内联，存在"隐式依赖"断链风险：
- subagent 可能忽略文件读取指令
- 文件路径可能因 worktree 切换或命名问题失效
- 无法在 prompt 层面强制保证上下文注入

---

## 解决方案

1. **02-code.md**：Generator spawn 前读取 Task Card + Sprint Contract（`.dev-gate-spec.{branch}`），文件不存在则 exit 1。prompt 模板新增 `{SPRINT_CONTRACT_CONTENT}` 占位符，spawn 时物理注入。
2. **01-spec.md**：spec_review spawn 前读取 Task Card，提取 DoD 章节，文件不存在则 exit 1。prompt 模板新增 `{TASK_CARD_DOD_CONTENT}` 占位符，spawn 时物理注入。

---

## 下次预防

- [ ] 新增 subagent spawn 时，检查 prompt 模板是否包含所有必要上下文占位符
- [ ] 若 subagent 需要读取文件，优先改为主 agent 读取后注入，而非 subagent 自主读取
- [ ] 在 DoD 中加入 `{PLACEHOLDER}` 字符串检查，确保模板更新与 pseudocode 同步
