# Learning: P0 修复 — RCA→auto-fix 无限重试，熔断机制失效

**Branch**: cp-03242101-11233a9d-7b76-4884-9909-62f12c
**Date**: 2026-03-25
**PR**: #1562

## 问题描述

Brain probe 失败时会触发 auto-fix，但存在两个 bug 导致死循环：
1. `capability-probe.js` 使用 `probe_${f.name}_${Date.now()}` 作为 signature，Date.now() 使每次 signature 唯一，3 次熔断永远无法触发
2. `dispatchToDevSkill` 的 guard 只查 `status='failed'`，不查 `queued/in_progress`，正在修复中仍会重复创建任务

### 根本原因

**bug 1**：signature 稳定性设计缺失。signature 的语义应为"同一类问题的唯一标识符"，用于 dedup 和熔断。混入 `Date.now()` 违背了这个语义，每次都是新 signature，熔断计数器永远不会累积到 MAX。

**bug 2**：guard 查询不完整。只看 `failed` 状态等于只看"过去已经失败的"，没有看"正在处理中的"。正确的去重逻辑应同时包含活跃状态（`queued/in_progress`）和历史失败状态。

## 修复方案

1. `capability-probe.js:343`: `probe_${f.name}_${Date.now()}` → `probe_${f.name}`
2. `auto-fix.js:158-179`: 单条 SQL 同时统计 failed_count 和 active_count，用 PostgreSQL FILTER 语法

```sql
SELECT
  COUNT(*) FILTER (WHERE status = 'failed') AS failed_count,
  COUNT(*) FILTER (WHERE status IN ('queued', 'in_progress')) AS active_count
FROM tasks
WHERE trigger_source = 'auto_fix'
  AND tags::jsonb ? $1
  AND updated_at > NOW() - INTERVAL '7 days'
```
