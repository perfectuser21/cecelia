# Learning: write-current-state.sh 完善

## 背景

为 `scripts/write-current-state.sh` 添加"最近 PR"（`dev-records` API）和"P0 Issues"（blocked/failed P0 任务）两个章节，使 CURRENT_STATE.md 覆盖任务要求的全部内容。

### 根本原因

脚本 `write-current-state.sh` 已存在但缺少"最近 PR"和"P0 Issues"两个章节。
测试文件 `scripts/__tests__/write-current-state.test.sh` 仅检查脚本结构（文件存在、语法、超时保护），不检查输出内容。
内容缺失问题只能通过 DoD `[BEHAVIOR]` 测试（运行脚本后读文件验证章节）来发现，而不是靠代码结构检查。

### 关键发现

1. **P0 数据注入安全**：通过环境变量（`P0_BLOCKED="$VAR" python3 -c ...`）传递 JSON，避免 shell 插值注入风险。直接用 `'''${VAR}'''` 插值方式在 JSON 含特殊字符时会崩溃。

2. **降级保护模式**：每个 `curl` 调用都遵循 `|| echo "{}"` 模式，Python 内部 `try/except` 打印降级文本而非 `exit 1`，确保 Brain 离线时全程不阻断 Stage 4。

3. **测试文件已预先存在**：`scripts/__tests__/write-current-state.test.sh` 在本次 PR 之前已存在，覆盖结构检查（文件存在、语法、worktree 兼容、--max-time、离线退出码）。内容完整性由 DoD `[BEHAVIOR]` 测试补充验证。
