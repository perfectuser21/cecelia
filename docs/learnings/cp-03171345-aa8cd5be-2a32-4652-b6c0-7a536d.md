## check-dod-mapping 对 ARTIFACT Test 命令的格式约束

### 根本原因

`check-dod-mapping.cjs` 对 `[ARTIFACT]` 类型的 Test 命令有白名单约束（只接受 node/npm/psql/curl 等），但 `[BEHAVIOR]` 和 `[GATE]` 类型接受 `bash -c` 包装的任意命令。

直接用 `manual:grep -c ...` → ARTIFACT 类型会触发"Test 命令必须包含真实执行命令"错误。

### 下次预防

- [ ] ARTIFACT 类型的文件存在性检查，Test 命令用 `manual:bash -c "grep -c '...' file"` 而非 `manual:grep -c ...`
- [ ] BEHAVIOR 和 GATE 类型可以直接用 `bash -c "..."`
- [ ] 写完 Task Card 后立即运行 `node packages/engine/scripts/devgate/check-dod-mapping.cjs` 本地验证
