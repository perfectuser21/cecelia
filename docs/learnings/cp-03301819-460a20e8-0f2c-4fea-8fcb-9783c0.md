# Learning: PipelineOutputPage 设计细节优化

**分支**: cp-03301819-460a20e8-0f2c-4fea-8fcb-9783c0
**日期**: 2026-03-31

### 根本原因

GenerationTab 组件最初设计时未考虑重新生成功能，因此没有 pipelineId prop，导致无法在组件内调用 POST /api/brain/pipelines/:id/run 接口。

生成阶段展示只有 completed_at 一个时间点，缺少 started_at（开始时间）和 review_issues（错误详情列表），用户看不到每个阶段的耗时和失败原因，无法定位执行问题。

这是典型的"初始功能只展示结果，后续需要升级为展示过程"的迭代模式，Stage 1 设计时应提前考虑可操作性（操作按钮）和可观测性（时间+错误）。

### 下次预防

- [ ] 前端组件需要调用 API 时，确保从父组件传入必要的 ID（pipelineId 等），而非依赖全局状态或路由 hook
- [ ] 生成阶段展示设计：started_at + completed_at + review_issues 是标准三件套，新建类似 Tab 时直接包含
- [ ] 整体重新生成和单步重新生成若调用同一接口，需在 UI 文案上做区分或在 PRD 中明确接口语义
