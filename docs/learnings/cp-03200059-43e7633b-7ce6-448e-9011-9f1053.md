# 激活CI L4 DevGate质量门禁能力开发记录

## CI DevGate L4层激活（2026-03-20）

### 根本原因

本次开发流程整体顺畅，主要遇到一个DoD测试命令路径错误问题。DoD测试最初指向不存在的`brain-ci.yml`文件，实际应该是`ci-l1-process.yml`。这是因为在编写DoD测试时，凭记忆引用了旧的CI文件名，没有先确认当前仓库中实际存在的CI文件路径。

### 下次预防

- [ ] 编写DoD测试命令前，先用`ls .github/workflows/`确认实际的CI文件名
- [ ] DoD测试命令涉及文件路径时，必须先验证文件存在性，避免凭记忆编写
- [ ] 对于CI文件引用，建议用通配符或动态查找的方式，减少硬编码路径依赖
- [ ] 继续使用per-branch Learning文件模式，避免并行开发时的文件冲突
