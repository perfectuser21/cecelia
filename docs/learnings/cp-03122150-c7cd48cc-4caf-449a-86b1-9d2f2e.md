# Learning: fix(profile) cortex 切换为 anthropic-api provider

## 本次任务

将 DB 中 `profile-anthropic` 的 `cortex.provider` 从 `anthropic`（bridge）改为 `anthropic-api`（直连），同时在 `FALLBACK_PROFILE.cortex` 中添加 `fallbacks` 数组。

## 关键发现

### 1. DEFINITION.md schema_version 两处同步（避坑）

修改 `selfcheck.js` EXPECTED_SCHEMA_VERSION 后，L2 facts-check 报错：
```
✗ schema_version: code=152 ≠ doc=151 (DEFINITION.md)
```

`DEFINITION.md` 中有**两处** schema_version 需要同步更新：
- 第 386 行：`| **schema_version** | ... | Schema 版本: 151 |`
- 第 676 行：`5. **Schema 版本** — DB 版本 >= '151'（>= 检查，向前兼容）`

两处都要改，否则 facts-check.mjs 失败。

### 2. FALLBACK_PROFILE.cortex fallbacks 结构

`llm-caller.js` 的 `callLLM()` 从 `agentConfig.fallbacks` 读取 fallback 列表：
```js
const fallbacks = agentConfig.fallbacks || [];   // [{model, provider}, ...]
const candidates = [primary, ...fallbacks];
```

在 `FALLBACK_PROFILE.cortex` 中添加 `fallbacks` 字段即可让 DB 不可用时也有 fallback 保护。

### 3. anthropic vs anthropic-api 的区别

| 方式 | 路径 | 特点 |
|------|------|------|
| `anthropic` | cecelia-bridge → claude -p | 走订阅，有并发限制 |
| `anthropic-api` | 直连 Anthropic REST API | 走 API key，快 5-8x |

DB profile 更新为 `anthropic-api` 后，cortex 调用会走 `callAnthropicAPI()`，无 bridge 中间层。
