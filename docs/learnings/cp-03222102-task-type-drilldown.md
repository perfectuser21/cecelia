# Learning: TaskTypeConfigPage 三层下钻架构

**Branch**: cp-03222102-task-type-drilldown
**Task**: a676faa7-960a-4db8-9488-ef95051edd19

### 根本原因

扁平表格模式（所有任务类型在一张表里）有两个核心问题：
1. 信息密度过高，用户看不到层级关系
2. 内联编辑体验差，只有少数行可编辑，绝大多数行没有交互感

### 解决方案

三层下钻架构（master-detail pattern）：
- 第一层：4个 ABCD 类别大卡片，展示约束说明和任务数量
- 第二层：点击类别后，左侧展示该类所有任务，右侧保留空态提示
- 第三层：点击任务行后，右侧 DetailPanel 滑入，展示完整详情+编辑表单

面包屑导航支持逐层返回（任务路由 > A类 > dev）。

### 下次预防

- [ ] 页面有层级关系时，优先用 master-detail 或 drill-down，不要用扁平表格
- [ ] 面包屑是必备组件，让用户随时知道自己在哪一层
- [ ] 详情面板（右侧）比模态框更好：不打断列表浏览，可以快速切换不同任务
- [ ] 只读字段和可编辑字段要在 UI 上有明确区分（约束说明文字 + 按钮状态）
