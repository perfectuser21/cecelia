# Learning: CI 覆盖率门禁扩展至 fix/refactor + L4 task-lifecycle E2E

## 背景
将 L3 CI coverage-delta job 从只覆盖 `feat` PR 扩展至 `fix`/`refactor` PR，同时新增 packages/quality/src/task-lifecycle.test.ts 任务生命周期 E2E 测试。

### 根本原因
1. **CI Config Audit 要求 [CONFIG] 标签**：修改 `.github/workflows/*.yml` 时，PR 标题必须含 `[CONFIG]` 或 `[INFRA]` 标签。此 PR 改了 ci-l3-code.yml 但标题没有 `[CONFIG]`，导致 CI Config Audit 失败。
2. **DoD 未勾选**：Task Card 的 DoD 条目必须在 push 前将 `- [ ]` 改为 `- [x]`，否则 DoD Verification Gate 失败。
3. **macOS stat 不兼容**：pre-existing 测试文件 `tests/hooks/pr-gate.test.ts` 和 `branch-protect.test.ts` 使用 `stat -c %a`（Linux专属），在 macOS 上报错。verify-step.sh 本地门禁因此无法通过。解法：换用 `statSync(HOOK_PATH).mode`（跨平台）。

### 下次预防
- [ ] 改 `.github/workflows/*.yml` 时，PR 标题必须带 `[CONFIG]` 标签
- [ ] DoD 条目的 `- [ ]` 必须在 `step_2_code: done` 写入前全部改为 `- [x]`
- [ ] 修改 packages/ 下文件前先在 macOS 跑 `npm test --workspace=<pkg>` 验证无平台兼容问题
