# Learning - DoD grep 必须与实际代码写法对齐（2026-03-12）

**Branch**: cp-03120340-d884a517-58ac-48ce-b84d-8c869d
**PR**: #894

### 根本原因

DoD 验收条目的 Test 命令写的是 `grep -c 'ORDER BY priority ASC'`，但实际 SQL 用的是 CASE 表达式排序（`ORDER BY CASE priority WHEN 'P0' THEN 0 ... END ASC`），字符串不匹配导致 grep 返回 0，CI DoD Verification Gate 失败。

根因是写 DoD 时按"语义"写了 grep pattern，而不是按实际代码内容写，没有本地执行验证就提交。

### 下次预防

- [ ] DoD 的每条 `Test: manual:bash -c "grep ..."` 命令，写完后必须在本地手动执行一遍，确认输出非零且 exit code 0，再 push
- [ ] PostgreSQL UPDATE 不支持直接 ORDER BY LIMIT，必须用子查询 `WHERE id IN (SELECT id ... ORDER BY ... LIMIT $1)`；MySQL 才支持直接写
- [ ] 流控常量命名：已有 `UNBLOCK_BATCH_LIMIT`、`QUARANTINE_RELEASE_LIMIT`，新增 `MAX_REQUEUE_PER_TICK`；命名风格不统一，后续统一为 `*_PER_TICK` 或 `*_LIMIT` 其中一种
