# 重构 executeCopywriting 圈复杂度（2026-04-03）

### 根本原因

`executeCopywriting` 原始实现将文案生成的所有路径（LLM 路径、静态降级路径）、findings 过滤逻辑、prompt 构建逻辑全部内联在一个函数体内，导致圈复杂度达到 26。核心问题是"单体函数反模式"：多个独立职责（过滤、构建 prompt、LLM 调用、静态模板渲染）没有被拆分为命名函数，使得每新增一个条件分支都直接累加主函数复杂度。

### 下次预防

- [ ] 写 executor 类函数时，超过 3 个 if/catch 分支即提取为命名子函数
- [ ] 扫描时若单个 executor 函数超过 50 行，先拆再写逻辑
- [ ] 新增 LLM 调用路径（try/catch + 正则解析）必须封装为独立函数，不内联
