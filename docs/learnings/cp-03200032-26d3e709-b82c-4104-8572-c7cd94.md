### 根本原因

`content-pipeline-executors.js` 使用 `require('@resvg/resvg-js')` 在 ESM 模块（`"type": "module"`）中加载渲染库，CommonJS `require()` 在 ESM 中不可用，导致每次 `executeExport` 运行时直接崩溃，卡片全部生成失败，`manifest.json` 中 `image_set.status` 始终为 `"failed"`。

### 修复方案

使用 `dynamic import()` + `pathToFileURL()` 替代 `require()`：

```javascript
// ❌ 旧：CommonJS require() 在 ESM 中崩溃
const { Resvg } = require('@resvg/resvg-js');

// ✅ 新：动态 import() + pathToFileURL 实现 ESM 兼容
const { Resvg } = await import(pathToFileURL(resvgDir).href);
```

关键点：必须将本地路径转换为 `file://` URL 格式，否则 Node.js 动态 import 无法解析绝对路径。

### 技术细节

- `WeChat 封面（900×383）`通过独立 SVG 直接渲染，而非从封面 PNG 裁切，更简单且不需要额外图像处理依赖
- `assembleMedia()` 返回 `{ cover, cards, coverWechat, count, errors }`，渲染失败时收集到 `errors[]` 而不是抛出异常，保证健壮性
- resvg 不可用时（测试/CI 环境），函数优雅降级：`count=0`、`errors` 有内容，不崩溃
