# Learning: CI Gate — manual: 命令白名单本地前置拦截

branch: cp-03231535-2d436535-a973-41cc-9687-bc55dd
date: 2026-03-23

## 根本原因

1. **DoD Test 路径解析**：Task Card 中 `tests/devgate/xxx.test.ts` 路径由 verify-step.sh 在 `$PROJECT_ROOT` 下解析，但 engine 测试文件实际在 `packages/engine/tests/`，导致 [BEHAVIOR] DoD 条目跑 `vitest` 时找不到文件。正确做法：[BEHAVIOR] 条目使用 `manual:node -e` 内联 Node.js 断言，直接调用被测模块。

2. **YAML step name 含冒号**：CI YAML 中 step name `Check manual: command whitelist` 因冒号触发 YAML 解析错误。必须用引号包裹：`- name: "Check manual: command whitelist"`。

3. **Task Card DoD 勾选未提交**：verify-step.sh step2 成功后自动将 `[ ]` 更新为 `[x]`，但该更改留在 working tree 未 commit。CI DoD 检查读的是 git 提交内容，因此看到未勾选项报错。Stage 3 push 前必须确认 task card 已提交最新状态。

4. **Engine 版本未 bump**：修改了 `packages/engine/hooks/`，L2 Consistency 要求 `package.json` 版本与 main 不同。需在同一 PR 内 bump 5 个文件（package.json / package-lock.json / VERSION / .hook-core-version / regression-contract.yaml）以及 feature-registry.yml。

## 下次预防
