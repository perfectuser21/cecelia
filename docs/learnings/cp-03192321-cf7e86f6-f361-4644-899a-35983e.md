## Brain 启动时自动触发 distill-learnings — Learning 注入链路打通（2026-03-19）

### 根本原因

`distill-learnings.js` 在 PR #1161 已实现同时写入 `knowledge` 和 `learnings` 两张表，
但脚本本身是独立 CLI 工具，没有任何代码引用它（grep 全仓库只有自身文件）。
脚本从未被调用，导致 `learnings` 表的 `best_practice` 记录为 0，
`learning-retriever.js` 无数据可用，100+ 篇历史 Learning 从未被注入到 /dev 任务 PRD。

根本原因是：实现了功能但没有"接线"——脚本孤立存在，触发点缺失。

### 下次预防

- [ ] 新增任何数据同步脚本（distill、backfill、seed 类）时，必须同时检查：是否有调用方？如果没有，必须在 Brain 启动或 CI 中接线
- [ ] 验证 Brain 数据链路时，不只看代码是否存在，还要检查相关表是否有实际数据（`SELECT COUNT(*) FROM learnings WHERE ...`）
- [ ] 修复注入链路类问题后，立即运行脚本验证端到端通路（代码写了 + 数据有了 = 链路通了）
