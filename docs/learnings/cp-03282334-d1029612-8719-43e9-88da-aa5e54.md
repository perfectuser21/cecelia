# Learning: Brain 关键路由集成测试补全

**分支**: cp-03282334-d1029612-8719-43e9-88da-aa5e54
**日期**: 2026-03-29

## 背景

为 Brain 4 个关键路由（/context、/tasks、/okr/current、/health）补集成测试，使用真实 PostgreSQL 连接。

### 根本原因

`/api/brain/context` 刚修复了 SQL bug（PR #1663），但无任何集成测试保护。CI 只有 mock DB 的单元测试，无法捕获 SQL 列名回归。

### 踩到的坑

1. **Task Card `## 成功标准` 区段不能用 checkbox**：`- [x]` 格式会被 `check-dod-mapping.cjs` 识别为 DoD 条目，要求 Test 字段，导致 L1 失败。成功标准应使用普通 `- ` 列表。

2. **新增 `packages/brain/src/` 下的 `.js` 文件需更新 brain.md**：L2 `Brain New Files` 检查对所有 `packages/brain/src/*.js`（包括 `__tests__/` 下的测试文件）要求同步更新 `.agent-knowledge/brain.md` 或 `DEFINITION.md`。

3. **goals.js 集成测试需 mock tick/okr-tick/nightly-tick**：goals.js 引入 30+ 传递依赖，需 mock `tick.js`、`okr-tick.js`、`nightly-tick.js` 避免 CI 中触发调度器副作用，但 `db.js` 保持不 mock（真实 DB 连接）。

### 下次预防

- [ ] Task Card 成功标准章节用 `- ` 纯文本，不用 `- [x]` checkbox
- [ ] 在 `packages/brain/src/__tests__/` 下新增 `.js` 文件后，在 `.agent-knowledge/brain.md` 中添加简要说明
- [ ] 新增 goals.js 相关路由测试时，记得 mock tick/okr-tick/nightly-tick 三个模块
