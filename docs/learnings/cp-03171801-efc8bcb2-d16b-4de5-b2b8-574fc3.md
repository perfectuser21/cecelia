# Learning: WriteStream fd:null 在 ubuntu-latest 上的 stdio 兼容性

**分支**: cp-03171801-efc8bcb2-d16b-4de5-b2b8-574fc3
**日期**: 2026-03-17

### 根本原因

`createWriteStream` 创建的 WriteStream 在文件实际打开前 `fd` 为 `null`。
在 macOS（hk-vps）上 `spawn` 可以接受 `fd:null` 的 stream（内部做了延迟处理），
但在 Linux（ubuntu-latest）上 `spawn` 在接收 stdio 参数时立即校验 fd，
因此抛出 `TypeError: The argument stdio is invalid. Received WriteStream { fd: null }`。

### 修复方案

在 `createWriteStream` 之后、`spawn` 之前，`await` WriteStream 的 `open` 事件：

```javascript
const logFile = createWriteStream('/tmp/cecelia-bridge.log', { flags: 'a' });
await new Promise((resolve, reject) => {
  logFile.once('open', resolve);
  logFile.once('error', reject);
});
// 此时 logFile.fd 已有效，可安全传给 spawn stdio
const child = spawn(..., { stdio: ['ignore', logFile, logFile] });
```

### 下次预防

- [ ] 凡是将 WriteStream 传给 `spawn stdio` 的代码，必须在 `open` 事件后才 spawn
- [ ] macOS 本地通过 ≠ ubuntu-latest 通过；涉及 stdio 的代码要在 Linux 环境验证
- [ ] 新增 bridge/子进程启动代码时，检查 stream 是否已 open
