# Learning — fix(brain): circuit_breaker.status HALF_OPEN 误报 all_closed

## 根本原因

`GET /api/brain/health` 的 `circuit_breaker.status` 计算逻辑只过滤 `OPEN` 状态，
HALF_OPEN 状态不被包含在 `openBreakers` 中，导致：
- `cecelia-run` 处于 HALF_OPEN（恢复中）时仍返回 `status: "all_closed"`
- `open: []` 为空，运维人员和 CURRENT_STATE.md 无法感知系统正在恢复

顶层 `status: "healthy"` 是正确的（HALF_OPEN 不应触发 degraded），但子字段误导性强。

## 诊断过程

1. CURRENT_STATE.md 显示 Brain "degraded" —— 实际是 write-current-state.sh 在熔断器 OPEN 时写入的，现在已过期
2. 当前 health API 返回 "healthy"，但 `circuit_breaker.states.cecelia-run.state: "HALF_OPEN"` 与 `status: "all_closed"` 矛盾
3. 根因：内容管道 P0 failures（"research findings 为空"）触发 cecelia-run 熔断器 3 次失败 → OPEN → 冷却 5 分钟 → 自动过渡 HALF_OPEN

## 修复

在 `packages/brain/src/routes/goals.js` 添加 `halfOpenBreakers` 检测：
- `all_closed`: 无 OPEN 也无 HALF_OPEN
- `recovering`: 有 HALF_OPEN（恢复中，允许探针）
- `has_open`: 有 OPEN（阻塞派发）

新增 `circuit_breaker.half_open` 数组，明确列出正在恢复的熔断器。
