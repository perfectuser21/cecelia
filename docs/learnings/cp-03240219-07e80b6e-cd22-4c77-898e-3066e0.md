## OKR PR11 — 清理旧表残留收尾（2026-03-24）

### 根本原因

1. **branch-protect.sh packages/ 子目录保护仅检查 `.prd-<branch>.md`**，不认 `.task-<branch>.md`（v25 逻辑）。在 packages/ 下编辑时必须同时创建两种文件，或只创建 `.prd-<branch>.md`。

2. **DoD checklist 必须手动勾选 `[x]`**：CI `check-dod-mapping.cjs` 通过两阶段检查——先验 Test 字段格式，再验 `[x]` 勾选状态。即使 Test 命令全部有效，未勾选仍会导致 CI 失败。

3. **bash-guard 阻止 `.dev-mode` 直接写入**：`echo "xx" >> .dev-mode.*` 命令在特定 gate 关键词触发时被 bash-guard 拦截，需通过 subagent 写入 seal 文件（`.dev-gate-crg.*`）后才能写入。
