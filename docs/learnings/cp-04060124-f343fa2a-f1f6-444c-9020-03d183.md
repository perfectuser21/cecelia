# Learning: 选题引擎测试 mock 断链修复

**任务**: [SelfDrive] [选题闭环] 基于数据驱动的AI自动选题引擎

## 根本原因

三个 topic 相关测试因模块级 import 导致的链式 DB 导入崩溃：
- `topic-selector.test.ts`: 导入 `topic-selector.js` → `llm-caller.js` → `account-usage.js` → `db.js` → `dotenv/config`（包不存在）
- `topic-selection-scheduler.test.js` / `topic-selection-trigger.test.js`: 导入 `topic-selection-scheduler.js` → `topic-suggestion-manager.js` → `db.js`（模块级 `import pool from './db.js'`）

这类测试文件只需要 mock 直接依赖（llm-caller 或 topic-suggestion-manager），就能阻断整条链。

## 下次预防

- [ ] 新建测试文件时，在 import 之前检查被测模块的 import 链，找出所有直接/间接导入 `db.js` 的路径并逐一 mock
- [ ] `topic-suggestion-manager.js` 在模块级初始化 pool（`import pool from './db.js'`），任何导入它的测试文件都必须加 `vi.mock('../topic-suggestion-manager.js', ...)`
- [ ] `llm-caller.js` 通过 `account-usage.js` 间接依赖 `db.js`，同样需要 mock
- [ ] 若测试只测试纯计算函数，先检查是否能通过单独 export 纯函数来避免加载整个模块
