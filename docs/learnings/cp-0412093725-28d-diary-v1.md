# Learning: 管家日报系统 v1 上线

**分支**: cp-0412093725-28dc56ab-156d-4cf1-9a08-24d26c
**日期**: 2026-04-12

## 做了什么

增强 `diary-scheduler.js`，在每日管家日报中增加两个新板块：
- **KR 进度**：查询 `key_results WHERE status='active'`，展示每个 KR 的名称和进度（含文本进度条）
- **异常告警**：查询今日 `status IN ('failed', 'quarantined')` 的任务数

同时新增 `diary-scheduler.test.js`（11 个测试用例）。

### 根本原因

diary-scheduler.js 虽已在 tick.js 中接入，但日报内容过于简单（只有 PR/决策/任务数），缺少推动 KR3 闭环所需的 KR 进度视角和异常感知能力。

### 下次预防

- [ ] `shouldGenerateDiary` 已从内部函数改为 export，未来需测试该触发逻辑
- [ ] 告警查询用 `updated_at::date` 而非 `completed_at::date`，因为 failed 任务 `completed_at` 为 NULL
