**分支**：cp-03300714-410f202f-b820-4864-9deb-f6fb25
**日期**：2026-03-30

## 根本原因

本次任务是新建一个 Pipeline 详情页（PipelineOutputPage），作为内容工厂的作品主页，展示内容生产的全生命周期（生成、发布、数据）。原有 ContentFactory 只有列表没有详情页，用户无法直接查看单个 Pipeline 的产出内容、发布状态和数据记录。

页面采用四 Tab 结构（Summary / 生成记录 / 发布记录 / 数据记录），Summary 展示汇总指标与执行进度，生成记录拉取 Brain API 真实数据，发布与数据记录暂用 mock 占位。路由设计为 `/content-factory/:id`，与列表页共享前缀，ContentFactory 列表行通过 `<Link>` 跳转。

技术选型上，深色背景 `#07050f` 使用 inline style 注入（Tailwind 不直接支持该颜色），标题渐变通过 `bg-gradient-to-r from-purple-400 to-pink-400 bg-clip-text text-transparent` 实现，整体风格参照 Justin Welsh 预览页。

## 关键决策

1. 路由设计为 `/content-factory/:id`，与已有的 `/content-factory` 列表页同前缀，语义清晰
2. 发布记录和数据记录均使用 mock 占位（8 平台），不引入新后端表
3. 深色背景 `#07050f` 通过 inline style 注入（Tailwind 不支持任意颜色），紫色渐变用 `bg-gradient-to-r from-purple-400 to-pink-400 bg-clip-text text-transparent`
