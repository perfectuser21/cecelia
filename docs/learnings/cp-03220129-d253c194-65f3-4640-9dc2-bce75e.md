# Learning: 重构 extractTaskSummary（圈复杂度 21 → 4）

**Branch**: cp-03220129-d253c194-65f3-4640-9dc2-bce75e
**Date**: 2026-03-22

## 背景

`packages/brain/src/auto-learning.js` 中 `extractTaskSummary` 函数的圈复杂度为 21，
超出阈值 10，由代码复杂度扫描任务触发重构。

### 根本原因

函数同时承担了两类职责：

1. **cecelia-run 合成结果处理**（exit_code/stderr_tail/failure_class 字段拼接）
2. **通用对象摘要提取**（属性优先链 + 类型归一化）

两类逻辑混在一个函数里，导致：
- 嵌套 if 深度达到 3 层
- `||` 短路运算 7 个（每个算一个决策点）
- 三元运算符叠加

总决策点数 ~20，大幅超标。

### 下次预防

- [ ] 新增函数时，若承担超过 1 类输入格式，立即拆分为命名子函数
- [ ] 函数里超过 5 个 `||` 运算时，提取辅助函数降低复杂度
- [ ] `if (typeof === 'object')` 嵌套内再嵌套 `if` → 必须抽函数

## 修复方案

将 `extractTaskSummary` 拆分为三个函数：

```
extractTaskSummary（主函数，4 决策点）
├─ _extractRunResult（处理 cecelia-run 合成结果）
└─ _extractObjectSummary（处理通用对象）
```

主函数仅做类型分派（null → string → run-result → generic-object），
每个子函数内部逻辑单一，复杂度均 ≤ 5。

## 验收

- ✅ 18/18 auto-learning 测试通过
- ✅ 字段优先顺序不变（error_details → error → message → result → findings → summary）
- ✅ 主函数行数从 25 行降至 10 行
