# Learning: 重构高圈复杂度函数的拆分模式

branch: cp-03220129-d253c194-65f3-4640-9dc2-bce75e
date: 2026-03-22

### 根本原因

`extractTaskSummary` 函数将多个职责混合在单一函数体内：
1. cecelia-run 合成结果处理（含 4 个条件分支）
2. 通用对象摘要提取（含 6 个 `||` 短路运算链 + 类型判断）
3. 顶层分派逻辑

三种职责叠加导致圈复杂度从 1 累计至 21，超过阈值 10 约 2 倍。

### 下次预防

- [ ] 当单一函数同时处理"分类 A 的分支"和"分类 B 的分支"时，立即提取为独立命名函数
- [ ] 连续 3 个以上 `||` 短路运算应提取为独立函数，命名体现其"选取优先级"语义
- [ ] 重构时保留字段优先顺序注释（`error_details → error → message → ...`），防止顺序漂移
- [ ] Task Card `## 成功标准` 必须用 `- [ ]` 格式，纯文本行 CI 会拒绝
