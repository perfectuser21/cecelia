# Learning: cortex insight → action 自动闭合机制

## 关键陷阱

### 1. selfcheck 版本 bump 需要同步更新 3 个测试文件

修改 `EXPECTED_SCHEMA_VERSION` 时，以下测试文件会硬编码期望值而失败：
- `desire-system.test.js` — D9 测试
- `selfcheck.test.js` — EXPECTED_SCHEMA_VERSION 测试
- `learnings-vectorize.test.js` — schema version 测试

**规则**：每次 migration + selfcheck 版本 bump，必须同步更新这 3 个文件。

### 2. PRD 成功标准必须用 `## 成功标准` 二级标题

`check-prd.sh` 按二级标题（`##`）匹配，不识别粗体 `**成功标准**:`。

### 3. Learning 文件必须在 PR 首次 push 时就存在

L1 Learning Format Gate 在每次 CI 运行时检查，不能等到最后一步才创建。

## 实现决策

- 用关键词检测（不用 LLM）避免超时影响主路径
- fire-and-forget 模式（setImmediate）不阻塞 recordLearnings
- payload 中冗余存储 source_learning_id（数据库列 + jsonb 双备份）
- applied=true 更新失败设计为非致命（task 已创建是主要目标）
