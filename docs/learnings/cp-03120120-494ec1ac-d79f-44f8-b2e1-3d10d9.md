# Learning: cortex insight → action 自动闭合机制

### 根本原因

selfcheck `EXPECTED_SCHEMA_VERSION` 版本 bump 时，三个测试文件硬编码期望旧版本号，没有随之更新，导致 L3 测试失败（3 个新 failure > baseline 0）。同时 PRD 的 `**成功标准**:` 粗体格式不被 `check-prd.sh` 识别（要求 `## 成功标准` 二级标题），Learning 文件缺失也导致 L1 失败。

### 下次预防

- [ ] 每次新建 migration + 修改 EXPECTED_SCHEMA_VERSION 时，必须同时更新 3 个测试文件：`desire-system.test.js`、`selfcheck.test.js`、`learnings-vectorize.test.js`
- [ ] PRD 成功标准必须使用 `## 成功标准` 二级标题（`check-prd.sh` 按标题匹配，不识别粗体）
- [ ] Learning 文件需在第一次 push 前就创建并提交（L1 每次 CI 运行都检查）

### 实现决策

- 用关键词检测（不用 LLM）避免超时影响主路径，fire-and-forget 模式不阻塞 recordLearnings
- tasks 表新增 `source_learning_id` 列，建立 insight→task 反向索引，去重查询效率更高
- applied=true 更新失败设计为非致命（task 已创建是主要目标）
