### 根本原因

Branch-protect hook 的 `find_prd_dod_dir()` 函数在 `packages/` 子目录开发时，只在遍历过程中检查 `.prd-${branch}.md`，不检查 `.task-${branch}.md`。

即使根目录有正确的 `.task-${branch}.md`，hook 返回到项目根时仍会判断"无 per-branch PRD"，输出 `__SUBDIR_NO_PERBRANCH_PRD__` 并阻止写代码。

该行为是 v25 的 `packages/` 子目录保护机制引入的，目的是防止旧任务的 `.prd.md` 被误用，但尚未同步支持 `.task-${branch}.md` 格式。

### 下次预防

- [ ] `packages/` 子目录开发时，**必须同时创建 `.prd-${BRANCH}.md`**（内容可简短，引用 task card 即可），branch-protect hook 才能通过
- [ ] `.dev-mode` 文件需要包含 `tasks_created: true` 字段，否则 branch-protect 会报 "Task Checkpoint 未创建" 错误
- [ ] Learning 文件的"根本原因"章节需要至少 3 行非空内容，否则 CI Learning Format Gate 失败
- [ ] 遇到 `packages/` 子目录开发被 hook 阻止时，优先创建 PRD 文件，不要尝试修改 hook 逻辑
