### 根本原因

Branch-protect hook 的 `find_prd_dod_dir()` 函数在 `packages/` 子目录开发时，只检查 `.prd-${branch}.md`，不检查 `.task-${branch}.md`。即使根目录有正确的 `.task-${branch}.md`，hook 仍会返回 `__SUBDIR_NO_PERBRANCH_PRD__` 并阻止写代码。

### 下次预防

- [ ] 在 `packages/` 子目录开发时，**同时创建** `.prd-${BRANCH}.md`（内容可简短，引用 task card 即可）
- [ ] task card 的格式在 branch-protect hook 的 find 函数里尚未支持，遇到 packages 子目录时必须加 `.prd-*.md`
- [ ] `.dev-mode` 文件需要包含 `tasks_created: true` 字段，否则 branch-protect 会拦截

### 技术决策

- tick 默认从 `false` 改为 `true` 是最小改动，利用 `?? true` 的 nullish coalescing 语义——已有记录的 DB 值不受影响，只改首次启动的默认行为
- `readCurrentState()` 用 `fileURLToPath(import.meta.url)` 定位仓库根，比硬编码路径更健壮
- 内容截断 2000 字符放在 `buildAnalysisPrompt` 而非 `readCurrentState`，保持原始数据可复用
