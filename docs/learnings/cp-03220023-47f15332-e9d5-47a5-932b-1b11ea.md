## branch-protect.sh .dod.md 归属校验（2026-03-22）

### 根本原因

branch-protect.sh v25 的 packages/ 子目录保护逻辑只认 `.prd-{branch}.md`，
不认 `.task-{branch}.md`（task card 格式）。
导致在 packages/engine/ 下开发时即使有 task card，hook 仍报错阻止写代码。
必须额外创建 `.prd-{branch}.md` 文件才能通过检查。

### 下次预防

- [ ] 在 packages/ 子目录开发时，同时创建 `.prd-{branch}.md` 和 `.task-{branch}.md`，确保 v25 保护通过
- [ ] 并行 agent 同任务时，提前检查是否有 OPEN PR 实现同功能，如已有 CI 通过的 PR 可评估直接关闭自己的 PR
