## 重构 formatItem 降低圈复杂度（2026-04-04）

### 根本原因
`formatItem` 函数内大量 `||` 链（每个 `||` 计入一个分支）和重复的 extras 构建逻辑（depth=1 和 depth>=2 几乎相同的代码块）导致圈复杂度达到 21，超出阈值 10。

### 下次预防
- [ ] 写新函数时，`||` 链超过 2 个就考虑提取辅助函数
- [ ] 重复出现的条件块（`if (x !== undefined && x !== null)`）改用 `!= null` 一次完成
- [ ] 相同的 if/else 分支体（仅参数不同）应提取为带参函数，而不是复制粘贴
