### 根本原因

KR3 进度停留在 25% 有两个原因：

1. **Brain API `checkKR3ConfigDB` 列名 bug**：PR #2348 已修复源码（`key→topic`），但运行中的 Brain 进程未重启，仍使用旧代码（返回 "column key does not exist"）。导致 Brain 无法感知配置状态。

2. **管理员 OpenID 虽已在代码中配置（PR #16 `BUILT_IN_ADMIN_OPENIDS`），但 Brain DB 中缺少对应 `decisions` 记录**，Brain 无法判断此项已就绪。

3. **WX Pay 商户号完全缺失**：`~/.credentials/wechat-pay.env` 不存在，仅有私钥文件（且为 PKCS#1 格式，微信支付要求 PKCS#8）。

---

### 解决步骤

1. 通过 `psql -d cecelia` 直接写入 `decisions` 表，插入 `kr3_admin_oid_initialized` 记录（绕过 Brain API bug）
2. 新增 `scripts/kr3-setup-wx-pay.sh`：引导 WX Pay 配置，自动检测 Brain API 是否有 column bug 并回退到 psql
