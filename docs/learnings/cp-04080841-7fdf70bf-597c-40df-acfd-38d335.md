# Learning: auth 故障遗留 rescue 清理盲区

**分支**: cp-04080841-7fdf70bf-597c-40df-acfd-38d335  
**日期**: 2026-04-08

### 根本原因

account3 于 2026-04-08 凌晨发生 OAuth token 失效，触发 198 个 pipeline_rescue 任务。
已有的 `cleanupDuplicateRescueTasks`（#2099）只取消"同分支多余副本"（每分支保留1条），
剩余 22 个单实例分支的 rescue 任务永远留在 quarantined 状态——
`recoverAuthQuarantinedTasks` 明确排除 pipeline_rescue，形成清理盲区。

### 修复方案

新增 `cleanupAuthQuarantinedRescueTasks`：
- 凭据健康 + auth circuit 全关时运行
- 批量取消所有 `quarantined` + `task_type=pipeline_rescue` + `failure_class=auth` 的任务
- 补全 `cleanupDuplicateRescueTasks` 之后的剩余清理

### 下次预防

- [ ] auth 故障恢复后，检查 credentials/health 中 auth_fail_count_24h
- [ ] 若有 quarantined pipeline_rescue，主动调用 POST /credentials/cleanup
- [ ] `cleanupDuplicateRescueTasks` 之后必须跟 `cleanupAuthQuarantinedRescueTasks`（顺序固定）
- [ ] Brain 重启后，新代码才生效（本 PR 不含重启操作，需单独协调）
