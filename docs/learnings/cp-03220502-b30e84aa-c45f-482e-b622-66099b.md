# Learning: 重构 capability-scanner.scanCapabilities（CC 42 → ≤10）

**分支**: cp-03220502-b30e84aa-c45f-482e-b622-66099b
**日期**: 2026-03-22

## 任务描述

将 `packages/brain/src/capability-scanner.js` 中 `scanCapabilities` 函数从圈复杂度 42 重构至 ≤10。

### 根本原因

原 `scanCapabilities` 函数将以下职责混杂在单一函数中：
1. DB 数据获取（3 个查询串行执行）
2. Brain 白名单判断
3. Brain 嵌入 source 判断
4. Skill 使用情况统计
5. Table 数据检查（带缓存）
6. 状态判定（failing/active/dormant/island）

每层嵌套的 if/else/for 分支都贡献圈复杂度，最终累积到 42。

### 解决方案

提取 9 个命名子函数，按职责单一原则划分：

| 函数 | 职责 | 降低的 CC |
|------|------|----------|
| `fetchScanData` | 并行 DB 查询 | 将循环体逻辑抽出 |
| `fetchEmbeddedSourcesActive` | cecelia_events 查询 | 隔离 try/catch |
| `checkTableHasData` | 单表检查+缓存 | 隔离 try/catch |
| `checkSkillUsage` | Skill 统计循环 | CC=6 独立函数 |
| `checkTableEvidence` | Table 证据循环 | CC=3 独立函数 |
| `determineStatus` | 状态判定 | CC=6 独立函数 |
| `evaluateCapability` | 单能力评估 | CC=5 协调者 |
| `buildSummary` | 汇总计算 | CC=1 |
| `scanCapabilities`（精简后）| 协调层 | CC=2 |

顺带优化：将 3 个串行 DB 查询改为 `Promise.all` 并行执行，减少扫描延迟。
