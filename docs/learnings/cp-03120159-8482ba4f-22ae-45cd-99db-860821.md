# Learning - liveness DEAD 改 requeue（2026-03-12）

**Branch**: cp-03120159-8482ba4f-22ae-45cd-99db-860821
**PR**: #886

### 根本原因

executor.js probeTaskLiveness 在双次确认进程 DEAD 后直接调用 `updateTaskStatus(failed)`，导致任务被永久标记失败。进程死亡通常是 OOM 或系统抢占（非代码 bug），应该重新入队而非永久失败。requeueTask() 已有完整的 retry + FAILURE_THRESHOLD + quarantine 逻辑，liveness DEAD 路径直接复用即可。

### 下次预防

- [ ] 任何"任务终止"逻辑在写 failed 之前，先判断原因是否可重试（OOM/SIGKILL/消失 = 可重试，code exit != 0 = 永久失败）
- [ ] 测试 requeueTask mock 时，按顺序准备 4 个 mockResolvedValueOnce（SELECT + UPDATE + SELECT learnings + INSERT learnings），否则查询对应错误导致抖动
- [ ] liveness 和 watchdog 共用 watchdog_retry_count 计数器，防止两路交替失败绕过 QUARANTINE_AFTER_KILLS 隔离
