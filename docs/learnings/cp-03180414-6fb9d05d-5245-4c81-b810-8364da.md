# Learning: 扫描 Learning 提炼工程规则写入 Brain 知识层

**失败统计**：CI 失败 3 次（历史任务），本地验证通过

## 根本原因

1. **任务三次失败后被重新调度**：Brain 重试机制正常工作，但每次失败都因 Brain 重启导致 in_progress 任务孤儿化，而非代码问题。
2. **知识 API 端点缺失**：knowledge 表虽存在，但没有 REST 端点，导致外部无法查询。
3. **distill 脚本初始版本用 pg npm 包**：在无 node_modules 的环境下失败，应使用 psql CLI。
4. **pg 在 DoD Test 命令中**：早期 DoD Test 直接调用 `const {Pool}=require('pg')` 而非使用 `psql`，在 CI 环境中失败。

## 解决方案

1. 使用 psql CLI（`execSync('psql ...')`）替代 pg npm 包，消除外部依赖
2. 通过 `ON CONFLICT (type, sub_area) WHERE type IS NOT NULL AND sub_area IS NOT NULL DO NOTHING` 保证幂等性
3. 新增 `packages/brain/src/routes/knowledge.js` 提供 GET /api/brain/knowledge 端点
4. migration 155 添加唯一索引支持 ON CONFLICT
