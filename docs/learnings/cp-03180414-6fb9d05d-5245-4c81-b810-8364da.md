## 失败统计
CI 失败 1 次，本地测试失败 0 次

## CI 失败记录
- 失败 #1：三重 CI 失败
  1. `distill-learnings.js` 使用 `import pg from 'pg'`，CI 没有 npm install，ERR_MODULE_NOT_FOUND → 改为 psql CLI subprocess
  2. `manual:curl` 在 CI check-dod-mapping.cjs 中被执行（不像 dod-execution-gate.sh 那样 DEFERRED）→ 改为 manual:node 文件内容验证
  3. Learning 文件缺失 → 补写 docs/learnings/ 文件

### 根本原因
1. **check-dod-mapping.cjs 与 dod-execution-gate.sh 行为不同**：dod-execution-gate.sh 会 DEFER curl 命令，但 check-dod-mapping.cjs 会直接执行所有 manual: 命令（包括 curl）。导致 BEHAVIOR DoD 中的 curl 测试在 CI 中失败。
2. **Brain scripts/ 下的脚本禁止依赖 npm 包**：CI 不做 npm install，import pg 会 ERR_MODULE_NOT_FOUND。
3. **Learning 文件必须在 push 前写好**：Learning Format Gate 在首次 push 时检查，不能事后补。

### 下次预防
- [ ] Brain scripts/ 下的脚本禁止 import npm 包（pg/axios 等），改用 psql CLI 或 Node 内置 fetch，确保在 CI 没有 node_modules 时也能运行
- [ ] `manual:curl` 用于 BEHAVIOR DoD 时，check-dod-mapping.cjs 会直接执行而非 DEFER；改用 `manual:node -e "readFileSync..."` 验证文件内容，或使用 tests/ 路径
- [ ] Learning 文件必须在第一次 push 前写好并加入 commit
