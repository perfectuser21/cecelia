# Learning: KR进度0%根因 — launchd游离进程 + Brain未重启

## 背景
- PR #2005 迁移了10个活跃 KR verifier
- PR #2007 修复了 context.js SQL（改用 `kr.progress` 列）
- 但 Brain 进程（PID 58875）是 4/7 21:20 启动的，比 PR 合并更早，始终运行旧代码

### 根本原因

**launchd 游离进程导致 deploy 失效**

Brain 在 launchd 管辖外手动启动（PID 58875），而 launchd 状态记录的是 PID 27216（已不存在）。
每次 `brain-deploy.sh` 调用 `launchctl kickstart -k` 时：
- launchd 尝试 kill PID 27216（不存在，空操作）
- 启动新进程，但 port 5221 已被 PID 58875 占用，新进程立即失败
- deploy 报告 "success"（因为 kickstart 命令本身成功返回），实际 Brain 代码未更新

**结果**：每次 Brain 代码变更后，deploy 均失败但无报警，Brain 一直运行旧代码。

- [ ] deploy 完成后应验证 `/api/brain/health` 进程启动时间 > deploy 时间（监控加固方向）
