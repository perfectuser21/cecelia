# Learning: 熔断器 HALF_OPEN 探针限制

## 根本原因

`circuit-breaker.js` 中 `isAllowed()` 在 HALF_OPEN 状态下对所有调用返回 `true`，没有限制探针数量。每次 tick（5秒一次）都会通过检查，导致多个探针任务并发派发，违反半开状态"渐进恢复"的设计意图。

## 修复方案

- 添加 `MAX_HALF_OPEN_PROBES = 1` 常量和 `probesSent: 0` 状态字段
- `isAllowed()` 在 HALF_OPEN 时检查 `probesSent < MAX_HALF_OPEN_PROBES`
- 新增 `recordProbeDispatched(key)` 函数消耗探针槽
- `tick.js` 在 circuit breaker 检查通过后立即调用 `recordProbeDispatched('cecelia-run')`
- OPEN→HALF_OPEN 自动转换时重置 `probesSent = 0`

## 下次预防

- [ ] `recordProbeDispatched()` 内部必须调用 `getState()` 触发 OPEN→HALF_OPEN 自动转换，否则直接操作 `breakers.get(key).state` 在时间推进后仍为 OPEN 状态，导致探针槽不被消耗
- [ ] HALF_OPEN 状态下的测试如需调用 `recordProbeDispatched()` 消耗探针槽，必须先通过 `getState()` 或 `isAllowed()` 触发状态转换
- [ ] 新增行为约束：所有 tick dispatch 路径在 `isAllowed()` 通过后必须调用 `recordProbeDispatched()` 消耗探针槽
