## 踩坑记录

### 根本原因

**坑 1：rebase 后 EXPECTED_SCHEMA_VERSION 断言与实际值不一致**
- 原 PR 将 selfcheck.js 的 EXPECTED_SCHEMA_VERSION 从 '202' 改为 '203'
- rebase 到 main 后，main 已推进到 '207'
- 若保留 '203' 断言，运行测试会失败（实际值为 '207'）
- 修复：将 selfcheck.js 和相关测试断言统一改回 '207'

**坑 2：测试文件需反映当前 main 的版本状态**
- 旧 PR 的 fix commit 更新了多个测试文件的版本断言（202→203）
- rebase 到更新的 main 后，这些断言需要再次更新到最新版本（207）
- 不能直接依赖旧 PR 的 fix commit，因为 main 已进一步更新

**坑 3：selfcheck.js 版本字段在多个测试中被校验**
- desire-system.test.js、learnings-vectorize.test.js、selfcheck.test.js
  三个文件均硬编码了 EXPECTED_SCHEMA_VERSION 的期望值
- 统一修改时需确保全部更新一致
