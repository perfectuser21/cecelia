## 修复任务成功率统计 NULL 语义 bug（2026-04-08）

### 根本原因
`getTaskStats24h()` 中 SQL 过滤器 `payload->>'failure_class' != 'auth'` 在 `failure_class` 为 NULL（绝大多数已完成任务未设置该字段）时返回 NULL 而非 true，导致这些任务被排除出 `total` 分母。但 `completed` 计数没有该过滤器，结果分子（239）远大于分母（8），产生 3063% 的虚假成功率。

### 下次预防
- [ ] SQL 中排除某值的过滤器一律使用 `IS DISTINCT FROM`（或显式 `IS NULL OR !=`），禁止裸用 `!= '...'` 处理可能为 NULL 的 JSONB 字段
- [ ] 成功率类统计写完后，检查 `分子 <= 分母` 是否成立（断言）
- [ ] 并行任务（SelfDrive 多 agent）开工前先查 main 最新 commit，避免重复修复同一问题
