## 修复成功率统计 NULL 语义 bug（2026-04-08）

### 根本原因

`getTaskStats24h()` 中使用 `payload->>'failure_class' != 'auth'` 过滤非 auth 失败任务。
PostgreSQL 中 `NULL != 'auth'` 返回 `NULL`（不是 `true`），导致 failure_class 为 NULL 的任务
被排除在 `total` 分母之外，而 `completed` 计数不受此过滤器影响。

结果：completed=239，total=8（只有8个任务有显式非空 failure_class），成功率 = 239/8×100 = 2987% ≈ 3063%。

### 下次预防
- [ ] SQL 中比较可能为 NULL 的 JSONB 字段时，优先使用 `IS DISTINCT FROM` 而非 `!=`
- [ ] 任何成功率/比率计算，上线前用 psql 验证分子 <= 分母（`SELECT CASE WHEN numerator <= denominator THEN 'OK' ELSE 'FAIL' END`）
- [ ] 对 `COALESCE(col, '') != 'auth'` 模式保持警惕 — 虽然规避了 NULL，但不如 `IS DISTINCT FROM` 语义清晰
