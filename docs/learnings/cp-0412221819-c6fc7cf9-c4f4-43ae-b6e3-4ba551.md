## Bridge /llm-call 500 — executor 降级 + startup 错误跳过重试（2026-04-13）

### 根本原因

1. PR #2308 修复了 thalamus→anthropic-api 主路径，但 `executeCopyReview` 和 `executeImageReview`
   在 `callLLM` 抛出异常时直接返回 `{ success: false }` → pipeline 硬失败，任务标记为失败，无法继续。

2. Bridge 500 重试逻辑不区分错误类型：
   - 瞬态限流（exit 1）→ 重试有价值
   - dyld/Library not loaded/ENOENT（启动级别崩溃）→ 重试 2 次全部浪费（每次 3ms 崩溃）

### 修复

1. `content-pipeline-executors.js`：`executeCopyReview` / `executeImageReview` LLM 失败时
   降级返回 `{ success: true, review_passed: true, llm_reviewed: false }`，
   允许 pipeline 继续执行后续步骤（export/publish）

2. `llm-caller.js`：`callClaudeViaBridge` 在 errText 含 `dyld|Library not loaded|ENOENT` 时
   标记 `isStartupError = true`，跳过重试直接进入 implicit anthropic-api fallback

### 下次预防

- [ ] 审核任务是非阻塞的：LLM 审核只是质量守门，不应阻断 pipeline 的生产流
- [ ] Bridge 的 500 响应体应始终含 `retryable` 字段（true/false），让 caller 端无需解析 stderr
- [ ] 瞬态故障 vs 基础设施故障要有不同处理策略
