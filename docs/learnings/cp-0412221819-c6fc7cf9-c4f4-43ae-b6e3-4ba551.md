# Learning: Bridge /llm-call 500 — content review 降级处理

**Branch**: cp-0412221819-c6fc7cf9-c4f4-43ae-b6e3-4ba551
**Date**: 2026-04-13

## 背景

多个文案审核任务因 Bridge /llm-call 500 错误失败，导致整个内容 pipeline 中断。

## 根因分析

### 根本原因

两层问题：

1. **上游根因**（已由 PR #2308/#2309 修复）：
   - thalamus 使用 anthropic（bridge）作为 primary，claude -p 在限流时 exit code 1 → Bridge 返回 500
   - dyld 库版本不匹配（libsimdjson.30.dylib 缺失）导致 claude CLI 无法启动
   - 已修复：thalamus 迁移到 anthropic-api，ENOENT 修复，implicit fallback 添加

2. **本 PR 修复的残留问题**：
   - `executeCopyReview` / `executeImageReview` 当 LLM 所有 fallback 耗尽时返回 `{ success: false }`
   - 这会导致 copy-review 任务标记为 failed，触发 copywriting 重试（内容已 OK，只是审核服务不可用）
   - dyld 类错误会触发 2 次无意义重试（各等 500ms/1000ms）

### 下次预防

- [ ] Review executor 的 LLM 失败应区分：**内容质量失败**（`review_passed: false`）vs **服务不可用**（`skipped_reason`，降级通过）
- [ ] Bridge 500 重试时先检测错误类型：dyld/ENOENT 等启动级别失败不需要重试
- [ ] 写新的 executor 函数时：LLM failure catch → 默认降级（不阻断 pipeline），而非 `{ success: false }`

## 修复内容

1. `llm-caller.js`：检测 dyld/Library not loaded/ENOENT 错误，跳过重试直接抛出（标记 `isStartupError`）
2. `content-pipeline-executors.js`：
   - `executeCopyReview`：LLM 调用失败 → `{ success: true, review_passed: true, llm_reviewed: false, skipped_reason }`
   - `executeImageReview`：同上
3. `llm-caller-bridge-retry.test.js`：新增 dyld 跳过重试测试
4. `content-pipeline-executors.test.js`：新增两个 LLM 失败降级测试
