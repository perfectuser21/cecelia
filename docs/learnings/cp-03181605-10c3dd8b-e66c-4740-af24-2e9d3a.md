### 根本原因

DoD 测试中 `node scripts/generate-ops-summary.mjs --days 8 --dry-run` 在 CI 中退出码 1：CI 环境没有运行 Brain API（localhost:5221），脚本调用 `fetchTasks` 时连接拒绝抛出异常，导致进程 exit(1)。

根本原因：脚本对 Brain API 不可用的情况没有优雅降级，而 DoD 测试在 CI 中会实际执行该命令。

### 下次预防

- [ ] 任何依赖 Brain API 的脚本都要提供 `fetchXxxSafe()` 包装（catch → 返回 `[]`），确保 Brain 离线时返回空数据而非 crash
- [ ] CI 环境没有 Brain API，所有 `manual:` 测试命令如果依赖 localhost 服务，必须能处理连接失败
- [ ] 新建脚本时，先在隔离环境（关闭 Brain）测试一遍 `--dry-run`，确认优雅降级行为
- [ ] Worktree CWD 会随 Bash 命令 `cd` 持久化漂移，写 agent seal 等文件前要用绝对路径或重置 CWD
- [ ] `feat:` PR 需要测试文件；纯运维/工具脚本用 `chore:` 前缀跳过该要求
- [ ] `gh run rerun` 复用旧 CI context（含旧 PR_TITLE），改 PR title 后必须 push 新 commit 才能触发新 CI 读到新 title
