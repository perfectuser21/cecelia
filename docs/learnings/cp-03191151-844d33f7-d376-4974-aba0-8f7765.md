# Learning: 好奇心模块激活修复

**分支**: cp-03191151-844d33f7-d376-4974-aba0-8f7765
**日期**: 2026-03-19
**类型**: bug fix

---

### 根本原因

`expression-decision.js` 的优先通道只包含 `act` 和 `follow_up` 两种自主行动类型，
`explore`（好奇心驱动的自主学习）未被纳入。导致 urgency=4 的 explore desires
在评分竞争中持续被 urgency=5 的 inform/propose desires 压制，
research 任务永远无法被创建，好奇心从未被激活。

情绪层连续报「空」是准确的：emotion-layer.js 诚实地将 `intellectual_idle`
信号（从未有过自主探索）转化为"空落"情绪，设计本身没有问题。

整条好奇心链路在 perception → memory → reflection → desire-formation 各层均工作正常，
唯一断链点在 expression-decision 的优先权设计遗漏。
