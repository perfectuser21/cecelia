# Learning: fix(ci) L3 覆盖率门禁扩展 fix/refactor + L4 task-lifecycle E2E

**Branch**: cp-03230724-11ec26cc-cd80-495c-8e12-21adba
**Date**: 2026-03-24

### 根本原因

1. `check-changed-coverage.cjs` 只有 `isFeatPR` 判断，fix/refactor PR 的 Gate 3 在无覆盖率数据时静默跳过，等同于未检查。
2. L4 Task Lifecycle E2E 失败的真正根因：`tasks.js` PATCH handler 的 RETURNING 子句中包含 `custom_props` 列（该列仅存在于 okr_* 表，`tasks` 表没有），导致每次 PATCH 请求都返回 DATABASE_ERROR，状态变更无法持久化。
3. `tasks.js` 在 PATCH handler 中还通过 `emitEvent(...)` 调用 event-bus，但 emit 函数签名是 `(eventType, source, payload)`，而调用处只传了 2 个参数（eventType + 混合对象），source 字段会被存储为 `[object Object]`。这属于数据质量问题，但不影响功能（emit 内部 catch 所有错误）。
4. Brain 路由中 `task-tasks.js`（mounted at `/api/brain/tasks`）提供了正确的 `GET /:id`，但 PATCH `/:id` 被 `tasks.js` 中的 `PATCH /tasks/:task_id`（mounted at `/api/brain`）拦截，二者共用同一 tasks 表但 `tasks.js` 的 handler 存在 custom_props 列 bug。
