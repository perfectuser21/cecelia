# Learning: 记忆系统 PR6 — Desire Unblock 接通建议管道

## 背景

Desire System 的通知类欲望（warn/propose/inform/celebrate/question）仅发飞书，未接通 suggestion_pipeline。

### 根本原因

`handleNotifyDesire` 函数只调用 `runExpression`（飞书路径），缺少写入 `suggestions` 表的逻辑。
`act`/`follow_up`/`explore` 类型已有管道路径，但通知类欲望被遗漏，导致高价值洞察无法转化为可执行建议出现在 dashboard。
欲望系统六层架构中，Layer 6 表达层只对接了飞书通知渠道，而忘记同时对接建议管道（suggestion pipeline）这条并联路径。

### 解决方案

1. 在 `handleNotifyDesire` 中解构 `score`（已有 expressionCandidate.score）
2. `score > 0.7` 时额外 INSERT 到 `suggestions` 表（source=`desire_system`）
3. 添加 `DESIRE_TO_SUGGESTION_TYPE` 映射：warn→alert, propose→insight_action, 其余→general

### 下次预防

- [ ] 新增欲望类型时，检查是否需要接通 suggestions 管道（不只是飞书）
- [ ] `handleNotifyDesire` 等函数需同时考虑两条输出路径：通知渠道 + 建议管道
- [ ] 写新的 desire handler 前先对比 act/follow_up/explore 的实现模式，确保一致性
