### 根本原因

1. **DoD Test 字段缺少 `manual:` 前缀**：DoD 验收项的 Test 字段写了 `node -e "..."` 但
   check-dod-mapping.cjs 的正则要求必须匹配 `tests/|contract:|manual:` 三种格式之一。
   缺少 `manual:` 前缀导致所有 DoD 条目被标记为"缺少 Test 字段"。

2. **`selfcheck.test.js` 版本期望未同步**：migration 179 更新了 `selfcheck.js` 的
   `EXPECTED_SCHEMA_VERSION` 从 '178' → '179'，但 PR #1456（同一任务前一次执行）忘记
   同步更新 `selfcheck.test.js` 中的 `expect(EXPECTED_SCHEMA_VERSION).toBe('178')`，
   导致 3 个 unit tests 失败。

3. **Learning 文件须在合并前提交**：Learning Format Gate 在 L1 Process Gate 阶段就检查，
   不能等到 PR 合并后再补。

### 知识点

- DoD check-dod-mapping.cjs 解析 Test 字段的正则：
  `/^\s*Test:\s*(tests\/[^\s]+|contract:[^\s]+|manual:.+)\s*$/`
- `manual:` 后支持的命令：`node`/`npm`/`curl`/`bash`/`psql`（CI 白名单）
- `grep`/`ls`/`cat` 不在白名单，文件内容检查统一用 `manual:node -e "require('fs').readFileSync(...)"`
- migration SQL 命名规则：`{3位数字}_{description}.sql`，migration runner 按文件名排序执行
- 并行 agent 同任务时检查已有 PR：同功能若有 PR 已存在，分析其 CI 失败原因，
  在新分支修复问题而非重复相同代码
