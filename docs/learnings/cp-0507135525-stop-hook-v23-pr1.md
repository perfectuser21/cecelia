# Learning：Stop Hook v23 PR-1 基础设施

## 背景
Stop Hook 5 天内 v18→v22 经历 5 次大重构 + ~20 次补丁，根因是用一个"档案"（state file）模拟"活动状态"，需要持续维护其活性导致复杂度爆炸——209 行代码堆满 cwd 路由 / session_id 双通道 / ghost rm / mtime expire / PreToolUse 8 段。本 PR 是 v23 重构序列的第 1 步：搭心跳模型基础设施，零回归。

## 根本原因
v22 的 209 行复杂度全部来自"考证档案是否有效"。**根因是混合"流身份"和"所有权"两个正交维度到单一 key**——cwd-as-key 防不住 assistant drift，session_id 路由防不住 sub-shell 覆盖，每次换主键都暴露新一面。心跳模型用文件存在性 + mtime（操作系统级事实）替代档案考证，从设计上消除"活性维护"这一类工作。
