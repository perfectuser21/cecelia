# generate-report POST Brain dev-logs（2026-03-19）

## 任务概述

修复 `generate-report.sh`：在生成质检 JSON 报告后，通过 `curl POST localhost:5221/api/brain/dev-logs`
将报告上传到 Brain `dev_execution_logs` 表，打通执行质量数据的上报闭环。

### 根本原因

`dev_execution_logs` 表和 `/api/brain/dev-logs` 端点在之前的 PR 中已建好，但 `generate-report.sh`
末尾从未调用过。质检数据只存在本地 `.dev-runs/`，Brain 无法感知执行质量趋势。
根本原因：建表/建端点和调用方是分两个 PR 做的，中间没有打通的验收标准。
