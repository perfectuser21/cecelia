# Learning: write-current-state.sh 新增最近 CI 状态章节

### 根本原因

`write-current-state.sh` 脚本在添加多个 API 章节时，Brain `dev-records` API
返回的是 `{"success": true, "data": [...]}` 包装格式，而非裸数组。
Python 直接做 `json.load()[:5]` slice 操作导致 `slice` 类型错误出现在输出中。

同时，`git reset --hard origin/main` 会让分支看起来"已合并"，branch-protect hook
拒绝后续所有文件写入操作，陷入死锁。应改用 `git rebase` 或保存 patch 后重新应用。

### 下次预防

- [ ] Brain API 响应格式不统一：调用前先检测 `isinstance(resp, dict)` 并取 `resp.get('data', resp)`
- [ ] 添加新 API 章节后必须实际运行 `bash scripts/write-current-state.sh` 检查输出内容，不能只做语法检查
- [ ] 解决合并冲突时用 `git rebase`（而非 `git reset --hard`），避免 branch-protect 僵尸状态
- [ ] `feat:` PR 必须同时提交：学习文件 + 测试文件 + 代码，三件一起
