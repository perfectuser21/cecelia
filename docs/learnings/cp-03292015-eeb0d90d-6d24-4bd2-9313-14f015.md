# Learning: spec_review Evaluator 内容注入

### 根本原因
01-spec.md Sprint Contract Gate 的 retry loop 里只说 "prompt = spec-review SKILL.md 全文 + Task Card 全文"，没有配套的伪码展示如何实现内容注入，导致主 agent 可能只传文件路径给 subagent（与 02-code.md Generator 部分已有的内容注入伪码风格不一致）。

### 下次预防

- [ ] Engine skill 步骤文档改动后，检查 loop/重试逻辑中是否有残留的旧路径引用，与新增的内容注入原则是否矛盾（维度 H blocker）
- [ ] 新增伪码块时，同步把 loop 内原来的简短描述更新为"见下方伪码"引用，避免两处描述并存造成歧义
- [ ] Planner subagent 生成的 Task Card 的 [GATE] Test 必须用 `try{...}catch(e){process.exit(1)}` 包含显式断言，不能只用 `accessSync`（无显式 process.exit 会被 check-dod-mapping 判为弱测试）
