# Learning: dispatcher 派发前 quota 冷却检查

## 根本原因

dispatchNextTask() 在配额冷却期间缺乏全局门控，导致持续发起失败派发，
增加无效 API 调用和噪音日志。

## 实现要点

1. **新模块独立**：`quota-cooling.js` 与 `executor.js` 的 `getBillingPause()` 并列，
   语义不同（billing pause = 显式配额耗尽，quota cooling = 通用冷却窗口）
2. **返回值区分**：冷却时返回 `{skipped: true, reason: 'quota_cooling'}`（非 dispatched:false），
   以便调用方区分"跳过"与"失败"
3. **容错降级**：`isGlobalQuotaCooling()` 调用用 try/catch 包裹，
   抛出异常时降级为不检查（继续派发），不影响正常流程

## DoD 类型选择

- `[BEHAVIOR]` 条目如果用 `grep` 验证，CI 会报"弱测试"错误
