# Learning: miniapp 端到端集成4个Bug修复

**分支**: cp-04090113-e0275cf6-20a8-4f48-bcc9-855037
**miniapp PR**: perfectuser21/zenithjoy-miniapp#11
**日期**: 2026-04-09

## 根本原因

### Bug 1: checkQuota 误阻断无限配额用户
`remainingQuota = -1` 代表"无限"，但 `remaining <= 0` 将其误判为耗尽，导致 premium 会员被拦截。

### Bug 2: WXML 不能调用 Page JS 方法
微信 WXML 模板 `{{method(arg)}}` 语法无效，只能读取 `data` 中的静态值。`formatDate(membership.expireDate)` 始终渲染为空。

### Bug 3 & 4: 文章云函数被手动禁用
`getRecommendArticles` 和 `getArticleDetail` 在某次开发中被改为返回空/禁用，没有还原，导致文章整个流程断路。

## 下次预防

- [ ] `-1 = unlimited` 的语义必须在配额判断条件里显式排除
- [ ] WXML 中禁止调用函数，格式化逻辑只放在 setData 前
- [ ] 手动禁用云函数时必须加 `// TODO: 恢复` 注释并在 PR 说明
- [ ] 端到端验证任务优先检查云函数是否被手动覆盖
