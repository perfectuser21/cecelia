# Learning - Math.min 截断 Bug 根因分析

**Branch**: cp-03110537-d7c5e7e3-25f8-40a0-880b-0e27d5
**PR**: #860
**关联修复**: PR #858

### 根本原因

`cecelia-bridge.js` 中的 `Math.min(timeout, BRIDGE_TIMEOUT_MS)` 将所有 per-request 超时截断至 120000ms（2分钟）。当 Cortex 调用 Opus 模型设置 300s 超时时，实际超时被静默截断为 120s，导致 Opus 推理在 121s 时收到 SIGTERM，进程 exit 143，Cortex 报离线。

### 下次预防

- [ ] 任何涉及"超时"的代码修改，需同时检查调用链上所有中间层是否也有截断逻辑
- [ ] `Math.min` 用于上限截断时，上限常量的语义（默认值 vs 硬上限）需明确分离
- [ ] 在 PR 审查中，对 `Math.min(userTimeout, hardCap)` 模式的代码需特别关注
