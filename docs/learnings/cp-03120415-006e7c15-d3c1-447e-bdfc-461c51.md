## /dev Step 10 分类写入 learnings DB（2026-03-12）

### 根本原因

CI 失败 2 次（共 3 次 push），根因如下：

1. **PRD 格式错误**：成功标准用了 `**成功标准**:` 粗体，`check-prd.sh` 要求 `## 成功标准` 二级标题。粗体在渲染上看起来像标题，但脚本用 `grep '^## '` 匹配，粗体不匹配。

2. **DoD 测试命令依赖 localhost**：DoD L29 使用 `curl localhost:5221` 集成验证，CI 环境无 Brain 服务，curl 失败；同时 DoD 所有项未标 `- [x]`，`check-dod-mapping.cjs` 检测到"未验证项"触发 FAIL。

3. **Learning 格式不符合规范**：第一版 Learning 用旧格式（`### [日期]` + `**预防措施**:`），check-learning.sh v2 要求 `## 标题（日期）` + `### 根本原因` + `### 下次预防` + `- [ ]` checklist 格式。

### 下次预防

- [ ] PRD 成功标准永远用 `## 成功标准` 二级标题，绝不用粗体 `**成功标准**:`
- [ ] DoD 测试命令只用静态命令（grep/ls/cat），绝对不依赖运行时服务（curl localhost）
- [ ] DoD 项代码完成后立即标记为 `- [x]`，不要等到 CI 失败才改
- [ ] Learning 文件格式：`## 标题`、`### 根本原因`（正文）、`### 下次预防`（`- [ ]` 列表）
