# Learning: 选题池 v1 — 主理人清单驱动

## 根本原因
内容 pipeline 失控的根因是"选题源头绕过了主理人"——NotebookLM 每天自动扩词 60-75 个关键词，直接创建 pipeline，没有人工审核，没有节奏控制。

## 解决方案
1. **禁用自动扩词** (`DISABLED = true`)：`topic-selection-scheduler.js` 中保留标志位，合并后即停止 AI 自动生成
2. **新建 topics 表**：主理人在 Dashboard 手写清单（title/angle/priority/status）
3. **节奏控制**：`topics_rhythm_config.daily_limit`（默认 1/天）控制每日触发量
4. **调度器**：`topic-pool-scheduler.js` 在窗口期从 `status='已通过'` 的 topics 里按优先级拉取 → 创建 content-pipeline task

## 关键设计决策
- **不删旧路由**：`/api/brain/topics` 原有路由（topic_suggestions 相关）保留，新 CRUD 路由用 `/pool` 子路径区分
- **Tab 而非新页面**：ContentFactory 加 Tab，避免新增导航入口带来的 PR 改动量
- **Migration 234**：同时创建 `topics` 表和 `topics_rhythm_config` 表，rhythm 独立表便于后续扩展

## 下次预防
