# Learning: 重构 gatherFailureContext 复杂度

## 根本原因
`gatherFailureContext` 圈复杂度 25 的来源：
- ctx 初始化中 8 个 `||` 短路运算符（每个计一分）
- 3 层嵌套条件分支（task/run/similar）
- payload 内部的 `&&` + 双 `||` 链

## 解决方案
提取 4 个辅助函数：
- `buildBaseContext(failure)` — 纯对象构建，去除 || 分支负担
- `enrichWithTaskMetadata(ctx, taskId)` — 单一职责：task 查询
- `extractPayloadFields(payload)` — 纯函数：payload 解析
- `enrichWithRunPayload(ctx, runId)` — 单一职责：run_events 查询
- `enrichWithSimilarFailures(ctx, reasonCode)` — 单一职责：同类统计

主函数降为：buildBaseContext + try(3 enrich) + catch = 复杂度 2

## 下次预防
- [ ] 新建异步函数时，若需要 3 个以上 if-嵌套+OR 链，直接拆子函数
- [ ] ctx 初始化 || 超过 5 个时提取 buildXxxContext() 模式
- [ ] 复杂度扫描 >15 时，先数 || 运算符数量作为首要切入点
