# Learning: cp-03222227-context-consistency-scan

## 任务概述
Stage 2 新增强制周边一致性扫描：改动文件 A 时扫描同目录文件的矛盾引用。

### 根本原因
- 系统只做精准手术式改动（只改 A），周边同模块文件 Y 引用了 A 旧行为却无人感知，导致矛盾积累、系统越改越脏
- code-review-gate 维度 H 只检查改动文件内部矛盾，缺少跨文件模块一致性检查
- 用户反馈：每次改功能只加了功能，没有同步清理周边矛盾内容

### 下次预防

- [ ] code-review-gate 维度 H 跨文件模块一致性检查：同模块文件有矛盾引用 = blocker，必须同 PR 修复
- [ ] worktree 被清理后恢复工作：先 `git log` 确认现有 commits，再决定需要重做什么
