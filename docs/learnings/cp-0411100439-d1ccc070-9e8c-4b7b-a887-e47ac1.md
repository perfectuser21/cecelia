# Learning: Dashboard KR5 阻断Bug清零 + 演示脚本

## 任务摘要
扫描并修复 Dashboard 3 大模块阻断 bug，编写 20 分钟演示脚本。

### 根本原因
Brain API 返回 `paused` 状态（pipeline 被暂停时），但前端 `STATUS_ICON`/`STATUS_COLOR`/`STATUS_BG`/`VERDICT_COLOR`/`VERDICT_LABEL` 常量均未覆盖此枚举值，导致：
- 列表页：`paused` verdict 渲染为 `⏳` + 原始字符串 "paused"（无中文标签）
- 详情页：状态标题显示 "— paused"（无样式图标）

### 下次预防

- [ ] 新增 API 状态枚举时，同步更新 `STATUS_*` 和 `VERDICT_*` 常量（两个文件：`HarnessPipelinePage.tsx` + `HarnessPipelineDetailPage.tsx`）
- [ ] 发现 API 枚举扩展时，搜索所有 `STATUS_ICON` / `VERDICT_LABEL` 引用点检查覆盖
- [ ] 所有状态映射对象应有 fallback 处理（已有：`status in STATUS_ICON ? status : 'not_started'`）
