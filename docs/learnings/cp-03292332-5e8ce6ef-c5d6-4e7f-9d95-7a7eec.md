## 任务信息

- Date: 2026-03-30

## 根本原因

/dev 工作流依赖多个 pipeline 安全属性（fail-closed、无上限孤儿处理、gate 链完整性、seal 格式），这些属性曾存在 bug，修复后缺乏回归测试保护。安全不变量一旦被代码变更静默破坏，可能导致：
- pr-review.yml API 错误放行 PR（fail-open 漏洞）
- stop-dev.sh 孤儿状态 N 次后放行（绕过循环控制）
- gate 链缺失导致未审查代码进入生产

## 关键设计决策

1. **只读不写的元测试**：测试文件仅读取现有 script/config 文件内容断言，不修改任何 hook/script
2. **变量赋值位置检测**：gate 链顺序测试使用 `indexOf('spec_seal_file=')` 而非 `indexOf('dev-gate-spec')`，避免注释中的首次出现干扰顺序断言
3. **ROOT 路径计算**：`resolve(__dirname, '../../..')` 从 `packages/engine/tests/` 向上 3 级到达 monorepo root

## 遇到的坑

1. **check-dod-mapping.cjs 路径解析**：`Test: tests/pipeline-integrity.test.ts` 必须相对于 projectRoot，`packages/engine/tests/` 路径不被接受，改为 `manual:node -e "...accessSync..."` 格式
2. **branch-protect.sh 两步前置**：per-branch PRD 文件 + `tasks_created: true`，缺一不可
3. **bash-guard 防抢跑**：`step_2_code: pending → done` 在 commit 前被拦截，必须先有代码提交
