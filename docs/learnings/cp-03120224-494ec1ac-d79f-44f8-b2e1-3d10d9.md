# Learning: Cortex Insight → Dev Task 自动闭合机制（PR #888）

**PR**: feat(brain): cortex insight → dev task 自动闭合机制
**Branch**: cp-03120224-494ec1ac-d79f-44f8-b2e1-3d10d9
**Date**: 2026-03-12

## 背景

cortex_insight 类型的 learning 写入 learnings 表后，`applied=false` 永远不会变化——因为没有任何机制把 insight 转化为可执行 task。本 PR 实现了 insight → dev task 的自动闭合链路。

## 关键设计决策

### 1. createTask() 新增 learning_id 参数 → 参数索引需同步更新测试

`createTask()` 的 `commonParams` 数组新增 `learning_id` 后，`domain`/`owner_role`/`delivery_type` 的参数索引全部 +1。
- 无 domain 路径：params.length 从 13 → 14
- 有 domain 路径：params[11] → params[12]，params[12] → params[13]，params[13] → params[14]
**教训**：每次给 `commonParams` 新增参数，必须同步更新 `actions.test.js` 中所有参数索引断言。

### 2. DoD test 路径必须精确

DoD 中的 test 路径 `packages/brain/__tests__/` 与实际 `packages/brain/src/__tests__/` 不一致，导致 CI 找不到文件（exit code 2）。
**规则**：DoD 中的 `manual:bash -c "grep -c ... <path>"` 路径必须与仓库实际结构一致，不能凭记忆写路径。

### 3. schema version 测试需同步所有关联文件

升级 `EXPECTED_SCHEMA_VERSION`（149 → 150）后，需要同步更新：
- `selfcheck.test.js`
- `desire-system.test.js`（D9 测试）
- `learnings-vectorize.test.js`（schema version match 测试）

### 4. cortex.js import actions.js 形成单向依赖

`cortex.js` 导入 `createTask` from `actions.js`（`actions.js` 不导入 `cortex.js`），形成单向依赖，无循环风险。

## 防止重复的规则

- [ ] 修改 `commonParams` 数组长度时，立即 grep actions.test.js 中所有 `params[N]` 断言并更新
- [ ] 新建 DoD test 路径时，用 `ls <path>` 验证文件存在后再写入 DoD
- [ ] 升级 schema version 时，先 `grep -r "149" packages/brain/src/__tests__/` 找出所有需要更新的测试
