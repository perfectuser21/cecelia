# Learning - Cortex Insight → Dev Task 自动闭合机制

**Branch**: cp-03120224-494ec1ac-d79f-44f8-b2e1-3d10d9
**PR**: #888

### 根本原因

新增 `learning_id` 到 `createTask()` 的 `commonParams` 数组后，`domain`/`owner_role`/`delivery_type` 参数索引全部 +1（params.length 13→14），导致 `actions.test.js` 中 3 个参数索引断言失败。Schema version 从 149 升到 150 后，`selfcheck.test.js`、`desire-system.test.js D9`、`learnings-vectorize.test.js` 三个测试文件中的硬编码版本号 '149' 未同步更新。DoD test 路径 `packages/brain/__tests__/` 与实际 `packages/brain/src/__tests__/` 不一致，CI 找不到文件（exit 2）。

### 下次预防

- [ ] 修改 `commonParams` 数组长度时，立即 `grep -n "params\[" packages/brain/src/__tests__/actions.test.js` 找出所有索引断言并同步更新
- [ ] 升级 schema version 时，先 `grep -r "'149'" packages/brain/src/__tests__/` 找出所有需更新的测试文件
- [ ] 新建 DoD test 路径时，用 `ls <path>` 验证文件存在后再写入 DoD，禁止凭记忆写路径
- [ ] `cortex.js` 导入 `actions.js` 形成单向依赖（无循环风险），但 `actions.js` 不能再导入 `cortex.js`
