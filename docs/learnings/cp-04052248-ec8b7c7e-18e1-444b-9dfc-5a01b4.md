# Learning: 团队 Dashboard v1 — 内容/数据一览化

## 根本原因
Dashboard 需要聚合多个 Brain API（发布结果、OKR、集群状态、社媒数据），其中部分数据源（publish_results、social/trending）目前为空因为数据采集任务尚在进行中。

## 下次预防
- [ ] Dashboard 页面必须对每个数据源单独 `Promise.allSettled`，不因一个 API 失败阻塞整体
- [ ] Empty State 设计：当数据为空时提示"数据采集中"而非报错
- [ ] Brain `publish-results/stats` 路由必须在 `publish-results` 路由**之前**注册，否则 express 会把 `/stats` 当成 `:id` 参数处理
- [ ] worktree 中 node_modules 不存在，tsc 需要用主仓库的 `node_modules/.bin/tsc`
