## [2026-03-12] 验证 PR #874 billing pause 熔断端到端路径

**失败统计**：CI 失败 1 次（DoD 格式），本地测试失败 2 次

### 根本原因

1. **DoD [BEHAVIOR] + grep 弱测试**：DoD 条目标注为 `[BEHAVIOR]` 但 Test 用 `grep -c`。CI check-dod-mapping.cjs 检测到 BEHAVIOR 条目使用弱测试（grep/ls）时直接 exit 1。正确做法是脚本内容检查用 `[ARTIFACT]`，运行时行为验证才用 `[BEHAVIOR]`。

2. **PATCH /tasks/:id 允许转换 queued→in_progress 是错的**：PATCH endpoint 的 `allowedTransitions` 只有 `pending→in_progress`，`queued` 不在其中。billing pause 的 `setBillingPause` 调用在 DB 事务之外，所以即使任务仍为 queued（UPDATE 影响 0 行），billing pause 全局标志依然会被设置。

3. **Learning 格式缺少 `### 根本原因` / `### 下次预防` / `- [ ]` checklist**：check-learning.sh 用 regex 匹配标题，必须有这三个元素。

### 下次预防

- [ ] DoD 中验证脚本内容的条目统一用 `[ARTIFACT]`，不要用 `[BEHAVIOR]`（BEHAVIOR 需要 curl 或 tests/ 验证）
- [ ] 调用 PATCH /tasks/:id 前先看 allowedTransitions，queued→in_progress 不合法，应直接跳过
- [ ] Learning 文件必须含 `### 根本原因`、`### 下次预防`、`- [ ]` 三要素，否则 check-learning.sh 报错
- [ ] Brain 运行时验证脚本（需要 localhost:5221）不能放入 DoD Test 字段，CI 环境无 Brain 服务
