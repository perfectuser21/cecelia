# Learning: [SelfDrive] 修正 Scanner 对内嵌能力的识别逻辑

## 任务概述
修复 Brain Scanner 对内嵌能力的识别逻辑，解决 Brain API 导入错误，实现准确的内嵌能力检测。

## 根本原因
1. **Brain API 导入缺失**: `packages/brain/src/routes/status.js` 缺少 `IDEMPOTENCY_TTL` 和 `ALLOWED_ACTIONS` 的导入，导致 Brain API `/api/brain/status` 端点返回错误
2. **功能已存在但未被识别**: `scanner-cli.js` 实际上已经存在完整的 Brain API 集成功能，只是 Brain 服务本身有问题

## 关键发现
- Brain 内嵌能力识别功能实际已经实现在 `scanner-cli.js` 中
- 问题的核心是 Brain 服务的导入错误，而非 Scanner 逻辑缺失
- 修复后 Scanner 能正确输出: `branch-protect: embedded`, `ci-devgate: embedded`, `watchdog: embedded` 等

## 技术细节
1. **修复导入**: 在 `status.js` 中添加缺失的导入
2. **验证功能**: 通过 `--test-brain-api` 和 `--scan-embedded` 参数验证功能正常
3. **DoD 验证**: 所有 6 个验收条件全部通过
