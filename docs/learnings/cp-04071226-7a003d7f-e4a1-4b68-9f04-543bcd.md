# Harness v3.1 Sprint Generate — Learning

## 概要

本次实现 Harness v3.1 的 4 个核心 Feature：Sprint Report Skill 部署、Contract GAN 防死循环、合同草案跨 worktree 持久化、v3.1 断链测试覆盖。

---

### 根本原因

1. **GAN 死循环风险**：Contract GAN 对抗没有上限保护，Proposer 和 Reviewer 可能反复循环无法收敛。
2. **草案跨 worktree 不可见**：Proposer 写的 contract-draft.md 只在当前 worktree 本地，Reviewer 在另一个 worktree 读不到最新版本。
3. **sprint_report 未入路由表**：skills-index.md 任务路由表缺少 `sprint_report → /sprint-report` 条目。
4. **测试覆盖缺口**：v3.1 新增的 Contract GAN 层（planner→propose→review 双向路由）没有对应测试。

---

### 下次预防

- [ ] 任何新增的 harness 任务类型，必须同时更新 `.agent-knowledge/skills-index.md` 路由表
- [ ] GAN 对抗类循环（propose/review）必须在实现时同步加 MAX_ROUNDS 防死循环保护
- [ ] 跨 worktree 文件共享场景，必须在 SKILL.md 里明确写 git push/pull 步骤
- [ ] 新增断链路由时，必须同步在 `harness-sprint-loop-v3.test.js` 补测试
- [ ] vitest binary 在主仓库根目录 `node_modules/.bin/vitest`，不在子包里
