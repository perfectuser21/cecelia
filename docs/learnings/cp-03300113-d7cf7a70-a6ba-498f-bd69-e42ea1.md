# Learning: pipeline-trace.sh 每 Stage 详细字段输出

branch: cp-03300113-d7cf7a70-a6ba-498f-bd69-e42ea1
date: 2026-03-30

### 根本原因

原 pipeline-trace.sh 每 Stage 仅输出单行摘要，缺少 seal 细节（planner/spec-review/generator/crg），无法快速定位 pipeline 异常位置和责任人。

### 解决方案

每 Stage 改为 2-4 行输出：标题行保持不变，追加缩进详情行展示 seal 字段。新增 `_json_array_field()` 函数处理 `files_modified` 数组；使用 `awk` 提取 Learning RCA 首行；用 `grep -oE` 从 PR URL 提取 `#NNN`。

- [ ] 添加新 seal 类型时同步更新测试 fixture
