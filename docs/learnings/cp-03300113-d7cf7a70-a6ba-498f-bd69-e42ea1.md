# Learning: pipeline-trace.sh 每 Stage 详细字段输出

branch: cp-03300113-d7cf7a70-a6ba-498f-bd69-e42ea1
date: 2026-03-30

### 根本原因

原 pipeline-trace.sh 每 Stage 仅输出单行摘要，缺少 seal 细节（planner/spec-review/generator/crg）。
这导致在排查 pipeline 异常时无法快速定位到具体责任环节（是哪个 seal 失败、是谁审查的）。
需要扩展为 2-4 行输出：标题行保留现有格式，追加缩进详情行展示各 seal 字段和关键状态。
