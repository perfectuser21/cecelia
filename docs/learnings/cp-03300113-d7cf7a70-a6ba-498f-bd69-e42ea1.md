## 变更摘要

将 pipeline-trace.sh 每个 Stage 从单行摘要升级为 2-4 行详细字段输出，涵盖 Stage 0 brain/confidence 信息、Stage 1 planner+spec seal 详情、Stage 2 generator+CRG seal 详情、Stage 3 PR number 提取、Stage 4 RCA 首行摘录。

### 根本原因

DoD Test 字段格式兼容性问题：check-dod-mapping.cjs 要求 Test: 路径以 `tests/`、`contract:` 或 `manual:` 开头，而 packages/engine/tests/ 路径不符合该规则，导致映射检查失败。

### 下次预防

- [ ] packages/engine/ 子树开发时，BEHAVIOR 类 DoD 条目优先使用 `manual:node -e "readFileSync().includes()"` 静态内容检查，避免路径格式不兼容问题
- [ ] [GATE] 类条目同样需要包含 `process.exit` 等断言逻辑，否则 check-dod-mapping.cjs 会报"无断言逻辑"错误
- [ ] tdd_red_confirmed: true 必须在写代码前加入 .dev-mode，否则 bash-guard.sh 会阻止 step_2_code 标记为 done
