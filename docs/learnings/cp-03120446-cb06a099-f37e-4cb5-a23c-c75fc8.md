# Learning: migration 152 — DEFINITION.md schema_version 必须与 selfcheck.js 同步

## 背景

PR #901 为 `learnings` 表新增 `quality_score` + `source_type` 字段（migration 152）。selfcheck.js 的 EXPECTED_SCHEMA_VERSION 从 '151' 更新到 '152'，但遗漏了 DEFINITION.md 两处同步，导致 L2 Consistency Gate 失败。

## 核心陷阱：DEFINITION.md 有两处 schema_version 需要同步

### 问题

`facts-check.mjs` 校验 `selfcheck.js` 中的 `EXPECTED_SCHEMA_VERSION` 与 `DEFINITION.md` 中的版本号是否一致。

`DEFINITION.md` 有**两处**记录 schema_version：
1. 第 386 行表格：`| **schema_version** | 迁移版本追踪 | Schema 版本: 151 |`
2. 第 676 行描述：`5. **Schema 版本** — DB 版本 >= '151'（>= 检查，向前兼容）`

只更新 selfcheck.js 而不更新 DEFINITION.md → L2 facts-check 失败：
```
✗ schema_version: code=152 (selfcheck.js:18) ≠ doc=151 (DEFINITION.md)
```

### 修复

每次更新 EXPECTED_SCHEMA_VERSION 时，必须同时修改 DEFINITION.md 两处：
```bash
grep -n "schema_version\|'151'" DEFINITION.md  # 先找到所有位置
```

## CI-safe 单测路径算法

从 `packages/brain/src/__tests__/` 向上解析路径时：
- `../` → `packages/brain/src/`
- `../../` → `packages/brain/`
- `../../migrations/` → `packages/brain/migrations/` ✓（正确）
- `../../../migrations/` → `packages/migrations/` ✗（错误，目录不存在）

**规律**：`__tests__` 在 `src/` 下，migrations 在 `packages/brain/` 下，应该用 `../../migrations/`。

## 应用规则

- 修改 `EXPECTED_SCHEMA_VERSION` 时，搜索 DEFINITION.md 中所有包含旧版本号的行并全部更新
- 新增 migration 时，同时更新 selfcheck.js + DEFINITION.md（两处）
- CI-safe 测试用 `fs.readFileSync` + regex，路径层级要仔细数
