# Learning: Engine CI 守护规则强化 (2026-03-17)

## Engine devgate 脚本开发经验

### 根本原因

本次开发遇到两个关键问题：

**问题1：文件编辑路径错误（误改主仓库而非 worktree）**
将 check-rci-health.mjs 和 check-coverage-completeness.mjs 创建在主仓库路径
`/Users/administrator/perfect21/cecelia/packages/engine/...` 而非 worktree 路径。
这是因为 Bash 的 CWD 被重置后，没有注意到 Edit 工具使用了主仓库绝对路径。
直到 branch-protect hook 阻止 rci-execution-gate.sh 的编辑时才发现。

**问题2：JSDoc 注释中含 `/**` 通配符导致 Node.js ESM 解析错误**
在 .mjs 文件的注释块中写了 `/**/*.test.ts` 这样的 glob 路径，
Node.js 的 ESM 解析器将 `*/` 误解为 JSDoc 注释块结束符，导致 SyntaxError。

### 下次预防

- [ ] 编辑文件前先确认路径前缀是 worktree 路径（`.claude/worktrees/`）而非主仓库路径
- [ ] ESM(.mjs) 文件的 JSDoc 注释中避免出现 `**/*.xxx` glob 格式，改用描述性文字替代
- [ ] 检测到 hook 阻止时，首先检查是否误操作了主仓库文件，再检查 PRD/dev-mode 缺失
- [ ] 开发后期出现"另一个 agent 已完成"情况：检查分支 git log 判断是否重复工作，合并剩余改动到已有 commit 即可
