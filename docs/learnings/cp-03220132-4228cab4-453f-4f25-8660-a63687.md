# Learning: 修复 ComplexityScanner 函数边界检测

**Branch**: cp-03220132-4228cab4-453f-4f25-8660-a63687
**Date**: 2026-03-22

### 根本原因

`ComplexityScanner.analyzeComplexity()` 使用固定 2000 字符截取（`content.slice(braceStart, braceStart + 2000)`）来提取函数体，导致相邻函数的分支逻辑被纳入当前函数的复杂度计算中。

`mapStatus`（实际 CC=2）被误报为 CC=26，因为截取窗口包含了其后 24 个函数的代码。

### 下次预防

- [ ] 任何 AST/静态分析代码涉及"函数体提取"时，必须用括号配对（brace counting）算法，禁止固定字符截取
- [ ] ComplexityScanner 新增测试：验证相邻函数互不影响
- [ ] Brain 任务生成器产出任务前，应先在本地运行一次扫描确认 CC 值，避免误报触发不必要的重构任务
