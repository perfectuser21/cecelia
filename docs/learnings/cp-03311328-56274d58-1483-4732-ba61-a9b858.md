创建于 2026-03-23 的 `4a2c8a08`（ZenithJoy 自媒体，有 4 个活跃 KR）排在第 6 位，被截断不可见。

KR 不是不存在，而是根本不被 API 返回，导致所有下游决策都看不到内容 KR 进度。

此外 migration 208 修复空表问题后，内容生成和自动发布两个 KR verifier 意外使用了完全相同的 SQL，导致语义不同的指标无法区分，一旦其中一个出问题两个同时出问题。
