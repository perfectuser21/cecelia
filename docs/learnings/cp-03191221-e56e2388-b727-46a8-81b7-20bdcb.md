# Learning: N8N 三平台统一 Webhook 路由升级

**Branch**: cp-03191221-e56e2388-b727-46a8-81b7-20bdcb
**Date**: 2026-03-19
**Task**: 升级 N8N content-publish webhook，支持抖音/快手/小红书三平台 + 图文/视频两类型统一路由

### 根本原因

N8N `flow-内容发布.json` 原只处理抖音和小红书图文，快手无 N8N 接入，视频类型无路由。
随着快手视频脚本（PR #1116）上线，需要统一入口管理三平台五路由。

### 技术决策

1. **responseMode 改为 responseNode**：原 `onReceived` 无法返回 400 错误，改为 `responseNode` 后可插入 `Respond to Webhook` 节点精确控制 HTTP 状态码
2. **向后兼容 targetPlatforms**：原 API 用 `targetPlatforms` 数组，新 API 用 `platform` 单值，准备节点同时支持两种格式避免破坏现有调用方
3. **共享解析通知节点**：5 条 SSH 路由共用同一 `解析通知` Code 节点，从 `$('准备').first().json` 读取平台和类型信息，避免 5 份重复代码
4. **Switch 节点 5 条规则**：douyin 单条规则（任意 contentType），kuaishou+image / kuaishou+video / xiaohongshu+image / xiaohongshu+video 各一条，fallbackOutput=0 兜底
5. **SSH 目标统一为 CN Mac mini（100.108.7.63）**：抖音分支改为 SSH 至 Mac mini 触发 scheduler.sh 转发 Windows PC，与快手/小红书保持同一 SSH 入口

### 下次预防

- [ ] N8N workflow JSON 涉及 `responseMode` 变更时，同步检查 caller 是否期望同步响应——`onReceived` 立即返回 200，`responseNode` 需等待节点执行完成
- [ ] 多平台路由扩展时，优先用 Switch 节点的 `outputIndex` 而非多个 IF 节点，减少节点数量
- [ ] feat 类型 PR 必须附带测试文件，N8N workflow 改动可用 `node --test` 验证 JSON 结构
