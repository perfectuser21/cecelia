# Learning: zenithjoy OKR 种子数据 + L4 brew PATH 修复

## 做了什么

1. 新增 migration 152，插入 ZenithJoy 2026 Q1 OKR (vision + 3 KR)，修复 zenithjoy 部门派发冻结
2. 修复 ci-l4-runtime.yml 的 `brew: command not found` 问题（self-hosted runner PATH 问题）

## 关键发现

### zenithjoy dispatch 冻结根因

- `lookupDeptPrimaryGoal(pool, 'zenithjoy')` 查询 `metadata->>'dept' = 'zenithjoy'`
- 数据库中无此记录 → 返回 null → heartbeat 任务 `goal_id=null`
- planner 无法为 zenithjoy 生成实质工作任务
- **修复**：migration SQL 插入 4 个 goals（1 vision + 3 KR），metadata 带 `dept: "zenithjoy"`

### CI L4 brew PATH 陷阱

- macOS self-hosted runner 不自动载入 `/opt/homebrew/bin` 到 PATH
- `brew: command not found` 导致 L4 所有 shard exit 127
- **修复**：在 checkout 后添加 "Set up Homebrew PATH" 步骤，用 `$GITHUB_PATH` 注入路径
- 注意：此问题已在 main 分支持续失败（预存在），非本 PR 引入

### 版本 bump 6 个文件（Brain 改动必须同步）

修改 Brain 版本时需同步更新的文件：
1. `packages/brain/package.json` — `"version"`
2. `packages/brain/package-lock.json` — 顶层 + packages 两处
3. `packages/brain/VERSION` — 纯文本
4. `.brain-versions` — 根目录，check-version-sync.sh 检查
5. `DEFINITION.md` — `**Brain 版本**:` 和 `| **schema_version** |` 两处
6. `package-lock.json`（根）— packages/brain 条目

### CI Config Audit

修改 `.github/workflows/*.yml` 时，PR title 必须包含 `[CONFIG]`，否则 L1 CI Config Audit 失败。

## 根本原因

zenithjoy 部门 goals 表从未播种 `metadata->>'dept' = 'zenithjoy'` 的记录，导致 `lookupDeptPrimaryGoal` 始终返回 null，heartbeat 任务无法正确绑定 goal，dept 调度链断裂。CI L4 brew PATH 问题则是 self-hosted macOS runner 的 shell 环境初始化不完整，`/opt/homebrew/bin` 未自动加入 PATH。
