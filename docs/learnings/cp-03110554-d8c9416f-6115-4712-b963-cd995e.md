# Learning: [配额轮转 PR1] quota_exhausted 状态基础设施

**Branch**: cp-03110554-d8c9416f-6115-4712-b963-cd995e  
**Date**: 2026-03-11

## 根本原因

配额耗尽的任务当前以 `failed` 状态落库，导致：
1. `handleTaskFailure` 递增 `failure_count`
2. 达到 `FAILURE_THRESHOLD=3` 后触发自动隔离（quarantine）
3. 污染失败率指标，引发误报警觉

## 解决方案

新增 `quota_exhausted` 终态：
- **migration 144**：为 `tasks.status` 添加文档注释，记录枚举值
- **selfcheck.js**：`EXPECTED_SCHEMA_VERSION` 从 143 → 144
- **quarantine.js** `shouldQuarantineOnFailure`：`status === 'quota_exhausted'` 时返回 `shouldQuarantine: false`
- **quarantine.js** `handleTaskFailure`：早返回 `{ quarantined: false, skipped: 'quota_exhausted' }`，不执行 UPDATE
- **planner.js** 去重查询：`NOT IN` 列表加入 `'quota_exhausted'`，防止重复创建同类任务形成僵尸循环
