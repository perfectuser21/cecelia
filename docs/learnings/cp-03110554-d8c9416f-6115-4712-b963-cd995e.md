# Learning: [配额轮转 PR1] quota_exhausted 状态

### 根本原因

- `tasks.status` 是 `VARCHAR(50)`，无现有 CHECK 枚举约束，直接添加索引即可记录新状态语义
- `quarantine.js` 中 `shouldQuarantineOnFailure` 和 `handleTaskFailure` 是 failure_count 累计的两个关键路径，均需加守卫
- `planner.js` 的 initiative 去重查询必须同步排除 `quota_exhausted`，否则配额耗尽任务仍会阻止新规划任务创建
