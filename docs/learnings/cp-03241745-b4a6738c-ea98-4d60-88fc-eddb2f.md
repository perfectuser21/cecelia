# Learning: CI Scope 修复 — scripts/ci 进 routing-map + workspace 加 L4

Branch: `cp-03241745-b4a6738c-ea98-4d60-88fc-eddb2f`
Date: 2026-03-25

### 根本原因

1. **CI 盲区**：`scripts/` 和 `ci/` 目录变更不触发任何 CI 层，是因为 `ci/routing-map.yml` 只注册了 `packages/*` 和 `apps/*` 下的子目录，基础设施目录未被覆盖。

2. **L4 缺失**：`workspace`（apps/api + apps/dashboard）的 `layers` 配置只有 `[l1, l3]`，缺少 L4 集成测试层，导致 Workspace 的 Brain proxy 集成从未被验证。

3. **Gate 0c 误报**：新增的 Check 4 代码使用了 `console.log()`，触发了 Gate 0c 调试代码检测。应统一使用 `process.stdout.write()` 模式与文件其他部分保持一致。

4. **TypeScript 严格类型**：`response.json()` 在 TS 严格模式下返回 `unknown`，直接使用 `data[0]` 访问属性会导致类型错误 `TS18046`。需要显式类型断言。

### 下次预防

- [ ] 新增基础设施目录（非 packages/ 非 apps/）时，必须同步在 `ci/routing-map.yml` 注册
- [ ] 新增 CI 脚本中的输出代码时，统一使用 `process.stdout.write()` 而非 `console.log()`（Gate 0c 会拦截后者）
- [ ] 写集成测试时，`response.json()` 必须加类型断言 `as Array<Record<string, unknown>>` 或具体类型，避免 TS18046
- [ ] PR title 含 `.github/workflows/**` 变更时，必须加 `[CONFIG]` 标签，否则 L1 CI Config Audit 失败
- [ ] Learning 文件必须在 push 前写好（含 `### 根本原因` + `### 下次预防` + `- [ ]` checklist），不能事后补
