# Learning: brain-new-files-check 只检查 brain.md，未覆盖 DEFINITION.md

### 根本原因

`brain-new-files-check` 独立 job（PR #1649）只检查 `.agent-knowledge/brain.md`，
但 `brain-l2` 内嵌步骤（同一 CI 文件）已支持 `brain.md` **或** `DEFINITION.md` 两选一。
同一文件中两处同语义检查使用不同标准，导致用 `DEFINITION.md` 更新时通过 `brain-l2` 但被独立 job 拒绝。

### 下次预防

- [ ] `brain-new-files-check` 与 `brain-l2` 内嵌步骤应保持同步，任一改动时同步更新另一处
- [ ] PRD 说"接受 X 或 Y"时，所有相关 CI 检查均需实现"或"逻辑，不能遗漏
- [ ] Task Card `## 成功标准` 改用有序列表而非 checklist，避免 check-dod-mapping 误判为 DoD 条目
