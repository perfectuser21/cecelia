# Learning: sprint_contract_propose verdict 纯文本提取失败

**Branch**: cp-04080701-5050a0fd-45a1-4fa9-98d1-43f3ec
**Date**: 2026-04-08

### 根本原因

`execution.js` 的 `sprint_contract_propose` 处理块只调用 `extractVerdictFromResult(result, ['PROPOSED'])`，该函数只能识别 JSON 格式的 `"verdict": "PROPOSED"` 文本。但 cecelia-run agent 有时输出纯中文描述（如"合同草案第 1 轮已提交...等待 Evaluator"），没有任何 JSON verdict 格式，导致 `proposeVerdict = null` → GAN 守卫拦截 → Reviewer 永远不被创建。

实证案例 `884b6db0`：result.result = "合同草案第 1 轮已提交：...等待 Evaluator (sprint-contract-reviewer) 开始挑战。" — 无 PROPOSED 关键字，无 JSON verdict。

### 下次预防

- [ ] harness propose/review 类任务的 verdict 提取必须有多层 fallback：
  1. JSON 格式 `"verdict": "..."` 正则（现有 extractVerdictFromResult）
  2. 纯文本关键字 `/\bPROPOSED\b/i` fallback
  3. 最终兜底：成功完成（newStatus=completed）时默认 PROPOSED
- [ ] sprint_contract_review 已有 `/\bAPPROVED\b/` fallback + 默认 REVISION，模式可参照
- [ ] 新增 harness 类型时，考虑加 "成功完成时默认 verdict" 的兜底逻辑
