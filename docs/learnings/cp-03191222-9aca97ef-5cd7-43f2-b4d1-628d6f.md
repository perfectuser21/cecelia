## 关键决策

**N8N 输出解析策略**：发布脚本通过 `process.stdout.write('\n' + JSON.stringify({...}) + '\n')` 输出标准 JSON，N8N Code 节点用正则 `/\{[\s\S]*\}/` 从 SSH 输出中提取最后一个 JSON 对象。这比在脚本里写入临时文件更简洁，N8N 无需额外文件读取步骤。

**Brain API 路由注册方式**：`publish-results.js` 使用 ESM `import { Router } from 'express'`，在 `routes.js` 的 `subRouter` 数组末尾追加，路径为 `/api/brain/publish-results`。

**抖音不修改**：抖音脚本在 zenithjoy repo 中，SSH 节点仍指向旧 VPS（已失效），这是 PR3 的职责范围，本 PR 不触碰。

## 遇到的坑

### 根本原因

**git stash 意外执行**：在诊断 branch-protect hook 失败时，误执行了 `git stash`。由于 worktree 的主分支（main）和当前分支（cp-...）的文件集不同，stash pop/restore 后导致约 450 个 `.dod-*`/`.prd-*` 文件出现 ` D`（未暂存删除）状态，污染了 working tree。

**引擎测试预存在失败**：`tests/dev/generate-feedback-report.test.ts` 中一个测试期望输出 `'323s'` 但实际得到 `'0s'`（无执行记录），该测试在隔离运行时通过，但在 CI 全套测试中失败——这是上游预存在问题，与本 PR 无关。

### 下次预防

- [ ] 在 worktree 中**永远不要手动 `git stash`**，使用 `git status` 查看状态即可，不需要 stash
- [ ] 如果遇到 stash 污染，`git checkout -- .` 可以快速恢复工作树到 HEAD 状态
- [ ] Branch-protect hook 检测 PRD/DoD 时：worktree 根目录 + `packages/workflows/` 两处都需要对应的 per-branch 文件（就近检测原则）
- [ ] N8N JSON 文件较大（~200KB），Write 前必须先 Read；若写入后 linter 自动格式化，需再次 Read 才能二次 Write
- [ ] 发布脚本 JSON 输出应用 `process.stdout.write` 而非 `console.log`，后者可能被其他日志淹没，难以从 N8N SSH 输出中提取
