# Learning: Brain API 图片扫描前缀匹配

## 背景
PR #1718 — fix(brain): [CONFIG] 图片扫描同时匹配 topic 和无连字符前缀

### 根本原因
`topic` 由 keyword 转换时保留连字符（`dan-koe` → `dan-koe`），但实际图片文件名可能使用无连字符形式（`dankoe-cover.png`）。
image-gen 工作流生成图片时，使用 slug 化的 topic 作为前缀，不同工具对连字符的处理不一致：有的保留连字符，有的去除。
filter 只检查一种前缀（带连字符）导致去掉连字符的图片文件全部漏匹配，`/output` 端点返回空 image_urls。
