# Learning: KR3-T1 内容模板梳理

**分支**: cp-03220805-c158aaef-a8ec-44d8-95ff-ed01a3
**日期**: 2026-03-22

## 背景

新增 5 个内容类型 YAML 模板（long-form/short-form/video-script/promo-copy/data-analysis），扩展 Cecelia 内容工厂支持的内容类型体系。

### 根本原因

content-types 目录原仅有 `solo-company-case.yaml` 一个示例模板，缺乏覆盖常见内容场景的完整类型定义，导致内容 Pipeline 只能处理单一内容类型。

### 下次预防

- [ ] 新增内容类型前先查阅 `content-type-validator.js` 的 REQUIRED_CHECKS，确保必填字段完整
- [ ] `content_type` 字段必须与文件名完全一致（registry 启动时会校验）
- [ ] 每个类型的 `template.generate_prompt` 需指定明确的产出物数量和格式，避免 Pipeline 执行器无法解析
- [ ] 新模板应在本地用 `js-yaml` 解析验证后再提交，防止 YAML 语法错误

## 技术细节

- `content-type-registry.js` 使用 ESM 模块，DoD 测试中不能用 `require()` 直接导入，需用 CommonJS 的 `js-yaml` 独立校验
- Pipeline 四阶段（research → generate → review → export）的提示词分别对应 `template.research_prompt`、`template.generate_prompt`、`template.review_prompt`
- `images.count` 根据内容类型差异显著（数据分析 8 张图 vs 视频脚本 1 张封面）
