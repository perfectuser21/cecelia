## [SelfDrive] 任务成功率诊断 — 失败模式分类方法（2026-04-10）

### 根本原因

1. **静默失败是最危险的模式**：83 例 content-pipeline/research 任务因 Orchestrator 未捕获异常而静默取消，`error_message` 为空，导致 Brain 无法识别、无法重试、无法告警。

2. **Quota 耗尽形成正反馈循环**：Quota 不足 → 触发大量凭据告警任务 → 告警任务本身也因 quota 耗尽而 quarantine → 问题无法自愈。

3. **级联取消不可逆**：父 pipeline 失败时子任务直接 cancel（而非 suspend），恢复父任务后无法重新激活子任务，损失被放大。

### 下次预防

- [ ] Pipeline Orchestrator 的所有 catch 块**必须**写入 `tasks.error_message`，error_message 为空的取消应视为告警信号
- [ ] Brain Tick 调度前检查 quota 余量，quota < 10% 时停止派发非关键任务
- [ ] 子任务级联失败改为 `suspended` 状态（可恢复），而非不可逆的 `cancelled`
- [ ] 任务成功率监控：按 task_type 分类，连续 5 分钟取消率 > 80% 自动触发 P1 告警
- [ ] 诊断类任务（SelfDrive）在分析阶段应直接查 DB，不依赖 Brain API 统计（统计口径可能不一致）
