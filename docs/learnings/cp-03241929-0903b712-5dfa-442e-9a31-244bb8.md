# Learning: fix suggestion-dispatcher 绕过丘脑直接建任务

**Branch**: cp-03241929-0903b712-5dfa-442e-9a31-244bb8
**Date**: 2026-03-25

### 根本原因

`suggestion-dispatcher.js` 历史实现直接 `INSERT INTO tasks`，绕过丘脑（Thalamus）统一任务创建权。
这是「任务创建权统一收归丘脑」架构规则的违反点，也是 Rumination→Desire 自循环闭环的最后断口。

### 修复方案

1. `thalamus.js` 新增 `EVENT_TYPES.SUGGESTION_READY` 和 `handleSuggestionReady()` 处理器
   - 在 `processEvent` 入口最前面专门处理此事件（绕过 quickRoute/L1 LLM）
   - 内含去重（`payload->>'suggestion_id'` in_flight 检查）和速率限制（队列积压 >= 5 暂缓）
   - 事务安全：创建任务 + 更新 suggestion.status 在同一事务中

2. `suggestion-dispatcher.js` 移除直接 INSERT，改用 `processEvent(SUGGESTION_READY)`
   - 通过 `_suggestion_dispatched` 标志判断丘脑是否实际创建了任务
   - 保留 Layer 层级描述字符串（thalamus-scope-layer.test.js 依赖）
