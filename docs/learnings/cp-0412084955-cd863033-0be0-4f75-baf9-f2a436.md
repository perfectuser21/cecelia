# Learning: Health 端点新增 evaluator_stats 字段

**任务**: harness_generate — Workstream 1/1
**分支**: cp-0412084955-cd863033-0be0-4f75-baf9-f2a436
**日期**: 2026-04-12

### 根本原因

Health 端点缺少 `evaluator_stats` 聚合字段，导致无法从单一 API 获取 Harness Evaluator 的历史运行统计。

### 实现方式

在 `packages/brain/src/routes/goals.js` 的 `/health` 路由 `Promise.all` 中新增一条 SQL 聚合查询：

```sql
SELECT
  COALESCE(COUNT(CASE WHEN status = 'completed' THEN 1 END)::integer, 0) AS passed,
  COALESCE(COUNT(CASE WHEN status IN ('canceled', 'failed') THEN 1 END)::integer, 0) AS failed,
  MAX(completed_at) AS last_run_at
FROM tasks
WHERE task_type = 'harness_evaluate'
  AND status IN ('completed', 'canceled', 'failed')
```

- 单条 SQL 一次获取 passed/failed/last_run_at，避免多次 DB 往返
- `.catch(() => null)` 实现降级容错：查询失败时不影响其余字段
- 应用层将 `passed + failed` 计算为 `total_runs`，无终态记录时返回零值对象

### 下次预防

- [ ] 新增聚合字段时优先考虑 `Promise.all` 并行执行，不阻塞现有查询
- [ ] SQL 聚合使用 `COALESCE` 保证空表场景返回 0 而非 null
- [ ] 查询包裹 `.catch(() => null)` 实现降级，应用层统一处理 null 情况
