# Learning: War Room 横向多列布局

## 任务
feat(dashboard): War Room 改为横向多列 dashboard 布局

## 根本原因

DoD 中引用测试文件使用了错误路径格式 `tests/apps/api/features/gtd/__tests__/warroom.test.ts`。
CI 的 `check-dod-mapping.cjs` 从项目根查找 `path.join(projectRoot, testRef)`，`tests/` 前缀要求文件在 `packages/brain/tests/` 或 `packages/engine/tests/` 等专属目录下。
但 `apps/api/` 下的测试文件路径不以 `tests/` 开头，需要改用 `manual:node -e "require('fs').accessSync(...)"` 进行文件存在性验证。

## 下次预防

- [ ] DoD 引用现有测试文件前，先用 `ls <path>` 确认路径存在
- [ ] `apps/` 路径下的测试文件必须改用 `manual:node -e "require('fs').accessSync(...)"` 格式，不要用 `tests/apps/...`
- [ ] `tests/` 前缀仅适用于 Brain/Engine 专属测试目录（`packages/brain/tests/`、`packages/engine/tests/`）
