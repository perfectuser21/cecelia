# Learning: War Room 横向多列布局

## 任务
feat(dashboard): War Room 改为横向多列 dashboard 布局

## 根本原因

DoD 中 `Test: tests/apps/api/features/gtd/__tests__/warroom.test.ts` 路径错误——CI 的 `check-dod-mapping.cjs` 从项目根目录查找 `tests/<path>`，但实际测试文件在 `apps/api/features/gtd/__tests__/warroom.test.ts`，`tests/` 前缀下不存在该文件，导致 DoD Verification Gate 失败。

## 下次预防

- [ ] DoD 中引用现有测试文件时，先用 `ls <path>` 确认路径存在
- [ ] `apps/` 路径下的测试文件必须改用 `manual:node -e "require('fs').accessSync(...)"` 格式，不要用 `tests/apps/...`
- [ ] `tests/` 前缀仅适用于 Brain/Engine 专属测试目录（`packages/brain/tests/`、`packages/engine/tests/`）
