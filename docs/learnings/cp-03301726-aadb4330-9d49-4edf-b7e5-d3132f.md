# Learning: content-export 完成后自动上传 NAS + 写 export_path

**分支**: cp-03301726-aadb4330-9d49-4edf-b7e5-d3132f  
**日期**: 2026-03-31

---

### 根本原因

`executeExport()` 执行完成后，仅在本地写入 manifest.json 和卡片文件，没有将内容同步到 NAS，`pipeline.payload.export_path` 字段始终为 null。

`_executeStageTask` 未对 executor 返回值中的 `export_path` 做 jsonb merge 写回 task.payload，导致 orchestrator 后续读不到路径。

`_handleExportComplete` 标记 pipeline completed 时也未从 task.payload 读取并写入 pipeline.payload.export_path，前端 GET /output 路由返回的 export_path 始终为 null，无路径可展示。

### 下次预防

- [ ] 新增文件写入操作时，检查是否需要同步到 NAS（内容归档场景）
- [ ] executor 返回值中的路径字段（如 export_path）需在 `_executeStageTask` 中显式 merge 到 task.payload，否则 orchestrator 读不到
- [ ] NAS 连接常量（USER/IP/BASE）应放在模块顶部 const，读取 `process.env` 并保留合理默认值，禁止硬编码在函数体内
- [ ] NAS 路径目录名使用 `pipelineId`（UUID），不要用 keyword slug（同一 keyword 多次 pipeline 会覆盖）
- [ ] code_review_gate 会检查凭据硬编码，提交前先过一遍安全维度

### 关键实现

```
executeExport() 返回 export_path
  ↓
_executeStageTask 检测 execResult.export_path → jsonb merge 写入 task.payload
  ↓
_loadPipelineContext 重新加载 task（已含最新 payload）
  ↓
_handleExportComplete 读 task.payload.export_path → jsonb merge 写入 pipeline.payload
  ↓
GET /output 路由读 pipeline.payload.export_path → 返回给前端
```
