## KR3 微信小程序关键 bug 修复（2026-04-08）

### 根本原因

1. **页面未注册**：zenithjoy-miniapp 在迁移时（Codex 操作）只保留了 5 个 tabBar/基础页面在 app.json，其余 5 个已实现页面目录从未注册，导致用户实际上无法进入这些页面。
2. **硬编码 API Key**：ai-features 页面在初始化阶段自动写入明文 API Key 到 storage，属于不应出现在前端代码的凭据泄露。
3. **openid 字面字符串**：微信云数据库服务端 `.where({ _openid: '{openid}' })` 的占位语法被直接用在了前端小程序代码中（chatHistory、membership），但前端只能依赖云数据库权限过滤，`'{openid}'` 在客户端是无效字符串查询。

### 下次预防
- [ ] 新增页面目录后立即在 app.json 中注册（作为 PR checklist 项）
- [ ] CI/pre-push 运行 `bash scripts/check-miniapp-health.sh` 验证页面注册完整性
- [ ] 前端禁止出现 `pat_` 或 `sk-` 开头字符串（可加 grep 扫描 hook）
- [ ] 云数据库查询在前端只用 `.get()`，openid 过滤交给服务端权限控制
