# fix(brain): kr-progress fallback 链路从未写入 progress（2026-04-08）

### 根本原因

`kr-progress.js` 中 `updateKrProgress` 函数存在两个 bug，导致 fallback 进度采集链路完全失效：

**Bug 1**（主因）：UPDATE SQL 写入了错误的表，且缺少字段赋值：
```sql
-- 错误：
UPDATE objectives SET updated_at = NOW() WHERE id = $1
-- 正确：
UPDATE key_results SET progress = $2, updated_at = NOW() WHERE id = $1
```
函数计算了 `progress` 但从未持久化到数据库。

**Bug 2**（次因）：`syncAllKrProgress` 查 `objectives UNION ALL key_results`，但 `okr_projects.kr_id` 只引用 `key_results.id`，传入 objectives ID 永远查不到 projects，fallback 对 objectives 永远跳过。

### 下次预防

- [ ] 写 UPDATE SQL 时：表名要对齐 FK 链路（`okr_projects.kr_id → key_results.id`），且写 UPDATE 同时确认有目标字段赋值（不能只更新 `updated_at`）
- [ ] `UNION ALL` 查询跨表时：确认各表的字段语义与下游 JOIN 条件一致，避免"查了但用不上"
- [ ] 单元测试 mock 要验证 UPDATE 参数（`expect(call[1]).toEqual([id, value])`），而不只验证 UPDATE 是否被调用
- [ ] 误入 vitest exclude 的纯 mock 单元测试要及时从 exclude 列表移出
