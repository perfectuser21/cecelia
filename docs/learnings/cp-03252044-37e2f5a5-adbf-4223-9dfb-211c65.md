# Learning: GTD 组件空数据 — apps/api task-system 表不存在时的代理模式

## 根本原因

`apps/api/src/task-system/` 中的 goals.js、projects.js 查询本地 cecelia 数据库中不存在的表（`goals`、`projects`）。
areas.js 查询 `ORDER BY sort_order` 但该列不存在，routes.js 缺少 `/areas` 路由注册（GTDArea/GTDProjects 组件直接调用 `/api/tasks/areas` 但无对应路由）。
真实数据存放在 Brain DB（port 5221）中：goals 在 `/api/brain/goals`，projects 在 `/api/brain/projects`。apps/api 应代理 Brain 而非本地查询不存在的表。

## 下次预防

- [ ] 新增 GTD 组件或 task-system 路由时，优先验证本地 cecelia DB 是否有对应表：`psql -c "\dt"` 确认
- [ ] 如果数据在 Brain DB，使用 `fetch(process.env.BRAIN_API || 'http://localhost:5221')/api/brain/xxx` 代理模式
- [ ] Brain 返回的字段名可能与前端期望不一致（如 `title` vs `name`，`area_kr` vs `kr`），写代理时检查前端组件的 interface 定义
- [ ] DoD 中 BEHAVIOR 测试不能用 `fetch('http://localhost:5211/...')` — CI 无运行中的服务器，改用文件内容检查：`node -e "const c=require('fs').readFileSync(...);if(!c.includes('...'))process.exit(1)"`
- [ ] DoD 中 `Test:` 字段必须加 `manual:` 前缀，否则 check-dod-mapping.cjs 报"缺少 Test 字段"
