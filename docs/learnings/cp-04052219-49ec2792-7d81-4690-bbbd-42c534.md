---

## 背景

实现护城河1：在 git push 前强制运行 TypeCheck + ESLint + unit test，60s 内完成，不通过不允许 push。

---

### 根本原因

没有本地 quickcheck 机制时，低质量代码可以直接 push → CI 失败 → 修复周期长（分钟级 CI vs 秒级本地）。

---

### 实现要点

1. **git hooks 在 worktree 中共享主仓库 hooks**：用 `git rev-parse --git-common-dir` 获取真正的 hooks 目录，而非 `git rev-parse --git-dir`（worktree 下两者不同）。

2. **node_modules 未安装时优雅跳过**：通过检查 `node_modules/.bin/tsc` 等文件是否存在决定是否运行对应检查，不报错、不阻塞。

3. **只检查变更包**：用 `git diff --name-only origin/main...HEAD` 检测变更包，避免全量检查超时。

4. **Engine unit test 只跑 parallel 组**：`vitest run --project=parallel` 仅跑无 shell 依赖的测试，避免 singleFork 串行测试慢。

5. **git push --no-verify 可跳过**：符合紧急情况需求。
