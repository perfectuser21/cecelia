# Learning: deploy webhook 非阻塞化

## 根本原因

Brain `POST /api/brain/deploy` 使用 `execSync` 同步执行 deploy 脚本，
导致 Node.js 事件循环在部署期间（最长 10 分钟）完全阻塞。
当部署进程遭遇 OOM kill（exit 137），Brain 进程本身也会被终止，deployState 丢失。

## 下次预防

- [ ] Node.js 服务端禁止用 `execSync` 执行长耗时命令，统一改用 `spawn`
- [ ] 新增 webhook 端点时检查是否有并发保护（running 时返回 409）
- [ ] 所有 `deployState` 字段修改需同步更新测试

## 关键决策

- `spawn` 替换 `execSync`：子进程独立运行，事件循环保持响应
- 并发保护在 auth 之后、状态更新之前检查（保证安全性优先）
- `output_tail` 限制 20 行防止内存无限增长
