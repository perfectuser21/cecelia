## 成功标准验证

### 根本原因
原 `integration.test.js` 仅有 `beforeAll` 建立数据，没有 `afterAll` 清理，每次运行都向 Workspace DB 写入测试 project/goal/task 且永不清除。累积后污染查询结果（如 `GET /api/tasks?status=queued` 返回历史测试垃圾数据）。

新测试文件设计时也发现 Brain API tasks 端点有两层：
- `GET /api/brain/tasks`（status.js，只读列表）
- `POST/PATCH /api/brain/tasks/tasks`（task-tasks.js，完整 CRUD）

Brain 任务无 DELETE 端点，cleanup 需要通过 `PATCH status=cancelled` 实现。
