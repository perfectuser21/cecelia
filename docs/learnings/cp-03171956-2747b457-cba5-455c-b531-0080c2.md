### 根本原因

migrate.js 使用 BEGIN/ROLLBACK 事务包裹每个 migration。当 xian-mac-m1 CI runner 的 pgdata 目录被同一 run 的重试复用时（`PG_DATA already exists, skipping initdb`），若上次运行中某 migration 部分 DDL 提交但 postgres 异常崩溃导致未能完整记录 schema_version，重试时该 migration 会再次执行。

具体不幂等操作：
1. `CREATE TABLE` 无 `IF NOT EXISTS` → "relation already exists"
2. `INSERT INTO schema_version` 无 `ON CONFLICT` → "duplicate key"
3. 缺少 `INSERT INTO schema_version` → runner 虽自动插入但版本追踪不自文档化

### 下次预防

- [ ] 所有新 migration 的 CREATE TABLE/INDEX 必须加 `IF NOT EXISTS`
- [ ] 所有新 migration 的 `INSERT INTO schema_version` 必须加 `ON CONFLICT (version) DO NOTHING`
- [ ] migration 文件中的版本号必须与文件名前缀一致（114 的 '113' 是审查遗漏）
- [ ] 新 migration PR review checklist 中加入幂等性检查项
