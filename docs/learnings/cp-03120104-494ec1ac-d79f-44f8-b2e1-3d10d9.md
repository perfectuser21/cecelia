# Learning - Cortex Insight → Action 自动闭合机制

**Branch**: cp-03120104-494ec1ac-d79f-44f8-b2e1-3d10d9
**PR**: #881

### 根本原因

cortex_insight 类型的 learning 写入 `learnings` 表后，`applied` 字段永远为 false，洞察积累但无任何行动闭合机制。self_model 连续 10 条重复洞察是观察者惰性的量化证据——insight 写入后没有自动触发修复任务的路径。

关键路径：`cortex.js` 的 `recordLearnings` 函数只做 INSERT，INSERT 成功后没有任何后续处理逻辑。`tasks.source_learning_id` 字段不存在，无法建立 insight→task 溯源关联。

修复：新增 `insight-to-task.js`（关键词检测 ≥2 命中 → 自动创建 dev task），在 `recordLearnings` INSERT 成功后 fire-and-forget 调用，migration 150 添加 `source_learning_id` 字段，task 创建成功后标记 `applied=true`。
