# Learning: cortex.js parseActionTags 实现

## 实现要点

### vitest vi.mock hoisting 与 const 暂时性死区（TDZ）

**问题**：`vi.mock('../db.js', () => ({ default: mockPool }))` 中引用了 `const mockPool = { query: vi.fn() }` —— `vi.mock` 被提升（hoisted）到文件顶部执行，此时 `mockPool` 尚在 TDZ，抛出 `ReferenceError: Cannot access 'mockPool' before initialization`。

**根本原因**：`vi.mock` 的 factory 函数在文件实际执行之前运行，`const` 声明在 TDZ 不可访问。

**修复方案（最简）**：在 factory 内直接 inline 创建对象，不引用外部变量：
```javascript
vi.mock('../db.js', () => ({ default: { query: vi.fn() } }));
```

**另一种方案**：使用 `vi.hoisted()` 将变量提升：
```javascript
const { mockPool } = vi.hoisted(() => ({ mockPool: { query: vi.fn() } }));
vi.mock('../db.js', () => ({ default: mockPool }));
```

**为何同项目其他测试文件可以用相同写法？** 可能与 vitest 版本或 transform 缓存有关，不应依赖此行为，新写测试时统一采用 inline factory 方案。

---

**Why**: vitest mock hoisting 是常见陷阱，影响所有使用 `vi.mock` + 外部变量的测试文件。
**How to apply**: 新建测试文件时，vi.mock factory 内不要引用文件顶层 `const` 变量。
