# Learning: cortex.js parseActionTags 实现

## vitest vi.mock hoisting 与 const 暂时性死区（2026-03-12）

### 根本原因

`vi.mock('../db.js', () => ({ default: mockPool }))` 中的 factory 函数被 vitest 提升（hoisted）到文件顶部执行，但 `const mockPool = { query: vi.fn() }` 此时还在暂时性死区（TDZ），导致 `ReferenceError: Cannot access 'mockPool' before initialization`。

`vi.mock` 的 factory 在模块 import 之前运行，`const` 声明在 TDZ 不可访问——这与同项目其他测试文件相同写法能通过，可能是 vitest transform 缓存行为不一致。

### 下次预防

- [ ] 新建 vitest 测试文件时，`vi.mock` factory 内不引用文件顶层 `const` 变量，改为 inline 创建对象：`vi.mock('../db.js', () => ({ default: { query: vi.fn() } }))`
- [ ] 需要在测试中访问 mock 对象时，改用 `vi.hoisted()`：`const { mockPool } = vi.hoisted(() => ({ mockPool: { query: vi.fn() } }))`
