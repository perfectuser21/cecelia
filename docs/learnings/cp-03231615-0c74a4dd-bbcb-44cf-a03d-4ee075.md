# Learning: P2P 对话查询能力——dispatch_query_task + execution-callback 飞书回调

Branch: cp-03231615-0c74a4dd-bbcb-44cf-a03d-4ee075
Date: 2026-03-23

### 根本原因

P2P 对话中 Cecelia 说"我去查"但不真正执行查询，有两个断链点：
1. `handleChat` 中 observeChat 调用未传入 `conversation_id`，导致 dispatch 的任务不知道回调给谁
2. `execution-callback` 路由未检查任务是否有 P2P 回调需求，任务完成后没有推送结果

### 修复策略

- 在 MOUTH_SYSTEM_PROMPT 新增 `dispatch_query_task` 类型，让 LLM 知道可以用这个信号派出真正的探查任务
- thalamus.js observeChat 新增 dispatch_query_task case：创建任务时在 payload.p2p_callback 存 open_id
- execution-callback 路由在 COMMIT 后、非阻塞地检查 p2p_callback 并调 sendFeishuToOpenId 回飞书
- ops.js fallback handleChat 路径补充 conversation_id: openId
