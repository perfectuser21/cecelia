# Learning: PostgreSQL NULL 比较导致成功率异常放大

**分支**: cp-04080139-123146c7-f5d8-499b-a34c-749b5a
**日期**: 2026-04-08

## 根本原因

`getTaskStats24h()` 中 `total` 字段的 SQL filter：

```sql
AND payload->>'failure_class' != 'auth'
```

在 PostgreSQL 中，`NULL != 'auth'` 返回 `NULL`（而非 `TRUE`）。
`completed` 任务的 `failure_class` 字段为 NULL，因此被排除出 `total` 计数。

结果：`completed=243, total=7` → 成功率 = 3471%（异常）。

## 修复

将 `!= 'auth'` 改为 `IS NULL OR != 'auth'`：

```sql
AND (payload->>'failure_class' IS NULL OR payload->>'failure_class' != 'auth')
```

修复后：`completed=243, total=255` → 成功率 = 95%（正常）。

## 下次预防

- [ ] 在任何 PostgreSQL `!=` 比较前，确认该字段是否可能为 NULL
- [ ] 对含 NULL 的字段用 `IS DISTINCT FROM` 或 `(IS NULL OR != value)` 替代 `!=`
- [ ] `failure_class` 字段仅在 failed/quarantined 任务中有值，completed 任务该字段为 NULL
