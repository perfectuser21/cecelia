# Learning: KR4 geo 网站进度卡在 25% — 根本原因与修复

### 根本原因

KR4 进度计算依赖 `okr_projects` 表的项目状态平均分：
- P1（geo网站部署）= `active`（50分）：网站实际已上线但状态未同步
- P2（SEO+内容首发）= `inactive`（0分）：未激活

两项平均 → 25/100 → progress_pct=25%。

根本问题是**KR verifier 数据依赖业务层手动维护**，而 agent 在完成工程交付后没有回写项目状态。Brain 也没有探针自动感知 geo 网站健康，无法驱动状态自动更新。

### 下次预防

- [ ] 完成工程交付（网站上线）后，立即回写对应 `okr_projects` 状态为 `completed`
- [ ] 新增外部产品探针时，同步检查 KR verifier 依赖项是否最新
- [ ] `okr_projects` 状态变更应由任务完成回调自动触发，不依赖人工记忆
- [ ] 外部 URL 类的产品（Cloudflare Pages / 小程序）需在 capability-probe 中注册健康探针

### 修复

- 新增 `probeGeoWebsite()` 函数（capability-probe.js），每小时检测 zenithjoyai.com/zh/ + blog + posts
- P1 okr_projects status: active → completed
- P2 okr_projects status: inactive → active
- KR4 进度：25% → 75%（下次 verifier 运行后生效）
