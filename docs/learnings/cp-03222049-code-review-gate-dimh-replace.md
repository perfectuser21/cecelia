2. **02-code.md 编辑规则第5条**：先删旧再写新，禁止堆叠 — 修改任何描述性内容时，必须同时删除被替代的旧内容

---

### 根本原因

AI 默认行为是**追加**，不是**替换**。改 A→B 时只加 B，旧的 A 描述保留。多轮迭代后同一文件 A、B、C 全在，全是过时信息，互相矛盾。

原有维度H只检测 warning 级别的"引用已删除路径"（被动发现），无法拦截"加新不删旧"这个核心问题。

### 下次预防

- [ ] 修改任何步骤说明/规则/文档段落时 → 先找到旧的对应描述，删除它，再写新的
- [ ] code-review-gate 维度H blocker 会检查：新增行描述新行为 → 旧描述是否有对应 `-` 删除行
- [ ] 典型易犯场景：changelog 加了新条目但正文旧说法还在；步骤说明更新了但原来的步骤没删

---

### 关键教训

**两种信息卫生问题要区分**：
1. **引用已删除内容**（warning）：引用了早就删掉的东西 → 被动发现
2. **加新不删旧**（blocker）：这次改动本身就没有删旧描述 → 在同一 PR 里就能检测

第2种问题更危险，因为它是每次改动都可能发生的，而不是偶发的引用错误。
