本地测试失败：0

## Use-After-Delete Bug 导致错误信息永远丢失（2026-03-17）

### 根本原因

`cecelia-run.sh` 第 427 行先 `rm -f "$wt_stderr_log"` 删除临时文件，第 431 行再 `cat "$wt_stderr_log"` 读取同一文件。文件已不存在，`cat` 失败，始终落入 `echo 'unknown error'` 分支。

导致至少 5 个 dev 任务（253e80d5/c74e6c32/7407c7d3/da7c4864/aa8cd5be）以 `worktree_creation_failed: unknown error` 失败，无法诊断真实原因（可能是磁盘空间不足、worktree 数量超限、flock 超时等）。

**代码位置**：`packages/brain/scripts/cecelia-run.sh` 第 422-435 行的 else 分支（worktree 创建失败路径）。

### 修复方式

先将 stderr 内容读入变量 `wt_stderr_content`，再删除临时文件，最终用变量写入 err_log：

```bash
# 修复前（use-after-delete）
cat "$wt_stderr_log" >&2
rm -f "$wt_stderr_log"
echo "worktree_creation_failed: $(cat "$wt_stderr_log" | head -3 || echo 'unknown error')" > "$err_log"

# 修复后
wt_stderr_content=$(cat "$wt_stderr_log" 2>/dev/null | head -10 || echo 'unknown error')
echo "$wt_stderr_content" >&2
rm -f "$wt_stderr_log"
echo "worktree_creation_failed: ${wt_stderr_content:-unknown error}" > "$err_log"
```
