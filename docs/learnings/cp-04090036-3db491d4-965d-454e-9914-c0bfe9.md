# Learning: KR3 冲刺编排 — 子任务 PRD 补全

## 根本原因
Brain 自动创建的 7 个 miniapp 子任务（由 kr3-sprint-planner 触发）description 字段为空，
只有 payload.description 有一行文字。夜间编排器可以调度任务，但 agent 拿到任务后
因缺乏足够上下文（无具体文件路径、无实现方案）而无法有效执行。

## 下次预防
- [ ] kr3-sprint-planner（或类似 sprint-planner 脚本）在创建任务时必须同时写入 `description` 字段（不只是 payload.description）
- [ ] 子任务创建后，父任务 "[SelfDrive]" agent 第一件事是验证 description 是否充分（> 200 字），不足则补全
- [ ] miniapp dev 任务应在 payload 中包含 `repo_path: /Users/administrator/perfect21/zenithjoy-miniapp`，避免 agent 找不到工作目录
