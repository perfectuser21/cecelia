# Learning: perception.js 删除后遗留的 snapshot=null 陷阱

### 根本原因

`perception.js` 在三层 Brain 重构中被删除，但 `routes.js` 的 `/status` 端点有两处
`const snapshot = null;` 硬编码遗留，而 `system_health.n8n_ok` 完全依赖 `snapshot?.snapshot_json?.n8n?.status === 'ok'`。
因 snapshot 永远为 null，n8n_ok 永远返回 false，触发 Brain 无限派发修复任务的死循环。

### 修复方案

在 `routes.js` 添加 `checkN8nHealth()` 函数：

```js
const N8N_API_URL = process.env.N8N_API_URL || 'http://localhost:5679';

async function checkN8nHealth() {
  try {
    const ctrl = new AbortController();
    const timer = setTimeout(() => ctrl.abort(), 3000);
    const resp = await fetch(`${N8N_API_URL}/healthz`, { signal: ctrl.signal });
    clearTimeout(timer);
    return resp.ok;
  } catch {
    return false;
  }
}
```

在 `/status` 端点的 `Promise.all` 中加入 `checkN8nHealth()`，结果赋值 `n8nAlive`，
替代已失效的 snapshot 路径。
