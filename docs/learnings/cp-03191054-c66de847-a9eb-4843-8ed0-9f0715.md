# Learning: cecelia-bridge.js cwd 隔离 + 飞书配置文档

**分支**: cp-03191054-c66de847-a9eb-4843-8ed0-9f0715
**日期**: 2026-03-19

## 背景

PR #1110 修复了 cecelia-bridge.cjs（生产文件）的 LLM session cwd 隔离问题，但 cecelia-bridge.js 未同步。本次补齐该漂移，并补充 .env.docker.example 中的 FEISHU_OWNER_OPEN_IDS 文档。

### 根本原因

1. **两文件漂移**：cecelia-bridge.js 和 cecelia-bridge.cjs 是独立维护的两份代码，.cjs 是生产运行版本，.js 是历史版本。PR #1110 只更新了 .cjs，导致 .js 缺失 cwd 隔离补丁。

2. **DoD 中包含本地凭据检查**：task card 中的 DoD 测试直接读取 `~/.credentials/feishu.env`，CI runner 上不存在此文件，导致 DoD 验证 Gate 失败。

### 下次预防

- [ ] 修改 cecelia-bridge 时，确保 .js 和 .cjs 两个文件同步变更
- [ ] DoD Test 命令必须能在 CI 环境中执行。读取本地凭据文件（~/.credentials/）的检查**不能**作为 CI DoD 测试；改为验证 .env.docker.example 或代码中的配置文档
- [ ] 凭据更新（FEISHU_OWNER_OPEN_IDS）属于本地运维操作，应在 PR 描述中注明"需要手动更新 ~/.credentials/feishu.env"，不要放在 DoD 测试中
