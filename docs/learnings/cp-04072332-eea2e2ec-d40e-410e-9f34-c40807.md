# Learning: KR5 Dashboard verifier 使用错误指标 — 永远 0%

## 任务
[SelfDrive] [紧急验证] KR进度更新机制失效排查 — 0%进度不可信

## 调查结论
KR verifier 机制本身工作正常（10个KR中9个进度正确）。根本问题是 Migration 223 中
KR5 Dashboard 的 verifier 指标选型错误，导致单个 KR 永远显示 0%。

### 根本原因
Migration 223 给 KR5 Dashboard 分配了 `page_views` 访问量查询（threshold=100），
但 `page_views` 表从未有数据写入（0条记录），导致 `progress = (0/100)*100 = 0%`
且永远不会改变。

KR5 的定义是"3大模块无阻断bug，可完整演示20分钟"，访问量与此无关。
正确指标是：关联 tasks（goal_id）的完成率，当前已验证 = 17%（1/6 任务完成）。

### 下次预防
- [ ] 新增 KR verifier 时，verifier 查询必须能立即返回非零值（否则很可能选错指标）
- [ ] 里程碑类 KR（如小程序上线、geo SEO）用 `SELECT 0` 占位是可接受的，但应注明 "待手动更新"
- [ ] Migration 审查要求：每个新 verifier 查询在 dev DB 上跑一次验证返回值合理
