# Learning: Dashboard KR5 可交付冲刺 — 数据链路修复

## 任务摘要
修复 Dashboard 3大模块数据链路，使 KR5 达到可演示状态。

## 变更清单
1. `apps/dashboard/src/pages/roadmap/RoadmapPage.tsx` — 修复 `/api/goals` → `/api/brain/goals`，`/api/tasks/projects` → `/api/brain/projects`
2. `apps/dashboard/src/components/PRProgressDashboard.tsx` — KR_GOAL_ID 修正为真实 KR5 ID
3. `apps/dashboard/src/pages/viral-analysis/ViralAnalysisPage.tsx` — 新建爆款分析页面
4. `apps/api/features/system-hub/index.ts` — 注册爆款分析路由 + 导航

### 根本原因

**RoadmapPage API 路径错误**：前端调用 `/api/goals` 和 `/api/tasks/projects`，均 404。原因是这两条路由从未在 Brain API 的对应路径注册——Brain 的 goals 路由挂载在 `/api/brain/goals`，projects 挂载在 `/api/brain/projects`。Vite proxy 将非 `/api/brain` 请求转发到 API Server（5211），而 API Server 无这两条路由。

**PRProgressDashboard goal ID 失效**：`KR_GOAL_ID` 硬编码了已删除的 goal UUID，导致 `{"error": "goal not found"}`，完成度仪表盘完全空白。当前 KR5（Dashboard可交付）goal ID 为 `d0e7ee21-5a76-4780-a719-bd6d97ad90e1`。

### 下次预防

- [ ] Brain API 路径规范：Dashboard 前端调用 Brain 数据时，统一使用 `/api/brain/*` 前缀，不应直接用 `/api/*`（后者指向 API Server）
- [ ] goal ID 硬编码隐患：PRProgressDashboard 等依赖特定 goal ID 的组件，建议通过 `/api/brain/goals?title=KR5` 动态查询，而非硬编码 UUID
- [ ] 新建页面时同步注册导航：ViralAnalysisPage 需在 `system-hub/index.ts` 中同时添加 `children` 条目、`routes` 条目、`components` 映射，三处缺一不可
