# Learning: 修复 content-pipeline 成功率崩溃

**分支**: cp-03251849-cf3cb5df-5772-4404-b0f7-1c3cb4
**日期**: 2026-03-26
**PR**: #1577

### 根本原因

`_parsePipelineParams` 中字段名拼写错误：使用 `pipeline.payload?.keyword`，
但 content-pipeline 任务的 payload 实际使用 `pipeline_keyword` 字段名（下划线分隔而非驼峰）。

由于 `pipeline.payload?.keyword` 为 undefined，keyword 回落到 `pipeline.title`，
导致传入 executeGenerate 的关键词是完整 title（`[内容流水线] 决策负担减半 2026-03-26`）而非干净关键词（`决策负担减半`）。

这触发了连锁失败：
- 内容模板使用错误关键词 → 占位符文本极短（copy ~170字 < 300门禁，article ~290字 < 1000门禁）
- executeReview 每次都返回 review_passed=false
- 4次 review 全失败后 pipeline 标记 failed，10个任务全军覆没
- 1小时成功率窗口捕获到这批失败，成功率跌至 24%

### 下次预防

- [ ] payload 字段名要有一处 SSOT 定义（目前 `pipeline_keyword` 散落在多处手写字符串）
- [ ] `_parsePipelineParams` 类似的字段提取函数应该有单元测试，覆盖 `pipeline_keyword` / `keyword` 两种字段名
- [ ] content-pipeline 集成测试应该验证 pipeline 完整运行（research → generate → review → export），而不只是单阶段 mock 测试
- [ ] executeReview 检测到"找不到产出目录"或"内容为空"时，应该记录到 task.error_message（目前是 success:true + review_passed:false，难以快速定位问题）
