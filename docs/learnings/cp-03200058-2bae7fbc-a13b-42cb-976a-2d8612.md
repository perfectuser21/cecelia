# Learning - 修正Scanner对内嵌能力的识别逻辑

## Scanner 内嵌能力识别逻辑修复（2026-03-20）

### 根本原因

Scanner 的 BRAIN_EMBEDDED_SOURCES 映射表不完整，只包含4条映射，而实际Brain内嵌能力有15+个。导致内嵌能力（如emotion-perception、memory-working等）无法通过 cecelia_events 源映射被正确识别，在扫描时误判为 island 状态而非 active。

具体问题：
1. BRAIN_EMBEDDED_SOURCES 映射缺失，只有 circuit-breaker、self-healing 等4个，漏掉了大量内嵌能力
2. 内嵌能力事件源不一致（有些写 memory_stream、有些写 working_memory），但映射表未覆盖
3. Scanner 依赖 BRAIN_ALWAYS_ACTIVE 白名单 + BRAIN_EMBEDDED_SOURCES 映射双重判断，映射不全导致误判

### 下次预防

- [ ] 新增 Brain 内嵌能力时，同步更新 capability-scanner.js 中的 BRAIN_EMBEDDED_SOURCES 映射
- [ ] 建立内嵌能力与事件源的标准映射规范，避免事件源命名不一致
- [ ] 在 capability-scanner 测试中增加内嵌能力识别的专门测试用例
- [ ] 定期审查 BRAIN_ALWAYS_ACTIVE 和 BRAIN_EMBEDDED_SOURCES 的一致性
- [ ] 考虑将内嵌能力映射配置化，而非硬编码在代码中
