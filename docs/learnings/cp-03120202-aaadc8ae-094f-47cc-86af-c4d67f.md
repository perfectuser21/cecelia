## Learning: perception.js 情感词过滤防止 learning_gap_signal 误报

### 问题描述

`learning_gap_signal` 使用 LIKE 关键词（不确定/不清楚/不知道/不太明白）在 memory_stream 中统计「知识盲点」，但日常情感表达（「不知道它们死在哪里」「不确定未来」）必然命中，导致每次 tick 都产生误报 → desire_system 积压重复 research 任务。

### 根因

SQL 语义过于宽泛：`content LIKE '%不确定%'` 无法区分技术上下文和情感表达。

### 修复方案

在 SQL 查询中添加 11 个情感词 `NOT LIKE` 过滤条件，覆盖最常见的情感表达场景：
- 感觉、担心、担忧、未来、将来（心理状态/时间感）
- 情感、生活、朋友（人际关系）
- 开心、难过、喜欢（情绪词）

### DoD 最佳实践

1. **所有验收项（包括 `[x]` 已完成项）都需要 Test 字段**，否则 DoD Verification Gate 报错
2. **DoD 验收项要标 `[x]`**：代码完成后、push 前，把所有完成的条目从 `[ ]` 改为 `[x]`

### 架构影响

无架构变更。`curiosity_accumulated`（section 13）和 `intellectual_idle`（section 14）信号是更精确的自主学习触发机制，此修复不影响这两个信号。
