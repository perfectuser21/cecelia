# Learning: KR 进度诚实化 — /audit 端点注册顺序 + DEFINITION.md 双处同步

## 背景

PR #885 实现 `GET /api/brain/tasks/goals/audit` 端点和 migration 150，将虚标 KR progress 修正为基于 initiative 实际完成率的真实值。

## 根本原因（2026-03-12）

### 根本原因

1. **Express 路由注册顺序陷阱**：`task-goals.js` 原有 `GET /:id`，新增 `GET /audit` 未注意必须在 `/:id` 之前注册，否则字符串 `'audit'` 会被当成 id 参数匹配。

2. **DEFINITION.md schema_version 双处未同步**：修改 `selfcheck.js` 的 `EXPECTED_SCHEMA_VERSION` 后，DEFINITION.md 有两处硬编码（line ~386 和 line ~676）均需更新，漏改导致 L2 facts-check 失败。

3. **PRD 成功标准标题格式错误**：`check-prd.sh` 要求 `## 成功标准` 二级 Markdown 标题，不接受粗体文字 `**成功标准**:`。

### 下次预防

- [ ] 在 Express router 文件新增路由时，先检查是否有 `/:id` 参数路由，若有则新静态路由放前面并加注释
- [ ] 修改 selfcheck.js `EXPECTED_SCHEMA_VERSION` 后立即运行 `grep -n "schema_version\|<旧版本>" DEFINITION.md` 确认两处全部更新
- [ ] PRD 成功标准用 `## 成功标准` 二级标题，不用粗体格式
- [ ] Learning 文件必须包含 `### 根本原因`、`### 下次预防`、`- [ ]` checklist 三个结构

## 补充：migration 审计日志惯用模式

```sql
WITH updated AS (
  UPDATE goals ... RETURNING goals.id, other_fields
)

```
