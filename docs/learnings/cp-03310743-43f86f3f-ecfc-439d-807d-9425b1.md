# Learning: fix(brain): executeResearch 加入 NotebookLM web search 步骤

**分支**: cp-03310743-43f86f3f-ecfc-439d-807d-9425b1
**日期**: 2026-03-31

## 背景

executeResearch 直接对空 notebook ask，没有先添加 web research sources，导致返回内容不足。

## 修复摘要

在 `notebooklm use` 之后、`ask` 之前加入三步：
1. `notebooklm source add-research` 添加 web research sources（deep 模式，no-wait）
2. `notebooklm research wait --timeout 300 --import-all` 等待导入
3. ask 完成后用 `listSources`+`deleteSource` 清空 notebook 以便复用

### 根本原因

executeResearch 只查询但不填充 source，notebook 为空时 ask 返回无意义内容。

- [ ] 调用 NotebookLM ask 前需确认 notebook 有有效 sources
- [ ] web search 步骤在 ask 之前（顺序不可颠倒）
- [ ] ask 后清空 notebook 是 pipeline 复用的必要条件
