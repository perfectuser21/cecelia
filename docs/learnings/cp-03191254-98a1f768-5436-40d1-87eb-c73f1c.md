## 根本原因

### 问题 1：check-dod-mapping.cjs 在 Step 1 要求 [ ] 全部勾选
verify-step.sh step1 调用完整的 check-dod-mapping.cjs，后者在「未验证项检查」阶段会对所有
未勾选 [ ] 的 DoD 条目返回 exit 1。这意味着 .dev-mode 的 step_1_taskcard: done 标记必须等
代码写完、DoD 验证完毕、所有 [ ] 改为 [x] 之后才能写入。

### 问题 2：.dev-mode 需要 tasks_created: true
branch-protect.sh 的 Edit 钩子检查 .dev-mode 是否含有 `tasks_created: true`，缺少会被拦截。
必须在开始写代码前把 `tasks_created: true` 写入 .dev-mode。

### 问题 3：feat PR 必须包含测试文件
L3 Test Coverage Required Gate 要求 feat 类型 PR 必须有对应测试文件改动。
纯 shell 脚本 + .cjs 也不例外，需要在 __tests__/ 下添加测试。

### 问题 4：publish-toutiao-article.cjs 原本不存在
PRD 提到的"Task 3 产出"实际未存在，本任务需要从头实现。
关键设计：用 module.exports 导出纯函数（loadContent/validateContent）以支持单元测试，
CLI 入口用 `if (require.main === module)` 包裹。
