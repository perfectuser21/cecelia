## exit_code=137(OOM) vs exit_code=1(代码错误) 失败分流（2026-03-23）

### 根本原因

`classifyDevFailure` 仅按错误文本模式匹配，未使用 `exit_code` 精确分流：
- exit_code=137（OOM/SIGKILL）→ 错误文本"Process exited with code 137"无法匹配任何模式 → UNKNOWN → 不重试（实际应降并发延迟重试）
- exit_code=1（代码错误）→ 同样落入 UNKNOWN → 不重试也不阻塞（实际应直接 blocked 等待人工）

### 下次预防

- [ ] 分类器应在文本模式匹配**之前**先检查 exit_code 值（特别是 137 和 1）
- [ ] exit_code=137 → RESOURCE_KILLED 类别，retryable=true，reduce_concurrency=true，15/30min 延迟
- [ ] exit_code=1 → CODE_ERROR + needs_blocked=true，不重试，execution.js 写入 status='blocked'
- [ ] 文本匹配的 code_error（TypeScript error 等）仍保持 retryable=true 不变，防止破坏现有逻辑
- [ ] execution.js 调用 classifyDevFailure 时必须传入 exit_code 参数
