# Learning: Engine CI 绿灯率加固 — DoD 守卫

## 任务背景
KR5 Engine CI 绿灯率 50% → 95%。分析最近 100 次 CI 运行（80 成功，7 失败，13 取消）。

## 根本原因
最近 7 次 CI 失败按类别：
| 根因 | 次数 | 状态 |
|------|------|------|
| Engine 版本漂移（.hook-core-version 遗漏） | 1 | ✅ PR 2291 修复 |
| DoD.md 含未勾选条目（harness-contract-lint） | 1 | ✅ 本次修复 |
| Brain 静态分析测试断言过时 | 2 | 自行修复（重构 PR 已更新测试） |
| Registry-lint（cleanup 未更新注册表） | 1 | N/A（清理完成后自然消失） |
| brain-diff-coverage（随 brain-unit 失败） | 1 | N/A（跟随 brain-unit） |
| harness-pipeline-closure 相关 brain-unit | 1 | N/A |

## 核心发现
`quickcheck.sh` 本地预检缺少 DoD 未勾选守卫，导致 agent 在 DoD 条目未全部验证时仍能 push，让 `harness-contract-lint` CI 步骤承担了本应在本地被阻止的错误。

## 下次预防

- [ ] 每次新增 CI check 类别，评估是否能在 `quickcheck.sh` 本地对应拦截
- [ ] `feature-registry.yml` changelog 必须和版本 bump 同步（已有 `bump-version.sh` 提示）
- [ ] Engine 版本 bump 务必用 `bash packages/engine/scripts/bump-version.sh`，不要手动改单个文件
