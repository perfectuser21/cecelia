# Learning: Brain v2 D1.7c-plugin2 — 4 scheduled job 拆 plugin

**日期**：2026-04-26
**分支**：cp-0426001004-d17c-plugin2
**Phase**：Brain v2 Phase D1.7c（接 D1.7b executeTick 抽出之后）

## 概述

把 `tick-runner.js` 的 4 个 inline scheduled job（pipeline-patrol / pipeline-watchdog / kr-health-daily / cleanup-worker）抽到独立 plugin。tick-runner 通过 `import * as xxxPlugin` 引用，每个 plugin 自带节流门 + 错误兜底。

与 builder-3 的 plugin1（dept-heartbeat 等）并行 PR，两人改 tick-runner.js 不同段落。

## 根本原因（为什么要 plugin 化）

executeTick 1843 行里嵌着 30+ scheduled job 调用，每个块约 10–25 行 inline，包含：
1. 节流门（`Date.now() - tickState.lastXxxTime >= INTERVAL_MS`）
2. tickState 写
3. 业务调用（runXxx / dynamic import）
4. .then().catch() 日志收尾

这种 inline 方式带来：
- **不可单测**：节流逻辑 + 业务调用混在一起，要 mock 整个 executeTick
- **不可热替换**：要改某个 job 的节流时间，必须改 tick-runner.js 重启 brain
- **常量重复**：D1.7b 把 `XXX_INTERVAL_MS` 重定义在 tick-runner（避免循环 import），三处常量同步漂移

plugin 化后：
- 每个 plugin 独立单测（mock 上游模块即可）
- 节流配置封装在 plugin 内（env 读 1 次，单点真理）
- tick-runner 只剩一行 `pluginX.tick(ctx).catch(...)`

## 关键设计：保 fire-and-forget 语义

原 inline 用 `.then().catch()` 不 await（除 kr-health-daily），plugin 包装时保持这个特性：

- 调用方式：`pluginX.tick(ctx).catch(err => console.warn(...))`
- plugin 内部 try-catch 兜底 → 返回 `{ error: ... }` 不 throw
- 外层 `.catch()` 是双保险

如果 plugin 改成 await，executeTick 总耗时会从 ~50ms 涨到秒级（pipeline-patrol 扫所有 worktree 慢），破坏 tick 节奏。

**例外**：kr-health-daily 原本就是 await（要 push 到 actionsTaken），plugin 也保持 await + 同步返回结果。

## 关键设计：依赖注入便于单测

plugin 内部 `dynamic import('./kr-verifier.js')` / `dynamic import('./cleanup-worker.js')` 在 vitest 下：
- vitest mock 静态 import 容易，mock dynamic import 在 vitest@1.x 行为不稳
- 解决：plugin 接受 `loadHealth` / `loadWorker` 注入参数，生产路径 fallback 到 dynamic import

测试时直接传 `loadHealth: async () => ({ getKrVerifierHealth: ... })`，绕过 mock 系统，零 vite TDZ 风险。

## 关键约束

1. **不动旧文件**：`pipeline-patrol.js` / `pipeline-watchdog.js` / `cleanup-worker.js` 保持原样，plugin 是 wrapper
2. **wire 命名固定**：DoD grep 检查 `pipelinePatrolPlugin` / `pipelineWatchdogPlugin` / `krHealthDailyPlugin` / `cleanupWorkerPlugin`
3. **常量收口到 plugin**：删 tick-runner.js 里 3 个 `XXX_INTERVAL_MS`，杜绝同步漂移
4. **MINIMAL_MODE 行为不变**：watchdog / cleanup-worker 保持 minimal 跳过
5. **与 builder-3 plugin1 协调**：两人改 tick-runner.js 不同段落，谁先 push 谁直接合，后合 rebase main
