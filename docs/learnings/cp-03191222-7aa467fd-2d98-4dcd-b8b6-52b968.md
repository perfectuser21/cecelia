### 根本原因

本次 CI L1 失败 1 次，根因是 Learning 文件需要在代码提交之前（或同批提交）写好。
CI check-learning.sh 要求 PR 必须包含 `docs/learnings/{branch}.md`，但本次 Learning 是在看到 CI 失败后才补写。

另外，branch-protect hook 在 `packages/` 子目录开发时，有多层 PRD/DoD 文件要求：
1. 根目录需要 `.prd-{branch}.md`（per-branch PRD，v25 packages 子目录保护）
2. `packages/workflows/` 下还需要同名 per-branch PRD（因 hook 就近检测，遇到子目录 `.prd.md` 会停止向上遍历）
3. 同理还需要 `.dod-{branch}.md` 在 `packages/workflows/` 目录

hook 的报错信息是"PRD 文件未更新（.prd.md）"——是因为它找到了 `packages/workflows/.prd.md`（旧文件）而不是 per-branch 新文件，使用旧文件路径检查 git 变更记录时当然找不到，从而误报。
