# Learning: verify-step.sh Gate 动态化修复

## 根本原因

verify-step.sh Gate 1 硬编码了 `packages/engine`，导致其他 packages 改动时跳过 npm test。Gate 2 只处理 `[BEHAVIOR]` 类型，`[ARTIFACT]` 和 `[GATE]` 类型的 DoD 条目被静默跳过，验收不完整。

## 修复内容

1. **Gate 1 动态检测**: 改为读 `git diff` 变更文件，按 `packages/<name>/` 模式提取受影响的 packages，对每个有 package.json + test script 的 package 运行 npm test
2. **Gate 2 扩展**: 同时处理 `[BEHAVIOR]`/`[ARTIFACT]`/`[GATE]` 三种 DoD 类型，消除静默漏验

## 下次预防

- [ ] DoD 类型新增时（如未来的 `[PERF]`），同步更新 verify-step.sh Gate 2 的类型列表
- [ ] verify-step.sh 的 package 检测逻辑如果变复杂，抽成独立函数并写单测
- [ ] CI 需要版本 bump 时，确保 version bump commit 在第一次 push 中（而非事后追加），避免 CI 在 bump 前触发
