## Engine 质量修复 5 处（2026-04-04）

### 根本原因

深度审查发现的积累问题，主要有三类：

1. **正则设计缺陷**：check-dod-mapping.cjs 的 `manual:[^\s]+` 正则在设计时未考虑 `node -e "..."` 命令含空格的场景，导致所有带引号的 manual 验证命令都无法被识别
2. **版本同步漏洞**：check-version-sync.sh 没有检查 hooks/VERSION，而这次新增检查后立即发现它已落后（14.1.1 vs 14.2.2），证明漏洞真实存在
3. **v14.0.0 废弃旧格式但代码未跟上**：branch-protect.sh 保留了 `.dev-mode` 无后缀回退，与 stop.sh v14.0.0 声明的"只支持 per-branch 格式"矛盾；对应测试也还在用旧格式创建文件

### 下次预防

- [ ] 修改 check-version-sync.sh 后立即在本地跑一次确认所有版本文件值相同
- [ ] 废弃旧格式时，同步检查：hook 代码 + 测试辅助函数 + 文档注释三处是否一致
- [ ] 正则涉及 shell 命令参数时，优先用 `[^\r\n]+`（匹配整行剩余）而非 `[^\s]+`（遇空格截断）
- [ ] 新增 CI 检查项时，确保被检查的版本文件已与当前主版本同步（避免新检查立即触发失败）
