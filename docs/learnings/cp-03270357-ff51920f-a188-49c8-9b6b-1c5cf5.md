# Learning: GTDWarRoom 绝对 URL 导致浏览器端暂无数据

## 根本原因

`GTDWarRoom.tsx` 在 `fetchData` 中硬编码了 `http://localhost:5221/api/brain/tasks`。
浏览器从非 localhost 域名（如 `perfect21:5211`）访问时，该绝对 URL 无法通过 vite proxy，
导致 `fetch` 失败，`Promise.all` 整体 reject，`catch` 将 `tree` 也清空，页面显示"暂无数据"。

## 修复

1. 改为相对路径 `/api/brain/tasks`（vite proxy 已配置 `/api/brain` → `localhost:5221`）
2. `Promise.all` → `Promise.allSettled`，两个请求独立容错，互不影响

## 下次预防

- [ ] 前端组件中禁止使用 `http://localhost:` 绝对 URL，必须走相对路径 + vite proxy
- [ ] 多个独立 fetch 请求使用 `Promise.allSettled` 而非 `Promise.all`，避免单个失败污染整体
