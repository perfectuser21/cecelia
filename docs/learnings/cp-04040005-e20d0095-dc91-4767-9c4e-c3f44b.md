# Learning: 重构高复杂度函数（contactFieldsToNotionProps）

### 根本原因
函数内有 10 个 if-else-if 分支（含 &&/|| 运算符），加上内嵌的 else 块再套 4 个分支，圈复杂度达 27。所有分支和辅助常量（10 个 Set）都在同一函数体内定义，导致函数臃肿且难以测试单一行为。

### 下次预防
- [ ] 类似的"大型 if-else 分支 + 内嵌条件"模式，第一次写时就拆为独立辅助函数
- [ ] 静态常量（Set/Map）放在模块级别，避免每次调用重建
- [ ] 分发逻辑（dispatch）与检测逻辑（detect by value）职责分离：各自一个函数
