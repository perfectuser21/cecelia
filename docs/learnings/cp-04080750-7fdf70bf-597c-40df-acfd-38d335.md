# Learning: 认证层故障深度排查与恢复

## 根本原因

account3 OAuth token 于 2026-04-05 过期后，连续 35 小时产生 295 次 auth 失败风暴，原因有三：

1. **`scanAuthLayerHealth` 未存在**：该探针（PR #2076）在 04-08 才合并，故障期间完全缺失
2. **`AUTH_FAIL_RATE_THRESHOLD = 5` 设置过高**：实际故障期间失败速率约 3 次/UTC-hour，阈值 5 永远不会触发
3. **`/api/brain/credentials/health` 缺失 token 到期时间**：操作者无法从 API 快速感知各账号 token 剩余时间

## 修复内容

- `credential-expiry-checker.js`：`AUTH_FAIL_RATE_THRESHOLD` 5 → 3；扫描窗口 1h → 4h
- `infra-status.js`：`/credentials/health` 端点新增 `token_expires_at`、`token_remaining_hours`、`token_status` 字段
- 直接恢复因 auth 失败被隔离的 `sprint_contract_propose` 任务（id: 6f475276）

## 下次预防

- [ ] Token 刷新后立即检查 `/api/brain/credentials/health` 中的 `token_status` 字段
- [ ] `token_status: expiring_soon` 出现时必须在 8h 内手动刷新对应账号的 OAuth token
- [ ] auth 失败率告警阈值 ≤ 3，窗口 ≥ 4h，防止低频失败风暴漏报
