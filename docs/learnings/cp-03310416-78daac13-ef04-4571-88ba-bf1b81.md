# Learning: content-type YAML 固定 notebook_id + orchestrator 传参 + export 清空 notebook

**分支**: cp-03310416-78daac13-ef04-4571-88ba-bf1b81
**日期**: 2026-03-31

### 根本原因

content-pipeline research 阶段依赖 `task.payload.notebook_id` 来定位 NotebookLM notebook，
但 orchestrator._startOnePipeline 创建 research 子任务时未将 typeConfig 中的 notebook_id 透传到 payload，
导致 research 执行时 notebook_id 为空，报错中止。

同时，export 完成后 notebook 内的源材料未清空，累积后干扰下一次 pipeline 的 research 阶段。

### 下次预防

- [ ] content-type YAML 新增字段时，需同步检查 orchestrator 是否将该字段传递给首个子任务
- [ ] 生命周期结束动作（清空/归档/重置）应在 export 阶段完成，确保资源可复用
- [ ] 新增 async 外部调用（notebook-adapter）时，用 try/catch 包裹并加降级判断（notebook_id 为空时跳过），避免阻断主流程
