# Learning: 快手视频发布脚本 publish-kuaishou-video.cjs

**Branch**: cp-03191204-d79e05e2-8b5d-409a-a1dc-58db5e
**Date**: 2026-03-19
**Task**: 新增快手视频发布能力（视频上传 + 分块 + 封面 + 标签）

### 根本原因

快手 publisher 只有图文发布能力，视频发布需求未被覆盖。
图文 API 方案（CDP Cookie + HTTP）证明稳定，视频发布沿用同一技术栈。

### 技术决策

1. **复用 CDP Cookie 提取模式**：与 `publish-kuaishou-api.cjs` 完全一致，不引入新的认证逻辑
2. **分块上传设计**：视频文件可能较大，采用 4MB 分块，小文件（≤4MB）走单块路径
3. **端点说明**：视频 API 端点基于快手 CP 内部 API 命名规律推断，若 404 需通过 Network 面板抓包更新
4. **纯函数测试**：与图文方案保持一致，业务逻辑层（网络/CDP）不测，只覆盖纯工具函数

### 下次预防

- [ ] 快手 API 端点不稳定，每次实际使用时先通过 Network 面板确认端点再执行
- [ ] 视频 API 与图文 API 端点格式一致（`/rest/cp/works/upload/video/token` 对应 `/upload/photo/token`），类推时先验证
- [ ] 新建 publisher 脚本时，先看现有同平台脚本结构再写，保持风格一致
