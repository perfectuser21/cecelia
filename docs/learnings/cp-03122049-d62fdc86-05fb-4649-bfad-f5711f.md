## Cortex 超时原因分类器（2026-03-12）

**背景**：Cortex L2 皮层 LLM 调用失败时，旧代码没有区分根因，日志只有 `Deep analysis failed: <message>`，无法快速定位是 quota / token / network 问题。

### 根本原因

`cortex.js` catch 块缺少结构化的错误根因分类。三类根因（HTTP 429 quota、HTTP 400 prompt 超长、timeout 网络超时）可以通过 `err.status`、`err.degraded`、`err.message` 关键词直接区分，但旧代码未做区分，只原始输出 error.message，导致 Brain 监控层无法按类型统计和告警。

### 下次预防

- [x] 新增 LLM 调用时，在 catch 块中调用 `classifyTimeoutReason(err)` 并输出结构化 JSON（含 `timeout_reason` 字段）
- [x] HTTP 状态码 + message 关键词作为分类依据（429→quota_exhausted，400+token/context→prompt_too_long，degraded/timeout→network_timeout）
- [x] 分类函数 export，确保单元测试可直接导入覆盖所有 case
- [ ] 后续为 `quota_exhausted` 和 `network_timeout` 接入 P1 级别 recordLLMError 统计，便于熔断阈值调优
