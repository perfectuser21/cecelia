# Learning: execution-callback harness verdict 未持久化 + ci_watch 无 pr_url 僵尸任务

### 根本原因
1. harness_evaluate/sprint_evaluate/contract_review 产生的 verdict 仅存在于内存中用于路由，未写入 `tasks.result`，导致调试时无法追溯各环节判断结果。
2. harness_generate/harness_fix 提取 pr_url 失败时（Generator 未产生 PR），仍会创建 payload.pr_url=null 的 harness_ci_watch 任务，该任务永远无法轮询到 CI 状态，形成僵尸任务。

### 下次预防
- [ ] 凡是 harness 任务产生决策字段（verdict/decision），必须同时写入 tasks.result
- [ ] createHarnessTask 创建 ci_watch 类型时，pr_url 为 payload 必填字段，缺失应抛错终止链路而非静默创建
