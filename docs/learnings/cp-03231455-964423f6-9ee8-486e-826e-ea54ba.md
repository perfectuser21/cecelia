# Learning: 重构 runReflection 降低圈复杂度

## 任务背景
Brain 代码复杂度扫描触发，`runReflection` 函数圈复杂度 39，阈值 10。

### 根本原因
函数承担 7 项职责（静默期检查、阈值校验、记忆去重、LLM 调用、哈希熔断、Jaccard 相似度去重、写入洞察），所有逻辑平铺于一个函数，导致大量嵌套 if/try-catch。

### 下次预防
- [ ] 新增功能时，单函数职责超过 3 项就立即提取子函数
- [ ] 每次提交前用 `node -e` 快速计数 if/catch/for 关键字，保持 < 10
- [ ] 提取模式：`_check*`（检查+早返回）、`_handle*`（处理分支逻辑）、`_write*`（副作用写入）
