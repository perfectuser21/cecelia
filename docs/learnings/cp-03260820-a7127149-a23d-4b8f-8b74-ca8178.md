### 根本原因

本次审计发现内容 pipeline 最大的断链在**数据采集层**：`social_media_raw` 数据库从未被创建。8 个平台的 scraper 脚本都连接这个不存在的数据库，导致采集完全无法工作。

### 关键审计发现

1. **P0 阻断**：`social_media_raw` DB 不存在 — 所有 scraper 连接立即失败
2. **P1 断链**：content-export executor 生成 manifest.json 后未写入 `zenithjoy.works` — 产出物是孤岛
3. **P1 断链**：5个平台（头条/微信/小红书/视频号/知乎）无 platform_post_id 提取 — 无法数据回流
4. **架构设计问题**：`publish_results`（cecelia DB）与 `zenithjoy.publish_logs` 是两套独立的发布记录系统，未打通

### 整体评分

| 层 | 评分 |
|---|---|
| 数据采集 | 0/10（DB 不存在） |
| 内容生产 | 8/10（逻辑完整） |
| 内容发布 | 9/10（8个平台已实现） |
| 数据回流 | 3/10（只有3/8平台） |
| **总体** | **5/10** |

### 下次预防

- [ ] 新系统上线前先验证所有依赖 DB 是否存在（`psql -U cecelia -l`）
- [ ] Scraper 脚本应在连接失败时打印清晰的诊断信息（当前静默失败）
- [ ] content-export executor 完成后应有写入 works 表的断言
- [ ] 发布结果写入应统一到一张表，消除 `publish_results` vs `publish_logs` 二义性
