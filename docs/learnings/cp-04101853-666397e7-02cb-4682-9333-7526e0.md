## fix(brain): proactiveTokenCheck 不清除 API 401 设置的 auth_failed（2026-04-11）

### 根本原因

`proactiveTokenCheck` 发现 token 文件未过期时会无差别清除 `auth_failed` 熔断，但未考虑熔断的来源。Anthropic API 返回 401 设置的熔断（`api_error`）与 token 文件过期设置的熔断（`token_expired`）语义不同：前者表示凭据无效，token 文件有效不代表 API 可用；后者表示 token 过期，token 刷新后应恢复。错误清除 `api_error` 熔断会导致同一账号不断被派发 → 不断 401 的死循环。

### 修复思路

在 `markAuthFailure` 的 entry 中增加 `source` 字段（默认 `api_error`）。`proactiveTokenCheck` 清除逻辑改为只清除 `source === 'token_expired'` 的熔断。

### 下次预防

- [ ] 新增"状态标记 + 清除"配对函数时，先问：**清除的条件是否和设置条件严格对称**？不对称则需要 source 字段区分
- [ ] `vitest 1.x` 项目禁止在 `describe/it` 内部放 `vi.mock()`：多个同模块 `vi.mock` 均被提升，后者覆盖前者，导致跨测试状态污染且本地（4.x）不可复现。应使用 `vi.mocked().mockImplementation()` 或顶层统一 mock
