# Learning：重构 cecelia-routes 降低函数复杂度

## 任务背景

Brain ComplexityScanner 报告 `cecelia-routes.js` 中 `mapStatus` 复杂度为 26（任务 PRD 误导），
实际是 PR #1353 已修复扫描器误报，`mapStatus` 本身复杂度仅为 2。
真正需要重构的是 `inferStepProgress`（15）和 `buildCheckpoints`（13）。

### 根本原因

Brain 自动生成任务时基于扫描时的快照，扫描器 bug（PR #1353 修复前）导致任务 PRD 中
函数名和复杂度信息有偏差。执行任务时应以实际当前代码状态为准，而非 PRD 中的数字。

### 下次预防

- [ ] 执行复杂度重构任务前，先运行 ComplexityScanner 验证当前实际状态，不盲信 PRD 数字
- [ ] 若 PRD 目标函数实际复杂度已达标，搜索同文件中其他超标函数一并处理
- [ ] branch-protect hook 需要 `.dev-mode.<branch>` 包含 `tasks_created: true`，创建时即加入

## 重构方案

将条件逻辑提取为独立辅助函数（early return 模式）：

- `parseStepIdx(stepStr, total)` → 解析并验证步骤索引（消除重复的 parseInt/isNaN 逻辑）
- `buildFailedProgress(stepStr, total)` → failed/cancelled 分支专用
- `buildRunningProgress(stepStr, total)` → in_progress/running 分支专用
- `resolveCheckpointStatus(status, stepNum, currentStepIdx)` → 将 buildCheckpoints 内嵌条件外移

`inferStepProgress`：15 → 4；`buildCheckpoints`：13 → 6；42 个测试全部通过。
