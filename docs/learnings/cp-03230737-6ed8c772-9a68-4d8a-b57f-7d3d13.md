## 根本原因

1. **[GATE] DoD 测试使用 `echo` 被 CI 误判为假测试**：`manual:bash -c 'bash -n script.sh && echo OK'` 中的 `echo` 触发了 CI fake-test 检测规则。正确做法是 `manual:bash -c 'bash -n script.sh'`——bash -n 本身的 exit code 就是断言，不需要额外 echo。

2. **`packages/engine/skills/` 改动必须 bump Engine 版本 + [CONFIG] 标签**：修改 `01-spec.md` 属于 skills/ 改动，需要同步更新 5 个版本文件并在 PR title 加 `[CONFIG]`。这一要求在 memory 中有记录但实践中容易遗漏。

3. **`readdirSync` 返回目录项导致 EISDIR**：DoD 测试遍历 `steps/` 目录时未过滤子目录（`.archive`），会在 `some()` 回调中抛出 EISDIR，整个测试失败。必须加 `.filter(f => statSync(dir+'/'+f).isFile())`。

## 下次预防

- [ ] DoD 中 `[GATE]` 条目的 Test 命令不要加 `&& echo OK`，bash 命令的 exit code 本身就是断言
