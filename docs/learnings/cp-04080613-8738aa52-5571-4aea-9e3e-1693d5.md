# Learning: Dashboard KR5 — Roadmap 页面字段映射修复

**分支**: cp-04080613-8738aa52-5571-4aea-9e3e-1693d5
**日期**: 2026-04-08

### 根本原因

Brain API (`/api/brain/projects`) 返回的字段名与前端 `Project` interface 期望的字段名不一致：

| API 实际字段 | 前端期望字段 | 影响 |
|---|---|---|
| `title` | `name` | 项目卡片标题空白 |
| `end_date` | `deadline` | 截止日期不显示 |

同样，`/api/brain/goals` 返回 `current_value`（字符串），而前端期望 `progress`（数字），导致 KR 进度条全部显示 0%。

### 修复方式

在数据获取后立即做字段规范化，不改动 interface 定义和渲染层：

```tsx
const normalizeProject = (raw) => raw.map(p => ({
  ...p,
  name: p.title ?? p.name ?? '',
  deadline: p.end_date ?? p.deadline ?? null,
}));

// goals: current_value 解析为 progress 数字
progress: parseFloat(g.current_value ?? metadata.metric_current ?? '0') || 0
```

### 下次预防

- [ ] 新增 API 端点时，同步检查前端 interface 字段名是否匹配
- [ ] Brain API 返回的项目字段应统一用 `title`（与 goals 保持一致），前端 interface 同步更新
- [ ] 字段映射问题应在 API 路由层或前端 adapter 层处理，不散布在组件内
