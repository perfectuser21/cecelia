# Learning: KnowledgeModules UI 重设计

### 根本原因

KnowledgeModules.tsx 和 KnowledgeModuleDetail.tsx 在实现时使用了 Tailwind 亮色 class（`bg-white`、`text-gray-900`、`border-gray-200`），与系统其他页面（InstructionBook）采用的暗色 inline style 体系不一致。

查看链接通过 `navigate('/knowledge/view?url=...')` 跳转内部路由，而 `output_url` 实际指向 `38.23.47.81:9998` 静态文件服务器，内部路由无法正确处理该绝对路径。

新功能页面在实现时未参考现有风格基准（InstructionBook），也未在 DoD 中明确外部链接必须用 `window.open` 的约束，导致两个问题在代码审查前未被发现。

### 下次预防
- [ ] 新增知识模块页面时，参考 InstructionBook inline style 暗色体系（`#0d0d0d` 背景，`#141414` 卡片，`#1f1f1f` border）
- [ ] 外部文件链接（38.23.47.81:9998）一律用 `window.open(absoluteUrl, '_blank')`，不通过内部 navigate
- [ ] DoD Test 字段中用 `node -e "...includes('#0d0d0d')"` 验证暗色主题，防止回退到亮色
