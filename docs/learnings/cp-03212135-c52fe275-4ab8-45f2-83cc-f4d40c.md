# Learning: Watchdog RSS 阈值启动时 sanity check

**分支**: cp-03212135-c52fe275-4ab8-45f2-83cc-f4d40c
**日期**: 2026-03-22

### 根本原因

Watchdog 的 RSS 阈值在模块加载时由 `os.totalmem()` 动态计算：

```js
const RSS_KILL_MB = Math.min(Math.round(TOTAL_MEM_MB * 0.35), 2400);
```

在 Crisis 模式（systemPressure >= 1.0）下，任何活跃任务都会被标记为候选杀进程，包括 RSS=1MB 的进程。若 `TOTAL_MEM_MB` 在某些环境（容器/测试）中上报异常偏小，`RSS_KILL_MB` 会极小，导致正常任务被频繁误杀。

该问题此前无任何启动时校验，只能通过运行时任务失败日志才能发现。

### 下次预防

- [ ] 任何依赖运行时系统状态计算的关键阈值，都应在 `selfcheck.js` 中添加合理性校验
- [ ] Watchdog 阈值校验标准：`TOTAL_MEM_MB >= 512`，`RSS_KILL_MB >= 50`
- [ ] 新校验项须是 WARN-only（非阻断），避免 Brain 因环境差异无法启动
- [ ] 测试时通过 `opts.watchdogThresholds` 注入，避免 mock 整个 watchdog.js 模块
