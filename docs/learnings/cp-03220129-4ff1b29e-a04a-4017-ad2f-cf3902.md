## 重构 createTask 降低圈复杂度（2026-03-22）

### 根本原因

`createTask` 函数将参数默认值处理、INSERT SQL 构建、去重逻辑全部混在同一函数体内，导致圈复杂度累积到 23。主要贡献者：11 个 `||` 默认值表达式、2 条 INSERT 路径（有/无 domain）、race condition dedup 路径。

### 下次预防

- [ ] 新增数据库写入函数时，参数默认值统一提取到 `buildXxxParams()` 辅助函数，避免在主函数体内堆积 `|| default`
- [ ] SQL 存在多个变体（有/无可选列）时，提取 `buildXxxStatement()` 返回 `{ sql, params }`，主流程只负责调用
- [ ] 函数超过 60 行时主动检查是否可以拆分辅助函数，不等到复杂度扫描报警才处理
