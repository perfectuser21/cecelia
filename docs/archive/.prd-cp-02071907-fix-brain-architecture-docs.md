# PRD: 修复 Brain 架构文档 - 澄清单 Brain 原则

## 问题描述

当前文档描述可能让人误解为"双 Brain"架构（US 和 HK 各有一个 Brain）。

**误导性描述**：
- "双服务器" - 暗示 US 和 HK 各有完整的 Brain
- "区域路由" - 暗示 Brain 会"迁移"到 HK
- "任务类型与路由" - 不清楚是 Brain 路由还是执行端选择

**实际架构**：
- **单 Brain**：只在 US 运行（唯一 tick、调度、决策、DB 写入）
- **HK 是执行端**：MiniMax executor、N8N workflows，不运行 Brain
- **执行端选择**：Brain 在 US 决策后，选择调用哪个执行端

## 解决方案

修改 DEFINITION.md 的 3 个关键部分：

1. **新增 2.3 单 Brain 原则** - 明确唯一调度源、唯一真相源、执行端无状态
2. **修改 4.6 任务类型与执行端映射** - 从"路由"改为"执行端选择"
3. **修改 8.1 US Brain + HK Runtime** - 明确单点 Brain + 多执行端架构
