## PRD - quickRoute USER_MESSAGE 快速路由

**需求来源**: 为 thalamus.js 的 quickRoute 函数添加 USER_MESSAGE 事件类型的快速路由规则

**功能描述**:
当前 quickRoute 已覆盖多种事件类型（HEARTBEAT, TICK, TASK_COMPLETED 等），但 USER_MESSAGE 仍然 fall through 到 Sonnet。
本次需要为 USER_MESSAGE 添加策略分级处理：
- intent=status_query → log_event（confidence: 0.85）
- intent=acknowledge → log_event（confidence: 0.9）
- 其他 intent → return null，交给 Sonnet

**涉及文件**:
- brain/thalamus.js（quickRoute 函数添加 USER_MESSAGE 分支）
- brain/thalamus.test.js（新增 quickRoute USER_MESSAGE 测试用例）
- DEFINITION.md（action_count / cortex_extra_actions 若有变动需同步）

**成功标准**:
1. USER_MESSAGE + intent=status_query → log_event，confidence=0.85
2. USER_MESSAGE + intent=acknowledge → log_event，confidence=0.9
3. USER_MESSAGE + 其他 intent（如 command）→ return null（进入 Sonnet）
4. ACTION_WHITELIST 包含 log_event（已存在，验证即可）
5. facts-check.mjs 通过
6. vitest 新增 quickRoute USER_MESSAGE 测试全部通过

**非目标**:
- 不修改其他事件类型的 quickRoute 逻辑
- 不新增 ACTION_WHITELIST 条目（log_event 已存在）
- 不修改 Sonnet 决策流程
