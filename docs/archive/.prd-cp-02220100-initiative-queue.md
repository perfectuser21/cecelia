## PRD - Initiative 队列管理机制

**需求来源**: 系统性问题修复 - 360 个 initiative 全是 active 状态，decomp-checker 给所有人同时补货，token 全烧在"生成工作"上
**功能描述**:
实现 Initiative 队列管理机制，限制同时 active 的 initiative 数量：
- 同时最多 10 个 active initiative（匹配 6 个并发 slot，留余量）
- 其余排队（pending 状态）
- 一个 initiative 完成 → 自动激活下一个 pending
- decomp-checker 只给 active 的 initiative 补货

**涉及文件**:
- `brain/migrations/047_initiative_queue.sql`（新建）
- `brain/src/initiative-closer.js`（新增 `activateNextInitiatives` 函数）
- `brain/src/tick.js`（Section 0.10：集成激活逻辑）
- `brain/src/decomposition-checker.js`（确认只处理 active initiative）
- `brain/src/__tests__/initiative-queue.test.js`（新建测试文件）
- `brain/src/selfcheck.js`（更新 EXPECTED_SCHEMA_VERSION）
- `brain/package.json`（版本号 bump：minor）

**成功标准**:
1. migration 047 执行后：没有活跃任务的 active initiative 改为 pending
2. `activateNextInitiatives()` 能从 pending 按 KR 优先级激活 initiative，总数不超过 MAX_ACTIVE_INITIATIVES(10)
3. tick.js Section 0.10 每次 tick 触发激活检查
4. initiative 完成关闭后立刻激活下一个 pending
5. decomp-checker 只给 status='active' 的 initiative 补货（已确认）
6. 所有测试通过（Q1-Q4 + DevGate）

**非目标**:
- 不修改 inactive/archived 的 initiative
- 不改变已完成的 initiative 状态
- 不添加 UI 界面（workspace 职责）
- 不改变任务调度核心逻辑
