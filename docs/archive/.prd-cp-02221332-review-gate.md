## PRD - v1.64.0 审查门控（Review Gate）

**需求来源**: OKR 渐进式拆解与验证系统 — PR 2/4
**功能描述**: 拆解完成后不直接激活下一层，进入 pending_review → Vivian 审查 → 通过后激活
**涉及文件**:
- `brain/src/review-gate.js` — 新文件，审查门控核心逻辑
- `brain/src/task-router.js` — LOCATION_MAP 新增 decomp_review → hk
- `brain/src/routes.js` — execution-callback 处理 decomp_review 完成
- `brain/src/decomposition-checker.js` — 新产出设为 pending_review
- `brain/src/selfcheck.js` — EXPECTED_SCHEMA_VERSION 050 → 051
- `brain/migrations/051-review-gate.sql` — decomp_reviews 表

**成功标准**:
1. shouldTriggerReview 正确判断是否需要审查
2. createReviewTask 创建 decomp_review 类型任务并路由到 HK
3. processReviewResult 根据 verdict 激活/修正/拒绝实体
4. task-router decomp_review → hk
5. execution-callback 处理 decomp_review 完成
6. Migration 051 创建 decomp_reviews 表

**非目标**:
- 不实现 progress-reviewer（PR 3）
- 不修改秋米 prompt（PR 4）
- 不实现 Vivian 的 LLM 审查逻辑（那是 skill 层面）
