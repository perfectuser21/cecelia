# Task Card: 角色精简 10→5 + 丘脑只做 domain 路由

## 需求（What & Why）
**功能描述**:
1. role-registry.js 精简为 5 个角色（CTO/CPO/CMO/CFO/COO），按业务领域划分不按工作类型。每个角色加 `task_types` 字段。
2. thalamus.js prompt 删除硬编码的意图→task_type 路由表，改为只输出 domain。丘脑只判断"这是哪个领域的事"，不直接决定 task_type。

**背景**: 当前丘脑绕过了 role-registry 角色体系，自己硬编码了关键词→task_type 映射。导致所有 coding 类请求直接变成 task_type=dev（1 PR），跳过了"这事有多大"的判断。10 个角色中部分有重叠（CTO/CISO/VP QA），部分从未使用（CFO skills 为空）。

**不做什么**:
- 不改 executor.js 的 skillMap
- 不改 task-router.js 的 LOCATION_MAP
- 不加角色判断大小的 LLM 调用（那是下一个 PR）
- 不改 /plan SKILL.md
