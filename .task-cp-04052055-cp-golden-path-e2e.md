# Task Card: Golden Path E2E 测试套件

## 需求（What & Why）

**功能描述**: 为 Cecelia Brain 编写 3 条 Golden Path 端到端测试，覆盖最核心的用户路径。任何一条 Golden Path 断了都是 P0 故障。

**背景**: 现有测试层以单元测试为主（mock DB），集成测试覆盖 health/tasks/context/okr。但缺少针对"完整调度链路"、"pipeline 创建→DB 落地→状态查询"、"deploy 端点完整性"的专项 Golden Path 测试文件，便于未来快速识别哪条核心链路断了。

**不做什么**: 不重写现有测试，不覆盖已有 brain-endpoint-contracts 和 critical-routes 的内容。Golden Path 测试聚焦"从 API 到 DB" 的完整链路验证。

## 成功标准

- [ARTIFACT] 文件存在于 packages/brain/src/__tests__/integration/golden-path.integration.test.js
- [BEHAVIOR] 3 条 Golden Path 测试在真实 PostgreSQL 环境下全部通过
- [BEHAVIOR] 测试使用真实 DB 写入+查询（不 mock pool），覆盖 tasks 表 CRUD + pipelines 创建 + health/deploy 端点响应结构

## 实现方案

**要改的文件**:
- 新建 `packages/brain/src/__tests__/integration/golden-path.integration.test.js`

**受影响函数/API**:
- `GET/POST/PATCH /api/brain/tasks` — task-tasks.js router
- `GET/POST /api/brain/pipelines` — content-pipeline.js router
- `GET /api/brain/health` — goals.js router
- `GET /api/brain/deploy/status` — ops.js router

**不改什么**: 不改生产代码，不改现有测试，只新增集成测试文件。

**测试策略**:
- 使用真实 PostgreSQL（与 CI brain-integration job 一致）
- beforeAll 插入测试数据，afterAll 清理
