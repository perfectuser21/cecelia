# Task Card: 三平台发布结果回调 + 飞书通知统一化

## 需求（What & Why）
**功能描述**: 为抖音/快手/小红书三平台建立统一的发布结果追踪和飞书通知机制。每个发布脚本完成后输出标准 JSON（success/platform/contentType/workId/url/error），N8N 解析结果后推送飞书通知，并写入 Brain PostgreSQL publish_results 表。

**背景**: 目前小红书有飞书通知，但无统一回调写入 DB；快手在 N8N 中无路由；抖音路由指向已失效 VPS。三平台结果无法统一追踪。

**不做什么**:
- 不修改抖音脚本（在 zenithjoy repo）
- 不修改 PR3 的 Webhook 路由逻辑
- 不修改其他平台（微博/头条/视频号等）

## 成功标准

1. [ARTIFACT] Brain DB 有 `publish_results` 表，迁移文件幂等执行
2. [ARTIFACT] Brain API `POST /api/brain/publish-results` 可写入发布结果
3. [BEHAVIOR] 快手发布脚本执行完成后输出标准 JSON（`{"success":true,"platform":"kuaishou","contentType":"video","workId":"xxx","url":"xxx"}`）
4. [BEHAVIOR] 小红书发布脚本执行完成后输出标准 JSON
5. [BEHAVIOR] N8N `flow-内容发布.json` 新增快手路由，调用 Mac mini 上的脚本
6. [BEHAVIOR] N8N 解析发布结果后调用 Brain API 写入 publish_results
7. [BEHAVIOR] N8N 飞书通知含作品 workId 和 url（成功时）或错误原因（失败时）
8. [GATE] CI 全部通过
