# PRD：branch-protect 分支日期警告 + cleanup.sh step_* 验证改为非阻塞

## 背景

当前存在两个 P2 问题：

1. `branch-protect.sh` 检查分支名格式（`cp-YYYYMMDD-*`），但不验证日期是否合理。AI 可能使用过时日期（如 `cp-20260101-xxx`）创建分支。

2. `cleanup.sh` 在执行前验证"所有 11 步都是 done"，但 step_* flag 现在只是展示用，不是流程控制依据（devloop-check.sh 已改为检查实际 PR/CI 状态），导致误阻塞合法清理操作。

## 目标

修复上述两个问题，提高开发工作流的健壮性。

## 范围

- `packages/engine/hooks/branch-protect.sh`：添加分支日期范围检查（只警告，不阻塞）
- `packages/engine/skills/dev/scripts/cleanup.sh`：将 step_* 验证从 exit 1 改为 WARN 警告，不阻塞清理

## 需求

### 需求 1：branch-protect.sh 分支日期检查

在分支名格式检查通过后，额外验证时间戳是否在合理范围内（今天 -2 天至今天）：

- 提取分支名中的日期（格式：`cp-YYYYMMDD-*`）
- 计算今天和 2 天前的日期
- 如果分支日期超过 2 天，输出 WARN 到 stderr，**不 exit 2**（长任务可能跨天）
- 兼容 macOS（`date -v-2d`）和 Linux（`date -d "2 days ago"`）

### 需求 2：cleanup.sh step_* 验证改为非阻塞

- 找到所有 step_* 验证逻辑（可能包含 exit 1）
- 将 exit 1 改为输出 WARN 并继续执行
- 确保清理操作不被 step_* flag 状态阻塞

## 成功标准

- branch-protect.sh 在分支日期超 2 天时输出 WARN，不退出
- branch-protect.sh 在分支日期正常时无额外输出
- cleanup.sh step_* 验证不再阻塞流程，改为 WARN 继续
- 兼容 macOS 和 Linux 日期命令

- feature-registry.yml 新增条目
