# Task: 修复 GTD 页面空数据 — 接通 Brain 真实表

## 背景
GTD 系列页面（/gtd/okr, /gtd/projects, /gtd/area）调用 apps/api 的 `/api/tasks/goals`、`/api/tasks/projects`、`/api/tasks/areas`，但这些路由查的本地表（goals/projects）不存在，导致前端永远显示空数据。

真实数据在同一个 cecelia DB 的 objectives/key_results/okr_projects/okr_initiatives/areas 表中。

## 改动范围
- `apps/api/src/task-system/goals.js` — GET / 改查 objectives + key_results（UNION ALL）
- `apps/api/src/task-system/projects.js` — GET / 改查 okr_projects + okr_initiatives
- `apps/api/src/task-system/areas.js`（新建）— GET / 查 areas 表
- `apps/api/src/task-system/routes.js` — 挂载 /areas

## 成功标准
- `GET /api/tasks/goals` 返回 objectives+key_results 数据（type: area_okr/kr）
- `GET /api/tasks/projects` 返回 okr_projects(type:project) + okr_initiatives(type:initiative)
- `GET /api/tasks/areas` 返回 areas 表数据
- /gtd/okr 前端不再显示 0 条
