# CeceliaPage V2 Phase 3 — 从聊天窗口到指挥室

## 背景

Phase 2 (PR #8) 完成了组件拆分（AmbientGlow、PulseStrip、ConsciousnessFlow、ActivityPanel），
但页面依然以聊天流为主体，80% 面积是对话记录。用户反馈"很弱"、"很乱"、"不知道能点啥"。

## 目标

将 CeceliaPage 从「聊天窗口」重构为「管家指挥室」。

## 设计依据

基于调研（Datadog/Grafana 运维监控、Temporal/Dagster 工作流编排、AgentOps/LangSmith AI 可观测、
Linear SaaS 设计、Home Assistant 智能家居、Notion 3.0 主动式 AI），提炼出以下原则：

- **5 秒规则**：用户 5 秒内判断「系统是否 OK」
- **倒金字塔**：最紧急的在最上面
- **渐进式披露**：总览 → 点击展开 → 详情
- **有事才显示**：Action Zone 无事隐藏，有事展开
- **聊天降级**：Cecelia 是自主运行的管家，用户来检查状况和做决策，不是来聊天

## 布局改动

### 当前（Phase 2）
```
PulseStrip
┌──────────────────┬─────────────┐
│ ConsciousnessFlow│ ActivityPanel│
│ (聊天+事件+任务  │ (运行/集群/ │
│  全部混在一起)    │  统计)      │
└──────────────────┴─────────────┘
```

### 目标（Phase 3）
```
PulseStrip
┌──────────────────────────────────┐
│ Action Required Zone (有事才展开) │
└──────────────────────────────────┘
┌──────────┬──────────┬────────────┐
│ Agent    │ 今日概览  │  事件流     │
│ 监控     │          │            │
└──────────┴──────────┴────────────┘
┌──────────────────────────────────┐
│ 💬 聊天抽屉 (默认折叠)            │
└──────────────────────────────────┘
```

## 具体组件改动

### 1. ActionZone（新组件）
- 待决策 Desire + 待确认 Task 集中展示
- 每项带操作按钮（派发/批准/跳过）
- 无事时完全隐藏（零高度），有事时自动展开

### 2. AgentMonitor（新组件，取代 ActivityPanel 的部分功能）
- 运行中 Agent：名称、任务、进度条、已用时间
- 队列：等待中的任务列表

### 3. TodayOverview（新组件）
- 完成/失败/排队计数
- 成功率进度条
- 集群节点状态（CPU/RAM）
- 今日焦点 KR

### 4. EventStream（新组件）
- 时间轴格式（非卡片），最新在顶
- 颜色编码：绿=完成，红=失败，蓝=进行中，灰=tick

### 5. ChatDrawer（重构 ConsciousnessFlow）
- 底部固定，默认折叠只显示输入框
- 点击展开显示聊天历史
- 保留语音、前端命令等全部聊天功能

### 6. CeceliaPage（重写）
- 新布局：PulseStrip → ActionZone → 三列网格 → ChatDrawer
- 保留 AmbientGlow 呼吸边框
- 保留 useCeceliaWS 和 useBriefing

## 不改的部分
- PulseStrip — 保持不变
- AmbientGlow — 保持不变
- useCeceliaWS / useBriefing hooks — 保持不变
- Brain 后端 — 不碰

## 成功标准

1. 打开页面 5 秒内能看到系统状态和待处理事项
2. 聊天区域默认折叠，不占主要视觉面积
3. 三列布局清晰展示：Agent 监控 / 今日概览 / 事件流
4. 有待决策事项时 Action Zone 自动展开，无事时隐藏
5. Vite 构建通过，无新增 TS 错误
