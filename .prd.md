# PRD: Worker 可观测性增强

## 问题

当前 worker 产生的数据"太骨架"，缺少关键字段：
- state.json 的 updatedAt 不动态更新
- summary.json 缺少 startedAt, duration, error, evidence
- result.json 太简单，缺少执行详情（特别是 skipped tests）

导致前台无法展示有意义的数据。

## 目标

修复 5 个关键字段，让数据可以支撑 Core Dashboard：

1. **state.updatedAt** - 动态刷新
2. **summary.startedAt** - 记录开始时间
3. **summary.duration** - 执行耗时（秒）
4. **summary.error** - 失败时的错误信息
5. **result 详细信息** - 包含 tests passed/failed/skipped

## 范围

**In Scope:**
- 修改 worker.sh 记录时间和计算 duration
- 修改 orchestrator/qa-run.sh 产生详细 test results
- 更新 state.json 时刷新 updatedAt
- summary.json 包含 evidence 字段

**Out of Scope:**
- 前台页面开发（下个 PR）
- Core API 开发（下个 PR）
- Heartbeat 定时任务配置（下个 PR）

## 验收标准

重新运行 worker 后：
- state.json 的 updatedAt 匹配当前时间
- summary.json 包含 startedAt, duration, error, evidence
- result.json 包含 tests.total, passed, failed, skipped 及 skipped_details
