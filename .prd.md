# PRD: 修复 heartbeat-inspector.js SQL 列名 + Docker 挂载

## 背景

Brain 的巡检模块 `heartbeat-inspector.js` 每 30 分钟执行一次，读取 HEARTBEAT.md 并结合系统快照判断是否需要行动。当前巡检每次都失败，报错：`column "type" does not exist`。

## 问题

1. `heartbeat-inspector.js` 中 `collectSystemSnapshot()` 的 SQL 查询使用了 `type` 列名，但 `cecelia_events` 表的实际列名是 `event_type`
2. 同文件末尾的 INSERT 语句也使用了 `type` 而非 `event_type`
3. prompt 构建中引用 `e.type` 应为 `e.event_type`
4. `docker-compose.yml` 需要持久化 HEARTBEAT.md 的 Docker 卷挂载配置

## 修复范围

- `brain/src/heartbeat-inspector.js`: 4 处 `type` → `event_type`
- `docker-compose.yml`: 添加 HEARTBEAT.md 卷挂载

## 验收标准

- 巡检不再报 SQL 错误
- 容器重启后 HEARTBEAT.md 可被读取
- 现有测试全部通过
