## PRD - 修复 decisions 表 schema 不匹配

**需求来源**: Brain tick 每次报错 `column "trigger" of relation "decisions" does not exist`
**功能描述**: decisions 表 schema 与 decision.js 代码期望的列不匹配。代码 INSERT (trigger, context, actions, confidence, status)，但表只有 (category, topic, decision, reason, status)。需要 ALTER TABLE 添加缺失列并放宽旧 NOT NULL 约束，使 Decision Engine (Stage 3) 正常工作。
**涉及文件**:
- `brain/migrations/009_fix_decisions_schema.sql` — 新增迁移脚本
- `brain/src/selfcheck.js` — 更新 EXPECTED_SCHEMA_VERSION 到 009
**成功标准**:
- `generateDecision({ trigger: 'tick' })` 不再报 column 错误
- decisions 表包含 trigger, context (jsonb), actions (jsonb), confidence (numeric), executed_at (timestamp) 列
- 旧列 category, topic, decision 改为 nullable（保留历史数据）
- selfcheck schema_version 升级到 009
- 现有 3 条战略决策数据不受影响
**非目标**:
- 不重构 decision.js 逻辑
- 不迁移旧数据到新列格式
