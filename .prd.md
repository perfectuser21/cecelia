# PRD: CI Gate — 新增 hook/lib 文件必须有 RCI 条目（CI L2 检查）

## 背景

当 PR 向 `packages/engine/hooks/` 或 `packages/engine/scripts/devgate/` 新增文件时，
没有任何门禁检查这些新文件是否已在 `regression-contract.yaml` 中注册 RCI 条目。
结果是新能力（新 hook、新 devgate 脚本）可以无 RCI 覆盖悄悄滑入生产，
RCI 扫描的覆盖率会因此被稀释，而不会触发任何告警。

## 目标

当 PR 新增 hooks/ 或 scripts/devgate/ 文件时，CI L2 强制要求该文件在 regression-contract.yaml 中有对应 RCI 条目。

## 成功标准

1. 新增 `check-new-files-need-rci.cjs`，导出 `scanMissingRci(addedFiles, contractContent)` 函数，扫描新增文件是否缺失 RCI 条目（require 可用）
2. CI L2 新增 `new-files-need-rci-check` job，检测 hooks/ 和 scripts/devgate/ 下新增文件是否有 RCI 覆盖
3. 三重降级保护：无新增目标文件 / 文件已有 RCI 覆盖 / 降级跳过 → 不失败
4. 非目标路径文件直接跳过（不属于 hooks/devgate 的文件不检查）
5. 测试文件覆盖率 ≥ 60%

## 实现要点

- `scanMissingRci(addedFiles, contractContent)` 接受文件路径数组和 YAML 字符串，返回缺失 RCI 的文件数组
- 目标路径：`packages/engine/hooks/` 和 `packages/engine/scripts/devgate/`
- CI 通过 `git diff --name-only --diff-filter=A origin/main...HEAD` 获取新增文件
- RCI 条目检查：扫描 contract YAML 中所有条目的 `file:` 字段，匹配文件名（相对 packages/engine/）
