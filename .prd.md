# PRD: Brain v2 Phase D1.7a — tick-state.js 抽出

## 背景

`packages/brain/src/tick.js` 已经膨胀到 2400+ 行，模块级散落 14 个 `let _lastXxxTime = 0` 节流计时器（L159-L173）外加 5 个 loop 控制态（_loopTimer/_recoveryTimer/_tickRunning/_tickLockTime/_lastConsciousnessReload）。

D1.7b 即将抽 executeTick 到 tick-runner.js（1600 行），D1.7c 要把 8 个 scheduled job 拆成 plugin。这两步都需要一个共享的、可单元测试的状态容器，否则 tickState 还得在 4-5 个文件之间到处传。

D1.7a 是先做这一步：**纯重构，零行为变化，把 19 个模块级 in-memory 变量收口为单例 `tickState` 对象**。

## 上游 PRD

`docs/design/brain-v2-c8-d-e-handoff.md` §5 D1.7+

## 成功标准

- 新建 `packages/brain/src/tick-state.js`，导出 `tickState` 单例 + `resetTickStateForTests()`
- `tickState` 包含 14 个 `lastXxxTime` 数值字段 + 5 个 loop 控制态字段（共 19 字段）
- `tick.js` 不再含 `let _lastXxxTime = 0` 模式（regex `let _last\w+Time` 不命中）
- `tick.js` 通过 `tickState.lastXxxTime` / `tickState.loopTimer` 等访问；原 `_resetLastXxxTime` 函数保留为 backwards-compat（重置 tickState.lastXxxTime）
- 所有 tick 相关 vitest 测试（tick-cleanup / tick-throttle / heartbeat-tick / tick-layer2-health / tick-state）全 pass
- 不动 executeTick 函数体（D1.7b 才做）；不动 30+ scheduled job 调用（D1.7c 才做）
