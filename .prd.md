# PRD: 统一系统时区为北京时间

## 功能描述
将 Cecelia Brain 生产环境的所有组件统一设置为北京时间（Asia/Shanghai, UTC+8），包括宿主机、Docker 容器、PostgreSQL 数据库，确保时间显示一致。

## 背景
- 当前 Brain 容器和 PostgreSQL 使用 UTC 时区
- 宿主机使用 Asia/Shanghai (CST) 时区
- 时区不一致导致日志和监控时间难以对比
- 用户习惯使用北京时间，UTC 时间戳需要手动转换

## 目标
1. Docker 容器使用北京时间
2. PostgreSQL 数据库使用北京时间
3. API 继续返回标准 UTC 时间戳（符合 ISO 8601）
4. 日志和内部处理使用北京时间

## 成功标准
- [ ] Brain 容器内 Node.js 时间显示 CST (GMT+0800)
- [ ] PostgreSQL timezone 设置为 Asia/Shanghai
- [ ] 容器重启后时区配置保持
- [ ] Brain API 正常运行，健康检查通过

## 技术方案

### Docker Compose 配置
```yaml
environment:
  - TZ=Asia/Shanghai

volumes:
  - /etc/localtime:/etc/localtime:ro
  - /etc/timezone:/etc/timezone:ro
```

### PostgreSQL 配置
```sql
ALTER DATABASE cecelia SET timezone TO 'Asia/Shanghai';
```

## 影响范围
- 配置文件：`docker-compose.yml`
- 数据库：PostgreSQL cecelia
- 容器：cecelia-node-brain

## 风险评估
- 风险：低
- 数据库时区修改需要重启连接
- Brain 容器需要重启应用新配置
