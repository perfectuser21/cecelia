## PRD - 实现执行状态实时展示 API 端点

**需求来源**: Brain 自动调度 — 前端已有 CeceliaRuns 和 RunDetail 组件，但后端缺少 `/cecelia/overview` 和 `/cecelia/runs/{runId}` API 端点，导致执行状态页面无法获取数据
**功能描述**: 在 Python FastAPI 后端创建 `cecelia_routes.py`，查询 `tasks` 表提供任务概览和详情数据，支撑前端 CeceliaRuns 和 RunDetail 页面实时展示
**涉及文件**:
- `src/api/cecelia_routes.py` (新建) — Cecelia 任务 API 路由
- `src/api/main.py` (修改) — 注册新路由
- `frontend/vite.config.ts` (无需修改) — `/api` 代理已覆盖

**API 端点设计**:

1. `GET /cecelia/overview` — 任务概览
   - 返回: `{ success, total_runs, running, completed, failed, recent_runs[] }`
   - 数据源: `tasks` 表，按 status 统计
   - recent_runs 映射字段: id, project→title, feature_branch→metadata, status, checkpoints→根据 dev step 推算

2. `GET /cecelia/runs/{runId}` — 任务详情
   - 返回: `{ success, run, checkpoints[], error }`
   - 数据源: `tasks` 表 + 推算 dev 步骤进度

**前端接口契约** (已有，不修改):
- CeceliaRuns.tsx 期望 `Overview { success, total_runs, running, completed, failed, recent_runs[] }`
- RunDetail.tsx 期望 `RunDetailResponse { success, run: TaskRun, checkpoints: Checkpoint[] }`
- TaskRun 接口: `{ id, prd_path, project, feature_branch, status, total_checkpoints, completed_checkpoints, failed_checkpoints, current_checkpoint, started_at, updated_at, completed_at, error, mode }`

**成功标准**:
- [ ] `GET /cecelia/overview` 返回有效数据，CeceliaRuns 页面正常显示
- [ ] `GET /cecelia/runs/{id}` 返回任务详情，RunDetail 页面正常显示
- [ ] 5 秒轮询不会对数据库造成过大压力
- [ ] TypeScript / Python 编译通过

**非目标**:
- 不修改前端组件（CeceliaRuns.tsx、RunDetail.tsx 已完善）
- 不实现 WebSocket 实时推送（现有 5 秒轮询足够）
- 不创建新数据库表（复用现有 tasks 表）
