## PRD - 实现执行状态实时展示组件

**需求来源**: 前端 DevTasks 和 CeceliaRuns 页面依赖的 API 端点（`/api/cecelia/overview`、`/api/dev/tasks`、`/api/dev/health`）在 Brain routes 中不存在，页面无法正常工作。需要实现后端 API 并优化前端轮询。

**功能描述**: 为 Brain 添加执行状态 API 端点，连接 executor.js 的进程追踪能力和 PostgreSQL 的任务数据，让前端能实时展示 Cecelia 执行器的运行状态。

**涉及文件**:
  - 修改: brain/src/routes.js（添加 /cecelia/overview、/dev/tasks、/dev/health、/dev/status 端点）
  - 修改: frontend/src/features/core/execution/pages/DevTasks.tsx（改用 5s 轮询 + 连接状态指示器）
  - 修改: frontend/src/features/core/execution/pages/CeceliaRuns.tsx（对齐新 API 响应格式）

**成功标准**:
  - GET /api/brain/cecelia/overview 返回 running/completed/failed 计数 + 最近 runs
  - GET /api/brain/dev/tasks 返回所有活跃 dev 任务的步骤状态
  - GET /api/brain/dev/health 返回追踪健康状态
  - DevTasks 页面显示活跃执行器任务（从 tasks 表 + activeProcesses）
  - CeceliaRuns 页面正常显示运行中/已完成/失败任务
  - 前端轮询间隔从 30s 缩短到 5s，带连接状态指示
  - 不引入新的测试失败

**非目标**:
  - 不实现 WebSocket/SSE（保持轮询，已够用）
  - 不修改 executor.js 的核心调度逻辑
  - 不新增数据库表或迁移
