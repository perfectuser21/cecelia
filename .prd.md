## PRD - 修复 similarity.js 中的 Jaccard 相似度算法

**需求来源**: Jaccard 相似度算法存在边界情况处理不当和性能问题

**功能描述**:
修复 `brain/src/similarity.js` 中的 `calculateScore` 方法，解决以下问题：

1. **除零错误**: 当 query 和 entity.text 都为空时，union.size 为 0，导致 NaN
2. **性能问题**: 使用 Array.includes() 的 O(n*m) 复杂度，应改为 Set 的 O(n) 查找
3. **大小写敏感问题**: 关键词提取后是小写，但匹配时使用原始大小写
4. **关键词加分无限制**: 可能导致分数虚高，应设置上限
5. **测试覆盖不足**: 缺少边界情况的测试

**涉及文件**:
- `brain/src/similarity.js` (修改 calculateScore 方法)
- `brain/src/__tests__/similarity.test.js` (添加边界情况测试)

**成功标准**:
1. 除零错误修复 - 空 tokens 返回 0.0 而非 NaN
2. 性能优化 - 使用 Set 进行 O(1) 查找
3. 关键词匹配修复 - 大小写不敏感
4. 关键词加分限制 - 最大 0.3
5. 所有现有测试通过
6. 新增边界情况测试通过
7. CI/CD 检查通过

**非目标**:
- 不改变算法的基本逻辑
- 不影响向后兼容性
- 不修改其他相似度计算方法
