# PRD - 实现执行状态实时展示组件

## 需求来源
任务来自 Brain 自动调度，任务类型：dev

## 功能描述
在 Core 前端添加实时显示 Cecelia 执行状态的组件，用于展示 Cecelia 任务执行的实时状态，包括：
- 当前执行的任务信息
- 执行进度（步骤）
- 执行状态（pending/running/completed/failed）
- 执行日志输出
- 任务元数据（项目、分支、PRD 等）

## 涉及文件
- `frontend/src/components/ExecutionStatus.tsx` - 新建：执行状态展示组件
- `frontend/src/services/api.ts` - 修改：添加获取执行状态的 API 调用
- `frontend/src/App.tsx` - 修改：集成执行状态组件到主界面
- `backend/src/routes/brain.ts` - 可能修改：确保提供执行状态 API 端点

## 成功标准
- [ ] 组件通过 WebSocket 实时更新状态，状态变更延迟 < 1秒；若 WebSocket 不可用则降级为每 2 秒轮询
- [ ] 组件能够通过 WebSocket 或轮询方式获取最新状态
- [ ] 显示任务的关键信息：项目名、分支、当前步骤、状态
- [ ] 能够展示执行日志（可选的详细模式）
- [ ] 使用 Core 前端统一的组件库，组件采用 Card 布局
- [ ] 支持移动端（320px-768px）、平板（768px-1024px）、桌面（>1024px）三种断点，移动端显示单列布局，桌面端显示双列布局，所有文本可读不截断

## 非目标
- 不实现任务的编辑或操作功能（仅展示）
- 不实现历史任务列表（仅展示当前/最新任务）
- 不实现复杂的筛选和搜索功能
