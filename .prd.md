# PRD: cto-review skill

## 背景

Brain 系统已有 `intent_expand` 和 `cto_review` 两阶段前置审查流程（#1083 已合并），
但 `/cto-review` skill 本体尚未创建。

## 目标

实现 `/cto-review` skill（`packages/workflows/skills/cto-review/SKILL.md`），
供 Brain 派发的 `cto_review` 类型任务在西安 Codex 执行。

## 功能描述

以 CTO 视角整体审查：enriched PRD + DoD + 核心代码 diff，
判断实现是否真正交付了用户意图。

审查维度：
1. 意图交付度：PRD 要求的有没有真正实现
2. DoD 质量：测试是否真实覆盖需求
3. 架构合规：有无越界或系统性问题

## 成功标准

- `packages/workflows/skills/cto-review/SKILL.md` 文件存在
- 包含完整的 5 步执行流程（Step 1~Step 5）
- 输出格式：`REVIEW_RESULT: PASS` 或 `REVIEW_RESULT: FAIL + FAIL_REASONS`
- 覆盖三个审查维度：意图交付度、DoD 质量、架构合规
