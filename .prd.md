# PRD: CI Gate — 新增 hook/lib 文件必须有 RCI 条目

## 背景

当 PR 新增 packages/engine/hooks/ 或 packages/engine/scripts/devgate/ 下的文件时，CI L2 检查该文件是否已在 regression-contract.yaml 中有对应 RCI 条目。防止新能力无 RCI 覆盖悄悄滑入生产。

## 成功标准

1. 新增 check-new-files-need-rci.cjs，导出 scanMissingRci 函数，接受 changedFiles 数组和 contractContent 字符串参数
2. CI L2 新增 new-files-need-rci-check step（检测 hooks/ 和 scripts/devgate/ 下新增文件）
3. 三重降级保护：scanMissingRci 空数组输入返回空数组（降级保护 1）；scanMissingRci 非目标路径返回空数组（降级保护 3）；scanMissingRci 已有 RCI 的文件返回空数组（降级保护 2）；scanMissingRci 缺少 RCI 的新文件返回该文件路径
4. 测试文件存在且格式合法（check-new-files-need-rci.test.ts）
