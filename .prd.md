# PRD: feat(brain): system_registry 表 + /api/brain/registry 接口

## 背景
在 Brain 加一张 system_registry 表，记录系统里所有东西（skill/cron/api/machine/integration）的位置和状态。加 /api/brain/registry 查询接口和 exists 检查接口。目标：Claude 创建任何东西前先查这里，彻底解决孤岛和重复问题。

## 成功标准
- `system_registry` 表已通过 migration 创建（含 name/type/location/status/description/metadata 字段）
- GET `/api/brain/registry` 支持 `?type=` / `?status=` / `?q=` 过滤查询
- GET `/api/brain/registry/exists` 支持 `?name=X&type=Y` 检查是否存在
- POST `/api/brain/registry` 支持注册新条目（upsert）
- PATCH `/api/brain/registry/:id` 支持更新状态/location
- 保持向后兼容（PRESERVE）：Brain 已有路由（server.js 中 /api/brain/tasks 挂载未被移除）
- 所有功能产出物（ARTIFACT）的运行时行为（BEHAVIOR）均经过验证，CI 门禁（GATE）全部通过

## 不做什么
不做 UI 可视化；不修改现有模块的注册逻辑（只提供表结构和查询接口）。
