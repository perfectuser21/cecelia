## PRD - MiniMax Provider 支持

**需求来源**: MiniMax Ultra plan（2000 prompts/5h）已验证可用，需要在 executor.js 接入
**功能描述**: 在 executor.js 新增 getProviderForTask() 函数，让所有无头任务默认走 MiniMax provider（成本 1/12），并将 provider 字段传递给 cecelia-bridge
**涉及文件**:
- brain/src/executor.js（新增 getProviderForTask 函数 + 修改 triggerCeceliaRun）
- brain/src/__tests__/minimax-provider.test.js（新增测试）

**成功标准**:
1. getProviderForTask() 默认返回 'minimax'
2. triggerCeceliaRun 的 bridge 请求 body 包含 provider 字段
3. 日志输出包含 provider 信息
4. getProviderForTask 从 module.exports 导出
5. 所有测试通过

**非目标**:
- 不修改 cecelia-run 或 cecelia-bridge（已完成）
- 不实现 provider 切换逻辑（后续迭代）
- 不修改数据库 schema
