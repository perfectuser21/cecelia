# PRD: P2P 对话查询能力——派任务+回调回飞书

## 背景

Cecelia 在飞书 P2P 对话里无法真正"查东西"。当用户问"zenithjoy dashboard 现在有什么"时，她只会生成"我去查一下"的空话，不会真正派出探查任务，结果永远回不来。

## 功能描述

1. orchestrator-chat.js P2P 流程里，mouth 判断需要查询时，能通过 `dispatch_query_task` 信号 dispatch 一个 exploratory/research 任务
2. 任务完成后通过 execution-callback 把结果发回原 P2P 对话（调飞书 send_message API 回复用户）
3. 任务派发时先发一条"正在查，稍等"给用户，避免无声等待

## 成功标准

- orchestrator-chat.js handleChat 函数保持正常工作，原有流程不受影响；MOUTH_SYSTEM_PROMPT 包含 dispatch_query_task 类型说明；在调用 observeChat 时传入 conversation_id
- ops.js fallback handleChat 调用传入 conversation_id: openId；thalamus.js observeChat 支持 dispatch_query_task 信号，在 tasks 表创建含 p2p_callback 的任务
- execution.js 在任务完成/失败后检查 p2p_callback 并发飞书消息；notifier.js 导出 sendFeishuToOpenId 函数，openId 为空时优雅降级返回 false（不抛异常）
- 核心模块关键导出存在，无明显结构缺失
