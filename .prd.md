# 修复孤儿进程泄漏

## 功能描述

修复 cecelia-run 和 zombie-sweep 中的孤儿进程泄漏 bug。

## 成功标准

cecelia-run 任务结束后不再残留 subagent 孤儿进程。

## 问题

cecelia-run 任务完成/超时后，claude subagent 进程变成孤儿（ppid=1），持续消耗 token 和内存。

### 根因

1. **cecelia-run cleanup 只杀 PGID**：`kill -TERM -PGID` 只杀同一进程组的进程，但 claude 的 subagent 和 Bash tool 启动的子进程会创建新的 process group，不在 PGID 范围内。

2. **zombie-sweep 匹配不到 subagent**：`pgrep -f 'claude.*-p'` 只匹配带 `-p` 参数的主 claude 进程，subagent 的命令是光秃秃的 `claude`，被漏掉。

### 影响

- 孤儿进程持续调 Anthropic API 消耗 token
- 占用内存导致 swap 95%
- 实测发现 9 个孤儿进程，最老的跑了 35 小时

## 修复方案

### 1. cecelia-run — 递归杀进程树

把 cleanup 函数和任务结束后的清理逻辑从 `kill -TERM -PGID` 改为递归 `pgrep -P` 杀整个进程树。

### 2. zombie-sweep.js — 扩大搜索范围

把 `sweepOrphanProcesses` 的搜索从 `pgrep -f 'claude.*-p'` 改为搜索所有 claude 进程，然后正确排除合法进程。
