## PRD - 修复 tick.js 中 getDispatchRate 未定义错误

**需求来源**: 紧急修复 - Cecelia 心跳系统无法执行

**背景**: 在尝试手动触发 tick 时，系统返回错误：
```
{"success":false,"error":"getDispatchRate is not defined","source":"manual"}
```

**问题分析**:
- `brain/src/tick.js` Line 1213 调用了 `getDispatchRate()`
- 但该函数未定义在当前作用域
- Line 18-19 导入了两个版本的 alertness 函数：
  - `getDispatchRateOld` from './alertness.js'
  - `getDispatchRateEnhanced` from './alertness/index.js'
- 其他地方（Line 791, 1198）使用的是 Enhanced 版本

**功能描述**:
修复 `getDispatchRate()` 函数调用错误，改为使用正确的 `getDispatchRateEnhanced()` 函数。

**涉及文件**:
- `brain/src/tick.js` - Line 1213

**技术方案**:
```javascript
// 修改前
const dispatchRate = getDispatchRate();

// 修改后
const dispatchRate = getDispatchRateEnhanced();
```

**成功标准**:
1. ✅ tick.js 代码修复完成
2. ✅ 手动触发 tick 不再报错
3. ✅ 相关测试通过
4. ✅ CI 通过

**非目标**:
- ❌ 不改动 alertness 系统的其他逻辑
- ❌ 不重构 tick.js 的其他部分
- ❌ 不修改函数签名或行为

**优先级**: P0（阻塞 Cecelia 心跳系统运行）

**预计影响范围**: 仅影响 tick 执行时的 dispatch rate 计算逻辑
