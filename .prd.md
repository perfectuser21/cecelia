## PRD - 修复 tsconfig 路径配置和缺失 workers.config.json

**需求来源**: Brain 自动调度 - tsconfig.json 的 @features/core/* 指向不存在的路径，与 vite 配置不一致。workers.config.json 被 import 但文件不存在。
**功能描述**:
1. 修复 tsconfig.json 的 @features/core/* 路径别名，使其与 vite.config.ts 一致（指向 src/features/core）
2. 将 workers.config.json 移入 git 管理的目录（features/core/execution/config/ 旁），修复 workers.config.ts 的 import 路径
3. 对齐 JSON 数据结构与 TypeScript 类型定义（JSON 用 teams，类型用 departments）

**涉及文件**:
  - 修改: frontend/tsconfig.json（修正 @features/core/* 路径）
  - 新建: frontend/src/features/core/execution/config/workers.config.json（从 untracked features/data/ 移入）
  - 修改: frontend/src/features/core/execution/config/workers.config.ts（修正 import 路径）
**成功标准**:
  - tsconfig.json 的 @features/core/* 路径与 vite.config.ts 一致
  - workers.config.json 存在于 git 管理的路径下
  - workers.config.ts 能正确 import workers.config.json
  - JSON 数据结构与 WorkersConfig TypeScript 类型定义匹配
  - `npm run build` 在 frontend 目录下不因这些文件报错
**非目标**:
  - 不修改 workers 业务逻辑
  - 不修改 vite.config.ts（它已经是正确的）
  - 不修改其他 tsconfig 路径别名
