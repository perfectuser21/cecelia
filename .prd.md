# PRD - 实现代码质量扫描基础设施

## 背景

自主任务生成机制的第一阶段，需要建立代码质量扫描基础设施。

## 具体需求

### 1. 扫描框架搭建
- 创建 `packages/brain/src/task-generators/` 目录
- 实现基础扫描器接口（Scanner Interface）
- 支持定期扫描调度（集成到 Brain tick）

### 2. 代码覆盖率扫描
- 集成现有测试覆盖率工具
- 识别覆盖率低于 70% 的模块
- 生成"提高XX模块覆盖率"类型任务

### 3. 代码复杂度分析
- 使用复杂度分析工具（如 complexity-report）
- 识别圈复杂度 > 10 的函数
- 生成"简化XX函数复杂度"类型任务

### 4. 未测试模块检测
- 扫描无测试文件的源代码模块
- 识别关键业务模块是否缺少测试
- 生成为XX模块添加测试类型任务

## 成功标准

- [ ] 扫描框架可运行并输出扫描结果
- [ ] 每天自动扫描并生成至少1个代码质量相关任务
- [ ] 生成的任务包含：模块路径、当前指标、目标指标
- [ ] 扫描结果存储到数据库（tasks 表或独立表）

## 技术方案

### 目录结构

```
packages/brain/src/task-generators/
├── index.js                 # 统一导出
├── base-scanner.js          # 基础扫描器接口
├── coverage-scanner.js      # 代码覆盖率扫描器
├── complexity-scanner.js    # 代码复杂度扫描器
├── untested-scanner.js      # 未测试模块检测器
└── scheduler.js             # 扫描调度器
```

### Scanner Interface

```javascript
class BaseScanner {
  async scan()     // 执行扫描，返回问题列表
  getName()       // 扫描器名称
  getThreshold()  // 获取阈值配置
  async generateTask(issue) // 将问题转换为任务
}
```

### 数据库设计

```sql
-- 扫描结果表
CREATE TABLE scan_results (
  id SERIAL PRIMARY KEY,
  scanner_name VARCHAR(100) NOT NULL,
  module_path VARCHAR(500),
  issue_type VARCHAR(100),
  current_value FLOAT,
  target_value FLOAT,
  task_id INTEGER REFERENCES tasks(id),
  scanned_at TIMESTAMP DEFAULT NOW()
);
```

### 集成 Brain tick

在 Brain tick 中添加扫描调度逻辑：
- 每天首次 tick 时触发扫描
- 或者每 N 次 tick 执行一次扫描
- 扫描结果转换为任务，插入 tasks 表
