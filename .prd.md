## PRD - 修复免疫系统 P1 断链

**需求来源**: 免疫系统审计发现剩余 3 个 P1 级别断链需要修复

**功能描述**:
修复 Brain 免疫系统的 3 个 P1 级别断链，完善免疫系统的限流、重试策略和隔离管理：

1. **Token bucket 没接**: `tryConsumeToken()` export 了但 dispatch 里没调用，限速形同虚设
2. **失败分类不用**: 分类了 BILLING_CAP/RATE_LIMIT/NETWORK 等，但 retry_strategy 没人读，任务不会按策略重试
3. **隔离区无自动释放**: 进去了只能手动放出来，没有定时重试机制

**涉及文件**:
- `brain/src/tick.js` - 在 dispatch 入口调用 tryConsumeToken()
- `brain/src/executor.js` - triggerCeceliaRun() 读取并应用 retry_strategy
- `brain/src/quarantine.js` - 实现隔离区 TTL 和自动释放机制
- `brain/src/__tests__/alertness-token-bucket.test.js` - Token bucket 限速测试
- `brain/src/__tests__/executor-retry-strategy.test.js` - Retry strategy 应用测试
- `brain/src/__tests__/quarantine-auto-release.test.js` - 隔离区自动释放测试

**成功标准**:
1. Dispatch 超过 QPS 限制时返回 `rate_limited`，任务进入延迟队列
2. BILLING_CAP 错误任务自动延迟到 reset 时间后重试
3. NETWORK 错误任务使用指数退避重试
4. 隔离区任务到期后自动进入 HALF_OPEN 探测，成功则释放
5. 所有新增测试通过
6. CI 通过

**非目标**:
- 不实现复杂的自适应限流算法
- 不修改现有 failure_count 计数逻辑
- 不重构整个 retry 系统架构
