# PRD: Tick Loop 诊断集成测试

branch: cp-04011104-integration-tick

## 背景
Tick loop 是 Brain 核心调度循环（每5秒运行一次），负责扫描 queued 任务并派发到 executor。
这条链路从未被真实集成测试覆盖，单元测试全部 mock executor，两者接口契约从未被真实验证。

## 目标
建立诊断型集成测试，记录 tick 循环的真实行为。发现 bug 时记录"实际行为 vs 预期行为"而非直接断言通过。

## 功能描述
1. 验证 POST /api/brain/tick 端点基础连通性
2. 创建 queued 任务 → 触发 tick → 检查状态是否变为 in_progress
3. 验证连续两次 tick 的幂等性（不重复 dispatch 同一任务）
4. 检查 cecelia_events 是否记录 tick 执行痕迹
5. 验证 GET /api/brain/tick/status 可观测字段完整性

## 成功标准
- tests/integration/tick.integration.test.ts 存在，含 ≥2 个测试用例
- 测试记录了 tick 的真实状态流转行为（包括 no_executor 回退路径）
- 无 vi.mock（真实集成测试，直接调 Brain HTTP API + 真实 DB）
- status: pending-ci（暂不接入 CI gate，作为诊断工具）
