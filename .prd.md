# PRD: Brain v2 Phase D1.7c-plugin1 — 4 scheduled job 拆 plugin

## 背景

D1.7b（PR #2625，已合）把 `executeTick` 函数体（~1626 行）从 `tick.js` 抽到 `tick-runner.js`，纯重构。
D1.7c 进一步拆分：把 tick-runner.js 内的 8 个 inline scheduled job 段落抽成独立 plugin 文件。

D1.7c 由两个 builder 并行：
- **plugin1（本 PR）**：dept-heartbeat / kr-progress-sync / heartbeat / goal-eval（4 个）
- **plugin2（PR #2626，已合）**：pipeline-patrol / pipeline-watchdog / kr-health-daily / cleanup-worker（4 个）

每个 plugin export 标准接口 `tick(now, tickState)`（或 plugin2 的 `tick(ctx)`），自带节流门、错误兜底、依赖注入测试支持。

## 上游 PRD

- 设计：`docs/design/brain-v2-c8-d-e-handoff.md` §5 D1.7c
- 团队：`brain-v2-d-team` Task #3（builder-3 完成）

## 成功标准

- 新建 3 个 plugin 文件：
  - `packages/brain/src/kr-progress-sync-plugin.js`（CLEANUP_INTERVAL_MS 节流，跑 kr-verifier + kr-progress fallback）
  - `packages/brain/src/heartbeat-plugin.js`（HEARTBEAT_INTERVAL_MS 节流，包装 runHeartbeatInspection；**仅成功后更新 lastHeartbeatTime**）
  - `packages/brain/src/goal-eval-plugin.js`（GOAL_EVAL_INTERVAL_MS=24h 节流，先 mark 后跑）
- 扩展 `packages/brain/src/dept-heartbeat.js`：增加 `tick(now, tickState, opts)` plugin 接口
  - 内部 dynamic import `consciousness-guard`，false 时跳过；true 时调 triggerDeptHeartbeats(pool)
  - 错误吞掉返回 `{triggered:0, skipped:0, results:[]}`，不向上抛
- `tick-runner.js` 4 处 inline scheduled job 改为 `await xxxPlugin.tick(now, tickState)` 形式
- import 命名：`deptHeartbeatPlugin / krProgressSyncPlugin / heartbeatPlugin / goalEvalPlugin`（namespace import）
- 4 个新 plugin 单测全 pass（38 个 case 覆盖：导出形状、节流门、成功路径、错误吞噬、时间戳更新规则）
- 老测试 `harness-pipeline.test.ts > tick-runner.js: triggerDeptHeartbeats 被 isConsciousnessEnabled() 门控` 同步到新位置（断言现移到 dept-heartbeat.js plugin tick 内）
- 与 plugin2 并行：本 PR 改 tick-runner.js 不同段落（imports / kr-progress-sync / goal-eval / heartbeat / dept-heartbeat-call），不冲突

## 风险与缓解

- **行为漂移**：plugin tick 必须保持原 inline 严格语义。挑战：goal-eval/kr-progress 是「先 mark 时间戳再 try」（即使 inner error 也不立即重试），heartbeat 是「仅成功后更新时间戳」。两套语义被显式测试断言锁定，避免改写时漏。
- **dept-heartbeat 守护位置**：原代码守护 `isConsciousnessEnabled()` 在 tick-runner.js 内联；D1.7c 后守护必须搬到 plugin 内部，否则 plugin 直接调用会绕过守护。本 PR 在 dept-heartbeat.js plugin tick 顶部 dynamic import + 守护 + 早返回，与原行为等价。
- **plugin2 冲突**：两个 builder 同改 tick-runner.js 不同段落。冲突点仅可能在 imports section（line 49 vs 87）和已删常量段。本地 rebase main 解决。
- **dynamic import 成本**：每个 plugin tick() 内部 dynamic import 模块。tick 主循环 2 分钟一次，import cache 后 cost 可忽略。
