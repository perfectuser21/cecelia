# PRD: [SelfDrive] 修复错误黑洞：错误传播、重试与可观测性

## 背景

Brain task ID: 9ec0e68d-5281-410c-92f2-72b5c58c9acd
Branch: cp-03210904-9ec0e68d-5281-410c-92f2-72b5c5

## 问题描述

当前系统存在以下静默失败路径：

1. **exit_code=1 不触发重试**：`classifyDevFailure()` 无法识别 `result.failure_class` 字段，
   导致 `code_error` 类失败返回 UNKNOWN → retryable: false，retry_count=0 的任务直接沉底。

2. **Dev 重试不同步 DB retry_count**：只更新 `payload.retry_count`，不更新 `tasks.retry_count`，
   max_retries 检查失效。

3. **进程日志未纳入 error_message**：result=null 时调试信息缺失。

## 功能描述

为当前高失败率问题补齐错误上报、重试策略和链路日志，减少静默失败，提升整体成功率与 MTTR。

## 成功标准

- `classifyDevFailure` 在 `result.failure_class='code_error'` 时返回 retryable: true
- `classifyDevFailure` 在 `result=null + exitCode=1` 时返回 code_error 可重试结果
- dev 重试路径的 UPDATE SQL 包含 `retry_count = retry_count + 1`
- result=null 时 error_message 尝试读取进程日志并追加
- 所有现有测试通过
