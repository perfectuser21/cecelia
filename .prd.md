# PRD: 发布决策层 — Fast/Safe Lane 风险分级

**Task ID**: a535dd31-a1f8-4b9a-812a-61d813eff5fe
**Branch**: cp-04052027-engine-fast-safe-lane

## 背景

`deploy.yml` 目前所有 PR merge 后走同一条路直接 deploy production，不区分风险高低。
Brain 核心文件（thalamus/tick/executor/migrate 等）改动与普通配置改动享受同等待遇，没有缓冲。

## 目标

在 `deploy.yml` 的 `changes` job 中加入风险等级检测，自动输出 `risk_level=high/low`，并据此分流：

- **Fast Lane**（low）：当前行为不变，自动 deploy
- **Safe Lane**（high）：deploy job 进入等待，输出明确告警，要求手动重触发或审批

## 高风险路径判断规则

以下路径改动 → `risk_level=high`：

```
packages/brain/src/thalamus.js
packages/brain/src/tick.js
packages/brain/src/executor.js
packages/brain/src/task-router.js
packages/brain/src/migrate.js
packages/brain/src/migrations/
packages/brain/src/selfcheck.js
packages/brain/src/routes/ops.js
```

其他所有路径 → `risk_level=low`

## 实现方案

1. `changes` job 新增 `risk_level` output，检测上述路径
2. 新增 `risk_gate` job：
   - 依赖 `changes` 输出
   - Fast Lane：直接 pass，输出绿色提示
   - Safe Lane：输出红色告警 + `::error::` annotation，job 失败，阻止 deploy 自动运行
3. `deploy` job 依赖 `risk_gate`（需要 risk_gate 通过才执行）

Safe Lane 失败时，告警信息说明：如确认改动安全，在 commit message 中加 `[SAFE-DEPLOY]` 后 push 空 commit 重触发，或手动运行 `brain-deploy.sh`。

## 成功标准

- 改动核心文件的 PR merge 后，risk_gate job 失败，deploy job 不执行
- 改动非核心文件的 PR merge 后，risk_gate pass，行为与现在一致（自动 deploy）
- 如 commit message 含 `[SAFE-DEPLOY]`，则强制 Fast Lane（bypass 机制）

## 不做的事

- 不改 Brain src 代码
- 不建真实 staging 环境（P2）
- 不引入 GitHub Environment protection rules（需要 repo 设置，成本高）
