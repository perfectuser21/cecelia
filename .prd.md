# PRD: 接通 content pipeline executor 的 Claude 调用

## 背景
当前 content-pipeline-executors.js 中，copywriting/copy-review/generate/image-review 四个阶段没有调用 Claude，而是用本地模板或静态规则。配置页面（ContentTypeConfigPage）里设置的 generate_prompt、review_prompt 等 prompt 根本没有被用到。

## 工作内容
1. executeCopywriting：用 typeConfig.template.generate_prompt + findings 调用 callLLM（Brain 统一 LLM 调用层），生成真实文案（社媒图文 + 公众号长文）
2. executeCopyReview：用 typeConfig.template.review_prompt + 文案内容调用 callLLM，返回每条 review_rule 的评分和 pass/fail（rule_scores 字段）
3. executeGenerate：用 typeConfig.template.generate_prompt 调用 callLLM，生成 SVG 卡片内容描述，写入 cards/llm-card-content.json
4. executeImageReview：用 typeConfig.template.image_review_prompt 或 review_prompt 调用 callLLM，审核内容质量
5. 第二次 rerun 时，把上一次的 review_feedback（task.payload.previous_feedback）带入 copywriting prompt

## 约束
- callLLM 调用失败时，各 executor 降级到原有本地逻辑，不中断 pipeline
- 不修改 executeResearch 和 executeExport
- 测试文件：tests/content-pipeline-llm.test.js

## 成功标准

- callLLM 已接入 executor：executeCopywriting/executeCopyReview/executeGenerate/executeImageReview 四个阶段均通过 callLLM 调用 Claude
- executeCopywriting 调用 callLLM 生成真实文案，生成结果写入 cards/copy.md 和 article/article.md，不是静态模板拼接
- executeCopyReview 返回 rule_scores 逐条评分（每条规则的评分），而非仅 issues 数组；找不到目录时返回 review_passed=false
- executeCopywriting 第二次 rerun 时，若携带 previous_feedback，prompt 中包含该 feedback 内容
- executeGenerate 调用 callLLM 生成卡片内容描述，写入 llm-card-content.json；返回 image_count 和 image_style 字段
- executeImageReview 调用 callLLM 审核内容质量，返回具体意见和质量分
- callLLM 失败时各 executor 降级到原有静态逻辑，不中断 pipeline；executeCopywriting 降级路径不报错（success: true）
- prompt 修改后，下次执行立即生效（无缓存）
