# PRD: quickRoute 扩展 - DEPARTMENT_REPORT + EXCEPTION_REPORT

## 背景

当前 quickRoute 未覆盖 DEPARTMENT_REPORT 和 EXCEPTION_REPORT，这两类事件有固定的快速处理规则，不需要 LLM 判断（低/中等严重度的情况）。

## 功能描述

为 DEPARTMENT_REPORT 和 EXCEPTION_REPORT 两类事件添加快速路由规则，避免无谓的 LLM 判断。DEPARTMENT_REPORT 直接归档，EXCEPTION_REPORT 按严重度分支处理。

## 目标
- DEPARTMENT_REPORT → log_event + archive（记录并归档），confidence: 0.9
- EXCEPTION_REPORT（severity: low/medium）→ log_event + analyze_failure，confidence: 0.85
- EXCEPTION_REPORT（severity: high/critical）→ null（交给 Sonnet/Opus 深度分析）

## 实施要求
- 修改 brain/src/thalamus.js 中的 quickRoute 函数
- 为每个新增分支添加单元测试（thalamus.test.js）
- 确保 facts-check 和 DevGate 通过

## 验收条件

- [ ] quickRoute 新增 EXCEPTION_REPORT 低/中严重度分支

- [ ] 单元测试覆盖所有新增分支（低、中、高三种 severity）
