## PRD - 执行状态实时展示组件

**需求来源**: Brain executor 派发任务后，前端缺少实时展示每个任务执行进度的组件。现有 SeatsStatus 只显示席位占用数量，不显示每个任务的详细执行状态（任务名、运行时长、CPU/内存、日志尾行等）。
**功能描述**: 增强 SeatsStatus 页面，添加实时执行详情面板，展示每个正在运行的 Claude 进程对应的任务信息、资源占用和执行时长，并显示今日执行历史记录。
**涉及文件**:
  - 修改: frontend/src/features/core/brain/pages/SeatsStatus.tsx（主要改动：添加执行详情面板和历史记录）
  - 修改: frontend/src/api/brain.api.ts（添加 execution-history 和 vps-slots API 封装）
  - 修改: brain/src/routes.js（增强 vps-slots 端点返回 task 详情，添加 active-executions 端点）
**成功标准**:
  - SeatsStatus 页面显示每个运行中任务的名称、优先级、运行时长、CPU/内存使用
  - 运行时长实时计时（每秒更新，不依赖 API 轮询）
  - 显示今日执行历史（成功/失败/运行中统计）
  - vps-slots API 返回任务详情（关联 activeProcesses Map 中的 task 信息）
  - 5 秒轮询间隔保持不变
  - 不引入新的测试失败
**非目标**:
  - 不添加 WebSocket/SSE（保持轮询架构）
  - 不修改 executor.js 核心逻辑
  - 不修改 N8N ExecutionStatus 页面
