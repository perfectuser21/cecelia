# PRD: 02-code.md 本地测试改为精准测试（去全量，去重复）

## 背景

当前 Stage 2 在 2.3.1 和 2.3.7 各跑一次全量 `npm test`。Brain 有 100+ 个测试，全量运行导致 Mac mini 内存不足（OOM），机器崩溃重启。传统团队做法是本地只跑与改动相关的测试文件，全量回归交给 CI（GitHub Actions ubuntu-latest）。

## 目标

修改 `packages/engine/skills/dev/steps/02-code.md`，将两处全量 `npm test` 合并为一处精准测试（仅跑改动相关文件）。

## 成功标准

1. 02-code.md 的 2.3.1 节改为精准测试逻辑（查找改动相关测试文件 → `vitest run`，找不到则跳过）
2. 02-code.md 的 2.3.7 节删除重复的全量 npm test，不再独立运行全量测试
3. 精准测试推导逻辑正确：通过 changed files 推导对应测试文件，无对应测试文件时自动跳过（exit 0 不报错），全量由 CI 负责
4. 原有 TDD 红绿灯结构（2.1.5 TDD先行）保持不变，不被破坏
5. Engine 版本已 bump（VERSION 与 package.json 同步）
6. feature-registry.yml 新增本次变更的 changelog 条目（targeted-local-test）

## 不做什么

- 不修改 CI 流程
- 不修改 TDD 红绿灯阶段（那个跑单个文件，是合理的）
