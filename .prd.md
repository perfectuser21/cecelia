# PRD: 内容工厂 — 内容类型绑定 notebook_id + Pipeline 步骤详情视图

## 背景
内容工厂 Pipeline 有两个核心体验缺失：
1. 每次新建 Pipeline 需要手动填 NotebookLM ID，应改为从内容类型配置自动带入
2. Pipeline 只显示整体状态色块，看不到每个阶段的输入/输出/提示词/失败原因

## 需求

### A. 内容类型绑定 notebook_id（ZenithJoy 前端 + Cecelia 后端）
- solo-company-case 等 packages/brain/src/content-types/solo-company-case.yaml 配置加 notebook_id 字段
- ContentTypeConfigPage.tsx 存在并含 notebook_id 编辑功能（ContentTypeConfigPage 注册到路由）
- 新建 Pipeline 时选内容类型后自动从配置带入 notebook_id

### B. Pipeline 步骤详情视图（ZenithJoy 前端）
- Pipeline 卡片点击展开显示 stages（Pipeline 卡片展开后显示真实 stage 状态）
- 每个 stage 显示：状态/耗时/失败原因（含 summary 字段）
- ContentFactory 在内容类型切换时自动 fetch config 并填入 notebook_id

## 成功标准
- solo-company-case.yaml 含 notebook_id 字段（packages/brain/src/content-types/solo-company-case.yaml）
- POST /api/brain/pipelines 接受 notebook_id 并写入 payload
- GET /api/brain/pipelines/:id/stages 返回含 summary 字段
- ContentFactory 在内容类型切换时自动 fetch config 并填入 notebook_id
- Pipeline 卡片点击展开显示 stages
- ContentTypeConfigPage.tsx 存在并含 notebook_id 编辑功能
- ContentTypeConfigPage 注册到路由（routes[] + components{}）
- 所有现有测试通过（GATE）
