# Task Card: /decomp skill PR 数量校准表 + Initiative 技术调研步骤

## 需求（What & Why）
**功能描述**: 在 /decomp skill（packages/workflows/skills/decomp/SKILL.md）中新增两项内容：
1. 拆解粒度校准表：明确 Task/Initiative/Scope/Project 各层级的 PR 数量、周期和判断标准，附反例说明
2. Initiative 技术调研步骤：规定每个 Initiative 的第一个 Task 应为技术调研（WebSearch），输出技术选型报告

**背景**: 当前 /decomp 缺少明确的粒度校准标准，导致拆解结果可能出现层级错配（如把 Scope 级别的工作量标记为 Initiative）。同时 Initiative 执行缺少技术调研前置步骤，容易直接开干而踩坑。

**不做什么**: 不修改 Phase 2/3/4 的执行逻辑，不改数据库 schema。
