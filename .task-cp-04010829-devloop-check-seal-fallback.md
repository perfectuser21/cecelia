## 背景

`.dev-mode` 文件消失时（会话压缩、worktree GC 等场景），`devloop-check.sh` 的直接执行入口（`devloop_check_main()`）会直接输出 `NO_ACTIVE_SESSION` 并退出，完全失明——无法告知 agent 当前处于哪个 stage、缺少什么、下一步应该做什么。

但 seal 文件现在已经 commit 进分支（P0 已修），可以从 seal 文件反推步骤状态：
- `.dev-gate-planner.{branch}` 存在 → step_1 已完成
- `.dev-gate-crg.{branch}` 存在 → step_2 已完成
- PR 已在远端创建（via `gh pr list`）→ step_3 已完成

## 目标

当 `.dev-mode` 不存在时，`devloop_check_main()` 从 seal 文件反推状态，输出明确的"当前推断 stage + 缺什么 + 下一步"，不误报 step_1 pending。

## 涉及文件

- `packages/engine/lib/devloop-check.sh`（修改 `devloop_check_main()` 函数，在无 .dev-mode 时添加 seal 反推逻辑）

## 实现方案

在 `devloop_check_main()` 的 `dev_mode_files` 为空分支（第 699-705 行）中，添加 seal 文件反推逻辑：
1. 扫描当前 worktree 目录中的 seal 文件（`.dev-gate-planner.*`、`.dev-gate-crg.*`、`.dev-gate-lite.*`、`.dev-gate-generator.*`）
2. 根据存在的 seal 文件推断已完成的 stage
3. 用 `gh pr list` 检测 PR 状态推断 step_3
4. 输出"推断状态 + 缺失信息 + 下一步 action"

## 成功标准

- `.dev-mode` 不存在时 `devloop-check.sh` 仍能输出正确 stage 推断
- 有 seal 文件时不误报 step_1 pending
- CI 通过
