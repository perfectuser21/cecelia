# PRD: verify-step.sh 和 check-dod-mapping.cjs 的 4-Stage Pipeline 兼容性修复

## 背景
在新的 4-Stage Pipeline 中，Stage 1 (Spec) 只写 Task Card 和 DoD 条目，DoD 验证在 Stage 2 才做。但 verify-step.sh 的 verify_step1() 调用 check-dod-mapping.cjs，该脚本要求所有 DoD 条目已勾选 [x]，导致 Stage 1 无法标记完成。

## 目标
让 check-dod-mapping.cjs 支持 `--format-only` 模式，只检查 DoD 格式（Test 字段存在、命令格式正确），不检查是否已勾选。verify-step.sh 在 Stage 1 使用此模式。

## 成功标准
- check-dod-mapping.cjs 接受 `--format-only` 参数，跳过未勾选检查
- verify-step.sh Stage 1 Gate 1 使用 `--format-only` 模式
- CI push 后仍做完整检查（不受影响）
- Engine 版本 bump 到 13.7.0
