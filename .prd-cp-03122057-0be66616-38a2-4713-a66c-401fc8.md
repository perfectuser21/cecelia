## PRD - feat(cortex): 添加 LLM 调用详细计时与 prompt token 估算日志

**需求来源**: Brain 自动调度任务
**功能描述**: 在 cortex.js 的 callLLM 调用前后记录关键可观测性指标，包括 prompt 估算 token 数、响应耗时、超时状态等，存入 decision_log.metadata 并提供统计 API 端点
**涉及文件**:
- `packages/brain/src/cortex.js` — 添加计时和 token 估算逻辑
- `packages/brain/src/server.js` — 新增 /api/brain/cortex/stats 端点
- `packages/brain/tests/cortex-timing.test.js` — 单元测试（新建）

**非目标**:
- 不调用外部 tokenizer API
- 不修改 cortex 的决策逻辑
- 不添加新的数据库表（使用现有 decision_log.metadata JSON 字段）

## 成功标准

- [ ] 每次 Cortex callLLM 调用后，decision_log 中的 metadata 字段包含 `prompt_tokens_est`、`response_ms`、`timed_out`、`error_type` 四个字段
- [ ] prompt_tokens_est 使用字符数 ÷ 4 快速估算（无需外部 API）
- [ ] /api/brain/cortex/stats 端点返回最近 24h 的调用总数、超时率、平均响应时间
- [ ] 单元测试覆盖计时逻辑和 token 估算函数
- [ ] 不影响现有 Cortex 调用路径的正常功能

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| 计时逻辑抛出异常 | 忽略，不中断 LLM 调用 | P3 | cortex_timing_error |
| decision_log metadata 写入失败 | 忽略，记录 warn 日志 | P3 | cortex_metadata_write_fail |
| /cortex/stats 查询超时 | 返回空数据 + 错误提示 | P3 | cortex_stats_query_fail |
