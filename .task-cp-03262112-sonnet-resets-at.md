# Task Card: account_usage_cache 新增 seven_day_sonnet_resets_at

## 需求（What & Why）

**功能描述**: Brain 的 `account_usage_cache` 表缺少 `seven_day_sonnet` scope 的独立 `resets_at` 字段。Anthropic API 对 five_hour、seven_day、seven_day_sonnet 三个窗口各自返回独立的 `resets_at`，但当前代码只存了前两个（`resets_at` + `seven_day_resets_at`），丢掉了 sonnet 的。

**背景**: 用户在使用 `/llm-quota` skill 查看账号用量时发现 Sonnet 的重置时间不正确——之前是把 7d all 的重置时间当作 Sonnet 的显示，但实际上它们是不同的。

**不做什么**: 不改 selectBestAccount 逻辑；不改前端 Dashboard。
