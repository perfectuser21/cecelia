## 实现方案

**要改的文件**:
1. `packages/brain/src/routes/task-goals.js` — 主路由：goals → objectives + key_results
2. `packages/brain/src/routes/task-projects.js` — 主路由：projects → okr_projects
3. `packages/brain/src/__tests__/routes/task-goals.test.js` — 更新 SQL 断言
4. `packages/brain/src/__tests__/routes/task-projects.test.js` — 更新 SQL 断言

**核心设计**:
- `task-goals.js` GET / : UNION ALL objectives + key_results；type 参数决定查单表还是 UNION
- `task-goals.js` GET /:id: 先查 objectives，未找到再查 key_results
- `task-goals.js` PATCH /:id: 先更新 objectives，0 行受影响再更新 key_results
- `task-goals.js` GET /audit: key_results LEFT JOIN okr_projects → okr_scopes → okr_initiatives
- `task-projects.js` GET /: FROM okr_projects，kr_id 直接过滤（非 project_kr_links 子查询）
- `task-projects.js` GET /:id: okr_projects，返回 id, title, NULL AS description, kr_id, NULL AS goal_id
- `task-projects.js` PATCH /:id: UPDATE okr_projects，name 字段映射为 title

**列映射**:
- objectives: id, title, status, area_id, owner_role, start_date, end_date, metadata, custom_props
- key_results: id, objective_id(→parent_id), title, status, area_id, owner_role, start_date, end_date, target_value, current_value, unit, metadata, custom_props
- okr_projects: id, kr_id, title, status, area_id, owner_role, start_date, end_date, metadata, custom_props
