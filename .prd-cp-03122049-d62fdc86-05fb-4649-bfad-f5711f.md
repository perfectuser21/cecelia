## PRD - Cortex 超时原因分类器

**需求来源**: Brain 自动调度 — 超时原因分类器，捕获并标注 quota / token / network 三类根因
**功能描述**: 在 cortex.js 的 LLM 调用错误处理中，添加 classifyTimeoutReason(error) 函数，输出结构化的 timeout_reason 字段并写入日志和任务 metadata
**涉及文件**:
- `packages/brain/src/cortex.js` — 修改 LLM 调用 try/catch，添加分类器
- `packages/brain/__tests__/cortex-timeout-classifier.test.js` — 单元测试（新增）

**非目标**:
- 不修改 thalamus.js 或其他大脑器官
- 不新增外部依赖
- 不修改数据库 schema（last_error 字段已存在）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| HTTP 429 / quota 耗尽 | 分类为 quota_exhausted，写日志 | P1 | cortex_quota_exhausted |
| HTTP 400 / context 超长 | 分类为 prompt_too_long，写日志 | P2 | cortex_prompt_too_long |
| 请求超时无响应 | 分类为 network_timeout，写日志 | P1 | cortex_network_timeout |
| 其他未知错误 | 分类为 unknown，写日志 | P2 | cortex_timeout_unknown |

## 成功标准

- cortex.js callLLM 错误处理增加 classifyTimeoutReason(error) 函数
- timeout_reason 枚举：quota_exhausted / prompt_too_long / network_timeout / unknown
- 每次 Cortex 失败时，在 Brain 日志中输出含 timeout_reason 的结构化 JSON
- 单测覆盖 classifyTimeoutReason 三种场景（quota/token/network）
- CI 全通过
