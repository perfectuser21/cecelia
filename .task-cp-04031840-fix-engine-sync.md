# Task Card: 修复 Engine 三处版本/文件同步问题

## 需求（What & Why）
**功能描述**: 修复审计发现的 3 个已确认真实问题
**背景**: engine 经历大规模重构（v13.78.x 清理 + v14.0.0 CI重设计 + v14.1.x Harness v2.0），但版本同步文件和 CHANGELOG 未跟上
**不做什么**: 不改逻辑代码，不修复未验证的 P1 问题（flock/realpath等）

## 成功标准

- hooks/VERSION 与 package.json 版本一致（均为 14.1.1）
- regression-contract.yaml 存在且版本字段为 14.1.1
- CHANGELOG.md 补全 v13.4.0 到 v14.1.1 的里程碑记录

## 验收条件（DoD）

- [x] [BEHAVIOR] hooks/VERSION 内容等于 14.1.1
  Test: manual:node -e "const v=require('fs').readFileSync('packages/engine/hooks/VERSION','utf8').trim();if(v!=='14.1.1')process.exit(1);console.log('OK:',v)"

- [x] [ARTIFACT] packages/engine/regression-contract.yaml 存在
  Test: manual:node -e "require('fs').accessSync('packages/engine/regression-contract.yaml');console.log('OK')"

- [x] [BEHAVIOR] regression-contract.yaml 版本字段为 14.1.1
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/regression-contract.yaml','utf8');if(!c.includes('version: 14.1.1'))process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] CHANGELOG.md 包含 v14.0.0 条目
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/CHANGELOG.md','utf8');if(!c.includes('[14.0.0]'))process.exit(1);console.log('OK')"

## 实现方案
**要改的文件**:
- `packages/engine/hooks/VERSION` — 改为 14.1.1
- `packages/engine/regression-contract.yaml` — 从 .bak 恢复，更新 version 字段
- `packages/engine/CHANGELOG.md` — 在 [13.3.0] 条目前插入 v13.4.0~v14.1.1 里程碑

**不改什么**: 不动 .bak 文件，不动任何逻辑代码
