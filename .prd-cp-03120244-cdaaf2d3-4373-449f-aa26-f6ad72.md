# PRD - KR 进度诚实化修复

**需求来源**: Brain 自动调度任务（KR审计发现虚标进度）
**功能描述**:
多个 KR 的 stated_progress=100 但实际 initiative 完成率远低于此。
需要：1) 审计端点暴露差距；2) 一次性修正数据；3) memory_stream 记录。

**涉及文件**:
- `packages/brain/src/routes/task-goals.js` — 新增 GET /goals/audit 端点
- `packages/brain/migrations/150_kr_progress_correction.sql` — 一次性 migration（记录修正历史）
- `packages/brain/src/server.js` — 无需改动（路由已注册）

**成功标准**:

## 成功标准

1. GET /api/brain/tasks/goals/audit 返回 JSON 数组，每条含 id/title/stated_progress/actual_progress/initiative_total/initiative_done/gap
2. 修正前：至少 3 个 KR 的 stated_progress 与 actual_progress 差距 > 20%（gap > 20）
3. 调用 POST /api/brain/tasks/goals/audit/apply 后，虚标 KR progress 更新为实际值
4. memory_stream 中有 source_type='kr_audit' 的修正事件记录

**非目标**:
- 不改 goals 表的 schema
- 不影响 Brain 现有调度逻辑
- 不做自动定期修正（只做一次性 apply）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| 数据库查询失败 | 返回 500 + 错误信息 | P3 | kr_audit_query_failed |
| KR 无关联 initiative | actual_progress = 0，正常返回 | P3 | - |
| apply 时 KR 已被删除 | 跳过，继续下一条 | P3 | - |
