# PRD - fix(brain): executor 孤儿检测路径补写 error_message 列 — 消灭静默失败

**需求来源**: Brain 自动调度任务，业务领域 agent_ops
**功能描述**: executor.js 孤儿检测路径在将任务标记为 failed 时，只更新 payload.error_details，不写 error_message 列，导致任务失败后 error_message=NULL，外部完全无法感知失败原因。在所有直接写 status=failed 的 DB 更新语句中，同时写入 error_message 列。
**涉及文件**: `packages/brain/src/executor.js`，对应单测文件
**成功标准**: 所有 failed 路径写入 error_message，单测验证

## 功能描述

### 背景
孤儿检测路径（executor.js startupSyncOrphans / 孤儿清理）在将任务标记为 failed 时，只更新 payload.error_details，不写 error_message 列，导致任务失败后 error_message=NULL，外部完全无法感知失败原因。

### 目标
在所有直接写 status=failed 的 DB 更新语句中，同时写入 error_message 列，提供结构化的 fallback 信息。

### 验收标准
- [ ] executor.js startupSyncOrphans：UPDATE tasks 同时写 error_message=failureClass+reason+timestamp
- [ ] executor.js liveness probe 超时路径：同上
- [ ] executor.js watchdog kill 进入 quarantine 的路径：同上（quarantine 是 failed 的特殊形式）
- [ ] error_message 格式：[孤儿检测|liveness超时|watchdog] reason=X at TIMESTAMP
- [ ] 单测：mock DB，验证 error_message 被写入

### 技术要点
- 目标文件：packages/brain/src/executor.js
- 搜索所有 `status = failed` 或 `status = 'failed'` 的 pool.query / client.query
- 新增字段：error_message = $N，值格式：`[orphan_detected] reason=${reason} at ${new Date().toISOString()}`
- 不影响 watchdog retry（requeue）路径，只影响最终 failed 路径

### 异常处理 & 报警规范

N/A - 无外部 IO，仅 DB 写入
