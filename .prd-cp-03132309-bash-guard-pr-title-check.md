## PRD - bash-guard.sh 新增 gh pr create title 格式验证

**需求来源**: AI 开发流程保障——防止 AI 忘记加 [CONFIG] 标签或写出不合规的 PR title
**功能描述**: 在 packages/engine/hooks/bash-guard.sh 中拦截 `gh pr create` 命令，检查 --title 参数，验证两条规则：
1. title 必须符合 Conventional Commits 格式（feat/fix/docs/chore/test/refactor/build/ci/style/perf/revert 前缀）
2. 如果改动文件包含 packages/engine/ 路径，title 必须包含 [CONFIG] 标签

**涉及文件**:
- `packages/engine/hooks/bash-guard.sh`（主要修改）
- `packages/engine/tests/hook-gates.test.ts` 或新建测试文件（新增测试用例）
- `packages/engine/feature-registry.yml`（新增 changelog 条目）
- `packages/engine/package.json`（版本 bump）
- `packages/engine/package-lock.json`（版本 bump）
- 根 `package-lock.json`（engine 条目 bump）
- `packages/engine/VERSION`（版本 bump）
- `packages/engine/.hook-core-version`（版本 bump）
- `packages/engine/regression-contract.yaml`（版本 bump）

**非目标**:
- 不验证 PR title 的其他格式（如 scope、长度）
- 不检查 PR body 内容
- 不修改其他 hook 文件

## 成功标准

1. `gh pr create --title "feat: 描述"` → 正常放行（Conventional Commits 格式正确）
2. `gh pr create --title "random text"` → bash-guard 拦截，exit 2，输出格式错误提示
3. 改动文件含 packages/engine/ 时，`gh pr create --title "feat: 描述"` → 拦截，提示缺少 [CONFIG]
4. 改动文件含 packages/engine/ 时，`gh pr create --title "[CONFIG] feat: 描述"` → 正常放行
5. title 提取失败（格式奇怪）→ 降级放行，不拦截
6. Engine 版本 bump（6 个文件），feature-registry.yml 有新条目
7. CI 全部通过

### 异常处理 & 报警规范

N/A - 无 IO 操作（bash-guard.sh 是本地 hook，纯 shell 脚本逻辑）
