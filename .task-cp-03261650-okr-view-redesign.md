# Task: feat(gtd): OKR页面重设计 — Vision置顶、聚焦OKR层、description可编辑

## 背景

GTD OKR 页面当前两个结构性问题：
1. 层级反了：Area 在顶层包含 Vision，但 Vision 是全公司唯一最顶层
2. 职责混淆：OKR 页面混入了 Project/Scope/Initiative，应该只显示到 KR 层

## 改动范围

### `apps/api/src/task-system/full-tree.js`
- 新增 `?view=okr` 查询参数
- 返回：Vision → Area → Objective → KR（4层）
- 原有无参数 full-tree 调用不变（兼容）

### `apps/api/features/gtd/pages/GTDOkr.tsx`
- 数据源改为 `/api/tasks/full-tree?view=okr`
- 顶层渲染 Vision（金色标签），Area 作为分组 section
- description 可内联编辑（铅笔图标 → textarea → blur 保存 PATCH）
- status 点击切换（保留）
- 不显示 Project/Scope/Initiative

## 成功标准

- Vision 显示在最顶层（所有 Area 在其下）
- OKR 页面只显示到 KR 层，无 Project/Scope/Initiative
- description 可内联编辑并保存
- KR 显示进度（current/target/unit）
