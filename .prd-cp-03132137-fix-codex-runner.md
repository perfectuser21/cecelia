# PRD: 修复 runner.sh — 改为单次大调用模式（对齐 Claude Code 无头工作流）

## 背景

并行测试发现 Codex runner 无法完成任务，根本原因是 runner.sh 分轮驱动模式（每轮 codex-bin exec 是新 session，失忆），而 Claude Code 是单次持久 session。

## 目标

将 runner.sh 改为单次大调用模式：
1. 第一次调用：发送完整 /dev 工作流 prompt（含 task-id、分支名、项目路径）
2. 调用完成后：devloop_check 验证是否完成
3. 若未完成（Codex 中途退出）：以带完整上下文的恢复 prompt 重启
4. 修复已知兼容性问题：--cwd 改为 cd、--sandbox full-access 改为 danger-full-access
5. 支持 CODEX_HOME 环境变量

## 成功标准

- runner.sh 不再使用 --cwd 和 --sandbox full-access（已知无效参数）
- CODEX_HOME 通过环境变量配置
- 第一次调用发送含完整上下文的综合 prompt
- 重试时发送带当前状态的恢复 prompt
- 有 TypeScript 测试文件覆盖主要逻辑
