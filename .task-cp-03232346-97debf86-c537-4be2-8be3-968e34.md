# Task Card: planner-domain-routing.test.js fixture 清理

## 需求（What & Why）

**功能描述**：修复 `planner-domain-routing.test.js` 的 test fixture，补充 afterEach 对新 OKR 表的显式清理，修正 createKR 的类型语义。

**背景**：
- `createKR` 插入 `goals(type='area_okr')` → 触发器同步到 `objectives`（KR 应同步到 `key_results`，语义错误）
- `afterEach` 只清理 `goals`/`projects` 旧表，依赖触发器级联清理新 OKR 表
- 触发器用 `EXCEPTION WHEN OTHERS` 静默捕获异常，失败时新表记录残留 → 测试间污染
- `key_results` 无 `domain` 列，无法直接 INSERT；改用 `type='area_kr'` 让触发器同步到 `key_results`，`goals.domain` 字段仍可用

**不做什么**：不修改 planner.js 逻辑；不修改数据库 schema。
