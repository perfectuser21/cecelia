# PRD: L4 CI 优化 4 shard + 堆内存 3GB

## 背景
L4 CI 当前配置 3 shard 并行 + 6GB 堆内存，但实际运行时每个 shard 只用 1-2GB 内存。
多余的内存预留在 M1 上造成资源浪费，且 3 shard 分配不均导致运行时间偏长。

## 目标
1. 将 shard 从 3 改为 4，缩短最长 shard 的运行时间
2. 将 NODE_OPTIONS 堆内存从 6GB 降到 3GB，释放 M1 资源
3. 更新所有相关注释和端口计算公式

## 成功标准
- [ ] matrix 包含 4 个 shard（1/4 ~ 4/4）
- [ ] NODE_OPTIONS max-old-space-size 为 3072
- [ ] 端口计算公式从 `* 3` 改为 `* 4`，范围 5500~5896
- [ ] GoldenPath E2E 条件更新为 shard '1/4'
- [ ] baseline 计算从 `/3` 改为 `/4`
