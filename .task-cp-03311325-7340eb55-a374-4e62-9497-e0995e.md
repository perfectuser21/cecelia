# Task Card: 社媒热点数据接入接口

## 需求（What & Why）

**功能描述**: 在 Brain 中新增一个 HTTP API 端点（`GET /api/brain/social/trending`），用于查询各平台（抖音、小红书、知乎、微博等）最近的热门内容数据，使内容生成系统能够消费来自 TimescaleDB 的社媒采集数据。

**背景**: HK VPS 上已运行 8 个平台采集器（douyin/kuaishou/xiaohongshu/toutiao/weibo/zhihu/channels/wechat），采集数据存入 TimescaleDB，但 Brain 的内容生成系统（content_topics 表）目前未与 TimescaleDB 打通，数据闭环 KR 进度为 0%。该接口是内容选题、发布等所有下游能力的基础依赖。

**不做什么**:
- 不修改现有采集器逻辑
- 不改动 content_topics 表结构
- 不实现内容生成系统对热点数据的自动消费（仅提供查询接口）
- 不实现采集数据的写入（TimescaleDB 由 HK VPS 采集器负责写入）

## 成功标准

1. [ARTIFACT] `packages/brain/src/routes/social-trending.js` 文件存在，并在 `src/routes.js` 中注册

3. [GATE] CI 全部通过（含 social-trending.test.ts 单元测试）

## 实现方案

**要改的文件**:
- `packages/brain/src/routes/social-trending.js`（新增）— TimescaleDB 跨库查询路由
- `packages/brain/src/routes.js`（修改）— 注册新路由
- `packages/brain/src/__tests__/social-trending.test.ts`（新增）— 单元测试（vitest + mock pool）

**Scope 锚定**: packages/brain/

**技术方案**:
- TimescaleDB 连接：独立 pg.Pool，连接参数从环境变量读取（TIMESCALE_HOST/TIMESCALE_DB/TIMESCALE_USER/TIMESCALE_PASSWORD）
- 当 TimescaleDB 不可达时：返回空数组 + 200（降级，不报错）
- 查询表：`v_all_platforms`（TimescaleDB 视图，统一各平台采集数据）
- 支持参数：`platform`（过滤平台）、`limit`（默认 50）、`days`（时间范围，默认 7）
