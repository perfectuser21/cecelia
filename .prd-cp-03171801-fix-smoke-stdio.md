# PRD: 修复 smoke.test.js stdio WriteStream ubuntu 兼容性

## 背景

PR #1022 将 L3/L4 CI runner 从 hk-vps（macOS）迁移到 ubuntu-latest 后，`smoke.test.js` 在 `coverage-baseline` job 中失败：

```
TypeError: The argument 'stdio' is invalid. Received WriteStream { fd: null }
❯ startCeceliaBridge  server.js:238:17
```

## 问题

`server.js:236` 用 `createWriteStream()` 创建 WriteStream 后立即传给 `spawn` 的 `stdio`。在文件真正打开前 `fd: null`，ubuntu 的 Node.js 拒绝此参数，macOS 接受但 ubuntu 不接受。

## 目标

修复 `startCeceliaBridge()` 中的 stdio 参数，使用整数文件描述符（`fs.openSync`）替代 WriteStream，消除平台兼容性问题。

## 成功标准

1. [BEHAVIOR] smoke.test.js 不再因 stdio WriteStream fd: null 报 TypeError
2. [ARTIFACT] server.js 使用 fs.openSync 获取 logFd 整数
3. [GATE] CI L3 coverage-baseline job 通过
