# /analyze - 内容分析

分析创作内容，提供洞察和改进建议。

## 触发方式

```
/analyze <文件名或主题>
/analyze 爆款
/analyze 最近
```

## 功能

### 1. 单篇分析

```
/analyze 不是你学得不够
```

输出：
- 主题、受众、结构
- 亮点（钩子、金句、案例）
- 可改进点
- 改写建议

### 2. 批量分析

```
/analyze 爆款           # 分析 is_hit: true 的文章
/analyze 深度帖         # 分析所有深度帖
/analyze 2025-12        # 分析 12 月内容
```

输出：
- 共同特征
- 结构模式
- 标题规律

### 3. 对比分析

```
/analyze compare 文章A 文章B
```

## 分析维度

| 维度 | 说明 |
|------|------|
| 主题 | 核心讨论什么 |
| 受众 | 写给谁 |
| 结构 | 开头-中间-结尾 |
| 钩子 | 开头是否抓人 |
| 金句 | 可传播的句子 |
| 行动 | 引导读者做什么 |

## 执行步骤

1. 根据参数定位文件（Glob 搜索 content/）
2. 读取文件内容和 frontmatter
3. 按维度分析
4. 输出分析报告
