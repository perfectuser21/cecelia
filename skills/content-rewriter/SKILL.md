# /rewrite - 内容改写

将内容改写成不同平台/格式的版本。

## 触发方式

```
/rewrite <文件名> <目标格式>
/rewrite 不是你学得不够 短贴
/rewrite 不是你学得不够 小红书
```

## 目标格式

| 格式 | 说明 | 字数 |
|------|------|------|
| 短贴 | 精简版，保留核心观点 | 100-200 字 |
| 小红书 | 分段+emoji+标签 | 300-500 字 |
| 公众号 | 完整文章格式 | 800-1500 字 |
| 推特 | 单条或 thread | 280 字/条 |
| 视频脚本 | 口语化，适合朗读 | 按时长 |
| 标题党 | 只改标题，更抓眼球 | - |

## 改写原则

1. **保留核心观点** - 不改变原意
2. **适配平台调性** - 小红书活泼，公众号正式
3. **优化钩子** - 开头更抓人
4. **加强金句** - 让观点更锐利

## 使用示例

### 深度帖 → 短贴

```
/rewrite 2025-11-14-不是你学得不够 短贴
```

输出：
```markdown
信息太多，不是学得不够。

塔勒布说：过滤比获取更重要。

建个自己的信息系统：
收集 → 处理 → 转化

不是十件事推到 30%，
而是一件事做到 90%。
```

### 深度帖 → 小红书

```
/rewrite 2025-11-14-不是你学得不够 小红书
```

输出带 emoji 和标签的版本。

## 执行步骤

1. 读取原文件
2. 提取核心观点和金句
3. 按目标格式重组
4. 输出改写版本
5. 可选：保存到新文件
