# Task Card: fix(engine): worktree 分支命名从分钟精度改为秒精度

## 需求（What & Why）
**功能描述**: 修改 `packages/engine/skills/dev/scripts/worktree-manage.sh` 中 timestamp 生成逻辑，从 `%m%d%H%M`（分钟精度）改为 `%m%d%H%M%S`（秒精度）。

**背景**: 同一任务在同一分钟内重试会生成完全相同的分支名 `cp-MMDDHHNN-slug`，导致 `fatal: a branch named ... already exists`，cecelia-run 立即失败，任务进入 quarantine 循环。

**不做什么**: 不改变分支名格式，不影响其他命名逻辑。

## 成功标准

> [ARTIFACT] worktree-manage.sh 第 145 行 timestamp 使用秒精度格式
> [BEHAVIOR] 同一分钟内两次调用生成不同分支名

## 验收条件（DoD）

- [ ] [ARTIFACT] worktree-manage.sh 使用 `date +%m%d%H%M%S` 秒精度格式
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/skills/dev/scripts/worktree-manage.sh','utf8');if(!c.includes('date +%m%d%H%M%S'))process.exit(1);console.log('OK')"

- [ ] [BEHAVIOR] timestamp 格式长度为 12 位（MMDDHHMMSS）
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/skills/dev/scripts/worktree-manage.sh','utf8');const m=c.match(/timestamp=\\\$(date \\+([^)]+))/);if(!m)process.exit(1);const fmt=m[2].replace(/%/g,'').replace(/[a-zA-Z]/g,'X');if(fmt.length!==10)process.exit(1);console.log('OK fmt='+m[2])"

- [ ] [ARTIFACT] Engine 版本已 bump（package.json/lock/VERSION/.hook-core-version/regression-contract.yaml/feature-registry.yml）
  Test: manual:node -e "const v=require('./packages/engine/package.json').version;const fv=require('fs').readFileSync('packages/engine/VERSION','utf8').trim();if(v!==fv)process.exit(1);console.log('version='+v)"

- `packages/engine/skills/dev/scripts/worktree-manage.sh` 第 145 行
