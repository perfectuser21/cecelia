# Task Card: 升级小红书和快手发布脚本（CDP UI自动化 + 音乐选择）

## 需求（What & Why）

**功能描述**：
把小红书和快手发布脚本升级为生产可用的纯自动化脚本，与抖音发布器保持一致的使用风格（指定内容目录 → 无人值守发布）。本次新增音乐选择功能。

**背景**：

**不做什么**：
- 不改抖音/头条/微博脚本
- 不支持视频发布（只做图文）
- 不改 N8N workflow（只改 Node.js 脚本）
- 音乐选择失败不阻断发布（降级跳过）

## 实现方案

**要改的文件**：
- `packages/workflows/skills/xiaohongshu-publisher/scripts/publish-xiaohongshu-image.cjs`（重写）
- `packages/workflows/skills/kuaishou-publisher/scripts/publish-kuaishou-image.cjs`（重写为 CDP UI 方案）
- `packages/workflows/skills/kuaishou-publisher/scripts/batch-publish-kuaishou.sh`（指向 image.cjs）

**SCP 方案（两个平台共用）**：
```
US Mac mini → rsync → xian-mac (100.86.57.69)
xian-mac → scp -i ~/.ssh/windows_ed → Windows (100.97.242.124)
```
SSH key 在 xian-mac 上：`~/.ssh/windows_ed`

**快手描述填写**（已验证）：
- 点击坐标 (793, 210) 聚焦描述框（class: _description_eho7l_59）
- 用 Input.insertText 插入文本
- 快手话题标签 ≤ 4 个

**快手发布按钮**（已验证）：
- TreeWalker 找最后一个"发布"文本节点
- scrollIntoView + Input.dispatchMouseEvent 点击

**音乐选择流程**（待本地验证）：
- 找"添加音乐"按钮 → 点击
- 等待音乐搜索框出现
- 输入搜索词（热歌/默认）
- 选择第一个结果
- 失败则 console.warn 并跳过

**Scope 锚定**：
- 只改 `packages/workflows/skills/` 下的发布脚本
- 不改 CI 配置、测试框架
