# PRD: PR Review 阻塞门禁 + Divergence 可执行检查

## 功能描述

将 PR Review workflow 从"建议模式"升级为"阻塞模式"，同时将 divergence_count 检查从 SKILL.md 文字描述提升为 devloop-check.sh 中的可执行断言。

## 背景

1. `.github/workflows/pr-review.yml` 使用 DeepSeek V3 进行 AI 代码审查，已加入 branch protection 必填 check，但当 review 发现🔴严重问题时 job 仍然成功退出（exit 0），不阻塞合并。
2. `divergence_count >= 1` 检查（Evaluator 至少要与主 agent 有 1 次分歧）当前只存在于 `packages/workflows/skills/spec-review/SKILL.md` 文字描述中，不是可执行代码，可被绕过。
3. 两处"软约束"需升级为硬门禁，才能真正保护代码质量和 Evaluator 独立性。

## 成功标准

1. `pr-review.yml` 改为直接调用 OpenRouter API，提取 review 文本，检测到🔴时以 exit 1 结束 job
2. `scripts/devgate/detect-review-issues.js` 独立脚本：stdin 包含🔴时 exit 1
3. `devloop-check.sh` 中 `check_divergence_count` 函数：值为 0 时返回非零退出码
4. feat PR 测试文件（pr-review-blocking.test.ts + devloop-check-divergence.test.ts）
5. CI 全部通过

## 不做什么

- 不改变 AI review 的 prompt 内容或审查逻辑
- 不修改 branch protection 规则本身
- 不重写 spec-review SKILL.md 的其他部分
- 不改变 devloop-check.sh 中其他现有检查项的行为
