## PRD - feat(brain): POST /api/brain/rumination/run — 手动触发 + force 模式绕过每日预算

**需求来源**: Brain 自动调度任务，任务类型 dev，领域 agent_ops
**功能描述**: 当前反刍预算今日耗尽（20/20），60条知识积压无法消化。需要在 routes/inner-life.js 新增 POST /api/brain/rumination/run 端点，内部调用 rumination.js 的 manualRunRumination()，加 force 参数控制是否绕过 daily_budget 检查。
**涉及文件**:
- `packages/brain/src/routes/inner-life.js` — 新增路由
- `packages/brain/src/rumination.js` — 新增 manualRunRumination() 函数
- `packages/brain/src/__tests__/rumination.test.js` — 单元测试

**成功标准**:
- POST /api/brain/rumination/run 存在且正常工作
- force=true 时即使 daily_count >= daily_budget 也能触发
- 响应包含消化数量和剩余积压
- 单元测试覆盖 force/非force 两种路径

**非目标**:
- 不修改自动 tick 触发的反刍逻辑
- 不改变 MAX_PER_TICK 限制（force 模式仍受此限制）

## 成功标准
- [ ] POST /api/brain/rumination/run 端点存在且正常响应
- [ ] force=false 时正常预算检查
- [ ] force=true 时绕过 daily_budget 直接消化
- [ ] 响应包含 digested 数量和 remaining 积压数
- [ ] 单元测试覆盖两种路径

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| rumination.js 不存在 manualRunRumination | 返回 500 + 错误信息 | P2 | rumination_manual_error |
| 预算已满且 force=false | 返回 200 + skipped=true | P3 | rumination_budget_exhausted |
| DB 查询失败 | 返回 500 + 错误信息 | P1 | rumination_db_error |
