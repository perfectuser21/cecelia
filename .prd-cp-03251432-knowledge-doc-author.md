# PRD: 知识文档系统完善 — 补全author字段+新建conversation_captures+放开design_docs类型

## 背景

知识文档系统多张表缺少author/made_by字段，无法区分人类决策和系统决策。同时缺少对话捕获表，导致Claude Code会话内容无法持久化。

## 功能描述

1. 给 decisions/learnings/knowledge/dev_records/ideas 各表加 author/made_by 字段
2. 新建 conversation_captures 表捕获会话摘要
3. 放开 design_docs 的 type CHECK 约束，支持 meeting/strategy/roadmap 等新类型
4. 新建 conversation-captures API 路由
5. 更新 strategic-decisions 路由支持 made_by/author 过滤

## 成功标准

- decisions表有 author/made_by/priority/area/alternatives/decided_at 字段
- conversation_captures表存在，含完整索引
- GET /api/brain/conversation-captures 可访问
- design_docs type支持 meeting/strategy/roadmap
- selfcheck.js EXPECTED_SCHEMA_VERSION = '194'
