# PRD: 内容类型注册表（YAML 配置化）— PR1: YAML schema 设计

## 背景
内容工厂 Initiative（330bce6f）—— 当前内容类型（solo-company-case）硬编码在 skill 里，无法扩展。
需要 YAML schema + 加载器，让新增内容类型只需创建一个 YAML 文件，无需改 Pipeline 代码。

## 目标
- 创建 packages/brain/src/content-types/ 目录
- 创建 solo-company-case.yaml 示例配置文件
- 创建 content-type-registry.js 加载器（导出 getContentType + listContentTypes）
- 单元测试覆盖加载 + 验证逻辑

## 成功标准
1. packages/brain/src/content-types/ 目录存在，含 YAML 文件
2. getContentType('solo-company-case') 返回完整配置对象
3. listContentTypes() 返回所有类型名称数组
4. 加载无效配置时抛出描述性错误
5. CI 全部通过
