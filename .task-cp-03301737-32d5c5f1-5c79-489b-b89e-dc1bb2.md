# Task Card: 发布后数据回收 — 各平台数据写回 DB

## 需求（What & Why）
**功能描述**: content_publish 任务完成 4 小时后，自动触发对应平台的 scraper 脚本，将阅读/点赞/播放等互动数据写入新表 `pipeline_publish_stats`，并通过 API 端点对外暴露汇总数据。

**背景**: scraper 脚本已存在于 `/perfect21/zenithjoy/workflows/platform-data/workflows/scraper/scripts/`，支持 douyin/kuaishou/xiaohongshu/toutiao/weibo/zhihu/channels/wechat 共 8 个平台。但 scraper 与 pipeline 之间没有自动关联，发布后数据无法自动回收。

**不做什么**:
- 不修改已有 scraper 脚本的内部逻辑
- 不改变现有 `/api/brain/pipelines/:id` 系列端点的响应结构
- 不实时（发布后立刻）抓取数据，4h 延迟是产品设计
- 不支持手动触发（本期只做自动触发）

## 成功标准
1. [ARTIFACT] 数据库迁移文件存在，`pipeline_publish_stats` 表可正常创建
2. [BEHAVIOR] content_publish 完成满 4h 后，Brain tick 自动触发对应平台 scraper 并将结果写入 `pipeline_publish_stats`
3. [BEHAVIOR] `GET /api/brain/pipelines/:id/stats` 返回该 pipeline 下各平台的互动数据汇总
4. [GATE] CI 全部通过
