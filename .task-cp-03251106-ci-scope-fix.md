# Task Card: CI scope 补全 — routing-map 路径扩展 + Brain-API 集成测试

## 需求（What & Why）

**功能描述**: 补全 CI routing-map 路径覆盖 + 新增 Brain↔Frontend API 集成测试

**背景**:
- `ci-helpers` 仅列举具体脚本文件，新增 scripts/ 根文件时无路由保护
- `ci-configuration` 仅列举 2 个 ci/ 文件，新增 ci/ 文件时无路由保护
- `apps/api/src/__tests__/` 只有 intent-recognition 测试，Brain-API proxy 层未被覆盖

**不做什么**:
- 不修改 ci-l4-runtime.yml
- 不修改 workspace routing-map 层级

## 成功标准

- ci-helpers 路径改为 `scripts/**`（排除 scripts/devgate/**），新增脚本自动覆盖
- ci-configuration 路径改为 `ci/**`，新增 ci 配置文件自动覆盖
- Brain-API 集成测试覆盖 proxy 成功/失败/timeout 三种场景

## 实现方案

**要改的文件**:
1. `ci/routing-map.yml` — ci-helpers paths 改为 `scripts/**`（排除 devgate）；ci-configuration paths 改为 `ci/**`
2. 新增 `apps/api/src/__tests__/brain-api-integration.test.ts`

**技术设计**:
- routing-map 路径扩展不影响 ci-evolution-check（只看 root: 字段）
- 集成测试 mock `fetch`，无需真实 Brain 服务
- 测试覆盖：GET /api/brain/tasks 成功、Brain 不可达 (503)、Brain 500 错误

**Scope 锚定**: ci/routing-map.yml + apps/api/src/__tests__/brain-api-integration.test.ts
