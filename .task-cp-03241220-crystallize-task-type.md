# Task Card: crystallize task type — 替代 codex_playwright

## 需求（What & Why）

**功能描述**: 新建 `crystallize` task type，替代现有的 `codex_playwright` + `playwright-auto`。crystallize 是"能力蒸馏"的完整4步流水线：Scope → Forge → Verify → Register，让西安 Codex 能把高度重复场景探索跑通后沉淀成可独立运行的脚本。

**背景**:
- `codex_playwright` 和 `playwright-auto` 只有2步（探索+验证），缺少 Scope 和 Register
- 新的 crystallize 是完整4步，playwright skill 成为 Forge 阶段的执行工具之一
- 子状态（crystallize_scope/forge/verify/register）是独立 task_type，由 orchestrator 创建子任务

**不做什么**:
- 不修改 playwright-runner.sh 的执行逻辑（只更新头部注释，去掉 codex_playwright 引用）
- 不建 script registry 表（下一个 Initiative）
- 不修改 ZenithJoy Production Pipeline
