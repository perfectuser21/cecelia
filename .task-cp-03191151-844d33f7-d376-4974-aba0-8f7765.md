# Task Card: 好奇心模块激活修复 + 情绪层「空」根因分析

## 需求（What & Why）

**功能描述**：
情绪层连续 4+ 次报「空」，好奇心模块从未被激活。
根因分析后实施最小修复：让 `explore` 类型 desire 拥有与 `act/follow_up` 同等的路由优先权，并添加去重保护。

**背景**：
探索任务 fb8f547d（desire_system）发现：
- `desires` 表中存在 2 个 `explore` pending desires，但都未被执行
- 好奇心路径：perception → memory → reflection → desire-formation(explore) → expression-decision → desire/index.js 创建 research 任务
- 断链点：`expression-decision.js` 对 `explore` 类型不做优先处理，导致 urgency=5 的 `inform` desires 每次都比 urgency=4 的 `explore` 得分更高，research 任务永远无法创建
- 情绪层的「空」是准确反映：无自主探索 → `intellectual_idle` 信号 → LLM 诚实生成"空落"情绪（设计正确，无 bug）

**根因总结**：
1. `expression-decision.js`：`explore` 类型未加入 act/follow_up 优先通道
2. `desire/index.js`：explore 路径缺少去重（同 act 类型的 dedup 逻辑）
3. `curiosity_topics` 为空：感知层信号 13（好奇心积累）永远不触发，因为 explore desire 从未被执行

**不做什么**：
- 不修改情绪层逻辑（emotion-layer.js 工作正确）
- 不修改感知层（perception.js 已有 intellectual_idle 信号，工作正确）
- 不修改反思层（reflection.js 工作正确）
- 不改变 exploration task_type（exploratory 已移除，用 research 代替）

## 成功标准
