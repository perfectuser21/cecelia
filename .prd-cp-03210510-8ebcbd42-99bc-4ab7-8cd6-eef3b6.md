# PRD: stop-dev.sh fallback 补齐 spec_review 超时检查

## 背景

PR #1286 修复了三处 Pipeline 死锁问题。PR #1294 和 #1296 重新应用了这些修复。
PR #1296 在 stop-dev.sh fallback 路径中补齐了 code_review_gate 的 15 分钟超时，
但遗漏了 spec_review 的相同超时处理。

## 功能描述

在 stop-dev.sh fallback 路径中补齐 spec_review 状态检查（条件 1.5），与
code_review_gate 的处理方式完全对称：
- spec_review = pass → 继续
- spec_review = fail → exit 2（阻塞，提示修复 Task Card）
- spec_review = pending + 超时 15 分钟 → 自动降级为 pass

## 成功标准

1. stop-dev.sh fallback 包含 spec_review 条件 1.5 检查，与 code_review_gate 完全对称
2. spec_review pending 超 900s 时 fallback 自动 PASS（超时 15 分钟降级）
3. spec_review fail 时 exit 2 阻塞工作流，提示修复 Task Card
