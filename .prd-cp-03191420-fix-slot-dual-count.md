# PRD: slot-allocator 双源计数取保守值

## 背景

`calculateSlotBudget()` 中 `totalRunning` 使用 ps 进程检测（`sessions.total`），
但 `ceceliaUsed` 和 `autoDispatchUsed` 使用 DB 查询。两个数据源不一致时，
ps 检测可能遗漏进程（如进程标题被覆盖），导致 totalRunning 偏低，从而过度派发。

## 成功标准

- `totalRunning` 取 `max(sessions.total, ceceliaUsed + autoDispatchUsed)`
- 新增测试验证：DB 计数大于 ps 检测时取 DB 值
- 新增测试验证：ps 检测大于 DB 计数时取 ps 值
- 所有现有测试继续通过
