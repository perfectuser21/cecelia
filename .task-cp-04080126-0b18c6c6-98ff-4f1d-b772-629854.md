# Task Card: KR3/KR4 进度采集链路修复

## 需求（What & Why）
**功能描述**: 修复 ZenithJoy KR3（微信小程序）和 KR4（geo SEO网站）的进度采集链路
**背景**: 两个问题：
1. kr-verifier.js 中 $2 参数类型歧义导致 ZenithJoy KR1/KR2 UPDATE key_results 失败（last_error = "inconsistent types deduced for parameter $2"）
2. KR3/KR4 verifier SQL 用 status 映射伪进度（active=50%，平均得 25%），不反映实际 okr_projects.progress

**不做什么**: 不改 OKR 结构、不改数据库 schema、不更新项目 progress 手动值
