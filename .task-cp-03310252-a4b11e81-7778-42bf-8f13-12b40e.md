# Task Card: 修复 ContentFactory + PipelineOutputPage UI 状态问题

## 需求（What & Why）

**功能描述**：修复内容工厂前端 4 个 UI 状态问题，让用户能实时看到 pipeline 执行进度，状态显示中文。

**背景**：
- ContentFactory.tsx 列表页无自动轮询，用户必须手动点刷新才能看到进度更新
- PipelineOutputPage.tsx 打开后不自动轮询，pipeline 执行中时状态不更新
- StatusBadge 缺少 'pending' 映射，导致显示原始英文 "pending"
- PipelineOutputPage 中 GenerationTab 的阶段状态文字直接用英文 icon 但不展示 stage-level status badge
- ContentFactory 列表中 pipeline 条目是 button，测试期望是 Link（指向 /content-factory/:id）

**不做什么**：
- 不修改 Brain API 后端逻辑
- 不改变 PipelineOutputPage 的四 Tab 结构
- 不改变 ContentFactory 表单逻辑

## 成功标准

## 成功标准

- ContentFactory pipeline 列表条目渲染为 `<Link to="/content-factory/:id">` 可点击链接
- ContentFactory 列表在有 `in_progress` 或 `queued` pipeline 时每 5s 自动轮询刷新
- PipelineOutputPage 在 pipeline 未完成时每 5s 自动轮询 pipeline 状态和 stages
- StatusBadge 包含 'pending' → '待执行' 映射，不显示原始英文
- PipelineOutputPage GenerationTab 显示所有 6 个阶段（含未开始的 pending 阶段）

## 实现方案

**要改的文件**：
1. `apps/api/features/content/pages/ContentFactory.tsx`
   - pipeline 列表条目：把 `<button onClick={toggleExpand}>` 改为 `<Link to={/content-factory/${p.id}}>` + 独立的展开按钮（或直接去掉展开，用 Link 导航）
   - 添加 StatusBadge 'pending' → '待执行' 映射
   - 添加自动轮询：`useEffect` 中检测 in_progress/queued pipeline，启动 5s setInterval

2. `apps/api/features/content/pages/PipelineOutputPage.tsx`
   - 添加自动轮询：pipeline 非完成状态时每 5s 刷新
   - 修复 GenerationTab：基于 PIPELINE_STAGE_ORDER 显示全 6 个阶段（用 pending 填充未创建的阶段）

**Scope 锚定**：仅修改以上 2 个 TSX 文件，不涉及后端、测试文件修改（测试已存在且期望行为）
