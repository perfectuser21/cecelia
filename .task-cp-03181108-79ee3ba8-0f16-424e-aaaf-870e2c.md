# Task Card: slot-allocator 积压感知降速

## 需求（What & Why）
**功能描述**: 在 slot-allocator.js 中新增队列深度感知，当 queued 任务数 > BACKPRESSURE_THRESHOLD(5) 时，将 tick.js 的 burst limit 从 2 压至 1，防止积压时连续多 tick 派发引发资源雪崩。
**背景**: tick.js 已有 MAX_NEW_DISPATCHES_PER_TICK=2 的 burst limiter，但无队列深度感知。积压 9-10 个任务时每 tick 仍尝试派发 2 个，连续多 tick 未消化会导致 N 个 agent 同时启动引发资源雪崩。
**不做什么**: 不过滤 task_type（统计全量 queued 任务），不修改 Pool C 预算计算逻辑，不改变积压=0 时的派发行为。

## 成功标准

1. [ARTIFACT] slot-allocator.js 新增 getQueueDepth() 和 BACKPRESSURE_THRESHOLD/BACKPRESSURE_BURST_LIMIT 常量导出
2. [ARTIFACT] calculateSlotBudget() 返回值新增 backpressure: { queue_depth, threshold, active, override_burst_limit } 字段
3. [BEHAVIOR] queue_depth=9 时 active=true，override_burst_limit=1，tick 每次最多派发 1 个
4. [BEHAVIOR] queue_depth=3 时 active=false，override_burst_limit=null，tick 每次最多派发 2 个（正常）
5. [GATE] CI 全部通过，现有测试不回归
