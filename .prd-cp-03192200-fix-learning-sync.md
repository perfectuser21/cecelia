# PRD: Learning 表同步修复

## 背景

`distill-learnings.js` 将 docs/learnings/ 的规则写入 `knowledge` 表，但 `learning-retriever.js` 从 `learnings` 表读取注入 /dev 任务。两张表不通，100+ 篇高质量 Learning 从未被注入。

## 目标

修改 `distill-learnings.js`，在写入 `knowledge` 表后同时写入 `learnings` 表，使 learning-retriever 能读到历史 learning 规则。

## 成功标准

- distill-learnings.js 在 INSERT knowledge 后同时 INSERT learnings
- 使用 content_hash 去重（ON CONFLICT DO NOTHING）
- 字段映射正确：name→title, content→content, sub_area→metadata.source_file
- learnings.category = 'process_improvement', learning_type = 'best_practice'
- source_branch 从 content JSON 的 source 字段提取
