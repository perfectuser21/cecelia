# Task Card: stop-dev.sh 跨 session orphan 隔离不对称修复

## 需求

**功能**: `packages/engine/hooks/stop-dev.sh:194` 的跨 session 隔离判断把"当前 session_id 空"的合法情况误判为"孤儿属于自己"，导致 Stop Hook 反复 block 无 `CLAUDE_SESSION_ID` 的 Claude Code session（headless/nested agent 环境常见）。

**背景**: 本机实测复现——Brain 派发 autonomous /dev 后，派出的 session 在 worktree 写了自己的 session_id 到 `.dev-lock`；主 session 的 `CLAUDE_SESSION_ID` 为空，Stop Hook 30+ 次误 block。

**不做**:
- 不改 self-heal 块（line 53 同样的 CLAUDE_SESSION_ID 要求，留给后续 PR）
- 不改 stop.sh router
- 不改 orphan 的自动清理路径（worktree gone 那条）

## 实现方案

**要改的文件**:
- `packages/engine/hooks/stop-dev.sh` — 修改 line ~194 跨 session 判断
- `packages/engine/tests/hooks/stop-hook-exit-codes.test.ts` — 补 2 个测试

**改动点**:
```bash
# 旧
if [[ -n "$_current_sid" && -n "$_orphan_sid" && "$_current_sid" != "$_orphan_sid" ]]; then

# 新
if [[ -n "$_orphan_sid" && "$_orphan_sid" != "$_current_sid" ]]; then
```

**判断真值表**（修复后）：
- current_sid 空 + orphan_sid 非空 → skip ✅（修复）
- current_sid 非空 + orphan_sid 非空 + 不同 → skip ✅（保留）
- current_sid 非空 + orphan_sid 非空 + 相同 → 不 skip（是自己的，走 block）
- current_sid 空 + orphan_sid 空 → 不 skip（都无主，走 block）
- current_sid 非空 + orphan_sid 空 → 不 skip（保守 block）
