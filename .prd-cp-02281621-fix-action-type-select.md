# PRD：修复 autumnrice/chat SELECT 缺少 action_type

## 问题

Brain 日志：
```
[autumnrice/chat] new version parse failed: null value in column "action_type" of relation "pending_actions" violates not-null constraint
```

根本原因：`routes.js` L1702 的 SELECT 语句没有查 `action_type`：
```sql
SELECT id, context, params, comments, status FROM pending_actions WHERE id = $1
```
所以 `action.action_type` 是 `undefined`，INSERT 时变成 null 报错。

## 修复

加上 `action_type`：
```sql
SELECT id, action_type, context, params, comments, status FROM pending_actions WHERE id = $1
```

Brain: patch bump（1.132.4 → 1.132.5）
