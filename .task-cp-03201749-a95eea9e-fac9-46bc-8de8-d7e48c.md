# Task Card: [SelfDrive] 修正 Scanner 对内嵌能力的误判

## 需求（What & Why）
**功能描述**: 修正 capability-scanner.js 中的内嵌能力识别逻辑，确保 Brain 内嵌运行的能力（意识层能力、架构性能力、Engine hooks 等）不被误判为孤岛。

**背景**:
- 当前 Scanner 将部分内嵌在 Brain 进程中的能力误判为孤岛
- migration 094 定义了 12 个 Cecelia 意识能力，但 Scanner 的 BRAIN_ALWAYS_ACTIVE 列表不完整
- Engine 层的 branch-protection、ci-devgate 等能力也需要特殊识别
- 这是后续孤岛激活决策的基础，必须准确

**不做什么**:
- 不改变扫描的基本逻辑（仍然检查使用数据）
- 不修改 capabilities 表结构
- 不影响真正孤岛的识别精度

## 成功标准
> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [ARTIFACT] 修正 capability-scanner.js 中的 BRAIN_ALWAYS_ACTIVE 列表
2. [BEHAVIOR] 所有 scope='cecelia' 的意识能力标记为 active 而非 island
3. [BEHAVIOR] Engine 层核心能力（hooks、devgate）正确识别为非孤岛
4. [GATE] 所有现有测试通过
