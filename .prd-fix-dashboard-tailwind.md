# PRD: 修复 Dashboard Tailwind CSS 扫描路径

## 问题

Monorepo 迁移后，`apps/dashboard/tailwind.config.js` 的 content 路径仍指向旧结构 `../core/features/`，
但实际 feature 组件已移到 `../api/features/`。导致 Tailwind purge 时扫描不到 222 个 feature 组件，
所有 utility class 被移除，CSS bundle 只有 34KB，全站排版崩溃。

## 修复方案

1. 更新 tailwind.config.js content 路径：`../core/features/` → `../api/features/`
2. 重新 build dashboard

## 成功标准

- CSS bundle 大小显著增加（从 34KB 到合理值）
- 所有页面字体大小、布局恢复正常
