# Task Card: PR2 — Migration 180 OKR 双向同步触发器

## 需求（What & Why）

**功能描述**：创建 migration 180，在旧 `goals`/`projects` 表上添加 AFTER INSERT/UPDATE/DELETE 触发器，将每次写操作自动同步到新 OKR 层级表（visions/objectives/key_results/okr_projects/okr_scopes/okr_initiatives）。

**背景**：
- migration 179 已完成一次性数据迁移（存量数据已在新表中）
- Brain 代码继续写旧表（goals/projects），导致新表数据未能实时更新
- 新旧表列差异大（priority/progress/embedding 等列不在新表），无法直接切换代码
- 触发器方案：零代码改动，新表始终与旧表保持同步

**不做什么**：不改任何 Brain 业务代码；不删旧表；不修改 selfcheck.js。

## DELETE 处理策略

- goals DELETE → 对应新表中 DELETE WHERE id = OLD.id（CASCADES 会自动删除子记录）
- projects DELETE → 对应新表中 DELETE WHERE id = OLD.id
- FK 约束（20张表引用 goals/projects）：触发器只删新表记录，不影响旧表的 FK 关系
- EXCEPTION 保护：任何触发器错误 RAISE WARNING 并返回 OLD/NEW，不阻断原始操作

## 成功标准

* migration 180 SQL 文件存在
* goals 表 INSERT type='vision' → visions 行同步（本地 DB 验证）
* projects 表 INSERT type='project' → okr_projects 行同步（本地 DB 验证）
* 触发器 EXCEPTION 不导致原始操作失败（RAISE WARNING 降级）
* pg_trigger 中注册 6 个触发器（goals ×3 + projects ×3）
