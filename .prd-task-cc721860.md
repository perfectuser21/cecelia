# PRD: Pipeline 小修复集合

## 背景

Engine pipeline 中存在 4 个小 bug，影响 DevGate 的可靠性和代码质量。合并为一个 PR 修复。

## 修复清单

### 1. Stage 1 正则 bug
- **文件**: `packages/engine/skills/dev/steps/01-spec.md` 第 105 行
- **问题**: `^- \[ \]` 无法匹配有缩进的 checkbox
- **修复**: 改为 `^\s*- \[ \]`

### 2. Fallback 路径缺 code_review 检查
- **文件**: `packages/engine/hooks/stop-dev.sh` 第 709-831 行
- **问题**: devloop-check.sh 加载失败时，fallback 跳过 code_review_gate
- **修复**: CI success 检查后、PR 合并检查前，添加 code_review 状态检查

### 3. cleanup_done 重复定义
- **文件**: `packages/engine/skills/dev/scripts/cleanup.sh` 和 `packages/engine/lib/devloop-check.sh`
- **问题**: 两处都定义了 `_mark_cleanup_done()`
- **修复**: cleanup.sh 删除本地定义，source devloop-check.sh 的版本

### 4. spec_review 降级时记录失败原因
- **文件**: `packages/engine/skills/dev/steps/01-spec.md` 第 221-224 行
- **问题**: spec_review 注册失败时只写 `spec_review_status: pass`，没记录降级原因
- **修复**: 额外写 `spec_review_degraded: true` + `spec_review_degraded_reason: <原因>`

## 成功标准

- 4 个修复全部到位
- 不引入新的回归
- CI 通过
