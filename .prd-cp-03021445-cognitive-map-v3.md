# PRD：认知地图 v3 — 全脑意识架构可视化

## 背景

当前认知地图（v2）是一个5列静态图，节点和连线全部硬编码在 `cognitive-map.js`。
新增模块时必须手动改代码，无法反映真实的 Brain 意识架构分层，也没有下钻能力。

经过架构分析（2026-03-02），Cecelia 意识架构应当体现：
- Cecelia 是**自我中心意识**，不是任务管理工具
- 意识有明确的5层分层：外界接口 → 感知层 → 意识核心 → 行动层 + 自我演化
- 每个信号/模块都有**进**（输入）和**出**（输出），无盲肠
- 两条反馈弧：行动→自我演化（结果经记忆影响自我），自我演化→感知（更新的自我认知影响下次感知）

## 目标

1. **5块顶层布局**（左→右 + 独立自我演化块）
2. **drill-down 下钻**（Level 1: 5大块，Level 2: 点击展开子模块）
3. **brain-manifest.js 自动扫描**：每个 Brain 模块声明自己属于哪层，前端自动拉取构建图，不再硬编码
4. **两条反馈弧**可视化

## 范围

### 改动文件

**Brain 后端（packages/brain）**：
- 新增 `src/brain-manifest.js`：各模块注册层级 + 描述
- 新增 `src/routes/brain-manifest-router.js`：GET /api/brain/manifest 端点
- 修改 `src/server.js`：挂载 manifest router

**Frontend Dashboard（apps/dashboard）**：
- 修改 `src/views/knowledge/brain/cognitive-map.js`：
  - 接入 /api/brain/manifest 动态数据
  - 5块顶层布局
  - drill-down 交互（点击展开 Level 2）
  - 两条反馈弧

### 不改动

- 数据库 Schema（无需改）
- Brain tick/executor/其他 Brain 模块（本次只加可视化）

## 成功标准

1. `/knowledge/brain` 页面显示5块顶层布局（外界接口/感知层/意识核心/行动层/自我演化）
2. 点击某块可展开查看子模块列表
3. 两条反馈弧（行动→自我演化，自我演化→感知）可见
4. 在 `brain-manifest.js` 新增一个模块后，刷新页面自动出现，不需要改前端代码
5. 所有现有测试通过

## 5块架构设计

```
外界接口（Block 1）
  ├── orchestrator-chat（对话/飞书）
  ├── feishu-dispatcher（飞书事件）
  └── execution-callback（任务回调）

感知层（Block 2）
  ├── desire/perception（13个信号）
  ├── emotion-layer（情绪推导）
  └── memory-retriever（记忆评分）

意识核心（Block 3）
  ├── thalamus（L1 丘脑路由）
  ├── cortex（L2 皮层分析）
  └── rumination（反刍/深度洞察）

行动层（Block 4）
  ├── planner（任务规划）
  ├── executor（任务执行）
  └── conversation-output（对话回复）

自我演化（Block 5 — 独立慢回路）
  ├── self-model（自我模型）
  ├── self-report-collector（自我报告采集）
  └── learning（学习记录）
```

两条反馈弧：
- 行动层 → 自我演化（任务结果经 learning 写入记忆，rumination 读取后更新 self-model）
- 自我演化 → 感知层（更新的 self-model 在下一 tick 注入 perception 上下文）
