# Task Card: L3 CI 按 commit type 智能跳过

## 需求（What & Why）

**功能描述**: 在 `ci-l3-code.yml` 中添加 `detect-commit-type` job，根据 PR title 提取 commit type，让 `fix:` / `docs:` / `style:` / `refactor:` / `chore:` 类型的 PR 跳过耗时 10-16 分钟的 L3。

**背景**: 每次 PR 无论类型都跑全量 L3，`qa-policy.yaml` 早已定义了规则但 CI 未接入。

**不做什么**:
- 不修改 L1/L2/L4 workflow
- 不修改 qa-policy.yaml

## 成功标准

> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [ARTIFACT] `ci-l3-code.yml` 中存在 `detect-commit-type` job
2. [ARTIFACT] 所有 L3 sub-jobs 均有 `needs: detect-commit-type` 和对应 if 条件
3. [BEHAVIOR] `fix:` 类型 PR → `should_run_l3=false`（本 PR 的 CI 运行验证 `feat:` → `true`）
4. [BEHAVIOR] `detect-commit-type` job 中存在 case 语句处理跳过逻辑
5. [GATE] Engine 版本 bump 到 12.88.0（5 个文件同步）
6. [GATE] CI L3 Code Gate 通过

## 验收条件（DoD）

- [x] [ARTIFACT] `detect-commit-type` job 存在于 ci-l3-code.yml
  Test: manual:bash -c "grep -c 'detect-commit-type' .github/workflows/ci-l3-code.yml"

- [x] [ARTIFACT] 所有 L3 sub-jobs 均有 should_run_l3 的 if 条件
  Test: manual:bash -c "grep -c 'should_run_l3' .github/workflows/ci-l3-code.yml"

- [x] [BEHAVIOR] 脚本中包含 fix/docs/style/refactor/chore 跳过 L3 的 case 逻辑
  Test: manual:bash -c "node -e \"const s=require('fs').readFileSync('.github/workflows/ci-l3-code.yml','utf8'); process.exit(s.includes('fix|docs|style|refactor|chore') ? 0 : 1)\""

- [x] [BEHAVIOR] l3-passed gate 在 should_run_l3=false 时直接通过（包含 SHOULD_RUN_L3 逻辑）
  Test: manual:bash -c "node -e \"const s=require('fs').readFileSync('.github/workflows/ci-l3-code.yml','utf8'); process.exit(s.includes('SHOULD_RUN_L3') ? 0 : 1)\""

- [x] [GATE] Engine 版本已 bump 到 12.88.0
  Test: manual:bash -c "grep -c '12.88.0' packages/engine/VERSION"

- [x] [GATE] CI L3 通过（verify via l3-passed check in l3-code.yml）
  Test: manual:bash -c "grep -c 'l3-passed' .github/workflows/ci-l3-code.yml"

**改动文件**:
- `.github/workflows/ci-l3-code.yml`（添加 detect-commit-type job）

- `packages/engine/features/feature-registry.yml`
