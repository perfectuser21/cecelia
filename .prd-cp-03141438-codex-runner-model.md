# PRD：runner.sh 新增 CODEX_MODEL 支持（v2.4.0）

## 背景

gpt-5.3-codex 是 ChatGPT Team plan 的默认 Codex 模型，但存在严格的 per-session quota 限制，导致任务频繁失败。gpt-5.4 是更新的模型，quota 更宽松，直接测试可正常运行。runner.sh 当前没有传入 `--model` 参数，完全依赖 Codex CLI 的默认选择，当 CLI 选择 gpt-5.3-codex 时就会 Quota exceeded。

## 目标

runner.sh 新增 `CODEX_MODEL` 环境变量支持，默认值 `gpt-5.4`，在调用 `codex-bin exec` 时通过 `--model` 参数传入，确保不会因默认选择 gpt-5.3-codex 而失败。

## 成功标准

1. runner.sh 读取 `CODEX_MODEL` 环境变量，默认 `gpt-5.4`
2. `codex-bin exec` 调用时加入 `--model "$CODEX_MODEL"` 参数
3. 日志输出显示当前使用的 model
4. 向后兼容：可通过设置 `CODEX_MODEL=gpt-5.3-codex` 恢复旧行为

## 范围

- `packages/engine/runners/codex/runner.sh`（核心改动）

## 优先级

P0 — 当前所有 Codex 任务均因 gpt-5.3-codex quota 失败，属于阻塞性问题
