# PRD: Brain 测试加固 Phase 1 - Smoke Test + Contract Test

## 背景

Cecelia Brain 有 110 个源码模块、268 个测试文件，但存在严重的测试质量问题：
- 51% 的测试用 mock 隔离了模块依赖，无法检测跨模块接口变更
- 0 个真正的集成测试（4 个名字含 integration 的只检查常量导出）
- 0 个 E2E 测试
- 16 个核心模块完全没有测试
- 覆盖率阈值全部为 0（statements/branches/functions/lines）
- 结果：改 A 模块时 B 模块悄悄坏掉，CI 绿灯放行

## 目标

建立三层测试安全网，让"改 A 坏 B"能被 CI 拦住。

## 成功标准

1. **Smoke Test**（1 个文件）：Brain 能启动、migrate、health check 返回 200、tick 跑一圈不 crash
2. **Contract Test**（6 个文件）：守住 6 条最危险的模块边界接口
3. **裸奔模块最小测试**（16 个文件）：每个零测试模块至少有 1 个测试验证主函数可调用
4. **覆盖率阈值**：从全 0 提升到 statements: 20, branches: 10

## 非目标

- 不重构任何现有模块
- 不改现有测试
- 不加新功能
- 不删死代码（测试加固完成后再做）

## 技术方案

### Smoke Test
- 文件：`packages/brain/src/__tests__/smoke.test.js`
- 真实启动 Express server（使用 CI PostgreSQL service）
- 执行 migrate
- GET /api/brain/health → 200
- POST /api/brain/tick（dry-run 模式或最小执行）→ 不 crash

### Contract Test
- 目录：`packages/brain/src/__tests__/contracts/`
- 不 mock 被测边界，使用真实模块调用
- 验证函数签名、返回值格式、错误处理

### 裸奔模块最小测试
- 对 16 个零测试模块，每个写 1 个最小测试
- 验证 require() 成功 + 主导出函数可调用

## 影响范围

- 只新增测试文件，不修改现有源码
- 修改 vitest.config.js 的覆盖率阈值
- brain-ci.yml 无需修改（已有 brain-test job）
