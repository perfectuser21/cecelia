# Task Card: 写 write-current-state.sh + 接入 /dev Stage 4

## 需求（What & Why）

**功能描述**：
创建 `scripts/write-current-state.sh` 脚本，负责查询 Brain API 并生成 `.agent-knowledge/CURRENT_STATE.md` 系统状态快照。同时将此脚本接入 `/dev` Stage 4（seal 阶段），确保每次 PR 合并后自动刷新状态。

**背景**：
`.agent-knowledge/CURRENT_STATE.md` 文件已存在，文件头声明 `source: write-current-state.sh`，但该脚本从未被创建。目前该文件是手动维护的，没有自动化更新机制。每次 PR 合并后系统状态可能已变化，但快照依然是旧的。

**不做什么**：
- 不改变 CURRENT_STATE.md 的格式（保留 frontmatter + 表格结构）
- 不修改 Brain API 端点
- 不改变 Stage 4 的合并逻辑，只在末尾添加调用

## 成功标准

1. [ARTIFACT] `scripts/write-current-state.sh` 文件存在且有可执行权限
2. [BEHAVIOR] 执行脚本后 `.agent-knowledge/CURRENT_STATE.md` 被更新，带有正确时间戳
   Test: `manual:node -e "const fs=require('fs');const c=fs.readFileSync('.agent-knowledge/CURRENT_STATE.md','utf8');if(!c.includes('generated:'))process.exit(1);console.log('OK')"`
3. [BEHAVIOR] Brain 离线时脚本不报错（exit 0）
   Test: `manual:bash -c "BRAIN_URL=http://localhost:9999 bash scripts/write-current-state.sh; echo exit_code:$?"`
4. [ARTIFACT] Stage 4 文件 `packages/engine/skills/dev/steps/04-ship.md` 末尾包含对 `write-current-state.sh` 的调用
   Test: `manual:node -e "const c=require('fs').readFileSync('packages/engine/skills/dev/steps/04-ship.md','utf8');if(!c.includes('write-current-state.sh'))process.exit(1);console.log('OK')"`
5. [GATE] CI 全部通过
