# Task: feat(gtd): OKR全树视图重建 — Vision层+完整字段+全展开

## 背景

GTD OKR 页面当前只是一个简陋的树：
- Scope 层默认折叠不可见（需点击4次才能展开到 Scope）
- 所有节点只显示 title + status，缺少 start_date / end_date / owner_role / description
- Vision 层完全缺失（visions 表有数据，objectives 有 vision_id FK，但未查询）
- 无法看到节点的 PRD/笔记（description 字段）

## 改动范围

### `apps/api/src/task-system/full-tree.js`
- Areas 查询加 Vision 子层：`visions` 表 via `visions.area_id` （或直接从 objectives.vision_id 聚合）
- 实际数据结构：当前 objectives 通过 `area_id` 直接挂 Area，通过 `vision_id` 可选挂 Vision
- 策略：如果 objectives 有 vision_id，在 Area 下先聚合 Vision 节点，再挂 Objectives；没有 vision_id 的 objectives 直接挂 Area
- 所有节点 SELECT 加字段：`start_date`, `end_date`, `description`, `owner_role`, `priority`

### `apps/api/features/gtd/pages/GTDOkr.tsx`
- TreeNode interface 加字段：start_date, end_date, description, owner_role, priority
- TreeRow 展示：每行加日期区间、owner、描述（折叠显示）
- 默认展开深度改为全展开（depth < 99 或不限制）
- TYPE_CONFIG 加 vision 类型（橙色/金色）

## 成功标准

- GTD OKR 页面显示 Vision 层（至少有"Cecelia 协助创建一个伟大的公司，要能盈利"这个 Vision）
- 每个节点显示 start_date / end_date（有数据则显示，无则空）
- 每个节点显示 owner_role
- Scope 层默认可见（不需要手动点击）
- /api/tasks/full-tree 的每个节点包含 start_date, end_date, description, owner_role 字段
