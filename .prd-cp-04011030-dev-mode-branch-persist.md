# PRD: .dev-mode.{branch} 状态层持久化

## 背景

devloop-check.sh 所有条件都读 .dev-mode，但 .dev-mode.* 被 gitignore 忽略，上下文压缩后文件消失，devloop-check 失明认为 step_1 pending，agent 从头开始。这是"状态层"的根本漏洞。

## 目标

每次写入 .dev-mode.{branch} 后立即 git add + commit 进功能分支，让 git worktree add 后能自动恢复 step 状态。

## 涉及文件

- `.gitignore`（移除 `.dev-mode.*` 的忽略，只保留忽略 `.dev-mode` 无后缀主 symlink）
- `packages/engine/skills/dev/steps/01-spec.md`（写 .dev-mode 后立即 git add + commit）
- `packages/engine/skills/dev/steps/02-code.md`（同上）
- `packages/engine/scripts/devgate/sprint-contract-loop.sh`（写完 seal 文件 + .dev-mode 后立即 git add + commit）

## 成功标准

- `.dev-mode.{branch}` 随 PR 提交，git worktree add 后自动恢复
- devloop-check.sh 能读到正确的 step 状态（不从头开始）
- CI 通过（engine-ci.yml）
