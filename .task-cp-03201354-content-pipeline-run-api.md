# Task Card: Content Pipeline Run API

## 需求（What & Why）
**功能描述**: 添加手动触发、子任务查询、产出查询三个 API 端点，task-router 改路由
**背景**: 内容工厂 pipeline tick 调度器停了，需要手动触发。前端需要查询子任务进度和产出物。
**不做什么**: 不改 orchestrator/executor 逻辑

## 成功标准
1. [ARTIFACT] POST /pipelines/:id/run 触发执行
2. [ARTIFACT] GET /pipelines/:id/stages 查询进度
3. [ARTIFACT] GET /pipelines/:id/output 查询产出
4. [BEHAVIOR] content-* 路由改 xian

## 验收条件（DoD）
- [ ] content-pipeline.js 包含 POST /:id/run 路由
  Test: manual:bash -c "grep -c 'router.post.*run' packages/brain/src/routes/content-pipeline.js"
- [ ] content-pipeline.js 包含 GET /:id/stages 和 /:id/output 路由
  Test: manual:bash -c "grep -c 'router.get.*stages\|router.get.*output' packages/brain/src/routes/content-pipeline.js"
- [ ] [BEHAVIOR] LOCATION_MAP 中 content-pipeline 路由到 xian
  Test: manual:bash -c "grep -c xian packages/brain/src/task-router.js"
