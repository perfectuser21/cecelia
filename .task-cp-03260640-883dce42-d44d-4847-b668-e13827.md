# Task Card: Capture Digestion Pipeline v1

## 需求（What & Why）

**功能描述**: 为已有的 captures 收件箱建立 AI 消化管道。Brain 扫描 inbox captures → 调 LLM 拆解为原子事件（capture_atoms）→ 用户在 /inbox 页面逐条确认路由到目标表。

**背景**: captures 表（收件箱）和 QuickCapture UI 已建好（PR #1597）。本任务是 Capture 基础设施的第二步：建立消化与路由能力。

**不做什么**:
- v1 不做自动路由（所有 atom 必须人工确认）
- 不做 embedding/向量检索
- 不做批量确认 UI（逐条操作）

## 成功标准

- capture_atoms 表存在，含 capture_id/content/target_type/target_subtype/suggested_area_id/status/routed_to_table/routed_to_id/confidence 字段
- events 表存在，含 name/date/event_type/location/people/area_id 字段
- Brain tick 中的 capture-digestion job 能将 inbox capture 拆解为 atoms（status 变为 processing）
- /inbox 页面新增 Captures Review tab，显示 pending_review atoms
- PATCH /api/brain/capture-atoms/:id 支持 confirm/dismiss 操作，confirm 后写入对应目标表
