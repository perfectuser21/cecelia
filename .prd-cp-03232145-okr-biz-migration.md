# PRD: Migration 182 — 新 OKR 表补充运营列

## 背景

Migration 181 已在旧 `goals`/`projects` 表上建立了双向同步触发器，但新 OKR 表（`visions`、`okr_objectives`、`key_results`、`okr_projects`、`okr_scopes`、`okr_initiatives`）仍缺少业务代码所需的运营列（`description`、`priority`、`progress`、`starvation_score`、`completed_at`、`last_dispatch_at`）。

导致现有 ~50 个 Brain 业务文件无法从旧表迁移到新表。

## 目标

Migration 182 为新 OKR 表补充运营列，并更新触发器函数同步这些列，使后续代码迁移成为可能。

## 成功标准

1. 新 OKR 表各层级有恰当的运营列
2. 触发器同步包含新增列（description、priority、progress 等）
3. selfcheck EXPECTED_SCHEMA_VERSION = '182'
4. 现有测试全部通过

## 不做什么

- 不修改任何业务代码文件（那是 PR4）
- 不迁移 `embedding`/`notion_*` 列（复杂度高，独立 PR）
- 不删除旧表（最终清理步骤）

## 技术方案

### 新增列（按表）

| 表 | 新增列 |
|---|---|
| `visions` | `description TEXT` |
| `okr_objectives` | `description TEXT`, `priority VARCHAR(10) DEFAULT 'P1'` |
| `key_results` | `description TEXT`, `priority VARCHAR(10) DEFAULT 'P1'`, `progress INTEGER DEFAULT 0`, `weight NUMERIC(3,2) DEFAULT 1.0` |
| `okr_projects` | `description TEXT`, `progress INTEGER DEFAULT 0`, `completed_at TIMESTAMPTZ` |
| `okr_scopes` | `description TEXT`, `progress INTEGER DEFAULT 0`, `completed_at TIMESTAMPTZ` |
| `okr_initiatives` | `description TEXT`, `priority VARCHAR(10) DEFAULT 'P1'`, `progress INTEGER DEFAULT 0`, `starvation_score INTEGER DEFAULT 0`, `completed_at TIMESTAMPTZ`, `last_dispatch_at TIMESTAMPTZ` |

### 触发器更新

更新 `sync_goal_to_okr_tables()` 和 `sync_project_to_okr_tables()` 同步新增列。

### selfcheck 更新

`EXPECTED_SCHEMA_VERSION = '182'`
