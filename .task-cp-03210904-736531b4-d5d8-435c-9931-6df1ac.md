# Task Card: 多平台内容发布 Stage 4 闭环激活

## 需求（What & Why）

**功能描述**: 修复内容发布管道 Stage 4（content-export 完成后自动触发多平台发布）的关键断链：
1. `_createPublishJobs` 未将 export findings（manifest_path、card_files）传入 content_publish 任务 payload，导致发布 skill 无法定位内容
2. executor 对 `content_publish` 任务构建的 PRD 不含内容路径，发布 skill 无从下手
3. `_createPublishJobs` 缺少 `description` 字段，任务描述为空

**背景**: content-pipeline 四阶段（research→generate→review→export）已闭合，Stage 4（export→publish）因上述断链导致 content_publish 任务被派发后失败。目标：至少 3 平台稳定自动发布。

**不做什么**:
- 不修改各平台 publisher scripts（已有且可用）
- 不改动 CDP/浏览器连接架构
- 不做发布结果飞书通知（另一个任务）

## 成功标准

1. [ARTIFACT] content_publish 任务 payload 包含 `manifest_path` 和 `card_files` 字段
2. [BEHAVIOR] executor 对 content_publish 任务构建含内容路径的 PRD（包含 manifest_path）
3. [BEHAVIOR] content-export 完成后创建的 content_publish 任务有非空 description 字段
4. [GATE] 所有现有测试通过，新增测试覆盖 manifest_path 传播

## 验收条件（DoD）

- [ ] [ARTIFACT] `_createPublishJobs` 接收 `findings` 参数并写入 `manifest_path`/`card_files` 到 payload
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/content-pipeline-orchestrator.js','utf8');if(!c.includes('manifest_path: manifestPath'))process.exit(1);console.log('OK')"

- [ ] [ARTIFACT] `_createPublishJobs` 写入非空 `description` 字段到 INSERT
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/content-pipeline-orchestrator.js','utf8');if(!c.includes('description'))process.exit(1);console.log('OK')"

- [ ] [BEHAVIOR] `executor.js` 为 `content_publish` 任务构建含 manifest_path 的 PRD
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/executor.js','utf8');if(!c.includes('manifest_path'))process.exit(1);console.log('OK')"

- [ ] [BEHAVIOR] `advanceContentPipeline` 将 findings 传给 `_createPublishJobs`
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/content-pipeline-orchestrator.js','utf8');if(!c.includes('_createPublishJobs(dbPool, pipeline, task, findings)'))process.exit(1);console.log('OK')"

- [ ] [GATE] 所有现有测试通过
  Test: manual:bash -c "cd packages/brain && npm test -- --run 2>&1 | tail -10"

## 实现方案

**要改的文件**:
1. `packages/brain/src/content-pipeline-orchestrator.js`
   - `advanceContentPipeline` (content-export 分支): 传 findings 给 `_createPublishJobs`
   - `_createPublishJobs`: 接受 `findings=null`，写 description + manifest_path + card_files
2. `packages/brain/src/executor.js`
   - `buildSkillPrompt`: 添加 content_publish 专用 PRD 构建逻辑
3. `packages/brain/src/__tests__/content-pipeline-orchestrator.test.js`
   - 更新 INSERT params 位置（description 使 payload 从 [5] 变为 [6]）
   - 新增 manifest_path 传播测试

**Scope 锚定**: Brain 包内，不涉及前端/engine/workflows
