# Task Card: 修复任务成功率统计逻辑（3063% 异常）

## 需求（What & Why）
**功能描述**: 修复 `getTaskStats24h()` 中的 SQL NULL 语义 bug，导致成功率显示 3063%。
**背景**: `payload->>'failure_class' != 'auth'` 在 failure_class 为 NULL（未设置）时返回 NULL 而非 true，导致大量完成任务被排除在 `total` 之外，但 `completed` 计数不受此过滤器影响。当前数据：completed=239、total=8 → 3063%。
**不做什么**: 不修改 `failure_class` 的赋值逻辑，不修改 `routes/stats.js`（该文件逻辑正确）。

## 成功标准

- [x] [BEHAVIOR] SQL 修复后，total >= completed（psql 直接验证）
  Test: manual:psql -U cecelia -d cecelia -c "SELECT CASE WHEN count(*) filter (where status IN ('completed','failed','quarantined') AND payload->>'failure_class' IS DISTINCT FROM 'auth' AND (completed_at > NOW()-INTERVAL '24 hours' OR updated_at > NOW()-INTERVAL '24 hours')) >= count(*) filter (where status='completed' AND completed_at > NOW()-INTERVAL '24 hours') THEN 'OK' ELSE 'FAIL' END as check FROM tasks WHERE task_type != 'pipeline_rescue'"

- [x] [ARTIFACT] 源码中用 IS DISTINCT FROM 'auth' 替代 != 'auth'
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/self-drive.js','utf8');if(!c.includes(\"IS DISTINCT FROM 'auth'\"))process.exit(1);console.log('OK')"

- [x] [ARTIFACT] infrastructure_failure 分类已加入返回字段
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/src/self-drive.js','utf8');if(!c.includes('infrastructure_failure'))process.exit(1);console.log('OK')"

## 实现方案
**要改的文件**: `packages/brain/src/self-drive.js`
**受影响函数**: `getTaskStats24h()` (line ~445)
**修改内容**:
1. `payload->>'failure_class' != 'auth'` → `payload->>'failure_class' IS DISTINCT FROM 'auth'`（`failed` 和 `total` 两处）
2. 在返回的 stats 对象中加入 `infrastructure_failure: auth_failed`（供外部消费和显示）
**不改什么**: SQL 的时间范围逻辑、dispatch-stats.js、routes/stats.js
