# Task Card: CI 覆盖率 Delta 检查 — 新改动行必须有测试覆盖

## 需求（What & Why）

**功能描述**: 在 `.github/workflows/ci-l3-code.yml` 中新增 `coverage-delta` job，
对 feat 类型 PR 验证新改动的行是否有测试覆盖（coverage delta check）。

**背景**:
- Brain 已配置 vitest + `@vitest/coverage-v8`，`vitest.config.js` 已输出 `json-summary` 格式
- `test-coverage-required` 只检查"是否有测试文件改动"，不检查实际覆盖率
- 新需求：对新增/修改的源码行进行覆盖率检查，delta 阈值 0.5%
- 使用 `anuraag016/Jest-Coverage-Diff@main` Action（兼容 vitest 的 json-summary 格式）

**不做什么**:
- 不修改 Brain 源码
- 不修改 vitest 测试文件（coverage 配置已满足需求）
- 不对 [CONFIG] 标签 PR 执行 coverage-delta（CI/配置变更豁免）

## 成功标准

1. [ARTIFACT] `.github/workflows/ci-l3-code.yml` 新增 `coverage-delta` job
2. [ARTIFACT] `coverage-delta` 在 `l3-passed` gate 的 needs 和检查逻辑中正确引用
3. [ARTIFACT] Engine 版本从 12.91.0 bump 到 12.92.0（5 个文件同步）
4. [ARTIFACT] `feature-registry.yml` 新增 12.92.0 changelog 条目
5. [BEHAVIOR] `coverage-delta` job 仅在 `should_run_l3 == 'true'` 且 `brain-unit` 成功时运行
6. [GATE] CI 全部通过

## 实现方案

**要改的文件**:
1. `.github/workflows/ci-l3-code.yml` — 新增 coverage-delta job + 修改 l3-passed gate
2. `packages/engine/VERSION` — 12.91.0 → 12.92.0
3. `packages/engine/.hook-core-version` — 12.91.0 → 12.92.0
4. `packages/engine/package.json` — version 12.91.0 → 12.92.0
5. `packages/engine/package-lock.json` — 两处 12.91.0 → 12.92.0
6. `packages/engine/regression-contract.yaml` — version 12.91.0 → 12.92.0
7. `packages/engine/features/feature-registry.yml` — 新增 12.92.0 changelog

**Scope 锚定**:
- coverage-delta job 放在 test-coverage-required job 之后
- needs: [detect-commit-type, brain-unit]
- if: should_run_l3 == 'true' && brain-unit.result == 'success'
- runs-on: ubuntu-latest（与 brain-unit 一致）
- l3-passed 的 needs 数组中添加 coverage-delta
- l3-passed 检查逻辑中添加对 coverage-delta 的条件检查（只有 brain 有变更时才要求通过）
