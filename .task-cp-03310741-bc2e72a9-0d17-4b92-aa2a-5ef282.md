# Task Card: Seal 文件 commit 到分支 — 防止上下文压缩后状态丢失

## 背景

上下文压缩后 worktree 消失，seal 文件（`.dev-gate-spec/planner/crg/generator-sprint/lite` +
`.sprint-contract-state`）全部丢失，导致 devloop-check 无法继续、Sprint Report 评分归零。

## 目标

Sprint Contract 完成后、第一次 push 前，把所有 seal 文件 commit 进功能分支。
这样 `git worktree add` 后 `git checkout` 自动还原所有防伪凭证。

## 实现方案

1. **sprint-contract-loop.sh**：`blocker_count == 0` 时，exit 0 前先 git add + git commit 所有 `.dev-gate-*.{branch}` 和 `.sprint-contract-state.{branch}`
2. **02-code.md**：Generator seal 写入后追加 git add + git commit 步骤
3. **04-ship.md**：push 前增加 seal 文件安全网 commit（补漏）
4. **Engine 版本 bump**：13.75.0 → 13.75.1（5 个文件）+ feature-registry.yml + generate-path-views.sh

## commit 类型

`fix(engine):` — 修复上下文压缩导致 seal 文件丢失的 bug
