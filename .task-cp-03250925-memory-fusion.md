# Task Card: Memory Fusion — Claude Code auto-memory ↔ Cecelia 文档系统

## 需求（What & Why）

**功能描述**：打通 Claude Code auto-memory（36个 memory/*.md 文件）与 Cecelia DB 文档系统（design_docs / decisions 表），让对话知识沉淀在 Dashboard 可见。

**背景**：

- design_docs 只有 1 条空日记

**不做什么**：

## 实现方案

### 1. memory-sync.js（新建）
- 扫描 `~/.claude-account1/projects/.../memory/*.md`
- 解析 frontmatter（name / type / description）
- `project` / `reference` 类型 → INSERT INTO design_docs
- `feedback` 类型 → INSERT INTO decisions（category='process'）
- 幂等：用 title 查重，已存在则跳过
- 导出 `memorySyncIfNeeded(pool)` 供 tick.js 调用

### 2. backfill-decisions.js（新建）
- 解析全局 `~/.claude/CLAUDE.md`（行为规则）
- 解析项目 `.claude/CLAUDE.md`（项目规则）
- 提取每条规则写入 decisions（category='architecture' 或 'process'）
- ON CONFLICT (topic) DO NOTHING（防止重复）

### 3. tick.js（修改）
- import memorySyncIfNeeded from './memory-sync.js'
- 在 tick 函数中每 N 轮调用一次（约每 30 分钟）
