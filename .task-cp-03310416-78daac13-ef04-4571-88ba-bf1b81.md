# Task Card: content-type YAML 固定 notebook_id + orchestrator 传参 + export 清空 notebook

## 需求（What & Why）

**功能描述**: solo-company-case 内容类型固定绑定指定 NotebookLM notebook，research 阶段子任务自动携带 notebook_id，export 完成后自动清空 notebook 源以供下次复用。

**背景**: content-pipeline 的 research 阶段需要知道目标 NotebookLM notebook 的 ID 才能正确写入源材料；export 完成后 notebook 中残留的源会干扰下一次 pipeline 的运行，需要自动清理。

**目标**: 
- solo-company-case.yaml 中固定声明 notebook_id
- orchestrator 创建 research 子任务时将 notebook_id 透传到 payload
- export 阶段完成后 notebook 内所有源被自动清空，notebook 恢复为空白可复用状态

## 成功标准

- solo-company-case.yaml 包含 `notebook_id: 1d928181-4462-47d4-b4c0-89d3696344ab`
- 启动 solo-company-case pipeline 时，research 子任务的 payload 中可观察到 notebook_id 字段
- export 阶段结束后，对应 notebook 的源列表为空（listSources 返回空数组）
