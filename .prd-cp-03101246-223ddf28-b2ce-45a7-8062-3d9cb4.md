# PRD - feat: GET /api/brain/projects/compare — KR进度 + 趋势数据

**需求来源**: Brain 自动调度任务，Initiative「项目对比数据 API」
**功能描述**: 在 PR #783（POST /api/brain/projects/compare/report）基础上，新增 GET /api/brain/projects/compare 端点，返回 KR 进度和历史趋势数据
**涉及文件**:
- `packages/brain/src/project-compare.js` — 新增 getCompareMetrics() 函数
- `packages/brain/src/routes.js` — 新增 GET /projects/compare 路由
- `packages/brain/src/__tests__/project-compare.test.js` — 追加测试

**成功标准**:
- GET /api/brain/projects/compare?ids=uuid1,uuid2 返回 200 且含 projects 数组
- projects[0].kr.progress 是整数（或 kr 为 null 当无关联 KR 时）
- projects[0].trend 是数组，每项含 week（如 "2026-W10"）和 completed（整数）
- ids 少于2个时返回 400
- 现有 generateCompareReport 的 7 个测试仍全通过
- Brain CI（brain-unit）通过

**非目标**: 不修改现有 generateCompareReport 函数

## 架构细节

### getCompareMetrics({ project_ids, format='json', trend_weeks=4 })

并行执行 3 个查询：
- 查询A: 项目基本信息 + KR 关联 (LEFT JOIN goals)
- 查询B: 任务统计（同现有 generateCompareReport SQL）
- 查询C: 历史趋势（按周统计已完成任务）

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| project_ids 少于2个 | 抛 {status:400} 错误 | P3 | compare_metrics_invalid_input |
| project_id 不存在 | 抛 {status:404} 错误 | P3 | compare_metrics_not_found |
| DB 查询失败 | 抛 500 错误 | P2 | compare_metrics_db_error |
