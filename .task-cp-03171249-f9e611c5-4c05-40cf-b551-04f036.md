# Task Card: fix(executor): syncOrphanTasksOnStartup 孤儿改 requeue

## 需求（What & Why）
**功能描述**: 修复 `syncOrphanTasksOnStartup` 逻辑，Brain 重启时不再将所有 in_progress 任务无条件标记为 failed。区分可重试孤儿（Brain 重启中断）和真实失败孤儿。
**背景**: Brain 重启时 in_progress 任务的进程随之终止，但这不等于任务失败——只是被打断了。当前代码将这些任务标记为 failed，导致数据丢失。
**不做什么**: 不改变进程存活任务的 rebuild 逻辑；不修改 startup-recovery.js。

## 成功标准

1. [BEHAVIOR] Brain 重启后，之前 in_progress 且 retry_count < max_retries 且 error_message 为空的任务变为 queued
2. [BEHAVIOR] retry_count >= max_retries 或 error_message 非空的孤儿任务保持标记为 failed
3. [GATE] 单元测试：startup-sync requeue 行为有断言
4. [GATE] CI 全部通过

## 实现方案
**要改的文件**:
- `packages/brain/src/executor.js` — `syncOrphanTasksOnStartup` 函数（约 2308 行）：更新 SELECT 加 retry_count/error_message/task_type，添加 requeue 判断逻辑
- `packages/brain/src/__tests__/startup-sync.test.js` — 新增 requeue 行为测试

**Scope 锚定**:
- max_retries 通过 LEFT JOIN retry_config 获取（per task_type），fallback 默认 3
- 可重试条件：`retry_count < max_retries AND (error_message IS NULL OR error_message = '')`
- requeue 时：status='queued', retry_count++, error_message='requeued after brain restart', 清空 started_at
- 不可重试时：维持现有 failed 逻辑（checkExitReason + error_details）
