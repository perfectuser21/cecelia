# Task Card: Gate 状态防伪 — seal 文件机制

## 任务描述

实现 seal 文件机制，消除 spec_review_status / code_review_gate_status 可被主 AI 直接伪造的 P0 漏洞。

## 改动范围

1. `packages/engine/hooks/bash-guard.sh` — 新增规则 5b：拦截无 seal 文件时写入 gate status
2. `packages/engine/hooks/branch-protect.sh` — 新增：Write/Edit .dev-mode 时验证 seal 文件
3. `packages/engine/lib/devloop-check.sh` — 修改条件 1.5/2.5：读 seal 文件而非仅读 .dev-mode
4. `packages/engine/skills/dev/steps/01-spec.md` — subagent prompt 必须写入 seal 文件
5. `packages/engine/skills/dev/steps/02-code.md` — subagent prompt 必须写入 seal 文件

## seal 文件规范

```
文件名：.dev-gate-spec.<branch>  （spec_review）
        .dev-gate-crg.<branch>   （code_review_gate）
位置：  worktree 根目录
格式：  JSON
{
  "verdict": "PASS" | "FAIL",
  "branch": "<branch>",
  "timestamp": "<ISO8601>",
  "reviewer": "spec-review-agent" | "code-review-gate-agent",
  "issues": [...]
}
```

## 成功标准

- 主 AI 直接写入 `spec_review_status: pass`（无 seal 文件）时被 bash-guard 或 branch-protect 拦截
- 主 AI 直接写入 `code_review_gate_status: pass`（无 seal 文件）时被拦截
- subagent 写入 seal 文件后，主 AI 可正常写入对应 gate status（验证通过）
- devloop-check.sh: seal 文件不存在但 .dev-mode 中有 status: pass → 返回 blocked（自认证检测）
- devloop-check.sh: seal 文件存在且 verdict=PASS → 接受（pass）
- devloop-check.sh: seal 文件不存在且 .dev-mode 无该字段 → pass-through（尚未运行）
