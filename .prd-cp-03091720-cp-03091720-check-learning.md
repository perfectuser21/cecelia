# PRD: check-learning HARD GATE

## 背景
Learning 条目格式不受约束，导致"注意 XXX"等无意义内容通过 CI。
当前 LEARNINGS.md 缺乏强制格式，知识沉淀质量无法保证。

## 目标
在 DevGate CI 新增 `check-learning` job，强制 Learning 条目必须包含结构化格式，确保每条 Learning 都有实质性内容。

## 成功标准
1. PR 的 LEARNINGS.md 新增条目必须包含"根本原因"章节（`## 根本原因` 或 `Root Cause`）
2. PR 的 LEARNINGS.md 新增条目必须包含"下次预防"章节（`## 下次预防` 或 `Prevention`），且至少有 1 条 checklist（`- [ ]` 格式）
3. 违反以上任一规则 → CI exit 1（HARD GATE）
4. 无新增 Learning 条目（PR 未修改 LEARNINGS.md）→ 跳过检查（exit 0）
5. `check-learning` 加入 `ci-passed` job 的 `needs` 列表，确保检查不可绕过
