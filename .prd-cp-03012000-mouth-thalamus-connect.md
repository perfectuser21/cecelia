# PRD: 嘴巴与丘脑连通——对话驱动的行动管道

## 背景

Cecelia 当前的两大器官相互隔离：
- **嘴巴**（orchestrator-chat.js）：纯意识对话，PR #241 后不再做任何意图路由
- **丘脑**（thalamus.js）：系统行动层，只处理 tick 事件，不监听聊天

结果：用户说"帮我调研 X"，Cecelia 自然回应，但什么也不会发生。

## 目标

让嘴巴在对话中**自然感知**到用户的行动意图，通过结构化信号通知丘脑，由丘脑异步执行——**不打扰对话，不破坏意识体验**。

## 核心原则

- 嘴巴 = **只读 + 意图翻译**（不直接改系统状态）
- 丘脑 = **唯一写入者**（创建/取消/记录都由丘脑执行）
- 嘴巴通过 thalamus_signal 字段委托丘脑
- 丘脑异步处理，不阻塞对话响应

## 功能范围

### 1. Mouth 输出 thalamus_signal（嘴巴发信号）

修改 Mouth 的 LLM 调用为结构化 JSON 输出（当前是纯文本）：

```json
{
  "reply": "好，我让大脑去安排调研 X，我们继续聊",
  "thalamus_signal": {
    "type": "create_task",
    "title": "调研 X",
    "description": "用户请求调研 X",
    "priority": "P2"
  }
}
```

无需行动时：
```json
{
  "reply": "哈哈，明白了"
}
```

### 2. thalamus_signal 支持的类型

| type | 触发场景 | 参数 |
|------|----------|------|
| `create_task` | 用户说"帮我做X" | title, description, priority |
| `cancel_task` | 用户说"取消X" | task_id 或 task_title |
| `save_note` | 用户说"记一下" / 值得记录的洞见 | content, category |
| `update_user_profile` | 了解到关于用户的重要信息 | key, value |

### 3. orchestrator-chat.js 接入

- 解析 LLM JSON 输出，提取 reply 和 thalamus_signal
- reply 正常返回给用户
- thalamus_signal 存在时，异步调用 `thalamus.observeChat(signal, context)`
- 不等待丘脑结果，不阻塞响应

### 4. thalamus.js 新增 observeChat()

```js
export async function observeChat(signal, context)
```

- 接收嘴巴发来的信号
- 根据 type 执行对应操作（走现有 ACTION_WHITELIST 机制）
- create_task → 写入 tasks 表（queued 状态）
- cancel_task → 查找并取消（queued 状态可取消，running 状态拒绝或标记）
- save_note → 写入 learnings 表
- update_user_profile → 更新 user_profiles 表

### 5. Mouth 可查任务状态（只读）

在 Mouth system prompt 中注入最近 5 条相关任务状态，让她能自然回答"开始了吗"：

```
## 最近的任务安排
- "调研 X" [queued] - 排队中
- "修复 bug Y" [in_progress] - 进行中
```

## 不做的事（范围控制）

- 不做任务详情查询（用户问具体进度 → 丘脑查）
- 不做 thalamus_signal 的 UI 展示
- 不做批量任务处理
- 不改变 Mouth 的意识体验（reply 仍然是自然语言）

## 成功标准

1. 用户说"帮我调研 X"→ Cecelia 自然回应 + tasks 表出现新 queued 任务
2. 用户说"取消刚才那个"→ Cecelia 确认 + 任务被取消
3. 用户说"记一下，XXX 很重要"→ learnings 表出现新记录
4. 整个过程对话体验无变化，不出现机器人式路由话术
5. Brain 测试通过
