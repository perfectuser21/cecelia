# PRD: branch-protect.sh 修复 monorepo 子目录错误复用根目录 PRD 的 bug

## Task 信息

- **Task ID**: 1ddd370a-b80c-429d-adc9-a464917d29a3
- **来源**: Brain 数据库

## 功能描述

`packages/engine/hooks/branch-protect.sh` 的 `find_prd_dod_dir()` 函数从被写入文件的位置向上查找 PRD/DoD 文件。

**Bug 场景**：
- 开发者在 `packages/workflows/skills/zhihu-publisher/scripts/` 下创建新文件
- `find_prd_dod_dir()` 向上找，没找到 PRD，一直找到仓库根目录
- 根目录有一个旧的 `.prd.md`（其他任务留下的）
- branch-protect.sh 用旧 PRD 做检查 → "PRD 未更新" 错误

**正确行为**：
- 如果文件在 `packages/xxx/` 子目录下，且查到的 PRD 目录是根目录，应该检查根目录是否有**当前分支**的 per-branch PRD（`.prd-{branch}.md`）
- 如果根目录也没有 per-branch PRD，则报错"请在根目录创建 .prd-{branch}.md"，而不是用旧的全局 `.prd.md`

## 涉及文件

- `packages/engine/hooks/branch-protect.sh` — 修复 `find_prd_dod_dir()` 函数
- `packages/engine/package.json` + 相关版本文件 — Engine 版本 bump
- `packages/engine/scripts/devgate/feature-registry.yml` — 新增 changelog 条目
- `packages/engine/__tests__/` 或测试脚本 — 新增测试用例

## 成功标准

- 在 `packages/workflows/` 子目录下开发时，若根目录只有全局 `.prd.md`（无 per-branch PRD），hook 报错拒绝
- 在 `packages/workflows/` 子目录下开发时，若根目录有正确的 `.prd-{branch}.md`，hook 通过
- 不影响在仓库根目录下直接开发的场景
- 不影响在 `packages/brain/`、`packages/engine/` 等子目录下有自己 PRD 文件的场景

## 非目标

- 不修改 DoD 检查逻辑（DoD 与 PRD 相同路径查找，一并受益）
- 不修改 CI 流程，仅修改本地 hook 行为
