# PRD: 疏通 Slot 分配系统

## 背景
Cecelia Brain 的 slot 分配链路被多层静态 Cap 截断，导致动态资源探测形同虚设。

## 问题
1. `platform-utils.js` 中 `MAX_PHYSICAL_CAP=10` 硬截断物理容量
2. `executor.js` 中 `MEM_PER_TASK_MB=800` 不符合实测（实际 300-400MB）
3. `capacity.js` 与 `platform-utils.js` 存在重复的物理容量计算（三个文件三个不同的值：800/500/350）
4. `slot-allocator.js` 中 Token 压力被当作并发限制（应只在选账户时轮转）
5. Pool A/B/C 三池静态预分配与动态探测结果矛盾
6. `CECELIA_BUDGET_SLOTS=7` 环境变量再次截断
7. 结果：7 slot 总预算，4 用户 + 1 headroom + 1 cecelia + 1 taskPool = 只剩 1 个可派发 slot

## 成功标准

1. `MAX_PHYSICAL_CAP` 提高到 20，`MEM_PER_TASK_MB` 统一为 400（三个文件一致）
2. slot-allocator 中 Token 压力不再限制 Pool C 并发数
3. 简化 Pool 分配：去掉固定预留，改为动态扣减
4. dispatchAllowed 基于 effectiveSlots 减去实际运行数
5. 所有现有测试通过
