## PRD - 修复 planner.js 中 quarantined 任务过滤

**需求来源**: Cecelia 欲望系统反馈 — "9轮诊断了，我自己去改。两个文件、不超过10行，修完你来看结果。"

**功能描述**:
修复 `packages/brain/src/planner.js` 中 `initiative_plan` 任务检测逻辑，使其正确排除 `quarantined` 状态的任务。

**问题分析**:
当前代码在检查是否已有 `initiative_plan` 任务时（planner.js:457），使用：
```sql
WHERE project_id = $1 AND task_type = 'initiative_plan'
AND status NOT IN ('completed', 'failed', 'cancelled')
```

这会将 `quarantined` 状态的任务视为"存在"，导致系统跳过创建新任务，使 Initiative 永远卡住。

**修复方案**:
在 planner.js:457 的排除列表中加入 `'quarantined'`，使系统在遇到隔离任务时创建新任务重试。

**涉及文件**:
- `packages/brain/src/planner.js`（行 457）

**成功标准**:
1. planner.js:457 的 SQL 查询排除 `'quarantined'` 状态
2. 本地测试：查询 SQL 语法正确
3. CI 通过（brain-ci.yml）
4. PR 合并到 main

**非目标**:
- 不修改其他任务选择逻辑（行 181、335等已正确使用 `IN ('queued', 'in_progress')`）
