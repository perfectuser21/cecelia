# PRD: 记忆系统 PR5 — Dopamine RPE（奖赏预测误差）

Brain Task ID: 8b7fcd20-6672-4b09-bdbb-52b84f876632

## 背景

当前 dopamine.js 只记录实际奖赏（actual reward），没有期望奖赏基线。
经典多巴胺神经科学模型的核心是 RPE（Reward Prediction Error）= actual - expected：
- RPE > 0：比预期好 → 正向学习信号，强化该类任务
- RPE < 0：比预期差 → 负向学习信号，降低同类任务优先级
- RPE ≈ 0：符合预期 → 维持现状

## 目标

1. 新增 `getExpectedIntensity(taskType, skill, dbPool)` — 查询同类任务历史平均奖赏强度
2. 新增 `computeRPE(actualIntensity, expectedIntensity)` — 纯函数计算 RPE 值
3. 新增 `recordExpectedReward(taskId, taskType, skill, dbPool)` — 任务开始时记录期望奖赏
4. `recordReward` 在写入实际奖赏时，自动查询并计算 RPE，写入 cecelia_events（event_type='rpe_signal'）

## 成功标准

- dopamine.js 存在 getExpectedIntensity、computeRPE、recordExpectedReward 三个函数
- computeRPE 返回 actual - expected，正负值语义正确
- recordReward 在有期望奖赏记录时自动写入 rpe_signal 事件
- getExpectedIntensity 查询最近同类任务的历史均值（N=10）
