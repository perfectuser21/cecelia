## PRD - Dashboard 添加 Codex 用量展示

**需求来源**: 用户直接描述 — 在前端 dashboard 上展示 Codex 5个账号的用量，与 Claude Code 账号展示风格一致

**功能描述**:

在 Cecelia dashboard 的 AccountUsagePage 和 LiveMonitorPage 中添加 Codex 账号用量展示。
数据从西安 Mac mini 的 codex-bridge（100.86.57.69:3458）的 /accounts 端点获取。

### 变更点

1. **Brain 后端**：添加 GET /api/brain/codex-usage 代理 API
   - 从 codex-bridge /accounts 拉取 5 个账号的用量数据
   - 缓存 3 分钟（与 Claude 账号一致）
   - 返回格式与 Claude account-usage 对齐

2. **前端 AccountUsagePage**：添加 Codex 区域
   - 展示 5 个账号（team1-team5）的用量
   - 指标：primaryUsedPct（5h）、secondaryUsedPct（7d）
   - 警告阈值：≥80% 红色
   - 与 Claude 账号同页面展示

3. **前端 LiveMonitorPage**：添加 Codex Ring
   - 5 个圆形进度圈，与 Claude 3个 Ring 同排展示
   - 颜色逻辑与 Claude 一致

**涉及文件**:
- `packages/brain/src/routes.js` — 添加 codex-usage 端点
- `apps/dashboard/src/pages/account-usage/AccountUsagePage.tsx` — 添加 Codex 区域
- `apps/dashboard/src/pages/live-monitor/LiveMonitorPage.tsx` — 添加 Ring

## 成功标准

1. GET /api/brain/codex-usage 返回 5 个 Codex 账号的用量数据
2. AccountUsagePage 展示 Codex 5 个账号的 5h 和 7d 用量百分比
3. LiveMonitorPage 展示 Codex 5 个 Ring 圆形进度圈

**非目标**:
- 不修改 codex-bridge 本身
- 不添加 Codex 账号选择/调度逻辑（已有）
