# Task Card — fix(brain): 修复 circuit_breaker.status 在 HALF_OPEN 时显示 all_closed 的误导性问题

## 背景
Brain API degraded 诊断任务发现：`cecelia-run` 熔断器处于 HALF_OPEN（恢复中）状态时，
`GET /api/brain/health` 仍返回 `circuit_breaker.status: "all_closed"`，
与 `states.cecelia-run.state: "HALF_OPEN"` 形成矛盾，使运维人员无法判断系统是否正在恢复。

**诊断结论**：
- degraded 根因：内容管道 P0 failures（research findings 为空）触发 cecelia-run 熔断器 OPEN
- 现状：熔断器已过渡到 HALF_OPEN（系统在自动恢复中）
- 代码 bug：`openBreakers` 只过滤 OPEN 状态，HALF_OPEN 不被包含 → `all_closed` 误报

## 成功标准

当 cecelia-run 处于 HALF_OPEN 时，`/api/brain/health` 返回：
- `circuit_breaker.status: "recovering"`（而非 `all_closed`）
- `circuit_breaker.half_open: ["cecelia-run"]`（方便运维识别）
- 顶层 `status: "healthy"`（HALF_OPEN 不触发 degraded，仅影响 CB 子字段）
