# PRD: feat(capture) — Capture Digestion 基础设施（PR1: 建表+API）

## 背景

captures 表（收件箱）已建好（PR #1597），用户可通过 QuickCapture 输入原始内容。
现在需要建立消化管道的基础设施：capture_atoms 表（拆解结果）+ events 表（生活事件）+ CRUD API。

## 目标

1. 建立 `capture_atoms` 表 — 存放 AI 拆解后的原子事件
2. 建立 `events` 表 — 存放生活事件（旅行/聚餐/看病等）
3. 提供 capture_atoms CRUD API — 支持 Review 流程
4. 提供 events CRUD API — 支持生活事件管理

## 成功标准

- capture_atoms 表存在，含 capture_id/content/target_type/target_subtype/suggested_area_id/status/routed_to_table/routed_to_id/confidence 字段
- life_events 表存在，含 name/date/event_type/location/people/area_id/description/capture_atom_id 字段
- GET /api/capture-atoms 返回 atoms 列表（支持 ?status=pending_review）
- POST /api/capture-atoms 创建 atom 返回 201
- PATCH /api/capture-atoms/:id 支持 confirm（路由到目标表）和 dismiss 操作
- GET /api/life-events 返回 events 列表
- POST /api/life-events 创建 event 返回 201
- Brain 迁移文件已创建（199_capture_atoms_events.sql）
