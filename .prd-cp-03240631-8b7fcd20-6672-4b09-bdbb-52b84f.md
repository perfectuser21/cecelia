# PRD: 记忆系统 PR5 — Dopamine RPE（奖赏预测误差）

## 背景

当前 dopamine.js 只记录实际奖赏，没有期望奖赏基线。PR5 目标：新增 RPE（Reward Prediction Error）计算：任务开始时记录期望奖赏（基于历史同类任务均值），完成时计算 actual-expected 差值作为 RPE，写入 cecelia_events。RPE>0 增强学习，RPE<0 降低同类任务优先级。

## 成功标准

- `packages/brain/src/dopamine.js` 新增 `computeExpectedReward`、`recordRPE`、`getRPEHistory` 三个导出函数
- `scanAndReward` 完成任务后自动调用 `recordRPE`，写入 `event_type='rpe_signal'` 到 `cecelia_events`
- RPE < 0 时写入 `brain_config` key `rpe_adjustments`，记录该 task_type 累积惩罚
- 现有所有导出和测试不受影响
