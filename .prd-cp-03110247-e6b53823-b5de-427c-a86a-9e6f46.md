## PRD - feat PR 变更行覆盖率硬门禁

**需求来源**: Brain 自动调度任务
**功能描述**: 在 CI 中新增三个确定性硬门禁，防止 AI 自我验证死循环（写代码没真测）
**涉及文件**:
- `packages/engine/scripts/devgate/check-changed-coverage.cjs`（新增）
- `.github/workflows/engine-ci.yml`（L3 Code Gate 新增覆盖率检查 job）
- `packages/engine/feature-registry.yml`（新增 changelog 条目）
- `packages/engine/VERSION` 等版本文件（bump）

**成功标准**:

## 成功标准

- feat: PR 无新增 .test.ts 文件时 CI 拒绝（exit 1）
- 新增 .test.ts 不 import 任何本 PR 修改的源码文件时 CI 拒绝（exit 1）
- 本 PR 变更行覆盖率 < 60% 时 CI 拒绝（exit 1）
- fix:/docs:/chore: PR 不受影响（向后兼容，直接 exit 0）
- engine-ci.yml L3 Code Gate 新增 changed-coverage job

**非目标**:
- 不修改现有测试文件
- 不改变覆盖率阈值配置方式（硬编码 60%）
- 不支持 .test.js（仅限 .test.ts）

### 异常处理 & 报警规范

N/A - 无 IO 操作（纯 CI 脚本，不涉及外部调用）
