# PRD: Pipeline 安全漏洞修复

## 背景

Cecelia Engine pipeline 存在 4 个安全漏洞，AI 可以通过以下方式绕过安全门禁：
1. 伪造 seal 文件（只需 `echo '{}' > .dev-gate-spec.branch`）
2. devloop-check.sh 加载失败时降级到无 seal 验证的旧路径
3. flock 不可用时静默返回 0 导致无锁并发
4. PR 合并前状态窗口期（检查 CI 通过后到合并执行之间状态可能变化）

## 目标

修复这 4 个安全漏洞，增强 pipeline 的 fail-closed 特性。

## 成功标准

1. 伪造 seal 文件（verdict 为空或非 PASS）被拦截
2. divergence_count 字段缺失时被拦截
3. stop-dev.sh fallback 路径直接 exit 2（fail-closed）
4. flock 不可用时返回 1 并输出错误
5. PR 合并前额外检查 mergeable 状态，CONFLICTING 时拦截

## 范围

仅修改 packages/engine 模块：
- `packages/engine/lib/devloop-check.sh`
- `packages/engine/hooks/stop-dev.sh`
- `packages/engine/lib/lock-utils.sh`
