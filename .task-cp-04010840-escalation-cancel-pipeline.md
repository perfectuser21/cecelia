# fix(escalation): cancel_pending 排除 content pipeline 子任务

**Task ID**: fc04ee9d-cddf-4b51-8605-1d0a944d354f
**Branch**: cp-04010840-escalation-cancel-pipeline
**Priority**: P0

## 问题

`cancelPendingTasks` 的 `task_type NOT IN` 排除列表缺少 content pipeline 子任务类型，
导致 Escalation 定期执行 `cancel_pending` 时会取消正在等待执行的 content-copywriting 等子任务，
pipeline 永久卡在 `in_progress` 状态（"永远等待中"）。

## 修复

在 `packages/brain/src/alertness/escalation.js` 的排除列表中加入所有 content pipeline 子任务类型。
