# Task Card: bash-guard.sh 补漏 main 分支 Bash 写文件拦截

## 需求（What & Why）

**功能描述**: bash-guard.sh 规则 3 只拦截特定代码扩展名（.js/.sh 等）的重定向写入，但 `git show <ref>:path > packages/xxx` 这类命令能绕过，导致可以在 main 分支直接用 Bash 覆盖源码文件。需要增加通用拦截：在 main/develop 分支时，任何重定向写入 `packages/`、`apps/`、`scripts/`、`hooks/` 目录的命令都被拦截。

**背景**: 发现了真实的绕过案例：`git show origin/main:packages/brain/src/routes.js > packages/brain/src/routes.js`，这条命令在 main 分支执行时 bash-guard.sh 没有拦截（Write/Edit 工具有保护，但 Bash 重定向没有）。

**不做什么**: 不改现有规则 3 的逻辑，新增规则 3b 作为补充。

## 成功标准

1. [ARTIFACT] bash-guard.sh 包含规则 3b，拦截对 packages/apps/scripts/hooks/ 目录的 Bash 重定向写入
2. [BEHAVIOR] main 分支时 `git show ... > packages/x` 类命令被拦截（exit 2）
3. [BEHAVIOR] cp-* 分支时同样命令放行（不拦截）
4. [GATE] engine CI 全部通过
