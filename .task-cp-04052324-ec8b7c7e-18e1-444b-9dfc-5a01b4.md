# Task Card: Dashboard v1 路由修复 + content analytics 对接

## 需求（What & Why）
**功能描述**: PR #1940 已合并 TeamDashboardV1（5个面板），但 `/api/brain/publish-results/stats` API 有路由冲突，导致500错误。PR #1941 已合并 content_analytics 表和 API，Dashboard 未接入。
**背景**: Brain server.js 第166行将 contentPipelineRoutes 挂载到 `/api/brain`，其 `GET /:id/stats` 路由在第177行的 brainRoutes（含 publishResultsRouter）之前，导致 `/publish-results/stats` 被误匹配。
**不做什么**: 不重写 Dashboard，不改 contentPipelineRoutes 路由结构。

## 成功标准
> [BEHAVIOR] /api/brain/publish-results/stats 返回 200（不再 500）
> [BEHAVIOR] fetchContentPerformance 使用 /analytics/content 端点

## 验收条件（DoD）

- [x] [BEHAVIOR] server.js 中 brainRoutes 挂载顺序在 contentPipelineRoutes 通用挂载之前
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/server.js','utf8');const bi=c.indexOf(\"use('/api/brain', brainRoutes)\");const ci=c.indexOf(\"use('/api/brain', contentPipelineRoutes)\");if(bi>ci)process.exit(1);console.log('ok, brainRoutes at',bi,'contentPipelineRoutes generic at',ci)"

- [x] [BEHAVIOR] fetchContentPerformance 使用 /analytics/content 端点
  Test: manual:node -e "const c=require('fs').readFileSync('apps/api/features/business/api/team-dashboard.api.ts','utf8');if(!c.includes('analytics/content'))process.exit(1);console.log('ok')"

- [x] [ARTIFACT] 测试文件存在
  Test: manual:node -e "require('fs').accessSync('apps/api/features/business/__tests__/team-dashboard-routing.test.ts')"

## 实现方案
**要改的文件**:
- `packages/brain/server.js`: 将 `app.use('/api/brain', brainRoutes)` 移到 `app.use('/api/brain', contentPipelineRoutes)` 之前
- `apps/api/features/business/api/team-dashboard.api.ts`: fetchContentPerformance 使用 /analytics/content
- `apps/api/features/business/__tests__/team-dashboard-routing.test.ts`: 新增测试

**受影响函数/API**: 
- `fetchContentPerformance()`: 改为调用 `/analytics/content?days=N`
- server.js 路由顺序

**不改什么**: TeamDashboardV1.tsx 组件逻辑不改，publish-results.js 路由不改
