# Task Card: 修复 smoke.test.js stdio WriteStream ubuntu 兼容性

## 需求（What & Why）

PR #1022 将 L3/L4 CI runner 从 hk-vps（macOS）迁移到 ubuntu-latest 后，`coverage-baseline` job 中 `smoke.test.js` 失败：

```
TypeError: The argument 'stdio' is invalid. Received WriteStream { fd: null, path: '/tmp/cecelia-bridge.log' }
❯ startCeceliaBridge  server.js:238:17
```

**根本原因**：`server.js:236` 用 `createWriteStream()` 创建日志文件，但 `WriteStream` 的 `fd` 在文件真正打开前是 `null`。ubuntu 上 `spawn` 拒绝接受 `fd: null` 的 WriteStream 作为 `stdio` 参数，macOS 上行为不同。

**修复**：将 `createWriteStream` 改为 `fs.openSync` 获取整数文件描述符（fd），直接传给 `spawn`，消除平台兼容性问题。

**不做什么**：不修改 smoke.test.js；不改变日志文件路径；不改变 bridge 启动逻辑。

## 成功标准

1. [BEHAVIOR] smoke.test.js 不再因 `stdio WriteStream fd: null` 报 TypeError
2. [ARTIFACT] `server.js` 的 `startCeceliaBridge()` 使用 `fs.openSync` 替代 `createWriteStream`
3. [GATE] CI L3 `coverage-baseline` job 通过

## 实现方案

**修改文件**：`packages/brain/server.js`

**改动**（lines 233-255）：
```
// 旧
const logFile = createWriteStream('/tmp/cecelia-bridge.log', { flags: 'a' });
const child = spawn(..., { stdio: ['ignore', logFile, logFile] })

// 新
const logFd = fs.openSync('/tmp/cecelia-bridge.log', 'a');
const child = spawn(..., { stdio: ['ignore', logFd, logFd] })
child.on('exit', () => { try { fs.closeSync(logFd); } catch (_) {} })
```

同时移除 `createWriteStream` import，改为 `import fs from 'fs'`（或补充 `openSync/closeSync`）。
