# Task Card: Dashboard KR5 冲刺 — 三大模块 Demo 阻断 Bug 修复

## 需求（What & Why）
**功能描述**: 修复 Dashboard 三大模块（OKR Roadmap / Collection / Viral Analysis）中影响 20 分钟演示的阻断 Bug
**背景**: KR5 目标是 Dashboard 可交付（58%→75%），三大模块需无阻断 bug 可流畅演示
**不做什么**: 不改 Brain 后端、不修改 Collection/Viral 页面（数据为 0 是正常数据问题）

## 成功标准

### Bug 1 — RoadmapPage: Now 列永远为空
- Brain projects 使用 `active` 状态，但 `classifyProject` 只检查 `in_progress`
- 修复：`active`/`in_progress`/`queued` → now，`planning` + kr_id → next
- 过滤：隐藏 `inactive` 项目（120个），避免 Later 列爆表

### Bug 2 — OKRPage: Objective 进度永远显示 0%
- Brain `area_okr` 的 `progress/current_value` 均为 null
- 修复：从所有子 KR 的平均值动态计算 Objective 进度

### Bug 3 — OKRPage: 显示 28 个 Objective（含 archived/cancelled）
- 修复：只显示 `status === 'active'` 的 Objective
