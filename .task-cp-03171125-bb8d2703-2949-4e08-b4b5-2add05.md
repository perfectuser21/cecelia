# Task Card: Engine CI 守护规则强化（RCI 健康 + 覆盖率完整性）

## 需求（What & Why）

**功能描述**:
在 CI 流水线中增加三类动态检查：
1. `check-rci-health.mjs` 接入 CI（+新增孤儿检测作为第4项检查）
2. 新建 `check-coverage-completeness.mjs`：源文件存在则必须有对应测试文件
3. `rci-execution-gate.sh` 扩展：在读取 quality RCI 之后，额外扫描 engine RCI 中可在 CI 执行的 `manual:bash` 条目

**背景**:
`check-rci-health.mjs` 已创建（v12.93.3 PR #1006）但未接入 CI。RCI 垃圾条目清理完成后，需要守护规则防止未来再次引入无效条目。覆盖率完整性检查和孤儿检测是双向保护机制。

**不做什么**:
- 不修改 regression-contract.yaml（已在 PR #1006 清理）
- 不运行 engine vitest 测试（已有独立 CI job）
- 不改动 packages/quality/

## 成功标准

1. [ARTIFACT] `check-coverage-completeness.mjs` 存在于 `packages/engine/scripts/devgate/`
2. [ARTIFACT] `check-rci-health.mjs` 增加第4项检查：孤儿 test 文件检测（test 引用的文件在 engine/ 外不存在对应 source 文件时报警）
3. [ARTIFACT] `ci-l1-process.yml` 中有 `check-rci-health.mjs` 调用步骤
4. [ARTIFACT] `rci-execution-gate.sh` 扩展：额外扫描 engine RCI P0 manual:bash 条目并执行
5. [GATE] CI 全部通过

## 验收条件（DoD）

  Test: manual:grep -q "engine-rci\|engine_rci\|ENGINE_RCI" packages/engine/scripts/devgate/rci-execution-gate.sh
