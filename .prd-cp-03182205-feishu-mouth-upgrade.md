# PRD: 飞书嘴巴升级

**分支**: cp-03182205-feishu-mouth-upgrade
**日期**: 2026-03-18

## 背景

当前飞书 P2P 消息处理存在以下问题：
1. 丘脑模型 (thalamus) 使用 MiniMax M2.5-highspeed，需迁移到 gpt-5.4-mini
2. P2P 消息没有防抖聚合（用户快速发多条消息会触发多次 LLM 调用）
3. 回复质量不稳定，缺乏 need_card 判断（长内容应用飞书卡片展示）

## 目标

1. thalamus 模型切换：model_profiles 表中 thalamus 从 MiniMax M2.5-highspeed → gpt-5.4-mini (provider: openai)
2. P2P 消息 3 秒防抖聚合：多条连续消息合并为一个请求
3. thalamus 同时判断：complete / intent / mouth_reply / need_card
4. need_card=true 时：先发飞书卡片（占位符），异步填充 handleChat 结果
5. need_card=false 时：直接发 mouth_reply 文字

## 成功标准

1. model_profiles 表中 thalamus.model = 'gpt-5.4-mini', provider = 'openai'
2. llm-caller.js 支持 openai provider
3. ops.js P2P 消息处理有 3 秒防抖窗口
4. thalamus.js 输出包含 mouth_reply 和 need_card 字段
5. ops.js 根据 need_card 值选择卡片或文字回复
6. migration 157 SQL 文件存在
7. selfcheck.js EXPECTED_SCHEMA_VERSION = '158'
8. 所有 brain 测试通过
