# PRD: decomp-checker type 匹配修正（area_okr 识别）

## 背景
decomposition-checker.js 的 Check A/B/C/D 只搜索 `type IN ('area_kr', 'kr')`，但 goals 表中实际的 KR 数据 type 为 `area_okr`。系统其他模块（kr-progress.js、planner.js、similarity.js）已正确包含 area_okr。

## 目标
修复 decomp-checker 的 SQL 查询条件，将 area_okr 加入 type 搜索范围，与系统其他模块保持一致。

## 成功标准
1. Check A/B/C/D 的 type IN 条件均包含 area_okr
2. 所有现有测试通过
