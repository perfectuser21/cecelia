# Task Card: L4 Runtime Gate 从 self-hosted macOS 迁移到 ubuntu-latest

## 需求（What & Why）
**功能描述**: 将 ci-l4-runtime.yml 的 brain-integration job 从 self-hosted macOS runner (xian-mac-m1) 迁移到 ubuntu-latest
**背景**: 西安 runner 只有 5 个且全部 busy，导致所有 PR 的 L4 无限排队阻塞。迁移到 ubuntu-latest 可以利用 GitHub 的弹性资源池，解除排队瓶颈。
**不做什么**: 不改变测试逻辑、shard 策略、baseline 机制、DevGate 检查

## 成功标准
1. [ARTIFACT] ci-l4-runtime.yml brain-integration job 的 runs-on 改为 ubuntu-latest
2. [BEHAVIOR] PostgreSQL + pgvector 在 ubuntu-latest 上正确安装并启动
3. [ARTIFACT] 删除所有 Homebrew 相关步骤（Set Homebrew PATH、Cache Homebrew、Install/Link Homebrew packages）
4. [BEHAVIOR] 动态端口隔离（shard 分片）机制保持不变
5. [GATE] CI 全部通过

## 验收条件（DoD）

- [ ] [ARTIFACT] runs-on 改为 ubuntu-latest
  Test: manual:node -e "const y=require('fs').readFileSync('.github/workflows/ci-l4-runtime.yml','utf8');if(!y.includes('runs-on: ubuntu-latest'))process.exit(1);console.log('PASS')"

- [ ] [ARTIFACT] 删除 Homebrew 相关步骤
  Test: manual:node -e "const y=require('fs').readFileSync('.github/workflows/ci-l4-runtime.yml','utf8');if(y.includes('Homebrew')||y.includes('brew'))process.exit(1);console.log('PASS')"

- [ ] [BEHAVIOR] PostgreSQL + pgvector 安装步骤替换为 ubuntu 风格
  Test: manual:node -e "const y=require('fs').readFileSync('.github/workflows/ci-l4-runtime.yml','utf8');if(!y.includes('apt-get')&&!y.includes('pgvector'))process.exit(1);console.log('PASS')"

- [ ] [GATE] 所有现有测试通过
  Test: manual:bash -c "node -e \"const y=require('fs').readFileSync('.github/workflows/ci-l4-runtime.yml','utf8');console.log('L4 workflow valid')\""

## 实现方案
**要改的文件**: `.github/workflows/ci-l4-runtime.yml`
**Scope 锚定**: 只改 brain-integration job 的 runner 和 PostgreSQL 安装方式，其余 job 不变
