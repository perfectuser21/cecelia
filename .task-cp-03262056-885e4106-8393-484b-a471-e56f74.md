# Task Card: Content Pipeline 配置可视化 — 前端编辑器 + Executor 注入

## 需求（What & Why）

**功能描述**: 在 Cecelia Dashboard 前端提供内容 pipeline 各内容类型的 prompt/规则可视化编辑界面，同时修改 Executor 从数据库读取配置替代硬编码常量。

**背景**: `content-pipeline-executors.js` 中 `BRAND_KEYWORDS`、`BANNED_WORDS`、prompt 均为硬编码。PR #1611 已合并 `content_type_configs` 表和 CRUD API（`GET/PUT /api/brain/content-types/:type/config`），本任务在其基础上构建前端编辑器和 Executor 注入。

## 成功标准

1. `/content/config` 路由可访问，显示所有内容类型的 prompt/规则编辑器
2. 用户修改 prompt 并点击保存，后端 `content_type_configs` 表更新成功（PUT 请求）
3. `executeCopyReview` 执行时优先使用 DB 中的 `copy_rules.brand_keywords`/`banned_words`，无 DB 配置时 fallback 到 YAML 默认值
