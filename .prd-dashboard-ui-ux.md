# PRD - 仪表盘 UI/UX 设计

## 需求来源
任务来自 Brain 自动调度：设计多维度数据展示界面

## 功能描述
设计一个综合的仪表盘界面，用于展示多维度的任务执行数据：

1. **平台维度**：展示各个平台（Notion、飞书、媒体平台等）的发布统计和成功率
2. **时间维度**：提供 24小时、7天、30天三个时间粒度的趋势分析
3. **成功率维度**：可视化成功/失败/进行中任务的占比和分布

设计需要平衡数据密度和可读性，支持快速洞察和深入分析。

## 成功标准

### UI 设计
- [ ] 响应式布局，支持桌面端和移动端
- [ ] 清晰的信息层级，关键指标突出显示
- [ ] 使用图表（柱状图、折线图、饼图）提升数据可读性
- [ ] 一致的设计语言，符合现有系统风格

### 功能完整性
- [ ] 平台维度：显示至少 5 个平台的统计数据
- [ ] 时间维度：支持切换 24h/7d/30d 三种时间范围
- [ ] 成功率维度：显示成功/失败/进行中任务的百分比
- [ ] 数据自动刷新，默认间隔 30 秒

### 用户体验
- [ ] 加载状态友好，使用骨架屏或加载动画
- [ ] 图表交互流畅，支持悬停查看详细数据
- [ ] 颜色搭配合理，支持浅色/深色主题
- [ ] 数据为空时显示友好的空状态提示

### 性能要求
- [ ] 首屏加载时间 < 2 秒
- [ ] 图表渲染流畅，无卡顿
- [ ] 数据更新不影响用户当前操作

## 非目标
- 不包含数据导出功能（后续版本）
- 不包含自定义仪表盘配置（后续版本）
- 不包含告警和通知功能（后续版本）
- 不实现实时数据流（使用定时轮询即可）

## 技术选型
- 图表库：Recharts 或 Chart.js
- 布局：CSS Grid + Flexbox
- 状态管理：React Context 或组件内状态
- 数据获取：使用现有的 API 端点

## API 依赖
- `GET /api/stats/platform` - 获取平台统计
- `GET /api/stats/timeline?range=24h|7d|30d` - 获取时间序列数据
- `GET /api/stats/success-rate` - 获取成功率数据
