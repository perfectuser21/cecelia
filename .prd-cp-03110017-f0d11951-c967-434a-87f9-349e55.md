## PRD - execution-callback 全字段皆空兜底 failure_class=no_diagnostic

**需求来源**: Brain 自动调度 - routes.js execution-callback 盲区修复
**功能描述**: 当 execution-callback 收到 result/exit_code/stderr/failure_class 四个诊断字段全部缺失时，自动注入兜底 error_message 和 failure_class，确保 tasks 表始终有可追溯的诊断信息
**涉及文件**:
- packages/brain/src/routes.js（execution-callback handler，约 4140-4200 行）
- packages/brain/__tests__/execution-callback-no-diagnostic.test.js（新建单元测试）

**成功标准**:
- POST /api/brain/execution-callback 只带 task_id+status=failed 时，DB 记录 failure_class='no_diagnostic'
- error_message 包含 "no diagnostic data"
- 不影响正常 callback（有任一诊断字段时不触发此兜底）

**非目标**: 不改变其他 callback 路径的行为，不修改 DB schema

### 异常处理 & 报警规范

N/A - 纯条件判断逻辑，无 IO 操作，兜底本身不会失败

## 成功标准
- [ ] 兜底条件：!result && !exit_code && !stderr && !failure_class && status === 'failed' 时注入
- [ ] error_message = "callback received with no diagnostic data (task_id: {task_id})"
- [ ] failure_class = "no_diagnostic"
- [ ] 单元测试覆盖：全字段皆空场景 + 有任一字段时不触发
