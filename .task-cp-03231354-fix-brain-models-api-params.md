# Task Card: 修复 BrainModelsPage 调整模型 API 参数名错误

## 需求（What & Why）

**功能描述**: 修复 `apps/dashboard/src/pages/brain-models/BrainModelsPage.tsx` 中 `handleSaveOrgan` 函数发送请求时参数名错误的 bug。

**背景**: 前端发送 `{ agentId, model, provider }`，但后端 PATCH `/api/brain/model-profiles/active/agent` 要求 `{ agent_id, model_id, provider }`，导致用户点击「保存」后报错 "agent_id and model_id are required"，无法切换模型。

**不做什么**: 不改后端 API；不改页面其他功能；不改样式。

## 成功标准

1. [ARTIFACT] `handleSaveOrgan` fetch body 使用 `agent_id`/`model_id` 字段名
2. [BEHAVIOR] 调用 `/api/brain/model-profiles/active/agent` 时参数名与后端匹配，不再报 400 错误
3. [GATE] Workspace CI 通过，无回归
