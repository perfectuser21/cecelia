# PRD: Token 全部耗尽时阻止任务派发

## 背景
PR #1124 把 token pressure 改为仅监控模式，不再限制 Pool C。但当所有账户 quota 都耗尽（available_accounts=0）时，Brain 仍然派发任务，导致任务秒失败 → requeue → 堆积循环。

## 目标
在 `calculateSlotBudget()` 中增加安全阀：仅当**全部账户耗尽**（token_pressure >= 1.0 且 available_accounts === 0）时阻止派发，轻度压力不限制。

## 成功标准

- token_pressure >= 1.0 且 available_accounts === 0 时 dispatchAllowed = false
- token_pressure >= 1.0 但 available_accounts > 0 时 dispatchAllowed = true
- 轻度压力（token_pressure < 1.0）不影响派发
- 现有测试全部通过
