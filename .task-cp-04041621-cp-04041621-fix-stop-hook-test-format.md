# Task Card: 修复 stop-hook 测试 per-branch 格式

## 需求（What & Why）
**功能描述**: 修复 stop-hook 相关测试文件中的 `.dev-mode` 旧格式引用，改为 `.dev-mode.{branch}` per-branch 格式。
**背景**: stop-dev.sh 在 v14.0.0+ 已改为只识别 `.dev-mode.{branch}` 格式，但测试文件仍在使用旧的 `.dev-mode`（无后缀）格式，导致测试实际上测的是"文件找不到 → exit 0"，没有覆盖任何真实逻辑。
**不做什么**: 不修改 stop-dev.sh 本身；不修改 hook-contracts.test.ts（已是正确格式）。

## 成功标准

> [BEHAVIOR] stop-hook 测试在运行后能找到 .dev-mode.{branch} 文件并触发实际逻辑

## 验收条件（DoD）

- [x] [ARTIFACT] stop-hook-exit-codes.test.ts 中所有 `.dev-mode` 旧格式引用改为 `.dev-mode.{branch}` per-branch 格式
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/tests/hooks/stop-hook-exit-codes.test.ts','utf8');if(c.match(/join\(tempDir, '\.dev-mode'\)/))process.exit(1);console.log('OK')"

- [x] [ARTIFACT] stop-hook-exit.test.ts 中 devModeFile 变量改为 per-branch 格式
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/tests/hooks/stop-hook-exit.test.ts','utf8');if(c.includes(\"join(tempDir, '.dev-mode')\" ))process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] stop-hook-exit-codes.test.ts 运行通过（vitest）
  Test: tests/hooks/stop-hook-exit-codes.test.ts

- [x] [BEHAVIOR] stop-hook-exit.test.ts 运行通过（vitest）
  Test: tests/hooks/stop-hook-exit.test.ts

## 实现方案
**要改的文件**:
  - `packages/engine/tests/hooks/stop-hook-exit-codes.test.ts`
  - `packages/engine/tests/hooks/stop-hook-exit.test.ts`

**关键改动**:
  1. stop-hook-exit-codes.test.ts：
     - `cleanup_done` 测试：需先创建 `git checkout -b test-branch`，然后用 `.dev-mode.test-branch`，同时创建 `.dev-lock.test-branch`
     - `retry_count` 测试：同上
     - `exit code consistency` 测试：去掉旧格式文件，改用正确 per-branch
  2. stop-hook-exit.test.ts：
     - `devModeFile` 从 `join(tempDir, '.dev-mode')` 改为 `join(tempDir, '.dev-mode.cp-test-branch')`（与 beforeEach 中创建的分支名一致）

**不改什么**: hook-contracts.test.ts 已是正确格式，不需改动
