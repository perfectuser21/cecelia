# Task Card: tick 感知模块移到 dispatch 前面

## 需求（What & Why）
**功能描述**: 把 tick.js executeTick() 中的感知/巡航模块从当前位置移到 alertness 评估之后、thalamus 处理之前，确保它们不受任何中间 return（thalamus dispatch、allGoalIds=0、canDispatch=false）影响。

**背景**: 当前 zombie sweep、Pipeline Patrol、health check、initiative-closer、kr-progress 等感知模块夹在 thalamus 和 canDispatch 之间。如果 thalamus 做了 dispatch_task 动作直接 return（第 1529 行），这些感知模块全被跳过。相当于"一紧张就停止思考"——应该反过来，感知永远运行。

**不做什么**: 不改变任何模块的内部逻辑、间隔检查条件、错误处理方式。纯位置移动。

## 成功标准
1. [BEHAVIOR] zombie sweep、Pipeline Patrol、health check、initiative-closer/scope/project completion、kr-progress sync 全部在 thalamus 处理之前执行
2. [BEHAVIOR] canDispatch=false 或 thalamus dispatch_task 时，感知模块仍然正常运行
3. [ARTIFACT] 移动后的代码块有"感知层：不受 canDispatch 限制"注释标记
4. [GATE] CI 全部通过
