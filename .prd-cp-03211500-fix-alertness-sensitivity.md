# PRD: Alertness 过敏修复

## 问题
1. 单次异常就能触发 PANIC/COMA，没有抖动稳定期
2. COMA 自动触发 drain_mode_requested=true，叠加 drain 自杀导致系统停摆
3. API 429 被当成严重信号

## 成功标准
1. PANIC 升级需要连续 3 次 critical 评估，单次 critical 最多升到 ALERT
2. COMA 状态不再设置 drain_mode_requested=true
3. API 429 错误不计入 operationHistory 错误率
4. 所有现有测试通过

## 修复方案
1. alertness/index.js: 添加 _consecutiveCriticalCount 计数器，连续 3 次 critical 才升级 PANIC
2. alertness-actions.js: activateShutdownSafety 中 drain_mode_requested 改为 false
3. alertness/metrics.js: recordOperation 过滤 429 状态码
