# Task Card: verify-step.sh 强制 Sprint Contract 四文件检查

## 需求（What & Why）

**功能描述**: 在 `verify_step1()` 中增加对 Sprint Contract 四个 seal 文件的检查：`.dev-gate-planner`（已有）、`.dev-gate-generator-sprint`（新增存在性检查）、`.dev-gate-spec`（新增 verdict=PASS + divergence_count>=1 检查）、`.sprint-contract-state`（新增 round>=1 检查）。

**背景**: 当前 `verify_step1()` 只检查 `.dev-gate-planner` 存在性 + Task Card Test 字段格式。主 agent 只需伪造一个 planner seal 文件就能绕过 Generator+Evaluator 对抗，直接把 `step_1_spec` 标记为 done，使 Sprint Contract 机制形同虚设。需要在 step1 验证时同时检查所有四个 seal 文件，确保整个 Sprint Contract 流程都真实执行过。

**不做什么**:
- 不修改 verify_step2() 或 verify_step4()
- 不修改 devloop-check.sh
- 不修改 CI 脚本或 DoD 映射逻辑

## 实现方案

**要改的文件**:
- `packages/engine/hooks/verify-step.sh` — 在 `verify_step1()` 中的 Planner seal 检查之后增加三个新检查

**Scope 锚定**: 仅修改 verify-step.sh，不动其他文件

**新增检查位置**: 在现有 `planner_seal_file` 检查块之后，`_pass` 调用之前插入三个新检查块：

1. **Generator seal 检查**：检查 `.dev-gate-generator-sprint.${BRANCH}` 文件存在
2. **Evaluator seal 检查**：检查 `.dev-gate-spec.${BRANCH}` 存在 + `verdict=PASS` + `divergence_count>=1`（用 node -e JSON 解析）
3. **Sprint Contract State 检查**：检查 `.sprint-contract-state.${BRANCH}` 存在 + `round>=1`（用 node -e JSON 解析）
