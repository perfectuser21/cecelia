# PRD - Brain 测试加固 Phase 2: Integration + Mock 瘦身 + 覆盖率

## 背景

Phase 1（PR #558）完成了 Smoke Test、Contract Test、裸奔模块测试。但仍有三个关键缺口：
- 0 个 Integration Test（金字塔倒挂：268 unit / 0 integration / 0 E2E）
- 51% 的测试 mock 依赖（降低测试可信度）
- 2 个文件 0% 覆盖率，routes/ 目录大面积低覆盖

## 目标

### A. Integration Test
- tick → planner → executor 真实链路测试（使用测试 DB）
- 验证跨模块交互：任务规划 → 选择 → 派发的完整流程

### B. Mock 瘦身
- 识别 mock 最重的测试文件
- 将可行的 mock 替换为真实模块调用

### C. 覆盖率提升
- dev-logs.js（0%）和 notion-sync.js（0%）添加基础测试
- routes/ 目录低覆盖文件补测试
- 覆盖率阈值提升：statements 40, branches 25, functions 20, lines 20

## 成功标准

1. 至少 1 个 Integration Test 测试 tick 完整链路
2. 至少 3 个测试文件 mock 用量减少
3. 0% 覆盖率文件消除
4. 覆盖率阈值提升到 40/25/20/20
5. npm test 全部通过
