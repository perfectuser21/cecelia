# Task Card: fix(brain): content-export resvg 渲染失败 — require is not defined

## 需求（What & Why）
**功能描述**: 修复 content-export 阶段中 renderPng 函数因使用 CommonJS require() 而在 ES module 上下文中失败的问题
**背景**: content-pipeline-executors.js 是 ES module（使用 export/import），但 renderPng() 直接调用 `require(resvgPath)`。ES module 没有全局 require，导致 "require is not defined" 错误。每次 content-export 任务执行，所有卡片渲染失败（count=0），generateCards 返回 false，pipeline 被标 failed，陷入无限重试循环。
**不做什么**: 不修改 generateCards 逻辑，不改 pipeline 调度架构，不修复 pipeline 状态 corruption（DB 直接清理）

## 成功标准

> [BEHAVIOR] renderPng 成功渲染 PNG 卡片（不再 require is not defined）
> [ARTIFACT] content-pipeline-executors.js 使用 createRequire 加载 resvg
