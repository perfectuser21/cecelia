## PRD - fix(thalamus): recordLLMError 新增结构化字段

**需求来源**: Brain 自动调度任务（agent_ops 领域）
**功能描述**: thalamus.js recordLLMError() 当前只记录 error message 字符串，缺少结构化字段，导致 Cortex 调用失败后无法快速定位是模型限流、超时还是网络问题。增强 recordLLMError() payload 结构：新增 http_status、elapsed_ms、model、provider、fallback_attempt 字段。
**涉及文件**:
- `packages/brain/src/thalamus.js` — recordLLMError() 函数
- `packages/brain/src/llm-caller.js` — Error 对象 / elapsed_ms 提取
- 相关测试文件

**成功标准**:
- recordLLMError() 接受 opts = { http_status, elapsed_ms, model, provider, fallback_attempt }
- 从 llm-caller.js 的 Error 对象中提取 error.status 作为 http_status
- 从 callLLM 返回值中提取 elapsed_ms（调用耗时 ms）
- recordLLMError 调用方传入结构化字段
- 单测：验证 payload 包含所有新字段
- recordLLMError DB 写入的 payload 含 http_status、elapsed_ms、model、provider

**非目标**: 不修改数据库 schema，payload 为 JSON 字段，无需迁移

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| opts 字段缺失 | 使用 null 降级，不抛出 | P3 | llm_error_partial_payload |
| DB 写入失败 | 已有降级逻辑，保持不变 | P2 | llm_event_write_fail |
