# PRD: Workspace 层级体验完整修复

## 背景

前台 4 个核心视图存在概念混乱和功能缺失：
1. Area 页显示的是 `goals.type='area_okr'`（OKR目标），而非真正的生活领域（areas 表）
2. OKR 页是平铺列表，没有 area_okr → kr 层级
3. Projects 页把 project 和 initiative 混排，看不到父子关系
4. Tasks 774 条数据，前台完全没有入口

## 目标

建立类 Notion 的五层体验：Area → OKR目标 → KR → Project → Initiative → Task

## 功能需求

### 1. Area 页（AreaDashboard.tsx）
- 改用 `/api/tasks/areas` 接口（areas 表，9 个生活领域）
- 显示 name / domain / 关联 project 数量
- 点击进入 AreaDetail 页（显示该 area 下关联的 projects）

### 2. OKR 页（OKRDashboard.tsx）
- 显示 `area_okr` 型 goals 作为父行，`kr` 型 goals 作为子行
- 子行有缩进，可展开/折叠（默认展开）
- 展示：title / status / priority / progress / weight

### 3. Projects 页（ProjectsDashboard.tsx）
- `type='project'` 作为父行，`type='initiative'` 作为子行（按 parent_id 分组）
- 子行有缩进，点击可展开/折叠
- 父行显示子 initiative 数量

### 4. 新增 Tasks 页（TasksDashboard.tsx）
- 新文件：`apps/api/features/planning/pages/TasksDashboard.tsx`
- 调用 `/api/tasks/tasks` 接口
- 显示：title / status / priority / project（initiative名称）/ area
- Brain 后端新增 `/api/brain/tasks/tasks` 路由

### 5. 导航注册
- 在规划模块的 tab 导航中加入 Tasks 入口

## 成功标准
- Area 页显示 9 个 areas 表条目（不是 OKR 目标）
- OKR 页展示 area_okr → kr 层级结构
- Projects 页展示 project → initiative 父子结构
- Tasks 页显示所有任务列表
