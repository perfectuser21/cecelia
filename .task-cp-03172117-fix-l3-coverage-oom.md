# Task Card: 修复 L3 coverage-baseline OOM 导致 main push CI 失败

## 需求（What & Why）
**功能描述**: 给 ci-l3-code.yml 的 `coverage-baseline` job 的 "Run coverage" 步骤加 `continue-on-error: true`，防止 vitest coverage worker OOM 杀死导致 main 推送 CI 失败。

**背景**: `coverage-baseline` 仅在 push 到 main 后运行（更新覆盖率基线），不是任何 gate 的依赖。但 "Run coverage" 步骤缺少 `continue-on-error: true`，OOM 时会让整个 workflow 标红。对比：`coverage-delta` job 的同名步骤已经有 `continue-on-error: true`（line 886）。

**不做什么**: 不改 vitest 配置，不调整 NODE_OPTIONS，不动其他 job。

## 成功标准

1. [ARTIFACT] `coverage-baseline` job 的 "Run coverage" 步骤有 `continue-on-error: true`
2. [ARTIFACT] `coverage-baseline` job 的 "Upload coverage baseline" 步骤也确认有 `continue-on-error: true`（防止 coverage 文件不存在时 upload 失败）
3. [GATE] CI 全部通过

## 验收条件（DoD）

- [x] [ARTIFACT] ci-l3-code.yml `coverage-baseline` job 的 "Run coverage" 步骤包含 `continue-on-error: true`

  Test: manual:node -e "const c=require('fs').readFileSync('.github/workflows/ci-l3-code.yml','utf8');const lines=c.split('\n');const uploadIdx=lines.findIndex((l,i)=>i>910&&l.includes('Upload coverage baseline'));const block=lines.slice(uploadIdx,uploadIdx+5).join('\n');if(!block.includes('continue-on-error: true'))process.exit(1)"
