# Task Card: Watchdog RSS 阈值启动时 sanity check

## 需求（What & Why）
**功能描述**: 在 Brain 启动时（selfcheck.js）验证 Watchdog 的 RSS 阈值配置是否合理，若阈值过小则发出 WARN。
**背景**: Watchdog 的 `RSS_KILL_MB = Math.min(TOTAL_MEM_MB * 0.35, 2400)`，依赖 `os.totalmem()` 运行时计算。若 `TOTAL_MEM_MB` 异常偏小（如容器/测试环境未正确上报），`RSS_KILL_MB` 会极小，导致正常任务被误判 crisis 杀掉。已有失败案例：任务 RSS=1MB 因 pressure>=1.0 触发 crisis 被杀。应在启动时 sanity check 发现此类配置问题。
**不做什么**: 不修改动态阈值计算逻辑；不阻断 Brain 启动（WARN 而非 FAIL）；不改变 Crisis 模式行为。

## 成功标准

1. [ARTIFACT] `selfcheck.js` 新增 Watchdog RSS Sanity 检查：`TOTAL_MEM_MB < 512` 或 `RSS_KILL_MB < 50` 时打印 WARN 日志
2. [BEHAVIOR] Brain 启动时（`runSelfCheck`）输出 Watchdog 当前阈值信息，异常时有 [WARN] 标记
3. [GATE] 所有现有测试通过，新增测试覆盖 sanity check 逻辑
