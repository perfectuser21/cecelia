# Task Card: worktree-gc 保护 blocked 任务的 worktree

## 需求（What & Why）
**功能描述**: worktree-gc.sh 在 --force 模式下会删超过 48h 的 worktree。当任务被 blocked（跨域依赖等待中），worktree 不能被删。修改 worktree-gc.sh 在删除前查询 Brain API 检查对应 branch 的任务是否是 blocked 状态，是则跳过。

**背景**: 将来实现跨域依赖阻塞机制时，agent 退出释放 slot，worktree 保留在磁盘上等恢复。如果 GC 在等待期间删了 worktree，agent 恢复时无法继续，工作成果丢失。

**不做什么**:
- 不改 blockTask/unblockTask 逻辑
- 不改 /dev 流程
- 不加新的 Brain API 端点（复用现有 tasks 查询）

## 成功标准

1. [BEHAVIOR] worktree-gc.sh 在删除前查询 Brain API，blocked 任务的 worktree 跳过删除
2. [BEHAVIOR] Brain 不可用时降级为原有逻辑（不因 API 故障阻塞 GC）
3. [ARTIFACT] worktree-gc.sh 新增 blocked 检查逻辑
4. [GATE] 所有现有测试通过
