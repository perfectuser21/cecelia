# Task Card: Pipeline Integrity Gate

## 需求（What & Why）
**功能描述**: 新增 `packages/engine/tests/pipeline-integrity.test.ts`，作为 meta-test 验证 pipeline 自身的安全不变量没有被削弱。测试覆盖：fail-closed 属性、反模式扫描（N次后放行）、gate 链完整性、关键文件存在性、seal 格式校验。

**背景**: Pipeline 安全属性是整个 CI/CD 系统的基础，但目前没有测试验证 pipeline 自身不被削弱。这个 meta-test 作为 CI 门禁的守护者，确保任何对 pipeline 文件的修改都不会引入安全漏洞（如 fail-open、N次后放行等）。

**不做什么**: 不测试业务功能；不修改现有 pipeline 逻辑；不修改 CI 流程配置（只验证）。

## 成功标准

1. [ARTIFACT] `packages/engine/tests/pipeline-integrity.test.ts` 文件存在，包含 4 个 describe 块
2. [ARTIFACT] `packages/engine/ci/known-failures.json` 中包含 4 条 known-failing 条目（fallback exit 2、flock 失败 return、verdict 字段检查、divergence_count 字段检查）
3. [BEHAVIOR] 运行 `npm test -- pipeline-integrity.test.ts` 时，未被 known-failures 标记的测试全部通过（文件存在性测试、pr-review.yml fail-closed 测试、反模式扫描测试）
4. [GATE] Engine L1 CI 通过（vitest 扫描到此测试文件并执行）
