# Task Card: Generator 独立 subagent — Stage 2 写代码从主 agent 拆出

## 需求（What & Why）
**功能描述**: 把 /dev Stage 2 的代码编写（2.2 写代码）从主 agent 内部拆成独立 Generator subagent。Generator subagent 只接收 Sprint Contract（Task Card）+ coding 规范（CLAUDE.md）+ 代码库上下文，不接收 Planner 的推理过程和 brain context。主 agent 变成纯编排者。

**背景**: 当前 Stage 2 中主 agent 既做探索、又做编码、又做验证，context 窗口压力大。拆出 Generator subagent 后：
1. Generator 的 context 只含必要信息（spec + 代码），质量更高
2. 主 agent 保持编排者角色，与 spec_review/code_review_gate 的 subagent 模式一致
3. 为后续并行 Generator 打基础

**不做什么**:
- 不改 Stage 1（spec）、Stage 3（integrate）、Stage 4（ship）
- 不改 spec_review 和 code_review_gate 的 subagent 调用方式
- 不改 Stop Hook 机制
- 不改 verify-step.sh 或 CI 检查逻辑

## 成功标准

- Stage 2 由主 agent spawn Generator subagent 完成代码编写（2.2 写代码部分）
- Generator subagent 的 prompt 只含 Task Card + CLAUDE.md + 代码库上下文（不含 brain context / planner reasoning）
- Stop Hook 仍然正常工作（.dev-mode / .dev-lock 不受影响）
- 主 agent 在 Generator 完成后继续执行 2.3 自验证 + 2.4 code_review_gate
- 现有测试不受影响
