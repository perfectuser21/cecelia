# Task Card: Engine 状态机三层防御 P0+P1

## 需求（What & Why）
**功能描述**: 堵住状态机双层绕行路：
1. bash-guard.sh 补 `.dev-mode` Bash 写检测 → 调用 verify-step.sh
2. verify-step.sh 通过后写 seal（step_N_seal: verified@timestamp）
3. stop-dev.sh / devloop-check.sh 补 seal 检查：step_N done 但无 seal → exit 2 强制补验

**背景**: AI 可以通过 Bash 工具（`echo ... > .dev-mode.branch`）绕过 PreToolUse:Write 拦截，
直接写入 `step_N: done` 而不经过 verify-step.sh 验证。三层防御堵住这条旁路。

**不做什么**: 不改变现有验证逻辑本身；不添加新的验证步骤类型。

## 成功标准

1. [ARTIFACT] bash-guard.sh 包含 `.dev-mode` 写检测，检测到 `step_N_xxx: done` 时调用 verify-step.sh
2. [BEHAVIOR] verify-step.sh 验证通过后向 .dev-mode 追加 `step_N_seal: verified@<timestamp>`
3. [BEHAVIOR] devloop-check.sh 检查：step_N done 但无 seal → status=blocked，要求重新验证
4. [GATE] CI 全部通过

## 验收条件（DoD）

- [x] [ARTIFACT] bash-guard.sh 规则5：Bash 写 .dev-mode.* 含 step_N_xxx: done 时调用 verify-step.sh，失败则 exit 2
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/bash-guard.sh','utf8');if(!c.includes('dev-mode'))process.exit(1);console.log('ok')"

- [x] [BEHAVIOR] verify-step.sh 验证通过后写 seal 到 .dev-mode
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/hooks/verify-step.sh','utf8');if(!c.includes('step_1_seal'))process.exit(1);if(!c.includes('verified@'))process.exit(1);console.log('ok')"

- [x] [BEHAVIOR] devloop-check.sh seal 检查：step_N done 但无 seal → blocked
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/lib/devloop-check.sh','utf8');if(!c.includes('seal'))process.exit(1);console.log('ok')"

- [x] [GATE] 所有现有测试通过
  Test: manual:bash -c "cd packages/engine && npm test -- --reporter=verbose 2>&1 | tail -20"
