## PRD - /dev Step 10 重写：分类写入 learnings DB

**需求来源**: Brain 自动调度（task_type: dev，domain: agent_ops）

**功能描述**: 重写 /dev Step 10 (Learning 记录)，将 Learning 内容按类型分类写入 Brain PostgreSQL learnings 表，同时保留现有 MD 文件格式兼容性。

**涉及文件**:
- `packages/brain/migrations/` — 新增 migration（learnings 表新增 learning_type、source_branch、source_pr 字段）
- `packages/brain/src/routes/learnings.js` — learnings 写入 API 端点（新建或扩展）
- `~/.claude/skills/dev/steps/10-learning.md` — Step 10 脚本重写（调 Brain API）
- `packages/brain/src/selfcheck.js` — 更新 EXPECTED_SCHEMA_VERSION

## 成功标准
- Step 10 完成后，learnings 表有新记录，包含 learning_type、source_branch、source_pr 字段
- 现有 docs/learnings/<branch>.md 文件仍然生成（向后兼容）
- Learning 分类由 LLM 自动判断，无需开发者手动填写
- CI L1 Learning Format Gate 继续通过

**非目标**:
- 不修改 L1 Learning Format Gate 的本地 MD 文件检查逻辑
- 不实现 Learning 查询 UI（只写入，不展示）
- 不修改 Brain planner 基于 learnings 的决策逻辑

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| Brain API 不可达 | 降级：只写 MD 文件，打印警告 | P3 | learning_db_write_skip |
| LLM 分类失败 | 默认分类为 best_practice，继续写入 | P3 | learning_classify_fallback |
| DB 写入失败 | 打印错误，不中断 Step 10 流程 | P2 | learning_db_write_error |
