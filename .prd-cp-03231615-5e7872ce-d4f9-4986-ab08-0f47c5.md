# PRD: CI Gate — 新增 hook/lib 文件必须有 RCI 条目

## 背景
当 PR 新增 packages/engine/hooks/ 或 packages/engine/scripts/devgate/ 下的文件时，CI L2 检查该文件是否已在 regression-contract.yaml 中有对应 RCI 条目。防止新能力无 RCI 覆盖悄悄滑入生产。

## 成功标准

1. 新增 check-new-files-need-rci.cjs，导出 scanMissingRci(changedFiles, contractContent) 函数
2. CI L2 新增 new-files-need-rci-check job（检测 hooks/ 和 scripts/devgate/ 下新增文件）
3. 三重降级保护（无变更/文件已有 RCI/非目标路径 → 跳过）
4. 测试覆盖率 ≥ 60%
