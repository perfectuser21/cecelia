# PRD: 记忆系统 PR4 — Rumination 质量改进

Brain Task ID: f6062d6e-f08c-4298-b1c5-57500edc5fd0

## 背景

当前 Rumination 仅从 `learnings` 表 FIFO 取输入，不利用 PR2 已写入的高显著性对话记录（memory_stream）。
PR4 让 Rumination 也消化 memory_stream 高显著性条目，形成"对话 → 记忆 → 反刍 → 洞察"完整闭环。

## 目标

1. 新增 `fetchMemoryStreamItems(db, limit)` — 读取 salience_score ≥ 0.7 的 conversation_turn 条目
2. runRumination 在 learnings 不足（数量 < limit）时，补充 memory_stream 条目
3. buildNotebookQuery 在 items 含 emotion_tag 时，将情绪上下文加入查询
4. 消化完成后，已处理的 memory_stream 条目 status 更新为 'ruminated'

## 成功标准

- rumination.js 存在 fetchMemoryStreamItems 函数
- runRumination 在 learnings 不足时补充 memory_stream 条目
- buildNotebookQuery 包含 emotion_tag 上下文（当有时）
- 处理后 memory_stream 条目 status = 'ruminated'
