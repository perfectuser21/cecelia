# Task Card: 执行日志记录器 + 真实质检报告

## 需求（What & Why）
**功能描述**: 在 /dev 工作流的每一步执行中记录事件日志（pass/fail/start），任务完成时用日志生成真实的质检报告，替代现在全是 unknown 的空壳报告。

**背景**: 当前 verify-step.sh 和 devloop-check.sh 有 pass/fail 判断但不记录历史；generate-report.sh 输出 L1/L2/L3 全是 unknown；开发者无法回溯任务过程中哪步出了问题。

**不做什么**: 不改 verify-step.sh 的验证逻辑本身；不改 devloop-check.sh 的状态机逻辑；不新增 Dashboard UI。

## 成功标准
1. [BEHAVIOR] verify-step.sh 每次 pass/fail 时写一行 JSON 到 `.dev-execution-log.jsonl`
2. [BEHAVIOR] devloop-check.sh 每次 blocked/done 时写一行 JSON 到同一日志文件
3. [ARTIFACT] 新增 `packages/engine/lib/execution-logger.sh`，提供统一的日志写入函数
4. [BEHAVIOR] generate-report.sh 从 `.dev-execution-log.jsonl` 读取数据，生成真实的质检报告（替代 unknown）
5. [GATE] 所有现有 Engine 测试通过

## 验收条件（DoD）

- [ ] [ARTIFACT] `packages/engine/lib/execution-logger.sh` 存在，提供 `_devlog_event()` 函数
  Test: manual:node -e "const fs=require('fs');if(!fs.existsSync('packages/engine/lib/execution-logger.sh'))process.exit(1)"

- [ ] [BEHAVIOR] verify-step.sh 在 `_pass()` 和 `_fail()` 中调用 `_devlog_event()`
  Test: manual:bash -c "grep -q '_devlog_event' packages/engine/hooks/verify-step.sh || exit 1"

- [ ] [BEHAVIOR] devloop-check.sh 在返回 blocked/done 时调用 `_devlog_event()`
  Test: manual:bash -c "grep -q '_devlog_event' packages/engine/lib/devloop-check.sh || exit 1"

- [ ] [BEHAVIOR] generate-report.sh 从 `.dev-execution-log.jsonl` 读取并填充 L1/L2/L3 状态
  Test: manual:bash -c "grep -q 'dev-execution-log.jsonl' packages/engine/skills/dev/scripts/generate-report.sh || exit 1"

- [ ] [GATE] Engine 测试通过
  Test: manual:bash -c "cd packages/engine && npm test 2>&1 | tail -5"

## 实现方案
**要改的文件**:
1. `packages/engine/lib/execution-logger.sh` — 新建，日志写入核心函数
2. `packages/engine/hooks/verify-step.sh` — 在 _pass/_fail 中接入日志
3. `packages/engine/lib/devloop-check.sh` — 在 blocked/done 返回点接入日志
4. `packages/engine/skills/dev/scripts/generate-report.sh` — 从日志生成真实报告

**Scope 锚定**: 只改 Engine 包内 4 个文件，不涉及 Brain/Dashboard/Workflows
