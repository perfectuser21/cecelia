# Task Card: 护城河1 — git pre-push 本地 quickcheck

## 需求（What & Why）

**功能描述**：在 `git push` 前强制运行基础检查（TypeCheck + ESLint + 关键 unit test），不通过不允许 push。

**背景**：AI agent 写完代码直接 push，所有问题都留给 CI 发现（8-15 分钟）。push 前 60 秒内完成基础检查，把明显错误拦在本地。

**不做什么**：
- 不重复跑全量 CI（只跑改动相关的子集）
- 不做 migration 冲突检查（已有 pre-push-check.sh 覆盖）
- 不替换 pre-push-check.sh（两者互补：quickcheck 做代码质量，pre-push-check 做 Brain 版本/migration）

## 成功标准

- `scripts/quickcheck.sh` 存在且可执行
- git push 时自动触发 quickcheck，失败则阻止 push
- TypeScript 错误、ESLint 报错、unit test 失败均能被拦截
- `--skip` 参数允许紧急绕过（打印警告）
- 60 秒内完成（路径过滤，非全量）

## 验收条件（DoD）

- [x] [ARTIFACT] `scripts/quickcheck.sh` 文件存在且有可执行权限
  Test: manual:node -e "const fs=require('fs');const s=fs.statSync('scripts/quickcheck.sh');if(!(s.mode&0o111))process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] quickcheck 能检测 TypeScript 错误（改动 packages/engine/ 时触发 tsc --noEmit）
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/quickcheck.sh','utf8');if(!c.includes('tsc --noEmit'))process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] quickcheck 能检测 ESLint 错误（只检查改动文件，非全量）
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/quickcheck.sh','utf8');if(!c.includes('eslint'))process.exit(1);console.log('OK')"

- [x] [BEHAVIOR] quickcheck 支持 `--skip` 参数跳过检查并打印警告
  Test: manual:bash -c "bash scripts/quickcheck.sh --skip 2>&1 | grep -q 'SKIP\|跳过\|skip' && echo OK || exit 1"

- [x] [ARTIFACT] `packages/engine/hooks/` 目录下存在 pre-push hook 文件调用 quickcheck
  Test: manual:node -e "const fs=require('fs');const files=fs.readdirSync('packages/engine/hooks/');const has=files.some(f=>f.includes('pre-push'));console.log(has?'OK':'FAIL');if(!has)process.exit(1)"

- [x] [BEHAVIOR] Engine 版本已 bump（hooks 有改动）
  Test: manual:node -e "const v=require('./packages/engine/package.json').version;const vf=require('fs').readFileSync('packages/engine/VERSION','utf8').trim();if(v!==vf)process.exit(1);console.log('版本一致:',v)"

## 实现方案

**要改的文件**：
1. `scripts/quickcheck.sh`（新建）— 主检查脚本
2. `packages/engine/hooks/pre-push.sh`（新建）— git pre-push hook，调用 quickcheck
3. `packages/engine/feature-registry.yml` — changelog 追加
4. `packages/engine/VERSION` — 版本 bump（已由 linter 更新为 14.3.5）
5. `packages/engine/package.json` — 版本 bump 到 14.3.5
6. `packages/engine/package-lock.json` — 版本 bump
7. `packages/engine/regression-contract.yaml` — 新增 quickcheck 条目（可选）
8. `.hook-core-version`（如存在）— 版本 bump

**受影响函数/API**：无（纯新增脚本）

**不改什么**：
- `scripts/pre-push-check.sh`（已有 Brain 版本/migration 逻辑，保留）
- `packages/engine/hooks/branch-protect.sh`（不动）
- `packages/engine/hooks/stop-dev.sh`（不动）
