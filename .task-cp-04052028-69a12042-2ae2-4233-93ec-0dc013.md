# Task Card: 修复 deploy 并发冲突 — ops.js 互斥锁 + deploy.yml concurrency group

## 需求（What & Why）
**功能描述**: 两处并发保护：
1. `ops.js` POST /api/brain/deploy 加并发锁：当 `deployState.status` 为 `running` 或 `rolling_back` 时，直接返回 409 阻止重复触发
2. `.github/workflows/deploy.yml` 的 `deploy` job 加 `concurrency` group，确保同一时间只有一个 deploy job 运行（cancel-in-progress: false，不取消已运行的）

**背景**: 多次 push 可能导致并发 deploy，互相竞争 pm2 restart / git checkout，造成部署失败或状态混乱。

**不做什么**: 不修改 deploy 流程逻辑、不改回滚、不动 staging deploy。

## 成功标准

- [BEHAVIOR] POST /api/brain/deploy 在 running 状态返回 409
- [ARTIFACT] deploy.yml deploy job 有 concurrency 配置

## 实现方案
**要改的文件**:
- `packages/brain/src/routes/ops.js`：在 POST /api/brain/deploy 认证通过后、更新状态前，检查 `deployState.status` 是否为 `running`/`rolling_back`，若是则 return 409
- `.github/workflows/deploy.yml`：在 `deploy` job 下加 `concurrency: group: deploy-production / cancel-in-progress: false`

**受影响函数/API**: `router.post('/deploy', ...)` 约第 2693 行

**不改什么**: rollback 端点、staging deploy、deploy-local.sh
