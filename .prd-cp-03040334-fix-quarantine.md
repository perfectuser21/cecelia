## PRD - 修复 quarantine.js 隔离区释放逻辑

**需求来源**: Desire System 反思循环分析 (3b72138b-433e-479e-b558-1f6c0041be51)

**功能描述**:
修复 quarantine.js 的三处问题，解锁 initiative_plan 任务补给线：

1. **改进分类器**：增强错误分类能力，减少误判
2. **调整阈值**：从 3 次失败隔离调整为 7 次失败隔离，降低过度隔离
3. **ALERT 级别不阻止释放**：ALERT 是警告级别，不应阻止任务从隔离区释放

**涉及文件**:
- `packages/brain/src/quarantine.js`

**成功标准**:
- [x] 分类器能正确识别常见错误类型（资源、配置、代码缺陷等）
- [x] 隔离阈值从 QUARANTINE_THRESHOLD=3 改为 7
- [x] ALERT 级别的任务可以从隔离区释放
- [x] 单元测试覆盖新逻辑
- [x] CI 通过

**非目标**:
- 不改变隔离区的核心架构
- 不修改 Brain 其他模块
- 不调整释放时机（仍然是每日早晨）

### 异常处理 & 报警规范（REQUIRED）

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| 分类器无法识别错误类型 | 归类为 UNKNOWN，仍然隔离 | P3 | quarantine_classifier_unknown |
| 释放逻辑判断失败 | 保持隔离状态，记录日志 | P2 | quarantine_release_logic_error |
| 数据库更新失败 | 回滚状态，重试 | P1 | quarantine_db_update_failed |
