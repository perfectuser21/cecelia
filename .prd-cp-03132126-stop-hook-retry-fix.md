# PRD: stop hook 重试上限统一 + 双 exit 0 终止条件合并

## 背景

`packages/engine/hooks/stop-dev.sh` 存在两个 bug：
1. 注释写的是"15次重试上限"，但代码检查的是 `RETRY_COUNT > 30`，注释与实现不一致
2. 逻辑顺序有问题：第31次重试时，如果 PR 已合并（完成信号），代码仍然先进入"超时失败"分支，而不是"正常完成"分支

`packages/engine/lib/devloop-check.sh` 存在两个冗余的 exit 0 终止条件：
1. `cleanup_done: true` → exit 0（cleanup.sh 写的）
2. `PR合并 + step_11_cleanup: done` → exit 0（devloop-check 自己判断的）

## 成功标准

1. `stop-dev.sh` 中的 MAX_RETRIES 常量统一为 30，注释与代码一致
2. `stop-dev.sh` 逻辑顺序修复：先调用 devloop_check 判断完成状态，再判断超时（先完成后超时）
3. `devloop-check.sh` 删除冗余的第二个 exit 0 路径，改为调用 `_mark_cleanup_done`（设置 cleanup_done: true），统一通过第一个路径退出
4. 现有测试中关于重试次数 15 的断言更新为 30
5. Engine 版本 bump（6个文件）
6. PR title 包含 [CONFIG]
