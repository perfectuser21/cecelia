# PRD：说明书章节沉浸式体验三连优化

## 背景

说明书手风琴布局 + SVG 配图已完成（PR #324），但体验还差最后一步：
- 点击章节只是就地展开，不像"打开了一章"
- 目录点击没有反应，是静态文字
- SVG 在手风琴里空间受限，不够舒展

## 需求

### ① 章节沉浸式视图（最重要）

点击章节 → 整个说明书区域切换为该章节的专属全屏视图：

```
← 返回说明书    第 2 章  👁 感知层
─────────────────────────────────
[ChapterDiagram SVG 全宽，高 420px]
─────────────────────────────────
[章节角色描述段落]
[2.1 模块 / 2.2 模块 / ...]
```

**交互细节：**
- 顶部固定导航栏：左侧「← 说明书」返回按钮，中间「第 N 章 · 章节名」
- SVG 全宽展示，高度 420px
- 进入/退出用淡入淡出动画（CSS transition，不引入新依赖）
- 返回按钮点击 → 回到手风琴列表

**状态管理：**
- 新增 `detailChapter: string | null` 状态（存 chapter.id）
- `detailChapter !== null` 时整个 ManualView 内容替换为 ChapterDetailView
- 原来的 `openChapter` 手风琴展开状态继续保留，兼容现有行为

### ② 目录点击跳转

目录区域的「第 N 章」点击 → 进入对应章节沉浸式视图

### ③ SVG 高度提升

ChapterDiagram 在沉浸式视图中高度 420px（viewBox 保持不变）

## 技术要求

- 修改 `apps/api/features/brain/pages/SuperBrain.tsx`
- ManualView 组件内部加 `detailChapter` state
- 新增 `ChapterDetailView` 子组件
- 动画用纯 CSS transition
- 暗色主题保持一致
