# Task Card: Knowledge System 补齐 — 导航 + 新增入口 + Markdown

## 需求（What & Why）

**功能描述**: 补齐 Knowledge & Documentation System 的核心缺口，让系统真正可用。

**背景**:
- 5个页面文件存在，路由也注册了，但 navGroups: [] → 侧边栏看不到入口
- DecisionRegistry 调用 /api/brain/decisions 但该端点返回丘脑决策（schema 不匹配），需专属路由
- DesignVault 用 `<pre>` 渲染 Markdown 内容，不格式化
- 所有页面只读，无法在前端新建内容

**不做什么**:
- 不做全文搜索（复杂度高，下次迭代）
- 不引入 marked 等新依赖（用内联简单 Markdown 渲染）
- 不改 Brain 数据表结构

## 成功标准

1. [ARTIFACT] knowledge/index.ts navGroups 非空，包含 knowledge 导航组
2. [ARTIFACT] packages/brain/src/routes/strategic-decisions.js 存在（GET/POST/PUT）
3. [ARTIFACT] server.js 注册 /api/brain/strategic-decisions 路由
4. [ARTIFACT] DecisionRegistry.tsx 使用 /api/brain/strategic-decisions 端点
5. [BEHAVIOR] DecisionRegistry 有「记录决策」按钮，能 POST 创建新决策
6. [BEHAVIOR] DesignVault 有「新建文档」按钮，能 POST 创建 design_doc
7. [BEHAVIOR] DesignVault 内容用简单 Markdown 渲染（# → h1，** → bold，- → li）
8. [BEHAVIOR] DevLog 展开后有评分输入（1-10），能 PUT 更新 self_score
9. [GATE] workspace CI 通过（前端 build 无报错）
