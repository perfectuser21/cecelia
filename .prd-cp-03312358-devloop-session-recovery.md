# PRD: devloop-check 会话压缩恢复入口

## 背景

`.dev-mode.*` 被 gitignore，worktree 消失后 stop hook 找不到当前 stage 进度。
会话压缩重启后 agent 无法知道自己在哪个 stage，需要手动重建状态。

P1（worktree 路径持久化）已在 Engine 13.75.2 合并。
本任务为 P2：让 `bash packages/engine/lib/devloop-check.sh` 直接可用作会话恢复入口。

## 目标

在 `devloop-check.sh` 末尾添加直接执行入口，会话恢复第一步永远是：

```bash
bash packages/engine/lib/devloop-check.sh
```

即可知道：
1. 当前分支和 .dev-mode 文件路径
2. 当前处于哪个 stage
3. 缺少哪些 seal 文件
4. 下一步应该做什么

## 约束

- 不破坏现有 stop hook 对 `devloop_check()` 的 source 调用
- 纯追加（不改现有函数）
- packages/engine 代码变更

## 成功标准

直接执行 `bash packages/engine/lib/devloop-check.sh` 可输出当前 stage 状态和 action。
