# PRD: Brain Session 隔离 + 飞书回复可靠性

## 背景
1. cecelia-bridge 的 `/llm-call` 端点 spawn `claude -p` 时没有指定 `cwd`，继承了 cecelia 仓库目录，导致 LLM 调用产生的 session 污染用户的 `/resume` 列表
2. `sendFeishuMessage` 没有检查飞书 API 返回值，发送失败完全无日志
3. `FEISHU_OWNER_OPEN_IDS` 存储了旧 App 的 open_id，与当前 App 不匹配

## 需求
1. `/llm-call` spawn claude 时指定 `cwd` 到仓库外目录（如 `/tmp/cecelia-llm`），隔离 session
2. `sendFeishuMessage` 检查 resp.json() 的 code 字段，非 0 时打印警告日志
3. 更新 `.env` 中 `FEISHU_OWNER_OPEN_IDS` 为正确值 `ou_b472f4ce7a9bf227b9f6ad4e557fb687`

## 成功标准
- `/llm-call` 的 claude 进程不再在 cecelia 目录下创建 session
- 飞书发送失败时 Brain 日志有明确错误输出
- FEISHU_OWNER_OPEN_IDS 指向当前 App 的 open_id
