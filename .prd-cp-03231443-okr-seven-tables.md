## PRD（需求）

### 背景

当前数据库用 `goals` 表混存 vision（type='vision'）和 KR（type='area_okr'），用 `projects` 表混存 project/scope/initiative（type 字段区分），没有 objective 层。

需要拆分为7张独立表，每张表有标准基础字段 + custom_props JSONB（前端动态属性系统已就绪，用户可从 UI 自行添加属性）。

### 目标

创建6张全新表 + 复用现有 tasks 表 = 7层OKR层级体系：

```
Vision (visions)
  └─ Objective (objectives)
       └─ Key Result (key_results)
            └─ Project (okr_projects)
                 └─ Scope (okr_scopes)
                      └─ Initiative (okr_initiatives)
                           └─ Task (tasks, 现有表)
```

### 旧表保留

`goals`、`projects` 不删除，仅新增，避免数据丢失。
