# Task Card: Registry Auto-Sync Phase 0（只读检查 + CI block）

## 需求（What & Why）
**功能描述**: 升级现有 `scripts/registry-lint.mjs` 为完整的 registry 同步检查脚本，验证 features/*.yml 中声明的文件（entry_files、tests、skills）全部真实存在，maturity 等级与实际文档/测试覆盖一致，feature_count 和 schema_version 与 system-registry.yml 一致。
**背景**: 当前 registry-lint 只检查 features_file 引用和旧格式 code_path。v2.0.0 schema 引入了 entry_files、tests、skills 等新字段，需要对应的完整性检查，避免 registry 声明与实际代码漂移。
**不做什么**: 不自动修复漂移（Phase 1 才做），不修改 registry 内容本身。

## 成功标准
> [BEHAVIOR] CI 中 registry-lint job 运行升级后的脚本，发现不一致时 exit 1 阻断合并
> [ARTIFACT] 升级后的 scripts/registry-lint.mjs 包含 6 项检查

## 验收条件（DoD）

- [ ] [BEHAVIOR] entry_files 存在性：features/*.yml 里每个 feature 的 code.entry_files 列出的路径全部存在（目录以 / 结尾的检查目录存在）
  Test: manual:node -e "const {execSync}=require('child_process'); const r=execSync('node scripts/registry-lint.mjs',{cwd:process.cwd(),encoding:'utf-8'}); console.log(r.includes('entry_files') ? 'PASS' : 'FAIL')"

- [ ] [BEHAVIOR] 测试文件存在性：tests.unit/integration/e2e/regression 列出的文件全部存在
  Test: manual:node -e "const {execSync}=require('child_process'); const r=execSync('node scripts/registry-lint.mjs',{cwd:process.cwd(),encoding:'utf-8'}); console.log(r.includes('tests') ? 'PASS' : 'FAIL')"

- [ ] [BEHAVIOR] skill 文件存在性：skills[].entry_file 非 null 时必须存在
  Test: manual:node -e "const {execSync}=require('child_process'); const r=execSync('node scripts/registry-lint.mjs',{cwd:process.cwd(),encoding:'utf-8'}); console.log(r.includes('skills') ? 'PASS' : 'FAIL')"

- [ ] [BEHAVIOR] maturity 诚实性检查：maturity >= 2 但 tests.unit 为空时报错，maturity >= 3 但 tests.integration 为空时报错，maturity >= 1 但 docs 全 null 时降级警告
  Test: manual:node -e "const {execSync}=require('child_process'); const r=execSync('node scripts/registry-lint.mjs',{cwd:process.cwd(),encoding:'utf-8'}); console.log(r.includes('maturity') ? 'PASS' : 'FAIL')"

- [ ] [BEHAVIOR] feature_count 一致性：system-registry.yml 的 feature_count 和对应 features/*.yml 的实际 feature 数量一致
  Test: manual:node -e "const {execSync}=require('child_process'); const r=execSync('node scripts/registry-lint.mjs',{cwd:process.cwd(),encoding:'utf-8'}); console.log(r.includes('feature_count') ? 'PASS' : 'FAIL')"

- [ ] [BEHAVIOR] schema_version 一致性：所有 features/*.yml 的 schema_version 和 system-registry.yml 一致
  Test: manual:node -e "const {execSync}=require('child_process'); const r=execSync('node scripts/registry-lint.mjs',{cwd:process.cwd(),encoding:'utf-8'}); console.log(r.includes('schema_version') ? 'PASS' : 'FAIL')"

- [ ] [ARTIFACT] 脚本零外部依赖或只用 node_modules 中已有的 js-yaml
  Test: manual:node -e "console.log(require('js-yaml') ? 'PASS' : 'FAIL')"

## 实现方案（必填）
**要改的文件**: `scripts/registry-lint.mjs`
**受影响函数/API**: 整个脚本重写，使用 js-yaml 进行准确的 YAML 解析替代正则
**不改什么**: CI 配置（`.github/workflows/ci.yml`）不需要改，已有 `registry-lint` job 会自动运行升级后的脚本
