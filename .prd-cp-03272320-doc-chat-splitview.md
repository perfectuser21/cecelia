# PRD: feat(dashboard): 文档+聊天分栏界面（Notion AI 风格）

## 背景
用户需要一个类似 Notion AI 的界面：左边是文档内容，右边是 Claude 聊天窗口。
聊天时 Claude 能实时更新左边的文档内容。文档数据来自 Brain 的 `design_docs` 表，已有完整的 CRUD API。

## 目标
在 Cecelia Dashboard 的知识库模块下，新增一个文档+聊天分栏页面：
- 左边：当前文档内容（markdown 渲染）
- 右边：Claude 聊天窗口（含模型选择器）
- 聊天时 Claude 可更新左边文档内容
- 记忆模式：文档全文 + 最近 10 条对话历史

## 技术方案

1. **Backend**：在 `packages/brain/src/routes/design-docs.js` 增加 `POST /:id/chat` 端点
   - 接收 `{ message, history, model }`
   - 用 `callLLM` 调用 Claude（走订阅，模型可选）
   - Prompt 包含文档全文 + 对话历史 + 用户消息
   - 返回 `{ reply, updated_content }` — Claude 如果要改文档，通过 `<doc-update>` 标记输出新内容
   - 自动保存 updated_content 到 design_docs 表

2. **Frontend**：新增 `DocChatPage.tsx` 页面
   - 路由：`/knowledge/doc-chat/:id`
   - 分栏布局（左 50% / 右 50%）
   - 左：fetch design_doc，渲染 markdown，聊天后自动刷新
   - 右：消息列表 + 输入框 + 模型选择器（Haiku/Sonnet/Opus）
   - 发消息 → 调 Brain `/api/brain/design-docs/:id/chat` → 更新左边
