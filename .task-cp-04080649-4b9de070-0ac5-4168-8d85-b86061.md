# Task Card: 数据闭环 v1 — 周报集成到 Dashboard

## 需求（What & Why）
**功能描述**: 将每周自动生成的内容周报写入 system_reports 表，使其在 Dashboard /reports 页面可见；同时支持手动触发生成。
**背景**: weekly-report-generator.js 已实现完整的 8 板块周报生成逻辑，但只写 working_memory，Dashboard 读 system_reports，两者未打通。
**不做什么**: 不改周报内容逻辑、不新增采集器、不修改数据库 schema。

## 成功标准
> [BEHAVIOR] 周报数据在 Dashboard /reports 列表中可见（weekly_report 类型）
> [BEHAVIOR] POST /api/brain/reports/generate with type=weekly_report 能手动触发并写入 system_reports
> [ARTIFACT] ReportDetailPage 能正确渲染 weekly_report 的文本内容

## 验收条件（DoD）

- [x] [BEHAVIOR] weekly-report-generator.js 在生成周报后写入 system_reports 表
  Test: manual:node -e "const {buildWeeklyReportText}=require('./packages/brain/src/weekly-report-generator.js');const r=buildWeeklyReportText('2026-W15','2026-03-30','2026-04-05',{count:2,topics:['副业']},{},{},0,[],[]);if(!r.includes('ZenithJoy'))process.exit(1);console.log('ok')"

- [x] [BEHAVIOR] POST /api/brain/reports/generate with type=weekly_report 返回 200 + report.id
  Test: manual:curl -s -X POST localhost:5221/api/brain/reports/generate -H "Content-Type:application/json" -d '{"type":"weekly_report"}' | node -e "const d=JSON.parse(require('fs').readFileSync('/dev/stdin','utf8'));if(!d.success)process.exit(1)"

- [x] [ARTIFACT] ReportDetailPage.tsx 存在 weekly_report 渲染分支
  Test: manual:node -e "const c=require('fs').readFileSync('apps/dashboard/src/pages/reports/ReportDetailPage.tsx','utf8');if(!c.includes('weekly_report'))process.exit(1);console.log('ok')"

## 实现方案
**要改的文件**:
- `packages/brain/src/weekly-report-generator.js` — 新增 saveWeeklyReportToSystemReports()，在 generateWeeklyReport() 中调用
- `packages/brain/src/routes/system-reports.js` — POST /generate 支持 weekly_report type，调用 generateWeeklyReport(pool, new Date(), {force: true})
- `apps/dashboard/src/pages/reports/ReportDetailPage.tsx` — 新增 weekly_report 类型的纯文本渲染分支

**受影响函数**: generateWeeklyReport(), buildWeeklyReportText(), router.post('/generate')
**不改什么**: 周报内容结构、topic heat scorer、content analytics、触发时间窗口逻辑
