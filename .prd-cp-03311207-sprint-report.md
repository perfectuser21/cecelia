## 目标

在 /dev Stage 4 Ship 阶段自动生成 Sprint Report，提供任务执行全程可视化。

## 成功标准

- Stage 4 Ship 结束时自动生成 `docs/reports/{branch}.md`
- Report 包含：Planner 隔离验证、Sprint Contract 对抗轮次+提案对比表、CI 门禁通过情况（每个 gate 状态）、综合评分（4个维度）
- Report 文件随 PR commit 进 git（在 Learning 之后 push 前生成）
- 生成脚本：`packages/engine/scripts/devgate/generate-sprint-report.sh`
- Stage 4 步骤文件（04-ship.md）加入生成 Report 的指令
- 脚本从本地 seal 文件和状态文件中读取数据，不依赖外部服务（无外部 HTTP 调用）

## 实现方式

从以下 seal 文件汇总数据：
- `.dev-gate-planner.{branch}` — Planner 输出
- `.dev-gate-generator-sprint.{branch}` — Generator 提案
- `.dev-gate-spec.{branch}` — Evaluator 裁决（含 consistent/divergence）
- `.sprint-contract-state.{branch}` — 轮次和 blocker_count
- `.dev-mode` — 各 Stage 完成状态
- CI 结果从 gh CLI 读取

评分维度（自动计算）：
- Planner 隔离：所有 Test=TODO → 10/10
- 对抗深度：轮次 × 分歧数，1轮0分歧 → 4/10
- CI 健康度：一次通过 → 10/10，需修复 → 扣分
- 留痕完整度：seal 文件是否完整 → 评分
