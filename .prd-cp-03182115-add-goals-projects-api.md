# PRD: 新增 GET /goals/:id + GET /projects/:id API 端点

## 背景

intent-expand skill 需要沿 `task.project_id → projects.kr_id → goals(KR).parent_id → goals(OKR).parent_id → goals(Vision)` 链路查询上下文。Brain 需要提供专用的只读端点，返回指定字段。

## 目标

在 `packages/brain/src/routes/task-goals.js` 和 `packages/brain/src/routes/task-projects.js` 中更新 `GET /:id` 端点，使其返回 intent-expand 所需的精确字段，并保持标准错误响应格式。

## 成功标准

- GET /api/brain/goals/:id 返回 `{id, type, title, description, parent_id, project_id}`
- GET /api/brain/projects/:id 返回 `{id, title, description, kr_id, goal_id}`
- 两个端点均有 404 处理（记录不存在时返回 `{"error": "goal not found"}` 或 `{"error": "project not found"}`）
- 所有现有测试通过
