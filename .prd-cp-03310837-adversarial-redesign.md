## 目标

重设计 /dev 4-Stage Pipeline 的对抗审查机制，对标 Anthropic 官方 Planner/Generator/Evaluator 三角对抗。

## 当前问题
- Planner 写了含 Test 字段的 DoD，Generator 照抄，没有独立思考
- spec_review（Evaluator）收到的 Task Card 含主 agent 已写好的 Test 字段，独立性被污染
- Sprint Contract 是主 agent 自己填 Test 字段后让 Evaluator 比对，不是两个独立 subagent 从零提案

## 修改范围
- packages/engine/skills/dev/lib/planner-prompt.md
- packages/engine/skills/dev/steps/01-spec.md（Sprint Contract Gate）
- packages/workflows/skills/spec-review/SKILL.md
- Engine 版本 bump

## 成功标准

- Planner 输出的 Task Card 不含任何非 TODO 的 Test 字段
- Sprint Contract 由独立的 Generator subagent + Evaluator subagent 各自从剥离版 Task Card 独立提案
- Orchestrator 比对两份提案，有分歧则展示给双方各自修正，N 轮直到收敛
