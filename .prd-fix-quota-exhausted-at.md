# PRD：修复 quota_exhausted_at 字段缺失

## 背景

Migration 145 的版本号与 `brain_config quota_reset_at` migration 冲突，导致 `tasks.quota_exhausted_at` 字段从未被创建。

`routes.js:2863` 的 `execution-callback` 端点在 UPDATE tasks 时引用了该不存在的字段，导致所有 Codex/Agent 任务完成后无法回写结果到 Brain，Brain 显示 degraded。

## 问题

```
column "quota_exhausted_at" does not exist
→ execution-callback 全部失败
→ 任务停在 queued / Codex 结果全丢
→ Brain degraded
```

## 目标

新建 migration 149，补建 `tasks.quota_exhausted_at` 字段，修复 Brain execution-callback 端点，恢复 Codex 任务结果回写能力。

## 成功标准

- `tasks.quota_exhausted_at` 字段存在于 DB
- `POST /api/brain/execution-callback` 返回成功（非 column 错误）
- Brain health 恢复 healthy
- selfcheck EXPECTED_SCHEMA_VERSION = 149
