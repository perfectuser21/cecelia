# Task Card: 动态产能模型

## 需求（What & Why）
**功能描述**: Brain 新增 capacity-budget API，基于历史实际完成数据动态计算产能（pr_per_slot_per_day），供 /decomp 和 /decomp-check 使用。冷启动用理论值，随数据积累自校准。

**背景**: 当前 /decomp SKILL.md 中的产能模型是写死的（12 slots、287 PR/day），但实际有 3 台 Mac Mini（26 slots）。校准表的 PR 数量也是固定值，导致拆解粒度不准。需要改为动态查询，让拆解精度随运行数据自动提升。

**不做什么**:
- 不改 slot-allocator.js 的调度逻辑
- 不改 account-usage.js 的 5h/7d 限制
- 不做产能仪表盘（那是另一个任务）

## 成功标准

1. [ARTIFACT] Brain 新增 `GET /api/brain/capacity-budget` API 端点
2. [BEHAVIOR] API 返回 `pr_per_slot_per_day`，冷启动返回理论值 25，有历史数据后返回实测值
3. [BEHAVIOR] API 返回 `confidence` 字段（theoretical/low/medium/high），基于 sample_size 动态变化
4. [ARTIFACT] /decomp SKILL.md 产能模型章节改为动态查询说明，删除写死数字
5. [ARTIFACT] /decomp-check SKILL.md 新增 PR 数量校验检查项
6. [GATE] 所有现有测试通过
