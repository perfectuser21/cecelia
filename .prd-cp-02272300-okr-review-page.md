# PRD: OKR 讨论专属页面 + 秋米重拆执行

## 目标

1. 提升 UX：将秋米对话从 InboxPage 卡片内嵌移到独立页面 `/okr/review/:id`
2. 增加执行能力：用户在对话中说「重新拆」时，后端自动重置 KR 状态触发重拆

## 用户故事

- 收件箱 OKR 拆解确认卡片 → 点「与秋米对话」→ 跳转到 `/okr/review/:id` 专属页面
- 页面展示：KR 信息、拆解结果、版本历史（同一 KR 的历次拆解）、秋米对话
- 对话中识别重拆意图 → 后端重置 KR → 显示通知 → 轮询新版本
- 页面底部「确认放行」→ 放行后返回收件箱

## 成功标准

- GET /api/brain/pending-actions/:id 端点返回单条数据
- GET /api/brain/pending-actions/:id/versions 端点返回同 KR 历史版本
- POST /autumnrice/chat 返回 `redecomp_triggered` 字段
- 点「与秋米对话」跳转到 `/okr/review/:id`（非内嵌展开）
- OkrReviewPage 正常渲染 KR 信息 + 对话区
- Brain v1.121.3，Workspace v1.6.0
