# Task Card: 激活多平台内容发布链路（≥3平台自动发布）

## 需求（What & Why）

**功能描述**：
现有发布基础设施（8个 publisher skills、content-pipeline-orchestrator、publish-results API）已全部就位，
但 `content_publish` 任务类型在 task-router.js 的 `VALID_TASK_TYPES` 和 `LOCATION_MAP` 中**均未注册**。

导致的核心问题：
- `getTaskLocation('content_publish')` 返回默认值 `'us'`（而非 `'xian'`）
- 所有 content_publish 任务被路由到 US 本机执行，无法触达西安 Codex Bridge 上的浏览器自动化环境
- `detectRoutingFailure` 检测到 `invalid_task_type:content_publish`，路由状态标为 `failed`

**背景**：
content-export 完成后，content-pipeline-orchestrator 会自动为 8 个平台创建 content_publish 任务（PR #1307 已实现），
但这些任务因路由错误无法正常派发到西安 Codex 执行发布脚本。

另外，发布成功率缺乏统计 API，无法验证「≥95% 成功率」目标达成。

**不做什么**：
- 不修改任何 publisher skill 脚本
- 不添加失败重试机制（scope 太大，单独任务处理）
- 不修改 content-pipeline-orchestrator
- 不修改 DB schema

## 成功标准

1. [ARTIFACT] `task-router.js` 中 `VALID_TASK_TYPES` 包含 `content_publish`
2. [ARTIFACT] `task-router.js` 中 `LOCATION_MAP` 包含 `content_publish: 'xian'`
3. [ARTIFACT] `task-router.js` 中 `SKILL_WHITELIST` 包含 `content_publish` 条目
4. [BEHAVIOR] `getTaskLocation('content_publish')` 返回 `'xian'`（通过单元测试验证）
5. [ARTIFACT] `GET /api/brain/publish-stats` API 返回各平台发布成功率统计
6. [GATE] CI 全部通过
