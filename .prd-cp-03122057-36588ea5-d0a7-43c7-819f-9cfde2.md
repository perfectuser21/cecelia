# PRD - feat(learning): OAuth 401 失败 Learning 聚合去重

## 背景

OAuth token 过期时，每次失败任务都写入 1 条 failure learning，导致 241 条积压中大量是重复的 OAuth 401 记录。这些噪音占据 Cortex context 的 top-N 位置，掩盖真正有价值的 insight。

## 目标

在写入 learning 时，检测是否存在同类 failure（相同 error_type + 相同 platform/category），若存在则合并（increment count）而非新增记录。

## 成功标准

- [ ] 新 learning 写入时检查：同 error_type + 同 source_category 的 learning 若 created_at > now()-24h 则合并
- [ ] 合并时 learning.occurrence_count +1，updated_at = now()
- [ ] learnings 表新增 occurrence_count INTEGER DEFAULT 1 字段（migration）
- [ ] 针对现有 241 条积压执行一次历史聚合（按 error_type 分组，合并为代表性记录）
- [ ] 聚合后 unprocessed learning 数量预计从 241 降至 < 30
- [ ] 单元测试覆盖聚合逻辑

## 技术要点

- 修改 packages/brain/src/actions.js recordLearning() 添加 upsert 逻辑
- 或在 learnings-received 路由中添加聚合检查
- 新增 migration 添加 occurrence_count 字段
- 提供一次性历史聚合脚本或 migration
