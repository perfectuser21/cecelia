# PRD - fix(executor): syncOrphanTasksOnStartup 孤儿改 requeue — 区分可重试 vs 真实失败

## 背景
Brain 重启时存在两个竞争实现：
- startup-recovery.js L228-237：已有正确逻辑，无条件 requeue in_progress 任务
- executor.js syncOrphanTasksOnStartup L2308-2384：错误逻辑，对所有孤儿无条件 fail

两者都在 Brain 启动时运行，executor.js 的 fail 会覆盖 startup-recovery.js 的 requeue，导致静默数据丢失。

## 目标
修复 syncOrphanTasksOnStartup，按 reason 分流：
- reason=process_disappeared → 改为 queued（Brain 重启中断，可重试）
- reason=oom_killed / oom_likely / killed_signal → 保持 failed（系统级真实失败）
- reason=timeout → 保持 failed（任务超时不应重试）

## 成功标准
- [ ] Brain 重启后，in_progress 任务（reason=process_disappeared）出现在 queued 状态
- [ ] oom_killed / killed_signal 任务仍保持 failed
- [ ] 单元测试：按 reason 分流行为有断言
