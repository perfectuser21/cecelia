# PRD: 修复 migration 103 desires_status_check 约束

## 背景

PR #334（Brain v1.163.0）新增了 migration 103，为 desires 表添加 completed_at/failed_at/effectiveness_score 字段，同时重建 desires_status_check 约束。

但约束只包含 `('pending', 'acted', 'completed', 'failed')`，遗漏了数据库中已存在的 `'expressed'` 和 `'acknowledged'` 两个状态值。

结果：Brain 启动时 migration 103 失败，服务无法正常运行。

## 问题描述

```
[APPLY] 103_desires_feedback.sql ...
[FAIL] 103_desires_feedback.sql: check constraint "desires_status_check" of relation "desires" is violated by some row
[FATAL] Migration failed
```

desires 表现有数据：
- expressed: 941 行
- acted: 66 行
- acknowledged: 1 行
- pending: 13 行

## 修复方案

修改 `packages/brain/migrations/103_desires_feedback.sql`，在 CHECK 约束中补充 `'expressed'` 和 `'acknowledged'`：

```sql
CHECK (status IN ('pending', 'acted', 'completed', 'failed', 'expressed', 'acknowledged'))
```

## 成功标准

- migration 103 能够成功执行
- Brain 可以正常启动（不再出现 FATAL migration 错误）
- Brain 版本 bump（patch）
