# PRD: 机器注册表 + Capability Tags 动态路由

## 背景

当前 task-router.js 中的 `LOCATION_MAP` 硬编码了 task_type → location（us/xian/hk），导致：
1. 新增机器需改代码
2. 路由绑定地点而非能力，无法负载均衡
3. HK MiniMax（不是自有机器）被错误路由

## 目标

用机器注册表（MACHINE_REGISTRY）+ 能力标签（capability tags）替换静态 LOCATION_MAP，实现基于机器能力的动态路由。

## 需求

### 1. TASK_REQUIREMENTS（task-router.js）
- 将 LOCATION_MAP 替换为 TASK_REQUIREMENTS：task_type → required tags[]
- dev/codex_dev/codex_qa 等需要 `has_git` 的任务 → 路由到 US M4
- research/talk/explore 等通用任务 → 路由到任意 `general` 机器（负载均衡）
- codex_playwright → `has_browser`
- heartbeat/daily_report 等 → `general`
- 导出 `getTaskRequirements(taskType)` 函数

### 2. MACHINE_REGISTRY（executor.js）
- 定义 3 台机器：US M4、西安 M4、西安 M1
- 每台机器有：id、url、type（claude_code/codex）、tags
- 移除 HK MiniMax 路由（不是自有机器）
- 新增 `selectBestMachine(requiredTags)` 函数：按 tags 过滤 → 负载均衡选最闲
- 更新 `triggerCeceliaRun()` 使用 tags 路由

### 3. 机器清单
- `us-m4`: claude_code, tags: ['has_git', 'general']
- `xian-m4`: codex, tags: ['general']
- `xian-m1`: codex, tags: ['general']

## 约束
- 向后兼容：不破坏现有调度流程
- 保留 selectBestBridge() 作为 Codex 机器负载均衡的底层实现
- 不改变 triggerCodexBridge() 和 triggerClaudeCodeBridge() 的内部实现

## 成功标准

1. `getTaskRequirements('dev')` 返回 `['has_git']`，`getTaskRequirements('research')` 返回 `['general']`
2. `triggerCeceliaRun()` 对 `dev` 任务路由到 US M4，对 `research` 任务路由到 xian-m4 或 xian-m1（负载均衡）
3. 不再有任何代码路径路由到 HK MiniMax（100.86.118.99）
