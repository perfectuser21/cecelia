## PRD - 微博新接口图文发布

**需求来源**: Brain 自动调度 — KR1 紧急自救任务
**功能描述**: 基于微博新内部 API 接口实现图文发布，替代现有 CDP UI 自动化方案
**涉及文件**:
  - 新建: `packages/workflows/skills/weibo-publisher/scripts/publish-weibo-api.cjs`
  - 新建: `packages/workflows/skills/weibo-publisher/scripts/__tests__/publish-weibo-api.test.cjs`
  - 修改: `packages/workflows/skills/weibo-publisher/SKILL.md`（新增 API 方案说明）

## 成功标准

- `publish-weibo-api.cjs` 能通过 CDP 提取 Cookie 并调用微博内部 HTTP API 完成图文发布
- 单元测试覆盖 parseCookieHeader、isRateLimit、isLoginError、buildPicUploadForm 等纯函数
- 异常处理：限频返回可重试错误，未登录返回不可重试错误

**非目标**:
  - 不需要 OAuth2 App 注册
  - 不修改现有 publish-weibo-image.cjs（两套方案并存）
  - 不修改 batch-publish-weibo.sh（后续可选接入）

### 技术方案（新 API vs 旧 CDP）

| 方案 | 原理 | 优势 | 劣势 |
|------|------|------|------|
| 旧 CDP 方案 | 控制浏览器 UI 操作 | 不需了解 API | 触发验证码、受页面改版影响 |
| 新 API 方案 | CDP 提取 Cookie → HTTP API 调用 | 稳定、快速、不触发验证码 | 依赖内部 API 格式不变 |

**流程**:
1. 连接 CDP → 提取微博会话 Cookie（SUB、SUBP、XSRF-TOKEN 等）
2. 逐张上传图片 → `picupload.weibo.com/interface/pic_upload.php` → 收集 pic_id
3. 发布内容 → `www.weibo.com/ajax/statuses/update` → 带 pic_ids 和文案

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| CDP 连接失败 | 重试 3 次（2s 指数退避）后报错 | P1 | `weibo_cdp_connect_failed` |
| Cookie 缺失/登录失效 | 不可重试，立即抛出 | P0 | `weibo_session_expired` |
| 图片上传失败 | 重试 3 次后报错 | P1 | `weibo_image_upload_failed` |
| 限频（发帖太频繁） | 可重试错误（等 60s 后重试） | P1 | `weibo_rate_limited` |
| 发布接口 500/502 | 重试 3 次后报错 | P1 | `weibo_post_failed` |
