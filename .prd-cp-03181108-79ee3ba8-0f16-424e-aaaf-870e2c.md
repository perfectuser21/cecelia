# PRD: feat(brain): slot-allocator 积压感知降速

## 背景

tick.js 已有 MAX_NEW_DISPATCHES_PER_TICK = 2 的 burst limiter（7a/7b 双路均检查）。但当前 slot-allocator.js 无队列深度感知：队列积压 9-10 个任务时，每 tick 仍尝试派发 2 个，连续多 tick 未消化积压会导致 N 个 agent 同时启动引发资源雪崩。

## 成功标准

- slot-allocator.js 新增 getQueueDepth() async 函数（查 tasks 表 status=queued count）
- calculateSlotBudget() 返回值新增 backpressure: { queue_depth, threshold, active, override_burst_limit } 字段
- active=true 当且仅当 queue_depth > BACKPRESSURE_THRESHOLD(5)
- tick.js Step 7 前读取 slotBudget.backpressure.override_burst_limit，若 active 则用 override 值替代 MAX_NEW_DISPATCHES_PER_TICK（effectiveBurstLimit）
- 单元测试：queue_depth=9 时每 tick 最多派发 1 个
- 单元测试：queue_depth=3 时每 tick 最多派发 2 个（正常）
- 现有测试不回归
