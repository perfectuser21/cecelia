# Phase 7.1：统一 claude 启动器 — Stop Hook session_id 全模式工作

## 背景

Phase 7（v17.0.0，PR #2443）修了 Stop Hook 多 worktree 路由机制——stop.sh 读 stdin JSON 拿 session_id、按 owner_session 精确匹配 .dev-lock。但只对 **headless 模式**有效，因为：

- **Headless (Brain 派)**：`cecelia-run.sh` 用 `claude -p --session-id <uuid>` → cmdline 有 `--session-id` → worktree-manage.sh 的 `_resolve_claude_session_id` 沿 PPID 链能解析 → owner_session 写正确
- **Interactive (用户手敲 `claude`)**：无 `--session-id` flag → cmdline 找不到 → owner_session 写 "unknown" → Stop Hook 拿到 Claude Code 自动生成的真 session_id 对比 "unknown" → **永远不匹配** → exit 0 放行

2026-04-20 实测：手动 /dev 场景下 Stop Hook 完全失效，导致 assistant "宣布完成"就真结束，没有阻止器循环。

## 核心洞察

**不是代码有两套模式——是"源头没写进去"的单点漏洞**：写 .dev-lock 时没拿到 session_id，匹配时拿到了也匹配不上。

用户 2026-04-20 明确主张：**一套规则、支持并行、头/无头统一**。

## 目标

让**所有** claude 实例（headless + interactive + 并行）都走同一个启动器，保证每个实例都有 session_id 且能被 worktree-manage.sh 和 Stop Hook 一致识别。

## 成功标准

1. `scripts/claude-launch.sh` 新建：
   - 若无 `$CLAUDE_SESSION_ID` → `uuidgen` 生成小写 UUID
   - `export CLAUDE_SESSION_ID=$SID`（子进程继承）
   - `exec claude --session-id "$SID" "$@"`
   - chmod +x
2. `packages/brain/scripts/cecelia-run.sh` 改成**调用 `claude-launch.sh`**（而不是直接 `claude --session-id $UUID`），`SESSION_UUID` 传过去或由 launcher 生成
3. `packages/engine/skills/dev/scripts/worktree-manage.sh::_resolve_claude_session_id` 改：
   - **优先读 `$CLAUDE_SESSION_ID` env var**
   - 读不到再 fallback 沿 PPID 链找 cmdline `--session-id`
   - 都读不到才写 "unknown"（保留现有行为作最后兜底）
4. `.claude/CLAUDE.md` 加 shell 配置说明：用户 `~/.zshrc` 或 `~/.bashrc` 加 `alias claude='bash /absolute/path/to/claude-launch.sh'`
5. 新测试 `packages/engine/tests/launcher/claude-launch.test.ts`：
   - 断言 launcher 无 env 时生成 session_id
   - 断言 launcher 有 env 时继承
   - 断言 launcher export 正确传给子进程
6. 新测试 `packages/engine/tests/launcher/session-resolve.test.ts`：
   - mock env var 路径：worktree-manage 能拿到
   - mock PPID fallback 路径：worktree-manage 也能拿到
   - 都没有时写 "unknown"
7. Engine 版本 18.0.0 → 18.1.0（minor，新增统一启动器）
8. 版本 6 处同步 + SKILL.md frontmatter version（7 处总）
9. feature-registry.yml 新 phase-7.1 条目
10. PR title 含 `[CONFIG]`

## 不做

- 不改 `hooks/stop.sh` 或 `hooks/stop-dev.sh` 的 session 匹配逻辑（Phase 7 已对）
- 不改 `devloop_check` 的完成条件判断（现有 SSOT 保留）
- 不改用户 shell config 本身（只在 CLAUDE.md 说明，用户自行 source）
- 不做 claude launcher 的 sub-command（launcher 只是 exec 包装）
- 不做 Phase 8.2/8.3

## 假设

- `uuidgen` 在 macOS/Linux 都可用
- claude CLI 支持 `--session-id <uuid>` flag（Phase 7 已验证）
- 用户在 shell 里给 claude 加 alias 是一次性操作，不破坏其他 tool 使用 claude CLI
