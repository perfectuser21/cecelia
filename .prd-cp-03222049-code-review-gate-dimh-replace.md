# PRD: code-review-gate 维度H升级 — 替代性内容未删旧内容=blocker

## 背景

当前 code-review-gate 维度H（信息卫生）只检测"diff 中新增行引用了已删除功能/路径"（warning 级别）。

但更根本的问题是：AI 改动内容时默认行为是**追加**而不是**替换**。改 A→B 时只加了 B，旧的 A 描述还留着。多轮迭代后同一文件里存在 A、B、C 互相矛盾的说法，AI 每次读都不知道哪个是真的。

## 成功标准

- code-review-gate 维度H 新增 blocker 级检查：diff 中任何文件（代码、.md、SKILL.md、yaml）新增了替代性描述，但旧的对应描述未在同一 PR 中被删除
- 02-code.md Stage 2 新增明确编辑规则：改动描述性内容时先删旧再写新，禁止堆叠
- 原有 warning 检查（引用已删除功能/路径、矛盾描述）保留不变

## 修复清单

1. **code-review-gate SKILL.md**：维度H 新增 blocker 项"替代性内容未删旧描述"，并说明检测方法（新增行描述新行为 → 检查旧描述是否有对应 `-` 删除行）
2. **02-code.md**：Stage 2 写代码前加明确编辑规则："修改任何描述性内容时，先删旧再写新，禁止在保留旧说法的同时加新说法"
