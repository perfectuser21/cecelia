## PRD - Cortex insight → action 自动闭合机制

**需求来源**: Brain 自动调度任务（cortex insight 积累但零行动）
**功能描述**: cortex.js 记录 insight 后，自动分析是否涉及代码级修复，若是则自动生成 dev task（写入 tasks 表），并标记 applied=true，防止重复触发
**涉及文件**:
- `packages/brain/src/cortex.js` — 在 record_learning 后追加 insight 分析逻辑
- `packages/brain/src/insight-to-task.js` — 新建：insight 分析 + task 创建核心逻辑
- `packages/brain/src/__tests__/insight-to-task.test.js` — 新建：单元测试
- `packages/brain/migrations/` — 可能需要确认 learnings.applied 字段已存在

**成功标准**:
- cortex_insight 类型的 learning 创建后，若内容含代码修复信号，自动生成 dev task
- 已有 task 的 insight 不重复触发（去重）
- applied=true 标记正确更新
- integration test 验证 insight→task 链路

**非目标**:
- 不修改已有的 thalamus.js / planner.js 逻辑
- 不改变 cortex 决策策略本身
- 不做 UI 展示

### 异常处理 & 报警规范

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| insight 分析失败（LLM 超时）| 降级：跳过 task 创建，记录日志 | P3 | insight_task_create_skipped |
| tasks 表写入失败 | 重试 1 次，失败记录日志 | P2 | insight_task_write_failed |
| applied 更新失败 | 记录日志，不阻断主流程 | P3 | insight_applied_update_failed |
