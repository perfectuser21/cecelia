# Task Card: QuickCheck 并发互斥锁

## 需求（What & Why）
**功能描述**: scripts/quickcheck.sh 加 flock/mkdir 互斥锁，防止并发 vitest 进程抢资源
**背景**: PR #2332 推送过程中，3 个并发 quickcheck 各自启动 vitest，互相拖慢到 25 分钟，push 卡死
**不做什么**: 不改 vitest 本身并发配置，不改 CI workflow

## 成功标准
- scripts/quickcheck.sh 在 git push 并发触发时，同一 repo 只跑一个 vitest
- 第 2 个实例 2 秒内拿不到锁 → 输出提示并 exit 0（不 block push）
- 锁在进程结束时自动释放（flock 的 fd / mkdir 的 trap）

## 验收条件（DoD）

- [x] [BEHAVIOR] quickcheck.sh 加了互斥锁（flock + mkdir fallback）
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/quickcheck.sh','utf8');if(!c.includes('flock -w 2'))process.exit(1);if(!c.includes('mkdir \"${_LOCK_DIR}\"'))process.exit(1)"

- [x] [BEHAVIOR] 锁文件路径为 .git/quickcheck.lock（不污染 /tmp）
  Test: manual:node -e "const c=require('fs').readFileSync('scripts/quickcheck.sh','utf8');if(!c.includes('quickcheck.lock'))process.exit(1);if(c.includes('/tmp/quickcheck'))process.exit(1)"

- [x] [BEHAVIOR] 新增测试文件 quickcheck-mutex.test.ts
  Test: manual:node -e "require('fs').accessSync('packages/engine/tests/scripts/quickcheck-mutex.test.ts')"

- [x] [BEHAVIOR] 测试包含并发互斥验证和自动释放验证
  Test: manual:node -e "const c=require('fs').readFileSync('packages/engine/tests/scripts/quickcheck-mutex.test.ts','utf8');if(!c.includes('两次并发调用'))process.exit(1);if(!c.includes('自动释放'))process.exit(1)"

- [x] [ARTIFACT] 所有改动文件无中文全角标点（避免 bash unbound variable 陷阱）
  Test: manual:node -e "const fs=require('fs');const files=['scripts/quickcheck.sh'];files.forEach(f=>{const c=fs.readFileSync(f,'utf8');const lines=c.split('\n');lines.forEach((l,i)=>{if(l.match(/\\\$[a-zA-Z_][a-zA-Z0-9_]*[，。（）]/))throw new Error(f+':'+(i+1)+' 变量后紧跟中文标点')})})"

## 实现方案
**要改的文件**:
- `scripts/quickcheck.sh` — 前 30 行加锁逻辑
- `packages/engine/tests/scripts/quickcheck-mutex.test.ts` — 新建

**不改什么**: vitest 并发配置、CI workflow、pre-push hook 其他逻辑
