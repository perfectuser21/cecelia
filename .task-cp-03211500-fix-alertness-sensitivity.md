# Task Card: Alertness 过敏修复（PANIC 抖动稳定期 + COMA drain 解耦 + 429 过滤）

## 需求（What & Why）
**功能描述**: 修复 Alertness 系统三个过敏问题：(1) PANIC 需要连续3次 critical 才升级，单次异常只升到 ALERT；(2) COMA 不再自动触发 drain_mode_requested=true；(3) API 429 错误不计入 error_rate 指标。
**背景**: 当前单次异常就能触发 PANIC/COMA，叠加 drain 自杀导致系统停摆。429 是正常限流不应算严重异常。
**不做什么**: 不改变 Alertness 等级定义，不改变 escalation 响应级别映射。

## 成功标准
> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [BEHAVIOR] PANIC 升级需要连续 3 次 critical 评估，单次 critical 最多升到 ALERT
2. [BEHAVIOR] COMA 状态不再设置 drain_mode_requested=true
3. [BEHAVIOR] API 429 错误不计入 operationHistory 错误率
4. [GATE] CI 全部通过
