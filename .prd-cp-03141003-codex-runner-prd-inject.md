# PRD: Codex runner.sh PRD 预注入

## 背景

Brain 只在 US VPS 运行，Codex 在西安 M4 运行。runner.sh 发送 `/dev --task-id XXX` 给 Codex 后，Codex 在 M4 侧尝试调用 `localhost:5221` 获取 PRD，但 M4 没有 Brain，导致每次任务失败。

## 问题

- runner.sh 发送 `--task-id` 给 Codex，Codex 在执行时需要访问 Brain
- M4 没有 Brain（`localhost:5221` 不通）
- Tailscale 到 US VPS 连接超时

## 解决方案

runner.sh 在 US 侧（Brain 可达）预拉 PRD 内容，嵌入 prompt 中，Codex 收到时无需再访问 Brain。

## 成功标准

- runner.sh v2.2.0 新增 `BRAIN_API_URL` 环境变量（默认 `http://localhost:5221`）
- runner.sh 在 US 侧预拉 PRD（`/api/brain/tasks/:id`），提取 title + description
- PRD 内容直接嵌入 `build_comprehensive_prompt()` 和 `build_resume_prompt()` 的 prompt
- 失败回调改用 `$BRAIN_API_URL`（不再 hardcoded localhost:5221）
- PRD 预拉失败时优雅降级（日志警告 + 降级到 --task-id 模式）
