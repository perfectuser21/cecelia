# Task Card: DevLog 数据接通 — dev_records 历史回填 + PR merge 自动写入

## 需求（What & Why）

**功能描述**：`dev_records` 表为空，DevLog 页面空白。需要两步修复：
1. 历史回填：从 GitHub 获取近 200 个合并 PR，批量写入 dev_records
2. pr-callback fallback：即使 task 匹配失败，也写一条 dev_records（task_id=null）

**背景**：
- Brain DB 只有 16 条测试任务，无 pr_url 数据
- pr-callback-handler.js 已有 dev_records 写入逻辑，但依赖 task 匹配
- task 匹配失败 → dev_records 永远空 → DevLog 页面空白
- GitHub 有 1528 个合并 PR，是唯一可用的历史数据源

**不做什么**：
- 不改 dev_records 表结构
- 不回填 prd_content / dod_items
- 不改前端

## 实现方案

**新建** `packages/brain/src/scripts/backfill-dev-records.js`
- `gh pr list --state merged --limit 200 --json number,title,headRefName,mergedAt,url`
- 解析 learning_summary：从 `docs/learnings/<branch>.md` 提取 `### 根本原因` 章节
- 批量 INSERT ON CONFLICT (pr_url) DO NOTHING
- 输出：写入行数统计

**修改** `packages/brain/src/pr-callback-handler.js`
- 在 matchTaskByBranchOrUrl 返回 null 时（无任务匹配），也写一条 dev_records
- 位置：现有 "4a. 写入 dev_records" 代码块上方添加 no-task fallback
