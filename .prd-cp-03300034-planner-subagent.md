# PRD: Planner 独立 subagent — Stage 1 Task Card 生成拆分

## 背景

当前 /dev Stage 1 中，Task Card + DoD 的生成由主 agent 直接完成。主 agent 同时承担多个职责：
读取 PRD、理解 Brain 上下文、生成 Task Card、填写 DoD、调用 spec_review。

Generator subagent（Stage 2）已成功从主 agent 拆出，证明 subagent 模式可行。
现在把 Stage 1 的 Task Card + DoD 生成也用同样的模式拆出。

## 功能描述

将 Stage 1 的 Task Card + DoD 生成拆为独立 Planner subagent：
- Planner subagent 只接收：用户任务描述 + docs/current/SYSTEM_MAP.md
- Planner subagent 不接收：CLAUDE.md coding 规范、brain context、代码库细节
- Planner 输出：Task Card + DoD（只说 WHAT，不说 HOW）
- 主 agent 变成纯编排者：读 PRD → spawn Planner → 拿到 Task Card → 继续 Sprint Contract Gate

## 成功标准

- Stage 1 由主 agent spawn Planner subagent 完成 Task Card + DoD 生成
- Planner subagent prompt 只含任务描述 + SYSTEM_MAP.md，无 coding 规范
- Planner 输出的 DoD 聚焦行为描述（WHAT），不含实现路径（HOW）
- Sprint Contract Gate（spec_review）仍然正常工作
- 主 agent 的 01-spec.md 中有明确的 Planner subagent 调用流程和隔离规则
