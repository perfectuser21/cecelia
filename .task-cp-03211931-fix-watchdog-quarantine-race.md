# Task Card: 修复 watchdog kill 死循环 — requeueTask 竞态导致 quarantine 永不触发

## 需求（What & Why）
**功能描述**: 修复 watchdog kill 后因竞态条件导致 quarantine 永不触发的 bug。

**背景**:
生产环境观察到任务 `f25828bb` 被 watchdog 连续 kill 4 次（06:16~06:25），但每次 `requeued: false, quarantined: false`。
根因：`requeueTask()` 要求 `WHERE status = 'in_progress'`，但在 watchdog 调用前，liveness probe 或 execution-callback 已将状态改为 `failed`。
结果：`watchdog_retry_count` 永远不递增，`QUARANTINE_AFTER_KILLS=2` 的阈值永远不触发，任务可通过恢复机制回到 queued 再次 dispatch，形成死循环。
同时 alertness 升级到 ALERT(3)，stop_dispatch 触发，系统停摆 13+ 小时。

**不做什么**:
- 不改 watchdog 的 kill 逻辑（检测和 kill 本身正常）
- 不改 liveness probe 的执行顺序
- 不改 alertness 恢复逻辑

## 成功标准

1. [BEHAVIOR] requeueTask 在任务非 in_progress 时仍能递增 watchdog_retry_count 并在超限时 quarantine
2. [BEHAVIOR] tick.js watchdog 处理中，requeueTask 返回 not_in_progress 且未 quarantine 时，做 fallback quarantine
3. [ARTIFACT] 相关单元测试覆盖竞态场景
4. [GATE] CI 全部通过
