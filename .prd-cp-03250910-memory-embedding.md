# PRD: 记忆系统 PR10 — conversation_turn embedding fire-and-forget

## 背景

`orchestrator-chat.js` 在保存用户消息和 Cecelia 回复到 `memory_stream` 表时
（source_type='conversation_turn'），调用了 `generateMemoryStreamL1Async`（生成 L1 摘要），
但没有调用 `generateMemoryStreamEmbeddingAsync`（生成向量 embedding）。

结果：这些 memory_stream 行的 embedding 字段为 NULL，无法参与向量检索（相似记忆召回）。

## 目标

在两处 memory_stream INSERT 后，fire-and-forget 调用
`generateMemoryStreamEmbeddingAsync(recordId, content, pool)`，
使对话消息也能被向量召回。

## 成功标准

1. orchestrator-chat.js 导入 `generateMemoryStreamEmbeddingAsync`
2. 用户消息插入后调用 embedding（fire-and-forget）
3. Cecelia 回复插入后调用 embedding（fire-and-forget）
4. 无 OPENAI_API_KEY 时静默 no-op（函数本身保证）
5. 新增测试验证 wiring 正确
