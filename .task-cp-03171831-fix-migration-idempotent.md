# Task Card: 修复 023 migration CREATE INDEX 不幂等

## 需求（What & Why）

CI L3 `coverage-baseline` job 中 `smoke.test.js` 启动 Brain server 失败：

```
[FAIL] 023_add_run_events_observability_v1.1.sql: relation "idx_run_events_task_id" already exists
[smoke] Server exited with code 1
```

**根本原因**：`023_add_run_events_observability_v1.1.sql` 中所有 `CREATE INDEX` 和 `CREATE UNIQUE INDEX` 语句都没有 `IF NOT EXISTS` 保护。CI PostgreSQL service 在跑多次测试后已存在这些索引，重复执行 migration 时报错，server 启动失败。

**修复**：为所有 `CREATE INDEX` / `CREATE UNIQUE INDEX` 语句加上 `IF NOT EXISTS`，使 migration 幂等。

**不做什么**：不改变表结构、视图、函数；不改 migration 编号；不改其他 migration 文件。

## 成功标准

1. [BEHAVIOR] 023 migration 可重复执行，不因 "already exists" 报错
2. [ARTIFACT] 023 migration 所有 CREATE INDEX / CREATE UNIQUE INDEX 均含 IF NOT EXISTS
3. [GATE] CI L3 smoke.test.js 通过

- [x] [GATE] Brain migration 文件语法正确（不含语法错误）
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/migrations/023_add_run_events_observability_v1.1.sql','utf8');if(!c.includes('CREATE INDEX IF NOT EXISTS'))throw new Error('no IF NOT EXISTS found');console.log('ok')"

## 实现方案

**修改文件**：`packages/brain/migrations/023_add_run_events_observability_v1.1.sql`

**改动**：将所有 `CREATE INDEX idx_*` / `CREATE UNIQUE INDEX idx_*` 改为 `CREATE INDEX IF NOT EXISTS idx_*` / `CREATE UNIQUE INDEX IF NOT EXISTS idx_*`，共 16 处。
