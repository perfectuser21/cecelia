# PRD: 基于依赖图的选择性 CI

## 背景

每个 PR 全量跑 L1-L4 所有 CI，不管改了哪个包，导致 CI 时间偏长、资源浪费。

## 目标

创建 `packages/engine/scripts/affected-packages.js`，基于 git diff 和路径映射计算受影响的包集合，统一选择性执行逻辑。

## 功能需求

1. 脚本接受改动文件列表参数（命令行参数或 stdin）
2. 输出 JSON 数组，如 `["brain","engine","dashboard"]`
3. 路径映射：`packages/brain/` → brain，`packages/engine/` → engine，等
4. 兜底规则：无法确定时（根目录文件、.github/workflows/）→ 输出所有包

## 成功标准

- `packages/engine/scripts/affected-packages.js` 存在且可执行
- 改动 brain 文件 → 输出包含 brain
- 改动 engine 文件 → 输出包含 engine
- 改动根目录文件 → 输出所有包（兜底）
- CI 全部通过
