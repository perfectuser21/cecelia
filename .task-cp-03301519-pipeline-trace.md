# Task Card: pipeline-trace.sh — 任意 branch 的执行全景视图

## 需求（What & Why）

**功能描述**：构建 `packages/engine/scripts/pipeline-trace.sh <branch>`，读取某个 branch 的所有 pipeline 证据文件，输出结构化执行摘要，让开发者能立即看清该 branch 的每个 Stage 状态、输入输出、关键 seal 字段、PR / CI 结果。

**背景**：/dev pipeline 执行中产生多个证据文件（.dev-mode、seal files、Learning），但没有统一的聚合视图。用户需要能一眼看清"这次执行有没有按照设计走"。

**不做什么**：不修改任何现有 pipeline 逻辑；不写入任何证据文件；只读取、聚合、展示。

## 成功标准

1. 传入 branch 名称，输出每个 Stage（0-4）的状态和时间戳
2. 显示 spec seal 的 verdict + divergence_count，crg seal 的 verdict
3. 显示 PR URL + CI 最近一次运行结论（pass/fail/pending）
4. 显示 Learning 文件存在性（含 ### 根本原因 判断）
5. 无该 branch 数据时输出清晰提示（不崩溃）
6. 可对正在进行中和已完成的 branch 使用
