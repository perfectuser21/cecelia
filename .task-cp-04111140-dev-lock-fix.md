# Task Card: dev-lock 丢失导致 stop hook fail-open — fail-closed + 重建逻辑修复

## 需求（What & Why）

**问题 1**：`stop-dev.sh` L76 `[[ -z "$DEV_LOCK_FILE" ]] && exit 0`  
dev-lock 找不到 → 直接放行。但 dev-lock 可能是被意外删除或从未正确创建，  
此时 dev-mode 还有未完成的步骤，Claude 却被允许退出（PR 未创建/CI 未完成）。

**问题 2**：`00-worktree-auto.md` L28-30 `cp "$DEV_MODE_FILE" "$DEV_LOCK_FILE"`  
dev-mode 只含 branch/step 字段，复制出的 dev-lock 没有 tty/session_id，  
`_session_matches()` 三路匹配全部失败 → 等同于 dev-lock 不存在 → exit 0。

**不做什么**：不改 `_session_matches` 逻辑，不改 harness 通道，不改 devloop-check.sh。

## 成功标准

- [x] stop-dev.sh：无 dev-lock 时扫描所有 worktree，发现未完成 dev-mode 则 block（exit 2）
- [x] 00-worktree-auto.md：重建 dev-lock 时生成含 tty/session_id 的完整格式，不再 cp dev-mode
- [x] stop-hook-exit-codes.test.ts：新增"dev-lock 丢失但 dev-mode 未完成 → exit 2"测试
