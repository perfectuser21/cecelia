# Task Card: Brain server.js 启动韧性修复

## 需求（What & Why）
**功能描述**: 修复 Brain server.js 启动时的三个致命问题：migration 失败直接退出、selfcheck 失败直接退出、uncaughtException 不退出导致僵尸进程。
**背景**: PG 临时断连时 migration 失败会导致 Brain 直接 exit(1) 而非重试；selfcheck 失败也直接退出而非降级运行；uncaughtException 只打日志不退出，进程变僵尸无法被 supervisor 重启。
**不做什么**: 不改 migration 逻辑本身，不改 selfcheck 检查项，不改业务路由。

## 成功标准

1. [BEHAVIOR] migration 失败时重试 3 次（间隔 5 秒），3 次都失败才 exit(1)
2. [BEHAVIOR] selfcheck 失败时降级启动（打 warn 日志），不再 exit(1)
3. [BEHAVIOR] uncaughtException handler 中加 process.exit(1)，让 supervisor 重启
4. [GATE] CI 全部通过

## 验收条件（DoD）

- [x] [BEHAVIOR] server.js migration 块包含 3 次重试循环
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/server.js','utf8');if(!c.includes('attempt <= 3'))process.exit(1)"

- [x] [BEHAVIOR] server.js selfcheck 失败时不 exit，打 warn 并继续
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/server.js','utf8');if(c.includes('Self-check failed. Brain will NOT start'))process.exit(1)"

- [x] [BEHAVIOR] uncaughtException handler 包含 process.exit(1)
  Test: manual:node -e "const c=require('fs').readFileSync('packages/brain/server.js','utf8');const m=c.match(/uncaughtException[\\s\\S]{0,300}/);if(!m||!m[0].includes('process.exit'))process.exit(1)"

- [x] [GATE] 所有现有测试通过
  Test: manual:bash -c "cd packages/brain && npm test 2>&1 | tail -5"

## 实现方案
**要改的文件**: packages/brain/server.js（1个文件）
**Scope 锚定**: 只改 3 处：第 61-66 行 uncaughtException handler、第 196-201 行 migration 块、第 203-207 行 selfcheck 块
