# Task Card: 动态 CI 守护规则 — RCI 清理 + 自动覆盖检测

## 需求（What & Why）

**背景**：
- `packages/engine/regression-contract.yaml` 中约 35 条契约已过时（引用不存在的文件、旧路径、重复ID）
- RCI 和 DevGate 是静态的：新功能加进来没有任何东西强制要求它必须有测试，时间一长必然腐烂
- `rci-execution-gate.sh` 只读 quality 包的 RCI，engine 的 RCI 从未被执行
- 删除功能时没有反向检查，RCI 条目会静默留存成垃圾

**功能描述**：
1. 清理 engine RCI 中过时/空头/重复的条目
2. 新增 `check-coverage-completeness.mjs`：源文件存在必须有对应测试
3. 新增 `check-rci-health.mjs`：RCI P0 条目必须有真实 test_command 且文件存在（含反向孤儿检测）
4. `rci-execution-gate.sh` 扩展支持 engine RCI
5. `known-failures` 条目加 expires 字段强制机制

**不做什么**：不重写现有测试文件；不改 stop hook 逻辑；不处理 quality/workflows 包的 RCI

## 成功标准

1. [ARTIFACT] engine RCI 中所有 `evidence.file` 指向不存在文件的条目已修复或删除
2. [ARTIFACT] `scripts/devgate/check-coverage-completeness.mjs` 存在并可运行
3. [ARTIFACT] `scripts/devgate/check-rci-health.mjs` 存在并可运行
4. [BEHAVIOR] 新增 hook 文件但无测试时，`check-coverage-completeness` 返回非零
5. [BEHAVIOR] RCI 有 P0 条目但 test_command 为空时，`check-rci-health` 返回非零
6. [BEHAVIOR] RCI 条目的 evidence.file 不存在时，`check-rci-health` 报警（孤儿检测）
7. [GATE] CI 全部通过
