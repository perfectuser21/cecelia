# Task Card: Dev Pipeline Codex 独立审查加固

## 需求（What & Why）

**功能描述**：修复 Dev Pipeline 中 spec_review / code_review_gate 两个 Codex Gate 的 5 个问题，使审查任务真正通过本机 Codex CLI 执行、携带完整上下文，并与开发任务的锁池隔离。

**背景**：
- spec_review / code_review_gate 当前走 cecelia-bridge（claude CLI），而非 Codex CLI，与注释"US 本机 Codex"不符
- preparePrompt() 给这两个类型只传 title/description，没有 Task Card 原文和 git diff，审查质量低
- 所有任务共用 /tmp/cecelia-locks/（12 slot），review 任务可能占满 dev 的 slot
- cecelia-run.sh 默认 MAX_CONCURRENT=12，但 Brain 物理上限是 10，数字不对齐
- code-review-gate SKILL.md 用 `gh pr diff` 获取 diff，本机 worktree 场景下应改用 `git diff`

**不做什么**：
- 不改 Codex CLI 的账号/模型配置
- 不改 spec-review SKILL.md 的审查维度和裁决逻辑
- 不改 xian Codex Bridge 路由（codex_dev/codex_qa 等保持不变）

## 成功标准

1. [ARTIFACT] `triggerLocalCodexReview()` 函数已在 executor.js 中存在，spec_review / code_review_gate 通过该函数执行
2. [BEHAVIOR] spec_review prompt 包含完整 Task Card 内容；code_review_gate prompt 包含 Task Card + git diff HEAD~1
3. [ARTIFACT] `/tmp/codex-review-locks/` 独立锁池，最多 2 个 review slot 并发
4. [ARTIFACT] cecelia-run.sh 默认 MAX_CONCURRENT 从 12 改为 10
5. [ARTIFACT] code-review-gate SKILL.md 输入章节改用 `git diff` 替代 `gh pr diff`
