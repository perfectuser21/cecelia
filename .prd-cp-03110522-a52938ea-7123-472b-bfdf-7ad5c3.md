# PRD - 修复 cecelia-bridge Math.min 截断：per-request timeout 不再被 120s 上限截断

## 需求来源

Brain 自动调度任务。

## 功能描述

cecelia-bridge.js line 74:
  const timeoutMs = Math.min(timeout || BRIDGE_TIMEOUT_MS, BRIDGE_TIMEOUT_MS);

llm-caller.js 传入 timeout=300000（5分钟，Cortex Opus 所需），bridge 用 Math.min 截断到 BRIDGE_TIMEOUT_MS=120000ms，导致 Opus 在 121s 时被 SIGTERM kill → exit 143 → Cortex 离线。

## 涉及文件

- `packages/brain/src/cecelia-bridge.js`（主要修改）

## 成功标准

- timeout=300000 传入时，bridge 实际等待 300s，不再截断到 120s
- 未传 timeout 时，行为不变（使用 BRIDGE_TIMEOUT_MS 默认 120s）
- CECELIA_BRIDGE_MAX_TIMEOUT_MS 环境变量可配置（默认 600s）
- Brain CI 通过
- Cortex LLM 调用不再 exit 143

## 非目标

- 不修改 llm-caller.js（已正确传入 timeout）
- 不修改 cortex.js（已正确设置 300000ms）

### 异常处理 & 报警规范

N/A - 纯逻辑修改，无新 IO 操作
