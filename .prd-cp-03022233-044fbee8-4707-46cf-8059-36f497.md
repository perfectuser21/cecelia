## PRD - Task Router 诊断 API + 日志增强 + 派发逻辑修复

**需求来源**: Brain 自动调度 - KR4 下 12 个 Initiative，7 天内未派发任何 Task
**功能描述**:
1. 新增 GET /api/brain/task-router/diagnose/:kr_id 端点，诊断指定 KR 下的任务派发状态
2. 增强 task-router.js 日志，记录路由决策过程
3. 修复派发逻辑问题，确保任务能够正常派发

**涉及文件**:
- packages/brain/src/task-router.js（日志增强 + 逻辑修复）
- packages/brain/src/server.js（新增诊断 API 端点）
- packages/brain/test/（相关测试）

**成功标准**:
- GET /api/brain/task-router/diagnose/:kr_id 返回完整诊断信息
- 诊断信息包含：KR 下的所有 Initiative、每个 Initiative 的 Task 状态、派发阻塞原因
- task-router.js 日志能够追踪路由决策过程
- CI 通过

**非目标**:
- 不修改前端 UI
- 不改变现有 API 契约
