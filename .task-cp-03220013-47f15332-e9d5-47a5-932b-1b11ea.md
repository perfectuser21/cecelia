# Task Card: branch-protect 新增 .dod.md 归属校验

## 需求（What & Why）
**功能描述**: 在 `packages/engine/hooks/branch-protect.sh` 中新增检查：若当前 worktree 的项目根目录存在 `.dod.md` 文件，且其 frontmatter 中有 `branch:` 字段，则校验该字段值是否与当前 git 分支一致。不一致时输出 WARNING 提示（非阻断）。

**背景**: 来自知识反刍条目"进入 worktree 后首先检查 .dod.md 是否属于本任务"。旧任务遗留的 `.dod.md` 可能残留在 worktree 中，若 branch 字段与当前分支不符，说明该文件来自其他任务，应提醒开发者核查。

**不做什么**: 不阻断写代码流程（不 exit 2），不修改 `.dod.md` 文件，不处理 `.dod-{branch}.md` 格式（有分支名的格式本身已安全）。

## 成功标准

1. [ARTIFACT] `branch-protect.sh` 新增 `.dod.md` branch 字段校验逻辑
2. [BEHAVIOR] `.dod.md` 的 `branch:` 字段与当前分支不匹配时，stderr 输出 `[WARN]` 警告信息
3. [BEHAVIOR] `.dod.md` 不存在、或无 `branch:` 字段时，不输出任何警告（静默放行）
4. [GATE] CI 全部通过

## 验收条件（DoD）
