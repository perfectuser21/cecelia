# Task Card: crystallize 执行闭环 — 补全3个断线

## 需求（What & Why）
**功能描述**: 补全 crystallize 任务类型的执行回传链路，使流水线能端到端运转。
**背景**: crystallize-orchestrator.js 实现了流水线状态机，但3个执行端连接缺失：playwright-runner.sh 不回传 Brain，execution.js 不调用 advanceCrystallizeStage，executor.js 不为 crystallize 使用 playwright-runner.sh。
**不做什么**: kr-progress.js bug 修复、Dashboard Roadmap UI 迁移

## 成功标准

1. [ARTIFACT] playwright-runner.sh 在成功/失败时 POST /api/brain/execution-callback
2. [ARTIFACT] execution.js 收到 crystallize_* 子任务完成时调用 advanceCrystallizeStage
3. [ARTIFACT] executor.js 对 crystallize_forge/crystallize_verify 使用 playwright-runner.sh runner 模式
4. [GATE] CI 全部通过

## 实现方案

### 缺口1：playwright-runner.sh 回传
- 添加 `send_callback` 函数：curl POST 到 BRAIN_API_URL/api/brain/execution-callback
- 成功时：发送 status="AI Done"，findings 含 script_path
- 失败时：发送 status="AI Failed"
- curl 调用加 `|| true` 防止 Brain 不可达时 runner 崩溃

### 缺口2：execution.js 新增 5c13 块
- 在 content-pipeline 块（5c11）后插入新块
- 检查 CRYSTALLIZE_STAGES 和 parent_crystallize_id
- 调用 advanceCrystallizeStage(task_id, newStatus, findingsObj)

### 缺口3：executor.js runner 模式
- 添加 isCrystallize 检查（crystallize_forge, crystallize_verify）
- runner: 'packages/engine/runners/codex/playwright-runner.sh'
- runner_args: ['--task-id', task.id]

- packages/engine/runners/codex/playwright-runner.sh
- packages/brain/src/routes/execution.js
- packages/brain/src/executor.js
