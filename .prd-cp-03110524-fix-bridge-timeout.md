## 功能描述

## 背景

cecelia-bridge.js line 74:
  const timeoutMs = Math.min(timeout || BRIDGE_TIMEOUT_MS, BRIDGE_TIMEOUT_MS);

llm-caller.js 传入 timeout=300000（5分钟，Cortex Opus 所需），bridge 用 Math.min 截断到 BRIDGE_TIMEOUT_MS=120000ms，导致 Opus 在 121s 时被 SIGTERM kill → exit 143 → Cortex 离线。

## 目标
修复 Math.min 截断，使 per-request timeout 可以超过全局默认值，同时引入 CECELIA_BRIDGE_MAX_TIMEOUT_MS 作为安全上限。

## 修改内容

### cecelia-bridge.js line 74
**改前：**
  const timeoutMs = Math.min(timeout || BRIDGE_TIMEOUT_MS, BRIDGE_TIMEOUT_MS);

**改后：**
  const MAX_LLM_TIMEOUT_MS = parseInt(process.env.CECELIA_BRIDGE_MAX_TIMEOUT_MS || '600000', 10);
  const timeoutMs = Math.min(timeout || BRIDGE_TIMEOUT_MS, MAX_LLM_TIMEOUT_MS);

### 无需改动
- llm-caller.js：已正确传入 timeout 字段
- cortex.js：已正确设置 300000ms 默认值
