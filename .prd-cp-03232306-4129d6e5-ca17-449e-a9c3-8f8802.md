# PRD - execution.js KR 断链修复 — strategy_session INSERT key_results 补 objective_id

## 背景
strategy_session 回调在 INSERT INTO key_results 时没有传 objective_id，导致创建的 KR 成为孤岛（没有挂到任何 Objective 下）。

## 功能描述
修复 `packages/brain/src/routes/execution.js` 的 strategy_session INSERT key_results 逻辑，在 INSERT 前查找或创建关联的 objective，确保每条 KR 都有 objective_id。

## 成功标准

- strategy_session 创建的 key_results 记录必须有 objective_id（不能为 null）
- 需要在 INSERT 前先查找或创建关联的 objective
- 若输出 JSON 携带 objective_id 则优先用；否则按优先级 fallback

## 约束

- 只改 execution.js，不改其他文件（除非必要）
- 如果找不到 objective，自动创建一个合理默认值
- 不影响现有的 execution 功能
