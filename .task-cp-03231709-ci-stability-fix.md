# Task Card: CI 稳定性加固 — 修复预存测试失败 + L4 Smoke Check

## PRD

### 背景
系统诊断发现两个直接导致 CI 不稳定的问题：
1. Brain 单元测试有 2 个预存失败（`actions-initiative-orchestration.test.js`），基线容忍值为 2，导致 CI 允许新 PR 引入额外失败而不报警
2. L4 没有任何 API 级别的 smoke check，Brain API 的破坏性变更无法被 CI 检测

### 目标
- 修复 2 个预存测试失败，将 `brain-test-baseline.txt` 降为 `0`
- 在 L4 `brain-integration` job 中添加 API smoke check，每次 Brain 有改动时验证基础 API 健康

### 成功标准
## 成功标准
- Brain 单元测试全部通过（0 failures）
- `brain-test-baseline.txt` = `0`
- L4 新增 API smoke check step，在 Brain 服务启动后验证 3 个端点
- CI L3 和 L4 全绿

### Non-Goals
- 不修复 `learning-effectiveness.test.js` 的本地 FK 问题（纯本地脏数据，CI fresh DB 无此问题）
- 不重构 createInitiative 的其他行为
- 不修改其他 CI 层
