# Task Card: OKR PR11 — 清理旧表残留收尾

## 需求（What & Why）

**功能描述**: 删除 Brain 源码中最后 4 处旧表（goals/projects）实际读写残留，确保旧表可以安全删除。

**背景**: PR10 已完成大部分迁移，但仍有以下 4 处残留导致旧表无法删除：
1. `actions.js` updateGoal 函数 — 三新表都找不到时 fallback `UPDATE goals SET`（第 509-522 行）
2. `planner.js` 任务创建校验 — 先查旧 `projects` 表再回退到新表（第 1192-1202 行）
3. `services/memory-service.js` getDetail — UNION 查询依次查 projects 和 goals（第 73-106 行）
4. `project-compare.js` — getCompareMetrics 和 generateCompareReport 查询 `projects`（第 111、256 行）

**不做什么**:
- 不删除旧表本身（仅清理业务代码中的查询）
- 不修改 notion-full-sync.js（Notion 同步模块有独立迁移计划）
- 不改变函数签名和对外 API

## 实现方案

**要改的文件**:
- `packages/brain/src/actions.js` — 删除第 509-522 行 fallback，加 return { success: false }
- `packages/brain/src/planner.js` — 第 1191-1202 行，删除旧 projects 查询，直接查新表 UNION ALL
- `packages/brain/src/services/memory-service.js` — getDetail 中第 73-106 行迁移到新 OKR 表
- `packages/brain/src/project-compare.js` — 第 111、256 行迁移，查 okr_projects/okr_scopes/okr_initiatives UNION ALL

**Scope 锚定**: 仅修改上述 4 个文件的旧表查询部分，不改函数签名

**memory-service.js 新表映射**:
- `projects` → 查 `okr_projects UNION ALL okr_scopes UNION ALL okr_initiatives`（level 字段需动态映射：okr_projects→project, okr_scopes→scope, okr_initiatives→initiative）
- `goals` → 查 `objectives UNION ALL key_results UNION ALL visions`（level 字段动态映射）

**project-compare.js 新表映射**:
- `getCompareMetrics` 第 111 行：`FROM projects p LEFT JOIN key_results` → 需查 okr_projects/okr_scopes/okr_initiatives UNION ALL 作为 p，再 LEFT JOIN key_results
- `generateCompareReport` 第 256 行：`FROM projects WHERE id = ANY($1::uuid[])` → 查 UNION ALL
