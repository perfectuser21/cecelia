# Task Card: 批量修复所有不幂等 migration

## 需求（What & Why）
**功能描述**: 修复 17 个 migration 中仍存在的不幂等操作，使它们可以在 CI DB 积累状态时安全重跑。

**背景**: CI 使用 self-hosted xian-mac-m1 runner，pgdata 目录在同一 run 的重试时被复用。若某 migration 中途失败导致 PostgreSQL 重启，transaction 会 ROLLBACK，但 schema_version 可能未记录。重跑时若 DDL 无 IF NOT EXISTS 保护则报 "already exists"。

**实际需要修复的内容**（经代码审查，17 个中只有以下需要改）:
- **013**: `CREATE TABLE` 无 `IF NOT EXISTS` + `INSERT INTO schema_version` 无 ON CONFLICT
- **052**: `INSERT INTO schema_version` 无 ON CONFLICT
- **078**: 缺少 `INSERT INTO schema_version`
- **112**: 缺少 `INSERT INTO schema_version`
- **114**: `INSERT INTO schema_version` 中 version 错误写为 '113'（应为 '114'）
- **119, 126, 127, 140**: 缺少 `INSERT INTO schema_version`

**不做什么**: 不改业务逻辑，不改约束内容，只加幂等保护。

## 成功标准
1. [ARTIFACT] 013 的 CREATE TABLE 改为 IF NOT EXISTS
2. [ARTIFACT] 013/052 的 schema_version INSERT 加 ON CONFLICT DO NOTHING
3. [ARTIFACT] 078/112/119/126/127/140 加 INSERT INTO schema_version ON CONFLICT DO NOTHING
4. [ARTIFACT] 114 的 version '113' 改为 '114'
5. [GATE] CI 全部通过
