# Task Card: 修复 runner.sh $CODEX_HOME） 变量边界 bug

## 需求（What & Why）
**功能描述**: 修复 `packages/engine/runners/codex/runner.sh` 中三处 `$CODEX_HOME）` 的 bash 变量边界问题。
**背景**: runner.sh 在 bash 3.2 + `LANG=en_US.UTF-8` + `set -u` 环境（西安 Mac mini M4）下运行时，全角括号 `）`（UTF-8: `\xef\xbc\x89`）的字节被 bash 3.2 当作变量名的一部分，导致展开 `CODEX_HOME）`（不存在的变量），触发 `unbound variable` 报错。team1 quota 超额后，account rotation 逻辑因第 432 行崩溃而无法切换到 team2-5，Codex 任务全部失败。
**不做什么**: 不修改 runner.sh 的业务逻辑，不改变账号切换行为。

## 成功标准

1. [ARTIFACT] `runner.sh` 第 332、432、445 行的 `$CODEX_HOME）` 均改为 `${CODEX_HOME}）`
2. [BEHAVIOR] bash 3.2 UTF-8 locale 下不再因全角括号触发 unbound variable
3. [GATE] CI 全部通过
