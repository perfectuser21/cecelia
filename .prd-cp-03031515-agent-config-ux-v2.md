# PRD：Brain Agent 配置 UI 重设计 v2

## 背景

PR #377 实现了动态扫描 + Toast，但 UI 交互模型仍然不理想：
- 当前：每行有一个小的 `API|无头` 切换 + 模型下拉，两个控件割裂
- 当前：MiniMax 只能选 API，无法走无头模式
- 问题：无法直观看到"当前用的是什么组合"，也无法一眼看到有哪些选项

## 核心概念澄清

调用方式有**两个独立维度**：

| | Anthropic 模型 (claude-*) | MiniMax 模型 |
|---|---|---|
| **直连 API** | Anthropic API | MiniMax API |
| **无头 (claude -p)** | Anthropic 无头 | MiniMax 无头 |

"MiniMax 无头"= 走 `claude -p` 无头调用，但 Claude Code 账号配置的 LLM provider 是 MiniMax。
需要用 Skill 时必须走无头模式（Skill 在 Claude Code 上下文内运行），但底层可以用 MiniMax 省钱。

## 目标

### 功能：折叠展开式选项卡

**折叠态（默认）**：
```
┌─ L2 皮层 ─────────────────── Anthropic · Sonnet 4.6 · 无头 ─ ▼ ┐
└────────────────────────────────────────────────────────────────┘
```
显示：agent 名称 + 当前选中的组合（provider · model · 调用方式）+ 展开箭头

**展开态（点击后）**：
```
┌─ L2 皮层 ──────────────────────────────────────────────────── ▲ ┐
│                                                                  │
│  ┌─ Anthropic ─────────────────────────────────────────────┐    │
│  │  [ API · Sonnet ] [ API · Opus ]                        │    │
│  │  [无头 · Sonnet ] [无头 · Opus ] ← 当前选中             │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  ┌─ MiniMax ───────────────────────────────────────────────┐    │
│  │  [ API · M2.5 Fast ] [ API · M2.5 ]                     │    │
│  │  [无头 · M2.5 Fast ] [无头 · M2.5 ]                     │    │
│  └─────────────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────────┘
```

选项按 **provider 分组**，每组内按 **模型** 排列，每个选项包含 API/无头两行。
只显示该 agent 的 `allowed_models` 里有的模型。

### 后端 provider 值映射

| UI 显示 | provider 字段值 | model 字段值 |
|---------|----------------|-------------|
| Anthropic API | `anthropic-api` | claude-* |
| Anthropic 无头 | `anthropic` | claude-* |
| MiniMax API | `minimax` | MiniMax-* |
| MiniMax 无头 | `minimax-headless` | MiniMax-* |

**注意**：`minimax-headless` 是新增的 provider 值，需要前端识别并在 UI 里正确显示。
后端是否已支持 minimax-headless 暂不确认，前端先实现 UI，后端逻辑独立。

## 范围

- 仅改 `apps/api/features/system/components/BrainLayerConfig.tsx`
- 不改后端、不改 model-registry.js、不改 API

## 成功标准

1. 每个 agent 默认折叠，显示当前选中的 provider+model+调用方式
2. 点击展开，显示所有可选组合（按 provider 分组）
3. 只显示该 agent allowed_models 里有的模型
4. 嘴巴（mouth）保持现有 2×2 网格不变
5. TypeScript 编译 0 错误
