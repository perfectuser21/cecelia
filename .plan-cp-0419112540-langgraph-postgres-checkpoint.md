# Implementation Plan: PostgresSaver 注入

## Task 1 — 修改 executor.js LangGraph 分支

**文件**: `packages/brain/src/executor.js`

**改动位置**: 行 2810 附近（`const { runHarnessPipeline } = await import(...)` 之后，`runHarnessPipeline(task, { ... })` 之前）。

**改动内容**: 在 `runHarnessPipeline` 调用前插入：

```js
// PostgresSaver: LangGraph 持久化 checkpointer（Brain 重启后从断点续跑）
const { PostgresSaver } = await import('@langchain/langgraph-checkpoint-postgres');
const checkpointer = PostgresSaver.fromConnString(
  process.env.DATABASE_URL || 'postgresql://cecelia@localhost:5432/cecelia'
);
await checkpointer.setup();  // 幂等建 checkpoints / checkpoint_blobs / checkpoint_writes
```

然后在 `runHarnessPipeline(task, { ... })` 的 opts 对象里加 `checkpointer,`。

**预期输出**: 文件包含 `PostgresSaver.fromConnString` 与 `checkpointer` 字段。

**测试命令**:
```bash

```
预期：`OK`

## Task 2 — 新增单元测试

**文件**: `packages/brain/src/__tests__/executor-langgraph-checkpointer.test.js`

**目标**: 用 vi.mock 拦截 `./harness-graph-runner.js` 的 runHarnessPipeline，断言 executor 在 LangGraph 分支路由时，调用 runHarnessPipeline 的 opts.checkpointer 不为空且是 PostgresSaver 实例。

**测试骨架**:

```js
import { describe, it, expect, vi, beforeEach } from 'vitest';

vi.mock('@langchain/langgraph-checkpoint-postgres', () => {
  const setup = vi.fn().mockResolvedValue(undefined);
  return {
    PostgresSaver: {
      fromConnString: vi.fn((url) => ({ __kind: 'PostgresSaver', url, setup })),
    },
  };
});

const runHarnessPipelineMock = vi.fn().mockResolvedValue({
  skipped: false, steps: 0, finalState: {},
});

vi.mock('../harness-graph-runner.js', () => ({
  runHarnessPipeline: runHarnessPipelineMock,
}));

vi.mock('../account-usage.js', () => ({
  isSpendingCapped: () => false,
  isAuthFailed: () => false,
}));

vi.mock('../docker-executor.js', () => ({
  executeInDocker: vi.fn(),
}));

describe('executor LangGraph branch - PostgresSaver checkpointer', () => {
  beforeEach(() => {
    runHarnessPipelineMock.mockClear();
    process.env.HARNESS_LANGGRAPH_ENABLED = 'true';
  });

  it('calls runHarnessPipeline with PostgresSaver checkpointer', async () => {
    const { dispatchTask } = await import('../executor.js');
    const task = { id: 'test-task-id', task_type: 'harness_planner', title: 'test' };
    // 模拟 executor 路由到 LangGraph 分支的最小环境
    // 这里只需确认 checkpointer 被传入 runHarnessPipeline
    // ...
  });
});
```

**现实考量**：executor.js 的 dispatchTask 有大量前置路由判断（Codex/西安/location），直接 import 跑会拖到依赖 DB 和无关模块。改成**更轻量的集成策略**：

**最终做法**：写一个纯字符串/AST 级的**单元测试**，不 spawn executor，改为用 `fs.readFileSync` + 正则检查 executor.js 结构：确认 `PostgresSaver.fromConnString` 出现在 `runHarnessPipeline` 调用之前、且 `checkpointer` 作为 runHarnessPipeline 的参数出现。

```js
import { describe, it, expect } from 'vitest';
import { readFileSync } from 'node:fs';
import path from 'node:path';

describe('executor.js — LangGraph PostgresSaver checkpointer 注入', () => {
  const src = readFileSync(
    path.resolve(__dirname, '../executor.js'),
    'utf8'
  );

  it('imports PostgresSaver from @langchain/langgraph-checkpoint-postgres', () => {
    expect(src).toMatch(/@langchain\/langgraph-checkpoint-postgres/);
    expect(src).toMatch(/PostgresSaver\.fromConnString/);
  });

  it('calls checkpointer.setup() before runHarnessPipeline', () => {
    const setupIdx = src.indexOf('checkpointer.setup()');
    const runIdx = src.indexOf('runHarnessPipeline(task,');
    expect(setupIdx).toBeGreaterThan(-1);
    expect(runIdx).toBeGreaterThan(-1);
    expect(setupIdx).toBeLessThan(runIdx);
  });

  it('passes checkpointer to runHarnessPipeline opts', () => {
    // 匹配 runHarnessPipeline(task, { ... checkpointer ... })
    const callBlock = src.match(/runHarnessPipeline\(task,\s*\{[\s\S]{0,500}?\}\)/);
    expect(callBlock).toBeTruthy();
    expect(callBlock[0]).toMatch(/checkpointer/);
  });

  it('uses DATABASE_URL fallback to localhost cecelia', () => {
    expect(src).toMatch(/postgresql:\/\/cecelia@localhost:5432\/cecelia/);
  });
});
```

**测试命令**:
```bash
cd packages/brain && npx vitest run src/__tests__/executor-langgraph-checkpointer.test.js
```
预期：4 tests passed。
