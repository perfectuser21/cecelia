# PRD：Brain Agent 配置中心体验改善

## 背景

PR #367 实现了 Brain agent 独立 API/无头调用方式配置，但存在两个体验痛点：

1. **切换后没有明确成功反馈**：用户不确定切换是否真的生效
2. **新增 agent 需要手改前端**：`BrainLayerConfig.tsx` 的 `layers` 数组是硬编码的，
   在 `model-registry.js` 添加新 agent 后，前端不会自动显示

## 目标

### 功能一：动态扫描（核心）
- `BrainLayerConfig.tsx` 的 `layers` 数组改为从 API 返回的 `agents` 数组动态生成
- 筛选 `layer === 'brain'` 的 agents
- 使用 `agent.name`、`agent.description`、`agent.allowed_models`、`agent.recommended_model`
- 以后在 `model-registry.js` 加新 brain agent → 前端自动出现，不需要改前端

### 功能二：切换成功反馈（体验）
- 切换模式或模型后，显示 toast 通知（"已切换到 API 模式 ✓"）
- 失败时显示红色 toast（"切换失败: xxx"）
- Toast 3 秒后自动消失

## 成功标准

1. 在 `model-registry.js` 添加新 brain agent 后，刷新 Dashboard 自动显示，无需改前端
2. 切换 API/无头后，页面显示绿色 toast 确认
3. 现有 5 个 brain agent（thalamus/cortex/reflection/memory/rumination）正常显示
4. 嘴巴（mouth）的 MouthModeSelector 保持不变

## 范围

- 仅改 `apps/api/features/system/components/BrainLayerConfig.tsx`
- 不涉及 Brain 后端、数据库、API
- 不改 `model-registry.js`（只是让前端能动态读取它）
