# PRD: 修复 initiative_plan 任务 skill 字段缺失

## 问题

`createInitiativePlanTask` 函数创建的任务缺少 `skill` 字段，导致 pre-flight check 失败（因为 dispatcher 不知道调用哪个 skill）。

## 根本原因

1. **decomposition-checker.js:96-100** - `createInitiativePlanTask` 创建任务时没有设置 `skill` 字段
2. **executor.js:886** - `TASK_TYPE_TO_SKILL_MAP` 定义了 `initiative_plan` → `/decomp` 的映射
3. **Pre-flight check 逻辑** - 检查 `skill` 字段是否存在，`null` 时失败

## 解决方案

### 方案 1：在创建任务时直接设置 `skill` 字段（推荐）

修改 `createInitiativePlanTask` 函数，在 INSERT 语句中添加 `skill='/decomp'`：

```sql
INSERT INTO tasks (title, description, status, priority, goal_id, project_id, task_type, skill, trigger_source)
VALUES ($1, $2, 'queued', 'P1', $3, $4, 'initiative_plan', '/decomp', 'brain_auto')
```

**优点**：
- 数据库中直接存储 skill，一致性更好
- Pre-flight check 无需修改

### 方案 2：Pre-flight check 支持通过 `task_type` 推断 `skill`

修改 pre-flight check 逻辑，当 `skill=null` 时，查询 `TASK_TYPE_TO_SKILL_MAP` 推断 skill。

**缺点**：
- Pre-flight check 需要知道 executor 的映射关系（耦合）
- 数据库中仍然 `skill=null`，不如方案 1 清晰

## 推荐方案

**方案 1**：在 `createInitiativePlanTask` 中直接设置 `skill='/decomp'`。

## 涉及文件

- `packages/brain/src/decomposition-checker.js` - 主要修改
- `packages/brain/src/routes.js:3418-3420` - 同样的 INSERT 语句需要修改（execution-callback 中创建 initiative_plan 任务）

## 成功标准

1. ✅ `createInitiativePlanTask` 创建的任务 `skill='/decomp'`
2. ✅ `routes.js` 中 execution-callback 创建的 initiative_plan 任务 `skill='/decomp'`
3. ✅ Pre-flight check 通过
4. ✅ 测试验证通过
5. ✅ CI 通过
