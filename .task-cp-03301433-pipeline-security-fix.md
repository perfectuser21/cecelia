# Task Card: Pipeline 安全漏洞修复（Seal 校验 + Fallback 关闭 + Flock 硬检查 + PR 二次确认）

## 需求（What & Why）
**功能描述**: 修复 4 个 pipeline 安全漏洞，防止 AI 通过伪造 seal 文件、降级执行、flock 静默跳过、合并前状态窗口期等手段绕过安全门禁。
**背景**: 当前 devloop-check.sh 只检查 seal 文件存在性而不验证内容，攻击面包括：(1) AI 可以 `echo '{}' > .dev-gate-spec.branch` 伪造通过；(2) stop-dev.sh fallback 路径降级到无 seal 验证的旧逻辑；(3) lock-utils.sh flock 不可用时静默返回 0 导致无锁并发；(4) PR 合并前无二次 mergeable 确认导致窗口期风险。
**不做什么**: 不改变 seal 文件格式本身，不影响正常流程中 verdict=PASS 的行为，不修改 CI 配置。

## 成功标准

1. [BEHAVIOR] 伪造的空 JSON seal 文件（`echo '{}' > .dev-gate-spec.branch`）无法通过 devloop-check.sh 的 seal 验证，被拦截为 blocked
2. [BEHAVIOR] stop-dev.sh 在 devloop-check.sh 未加载时直接 exit 2 拒绝执行，不降级到旧内联逻辑
3. [BEHAVIOR] lock-utils.sh 在 flock 命令不可用时返回 1（而非 0），调用方可检测失败
4. [BEHAVIOR] devloop-check.sh 在执行 `gh pr merge` 前额外检查 PR 的 mergeable 状态，CONFLICTING 时拦截
5. [GATE] CI 全部通过

**要改的文件**:
