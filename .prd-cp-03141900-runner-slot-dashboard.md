# PRD: 实时 Runner/Slot 面板 + 动态 Slot 推荐器

## 背景

当前 LiveMonitor 页面只显示简单的 slot 圆点状态，无法看到：
- 每个 slot 使用哪个账号、跑什么模型、执行什么任务
- Claude 账号的剩余配额与重置时间
- 消耗记录（任务维度的 account_id + duration_ms）
- 应该开几个 slot 才能把配额刚好用完

## 目标

Phase 1+2 一次性实现，让用户实时看到系统全貌并能动态优化 slot 配置。

## 功能需求

### 1. LiveMonitor 实时 Runner 面板（Phase 1）

**数据来源**：新 API `/api/brain/runner-status`

面板展示：
- 10 个 slot 分三区：user（2个）/ cecelia（2个）/ taskPool（6个）
- 每个活跃 slot 显示：
  - 账号名（account1/account2/account3 或 codex-slot-N）
  - 模型（claude-sonnet-4-6 等）
  - 任务名（截断 30 字符）
  - 运行时长
- 空闲 slot 显示灰色 "空闲"

### 2. Claude 账号配额条（Phase 1）

三个 Claude 账号各一条进度条：
- 5h 窗口：已用/上限 请求数
- 7d 窗口：已用百分比
- 下次重置倒计时

**数据来源**：`/api/brain/tick/status` 的 tokenPressure 字段（已有）

### 3. execution_callback 消耗记录（Phase 2）

在 Brain 的 `execution_callback` 处理完成时，写入新表 `task_execution_metrics`：
- task_id, account_id, duration_ms, est_requests（estimate: duration_ms / 30000）
- migration 编号：下一个可用编号

### 4. 动态 Slot 推荐器（Phase 2）

新 API `/api/brain/slot-recommendation` 返回：
```json
{
  "recommended_slots": 4,
  "formula": "floor(remaining_requests / remaining_hours / avg_requests_per_task)",
  "inputs": {
    "remaining_requests": 120,
    "remaining_hours": 4.5,
    "avg_requests_per_task": 6.5
  }
}
```

Dashboard 在配额条下方展示推荐 slot 数量。

## 成功标准

- `/api/brain/runner-status` 返回 10 个 slot 的实时状态（含 account/model/task）
- LiveMonitor 渲染三区 slot 面板（非空 slot 显示详情）
- 三个 Claude 账号配额条显示 5h/7d 窗口
- `task_execution_metrics` 表在 execution_callback 完成时写入记录
- `/api/brain/slot-recommendation` 返回推荐 slot 数
- Dashboard 展示推荐 slot 数

## 技术约束

- Brain 后端：Node.js + PostgreSQL（packages/brain/）
- Frontend：React + TypeScript（apps/dashboard/）
- 不改 tick 逻辑，不改 slot budget 算法
- migration 编号接续现有最高编号
