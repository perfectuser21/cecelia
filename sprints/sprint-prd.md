# Sprint PRD

## 产品目标

Brain 调度系统对 P1 任务的处理时效缺乏可见性：一旦 P1 任务进入 queued 状态后无 agent 领取，系统会静默等待，运维人员无从感知阻塞。本功能为 Brain tick loop 添加卡住检测能力，当 P1 任务 queued 超过 30 分钟仍未被处理时，自动在日志中记录告警，让运维人员能及时发现并介入。

## 功能清单

- [ ] Feature 1: P1 任务卡住检测 — tick 每次执行时检查所有 P1 queued 任务，若等待时长超过 30 分钟则记录 `[WARN][stuck-task]` 告警日志
- [ ] Feature 2: 告警日志格式标准化 — 日志含任务 ID、任务标题、已等待分钟数，便于后续检索和告警接入

## 验收标准（用户视角）

### Feature 1: P1 任务卡住检测
- 当一个 P1 任务处于 queued 状态超过 30 分钟时，运维人员能在 Brain 日志中看到告警条目
- 当 P1 任务在 30 分钟内被 agent 领取后，不产生告警日志
- 每次 tick 执行时均会检查，而不是只检查一次

### Feature 2: 告警日志格式
- 告警日志包含：任务 ID、任务标题、已等待时长（分钟）
- 日志格式可通过 `grep [WARN][stuck-task]` 快速筛选
- 同一任务在每次 tick 检查时均输出告警（持续告警，直到任务被处理）

## AI 集成点

不适用。本功能为纯 Node.js 定时检查逻辑，无需 AI 能力。

## 不在范围内

- 不发送通知（Slack/邮件/webhook），只记录日志
- 不自动重试或重派任务
- 不检测非 P1 优先级（P2/P3）任务
- 不修改任务状态
- 不引入新的数据库表或迁移
