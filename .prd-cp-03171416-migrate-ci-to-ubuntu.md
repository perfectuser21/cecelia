# PRD: migrate L3/L4 hk-vps CI jobs to ubuntu-latest

## 背景

HK VPS（8GB RAM）运行 8 个并发 GitHub Actions runner，每个 vitest 进程使用 20GB+ 虚拟内存，
导致 OOM 崩溃，机器失联，CI 全部卡死。

## 目标

将 L3 和 L4 中依赖 `hk-vps` self-hosted runner 的 jobs 迁移到 `ubuntu-latest`，
彻底消除 HK VPS OOM 风险，同时提升 CI 并行度。

## 成功标准

## 成功标准

- `ci-l3-code.yml` 无 hk-vps 引用（grep 返回 0）
- `ci-l4-runtime.yml` 无 hk-vps 引用（grep 返回 0）
- L4 brain-integration 仍在 xian-mac-m1 上运行
