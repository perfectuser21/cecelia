# Task Card: 修复 bash-guard 过激拦截 SKILL.md

## 需求（What & Why）

**功能描述**：移除 bash-guard.sh 中对所有 SKILL.md 路径的过激拦截。原有逻辑基于"symlink 指向 git 仓库"的假设，但现在 `~/.claude/skills/` 下的 skill 已全部改为独立真实目录（只剩 `dev` symlink 指向 engine），该拦截规则已过时。

**背景**：
- 原规则：bash 命令中凡含 `.claude/skills/*/SKILL.md` 路径字符串（包括读取），一律拦截并要求走 /dev
- 现状：26 个 skills 已从 symlink 转为独立目录，不再在 git 追踪范围内
- 遗留问题：bash-guard 仍拦截对这些独立 skills 的任何 bash 操作（含读取），导致无法正常使用

**不做什么**：不修改 branch-protect.sh，不改变对 `dev` skill（engine symlink）的保护。

## 成功标准

1. [ARTIFACT] `packages/engine/hooks/bash-guard.sh` 移除旧的 SKILL.md 全路径拦截逻辑（`SKILL_PATH_PATTERN` 相关代码块）
2. [BEHAVIOR] 对 `~/.claude/skills/canvas-project/` 等独立 skill 执行 `ls`、读取操作，bash-guard 不再拦截
   Test: `manual:node -e "require('fs').readdirSync(require('os').homedir()+'/.claude/skills/canvas-project')"`
3. [BEHAVIOR] 对 `~/.claude/skills/dev/SKILL.md`（engine symlink）的写入操作，branch-protect 仍阻止（bash-guard 不负责此检查，branch-protect 负责）
   Test: `manual:node -e "const r=require('child_process').execSync('bash /Users/administrator/.claude-account1/hooks/branch-protect.sh <<\\'EOF\\'\\n{\"tool_name\":\"Write\",\"tool_input\":{\"file_path\":\"/Users/administrator/.claude/skills/dev/SKILL.md\"}}\\nEOF',{env:{...process.env}});console.log(r.toString())" 2>/dev/null || true`
4. [GATE] CI 全部通过
