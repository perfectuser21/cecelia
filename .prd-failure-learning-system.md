## PRD - Cecelia 失败学习系统

**需求来源**: Cecelia 识别重复失败模式，进隔离区时自动分析根因，写入 learning，经反刍后形成策略调整，不再犯同类错误。

**功能描述**:
当任务失败进入隔离区时（retry_count >= 3），系统自动调用 LLM 分析失败根因，将分析结果写入 learnings 表，经反刍系统聚类分析后形成策略调整，写入 self-model，防止同类错误重复发生。

**涉及文件**:
- `packages/brain/src/routes.js` 或 `packages/brain/src/quarantine.js` - 隔离区逻辑修改
- `packages/brain/src/rumination.js` - 反刍系统提示词修改
- 相关测试文件 - 单元测试

**成功标准**:
1. 任务进隔离区时自动触发 LLM 分析，写入 category='quarantine_pattern' 的 learning
2. LLM 分析失败时降级跳过，不影响隔离主流程
3. 反刍系统能识别隔离失败记录并追加分析指令
4. 所有单元测试通过
5. 不影响现有隔离区逻辑（release、状态更新等）

**非目标**:
- 不修改 self-model.js（使用现有的 rumination → updateSelfModel 链路）
- 不改变隔离区的现有工作流程
- 不影响正常任务的执行流程
