## PRD - 实现 Brain tick 48h 简报定时检查逻辑

**需求来源**: Brain 自动调度任务
**功能描述**: 在 tick.js 中实现 48 小时简报定时检查功能，包括数据库表创建、API 端点、以及 cortex 简报生成逻辑（mock）
**涉及文件**:
- `packages/brain/src/tick.js` - 添加 check48hReport() 函数并在 tick 中调用
- `packages/brain/src/server.js` - 添加 /api/brain/reports 查询端点和手动触发端点
- `database/migrations/` - 创建 system_reports 表的迁移脚本

**成功标准**:
- [ ] tick.js 添加 check48hReport() 函数，每次 tick 检查是否距离上次简报超过 48h
- [ ] system_reports 表存在，包含 type/content/created_at 等字段
- [ ] GET /api/brain/reports 可查询历史简报
- [ ] POST /api/brain/reports/trigger 可手动触发简报生成
- [ ] 简报生成逻辑（mock cortex 调用，生成简单的系统状态摘要）

**非目标**:
- 不需要实现真实的 LLM 调用（用 mock 实现）
- 不需要前端 UI
- 不需要推送/通知功能

### 异常处理 & 报警规范（REQUIRED）

| 异常场景 | 处理方式 | 报警级别 | eventType |
|----------|----------|----------|-----------|
| 简报生成失败 | 记录错误日志，下次 tick 重试 | P2 | report_generation_failed |
| 数据库写入失败 | 记录错误日志，不阻塞 tick | P1 | report_db_write_failed |
| system_reports 表不存在 | 自动创建表 | P0 | report_table_missing |
