# Task Card: 新增 GET /goals/:id + GET /projects/:id API 端点

## 需求（What & Why）

**功能描述**: 在 `packages/brain/src/routes/task-goals.js` 和 `packages/brain/src/routes/task-projects.js` 更新 `GET /:id` 端点，使其返回 intent-expand 所需的精确字段。

**背景**: intent-expand skill 需要沿 `task.project_id → projects.kr_id → goals(KR).parent_id → goals(OKR).parent_id → goals(Vision)` 链路查询上下文。Brain 的 goals/:id 和 projects/:id 端点已存在但使用 SELECT *，需要改为指定字段并统一错误格式。

**不做什么**:
- 不新增数据库 migration（纯只读端点，不改 schema）
- 不 bump schema version
- 不做写操作（POST/PUT/DELETE）

## 成功标准

1. [ARTIFACT] `GET /api/brain/goals/:id` 返回 `{id, type, title, description, parent_id, project_id}` 字段
2. [ARTIFACT] `GET /api/brain/projects/:id` 返回 `{id, title, description, kr_id, goal_id}` 字段
3. [BEHAVIOR] 两个端点均有 404 处理（记录不存在时返回 `{"error": "goal not found"}` 或 `{"error": "project not found"}`）
4. [GATE] CI 全部通过
