# Task Card: 记忆系统 PR3 — 条件注入 WORLD_STATE + distilled docs token 预算

## 需求（What & Why）

**功能描述**:
buildMemoryContext 在 chat 模式下无论查询内容是什么都加载 WORLD_STATE（OKR/项目列表），
导致闲聊/情感类对话浪费 token，且 system prompt 没有总长度监控。

**背景**: PR2 已将对话写入 memory_stream。PR3 让 ContextBuilder 更智能：
仅在与 OKR/任务/项目相关的查询时注入 WORLD_STATE；同时对 distilled docs
的 USER_PROFILE + WORLD_STATE 设置 token 上限防止溢出。

**不做什么**:
- 不重写 buildUnifiedSystemPrompt 整体结构
- 不修改 SOUL/SELF_MODEL 加载逻辑（必注入）
- 不创建新的 context-builder.js 文件

## 成功标准

1. [ARTIFACT] memory-retriever.js 新增 `isWorldStateQuery(query)` 函数，识别 OKR/任务/项目相关查询
2. [BEHAVIOR] buildMemoryContext chat 模式：WORLD_STATE 仅在 isWorldStateQuery=true 时注入
3. [BEHAVIOR] distilled docs USER_PROFILE + WORLD_STATE 合计 token 上限 1200（超出截断）
4. [BEHAVIOR] block 总字符超 32000 时 console.warn 记录（约 8000 tokens）
5. [GATE] CI L1-L4 全部通过
