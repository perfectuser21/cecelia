# Task Card: 三阶段 seal 对齐检查 — Planner→Contract→Generator 强制链路

## 需求（What & Why）

**功能描述**: 在 devloop-check.sh 中增加三道 seal 前置检查，确保 Sprint Contract（step_2_code 开始前）和 Generator（Stage 3 开始前）无法在没有对应 seal 文件的情况下推进；同时在 Planner subagent 和 Generator subagent 的 prompt 中加入写 seal 文件的强制指令。

**背景**: 当前 devloop-check.sh 已有 spec_review seal（条件 1.5）和 code_review_gate seal（条件 2.5）防伪机制，但缺少以下三道检查：
1. Sprint Contract（step_2_code）开始前：必须有 .dev-gate-planner.{BRANCH}（Planner subagent 完成的证明）
2. Generator（step_2_code 写代码前）开始前：必须有 .dev-gate-spec.{BRANCH}（spec_review PASS 的证明）
3. Stage 3（push/PR 前）开始前：必须有 .dev-gate-generator.{BRANCH}（Generator subagent 完成的证明）

这三道检查确保 Planner→Sprint Contract→Generator 强制按顺序执行，缺任何一个就 exit 2 打回，不等 CI 就在本地拦住。

**不做什么**:
- 不修改 spec_review seal 和 code_review_gate seal 的现有逻辑
- 不修改 Brain 数据库或调度逻辑
- 不修改 CI 脚本

---

## 成功标准

> [ARTIFACT] = 产出物（文件/API/配置）
> [BEHAVIOR] = 运行时行为（系统如何响应输入）
> [GATE] = 门禁（CI/测试/代码质量）

1. [BEHAVIOR] 当 .dev-gate-planner.{BRANCH} 不存在时，devloop-check.sh 在 step_2_code 检查前 exit 2
2. [BEHAVIOR] 当 .dev-gate-spec.{BRANCH} 不存在（且 step_1_spec done）时，devloop-check.sh 在 step_2_code 检查前 exit 2（现有条件 1.5 已处理 spec seal 缺失，本任务在 step_2_code 检查之前插入额外的 planner seal 检查）
3. [BEHAVIOR] 当 .dev-gate-generator.{BRANCH} 不存在时，devloop-check.sh 在条件 3（PR 创建检查）前 exit 2
4. [ARTIFACT] planner-prompt.md 包含写 .dev-gate-planner.{BRANCH} 的指令
5. [ARTIFACT] 02-code.md Generator subagent prompt 模板包含写 .dev-gate-generator.{BRANCH} 的指令
6. [GATE] CI 全部通过
