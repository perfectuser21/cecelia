# Task Card: feat PR 测试覆盖强制检查

## 需求（What & Why）

**功能描述**: 在 `ci-l3-code.yml` 新增 `test-coverage-required` job，对 feat 类型 PR 强制要求包含测试文件改动。若无测试文件改动则 fail，有则 pass 并输出找到的测试文件。

**背景**: 当前 feat 类型 PR 只要老测试绿了就能通过 CI，没有验证新功能是否带测试。`detect-commit-type` job 已有 `should_run_l3` 输出（feat/feat! → true，fix/docs → false）。

**不做什么**: 不修改现有 CI 流程，不改变 fix/docs/chore 类型 PR 的行为，不引入新依赖。

## 成功标准

1. [ARTIFACT] `ci-l3-code.yml` 新增 `test-coverage-required` job，needs `detect-commit-type`，if `should_run_l3 == 'true'`
2. [BEHAVIOR] feat 类型 PR 无测试文件改动时，`test-coverage-required` job fail 并输出清晰错误信息
3. [BEHAVIOR] feat 类型 PR 有测试文件改动时，`test-coverage-required` job pass 并输出找到的文件列表
4. [ARTIFACT] Engine 版本从 12.89.0 bump 到 12.90.0（5个文件同步）
5. [ARTIFACT] `feature-registry.yml` 添加 12.90.0 changelog 条目
6. [GATE] CI 全部通过（engine-ci.yml 绿）

## 验收条件（DoD）

- [x] [ARTIFACT] `ci-l3-code.yml` 中存在 `test-coverage-required` job
  Test: manual:grep -c "test-coverage-required" .github/workflows/ci-l3-code.yml

- [x] [ARTIFACT] job 有正确的 `needs` 和 `if` 条件
  Test: manual:grep -A5 "test-coverage-required:" .github/workflows/ci-l3-code.yml | grep -c "detect-commit-type"

- [x] [BEHAVIOR] job 能识别测试文件（包含 .test.ts, .spec.ts, .test.js, .spec.js, tests/, __tests__/, *_test.py, test_*.sh）
  Test: manual:grep -c "\.test\.\|\.spec\.\|tests/\|__tests__/" .github/workflows/ci-l3-code.yml

- [x] [BEHAVIOR] 无测试文件时有明确 fail 消息
  Test: manual:grep -c "❌\|ERROR\|No test" .github/workflows/ci-l3-code.yml

- [x] [ARTIFACT] Engine 版本 5 个文件同步更新到 12.90.0
  Test: manual:grep -c "12.90.0" packages/engine/VERSION packages/engine/package.json packages/engine/.hook-core-version

- [x] [ARTIFACT] feature-registry.yml 包含 12.90.0 changelog 条目
  Test: manual:grep -c "12.90.0" packages/engine/features/feature-registry.yml

- [x] [GATE] CI 通过检查
  Test: manual:ls .github/workflows/ci-l3-code.yml

## 实现方案

**要改的文件**:
- `.github/workflows/ci-l3-code.yml` — 新增 `test-coverage-required` job
- `packages/engine/VERSION` — bump 12.89.0 → 12.90.0
- `packages/engine/.hook-core-version` — bump 12.89.0 → 12.90.0
- `packages/engine/package.json` — bump 12.89.0 → 12.90.0
- `packages/engine/package-lock.json` — bump 12.89.0 → 12.90.0（两处）
- `packages/engine/regression-contract.yaml` — bump 12.89.0 → 12.90.0
- `packages/engine/feature-registry.yml` — 添加 12.90.0 changelog 条目
- 运行 `bash packages/engine/scripts/generate-path-views.sh`

**Scope 锚定**: 只改 CI 配置和 Engine 版本文件，不改 Brain、前端、测试文件。
