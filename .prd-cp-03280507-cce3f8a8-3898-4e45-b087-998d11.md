# PRD: 修复 KR 进度可见性 — 接通 kr-verifier → OKR 显示链路

## 背景
KR 进度在 Brain 中全部显示为 0%，但系统实际已有真实进度：
- `kr-verifier` 已正确采集数据并写入 `key_results.progress`（管家闭环 = 30%）
- 但 `/api/brain/okr/current` API 读取 `current_value/target_value`（均为 0）
- 两套字段断连导致仪表板永远显示 0%

## 需求
1. 修复 `/api/brain/okr/current` 路由：优先使用 `progress` 字段（kr-verifier 写入），fallback 到 `current_value/target_value`
2. 修复 `decision.js` 中 `compareGoalProgress()` 的 SQL GROUP BY 错误，消除每次 tick 的 decision engine 报错

## 成功标准
- `/api/brain/okr/current` 返回管家闭环 KR 的 progress_pct > 0（真实值 30%）
- `compareGoalProgress()` 不再报 GROUP BY SQL 错误
- 现有测试通过
