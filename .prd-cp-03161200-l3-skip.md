# PRD: L3 CI 按 Commit 类型跳过不必要的检查

## 背景

当前每次 PR（无论 fix/feat）都跑全量 L3，L3 在 HK VPS 要 10-16 分钟，造成大量时间浪费。

## 目标

在 `ci-l3-code.yml` 中添加 `detect-commit-type` job，根据 PR 标题提取 commit type，让 L3 的各个子 job 在 `fix:` / `docs:` / `style:` / `refactor:` / `chore:` 类型时自动跳过。

## 成功标准

1. `ci-l3-code.yml` 包含 `detect-commit-type` job，输出 `commit_type` 和 `should_run_l3`
2. `fix:` 类型 PR title → `should_run_l3=false`
3. `feat!:` / `BREAKING:` 类型 → `should_run_l3=true`
4. `feat:` 类型 → `should_run_l3=true`（保守策略）
5. 无法识别 commit type → `should_run_l3=true`（保守策略）
6. `docs:` / `style:` / `refactor:` / `chore:` → `should_run_l3=false`
7. Engine 版本 bump 到 12.87.0
