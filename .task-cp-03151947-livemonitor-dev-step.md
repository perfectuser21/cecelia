# Task Card: Dashboard LiveMonitor /dev 实时步骤进度面板

## 需求（What & Why）

**功能描述**: 在 Dashboard LiveMonitor 左列新增 DEV STEPS 面板，读取 Brain tasks 表中 `custom_props.dev_step` 和 `custom_props.dev_step_name` 字段，实时展示当前活跃 `/dev` 任务的步骤进度。

**背景**: PR #964 已实现 `stop-dev.sh` 在每步完成后向 Brain PATCH `custom_props.dev_step`（数字）和 `custom_props.dev_step_name`（字符串），但 Dashboard 无法展示这些信息。用户需要在 LiveMonitor 中看到哪些 /dev 任务在跑、跑到哪一步了。

**不做什么**:
- 不修改 Brain API（只读取已有 `custom_props`）
- 不新增后端端点（复用现有 `GET /api/brain/tasks?status=in_progress&task_type=dev`）
- 不显示非 dev 类型的 in_progress 任务（只看 task_type=dev）

## 成功标准

> [ARTIFACT] 产出物 / [BEHAVIOR] 运行时行为 / [GATE] 门禁

1. [ARTIFACT] LiveMonitorPage.tsx 中新增 `DevStepPanel` 组件，在左列 RUNNER 面板之后渲染
2. [BEHAVIOR] 面板只显示 `task_type=dev` 且 `status=in_progress` 的任务
3. [BEHAVIOR] 每个任务显示：步骤编号（`custom_props.dev_step`）、步骤名称（`custom_props.dev_step_name`）、任务标题（截断 30 字符）
4. [BEHAVIOR] 若 `custom_props` 无 `dev_step`，显示"步骤未知"（灰色）
5. [BEHAVIOR] 数据与现有 `activeTasks`（已有 5s 轮询）同步，无需额外请求
6. [GATE] CI 全部通过
